# Codex 速用提示词

这份是短版。

适合两种场景：

1. 你懒得翻长手册，只想直接复制
2. 你现在最头疼的问题就是文太平、没情绪、读者没感觉

## 最常用 10 句

### 1. 长篇开书

```text
story-long-write，帮我开一本长篇。
平台：番茄
题材：都市脑洞
情绪：持续爽感+强期待感
对标：高钩子开局
要求：直接做题材定位、核心设定、前三卷卷纲、前30章细纲，并写入文件。
```

### 2. 长篇前 30 章细纲

```text
story-long-write，基于当前项目设定和卷纲，做前30章细纲。
每章都要有：核心事件、目标情绪、章首钩子、爽点、章尾钩子、伏笔。
直接写入大纲目录。
```

### 3. 写第 1 章

```text
story-long-write，写第1章。
按现有细纲、角色状态、伏笔表来写。
前3句必须是事件/对话/动作/信息炸弹，不要天气开头。
写完直接落文件。
```

### 4. 日更续写

```text
story-long-write，日更。
先读取上下文、最近正文、角色状态、伏笔、时间线，再直接续写下一章。
不要重做设定，不要空转。
```

### 5. 回炉某章

```text
story-long-write，回炉第8章。
问题：冲突起得慢，情绪不够炸。
不要改后续大方向，给我最小改动重写版本，并直接落文件。
```

### 6. 写盐言短篇

```text
story-short-write，帮我写一个盐言短篇。
题材：婚恋反转
情绪：意难平+最后一击反转
字数：1.2万
要求：直接做核心框架、小节大纲、正文、自检，并写入文件。
```

### 7. 只做短篇大纲

```text
story-short-write，基于当前设定只做小节大纲，不写正文。
每节写清：主事件、情绪、钩子、读者新获知的信息、伏笔/物件、目标字数。
反转放在后半段。
```

### 8. 审查当前章节

```text
story-review full，审查我当前这章。
重点找问题，不要夸。
优先检查：结构和钩子、角色动机和对话、节奏和情绪曲线、事实冲突和伏笔断线。
```

### 9. 去 AI 味

```text
story-deslop，处理当前章节。
只改表达，不改剧情和信息。
优先处理套话、书面腔、机械对话标签、直接说情绪。
改完给我修改后版本。
```

### 10. 查询项目状态

```text
story，当前这本书写到哪了？还有哪些伏笔没收？谁的状态刚发生了变化？
```

## “文很平”到底卡在哪

很多时候不是文笔差，而是这 5 个地方空了：

1. 没先定本章情绪目标，只是在交代信息
2. 角色有事件，没有代价
3. 有冲突，没有升级
4. 有想法，没有身体反应和动作
5. 结尾没有余波，没有钩子，没有没说出口的话

一句话说透：

**平，不是因为词不漂亮，是因为人物没疼、没急、没失去、没硬撑。**

所以只改文风通常不够，最好从 `细纲 → 场景 → 对话 → 正文` 四层一起动。

## 专治“文平”的提示词

### 1. 写之前先校准情绪

这句很实用，适合长篇和短篇。

```text
story-long-write，先别急着写第12章。
先告诉我：
1. 本章目标情绪是什么
2. 本章哪一段开始升温
3. 哪一段必须爆
4. 爆完后余波怎么留
5. 主角和配角各自最疼的一刀是什么
确认后再写正文，并把这条情绪曲线落实到场景里。
```

如果是短篇：

```text
story-short-write，先别写正文。
先把这篇的情绪曲线给我拉出来：
开头情绪、中段情绪、反转情绪、结尾情绪。
每一段都要说清楚：人物想要什么、怕失去什么、被戳中的点是什么。
确认后再写。
```

### 2. 不改文风，先改细纲

如果前面的细纲平，后面正文一定平。

```text
story-long-write，重查前10章细纲。
不要修文笔，专查情绪曲线。
逐章标出：
1. 目标情绪
2. 冲突级别
3. 代价
4. 钩子强度
5. 哪一章和前后相比最平
把连续两章以上情绪持平的地方重做。
```

短篇版：

```text
story-short-write，重查这个短篇的小节大纲。
找出哪些节只有信息，没有情绪推进。
把每节补上：欲望、阻碍、代价、误导、余波。
不要直接写正文，先把大纲救活。
```

### 3. 强制场景里出现“疼点”

```text
story-long-write，这一章太平了。
不要只润色，按“压抑→升级→爆发→余波”的曲线重写。
每个关键场景都必须写出：
1. 人物当下想得到什么
2. 他最怕失去什么
3. 他嘴上怎么装没事
4. 他身体的真实反应是什么
5. 这个场景结束后，局面比开始更糟还是更紧
按这个标准重写。
```

### 4. 专门处理“有剧情没感觉”

```text
story-long-write，这章现在是“有剧情，没感觉”。
请先指出：
1. 哪些段落只有信息推进，没有情绪推进
2. 哪些冲突来得太轻
3. 哪些角色反应不够具体
然后重写这些段落。
重写时少讲道理，多写动作、停顿、嘴硬、失控、沉默和反常。
```

### 5. 强化人物反应

这个非常有效。很多“平”就是因为角色反应像木头。

```text
story-long-write，重写这一场的人物反应。
要求：
1. 不要直接说“他很难过”“她很愤怒”
2. 改成身体反应、动作、视线、停顿、说一半咽回去的话
3. 每个主要人物至少给我一个有辨识度的反应
4. 不要所有人反应都一个样
```

### 6. 强化对话张力

很多“平”不是情节平，是对话太直。

```text
story-long-write，这段对话太直白了，像在交代剧情。
重写时加入：
1. 潜台词
2. 打断
3. 回避
4. 反问
5. 说一半停住
6. 用动作代替部分台词
不要让角色把心里话全说出来。
```

短篇也能直接用：

```text
story-short-write，这段对话太平。
我要的是拉扯感，不是解释会。
请按“想靠近→被刺痛→嘴硬回避→留一个更痛的尾巴”重写。
```

### 7. 强化感情拉扯

如果你写的是感情线，这句很好用。

```text
story-long-write，把这一场感情戏重写得更有拉扯。
不要甜言蜜语堆砌，要写：
1. 两个人各自想靠近什么
2. 他们各自不敢承认什么
3. 这一场是谁先试探
4. 谁先嘴硬
5. 谁先露破绽
6. 最后留下哪句没说出口的话
```

### 8. 强化爽点爆发

有些“平”是因为爆发来得太软。

```text
story-long-write，这个爽点释放得不够狠。
请重写这段，按这个顺序打：
1. 先压住主角
2. 再放大代价
3. 再揭信息差
4. 最后让主角狠狠干回来
要求爆发要干脆，不要边爆边解释。
```

### 9. 强化虐点和意难平

```text
story-short-write，这段虐点不够疼。
不要只是把事实说出来，要让我看到：
1. 人物明明想说却没说
2. 人物明明想留却只能走
3. 人物表面平静，身体已经绷不住
4. 结尾留一个回想起来会二次刺痛的物件或动作
按这个标准重写。
```

### 10. 让结尾留下感情

很多文平，是因为结尾收得像写总结。

```text
story-long-write，这章结尾太像总结陈词了。
重写结尾：
1. 不要升华
2. 不要解释主题
3. 用一个动作、一个物件、一个没说完的反应收尾
4. 让读者心里悬一下，或者疼一下
```

## 一个最好用的万能句

如果你懒得挑，先用这句：

```text
story-long-write，这章太平了。
不要只润色文风，先查情绪曲线和冲突升级。
把“只有信息没有感觉”的段落找出来，按“人物想要什么、怕失去什么、身体怎么反应、嘴上怎么硬撑、结尾留下什么余波”重写。
直接落文件。
```

短篇版：

```text
story-short-write，这篇太平了。
不要只改字句，先把情绪曲线拉开，再重写关键场景。
我要的是疼、憋、刺、翻，不要平推。
直接改到文件里。
```

## 额外提醒

提示词能解决很大一部分“平”，但有个边界：

如果你一开始就没把 `目标情绪 / 冲突代价 / 人物欲望 / 反转时机` 放进大纲，后面只靠润色，效果有限。

最稳的顺序永远是：

1. 先修细纲的情绪曲线
2. 再修场景里的动作和反应
3. 再修对话张力
4. 最后才是去 AI 味和文字润色
