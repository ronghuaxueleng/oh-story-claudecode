# Codex 提示词手册

这份手册只面向一种使用方式：

1. 你已经把 `oh-story-claudecode` 安装成 skill
2. 你在 `Codex` 里使用它
3. 你对 `Codex` 说自然语言，让它触发对应 skill

这不是给通用模型的 system prompt，也不是给别的宿主的配置文件。
下面所有内容，都是你可以直接复制到 `Codex` 里说的话。

---

## 使用原则

### 1. 最稳的写法：点名 skill + 说清任务

比起只说“帮我写小说”，更推荐这样说：

```text
story-long-write，帮我开一本番茄男频长篇。
```

```text
story-short-write，帮我写一个盐言短篇。
```

这样路由最稳，不容易跑偏。

### 2. 少讲抽象感受，多给可执行信息

尽量在一句话里带上这些信息：

- 平台：`番茄 / 起点 / 晋江 / 盐言 / 七猫`
- 类型：`都市 / 玄幻 / 悬疑 / 现言 / 古言 / 复仇 / 家庭`
- 目标情绪：`爽 / 拉扯 / 反转 / 意难平 / 悬疑 / 共鸣`
- 对标作品：`有就给，没有就算`
- 当前动作：`开书 / 建细纲 / 写第X章 / 回炉 / 审查 / 去AI味`

### 3. 让 Codex 直接落文件

如果你不想只看对话输出，要明确说：

```text
直接写入项目文件，不要只回在对话里。
```

### 4. 模糊请求能用，但精确请求更好

能说：

```text
帮我开书。
```

更推荐：

```text
story-long-write，帮我开一本番茄女频古言重生文，目标情绪是压抑后的强反杀，对标《嫡嫁千金》这种开局压迫感。直接先做题材定位、核心设定、前三十章细纲。
```

## 通用母版

下面这个母版，适合绝大多数场景。把花括号替换掉就能用。

```text
{skill 名}，我要做{任务}。
平台是{平台}，题材是{题材}，目标情绪是{情绪}，目标读者是{读者画像}。
对标作品是{对标作品}。
请直接按这个项目的标准流程执行，并把结果写入文件，不要只停留在对话里。
如果信息不够，你先补最关键的问题；如果已经够了，就直接开始。
```

## 长篇

### 1. 从零开书

```text
story-long-write，帮我开一本长篇网文。
平台是番茄，题材是都市脑洞，目标情绪是持续爽感+强期待感。
对标作品是《我不是戏神》《十日终焉》这种高钩子开局。
请按这个项目的方法直接做：
1. 题材定位
2. 核心设定
3. 主角设定
4. 世界观骨架
5. 核心冲突
6. 前三卷卷纲
7. 前30章细纲
直接写入项目文件，不要只给我聊天式建议。
```

### 2. 只有一个模糊想法，让它帮你定方向

```text
story-long-write，我想开一本长篇，但现在只有一个模糊感觉：
我想让读者觉得压抑、憋屈、然后狠狠干翻回来。
我个人优势是节奏感还可以，脑洞一般。
请你先帮我反查题材方向，再给我3个可写方案，每个方案都要包含卖点、目标平台、对标方向、风险点，最后推荐一个最稳的。
```

### 3. 只做核心设定，不急着写正文

```text
story-long-write，基于我这个想法先做核心设定，不要急着开写正文。
题材：玄幻升级
平台：番茄
目标情绪：一路压着读者往下看，爽点爆发要干脆
核心灵感：主角能提前看见别人死亡前的最后一段记忆
请输出并落文件：
1. 一句话梗概
2. 主角设定
3. 金手指规则
4. 世界观骨架
5. 主线冲突
6. 设定/关系.md
7. 设定/题材定位.md
```

### 4. 做前三十章细纲

```text
story-long-write，基于当前项目里的设定和卷纲，给我做前30章细纲。
要求：
1. 每章都要有核心事件、目标情绪、章首钩子、爽点、章尾钩子
2. 前3章必须完成人设基点、切入点基点、金手指基点
3. 不要平推，要有升级和信息差
4. 直接写入大纲目录
```

### 5. 写第 1 章 / 第 X 章

```text
story-long-write，写第1章。
按项目现有设定、细纲、角色状态、伏笔表来写。
要求：
1. 前3句必须是事件/对话/动作/信息炸弹
2. 不要天气开头，不要背景说明开头
3. 段落短，适合手机阅读
4. 本章必须有明确爽点或强钩子
5. 写完直接落到正文文件
```

```text
story-long-write，继续写第12章。
先读取上一章、本章细纲、追踪/伏笔.md、追踪/角色状态.md，再动笔。
这章目标情绪是“表面平静，底下快炸了”，结尾必须留一个强钩子。
写完更新角色状态和伏笔追踪。
```

### 6. 日更续写

```text
story-long-write，日更。
先读取当前项目的上下文、最近正文、角色状态、伏笔、时间线，告诉我你理解的当前位置，然后直接续写下一章。
保持现有风格，不要重做设定，不要空转。
```

### 7. 回炉 / 重写某章

```text
story-long-write，回炉第8章。
问题是这一章不够炸，冲突起得慢，角色反应也太平。
不要改后续大方向，先判断这章该保留什么、该重写什么，然后给我最小改动重写版本，并直接落文件。
```

### 8. 查询项目状态

```text
story，沈栀现在什么状态？
```

```text
story，当前这本书写到哪了？还有哪些伏笔没收？
```

```text
story，第一卷现在的主线冲突是什么？角色关系里谁和谁已经翻脸了？
```

## 短篇

### 1. 从零写一个盐言短篇

```text
story-short-write，帮我写一个盐言短篇。
题材是婚恋反转，目标情绪是意难平+最后一击反转。
字数目标1.2万字，第一人称，对标盐言高赞虐文。
请按项目流程直接做：
1. 核心框架
2. 核心反转
3. 情绪曲线
4. 小节大纲
5. 正文
6. 自检文件
直接写入文件，不要只在对话里展示。
```

### 2. 只有一个故事核

```text
story-short-write，我有个短篇故事核：
“女主一直以为丈夫不爱她，直到丈夫死后她才知道自己这些年一直活在他安排好的保护里。”
请你把它扩成一个适合盐言的短篇方案。
要给我：
1. 标题方向
2. 一句话梗概
3. 核心反转
4. 3条真铺垫
5. 1条误导线索
6. 小节大纲
```

### 3. 只做小节大纲

```text
story-short-write，基于当前设定只做小节大纲，不写正文。
要求每节一行，写清：
主事件、子事件、情绪、读者新获知的信息、钩子、伏笔/物件、动静、对话密度、目标字数。
反转必须放在后半段，不要太早亮底牌。
```

### 4. 直接写正文

```text
story-short-write，按现有小节大纲直接写正文。
要求：
1. 开头100字内给足有效信息
2. 每句话都要有功能
3. 默认第一人称
4. 不要解释主题，不要写成散文
5. 反转段要打满，不要轻轻揭开
写完后继续做自检，不要停在初稿。
```

### 5. 重打反转

```text
story-short-write，这个短篇的反转不够狠。
请你不要推翻整个故事，只重做反转设计：
1. 先指出原反转为什么弱
2. 给我3个更强的反转方案
3. 选1个最适合盐言传播的
4. 反推需要补哪些铺垫
5. 更新小节大纲
```

## 扫榜

### 1. 长篇扫榜

```text
story-long-scan，帮我扫番茄最近适合新作者切入的长篇方向。
我更想写男频，擅长节奏，不擅长特别复杂的世界观。
请给我：
1. 当前值得写的3个方向
2. 每个方向的读者情绪需求
3. 常见卖点
4. 竞争风险
5. 最适合我先开的一个方向
```

### 2. 短篇扫榜

```text
story-short-scan，帮我看短篇现在什么题材更容易出传播。
重点看盐言、番茄短篇这类方向。
我偏好婚恋、家庭、复仇。
请直接给我高频情绪、常见反转核、适合新手的入场方式。
```

## 拆文

### 1. 长篇拆文

```text
story-long-analyze，帮我拆《{书名}》。
重点不要只复述剧情，要提炼：
1. 开篇钩子
2. 爽点节奏
3. 角色功能位
4. 可借用的剧情模块
5. 黄金三章为什么成立
如果能深度拆，就直接按项目结构输出到拆文库。
```

### 2. 短篇拆文

```text
story-short-analyze，帮我拆这个短篇。
重点看：
1. 故事核
2. 情绪曲线
3. 反转设计
4. 铺垫和误导
5. 结尾为什么有传播感
不要泛泛而谈，要给我可复用的写法。
```

### 3. 精细拆解

```text
story-short-analyze，精细拆解这篇短篇。
我要的不只是总结，我要你把它拆成我之后能复用的小模块：
开头钩子、铺垫方式、误导方式、揭露方式、结尾余韵。
```

## 审查

### 1. 完整审查

```text
story-review full，审查我当前这章。
重点找问题，不要夸。
优先检查：
1. 结构和钩子
2. 角色动机和对话
3. 节奏和情绪曲线
4. 事实冲突和伏笔断线
按严重程度给我列问题。
```

### 2. 轻量审查

```text
story-review lean，审查最近两章。
我现在最担心的是剧情节奏和设定一致性，别把篇幅浪费在文风表扬上。
```

### 3. 单人快速审查

```text
story-review solo，快速审一下这一章，告诉我最该先改的3个问题。
```

## 去 AI 味

### 1. 对正文去 AI 味

```text
story-deslop，处理当前章节。
要求：
1. 只改表达，不改剧情和信息
2. 优先处理套话、书面腔、机械对话标签、直接说情绪
3. 改最少的字，拿到最大的去味效果
4. 不要整段乱删
输出修改后版本，并说明关键改动点
```

### 2. 对短篇整体去 AI 味

```text
story-deslop，给这篇短篇全文去AI味。
重点看开头、对话、结尾。
如果有明显的“模型腔”“解释腔”“总结升华腔”，直接下手改。
不要改剧情走向。
```

## 推荐工作流

### 长篇从 0 到 1

按这个顺序对 `Codex` 说：

```text
story-long-scan，帮我看番茄现在什么方向适合我。
```

```text
story-long-analyze，帮我拆《{对标书名}》。
```

```text
story-long-write，帮我开书，直接做题材定位、核心设定、前三卷卷纲、前30章细纲。
```

```text
story-long-write，写第1章。
```

```text
story-review full，审查第1章。
```

```text
story-deslop，处理第1章。
```

### 盐言短篇从 0 到 1

```text
story-short-scan，看看最近容易出传播的短篇方向。
```

```text
story-short-analyze，拆一篇你想模仿结构的短篇。
```

```text
story-short-write，帮我做核心框架、小节大纲、正文、自检。
```

```text
story-deslop，全文去AI味。
```

## 不推荐这样说

### 1. 太空

```text
帮我写个好看的小说。
```

问题：没有平台、题材、情绪、任务动作，Codex 只能先猜。

### 2. 太散

```text
我想写个女频，也可能男频，现代古代都行，最好有点悬疑也有点感情，还要能赚钱。
```

问题：信息太多但没有优先级。

### 3. 只要结果，不给约束

```text
写第10章。
```

如果项目状态清楚，这句话能用；如果项目状态不清楚，最好补一句：

```text
story-long-write，写第10章，按现有细纲和角色状态继续，不要另起炉灶。
```

## 最后给一个极简版

如果你懒得组织话，至少按这个格式说：

```text
{skill 名}，我要做{任务}。
平台：{平台}
题材：{题材}
情绪：{情绪}
对标：{对标}
要求：直接按项目流程执行，并写入文件。
```

例如：

```text
story-short-write，我要做一个盐言短篇。
平台：盐言
题材：婚恋反转
情绪：意难平
对标：高赞虐文
要求：直接按项目流程执行，并写入文件。
```
