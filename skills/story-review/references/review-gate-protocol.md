# 审查门禁协议

> 本文件把 `story-review` 的最低硬规则压成统一协议。执行任意 `full / lean / solo` 审查时，优先读取本文件；其他 references 负责补充方法，不再承担主门禁入口。

---

## 一、默认硬审范围

无论用户有没有追加检查项，默认都必须审：

1. 正文纯净度
2. 一致性
3. 题面兑现
4. 场面兑现
5. 章尾硬规则
6. 人物显影
7. AI味 / 解释腔 / 作者代判句
8. 追踪反校验
9. 连续章节留存
10. 并入裁决
11. 篇幅预算

规则：
- 用户追加项只能加码，不能替代默认硬审
- 命中门禁层，必须给失败码、并入裁决和施工动作
- 命中需中止推进的问题时，必须同步给 `建议接管态`：`H1-A / H1-B / 无`
- 若任务口径是纯测试 / 基准验证 / 回归验证，建议接管态只能是 `H1-A` 或 `无`
- 若任务口径是生产修文，且失败码不是 `F0`，建议接管态默认是 `H1-B`

补充功能场景反说明书审查：

- 若章节命中 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，必须额外审：
  - 规则信息是否压在在场动作里
  - 是否真的发生了 `试用 / 交易 / 排队 / 讨价 / 受阻 / 改规矩 / 利益变化`
  - 该段删掉后，场面推进会不会明显变弱

---

## 一点五、功能场景反说明书硬闸

以下情况默认至少 `F2`：

- 功能场景名义上有人在场，实际主要在讲制度、流程、福利、玩法、资格或规则介绍
- 连续 `4` 句及以上主要功能是在解释规则，而不是推动在场人物动作
- 该段没有明确的 `试用 / 交易 / 排队 / 讨价 / 受阻 / 改规矩 / 利益变化`
- 删掉该段后，人物关系和场面推进几乎不受影响
- 该段可以直接改写成产品说明 / 制度公告 / 服务介绍，而意思几乎不变

执行口径：

- 同时命中上面 `5` 条中的 `4` 条，默认按“功能场景写成规则说明书”处理，至少 `F2`
- 若只是单句解释偏重，但段落主功能仍是场景推进，可降到 `F1`
- 若连续 `2` 章以上都把功能场景写成说明书，或因此导致题面一直停在“知道规则更多”，默认升级为 `F3`

排误伤规则：

- 规则多，不等于说明书
- 若规则信息始终压在 `点单 / 试吃 / 结账 / 排队 / 抢位 / 争执 / 改口 / 改规矩 / 受阻 / 站位变化` 里，不得仅因信息密度高判死
- 若删掉规则信息会直接削弱当前交易、争抢、盘面变化，也不得误判为说明书

---

## 二、解释句密度硬闸

以下情况默认至少 `F2`：

- 开头 `500` 字内解释句 `>= 2`
- 任意连续 `300` 字内解释句 `>= 2`
- 单章累计解释句 `>= 3`
- 章尾最后 `1-2` 句承担意义总结 / 局势翻译 / 作者代判功能

解释句命中后，必须继续判根因：
- `G1 信息抢跑`
- `G2 现场承载不足`
- `G3 提纲语言漏出`
- `G4 人物失声`

不得只写“解释腔偏多”。

### 解释句黑名单

以下句式默认视为高危解释句，不得按普通解释腔轻放：

- `他只知道一件事`
- `他这才明白`
- `今夜若……他会死`
- `最后留下来的，是……`
- `情报一个字没错`
- `就是现在`
- `就这一瞬，够了`
- `但这份……撑不了多久`
- 任意“先下结论，再补动作”的短句

规则：
- 开头 `500` 字内命中黑名单 `>= 2`，默认至少 `F2`
- 单章其他位置命中黑名单，也不得只做词面替换
- 修法必须回到动作 / 身体反应 / 即时念头
- 若黑名单句式集中在开头或章尾，必须继续判 `G1/G2/G3/G4`

---

## 三、人物显影硬闸

以下情况默认至少 `F2`：

- 主角 / 核心反派 / 核心盟友 / 高价值角色首次强出场少于 `2` 项锚点
- 连续登场章节未复现 `1` 项稳定锚点
- 删除角色名后，主角组与核心配角台词仍明显混同
- 角色功能明确，但正文只给功能不给人脸、动作、说话区分
- 高价值角色只在设定里“好看 / 压人 / 值得看”，正文无可脑补细节

若角色卡已标记外形 / 动作 / 说话 / 气场锚点，正文却未连续兑现，默认至少 `F2`。

### 配角工具嘴失效边界

以下情况默认至少 `F1`：

- 某角色只在关键道具 / 关键信息 / 关键反转出现时补一句 `认出 / 确认 / 抬价 / 解释`，且前文无站位铺垫

以下情况默认上升为章节层问题：

- 某角色承担章尾翻面功能，却只靠一声失口或一句“怎么会是这个”顶住

---

## 四、高潮双段兑现硬闸

所有章节都必须拆成两段审查：

- `第一爆点`
- `高潮后续收益`

如果只有前者成立，而后者只剩：

- 对方变脸
- 黑幕更深
- 事情不简单
- 下一章再处理

默认至少 `F2`。

### 高潮后续收益认定

以下内容不得单独算作高潮后续收益：

- 黑幕更深
- 某物很重要
- 某人认得它
- 局势更复杂
- 事情不简单

合格的高潮后续收益至少命中以下之一：

- `命`
- `人`
- `名`
- `权`
- `物证`
- `站队`
- `规矩变化`
- `垫背者落地`

### 连续收益链豁免

以下情况，允许把“单一爆点触发的连续失血链”整体认定为合格的高潮后续收益，不得仅因其链条连续，就机械判成“只有一口收益”：

- 第一爆点落地后，连续引发 `控制权转移`
- 连续引发 `敌方失血`
- 连续引发 `我方抬位`
- 连续引发 `站位变化 / 队形变化 / 规矩变化`

适用前提：

- 每一拍都必须带来可见的现实变化
- 不得只是“认出来了 -> 继续解释 -> 继续确认”
- 至少命中上面现实变化中的 `2` 项

命中该豁免时：

- 审查必须先按“连续收益链”记录
- 不得仅因“不是两个明显分段的翻面块”就判 `F2`
- 若仍不过线，必须具体指出是“哪一拍没有造成现实变化”，不能笼统写“链条太完整”

### 后半章信息场失效边界

若单章后 `1/3` 主要在：

- 抬黑幕
- 抬线索价值
- 确认价值
- 变脸失声
- 解释局势

却没有新的现实翻面，默认至少 `F2`。

但若后 `1/3` 虽呈现连续处置链 / 连续翻面链，却同时满足以下任意 `2` 项以上：

- `文书 / 令牌 / 物证 / 兵器 / 车马` 的控制权发生转移
- 敌方被迫 `改动作 / 改站位 / 改判断 / 改口`
- 我方获得 `贴身控制 / 新站位 / 官面口径 / 明确保护`
- 现场其他人发生 `站队变化 / 递缰 / 横枪 / 改边 / 退缩`

则不得仅因“链条完整”或“处置连续”判为信息场；必须继续下钻判断：

- 是不是每一拍都带来现实变化
- 是不是只是同一结果被换说法重复报送
- 是不是读感问题主要来自句噪，而不是章节收益缺失

若此问题落在第 `1` 章章尾，默认从严处理。

### 首章双闸分离

首章必须分开裁决：

- `章尾是否合格`
- `高潮兑现是否过线`

规则：
- 章尾合格只代表“最后一拍停在了现场”，不代表整章已经过线
- 若首章满足以下任一情况，默认至少 `F2`：
  - 章尾合格，但高潮后续收益未成立
  - 章尾合格，但题面另一半未露头
  - 章尾合格，但外部盘面未改写
- 不得用“章尾已经有危险 / 发现 / 反扑”掩盖“整章只做到活下来 + 敌人来了 + 线索拿到了”
- 若首章章尾只是把刀送到门口、把物证送到眼前、把人送到脸前，而主角还没拿到任何现实收益，仍不得判 `F0`

## 五、事实层证据等级硬闸

### 1. 预案层

只允许写未来内容：

- `大纲/`
- `卷纲/`
- `细纲/`
- `追踪/总施工单_*.md`
- `追踪/执行计划_*.md`

### 2. 事实层

只允许写已落地内容：

- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报台账.md`
- `追踪/伏笔.md`
- `设定/角色/*.md` 中的“当前已落地事实”区块

硬规则：
- 审查追踪时，必须从正文抽事实，不得默认相信细纲或闸门
- 角色卡不得提前写未发生伤情、关系变化、身份变化
- 若事实层抄入预案层内容，直接按 `F5`

### 事实层措辞降级规则

事实层文件不得把“线索 / 强指向 / 高度疑似 / 死前吐词 / 临死失口”写满成“已确认 / 已主使 / 已执行 / 已坐实 / 第一批物证 / 活口口供”。

默认允许的表述：

- `获得指向 X 的线索`
- `高度疑似`
- `死前吐词 / 临死失口`
- `与 X 有关联迹象`
- `怀疑 X 方向仍有埋杀链`

默认禁止的表述：

- `确认 X 仍有埋杀局`
- `X 已派人灭口`
- `X 是灭口执行者`
- `第一批物证外露`（若正文仍只是线索）
- `活口口供`（若正文并无活口）

若事实层使用了禁止表述，而正文未完成同等级坐实，直接按 `F5`。

### 情报兑现时限硬闸

若核心卖点包含 `情报流 / 系统流 / 预知流 / 任务流`，必须同时满足：

- `追踪/情报台账.md` 写明 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果`
- 情报在当前章或后 `1` 章内转成现实收益
- 情报首次出现只用 `「」` 包住情报本体，而不是混在普通叙事里
- 情报首次出现时交代来源或触发机制，而不是凭空掉落

若连续出现“情报只负责提醒、解释、抬线索价值”，而没有兑现成现实战果，默认至少 `F3`。

若情报没有被 `「」` 包裹，或首次出现没有来源 / 触发机制，默认至少 `F2`。

补充：
- 不得把“下一章可直接反制 / 已经布好局 / 即将兑现”视为当前章的 `实际战果`
- 若 `追踪/情报台账.md` 缺失，而正文已出现第一条正式情报，默认至少 `F5`

## 六、反校验权限硬闸

`反校验_第XXX章.md` 必须至少检查：

1. 正文 vs 细纲
2. 正文 vs 写前闸门
3. 正文 vs 追踪主表
4. 正文 vs 角色卡

若存在以下任一情况，`反校验` 不得自报 `通过`：

- 细纲仍保留旧证据来源
- 写前闸门仍保留旧战果口径
- 追踪主表存在把线索写满成结论
- 角色卡“当前已落地事实”仍使用正文未坐实的判断

命中任一项时：

- 各分项可写 `待回修`
- `最终反校验结果` 只能写 `未通过`

禁止：

- 先写 `通过`，再等 review 打回来
- 把“尚可理解”当成“已对齐”

若任一组未通过，或旧口径未清而自报通过，默认至少 `F5`。

---

## 七、首章额外高压标准

第 `1` 章在通用协议上再加：

- 前 `300` 字内必须亮出主卖点死局
- 只活命不够，必须显出题面另一半的第一口势
- 后 `1/3` 不得主要靠线索收纳和意义总结撑住
- 开头 `500` 字内黑名单解释句不得超过 `1`
- 章尾前必须至少出现一口“敌方被迫改动作 / 改站位 / 改判断”的外部现实变化

若首章只做到：

- 主角活下来了
- 敌人来了
- 线索拿到了

但没有更硬的外部盘面改写，默认仍至少 `F2`。

---

## 八、章尾单独否决权

以下章尾默认不过线：

- `预告型`
- `总结型`
- `信息型`
- `平收型`
- `污染型`

合格章尾只认现场结果，不认口头承诺。

至少命中以下之一，才可判章尾过线：

- 新危险已经咬上来
- 新反扑已经启动
- 新人物 / 新敌意 / 新物证当场落地
- 某人失口、露底、翻面
- 某条规矩下一秒就要被打破
- 现场盘面已被推进到新的不可逆局势

`不是盖章句` 不等于 `章尾合格`。只要落点仍是预告、总结、平收、解释，照样判不过线。

若章尾最后 `1-2` 句主要功能是：

- 总结主角身份变化
- 总结本章胜负变化
- 总结局势已经变化
- 替读者解释“这一下意味着什么”

即使前面的动作、失态、压制、物证都成立，仍默认至少 `F2`。

只要单章章尾不过线，即使整章主体尚可，该章也默认 `不可并入主正文`。

## 九、失败码落点

- `F0`：过线
- `F1`：句子层失败
- `F2`：章节层失败
- `F3`：机制层失败
- `F4`：污染失败
- `F5`：流程 / 追踪失败

凡命中以下任一项，`F5` 不得留空：

- 事实层把线索口径写满成结论口径
- `反校验_第XXX章.md` 在旧口径未清时自报 `通过`
- 追踪主表 / 角色卡 / 时间线 / 闸门 / 细纲 保留双版本或旧证据来源
- 项目文件出现 `活口口供 / 已主使 / 已执行 / 第一批物证` 等超前口径，而正文未同级坐实
## 脚本结果消费硬规则

- 若项目存在 `scripts/scene_lint.py`，审查前默认优先运行 `scripts/scene_lint.py --json`
- 主线程必须显式区分三层：
  - `脚本 error`：可直接进入门禁层
  - `脚本 warn`：仅作预警，不自动进门禁层
  - `人工复看确认`：主线程结合正文证据后，才可升级为正式问题
- 默认裁决：
  - `summary.errors > 0`：不得忽略
  - 仅 `字数.warn`：不得单独判 `F2`
  - `章尾.warn / 解释.warn / 结构.warn / 情报.warn`：必须回看对应正文，再决定是否入单
- 不得只因脚本有 `warn` 就给章节判不过线
- 也不得只因脚本退出码为 `0`，就跳过章尾、解释腔、情报兑现等人工核查
