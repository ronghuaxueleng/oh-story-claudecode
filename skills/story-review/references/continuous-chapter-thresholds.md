# 连续章节审查阈值

用途：

- 统一承载 `story-review` 中与连续章节、连续章尾失败、连续回炉方式判断有关的默认数量阈值。
- 主 skill 只引用“达到连续章节升级阈值”，不在主规则区写死数字。

## 默认阈值

### 1. 章尾失败升级到留存层问题

- 默认最小触发阈值：`3` 章中有 `2` 章章尾不过线

### 2. 章尾持续平收触发章节组重组建议

- 默认最小触发阈值：连续 `3` 章及以上以预告 / 总结 / 平收章尾收束

### 3. 连续多章额外做回炉方式判断

- 默认最小触发阈值：`3` 章及以上

### 4. 连续章组追加前移收益检查

- 默认适用范围：`3` 章及以上同一章组

说明：

- 这些阈值是默认值，不是固定结构。
- 若项目在 `追踪/阶段审查计划.md` 中另行声明了连续章节审查阈值，优先按项目计划执行。
