## 四、门禁核验

### 1. 首章高压

- [ ] 非首章，跳过
- [ ] 已核验首章门禁
- 死因定位：
  - [ ] 开局入局不足
  - [ ] 黑名单解释句超阈值
  - [ ] 题面另一半未露头
  - [ ] 外部盘面未改写

### 2. 解释句 / 黑名单

- [ ] 开头 `500` 字解释句过线
- [ ] 开头 `500` 字黑名单过线
- [ ] 单章解释句密度过线
- [ ] 已下钻到 `G1/G2/G3/G4`
- [ ] 已单独判断“命中变少”是否只是把正文磨成更安全的验收稿

### 3. 章尾

- 章尾判型：
  - [ ] 合格
  - [ ] 预告型
  - [ ] 总结型
  - [ ] 信息型
  - [ ] 平收型
  - [ ] 污染型
- [ ] 已单独抽查最后 `5-15` 行
- [ ] 已判断是否命中“最后一句总结味”
- [ ] 已单独判断“章尾前一拍合格，但最后一拍是否继续砸出新结果”
- [ ] 若最后一句只是压话 / 止语 / 预告，已判不过线
- [ ] 若本章尝试用“新命令”收章，已继续追问是否出现“立即执行 + 即时可见后果”；若没有，仍不得按章尾过线放行

### 4. 人物显影

- [ ] 首次强出场锚点过线
- [ ] 连续登场稳定锚点过线
- [ ] 台词可区分
- [ ] 高价值角色可脑补
- [ ] 角色卡锚点已回落正文
- [ ] 首写未命中人物显影时，本轮已明确抓出具体缺口
- [ ] 已单独判断角色是否只剩 `压场 / 落字 / 传话 / 护口径 / 背书` 功能位
- [ ] 若只剩功能位，已写清缺的是 `私心 / 声线 / 习惯动作 / 风险代价` 哪一层
- [ ] 已单独判断“值钱句”是不是长在人物身上，而不是长在作者嘴上
- [ ] 若该句任意角色都能换着说，已判为热闹句，不算人物句
- [ ] 若该句是 `最值钱的不是……是…… / 先乱的不是……是…… / 真正值钱的是……` 这类作者判价句式，已先判为作者句；除非它是角色原话且直接造成现场改动作，否则不得算人物句
- [ ] 若主角已出现固定判断顺序、旧伤动作、冷口判断句，已继续追问这些是否足以坐实“私人质地”，而不只是“去工位化成立”
- [ ] 若主角句仍主要只服务于报后果 / 压对手 / 推改盘，且答不出更明确私人失去物，默认仍只判“脱离工位主角”，不得高判成稳定私人质地
- [ ] 若命中“非人话表达 / 生造口气句”，已明确写出这句到底更像 `角色口条失真 / 作者造句感过重 / 模板未本地化`
- [ ] 若某句问题是“不像人话”而不是“信息错误”，已在报告中显式标注 `非人话表达 / 生造口气句`，而不是只写“别扭 / 可优化”
- [ ] 若主角想高判到“稳定私人质地”，已继续追问他是否给“将坏未坏”的状态起出只属于自己的名字，且该名字是否长在本题材载体上
- [ ] 若高价值角色已命中稳定 `A3` 四连，已继续追问这些后果是不是长在该人物专属控制链上，而不是任意高位角色都能套
- [ ] 若 `后续稳定权下滑 / 自己人裂口` 只是可替换的标准件，没有和角色本人最护的一口绑定，默认记为“规则成立但写机械了”
- [ ] 若高价值角色已命中稳定 `A3` 四连，已继续追问她/他真正没丢掉的那一口是否已经露头；若仍只写到 `撤签 / 失权 / 裂口`，默认不得高判成更高层人物魅力

### 5. 高潮兑现

- 第一爆点：`成立 / 未成立`
- 高潮后续收益：`成立 / 未成立`
- 兑现类型：`单章兑现 / 章组兑现 / 情节库兑现`
- 若非单章兑现：
  - 兑现组范围：
  - 本章是第几拍：
  - 本章已拿到的中间收益：
  - 最迟完整兑现章：
- 收益类型：
  - [ ] 命
  - [ ] 人
  - [ ] 名
  - [ ] 权
  - [ ] 物证
  - [ ] 站队
  - [ ] 规矩变化
  - [ ] 垫背者落地
- 若为连续收益链：
  - [ ] 控制权转移
  - [ ] 敌方失血
  - [ ] 我方抬位
  - [ ] 站位 / 队形变化
- [ ] 后半章未滑成信息场
- [ ] 若纸面新口径已落，已核对是否只是 `名分落袋`
- [ ] 若高判到 `实利起效`，已核对是否存在“被动承认的一击”
- [ ] 若首写未命中高潮兑现，本轮已明确抓出具体缺口
- [ ] 已复核是否存在“模板写太快”：对手第二层反压缺失 / 第三方迟疑成本缺失

连续收益链判定规则：
- 若“控制权转移 / 敌方失血 / 我方抬位 / 站位变化”命中 `2` 项以上，不得仅因链条连续判 `F2`
- 若仍判不过线，必须写明是“哪一拍没有造成现实变化”

### 6. 情报兑现

- [ ] 非情报流，跳过
- [ ] 已入 `情报台账`
- [ ] 已限章兑现
- [ ] 情报本体格式合规
- [ ] 来源 / 触发机制已交代
- [ ] 未出现“只抬线索不兑现”
- [ ] 若命中广播 / 系统 / 轮次 / 倒计时 / 复核口令，已先判 `规则执行口` 还是 `正式情报文本`
- [ ] 若属于规则执行口，未因脚本误报而机械补假情报流

### 6.5 解释腔替代现场结果

- [ ] 已对当前审查范围做过一次从头到尾的整段复扫，不是只看重点句/章尾句
- [ ] 已检查关键收益位后，是否又补了作者代判句
- [ ] 若出现 `这就是现实起效 / 现在不是…… / 这就不是只……` 一类句子，已判是否可删
- [ ] 若出现 `这是第一口…… / 第二口外部反应也来了 / 这才是……的真值 / 这不是帮腔 / 这不是只下场 / 问，就是要写` 一类模板节点标签句，已判是否可删
- [ ] 若可删而不影响现场意义，已记为 `解释.error`
- [ ] 首写测试口径下，已明确区分“观察项”还是“主否决项”
- [ ] 若所谓“金句”本质是在给模板节点、收益节点、人物价值下定义，已判为作者盖章句
- [ ] 若所谓“金句”只是旁白替读者判“今晚最值钱的是什么”，已判为作者判价句，不得拿来充当人物锋味
- [ ] 若所谓“人物句”本轮走的是冷口判断句路线，已追问它是否同时挂住 `私利 / 代价 / 现实改盘`；若没有，仍按“句子好听但归属不稳”处理
- [ ] 若所谓“主角人物句”已挂住 `私利 / 代价 / 现实改盘`，仍继续追问“删掉角色名后，是否还能认出只有主角会这么说”；若不能，仍按“有效推进句 / 工位句”处理，不高判成稳定人物句
- [ ] 若所谓“主角人物句”想再上推一档，已继续追问它是否给“将坏未坏”的状态起出只属于主角的题材化名字；若没有，默认仍按“有效但未坐实不可替换”处理
- [ ] 若所谓“高价值角色人物句”只是站在高位替主角补公道、补规则、补裁决，而没有先压自己最想护的人 / 物 / 名分，仍按“高位改线器”处理
- [ ] 若已经写了“先压自己人 / 自己物 / 自己名分”，仍继续追问她/他自己是否当场失去现实位 / 顺位 / 面子 / 名分；若没有，仍不得高判为 `可脑补`
- [ ] 若正文想把高价值角色高判到 `A3`，已继续追问她/他是否立刻失去 `后续稳定权 / 复签权 / 背书口 / 执行口 / 自己人信任`；若没有，默认仍只可判到 `A2` 或 `A3 候选`
- [ ] 若正文想把高价值角色高判到 `A3`，已继续追问她/他自己边是否有人立刻裂口 / 改口 / 不再替她/他背书；若没有，默认仍只可判到 `A2` 或 `A3 候选`
- [ ] 若正文想把高价值角色继续高判到“更高层人物魅力”，已继续追问她/他真正没丢掉的那一口是否已被正文坐实为 `秩序 / 手感 / 压场核心 / 更深层权力核心`；若没有，默认仍只可判“规则成立”
- [ ] 若这条“先付代价再定场”已经成立，仍继续追问正文是否又补了作者代判句 / 作者收义句替它盖章；若删掉后规则仍成立，默认记为“规则成立但被作者补义削弱”
- [ ] 若正文出现 `我若先把自己人……谁还认…… / 我若先护自己人……谁还认…… / 我若先把自己人塞进……谁还认……` 一类直接翻义句，已单独判断：删掉后规则是否仍成立；若仍成立，默认不得高于 `A2`
- [ ] 若正文出现 `现在争的已经不是……而是…… / 他最怕的不是……是…… / 这章不能停在……` 一类旁白代判 / 章节工位句，已单独判断：删掉后现场链是否仍成立；若仍成立，默认记为“隐性补义”
- [ ] 若正文出现 `不是X，是Y` 一类变体直翻义句，已单独判断：删掉后现场链是否仍成立；若仍成立，默认按“模板写直 / 作者代判句”处理，不得高判到 `A3`
- [ ] 若直翻义句已清，但仍存在隐性补义，同时高价值角色本人没有更硬的可见自损，默认仍不得高于 `A2`
- [ ] 已把 `我看着他，只觉得…… / 她大概没想到…… / 他显然…… / 那时候我脑子里全是……` 这类句型作为一类批量排查，不是只改单句
- [ ] 已把 `喜欢 / 委屈 / 体面 / 难堪 / 自尊 / 不甘` 等抽象词对应的对象是否落地逐条复核；若读者需要自己补宾语，已单列问题
- [ ] 已复核是否存在 `前半写环境/状态，后半突然跳耳鸣/发闷/发麻/心里一下静了` 的感官逻辑断裂句；若有，已单列为 `感官逻辑断裂句 / 状态-感官硬焊句`
- [ ] 已复核听觉描写是否超出“可听见层”直接替人物判到身体内部体感；若有，已单列为 `听觉描写解释过满`
- [ ] 已复核是否存在 `进去了 / 起效了 / 听进去了 / 扎到了` 一类效果判词句；若有，已单列为 `效果判词句 / 起效代判句`
- [ ] 已复核是否存在“当前明明是第三方场景，却被正文说成对方场子/对方婚礼”的指代错位；若有，已单列为 `事实指向错位句 / 场景指代错误`
- [ ] 已复核是否存在“当前明明是在防伤害/防升级，却只用了碰/弄/来一下这类轻动词”的风险落轻句；若有，已单列为 `风险等级落轻句 / 动作结果写虚句`

### 6.6 假进步 / 过闸稿

- [ ] 已区分“问题减少”与“正文变好”不是一回事
- [ ] 已判断这次回修有没有把角色磨成更会解释、更统一的安全口条
- [ ] 已判断这次回修有没有把后果改写成总结句 / 关系判断句 / 成熟对话块
- [ ] 已判断这次回修有没有同时磨掉人物毛边、私心、嘴硬、遮掩

### 7.5 入口链路承载

- [ ] 若规则只在高层协议存在、入口模板未承载，已判为 `F5`
- [ ] 已区分“正文失败”还是“入口链路失败”
- [ ] 已检查写后验收 / 角色状态 / 章组总表是否把过程收益高判成结果收益
- [ ] 已检查写后验收是否把模板使用率/模板效果直接写成 `全写 / 全进正文 / 全命中 / 全是`
- [ ] 已检查方法论资料利用率是否被一口气写满为“全部已落地且有效”

- [ ] 若方法论资料利用率被全部写成“已落地且有效”，已继续核对是否至少保留 `待复审确认 / 仅读取未落地` 的保守口径；若没有，已单列为 `追踪口径写满风险`
- [ ] 若追踪口径写满，已单列为 `F5`，未与正文质量问题混写
