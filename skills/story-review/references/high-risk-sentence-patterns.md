# 高危句型黑名单

用途：把“解释腔 / 作者代判 / 人物口条失真”收束成可执行的具体句型，避免审查时只会泛泛说“有点 AI 味”。  
适用阶段：`story-review lean/full/solo`。  
执行原则：命中后必须判它到底伤在 `人物`、`解释`、`章尾`、还是 `作者盖章`，不得只记“语言有点问题”。

## 一类：作者盖章型

- `不是……是……`
- `真正……的是……`
- `最值钱的是……`
- `最要命的是……`
- `因为这……`
- `这意味着…… / 这说明…… / 这证明…… / 这就说明……`

默认判法：
- 若压在章尾、第二口收益、外部反应位，优先判 `解释腔 / 作者盖章`
- 若删掉后事件仍成立，至少 `F1`

## 二类：两拍判词句

- `不怕……怕的是……`
- `不是来……的，是来……的`
- `这不是狠，是老`
- `这不是认输，是最后一口反压`

默认判法：
- 若句子主要魅力来自“判词好记”，而不是现场动作成立，判 `作者代判句`
- 若说这句话的角色本不该这么会说，同时判 `角色口条失真`

## 三类：排比抒情台词

- `带着什么去？带着……还是带着……`
- `因为穷，因为断供，因为没人……`
- 连续两拍以上整齐对称句
- 岗位/对象/苦处列表式台词

默认判法：
- 关键配角命中时，优先核角色卡；若违背角色卡，至少 `F1`
- 首章命中时，不得只降为观察项

## 四类：旁白替人物补义

- `他比谁都清楚……`
- `因为一旦……就……`
- `谁都知道……`
- `这话比什么都硬`
- `围观不是刀，可围观会把刀磨快`

默认判法：
- 优先判 `作者替人物补义`
- 若集中出现在首章前半或高潮后半拍，必须额外检查是否已经吃掉人物反应和第三方现实动作

## 五类：非人话表达 / 生造口气句

- `轮不到你往里垫`
- `我先把这句话塞进去`
- `你少往这事里挂口`
- `他把那点体面往回兜`
- 其它“字面能懂，但不像真人会这么说”的临时拼装口语

默认判法：
- 优先判 `角色口条失真` 或 `作者造句感过重`
- 若该句换成 `插嘴 / 掺和 / 接话 / 打岔 / 顶话 / 垫话` 等常见口语后，功能完全不变，说明原句的主要问题不是信息，而是“不像人话”
- 若项目已有口气模板、对标原文嘴型或声纹卡，正文仍出现此类句子，优先判 `口气模板未本地化`
- 报告里不得只写“有点别扭”，必须明确标为 `非人话表达 / 生造口气句`

## 六类：人物反应代判句 / 作者替人物想

- `她大概没想到我会这么平`
- `他显然没料到我会这么做`
- `他这才意识到事情不对`
- `她终于明白自己错在哪`
- `我看着他，只觉得空`

默认判法：
- 优先判 `人物反应代判句` 或 `作者替人物想`
- 若删掉后现场动作、表情、站位、沉默仍足够成立，至少记 `F1`
- 若删掉后整段直接发空，说明问题不只是句子，而是 `人物反应承载物缺失`
- 修法优先回到可见反应：`眼神停住 / 手顿住 / 没接上话 / 笑僵住 / 脸色变 / 站位退`
- 报告里不得只写“心理描写略多”，必须明确指出它是在“替谁想”

## 七类：抽象词悬空 / 对象省略过度

- `那时候我脑子里全是喜欢`
- `她心里只剩委屈`
- `他最在意的就是体面`
- `我忽然觉得难堪`

默认判法：
- 优先判 `抽象词悬空` 或 `对象省略过度`
- 若读者必须自己补出“喜欢谁 / 委屈什么 / 体面给谁看 / 难堪因为什么”，至少记 `F1`
- 修法优先补回对象、来源、落点，不要只换个更文艺的抽象词
- 报告里不得只写“有点虚 / 有点飘”，必须写清缺的是哪一层对象信息

## 八类：感官逻辑断裂句 / 状态-感官硬焊句

- `门一关，客厅里一下静下来，静得我耳朵都嗡`
- `我看着屏幕，心里一下静了`

默认判法：
- 优先判 `感官逻辑断裂句` 或 `状态-感官硬焊句`
- 若前半写的是环境/现场状态，后半突然跳成另一条身体感受，且中间没有自然过桥，至少记 `F1`
- 修法优先：
  - 保留单一状态
  - 拆成两拍补过桥
  - 把后半改成前半自然能推出的同感官反应
- 报告里不得只写“感觉怪怪的”，必须写清前后两半分别在写什么

## 九类：听觉描写解释过满

- `那头先是一阵喘，不是哭，是气急了又压着，憋得胸口发闷那种喘`

默认判法：
- 优先判 `听觉描写解释过满`
- 若文本只是“听见”，却直接替人物补到身体内部体感或情绪细因，至少记 `F1`
- 修法优先回到可听见层：喘、断句、卡壳、呼吸乱、接不上话
- 报告里不得只写“有点满”，必须写清哪一层属于作者多判

## 十类：效果判词句 / 起效代判句

- `我知道，我这句话进去了`
- `这句一出去，起效了`
- `我知道，他听进去了`

默认判法：
- 优先判 `效果判词句` 或 `起效代判句`
- 若句子主要功能是替读者总结“话已经起效”，而不是呈现现场反应，至少记 `F1`
- 修法优先改成可见反应：沉默、停顿、没接上话、呼吸乱、变脸、退半步
- 报告里不得只写“有点像总结”，必须写清它在替哪一层效果盖章

## 十一类：事实指向错位句 / 场景指代错误

- 当前是朋友婚礼，却写成 `不然我来喝你喜酒？`
- 当前是第三方场子，却写成“你的场子”

默认判法：
- 优先判 `事实指向错位句` 或 `场景指代错误`
- 若句子功能成立，但事实归属、场景归属、代词指向错了，至少记 `F1`
- 修法优先先改正事实指向，再保住原句的反问/嘲讽/顶嘴功能
- 报告里不得只写“逻辑不顺”，必须写清到底是 `谁的场景被说成谁的场景`

## 十二类：风险等级落轻句 / 动作结果写虚句

- 当前要拦的是“伤人”，却写成 `别让他碰人`
- 当前要防的是“动手升级”，却只写成 `别让他乱来`

默认判法：
- 优先判 `风险等级落轻句` 或 `动作结果写虚句`
- 若句子承担的是止损/制止/防伤害功能，却把真实危险写轻、写虚，至少记 `F1`
- 修法优先先判真实风险，再把后果写实：`伤到人 / 动手 / 砸到人 / 波及别人`
- 报告里不得只写“这句不够狠 / 不够准”，必须写清它把哪一层风险落轻了

## 首章强制结论

若第 1 章命中本卡任意句型：
- 必须写 `首章语言是否过线`
- 必须在 `发现的问题` 中单列句子，不得吞成观察项
- 若仍判可并入，也必须写明这是 `F1 语言 finding`

## 修法指向

审查报告不能只写“删掉更自然”。必须指出替代承载物：
- 改成动作
- 改成物证
- 改成他人失态
- 改成现实后果
- 改成角色更符合设定的短句
- 若问题属于 `非人话表达 / 生造口气句`，还必须补一句：`建议回退到该角色平时会说的常用口语层，不要为了新鲜感硬造搭配`
