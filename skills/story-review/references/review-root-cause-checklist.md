# review-root-cause-checklist

用于 `story-review full agents` 或主线程复盘时，专门把“问题是什么”继续下钻到“为什么会出现这个问题”，避免只开正文施工单、不追流程根因。

## 使用时机

- 用户明确要“找原因”“不要改正文”
- 同类问题连续复发两本以上
- skill 明明写了规则，但冷启动正文仍重复出现相同失败

## 根因拆解顺序

1. 先判 `skill 无规则` 还是 `已有规则未执行`
- 若相关硬闸在 `story-long-write / story-review` 中根本不存在，归为 `skill 缺口`
- 若规则已存在，但正文、写后验收、章组总表仍直接越过，归为 `执行漏步`

2. 再判是 `正文问题` 还是 `追踪高判`
- 正文只做到 `资格 / 名分落袋 / 现场暂控`，追踪却写成 `实利起效 / 实利落袋 / 稳定掌控 / 开始掌某口分配`
- 章组总表明写收益偏低，却仍放行 `可并入主正文`
- 这类一律归为 `口径高判`

3. 再判是 `模板未落地` 还是 `模板已落地但过程写窄`
- 已有第一爆点、第二口收益、章组末次兑现、外部现实变化：不得再写成“模板未用”
- 若中段主要靠查证、表态、挂名、等高位到场推进，归为 `过程写窄`

4. 再判人物为何变薄
- 角色卡缺失：`卡缺`
- 有卡但正文只承担认账/背书/记门/见证：`接口功能化`
- 有人物站位但缺代价、软肋、反作用：`人物肉量不足`

5. 最后判语言问题是因还是果
- 若删掉盖章句后，事件仍成立，语言问题多半是结果层
- 若删掉盖章句后，段落直接空掉，说明正文本体承载不足，语言问题只是暴露机制问题

## 固定输出字段

- `真问题`
- `根因类型`：`skill 缺口 / 执行漏步 / 口径高判 / 正文骨架问题 / 人物接口掉线 / 语言外溢`
- `证据文件`
- `为什么不是其他原因`
- `是否需要补 skill`
- `是否需要改执行顺序`

## 当前高频根因提示

- `查证链过长，现实后果链不足`
- `章组总表低档收益却提前放行`
- `模板已落地但过程写窄`
- `角色在关键章退化成功能接口`
- `作者代判句回弹来自动作链不足，而非单纯文风差`
