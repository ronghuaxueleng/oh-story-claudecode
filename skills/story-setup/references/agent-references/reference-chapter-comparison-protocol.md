# 参考章节对比协议

适用场景：
- 用户要求“用原文参考再写一版”
- 用户要求“和现稿对比一下”
- 用户要求判断“差在哪”

## 前置输入

执行前必须先读：
1. `参考书使用边界.md` 或同类边界说明
2. 本章参考输入卡
3. 本章单章写前卡
4. 本章现稿
5. 对应参考原文章节

## 禁止事项

- 禁覆盖现稿
- 禁把参考原文章节顺序直接平移成本章正文
- 禁只给“像不像”的空泛评价

## 操作流程

1. 先确认本轮可借的是发动机层、表演层，不可借的是设定结论、关键事件顺序、角色命运落点。
2. 读取本章写前卡，锁定本章必须完成的事件责任、人物责任、章尾责任。
3. 读取参考原文，只抽可迁移的表演层资产：
   - 压场节拍
   - 群像抬轿
   - 信息抖落顺序
   - 句式冲击方式
   - 章尾吊法
4. 单独写一份“参考驱动版”，不得覆盖现稿。
5. 输出“现稿 vs 参考驱动版”结构化对比单。
6. 结论只允许是：
   - `局部吸收`
   - `局部重写`
   - `整章回炉`

## 对比维度

- 情节层：是否完成本章任务
- 压场层：是否有递进、有见证、有落点
- 群像层：配角是在报信息，还是在给主角抬轿
- 语言层：句子是否有劲道、是否缺少现场感动作词
- 章尾层：是收平，还是留钩

## 输出结论

必须分三段写：
- 现稿更稳的点
- 参考驱动版更强的点
- 处理建议：保留什么、吸收什么、不建议覆盖什么
