# 高敏同桥仿写与外部分块审计回修实战手册

这份手册只管高敏同桥仿写与高危回修。

不管正常自由创作。

这份文件的定位是 `执行层手册`，不是共享判断正文。

共享判断、共享自检、同桥不过检判断、作者站位判断，统一维护在：

- `../../../story/references/short-high-risk/reference-index.md`
- `../../../story/references/high-risk-rewrite-governance.md`

这里不再重复维护那批共享正文，只保留：

- 写作侧怎么选母稿
- 怎么保护母稿活气
- 怎么做块级最小手术
- 怎么挂失败码与回修验收

适用场景：

- `同桥仿写`
- `原情节实验`
- `对标重写`
- `外部分块审计长期卡高`
- `已经改很多轮，开始越改越像施工稿`

## 一、先记死这 4 条总原则

共享高敏判断先读：

- `../../../story/references/short-high-risk/reference-index.md`

这 4 条只补写作侧执行动作，不替代共享判断层。

### 1. 规则只用于否决，不用于生成

规则可以用来：

- 找雷
- 否决
- 验收

规则不能用来：

- 直接生产全文
- 发明最安全讲法
- 替代正文的自然受力

### 2. 母稿优先于安全稿

母稿不是最低分那版。

母稿是更像人的那版。

判断母稿至少看 4 条：

1. 人物更像人在反应
2. 信息更像慢慢漏出来
3. 段落更像在活，不像在交付任务
4. 就算有雷，也雷在桥段，不雷在整篇语气

### 3. 一轮只拆一个活结

不要一轮同时追：

- 去 AI 味
- 降检
- 更像原文
- 更顺结构
- 更文

一轮只准拆一个主炸点。

### 4. 外部检测报告优先按“块”看，不按你的章节名看

外部分块审计给出的疑似段，不是在说哪句话不自然。

它常常是在说：

- 这一块太像完整成品
- 这一小窗太会交代
- 这一条推进链太顺

## 二、命中高敏同桥时的强制流程

进入这一节前，默认已经完成两件事：

1. 读过共享高敏判断入口
2. 知道当前问题是块病、桥病，还是句病

### 第 0 步：先分任务类型

先判这次属于哪一种：

1. `自由创作`
2. `同桥仿写`
3. `局部高危回修`
4. `整篇已经被规则接管`

只有 2、3、4 才加载这份手册。

### 第 1 步：如果已经有多版稿，先选母稿

不要从最新稿直接继续补丁。

先判断哪一版更像母稿。

### 第 2 步：改稿前必须写母稿保护卡

至少写 3 条：

1. 这段最值钱的活气是什么
2. 这段最不能动的气口是什么
3. 这段最容易被改死的地方是什么

写不出来，不准改正文。

### 第 3 步：先挂 4 条最低限制，再写活稿

写活稿时只挂：

1. 不要总结句
2. 不要高功能对白
3. 不要替人物解释心理
4. 不要整齐交代结果

除此之外，不要一边写一边满脑子审计。

### 第 4 步：写完后先做命名式滑窗审计

不要先看句子。

先问：

1. 这 600-1200 字能不能被一句话命名
2. 这一窗是不是完整事件包
3. 它是不是：
   - 线索 -> 处理 -> 见效/翻车
   - 受伤/离婚/翻车 -> 恢复 -> 再相遇 -> 切断
   - 回忆 -> 主题总结 -> 自我宣言

能被一口命名，就先判危险。

### 第 5 步：只做最小手术

只允许：

1. 改一个活结
2. 最多动相邻两三句
3. 不为一个炸点重写整章

禁止：

1. 整段回炉
2. 为了安全补一串缓冲物
3. 把人物都写成更克制、更含糊的一张脸
4. 把后果写成模糊摘要

## 三、这类实战里最关键的 10 个发现

这一节只保留“写作侧怎么施工”的发现。

如果是：

- 为什么原版同桥不过检
- 什么叫作者站位过高
- 什么叫短篇高敏标准承载方式

回共享索引，不在这里各写一套平行版本。

### 1. 证据前优先隔“现实后果”，不要优先隔“时间空档”

高风险：

- 办完手续
- 过几天
- 证据来了

正确方向：

- 离婚当天继续挑衅
- 公开场继续偏向对方
- 请帖/公开刺激物先到
- 然后证据晚一点、歪一点闯进来

### 2. 证据章内部也不能太标准

高风险内部链：

- 看见
- 认出来
- 卡住
- 拖两天
- 再处理

如果内部处理链太齐，照样会炸。

### 3. 婚礼/公开爆炸章不要做完整播报

优先抓：

- 谁先站起来
- 谁先没动
- 谁被人把东西塞到眼前
- 谁脸色先白

不要把整场事件播成说明书。

### 4. 后果要“听来补来”，不要“总结清算”

优先写：

- 今天听一嘴
- 明天补一嘴
- 版本不一样

不要写成：

- 后果一
- 后果二
- 后果三

### 5. 尾声入口必须让给真正核心

高风险：

- 先和前任完整偶遇
- 完成一场切断戏
- 再去墓园/旧人/真正核心

正确方向：

- 前任线在余波区就结束
- 真尾声前不再给她完整对手戏
- 最终入口直接给真正核心

### 6. 尾声不要写标准主题句和宣言句

高风险：

- 这世上……
- 我终于……
- 以后我会……

正确方向：

- 停在物件
- 停在身体感
- 停在动作
- 停在没说透的闷住

### 7. 同桥能不能过，不只看事件，要看“受力方式”

原文普通事件不炸，常常因为：

- 伤人的是具体小事实
- 偏心有先后顺序
- 难堪来自公开秩序
- 情绪里带不体面的羡慕、狼狈、丢脸
- 人没有在替作者总结

### 8. 人物必须“不同脸”

写前先给主要人物各写 4 件事：

1. 他说话怎么伤人
2. 他身体怎么反应
3. 他遇事先偏向谁
4. 他失控时会干什么

写完后遮掉名字测试。

换个名字还能成立，说明这句不属于任何人。

### 9. 写作手法不能只做表面变化

多空几行、插两句回忆、换成倒叙开头，

如果主任务闭环没拆，

检测器还是会吞成完整成品块。

### 10. 高敏节点一定要硬断场

默认优先断章的位置：

- 偏心刚坐实
- 物件刚丢
- 证据刚到手
- 婚礼/宴会/公司冲突刚炸
- 手术日刚被点到
- 尾声对象刚出现

## 四、命中高敏同桥时的固定失败码

### 失败码 A：证据触发顺序失败

判定条件：

- 证据紧贴上一事件自然弹出
- 中间只隔时间，不隔现实后果
- 证据内部处理链过于整齐

### 失败码 B：尾声入口归属失败

判定条件：

- 真尾声前还给了前任/次要关系一整场
- 尾声块里有完整主题总结或自我宣言

### 失败码 C：规则接管写作权

判定信号：

1. 每个炸点都被加了差不多的缓冲物
2. 人物越来越不直接说话
3. 后果越来越像作者故意糊成摘要
4. 稿子越来越安全，但越来越不疼

中 2 条以上，就别沿这版继续补。

### 失败码 D：假拆法失败

判定条件：

- 看起来换了手法
- 实际主链一点没断
- 检测器滑窗还能一句话命名

### 失败码 E：同脸失败

判定条件：

- 去掉名字后分不出谁是谁
- 多个角色都在用同一套“停一下、低声、没说完”

## 五、回修时的固定自检顺序

不要一上来先润词。

按这个顺序查：

1. `外部分块 / 滑窗块`：这块能不能被命名成完整成品
2. `主任务`：这一块是不是同时做了太多事
3. `顺序`：哪一步太像标准平台桥
4. `入口`：证据/尾声入口是不是太明白
5. `人物`：是不是又写成同脸
6. `句壳`：最后才查总结句、解释句、功能对白

---

## 六、哪些东西能学，哪些不能学

### 可以优先学

1. 桥段承重件
2. 外部秩序如何压人
3. 偏心先后顺序
4. 具体伤人物件
5. 后果回灌方式
6. 不体面情绪
7. 不同角色的反应语法

### 不能直接学

1. 高敏桥的标准平台承载顺序
2. 原文偶尔过检时的表面桥壳
3. 容易长成成熟尾包的收口句
4. 作者替人物讲明白的判断句

---

## 七、最重要的一句执行口令

以后命中高敏同桥，就先执行这一句：

`先保母稿的笨、拖、空、断、没说透；规则只用于否决，不用于生成；一次只拆一个活结；外部分块怎么切，就按那个块修。`

---

## 八、固定自检清单

每轮高敏回修后，必须逐条复检下面 14 项。

### 1. 母稿保护

1. 这版还像母稿吗
2. 有没有比母稿更工整
3. 有没有比母稿更会交代

### 2. 证据触发

4. 证据前隔开的，是现实后果还是时间空档
5. 证据入口是不是还能被读成标准按钮
6. 证据章内部是不是还能被读成：认出来 -> 卡住 -> 拖着 -> 再处理

### 3. 后果回灌

7. 后果是不是零碎听来，而不是完整清算
8. 有没有把外界余波整理成条目或说明书

### 4. 尾声入口与收口

9. 尾声入口是不是被次要关系抢走了
10. 尾声最后有没有主题句
11. 尾声最后有没有自我宣言句
12. 收口是不是停在物件、身体感、动作，而不是停在总结

### 5. 人物与块病

13. 遮掉名字后，主要人物还能不能分出来
14. 当前高风险窗能不能被一句话命名成完整成品包

只要 4、5、6、9、10、11、14 任一项答案偏“是”，当前段不算通过。
