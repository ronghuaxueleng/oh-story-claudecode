# Story Format Rules

正文文件的本地补充规范。当用户直接编辑正文文件时自动加载。

## 优先级

- 项目根 `CLAUDE.md`、`写作执行铁律.md`、当前会话实际调用的 `story-long-write` / `story-review` skill，高于本文件。
- 本文件只补充正文层的排版与表达注意项，不得替代写前闸门、短执行核、脚本门禁、写后验收。
- 若本文件与项目根铁律冲突，以项目根铁律为准。

## 绝对禁止

1. **禁止把排版规则当作剧情质量替代品**：段落切分服务于动作、情绪和信息推进，不得只靠切短句冒充节奏。
2. **禁止段落间空行**：正文段落之间不加空行，保持紧密节奏。
3. **禁止「他说」「她道」**：对话用动作代替对话标签。
4. **禁止大段描写堆砌**：描写必须穿插动作或对话，不连续超过 3 段纯描写。
5. **禁止跳出视角**：已确定视角（第一人称/限知第三人称）后，不切换到其他角色内心。

## 格式规范

- 对话独立成行，用冒号或动作引出
- 段落以动作、受力、情绪或信息改动为单位划分
- 章节标题用 `## 第X章 章名` 格式，标题后保留一个 Markdown 空行
- 正文段落之间不加空行
- 章节之间不使用 `---`、水平分隔线或额外空白行
- 单段长短不设固定字数上限，以“读者一眼能吞下一个动作单元”为准；若连续长段导致动作和受力点模糊，才视为问题

## Examples

### Correct — 段落按动作单元切分
```
沈栀抬手，灵力从指尖涌出。
面前的结界出现一道裂缝，碎纹像蛛网般蔓延开。
她咬紧牙关加大输出，整条手臂开始发颤。
```
每段承载一个清晰动作或受力单元，读者能顺着动作往下吞。

### Wrong — 多个动作硬塞成一团
```
沈栀抬手灵力从指尖涌出面前的结界出现一道裂缝碎纹像蛛网般蔓延开她咬紧牙关加大输出整条手臂开始发颤最后结界轰然碎裂碎片向四周飞溅。
```
多个动作和结果挤在同一段，读者难以停顿，也难以抓住受力点。

### Correct — 对话用动作引出（无「他说」）
```
沈栀将茶杯往桌上一顿。
"你到底想说什么？"
陆衍没接话，只是看着窗外。
```
对话通过动作和上下文引出，不使用「他说」「她道」等标签。

### Wrong — 使用对话标签
```
沈栀将茶杯往桌上一顿。
她说道："你到底想说什么？"
陆衍没接话，他轻轻叹道只是看着窗外。
```
「说道」「叹道」属于禁止的对话标签，应用动作替代。

### Correct — 章节标题后无多余空行
```
## 第X章 章名

陆衍推开门时，屋内已经坐了三个人。
沈栀坐在最里面的角落，手里捏着一张纸条。
```
章节标题 `##` 与正文之间仅一个空行，正文段落间无空行。

### Wrong — 段落间多余空行
```
## 第X章 章名


陆衍推开门时，屋内已经坐了三个人。


沈栀坐在最里面的角落，手里捏着一张纸条。
```
段落之间出现多余空行，破坏紧凑节奏。
