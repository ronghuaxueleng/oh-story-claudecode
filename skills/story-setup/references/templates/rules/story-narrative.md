# Story Narrative Rules

拆文、对标分析和设定文件中的本地补充规则。

## 优先级

- 项目根 `CLAUDE.md`、`写作执行铁律.md`、`story-long-write` 的质量层与门禁层规则高于本文件。
- 本文件只补“叙事与设定层的常驻提醒”，不得代替写后验收、脚本裁决和失败码。

## Rules

1. **设定交叉检查**：新增或修改设定时，必须与已有设定交叉检查矛盾。检查方法：`rg` 已有设定文件中的关键词，确认不冲突。
2. **语言风格一致**：角色对话必须匹配角色设定中的语言风格档案。
3. **世界观规则明确**：力量体系、社会规则等“可能”和“不可能”的事必须明确记录在设定文件中，且在正文中一致遵守。
4. **悬念必须有文档记录的真实答案**：设置的每个悬念/谜团，在追踪/或设定/文件中必须有明确的真实答案记录。
5. **角色动机和关系内部自洽**：角色行为必须符合其设定的动机，角色关系变化必须有合理触发事件。
6. **禁止 AI 腔**：
   - 禁止章末总结/升华/哲理收尾
   - 禁止“他感到/觉得”直述情绪，用身体反应替代
   - 禁止连续 2 段以上纯心理活动，必须穿插动作/对话
   - 禁止万能比喻
   - 禁止所有角色说话方式一样
   - 禁止把模板节点、验收术语、作者判题句直接写进正文；模板可以驱动正文，但不能在正文里点名自己
   - 禁止“像人话但不是人真会说的话”；优先回退常用口语层
   - 禁止作者替人物和读者做结论；优先改成现场动作、停顿、沉默、接不上话
   - 禁止把关键情绪、关系判断写成不带对象的空词；喜欢谁、委屈什么、体面给谁看必须落地
   - 禁止把紧密段落写成整页极短句密排；断段按镜头变化，不按短句数量
7. **对话三功能检验**：每段对话删掉后检查，三项全否则删除。
8. **人物不能只当接口**：关键配角至少要在正文里显出岗位、怕什么、要什么、此刻冒什么风险；若只能承担传话、送证据、替主角解释，默认人物层不过线。
9. **先保句子功能再改句**：改对白时先判断这句是陈述、反问、嘲讽、顶嘴、求情还是敷衍；禁止把原本有锋利功能的句子修成平陈句。

## Examples

### Correct — 设定交叉检查
```
新增设定：灵力等级分为九层，最高为第九层“破境”。
执行动作：
1. rg 设定/ 目录中所有文件，确认没有其他灵力体系描述
2. rg 追踪/伏笔.md 中“灵力”关键词，确认无矛盾伏笔
3. 更新追踪/上下文.md：[2024-05-05] 新增灵力九层体系
```

### Wrong — 设定未经交叉检查
```
直接在设定/世界规则.md 中添加“灵力等级分为十层”。
结果：已有设定文件中记载“灵力最高为九层”，两处矛盾。
且追踪/伏笔.md 中有一条伏笔依赖“九层”设定。
```
