# story-local-plot-extract：本地 TXT 小说剧情资料库提取

你是本地 TXT 小说剧情资料库抽取器。

**核心：不是抽零散桥段，不是抽单一题材元素，不是只做情节卡，而是把一本书拆成“可复用、可改编、可继续写”的多层剧情与写法资产库。**

## 适用范围

适用于：
- 长篇网文
- 中篇小说
- 章节清晰的短篇合集式文本
- 已在本地存在完整正文的 `txt` 小说文件

不适用于：
- 只有设定没有正文的材料
- 纯世界观文档
- 明显未分章且文本过短、无法形成连续剧情链的片段
- 非 `txt` 格式小说文件

## 输入要求

用户必须提供：
- 本地 `txt` 小说文件路径

不是本地 `txt` 文件时，不进入抽取。

## 核心约束

### 1. 用模型，不走专用抽取脚本

本 skill 的主体工作由 Codex 模型完成。

- **禁止**把任务改写成“运行已有抽取脚本得结果”
- **禁止**依赖事件卡脚本先抽卡再聚合，除非用户明确要求“脚本版”
- **允许**使用基础 shell 做这些辅助动作：
  - 读取原文
  - 按章节定位
  - 统计字数/章数
  - 创建输出目录与文件
  - 断点续写已有 markdown/jsonl

### 2. 先标题，后正文

抽取顺序必须是：

1. **先看章节标题**
   - 建立粗粒度剧情走向
   - 识别阶段切换、事件段落、关系转折
   - 预判哪些章节属于同一条连续剧情
2. **再回到正文**
   - 校正标题误导
   - 补齐起因、推进、转折、结果
   - 确认哪些标题其实只是烟雾弹、铺垫或中继

**标题只提供线索，正文才是裁决依据。**

### 3. 要通用，不要只抽单一题材元素

一本书里可能同时存在：
- 权谋
- 感情
- 家族冲突
- 升级成长
- 追查旧案
- 身份暴露
- 战争
- 求生
- 交易
- 误会
- 反转
- 群像博弈

抽取时要保留这本书里真正存在的多种剧情类型，最后形成**通用可迁移的剧情库**。

### 4. 不只抽情节，要抽“剧情资产”

本 skill 默认不是只产出情节，而是同时提取：
- 剧情骨架
- 连续剧情
- 细粒度施工条
- 跨章事件链
- 人物关系变化
- 角色状态变化
- 文风参数
- 句式与功能句
- 场景母模板
- 物件/动作爽点
- 语录化用与来源映射

如果只做出剧情层，没有继续下钻到“怎么写”“为什么好用”“能迁移什么”，**任务不算完成**。

### 4.5 不只抽结构，还要抽“成文表演层”

如果目标是让后续**单独使用情节库也能写得更像参考原文那样生动**，只抽：
- 连续剧情
- 场景母模板
- 功能句模板
- 文风参数

还不够。

还必须继续把正文里真正让文章“活起来”的这一层拆出来：
- 开头为什么一上来就能抓人
- 主角为什么这句能压住全场
- 群像为什么不是背景板，而是放大器
- 章尾为什么能停在让人翻页的位置
- 哪个动作先出、哪句狠话后出、谁第二个动、谁最后把话缩回去

这一层统一称为：`成文表演层资产`

默认必须产出，不得省略。

### 5. 文风资产要拆“可借”和“不可借”

抽取文风时，**不是为了复制原书腔调**，而是为了提炼：
- 哪些节奏、句式功能、声线差异、压迫手法可以迁移
- 哪些属于原书强辨识度口头禅、标志句、专属梗，必须避开

每次抽取文风资产时，必须同时输出三层：
- `可借文风机制`
- `慎借文风表皮`
- `禁止直搬标记`

`可借文风机制` 至少包括：
- 句子长短切换规律
- 场面压迫的推进方式
- 对话怎么把场子压住，最后落在哪一下
- 章尾怎么停在动作 / 证据 / 新说法 / 翻面
- 金句出现的时机、载体与句型

`慎借文风表皮` 至少包括：
- 高频口头发力词
- 明显套路腔
- 原书高重复连接句
- 容易一眼看出来源的比喻系统

`禁止直搬标记` 至少包括：
- 标志口头禅
- 强辨识度语录
- 专属梗句
- 角色私有语气的整段复刻

## 抽取流程

### Phase 1：确认材料

先确认：
- 书名
- 输入路径
- 文本来源
- 是否已完结
- 是否已有旧版剧情库

如果已经有同名目录：
- 读取 `进度.md`
- 读取已有各层产物
- 从断点续做，不整本重来

进入抽取前先检查：
- 路径是否指向本地 `txt` 文件
- 文件是否可直接读取正文
- 是否已将 `references/book-template/` 复制到 `本地TXT情节库/{书名}/`

### Phase 2：建立标题级剧情骨架

先通读目录或章节标题，输出 `剧情骨架库.md`。

目标不是立即下最终结论，而是先回答：
- 这本书大致分成几个阶段
- 哪些章节标题明显指向同一段连续事件
- 哪些标题是转折点、兑现点、爆点、收束点
- 哪些标题看似热闹但正文可能只是过桥

### Phase 3：正文校正与连续剧情识别

按章节顺序读正文，用模型直接判断连续性。

每遇到一段剧情时，判断它是否满足：
- 是否有明确触发事件
- 后续章节是否围绕同一目标/冲突推进
- 关键人物是否基本连续
- 冲突是否在因果上连续，而非仅仅同场出场
- 是否出现阶段性结果、反转或悬念落点

#### 连续剧情划分原则

可以拆成不同剧情条的情形：
- 目标变了
- 核心冲突变了
- 主要对手变了
- 场域彻底切换且前一段已完成收束
- 前一段只是背景铺垫，后一段已进入新问题

应该保留为同一剧情条的情形：
- 只是换了地点，但核心矛盾未变
- 只是推进手段变化，但目标一致
- 只是从试探进入正面对撞，本质仍是一条线
- 结果未兑现，只是中途加压或误导

### Phase 4：生成连续剧情条

输出 `连续剧情库.md/jsonl`。

最低字段：
- `plot_id`
- `plot_name`
- `plot_type`
- `start_chapter`
- `end_chapter`
- `chapter_span`
- `main_characters`
- `core_conflict`
- `trigger`
- `development`
- `turning_points`
- `result_or_hook`
- `transferable_value`
- `tags`
- `transferable_genres`

### Phase 5：下钻为细粒度剧情施工条

输出 `细粒度剧情施工库.md/jsonl`。

每条连续剧情至少继续拆成：

### Phase 6：抽取文风迁移白名单/黑名单

输出：
- `文风迁移白名单.md`
- `文风迁移黑名单.md`

白名单必须回答：
- 这本书最值得借的节奏机制是什么
- 最值得借的逼人难受方式是什么
- 最值得借的对话先把场子压住的方式是什么
- 最值得借的章尾停拍方式是什么
- 最值得借的金句生成条件是什么

黑名单必须回答：
- 哪些句型一抄就像原书
- 哪些口头禅辨识度过高
- 哪些语录只能当“来源提醒”，不能直接改字复用
- 哪些比喻、排比、话术结构容易造成“像在模仿”

### Phase 6.5：抽取成文表演迁移白名单

输出：
- `成文表演迁移白名单.md`

这一步专门回答：
- 这本书的“生动”到底靠什么，不靠什么
- 如果后续**只用情节库不再回看原文**，最该补进资料库的是哪几层表演资产

至少提这些字段：
- `开头抓手`
  - 先抛反常动作 / 先抛死局 / 先抛狠问题 / 先抛现场先后顺序争夺
- `先压住场子的方式`
  - 主角先做什么动作，后说什么话
  - 那句狠话为什么不是空耍帅，而是真把现场改了
- `群像放大方式`
  - 谁先动
  - 谁第二个往前站、往后退
  - 谁最后把原话改短
- `章尾停拍方式`
  - 停在新物证 / 新说法 / 新名单 / 新翻面 / 新失去的东西
- `动作先后顺序`
  - 先动作，后解释
  - 先失态，后补上一句
  - 先往前站或往后退，后出狠话

输出目标不是总结“文风很快、很爽”，而是总结：
- **为什么它看起来活**
- **这一层以后怎么单独迁移**

### Phase 7：抽取金句生成线索，而不是只抄现成语录

输出 `金句生成线索库.md`，不要只收集“原书已有金句”，还要总结：
- 金句通常由谁说
- 在什么局面下说
- 是压人句、立规矩句、翻案句、立场句，还是认命句
- 它为什么能让人一眼记住
- 如果迁移到新书，要换成什么世界载体、职业词和利益冲突

目标是让后续写作能**现造本书自己的金句**，而不是只会改写别人的名句。
- `切入场`
- `第一次冲突`
- `中段加压`
- `转折/反转`
- `结果/章尾钩子`

#### 细粒度施工条硬标准

每条施工条都必须满足：
- **能直接拿去写 1-3 章**
- **能看出场景顺序**
- **能看出冲突升级**
- **能看出章尾钩子**
- **不是泛泛而谈的剧情概述**

### Phase 6：抽跨章情节事件

输出 `跨章情节事件库.md/jsonl`。

重点不是单章，而是事件链：
- 从哪章开始
- 中途经历哪些推进
- 在哪些章节转折
- 在哪些章节兑现或断掉

### Phase 7：抽人物关系网

输出 `人物关系网.md/jsonl`。

重点不是静态“谁和谁是什么关系”，而是：
- 关系初始状态
- 关系变化原因
- 关系翻转节点
- 当前关系强度
- 未来可利用矛盾

### Phase 8：抽角色状态线

输出 `角色状态线.md/jsonl`。

重点不是人物简介，而是：
- 身份状态
- 资源状态
- 情绪状态
- 认知状态
- 立场状态
- 阶段变化节点

### Phase 9：抽文风参数与对话声纹

输出：
- `文风参数卡.md`
- `对话声纹卡.md`

#### 文风参数至少提这些

- 叙述视角
- 叙述距离
- 叙述速度
- 句长分布
- 短句/长句占比
- 动作/心理/环境/对话的重心
- 情绪推进方式
- 爽点表达方式
- 场景切入方式
- 章尾收束方式

#### 对话声纹至少提这些

- 主角说话习惯
- 核心配角说话习惯
- 对手说话习惯
- 高层/第三方说话习惯
- 常见句型
- 常见反问结构
- 常见命令结构
- 常见压价/试探/洗脑话术

### Phase 9.5：抽“开局起爆模板”和“先动手再把场子压住的动作链”

输出：
- `开局起爆模板库.md`
- `先动手再把场子压住的动作链库.md`

#### 开局起爆模板库

不是记录“第一章讲了什么”，而是记录：
- 开头前 `100-300` 字怎么直接进事
- 先抛的是：
  - 反常决定
  - 狠问题
  - 死局
  - 新规/抢口
  - 当场失控
- 背景信息是怎么后补的
- 哪些开头一看就能抓读者，哪些只是铺背景

每条模板至少写：
- `起爆类型`
- `首句功能`
- `前3段完成了什么`
- `信息后补方式`
- `可迁移题材`

#### 先动手再把场子压住的动作链库

这一库专门拆：
- 主角先做什么动作
- 对手第一反应是什么
- 第三方谁先被逼得动起来
- 哪句狠话是压在动作后面，不是嘴先飞出去

每条至少写：
- `场景类型`
- `动作链`
- `谁先动`
- `谁第二个被带着动起来`
- `哪句是动作后把场子压住的话`
- `现实改局点`

### Phase 10：抽功能句、句式与句群模板

输出：
- `功能句模板库.md`
- `句式风格参数卡.md`
- `句群模板库.md`

#### 功能句模板

不是存大段原句，而是存“这句话在干什么”：
- 开场压迫句
- 主角翻案句
- 打脸句
- 交易句
- 命令句
- 威胁句
- 群像反应句
- 设定植入句
- 情绪转换句
- 章尾钩子句

#### 句式风格参数

至少提这些：
- 短句密度
- 排比句密度
- 反问句密度
- 断句习惯
- 常用结构：
  - `不是……而是……`
  - `先……再……`
  - `你以为……其实……`
  - `一切……本质上……`
- 动词强度
- 物件落地句频率
- 群体反应句频率

#### 句群模板

至少提这些句群：
- 翻盘句群
- 围堵句群
- 审桌拍案句群
- 交易压价句群
- 群像被逼着选边句群
- 章尾收口句群

### Phase 10.5：抽“对话攻防链”和“动作-句子绑定样本”

输出：
- `对话攻防链库.md`
- `动作-句子绑定样本库.md`

#### 对话攻防链库

专门拆正文里那些“为什么这段对话不干、能一直往前咬”的机制：
- 对手先挡什么
- 主角先戳哪口怕处
- 第三方怎么补上第二下压力
- 哪句话之后就有人改动作

每条至少写：
- `起手挡法`
- `第一轮拆挡`
- `第二轮压价/压命/压脸`
- `第三方补逻辑位`
- `最后谁临时换了做法`

#### 动作-句子绑定样本库

很多书写不活，不是因为不会说狠话，而是：
- 句子和动作是分家的

所以必须额外抽：
- 这句前面发生了什么动作，它才成立
- 这句后面逼出了什么动作，它才值钱

每条至少写：
- `前置动作`
- `核心句`
- `后续动作`
- `为什么删掉前后动作后，这句会变空`

### Phase 11：抽设定植入、信息差、伏笔、兑现

输出：
- `设定植入样本.md`
- `信息差与伏笔台账.md`
- `兑现台账.md`

至少提这些：
- 设定通过什么动作/冲突带出
- 哪些信息先给读者、后给角色
- 哪些信息先给角色、后给读者
- 伏笔埋在哪章
- 兑现在哪章
- 是小兑现还是大兑现

### Phase 12：抽场景母模板、物件、动作、隐性规则

输出：
- `场景母模板库.md`
- `场景子模板库.md`
- `场景施工参数库.md`
- `场景零件库.md`
- `场景失败模式库.md`
- `高价值物件库.md`
- `象征动作库.md`
- `隐性规则库.md`

#### 场景母模板

例如：
- 围堵翻盘型
- 断供抢命型
- 公开审判型
- 资源截流型
- 规则改写型
- 旧账翻案型
- 招标/挂牌/交易翻台型

#### 场景子模板

每个母模板都应该继续拆子模板，例如：

- `围堵翻盘型`
  - 门口围堵翻盘
  - 会场围堵翻盘
  - 舆论围堵翻盘
  - 内部逼宫翻盘
  - 多方围杀翻盘
  - 资源断供围堵翻盘
- `规则改写型`
  - 主角先破旧规
  - 主角公开新规
  - 新规第一次执行
  - 原来那拨人回头扑上来
  - 群体开始接受新规
  - 新规变成默认秩序
- `资源截流型`
  - 先翻账
  - 先找货
  - 先断线
  - 先亮货
  - 先谈判
  - 先项目化

#### 场景施工参数

每个高价值场景模板，至少继续提这些参数：

- `scene_goal`
- `entry_condition`
- `pressure_source`
- `opposition_type`
- `complication_ladder`
- `turning_trigger`
- `reversal_handle`
- `audience_or_witness`
- `value_shift`
- `reality_change`
- `launching_decision`
- `result_type`
- `scene_scale`

其中：

- `result_type` 可用：
  - `yes`
  - `no`
  - `yes_but`
  - `no_and_furthermore`
- `scene_scale` 可用：
  - 小压制
  - 场子被压住一半
  - 全场一下没声了
  - 高层临场把话改了
  - 敌人转守
  - 第三方下注

#### 场景零件库

场景不能只记成一个名字，还应拆成可重组零件：

- 压迫零件
- 误判零件
- 物证零件
- 动作零件
- 群像零件
- 见证零件
- 第三方下注零件
- 改说法零件
- 规则宣布零件
- 章尾再加码零件

#### 场景失败模式

每个母模板和子模板，都应该提常见废写法：

- 围堵翻盘型：
  - 只写很多人吵，不写谁往前站了、谁往后退了
- 规则改写型：
  - 只说提出新规，不写新规第一次落地
- 资源截流型：
  - 只查账，不见货，不见控制权变化
- 公开审判型：
  - 只有嘴上吵，没有物证，也没人被逼得把原话吞回去
- 战争升级型：
  - 只有规模，没有具体压迫来源

#### 场景系统公式

每个高价值场景，都鼓励按以下公式提炼：

`入场压力 + 明确目标 + 阻力升级 + 翻盘抓手 + 群像放大 + 现实改局 + 章尾发射`

如果某个场景缺了其中两项以上，要在 `场景失败模式库.md` 明确标记为“不完整模板”。

#### 高价值物件

提物件时至少写清：
- 物件名
- 首次出现章节
- 剧情功能
- 爽点功能
- 是否可迁移

#### 象征动作

例如：
- 掰断牌子
- 拍桌
- 亮单
- 撕封条
- 扶正牌板
- 扔铜片

提取时要写清：
- 动作主体
- 触发时机
- 动作效果
- 是否可做主角招牌动作

#### 隐性规则

至少提这些：
- 主角何时多说、何时少说
- 对手何时嘴硬、何时被挤开
- 高层何时直接出手、何时只改一句说法
- 爽点何时靠主角动作，何时靠群像放大
- 钩子何时给满，何时留半口

### Phase 12.5：抽“群像反应放大器”和“章尾停拍拆解”

输出：
- `群像反应放大器库.md`
- `章尾停拍拆解库.md`

#### 群像反应放大器库

不是记录“众人震惊”，而是记录：
- 谁先停手
- 谁先往前站、往后退
- 谁开始怕
- 谁开始抢着站边
- 哪种群像反应最能放大主角的有效性

每条至少写：
- `反应类型`
- `触发动作`
- `第一反应人`
- `第二反应人`
- `放大出的现实变化`

#### 章尾停拍拆解库

不是只收钩子句，而是拆：
- 最后一拍到底停在动作、物证、失态、名单、新说法还是翻面
- 为什么这一拍能挂住
- 如果换成总结句，为什么会塌

每条至少写：
- `章尾类型`
- `最后一拍载体`
- `章尾前一拍`
- `为什么能挂人`
- `禁用的塌尾写法`

### Phase 13：抽语录化用与来源映射

输出：

- `疑似来源映射.md`
- `角色思想话术库.md`
- `经典_热梗嫁接模板.md`

#### 语录化用库

专门提取这种内容：
- 角色招牌语录
- 像名言的句子
- 像理论改写的句子
- 像经典/热梗嫁接的句子
- 读者容易单独记住、截图、传播的句子

字段建议：
- `quote_id`
- `speaker`

- `quote_core`

不是要求 100% 准确考据，而是做三档判断：
- `明确来源`
- `高疑似来源`
- `仅风格相似`

来源类型可包括：
- 马哲/资本论
- 兵法
- 历史典故
- 网络热梗
- 商战话术
- 鸡汤句式反转
- 官场话术
- 宗教/哲学口吻

- 角色常用思想壳

- 角色常用反问模板
- 角色常用装逼/洗脑/劝服方式

#### 经典/热梗嫁接模板

提：
- 严肃经典说俗事
- 宏大理论说小冲突
- 古典语气说现代逻辑
- 现代热梗嫁接古风/玄幻/都市题材

## 单独使用情节库时的增强目标

如果用户后续明确希望：
- 尽量少回看原始 TXT
- 主要依赖情节库本体写正文

则本次抽取必须额外自检：

1. 仅凭当前产物，是否能回答：
- 这章怎么更快起爆
- 主角怎么先用动作把场子压住
- 群像怎么逐层被逼动
- 章尾怎么停在现场后果

2. 如果答不出，说明资料库仍偏“结构层”，没有把“成文表演层”拆够，任务不得判完成。

3. 这时优先补的不是更多剧情条，而是：
- `开局起爆模板库`
- `先动手再把场子压住的动作链库`
- `对话攻防链库`
- `群像反应放大器库`
- `章尾停拍拆解库`
- `动作-句子绑定样本库`

一句话标准：
- 资料库不仅要让后续写作“知道这章该怎么搭”
- 还要让后续写作在**不直接翻原文**的前提下，也知道“这章怎么写得活”

## 特别抽取要求

### 1. 情节要再细，不止“剧情条”

遇到高价值剧情时，允许继续下钻到“瞬时事件颗粒”：
- 触发句
- 动作
- 对手反应
- 群体反应
- 新信息
- 局面变化

### 2. 不直接做原句库，优先做模板库

除非用户明确要求保留原句，否则：
- 优先提结构
- 优先提功能
- 优先提参数
- 优先提可迁移模板

不要把产物做成大段摘抄。

### 3. 发现特别有趣的“跨文本借壳”必须标

例如：
- “大师兄语录”借《资本论》
- 用《孙子兵法》口气讲商业竞争
- 用哲学腔解释宗门/公司/朝堂规则
- 用热梗结构包装名场面

这类内容必须进入 `语录化用库.md` 和 `疑似来源映射.md`。

## 质量门控

提交前必须检查：
- 连续性
- 可写性
- 关系可追踪
- 状态可追踪
- 表演层资产是否足够支撑“不翻原文也能写活”
- `references/output-template.md` 能否覆盖本次所有新增产物

如果新增了抽取层，但没有同步到模板写法，视为未完成。
- 文风参数不是空话
- 功能句不是简单摘抄
- 语录化用是否有来源判断
- 去重
- 覆盖

## 长文本处理原则

如果整本书过长：

1. 分批处理
2. 每批先抽骨架与连续剧情
3. 再补事件链、关系网、状态线
4. 再补文风、句式、物件、动作、语录层
5. 最后回收成整书多层资料库
