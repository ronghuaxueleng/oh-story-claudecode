# 补层与断点续补规则

## 抽取完成标准补充

本 skill 不允许停在“只把剧情主库抽出来”。

只要用户目标包含以下任一类：
- 单独使用情节库也要能写活
- 想让写出来更像参考正文的生动度
- 想减少后续反复回炉
- 想继续补到某一章断点

则默认必须继续补足以下两层：

### 1. 表演核层

至少包括：
- `开局起爆模板库.md`
- `压场动作链库.md`
- `群像反应放大器库.md`
- `对话攻防链库.md`
- `动作-句子绑定样本库.md`
- `章尾停拍拆解库.md`

### 2. 写作调用层

至少包括：
- `写作调用规程.md`
- `单章写前卡模板.md`
- 如用户继续要实战化，再补：
  - `单章写前卡_示例.md`

要求：
- 单章写前卡模板不得默认回灌 `掌口 / 改口 / 执行口 / 解释口 / 资源口 / 口令 / 口子 / 第一刀 / 第二刀 / 补刀 / 追刀 / 落刀 / 解释权 / 执行权 / 决定权 / 资源位 / 头位 / 控盘 / 抢势 / 盘面` 这类施工摘要词
- 必须优先产出“谁先动、谁先补、谁先分、谁先放、谁被迫认账、谁失去先手、谁先吃亏、谁先退、谁先交东西、谁先让地方”这类动作结果词

## 断点续补规则

如果用户明确说：
- “继续”
- “全部补上”
- “补到第521章”
- “不要再提取情节了，做第二个方案”

则执行顺序固定为：

1. 先判断当前目标是：
  - `剧情主库续补`
  - `写法扩展库续补`
  - `表演核续补`
  - `文风/语录层续补`
  - `调用规程层续补`
2. 用户如果明确说“不要再提取情节”，则禁止继续补 `连续剧情库` 以外的新剧情层。
3. 优先补表演核、文风/语录层、调用规程层。
4. 每轮续补后必须同步：
  - `进度.md`
  - `抽取说明.md`
  - `_meta.json`

## 写法扩展层优先级补充

当用户目标是“让情节库单独使用时也像参考原文那样生动”，优先级固定为：

1. `表演核层`
2. `文风/语录层`
3. `写作调用层`
4. 最后才是继续抽新的剧情主库
