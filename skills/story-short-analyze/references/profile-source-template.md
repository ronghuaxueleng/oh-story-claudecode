# profile_source 模板

这份文件用于 `写作资产/profile_source.md`。

定位：

- 这是模型先提的原始规则层
- 不是最终 `book.profile.json`
- 作用是把脚本难抽出的隐性信息先固定下来，再交给脚本标准化

使用要求：

- 必须基于原文和完整拆书结果填写
- 不要写空泛题材介绍
- 每个字段尽量给“短语 + 简短说明”
- 能落到桥段、物件、动作、场面、句壳，就不要只写抽象结论
- 先做 `样本分级`，再决定哪些字段能进正向 DNA，哪些只能进禁写规则
- `桥段承重件` 这一节不是只给 `must_keep` 用的，还要让脚本能抽出：
  - `opening_pattern`
  - `recommended_sequence`
  - `why_order_matters`
  - `fake_signals`
  - `why_original_passes`

桥段字段映射口径：

- `原文怎么起手` -> `opening_pattern`
- `承重件` -> `must_keep`
- `不能丢的顺序` -> `recommended_sequence`
- `为什么这个顺序不能乱` -> `why_order_matters`
- `最容易写假的点` -> `fake_signals`
- `原文为什么能过` -> `why_original_passes`
