
# 网文质量检查清单

> **定位**：story-short-analyze（短篇拆书）的质量自检清单，评估拆解对象的成文质量时逐项核对。
> 拆书管道本身的质量门控见 `output-templates.md`「质量门控必填字段」；所有数值阈值的唯一权威定义见 `material-decomposition.md`「质量标准」与「节奏分析」，本清单不自定义数值。

## 一、通用检查

### 章节结构
- [ ] 开头有钩子（不是天气/风景/日常开场）
- [ ] 中段有推进（有事件发生）
- [ ] 局势有变化（读完这章，世界跟之前不一样了）
- [ ] 结尾落在变化上（不是总结）

### 开篇检查（前 300-500 字）
- [ ] 有钩子，能抓住注意力
- [ ] 不从天气/风景/日常开始
- [ ] 主角快速出场
- [ ] 卖点或危机可见
- [ ] 前三句各自有功能，不是三句一个意思
- [ ] 第一屏不只是交代处境，还给了结果期待或危险预期

### 章节推进
- [ ] 有核心事件
- [ ] 局势有变化
- [ ] 不是水字数（删掉这章会影响理解吗？不会 = 水了）
- [ ] 推进了主线/关系/设定中的至少一项
- [ ] 中段不是只重复受气，而是又加了一层事，或者人物进了新状态

### 信息传递
- [ ] 没有大段设定说明文
- [ ] 信息跟着冲突走（通过事件传递设定）
- [ ] 设定量可控（一章不超 3 个新概念）

### 场景检查
- [ ] 场景有目标（人物要什么）
- [ ] 场景有阻碍（什么挡着）
- [ ] 场景有变化（结束后跟之前不同）
- [ ] 人物在做事情，不是在感觉事情
- [ ] 没有可删除的段落
- [ ] 主角不是纯承受，至少出现过关键非标反应或主动动作

### 章尾
- [ ] 结尾落在变化上
- [ ] 有危机/决定/发现/反转中的至少一个
- [ ] 不是总结式结尾
- [ ] 拉住读者翻下一页
- [ ] 结尾不是只把结果说完，而是让位置、关系或认知发生了变化

### 语言
- [ ] 没有空洞的抒情段落
- [ ] 没有连续多段同一情绪
- [ ] 对话符合人物身份（不同人说话方式不同）
- [ ] 情绪通过动作落地（不是直接说"他很难过"）
- [ ] 强情绪不只靠抽象词，能看见眼神、身体、说话或动作变化

### 连载连续性
- [ ] 没有遗忘之前的承诺/伏笔
- [ ] 没有突然塞入大量新设定
- [ ] 伏笔有推进
- [ ] 故事引擎还在运转

### 水（filler）检测
以下信号出现 = 可能水了：
- 全章对话没有任何新信息
- 同一个情绪写了 3 段以上
- 场景描写超过 500 字但不推进剧情
- 角色回忆之前发生的事但没有任何新视角
- 连续 2 章以上没有冲突
- 连续几段只是同类羞辱换说法，没有第二层升级

## 二、长篇专项

### 黄金三章检查
- [ ] 第一章前 500 字有钩子？
- [ ] 主角第一章就出场？
- [ ] 第一章有事件（不是纯铺垫）
- [ ] 第二章有升级（矛盾加深）
- [ ] 第三章有追读理由（给出继续看的动力）
- [ ] 前三章有至少 2 个爽点？
- [ ] 世界观没有大段说明文？
- [ ] 每章结尾有悬念？

### 节奏检查
- [ ] 最近 5 章是否有明确进展？
- [ ] 爽点间隔是否超过 5000 字？
- [ ] 有没有连续 2 章以上没有冲突？

### 人物检查
- [ ] 主角行为符合人设？
- [ ] 配角是否有存在感？
- [ ] 反派逼格是否匹配当前阶段？

### 五维评分标准

每个维度 0-100 分，根据评分结果选择精修策略。

### 维度 1：核心一致度
检查：关键冲突、关键行动、人物动机是否前后一致。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 人物动机突然改变无铺垫 | critical | 补充动机转变的触发事件 |
| 核心冲突前后不一致 | high | 回溯修改冲突设定 |
| 关键行动与人物性格矛盾 | high | 调整行动或补充解释 |
| 次要矛盾遗忘 | medium | 回收或弱化 |

### 维度 2：表层重写度
检查：句式与措辞是否足够原创，避免套路化表达。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 照搬原句导致语气不自然 | medium | 只在确有 AI 腔时改写，正常原句可保留 |
| 大量使用 AI 标志词 | high | 替换为具体描述 |
| 同一句式重复出现 | medium | 变换表达方式 |
| 描写过于文学化 | medium | 改为口语化/动作化 |

### 维度 3：格式一致度
检查：段落结构、字数分配、开头结尾格式是否统一。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 段落长度差异过大 | medium | 调整段落划分 |
| 章节字数偏离目标 | **high** | 写作/大纲修复时先回到细纲补足计划内情节点，再展开；去AI味已有正文时不得新增剧情 |
| 格式混乱（对话/描写不统一）| low | 统一格式 |

### 维度 4：可读性
检查：是否有啰嗦、AI 腔、空泛总结、套路修辞。

| AI 腔特征 | 如何识别 | 如何修复 |
|-----------|----------|----------|
| 空泛总结 | 「他终于明白了」「一切尽在不言中」 | 删掉，用行动代替 |
| 套路修辞 | 「命运仿佛在和他开玩笑」 | 删掉或换成具体描述 |
| 情绪标签 | 「他感到一阵悲伤」 | 改为行为表现 |
| 大段心理描写 | 连续 3 句以上内心独白 | 压缩为 1 句或转为对话 |

### 维度 5：逻辑连贯
检查：句间/段间是否通顺，有无设定冲突。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 设定前后矛盾 | critical | 查找并统一设定 |
| 时间线错误 | high | 标记时间线并修正 |
| 角色信息不一致 | high | 建立角色档案对照 |
| 因果链断裂 | medium | 补充过渡 |

### 精修策略

根据五维评分结果，选择精修策略：

| 主要问题 | 策略 | 说明 |
|----------|------|------|
| 核心一致度低 | rewrite | 围绕核心冲突重写相关段落 |
| 字数超标 | compress | 删减不推动剧情的内容 |
| AI 腔重 | de_ai | 替换禁用词、改写句式 |
| 小问题多 | polish | 打磨语言细节 |

## 三、短篇专项

### 仿写 / 融合可用度检查

如果这篇拆文后面不是只做欣赏分析，而是要继续拿去：

- 仿写
- 融合
- 去原作化后再写

那就不能只看“故事核、情绪线、反转机制有没有拆出来”，还要看有没有拆到能直接拿去写。

这组检查通过后，才算达到 `可直接仿写层 = ready-for-write`。如果没过，不要把这篇拆文判成“已备料完成”。

必查项：
- [ ] 已明确这篇属于什么流，不只是题材名，而是成文流法（如追妻情绪流/全员背叛流/婚恋清算流）
- [ ] 已拆出主梗、1-3 个副梗，以及它们各自负责开头/中段/高潮哪一层功能
- [ ] 已能用一句组合公式概括这篇（如“全员背叛 + 装穷 + 女兄弟梗”）
- [ ] 已有 `导语拆解表`，知道前 20 / 60 / 80 字分别完成了什么
- [ ] 已有 `顺序事件表`，能顺着说清原文怎么一步步往前推
- [ ] 已有 `物件表`，知道关键物件伤的是体面、旧伤、亲情、身份，还是关系位置
- [ ] 已有 `动作表`，知道谁抢、谁扔、谁拦、谁护、谁走、谁留证
- [ ] 已有 `对白功能表`，知道不同角色的话是在挑衅、压人、找补、装无辜，还是切割关系
- [ ] 已有 `对话衔接表`，知道对白怎么接、哪里用动作垫、哪里靠称呼变化加压
- [ ] 已有 `误判表`，知道读者和人物先被误导到哪一步，后来从哪翻
- [ ] 已有 `钩子表`，知道每一拍末尾留了什么、读者下一步在等什么
- [ ] 已有 `微动作表`，知道情绪不是只靠解释句，而是怎么落到动作上
- [ ] 已有 `安静压迫场表`，知道不靠大吵也能怎么把场压住
- [ ] 已有 `样本分级与可学层`，知道这本是 `A类正样本 / B类骨架样本 / C类负样本`
- [ ] 已明确 `是否可用于DNA提取`，不会把整本默认当正样本
- [ ] 已补 `原文检测结论 / 原文整体分数 / 原文高风险块 / 分数使用口径`
- [ ] 已明确原文是“整本高”还是“局部桥段高”，不会把高分原文误当低风险 DNA 源
- [ ] 已拆清 `可学层 / 禁学层`，知道这本到底能学骨架、承重件、动作层，还是能学句法层
- [ ] 已有 `作者DNA指纹`，知道这个作者稳定怎么切句、怎么给情绪落地、哪些小动作会反复承担压力
- [ ] 已有 `仿写约束_禁写清单`，知道哪些总结句、解释句、过渡句、假口语一写就会假
- [ ] 已有 `同桥段过检规则`，知道相同桥段里原文为什么能过、仿稿最容易假在哪、承重件不能丢什么
- [ ] 16 张 `可直接仿写_*.md` 都不只是有表，而且每张都补了 `可直接借的承重结构`
- [ ] 16 张 `可直接仿写_*.md` 都补了 `迁移顺序提醒` 或 `总迁移顺序提醒`
- [ ] 16 张 `可直接仿写_*.md` 都补了 `为什么这个顺序不能乱`
- [ ] `作者DNA指纹` 不是空标签，至少能拿出 3 组原文级证据说明句长切句/口气差/动作替代
- [ ] `作者DNA指纹` 里已经写出至少 2 组“明显不像这个作者会写”的反面句型
- [ ] `仿写约束_禁写清单` 不是只有禁令，每条后面都回答了“为什么假”
- [ ] `仿写约束_禁写清单` 已覆盖空情绪、假口语、补字废话、统一过渡句这 4 类高频坏味
- [ ] `同桥段过检规则` 不是只有桥段名，至少写清起手件、承重件、假点、保留件
- [ ] `同桥段过检规则` 里至少有 1 组以上“原文能过 / 仿稿会假”的正反对照

判定口径：

- 缺上面 10 项里的任意 2 项，默认这篇拆文还停在概括层
- 缺 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 里的任意 1 项，默认还没到可直接安全仿写层
- 这时候不要急着做融合稿，先回去补细
- 如果只有“故事梗概 + 起承转合 + 题材总结”，那不叫可直接仿写
- 如果没做 `样本分级`，默认不能直接提取整本 DNA
- 如果没写 `原文分数快照`，默认不能直接提取整本 DNA
- 如果明明是 `B类骨架样本`，却还在提句法层 DNA，默认拆错
- 如果连“这是什么流、靠哪几个梗成立”都说不清，默认题面层还没拆稳
- 如果没有 `同桥段过检规则`，默认还不能直接拿去做同桥段融合、去AI味回修、或原桥改写
- 如果表格里只有现象总结，没有 `承重结构 / 迁移顺序 / 顺序原因`，默认还没到施工层
- 如果这 3 份文件虽然存在，但没有原文证据、没有反面写法、没有桥段承重件，也默认不能放行
- 真正能进入下游写作的拆文，至少要能让人直接抽导语起法、抽事件、抽物件、抽动作、抽口气、抽衔接、抽误判、抽钩子、抽微动作、抽安静压迫场

和下游对齐的准出结论：

- `ready-for-write`：10 项基础表 + `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 全齐，且能回答“为什么能直接拿去写，为什么同桥段不会写假”
- `ready-for-write` 额外要求：能用一句话说清“这个作者最稳定的真人感落在哪三件事上”，并能指出“最该拦掉的三类 AI 坏句”
- `need-more-deconstruction`：缺任意 2 项以上，或只能抽概括结论，或缺 3 份硬门槛文件中的任意 1 份
- `blocked-on-assets`：骨架有了，但 `对话衔接 / 微动作 / 安静压迫场 / 同桥段过检规则` 这类表演层与过检层资产明显不够，写出来大概率会空

### 虐爽节奏检查

短篇特有的检查项——虐点和爽点的分布是否合理：

```
理想分布：虐1 → 虐2 → 虐3 → 爽1(小) → 虐4 → 爽2(大)
禁忌分布：虐1 → 虐2 → 虐3 → 虐4 → 虐5（无爽点，读者流失）
禁忌分布：爽1 → 爽2 → 爽3 → 爽4（无铺垫，爽感疲劳）
```

检查规则：
- 每 300 字至少 1 个小冲突
- 虐点间隔不超过全文的 30%
- 最大爽点必须出现在全文的 70-85% 位置
- 结尾必须有情绪落点
- 爽点不能只靠处刑，最好带位置变化、迟到惩罚或价值抬高

### 反套路检查

- [ ] 表面题材明确，但真正抓人的地方不是只靠题材本身
- [ ] 有真正特别的地方，不是常规桥段直推到底
- [ ] 中段又加了一层事，不是单线打脸
- [ ] 主角至少有 1 处非标反应
- [ ] 高潮不只是兑现，还能再往上抬一下，或者让人回头重看前文
- [ ] 结尾不只是处刑，还把整篇文往上抬了一下
- [ ] 题面和导语先扎中了关系错位或最想看的后果，不是平铺背景
- [ ] 如果目标是后续仿写，已经能从拆文里直接抽到“事件 / 物件 / 动作 / 口气 / 误判 / 钩子”，而不是只能抽概括结论

### 追妻专项检查（追妻题材适用）

- [ ] 旧情基础清楚，不是平地追妻
- [ ] 男主不是突然变坏或突然醒悟，而是有一步步站错边的过程
- [ ] 中间至少有 1 次他差点看明白却没接住
- [ ] 真相后他没有立刻全懂，存在认错重点或补错方式
- [ ] 最后“追不回”成立，不是嘴上说晚了，实际上还能轻松和好

### 对话密度检查

> 对话占比合格标准见 `material-decomposition.md`「节奏分析」。

| 指标 | 合格标准 | 警戒线 |
|------|----------|--------|
| 对话占全文比 | 见 `material-decomposition.md`「节奏分析」 | <30%（太干）或>75%（太水） |
| 每章对话条数 | 12-18条/千字 | <8条（缺交互）或>25条（碎片化） |
| 伤人性对话占比 | 35-45%（虐文）/ 20-30%（爽文） | 虐文<20%（不够痛）/ 爽文>40%（太压抑） |

### 主角冷静度检查（打脸/复仇文专用）

| 检查项 | 标准 |
|--------|------|
| 主角是否有标志性冷静动作？ | 必须有（端水杯/整理西装/转笔等） |
| 主角是否有情绪失控场面？ | 复仇文最多1次，且必须在前30% |
| 反派是否比主角更歇斯底里？ | 必须是——反差是爽感来源 |
| 主角台词是否短于反派？ | 平均短30-50%——越短越有力 |
| 是否有"审判式对话"？ | 至少2处（主角提问→对方自爆） |

### 证据链完整性检查（复仇/打脸文专用）

| 检查项 | 标准 |
|--------|------|
| 证据是否分章释放？ | 至少分3次揭露，不能一次全给 |
| 每个证据是否有铺垫？ | 前文必须埋过线索 |
| 反派是否每次都先得意再被打脸？ | 必须是——先扬后抑 |
| 最终证据是否最致命？ | 最后一个证据要改变全局认知 |
| 是否有"定时炸弹"证据？ | 至少1个主角提前布局的证据 |

### 毒点/代入感/震惊/开头/结尾/情绪/期待感速查

> 详细的毒点分类、识别方法和修复方案见 `anti-ai-writing.md`。

| 检查项 | 标准 |
|--------|------|
| 爽文不爽 | 金手指效果必须展示清楚 |
| 压制无目的 | 每次压制必须服务于后续爆发 |
| 反派结局与主角无关 | 反派之死必须因主角行动导致 |
| 题材很熟但没有特别的地方 | 容易套路化，必须补中段再加的事或非标反应 |
| 经济/战力崩坏 | 设定前后一致，以普通人为锚点 |
| 女主/女配降智 | 人设正常可写，老套路别写 |
| 读者期待长时间不满足 | 适时满足或引入新期待 |
| 主角行为不可理解 | 必须能理解、能共鸣 |
| 氛围被突兀梗破坏 | 氛围 > 突兀梗（乐子文除外） |
| 钩子不回收 | 钩子不回收 = 烂尾 |
| 震惊分层 | 点→网→深度，不能只写点震惊 |
| 震惊阶梯递进 | 阶梯式上升，非直线 |
| 震惊有广度 | 关系网都要有反应 |
| 前50字有冲突/异常 | 不能是背景铺垫 |
