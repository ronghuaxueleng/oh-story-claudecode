# 章法手法硬拆清单

这份文件只解决一个问题：

`拆短篇时，不能只写“用了倒叙/插叙/反转/留白”，必须拆清这些章法手法到底压在哪个桥上、为什么成立、为什么换一种顺序就会假。`

适用场景：

- 后续要 `仿写 / 融合 / 原情节实验 / 去AI味回修`
- 用户关心 `为什么原文普通桥不过检、仿稿却会高`
- 这篇文明显靠 `章法、断场、入口分配、时间操控、信息漏出顺序` 成立

---

## 一、必须硬拆的 8 类章法问题

### 1. 开场入口

必须回答：

- 这篇从哪一秒切入
- 为什么不是从更早的铺垫切
- 开场第一屏先亮的是 `关系 / 冲突 / 后果 / 误判 / 场面秩序` 哪一类
- 如果顺着常规时间线从头写，会不会立刻变成功能化说明开头

### 2. 时间线手法

必须回答：

- 是 `线性 / 插叙 / 倒叙 / 双线交叉 / 回忆插片 / 闪回`
- 每次时间跳跃具体落在哪个桥上
- 这次跳跃补的是 `因果 / 误判 / 旧伤 / 反差 / 认知回收` 哪一类
- 如果把它改成平铺直叙，会损失什么

### 3. 回忆和插片的职责

必须回答：

- 回忆是不是只解释背景
- 回忆有没有改写当前这场局
- 回忆是补情绪、补旧伤，还是补认知反差
- 回忆插入的位置为什么在这里，不在前一段或后一段

### 4. 断场位置

必须回答：

- 这一章 / 这一场最该断在哪一句
- 断场前最后放出来的，是 `动作 / 误判 / 后果 / 空白 / 场面位移` 哪一种
- 为什么这里要硬断，不顺手收全
- 如果不断，会不会直接被读成完整成品包

### 5. 入口分配

必须回答：

- 哪条线只能停在余波区
- 哪条线有资格吃尾声入口
- 最终收口是给 `前任线 / 核心关系线 / 旧人线 / 物件线 / 身体感`
- 为什么这个入口给他，不给另一个人

### 6. 信息漏出顺序

必须回答：

- 关键事实是 `先伤人再解释`，还是 `先解释再伤人`
- 偏心、旧伤、真相、后果，谁先落地
- 有没有刻意延后关系判断
- 原文有没有避免“拿到就懂、懂了就做、做了就见效”的整齐闭环

### 7. 场面播报密度

必须回答：

- 大场面是完整转播，还是只抓几下关键画面
- 哪些信息故意不播全
- 哪几下位移、脸色、站位已经足够伤人
- 如果把整场补成说明书，会不会立刻变假

### 8. 收口方式

必须回答：

- 结尾停在 `物件 / 动作 / 身体感 / 空白 / 未说透`
- 有没有标准主题句、宣言句、总结句
- 结尾是在“讲明白”，还是在“闷住”
- 如果把尾声再写满一场完整告别戏，会不会掉成模板尾巴

## 二、拆书时必须补的章法结论

如果这篇文要进入后续仿写或高敏回修，下面这些结论必须写进产物：

### 写进 `写作手法.md`

至少补：

1. `开场入口为什么选这里`
2. `时间线类型 + 每次跳跃的职责`
3. `回忆/插片为什么插在这里`
4. `关键断场点`
5. `尾声入口给了谁`
6. `结尾为什么停在这里`

### 写进 `同桥段过检规则.md`

至少补：

1. 这个桥在原文里靠什么章法藏住成品感
2. 这个桥最怕被写成哪种标准顺推顺序
3. 哪个断场点不能丢
4. 哪个信息必须延后漏出

### 写进 `作者DNA指纹.md`

至少补：

1. 这个作者常把解释压后还是压掉
2. 这个作者常把结尾停在哪类落点
3. 这个作者什么时候用回忆，什么时候坚决不用

- 重要事件是否被写得过于完整，反而没有余波、偏移或不均匀停顿

重点看：

- 是否反复走“提问—回答—确认—解释”的同一回路
- 配角是否只负责递信息、确认主角判断或推动下一拍
- 是否存在答非所问、误听、抢话、找补、闲枝、沉默或没有被追问的半句话

## 五、全局审计的输出位置

以下结论必须同时进入：

1. `拆文报告.md` 的 `### 全局成文形状审计`
2. `写作手法.md` 的 `## 10. 全局成文形状审计`
3. `写作资产/样本分级与可学层.md` 的 `## 4.4 全局成文形状审计`

只在某个细节表或模型备注里出现，不算完成。缺原文证据、只写形容词、没有区分
可学层与禁学层，均不得进入正式写作资产收口。

## 六、不合格信号

只要出现下面任一类，默认章法还没拆到位：

- 只写“用了倒叙/插叙”，没写为什么
- 只写“有回忆”，没写它改写了当前哪场局
- 只写“这里断章效果好”，没写不断会怎样
- 只写“结尾留白”，没写留白具体停在哪
- 只写桥段名，没写章法怎么把桥藏在生活里
