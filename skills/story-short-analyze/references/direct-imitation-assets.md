# 可直接仿写层资产合同

适用场景：

- `仿写`
- `融合`
- `去原作化后再写`
- `先拆后写`

原则：

- 这些表是接口，不是让 skill 主文件塞进大词库
- 表里记录的是功能、顺序、口气差异、表演层资产，不是堆漂亮话
- 缺关键表时，优先回上游补拆，不要拿概括结论硬写
- 16 张 `可直接仿写_*.md` 不是“有表就算完成”，每张表都必须补齐固定施工层

固定施工层：

1. `可直接借的承重结构`
2. `迁移顺序提醒`（跨事件主链可改用 `总迁移顺序提醒`）
3. `为什么这个顺序不能乱`

用途解释：

- `可直接借的承重结构`：回答这张表后面真正值得搬走的骨架，不是泛泛而谈的优点
- `迁移顺序提醒`：回答这些件挪去新稿时先写什么、后写什么、哪一步不能抢跑
- `为什么这个顺序不能乱`：回答原文为什么成立、仿稿为什么最容易在顺序上写假

验收口径：

- 缺任意 1 段，默认该表还停在分析层
- 三段写成空话，也默认未完成
- 如果只能写“这段很爽 / 这个桥很虐”，说明还没拆到施工层

## 最小字段

### 1. 导语拆解表

| 字段 | 说明 |
|------|------|
| 前20字 | 是否已经起事 |
| 前60字 | 是否完成关系错位 / 事故信息 / 后果预期中的至少 2 项 |
| 前80字 | 是否出现第二推进点 |
| 第一句功能 | 越界现场 / 结果先行 / 关系错位 / 反常反应 |
| 第二句加码 | 抬后果 / 抬危险 / 抬掉位 |
| 继续滑动力 | 读者为什么会继续往下滑 |

### 2. 顺序事件表

| 字段 | 说明 |
|------|------|
| 编号 | `01 / 02 / 03 ...` |
| 谁做了什么 | 必须具体到人 |
| 具体动作 | 不接受空泛概括 |
| 这一拍功能 | 起事 / 压位 / 留钩 / 翻刀 / 抬后果 |
| 可迁移写法 | 换壳后还能怎么用 |

### 3. 物件表

| 字段 | 说明 |
|------|------|
| 物件 | 关键承重件 |
| 首次出现 | 节位 / 场面 |
| 后续回收 | 回收点 |
| 伤害层 | 体面 / 旧伤 / 身份 / 关系位置 / 亲情 |
| 可替换功能件 | 换题材后的同功能件 |

### 4. 动作表

| 字段 | 说明 |
|------|------|
| 动作 | 抢 / 扔 / 拦 / 护 / 留证 / 装无辜等 |
| 谁做 | 动作发起人 |
| 对谁做 | 动作承受人 |
| 当下效果 | 当场怎么压位 |
| 后续效果 | 为后文埋了什么 |

### 5. 对白功能表

| 字段 | 说明 |
|------|------|
| 角色 | 主角 / 压迫者 / 接住者 / 旁观者等 |
| 典型说法类型 | 挑衅 / 找补 / 压人 / 装无辜 / 切割关系 |
| 这类话负责什么 | 拉仇恨 / 洗白自己 / 逼位 / 翻盘 |
| 口吻特征 | 称呼、长短句、先忍还是先刺 |
| 明显不像他说的话 | 用来拦错配口气 |

### 6. 对话衔接表

| 字段 | 说明 |
|------|------|
| 上句功能 | 上一句在逼什么 |
| 下句接法 | 反问 / 冷接 / 装傻 / 转移 / 压回 / 停顿后接 |
| 动作垫句 | 有没有用动作垫一下再说 |
| 称呼变化 | 是否改称呼制造关系变化 |
| 衔接效果 | 为什么这一接更刺人 / 更压人 / 更显心冷 |

### 7. 误判表

| 字段 | 说明 |
|------|------|
| 误判方 | 读者 / 角色 / 群体 |
| 先误判了什么 | 早期认知 |
| 从哪开始翻 | 触发点 |
| 翻完后前文怎么变 | 重读价值 |

### 8. 钩子表

| 字段 | 说明 |
|------|------|
| 位置 | 段末 / 节末 / 场末 |
| 钩子内容 | 留了什么问号或后果 |
| 钩子类型 | 越界现场 / 结果先行 / 信息差 / 掉位后果等 |
| 读者下一步在等什么 | 明确期待 |
| 回收位置 | 哪一拍兑现 |

### 9. 微动作表

| 字段 | 说明 |
|------|------|
| 动作本体 | 捏杯口 / 把话咽回去 / 手指停住等 |
| 对应情绪 | 心冷 / 发紧 / 憋屈 / 防御 |
| 角色状态 | 忍 / 试探 / 硬撑 / 装平静 |
| 替代的解释句 | 替掉了哪类“她很难过”式解释 |
| 可迁移题材 | 适合迁去什么题材 |

### 10. 安静压迫场表

| 字段 | 说明 |
|------|------|
| 场面压力来源 | 谁压谁，靠什么压 |
| 谁没说话 | 沉默者是谁 |
| 环境音 / 物件 | 关门声、筷子声、手机震动等 |
| 未说破结果 | 大家都知道但没说破的后果 |

### 11. 人物偏手表

| 字段 | 说明 |
|------|------|

### 12. 失控说话表

|------|------|

### 13. 烂关系漏出表

| 谁先看见 | 谁最先被迫反应 |
| 为什么比吵架更有效 | 关系是怎么自己漏出来的 |
| 可迁移桥段 | 换壳后还能怎么用 |

### 14. 外部秩序表

| 字段 | 说明 |
|------|------|
| 秩序来源 | 医院 / 家宴 / 公司 / 民政 / 舆论 |
| 谁掌控秩序 | 护士 / 长辈 / 客户 / 律师 / 工作人员 |

### 15. 公开炸场表

| 字段 | 说明 |
|------|------|

| 关键硬牌 | 录音 / 协议 / 病历 / 证据 / 账目 |
| 谁出的牌 | 女主 / 外部人 / 事实本身 |
| 世界怎么反应 | 外部人看见了什么 |
| 炸后第一后果 | 关系或秩序怎么立刻改变 |

### 16. 后果链表

| 字段 | 说明 |
|------|------|
| 起点实锤 | 掉位 / 实锤 / 炸场从哪里开始 |
| 女主第一现实动作 | 签字 / 搬离 / 找律师 / 撤资 |
| 男主第一波反应 | 误判 / 压住 / 追错点 |
| 外部秩序怎么继续压 | 不是只靠两人拉扯 |
| 最终新状态 | 谁已不在原地 |

### 17. 原文细节库最小要求

每一类细节库至少都要回答：

1. 具体发生了什么
2. 这个细节为什么有用
3. 它压的是谁、压在哪
4. 后续能迁到什么新桥段
5. 它对应的角色 / 情绪 / 反转是什么

### 18. 写作资产最小要求

每一份写作资产至少都要回答：

1. 这个资产解决写作中的什么问题
2. 它来自哪些高频原文机制
3. 它更适合开头 / 中段 / 高潮 / 结尾哪一段
4. 它最容易写假的地方是什么
5. 写新稿时最短怎么调用

### 19. 作者DNA指纹

最少要覆盖：

1. 这个作者最稳定的句长、停顿、切句习惯
2. 叙述视角常见贴脸距离
3. 最常用的情绪落点方式
4. 最常见的动作替代表达
5. 最高频的“旧伤触发器”
6. 最容易被 AI 写丢的口气差
7. 明显不属于这个作者的空话、总结句、抒情句

额外硬要求：

8. 至少 3 组原文级证据，说明这个作者怎么切句、怎么停、怎么在动作后补一刀
9. 至少 3 组原文级证据，说明不同角色说话的口气差到底落在哪
10. 至少 2 组反面句型，明确“这种句子虽然通顺，但不像这个作者会写”

写法提醒：

- 不接受只写“文风细腻 / 情绪克制 / 擅长虐心”
- 必须尽量落到句法、停顿、动作、称呼、站位
- 能写“替代关系”最好：这个作者常用什么动作替掉什么解释句

### 20. 仿写约束_禁写清单

最少要覆盖：

1. 这个题材下最容易写出 AI 味的句式
2. 不能乱写的解释句、总结句、空情绪句
3. 不能泛化的桥段推进词
4. 不能把人物写得过于讲理、过于整齐的地方
5. 哪些位置必须落到动作、物件或选择，不能只讲感受
6. 哪些“补字数方式”是废话，必须禁止

额外硬要求：

7. 每条禁写项后面都要补一句“为什么假”
8. 至少覆盖 4 类高频坏味：空情绪、假口语、万能过渡句、补字废话
9. 至少举 2-3 组“看起来通顺但会把人味冲掉”的句式

写法提醒：

- 不要只写“少用总结句”
- 要写成“哪类总结句不能出现在什么位置，为什么一出就假”
- 最好能直接写成后续写作时可扫描的拦截词或拦截句型

### 21. 同桥段过检规则

这是专门回答“为什么同桥段原文能过、仿写容易假”的文件，最少要覆盖：

1. 该书里最值钱的 3-5 个高频桥段
2. 每个桥段原文是怎么起手的
3. 原文靠什么过检：动作、物件、旧伤、口气、秩序还是后果
4. 新稿最容易写假的点
5. 改写时必须保留的承重件
6. 绝对不能出现的 AI 过渡句或作者总结句

额外硬要求：

7. 至少 1 组“原文能过 / 仿稿会假”的正反对照
8. 要明确这个桥是靠动作、物件、旧伤、口气、秩序还是后果承重
9. 要明确“桥本身俗不俗”不是重点，重点是“怎么把俗桥写得不像加工成品”

字段化要求：

每个桥段除了写给人看的说明，还必须能稳定映射到后续 `book.profile.json` 的这 7 类字段：

1. `opening_pattern`
2. `must_keep`
3. `must_avoid`
4. `fake_signals`
5. `recommended_sequence`
6. `why_order_matters`
7. `why_original_passes`

对应口径：

- `原文怎么起手` -> `opening_pattern`
- `原文承重件 / 必须保留件` -> `must_keep`
- `绝对不能出现` -> `must_avoid`
- `最容易写假的点 / 新稿高风险假点` -> `fake_signals`
- `推荐迁移顺序 / 不能丢的顺序` -> `recommended_sequence`
- `为什么这个顺序不能乱` -> `why_order_matters`
- `原文为什么能过` -> `why_original_passes`

如果只写“承重件 + 假点”，没写起手和顺序原因，默认还没拆到可直接给 profile 吃的层。

写法提醒：

- 不要只写“这个桥很虐/很爽”
- 要写“从什么脏细节起手，靠什么拖住场面，哪一步最容易被 AI 写成成品模块”
- 如果桥段的承重件其实是秩序场或旧伤件，一定写出来，不要只盯对白

### 22. 16 张表的统一施工层要求

每张表在表格后都要继续回答 3 件事：

1. `可直接借的承重结构`
   - 这张表里哪几项是绝对不能丢的骨钉
   - 哪类内容只是表皮，哪类内容才是真承重
2. `迁移顺序提醒`
   - 这些件换到新稿时先上什么、后上什么
   - 哪一步不能抢跑，哪一步不能省略
3. `为什么这个顺序不能乱`
   - 为什么原文这样排会成立
   - 为什么仿稿最容易把这里写成 AI 成品模块

高频误区：

- 只有“优点总结”，没有施工顺序
- 只有“这桥很炸”，没有承重件
- 只有“多用动作少用总结”，没有写明具体哪一步不能乱
- 只有表名和现象，没有说明为什么相同桥在新稿里会假

## 缺库处理

- 缺 `导语拆解表`：不要直接写开头
- 缺 `顺序事件表`：不要直接重组骨架
- 缺 `对白功能表` 或 `对话衔接表`：不要硬写对白
- 缺 `微动作表`：不要只靠情绪词和解释句顶场
- 缺 `安静压迫场表`：高潮和公开场容易只会写吵
- 缺 `人物偏手表` / `烂关系漏出表` / `后果链表` 任意 1 张：高情绪关系文默认还没拆透
- 缺 `原文细节库` 任意 2 类以上：不要直接做融合或去原作化
- 缺 `写作资产` 里的 `母结构 / 主冲突 / 情绪母线` 任意 1 份：不要直接起大纲
- 缺任意 2 张以上基础表，或缺任意 1 张关键成文活层表：默认还停在概括层，先回拆文阶段补

统一状态词：

- `ready-for-write`：analyze 侧已拆到可直接仿写层，write 侧可直接开写
- `need-more-deconstruction`：还停在概括层，必须回 analyze 继续补拆
- `blocked-on-assets`：骨架可用，但表演层资产不够，最容易写出空场和假对白
- `ready-for-fusion`：已具备 16 张表 + 原文细节库 + 写作资产，可直接做融合版 / 去原作化版本

放行前再加 1 条硬检查：

- 如果 16 张表里任意一张没有 `承重结构 / 迁移顺序 / 顺序不能乱的原因`，一律不能标 `ready-for-write`

---

## 主文件中该怎么用

- `story-short-analyze`：把 16 张表 + 原文细节库 + 写作资产当作固定产出合同
- `story-short-write`：把这些文件当作固定读取合同
- 需要扩充例句、词库、反例时，优先放进独立 reference，不把主 skill 写成大词表

拆文侧延伸参考：

- `导语拆解表` 对应 [opening-deconstruction-library.md](opening-deconstruction-library.md)
