## 短篇 vs 长篇：核心差异

| 维度 | 长篇 | 短篇 |
|------|------|------|
| 篇幅 | 数十万字、数百章 | 5000-15000字、6-12节 |
| 弧线 | 多线交织、多卷结构 | 单弧线、压缩三幕 |
| 人物 | 群像、分级体系 | 精简（3-5人）、功能高度浓缩 |
| 世界观 | 复杂设定、力量体系 | 现实为主、极少特殊设定 |
| 核心驱动 | 爽点循环、升级感 | 爆点设计、反转冲击 |
| 读者粘性 | 追更驱动 | 一口气读完 |
| 分析重心 | 结构完整性、伏笔管理 | 情感设计、写作手法、节奏 |

**核心：短篇靠共鸣和爆点驱动。拆文就是看它用什么故事核、怎么铺垫、在哪里引爆。**

## 拆解思路

### 核心原则

| 原则 | 说明 |
|------|------|
| 故事核驱动 | 先找到故事的核心梗（一句话），所有分析围绕故事核展开。故事核变了，整篇的分析方向都会变 |
| 读者视角优先 | 每一步从读者体验出发——读者看到了什么？感受到了什么？为什么在意？不猜测创作者动机，只分析文本实际制造的读者体验 |
| 可复用性导向 | 每个分析点都要输出可迁移的结构、功能位或风险阈值；纯欣赏型分析不合格 |
| 爆点为中心 | 短篇拆文的核心问题是——什么让读者情绪爆发？什么让读者想转发？找到爆点，拆解它的铺垫和释放机制 |
| 共鸣决定传播 | 一篇短篇能火，不是因为写得好，是因为引发了共鸣。共鸣层次越多，传播性越强 |

### 资料集合反推原则

当输入不是一篇完整成品，而是一批原文、一组题材资料、方法论文、热梗积累或题卡合集时，拆法要切换。

这时候核心不是“整理资料”，而是反推出一套真的能拿去继续拆书的方法。

要回答的是：

1. 这批文本真正重复出现了什么，不是别人声称自己在教什么
2. 哪些规律能被多个样本反复证明
3. 哪些只是表层题材词，哪些才是底层结构动作
4. 以后再拆别的书时，应该先看什么，再看什么，再怎么验

硬要求：

- 不把资料原话直接当结论，先回样本文本验证
- 不只做知识点摘抄，必须反推成拆书动作
- 不只记名词，要落到“怎么拆、拆什么、怎么验”

还要先分清材料到底属于哪一层：

- `结构型材料`：适合反推故事走法、事件链、高潮回收
- `方法型材料`：适合反推开头钩子、情绪拆法、信息密度、自检口径
- `素材型材料`：适合回收微动作、眉眼细节、说话动态、题材局部件

素材型材料不要硬拿来证明结构规律，但一定要回收成表演层资产。

### 分析顺序（从外到内，层层递进）

```
故事核 → 结构 → 人物 → 情感线 → 写作手法 → 共鸣+爆点性+话题性 → 综合评估
  │         │       │       │          │              │                  │
  │         │       │       │          │              │                  └─ 可借鉴结构
  │         │       │       │          │              └─ 为什么能火
  │         │       │       │          └─ 怎么写出来的
  │         │       │       └─ 读者感受到了什么
  │         │       └─ 谁在推动故事
  │         └─ 怎么布局的
  └─ 写的是什么（锚定一切分析的起点）
```

### 每个阶段的核心问题

拆文不是填表，是回答问题。每个阶段有一个核心问题：

| 阶段 | 核心问题 | 回答不了说明 |
|------|----------|-------------|
| 故事核 | 一句话讲清楚这个故事卖什么？ | 还没读懂，别急着拆 |
| 结构 | 文本如何布局？开端/发展/高潮/结局各占多少？ | 布局决定了节奏，不知道布局就不知道节奏 |
| 人物 | 谁是主人公？谁是主动型角色？谁是被动型角色？ | 不知道谁在推故事，就不知道故事怎么动的 |
| 情感线 | 读者的情绪是怎么被一步步拉扯的？ | 这是短篇的核心价值，拆不出来等于白拆 |
| 写作手法 | 文本用了什么手法让读者产生这种体验？ | 手法必须被拆成可复用功能位，否则无法迁移 |
| 共鸣+爆点 | 为什么这篇能火？触发了什么共鸣？ | 不回答这个问题，拆文就变成了文学鉴赏 |
| 综合评估 | 哪些结构可复用？复用条件和风险是什么？ | 不能指导下一份 artifact 的拆文没有意义 |

### 拆解约束

1. **禁止主观评价** — 不评价好坏，只拆解机制。输出中不得出现"写得好/写得差"类评价
2. **从体验反推手法** — 先定位读者情绪高峰，再反推文本用什么手法制造了这个高峰
3. **深度优先于全面** — 核心维度（故事核/爆点/共鸣）必须深入，可选维度按需
4. **可复用性导向** — 每个分析点必须输出复用条件、可迁移做法和风险边界

## 资料集合拆法

### 什么时候按这套走

出现下面任一情况，就别只做单篇拆文，要切到这套：

- 用户给的是一批原文或一组题材材料
- 材料里混着原文、拆文笔记、热梗积累、起盘方法
- 用户目标不是拆这一篇，而是反推出“以后怎么拆”

### 第一步：先分材料，不要混读

先把材料分成四类：

- 原文样本：真正的成品文本、可验证桥段
- 总结话术：别人归纳出来的规则、公式、口号
- 题材素材：热梗、题卡、导火索、场景清单
- 方法草图：别人教你怎么开头、怎么起盘、怎么融合

拆的时候，优先级固定：

1. 先看原文样本
2. 再拿总结话术去原文里找证据
3. 再看题材素材到底在原文里怎么被使用
4. 最后才反推出方法

### 第二步：从原文里反推 5 件事

不是先做分类，而是先回答这 5 个问题：

1. 开头怎么抓人
2. 题面和导语为什么一眼就能让人点开
3. 情绪为什么不是一句“她很难受”，而是一步步把人拖进去
4. 人物为什么一开口就不像同一个人在说话
5. 结尾为什么能让人读完还挂着

每个问题都要回到原文里找重复动作，而不是抄总结句。

中段、高潮、结构这些，再往下继续拆。

反推时也要顺手记“哪些是开头禁区、正文禁区”。

例如：

- 开头还在解释，没起事
- 第一屏只有背景，没有后果
- 正文连续解释，不给动作
- 回忆很多，但不改现在这场局
- 同类伤害重复，没有升级

这些不是单篇问题，往往就是后面写作最该提前避开的坑。

### 第三步：把表层题材词拆成底层结构

不要满足于“这是追妻”“这是家庭反杀”。

必须继续往下拆成：

- 最先把读者拽进去的事是什么
- 中段怎么再加一层更麻烦的事
- 主角什么时候开始不再纯挨打
- 高潮为什么一定要放在这个场合
- 结尾为什么不只是处刑

### 第四步：按 4 条线拆

#### 1. 结构线

看它最常见的故事走法：

- 开头先给什么
- 中段靠什么升级
- 高潮怎么兑现
- 结尾怎么回收

还要继续追：

- 它是不是靠连续 4 到 6 个核心事件往前推
- 中段有没有第二层升级，不只是重复拉扯
- 爽点是单次打脸，还是整局回收

#### 2. 事件线

看它怎么一步一步往前推：

- 导火索
- 越界
- 升级
- 切割
- 当众摊开的场面
- 关键证据
- 高潮后有没有再往上抬一下
- 事情结束后还留下了什么感觉

拆事件线时，优先看这些高频承重件：

- 主角是不是先忍、先退、先配合，等对方把局做大
- 真正翻盘靠的是不是一张能落地的牌
- 高潮是不是放在对方最怕丢脸的公开场

#### 3. 情绪线

情绪一律按“主角当前处境带给读者什么感觉”来拆：

- 先替她委屈
- 还是先替她发冷
- 还是先替她丢脸
- 哪一步变成疼
- 哪一步变成等她反击
- 哪一步变成掌控感

注意：

- 不要只写“难受、绝望、崩溃”这种抽象词
- 要追她当下落到了哪里：眼神、嘴角、手、呼吸、站位、说话方式、动作选择
- 同一种受气连着写两三次，不叫递进，叫打转

再补一句总口径：

- 这里的“情绪”不是人物内心独白有多满，而是文本先把什么感觉砸给读者

所以拆的时候别急着写：

- 难过
- 痛苦
- 崩溃

先写：

1. 谁做了什么越界
2. 主角当下被逼到了什么位置
3. 读者最先替她感觉到什么
4. 这一节比上一节更难受在哪

如果第四条答不出来，说明这一节大概率只是在重复受气，不是在升级。

#### 3.0 情绪拆法的最小模板

按原文去拆时，不要只写：

- 这里虐
- 那里爽

至少要拆成这三层：

1. `带虐点的剧情`
   是具体发生了什么
2. `虐点内核`
   这件事真正扎人的地方是什么
3. `情绪结果`
   这件事会让主角和读者先感觉到什么

示意：

- 带虐点的剧情：他表面温柔，背地里却想让她和孩子一起消失
- 虐点内核：温柔面具下是毫无人性的狠毒
- 情绪结果：绝望、心寒、发冷

这样拆，后面才能把别人的情节换成自己的情节，只借“扎人的内核”和“情绪走法”。

#### 3.0.2 情绪阶段怎么验

拆完每个关键节点，再给它落一个阶段标签：

- `受气`
- `看清`
- `抽身`
- `反击`
- `余波`

不是每篇都五个都重，但大多数短篇都会经过其中 3 到 5 个。

验法：

1. 这一节点如果删掉，主角是不是少走了一步
2. 这一节点之后，关系位置有没有更偏
3. 这一节点有没有把后面的情绪准备好

如果一个节点既没有阶段变化，也没有关系变化，多半只是热闹，不是承重点。

#### 3.0.1 情绪浓度怎么分

原文里能稳定复用的，大致是三档：

- `一般情绪`：失望、意外、无奈、心累、遗憾
- `中等情绪`：痛苦、羞辱、无助、绝望、愤怒、心寒、死心
- `强烈触点`：践踏尊严、栽赃嫁祸、谋财害命、精神控制、众叛亲离、致残害命

拆的时候要看两件事：

1. 这篇前段主要压在哪一档
2. 付费点或高潮前，有没有把情绪再往上推一档

如果开头已经用了强烈触点，后文却一直掉回普通吵架和普通委屈，说明情绪浓度塌了。

#### 3.1 人物口气线

除了情绪线，还要补一条“人物怎么说话”的线。

至少拆这 5 件事：

- 常用称呼：对谁怎么叫
- 句子长短：是短句顶人，还是绕着说
- 常见口头习惯：会不会先讽一句、先装没事、先找补
- 情绪上头时会怎么变：更冷、更碎、更冲，还是突然不说了
- 不会说什么：哪些书面词、解释腔、体面话不属于这个人

如果拆完以后，几个主要角色的口气还能互换，说明没拆到位。

#### 3.1.1 人物反应线

除了口气，还要拆“这个人遇事怎么反应”。

至少看这 4 种时刻：

1. 受伤时怎么反应
2. 动心时怎么反应
3. 被误解时怎么反应
4. 失去时怎么反应

不要只记人设标签，要记稳定反应模式。

#### 3.2 追妻题材要额外追一条线

如果是追妻题材，别只拆女主受了多少伤，还要专门追男主那条感情线：

1. 旧情基础在哪
2. 关系是被什么误会或旧伤砸坏的
3. 他从哪一步开始一次次站错边
4. 中间有没有“差点看明白”的机会
5. 真相到了以后，他第一反应认错了什么
6. 最后他是怎么发现已经追不回来的

这条线拆不出来，追妻文就只剩表面流程。

还要再验一句：

- 最后的最大爽点，是“他终于表白”，还是“她已经不需要了”

后者通常更值钱。

#### 3.3 反直觉线

有些文不新，不是因为题材老，而是每一步都太像常规答案。

所以还要专门追：

1. 读者第一眼会以为什么
2. 真正值钱的东西是不是藏在第二层
3. 主角有没有做出不按常规来的关键选择
4. 结尾有没有改掉读者原本的默认预期

反直觉不是乱拧，必须前面有铺垫、后面能回收。

#### 4. 素材线

看高频重复出现的承重件，而不是杂项：

- 导火索
- 关系场
- 让事情变得更麻烦的那层事
- 当众摊开的场面
- 关键证据
- 切割动作
- 结尾落点

具体素材条目底库见 `short-story-material-bank.md`。拆资料时，如果已经确认某类素材在样本里反复出现，就按那份底库统一命名、统一归类，别每次临时起名。

素材线还要继续往下拆，别只停在“这篇用了哪些梗”。

至少再追这 4 层：

1. `表演层`
   微动作、眉眼、身体反应、哭笑、停顿、称呼变化
2. `对话层`
   说话动态、句长变化、口头习惯、情绪上头后的失真方式
3. `题材局部件`
   古言称谓、自称、官职、店铺名、法器、阵法、功法、器物、服饰
4. `接场层`
   转场句、留问号的句子、二次回看会变意思的前文细节

拆这层时要记住：

- 表演层资产是拿来替代解释句的，不是拿来堆华丽词
- 对话层资产是拿来区分人物声道的，不是拿来做词库展示
- 题材局部件要看它有没有参与对抗、掉位、翻盘
- 接场层要看它是不是把上一场的劲带进下一场

#### 4.1 表演层怎么拆

不要只记“这里描写细腻”，直接拆成：

1. 这段主情绪是什么
2. 作者先给了哪个身体或表情细节
3. 这个细节暴露了什么关系位置
4. 后面有没有再补解释句，还是就靠动作顶住

重点优先看：

- 手：攥、松、抖、停、藏、移开
- 眼：看、躲、停、冷、红、发空、笑意不达眼底
- 嘴：抿、张口又停、咬字变硬、声音发虚
- 呼吸和站位：靠近、后撤、僵住、贴墙、站不稳
- 哭笑：忍、压、失控前一瞬、笑不对眼

如果一段里细节很多，但换掉也不影响关系和情绪，那些大多是装饰，不算承重。

#### 4.2 对话层怎么拆

对话别只统计占比，要看“怎么说”。

至少拆这 5 件事：

1. 常用称呼怎么变
2. 句子平时多长
3. 急了以后变短、变碎、变冲，还是反过来更轻
4. 常见护脸方式是什么
5. 哪些书面解释句不属于这个人

再补一条：

- 关键场里，对话结束后，关系位置有没有立刻变化

如果一段对话说了很多，但谁都能替谁说，说明声音没拆出来。

#### 4.3 题材局部件怎么拆

古言、权谋、玄幻这些题材，不能只记有多少名词，要拆“这些名词有没有用”。

优先追：

- 称谓变化有没有带出尊卑、亲疏、翻脸
- 官职、家世、门派、势力名有没有真实影响谁能压谁
- 店铺、宅院、法器、阵法、器物是不是承接了会面、做局、掉马、切割
- 服饰、妆容、佩饰是不是只管好看，还是顺手带出身份和处境

拆完后最好能回答：

- 这个题材件删掉，剧情还成立吗
- 它有没有把冲突、身份、危险抬高
- 它后面有没有再被回收

#### 4.4 转场、伏笔、悬念怎么拆

拆转场，不是看“有没有切场”，而是看：

1. 上一场剩下的情绪有没有被带过去
2. 下一场是不是接着前一场的后果，而不是硬开新场
3. 有没有用物件、称呼、动作、半句话把两场粘起来

拆伏笔和悬念时，至少列这三次：

- 第一次出现时，读者以为它是什么意思
- 第二次出现时，意味有没有变重
- 最后回收时，前面两次有没有被重新解释

### 第五步：专门验“为什么不套路”

这一步不能省。

每批材料都要追这 6 个问题：

1. 这篇表面上是什么题材
2. 真正让它不普通的地方是什么
3. 中段又加了哪一层更麻烦的事
4. 主角哪一步不是标准反应
5. 哪个高潮回头改写了前文理解
6. 结尾除了处刑，还有没有位置变化、价值抬高、迟到惩罚

少了这一步，最后很容易只学到“顺”，学不到“狠”。

还要再补一个“为什么没拼盘”的检查：

1. 母结构是不是只有一个主骨架
2. 中段那层新事是不是确实抬高了后果
3. 表演层和题材件是不是都在给主骨架服务
4. 有没有只是好看、其实删了也不影响推进的花件

如果一篇材料看起来什么都有，但删掉一半设定也还能照走，那些大概率只是装饰，不是值钱点。

### 第五步半：顺手反推“为什么会被编辑打回”

如果用户后面要拿这些规律去写，不只要说它哪里好用，还要能反推常见不过稿点。

最常见的三类：

1. `套路化`
   重点查：
   - 有没有第二层更麻烦的事
   - 有没有副升级器
   - 高潮有没有二次翻一下，不只是把真相讲出来
2. `张力不足`
   重点查：
   - 开头有没有后果预期
   - 中段有没有连续升级
   - 高潮前有没有卡着读者往下等
3. `文笔一般`
   重点查：
   - 是不是解释句太多
   - 是不是人物口气没分开
   - 是不是只有句子顺，没有动作、关系、情绪承重

这一步最好单独写出来，方便后面直接拿去做回炉单。

### 第六步：最后才收成拆书方法

前面都走完后，最后收口时，应该能得到这套可复用拆法：

- 先看开头钩子的真实作用
- 再看故事怎么走，以及中段怎么再加事
- 再看事件链怎么防平
- 再看情绪怎么持续加压
- 再看高潮后有没有再往上抬一下
- 再看结尾有没有把整篇文抬高

这才叫从资料里反推出拆书方法。

最后收口时，最好再顺手给出一条“调用顺序”。

不是只说有哪些素材，而是告诉后面写的时候该怎么先后用：

1. 先拿母结构和事件链
2. 再拿中段那层更麻烦的事
3. 再拿公开场和关键证据
4. 再拿情绪阶段和人物口气
5. 最后才拿微动作、眉眼、题材局部件

这样后面写的人不容易把骨架件和后补件用反。

### 第六点五步：如果后面要仿写，必须补到“可直接仿写层”

很多拆文看起来信息很多，但后面一写还是发飘，问题通常不是分析不够长，而是没拆到能直接用。

如果这批拆文后面是为了：

- 仿写
- 融合
- 去原作化后再写

那在收口前必须再补 10 张表：

1. `导语拆解表`
   - 前 20 / 60 / 80 字分别完成了什么
   - 第一刀怎么扎，第二句怎么加码
2. `顺序事件表`
   - 顺着列清楚每一拍谁做了什么
   - 不能只写“中段升级”
3. `物件表`
   - 关键物件伤的是哪一层，后面还能换成什么
4. `动作表`
   - 谁抢、谁扔、谁拦、谁护、谁走、谁留证
5. `对白功能表`
   - 这类话在文里负责挑衅、找补、压人，还是切割关系
6. `对话衔接表`
   - 上句怎么逼，下句怎么接，哪里用动作垫，哪里用称呼变化
7. `误判表`
   - 读者 / 角色先被误导到哪一步，后来怎么翻
8. `钩子表`
   - 每一段末尾到底留了什么，读者下一步在等什么
9. `微动作表`
   - 情绪落在哪个小动作上，替掉了哪类解释句
10. `安静压迫场表`
   - 哪些场面不靠大吵，照样能把人压住

只要这 10 张表没出来，就说明拆文还停在概括层，不算真正拆到能直接仿。

如果用户还要做平台适配，再顺手补一条：

11. 最后再看这套东西更像哪个平台口气

主要看：

- 开头快不快
- 情绪是更直，还是更有后劲
- 高潮是更偏公开翻车，还是更偏价值回收
- 结尾是更狠，还是更留余味

## 拆解流程 5 阶段

> **编号说明**：Phase 1（确认拆解对象+题材路由）是 SKILL.md 处理的顶层工作流阶段。本文件描述拆解管道的 Stage 2-6。

### Stage 2：结构分析 + 情节节点提取

- 识别章节/小节分隔符（###01、第一章、01 等）
- 提取标题，记录每节字数
- 提取**故事核**（分三层概括）：
  - **设定**：故事的前提/触发条件（如"重生三次被母弟杀害"）
  - **主题**：核心矛盾/价值观冲突（如"极端重男轻女"）
  - **核心行动**：主角做了什么（如"放任母弟自毁"）
  - 合并为一句：如"重生三次被母弟杀害的女孩，第四世放任她们自毁"
- 写**故事梗概**（200-500字概括）
- 识别 POV（第一/第三人称、全知视角）
- 识别叙事时间线（线性/插叙/倒叙/双线交叉）
- 将全文按功能划分为 4-6 段（必须包含开端/发展/高潮/结局，可酌情增加转折/过渡段），标注每段字数范围和占比

> **灵活分节**：部分故事只有2-3章但内含数字编号小节，此时"节"应以叙事功能单元为准，而非物理章节。章节过少时以小节/编号段为分析单位。

#### B. 情节节点提取

情节节点 = 推动故事发展的关键事件或信息释放。

密度原则：每 200-300 字一个节点，每节 3-8 个。全文目标节点数按字数分档，以下表为唯一权威：

| 全文字数 | 目标节点数 | 说明 |
|----------|-----------|------|
| ≤5000 | 15-25 个 | 精短型，每个节点必须有效 |
| 5000-8000 | 25-40 个 | 标准型 |
| 8000-12000 | 35-55 个 | 中长型 |
| >12000 | 45-60 个 | 长短篇（上限） |

每个节点提取：

| 字段 | 说明 |
|------|------|
| 序号 | 严格时间顺序 |
| 类型 | 情绪节点/信息节点/冲突节点/转折节点/对话节点/氛围节点 |
| 描述 | 客观白描，只记录发生了什么（禁止叙事框架词） |
| 原文引用 | ≤300 字直接引用 |
| 情绪标记 | 情绪类型 + 强度（-9 到 +9） |
| 涉及人物 | 全名 |
| 手法标记 | 使用了什么手法（如有时） |

### Stage 3：情感线 + 爆点分析

#### A. 情感曲线

从情节节点数据生成情感曲线。节点数 = 至少 max(5, 节数) 个，确保每节至少一个情绪节点。

曲线特征提取：
- 起始情绪
- 走向模式（上行/下行/波浪/V形/倒V/阶梯/断崖/压缩弹簧）
- 极值（最高点、最低点及其位置百分比）
- 方向差（起始到结尾的净变化）
- 翻转次数（正负交替次数）

#### B. 爆点分析

分析故事的情感爆发设计：

| 维度 | 问题 |
|------|------|
| 铺垫 | 什么让读者开始在意？前多少字完成？ |
| 积累 | 什么在积累情绪势能？积累多久？ |
| 延迟 | 有没有延迟释放？信息差/误解/悬念？ |
| 爆发点 | 哪个瞬间释放全部情绪？精确到句子 |
| 余波 | 释放后有没有余震？多个角色反应？ |
| 印象 | 情绪释放后留下什么？读者记住什么？ |

**叠加爆点**：有些故事的爆点是分层的——表层爆点（如死亡/分离）刚引爆，深层爆点（如真相揭示/认知颠覆）在前一个余波中二次爆发。分析时需标注：
- 是否存在叠加爆点？
- 表层爆点和深层爆点分别是什么？
- 叠加效果（1+1>2 的增效在哪里？）

**多爆点串联**：部分故事（豪门复仇、舆论反转）有多个独立爆点逐个引爆，每个爆点自成一体但互相呼应。分析时需标注：
- 有几个独立爆点？分别是什么？
- 爆点之间是递进（一个比一个爽）还是并列（不同维度的满足）？
- 爆点间隔是否合理（间隔太远=情绪断裂，太近=疲劳）？

#### C. 期待感分析

逐节追踪读者的期待状态：

| 节 | 制造的期待 | 满足/升级/悬置 | 剩余期待 |
|----|-----------|---------------|---------|
| 1 | {期待A} | {制造} | {A} |
| 2 | {期待B} | {升级A} | {A+, B} |
| ... | | | |

规则：不超过两节不制造新期待，不超过一节不推进核心期待。

### Stage 4：反转设计 + 写作手法

#### A. 反转分析

**前置反转检查**：是否有在故事时间线之前就已存在的谎言/误判。

> **无反转型/报应型**：有些故事（复仇爽文、重生文）没有传统反转，爽感来自看反派因果报应逐步兑现。遇到此类故事，跳过反转机制拆解，改为分析"报应设计"——反派如何一步步走向毁灭，每一步的铺垫和满足感。使用报应设计表：每条恶行→对应的报应→爽感来源。理想情况下每条报应与恶行形成精确对应（"以彼之道还施彼身"）。
>
> **主动报应 vs 反派自毁**：报应设计分两种——①主角设局让反派自爆（如香水复仇、比赛设局）；②主角不复仇，反派内部反目自己作死。第二种更真实但爽感较低，分析时标注类型。

**反转机制拆解**（存在反转时执行）：
- 铺垫线索（≥2条，含原文位置）
- 误导方向（文本把读者往哪引）
- 真相揭示（怎么揭开的）
- 合理性（回看得不失真）
- 时机（全文百分之几位置）
- 铺放比（铺垫字数 : 释放字数）

**反转链**：部分故事（舆论反转文、套娃反转文）包含多个连续反转，前一个反转的结论被后一个反转推翻。分析时需按顺序列出每个反转：
- 每个反转：类型 + 触发事件 + 读者认知变化
- 反转之间的因果关系（前一个反转如何为后一个反转铺垫）
- 反转链的整体效果（递进/叠加/出其不意）

**套娃反转的质量检验**：套娃反转好不好，看反转之间有没有因果关系——反转A必须解释或加深反转B，而不是随便堆砌。检验方法：删掉任意一层反转，其他反转还成立吗？如果成立，说明反转之间没有因果关联，只是堆砌。

**伏笔式反转**：部分反转不是通过新信息揭示，而是开头早就给过的信息，读者知道真相后回头看含义完全变了。这就是伏笔的经典用法——不铺新线索，只是让读者自己"恍然大悟"。分析时标注：伏笔原文位置、初读时读者的理解、知道真相后的新理解。

#### B. 写作手法分析

逐项分析以下维度的使用情况：

| 维度 | 分析内容 |
|------|----------|
| POV策略 | 人称选择的效果、叙事距离、可靠性 |
| 对话手法 | 密度比、潜台词率、模式识别 |
| 时间操控 | 时间跳跃、场景压缩、平行剪辑 |
| 信息控制 | 读者知道 vs 角色知道、隐瞒/揭示时机 |
| 感官锚定 | 五感分布、主导感官、感官切换 |
| 句式节奏 | 长短句交替、节奏加速/减速、关键句位置 |
| 对比设计 | 角色对比、前后对比、环境对比 |
| 意象/物件 | 重复出现的物件、意象、象征 |
| 留白手法 | 不写之写、省略的艺术、暗示 |
| 专业技能武器化 | 主角用职业特长作为复仇/对抗工具（调香师用香水、AI工程师用AI、律师用法律）。职业技能越专业，手段越可信越有新意 |
| 倒计时框架 | 用明确时限（10天/5小时/1分钟）制造紧迫感。每一段倒计时都是情感催化剂，读者和主角一起倒数 |
| 双视角叙事 | 同一事件从两个角色视角分别体验。先跟A感受痛苦，再跟B感受悔恨。读者情感判断被迫修正，无法简单站队 |
| 弹幕/评论互动 | 故事内的"读者评论"成为角色——提供任务、吐槽、剧情预告。弹幕本身可以有变化弧线（从自信到崩溃） |
| 反差萌 | 角色想装一种人却总是暴露本性，每次"装失败"都让对方更喜欢。注意识别重复升级的模式 |

每项手法标注：使用位置、效果评价、可复用条件。

### Stage 5：人物分析 + 开头结尾

#### A. 人物提取

短篇人物精简（通常 3-5 人），但每个人物承担多重功能。

**人物分类**（二维体系）：

维度一 — 叙事角色（这人在故事里的位置）：
- **主人公**：故事的第一视角人物或核心焦点人物
- **重要配角**：戏份重、有独立弧线的人物
- **功能人物**：承担特定叙事功能（催化剂/对照组等）

维度二 — 行动角色（这人在故事里怎么动）：
- **主动型**：主动推动剧情发展、做出关键选择
- **被动型**：被剧情推着走、被动承受
- **转变型**：从被动转为主动（或反过来）

> 同一人物可以同时是"主人公"+"被动型"（如前期的乔青），也可以从"被动型"转变为"主动型"。两个维度独立标注。

**双主人公结构**：部分故事（BE虐文、双向视角文）存在两个主人公，各自驱动故事的不同阶段。识别标准：①两人都有完整的情感弧线（不是扁平的）②故事分为明显的"A段"和"B段"，各自以不同角色为中心③读者对两人都有强烈的共情。标注时两人都标为"主人公"，并注明各自主驱动的章节范围。

| 字段 | 说明 |
|------|------|
| 姓名 | 全名 |
| 分类 | 叙事角色{主人公/重要配角/功能人物} + 行动角色{主动型/被动型/转变型} |
| 人物性格 | 从行为归纳，不从标签推导 |
| 内在矛盾 | 核心心理冲突（不是外部遭遇） |
| 关键台词 | 最能代表该角色的 1-2 句话 |
| 弧线 | 始→转→终（标注触发事件和位置）。无明显弧线标注"扁平" |
| 功能标签 | 在故事中承担什么叙事作用 |

**人物功能标签**（7 种）：

| 标签 | 定义 | 示例 |
|------|------|------|
| 情绪承载者 | 承载读者共情的主要载体 | 第一人称主角 |
| 压迫源 | 制造负面情绪的角色 | 冷漠/控制的男主 |
| 反转触发器 | 触发关键反转的角色 | 绑架事件中的第三方 |
| 道德锚点 | 代表正常价值观的角色 | 关心/担忧主角的长辈 |
| 对照组 | 与主角形成对比的角色 | 温和的配角 vs 冷酷的主角 |
| 催化剂 | 推动剧情但不直接参与核心冲突 | 连接/温暖型角色 |
| 自毁型 | 自身行为导致自我毁灭的角色 | 重男轻女的母亲、偏执的家属 |

一个人物可以有多个标签。

#### A2. 人物关系

从情节节点中提取人物间的关系和演变：

| 字段 | 说明 |
|------|------|
| 关系对 | A ↔ B（全名） |
| 关系本质 | 一句话概括关系核心（如：单向依附→对等拒绝） |
| 演变轨迹 | 节N:状态A → 节M:状态B，标注触发事件 |
| 情绪贡献 | 这段关系对情感线的贡献 |

提取策略：
- 只提取有直接互动或明确情感描述的人物对
- 同一对人物保留最终状态，历史变化写入演变轨迹
- 关注关系转折的精确位置和触发原因

#### B. 开头分析

- 前 3 句原文引用
- 前 50 字分析：有没有冲突/异常？
- 前 100 字分析：读者是否知道核心矛盾？
- 钩子类型：悬念/冲突/反差/代入/信息差
- 开头情绪强度（1-10）
- 代入感评估：强（立刻在意主角）/ 中（需几段投入）/ 弱（缺乏共鸣点）
- 信息密度评估：高/中/低
- 声音辨识度：强（一句话就能辨认是谁在说话）/ 中（有特征但不够突出）/ 弱（角色声音同质化）

#### C. 结尾分析

- 最后一段原文引用或概括
- 结尾类型：HE满足/BE遗憾/开放式/反转余韵/留白
- 结尾情绪强度（1-10，参考标准：虐文≥8、爽文≥7、治愈文≥6、通透满足≥5、震惊反转≥7、**BE意难平≥8** 为合格）
- 情绪落点（读者离开时在想什么）
- 余韵设计
- 传播欲（读者会不会推荐）
- 收束完整性（所有钩子是否回收）
- 价值观传达：这篇故事最终想说什么？（如"爱情只是调味剂，不是必需品"）

#### D. 首尾呼应

对比开头和结尾的元素呼应：

| 呼应元素 | 开头位置 | 结尾位置 | 呼应方式 | 效果 |
|----------|----------|----------|----------|------|

**重读发现**：部分故事的开头包含特殊的伏笔——开头的某句话在初读时是一种含义，读完真相后回头再看含义完全变了。分析时标注：哪些开头语句有这种效果？真相揭示后含义如何变化？

#### E. 钩子回收检查

列出全文埋设的所有钩子及其回收状态：

| 钩子 | 埋设位置 | 类型 | 回收位置 | 回收方式 | 状态 |
|------|----------|------|----------|----------|------|

未回收的钩子标注为「留白」或「遗漏」。

### Stage 6：综合评估报告

综合所有阶段数据，输出评估报告：
- 故事核评价
- 五维评分（开头吸引力、情感拉扯力、反转设计、节奏控制、结尾余韵）
  > **甜宠/喜剧类替代维度**：当故事类型为甜宠/喜剧/反差萌时，"反转设计"维度天然低分（此类故事通常无反转），可替换为"反差萌浓度"，"情感拉扯力"替换为"甜度曲线"。替代维度：开头吸引力、甜度曲线、反差萌浓度、节奏控制、结尾余韵
- 爆点性 + 话题性
- 共鸣层次分析
- 可复用结构（至少 3 条，含适用场景）
- 同类型写作动作

## 情节节点提取规则

### 节点类型（6 种，不得自创）

| 类型 | 定义 | 识别特征 |
|------|------|----------|
| 情绪节点 | 角色内在情绪发生明显转变 | 情绪词、行为变化、内心独白 |
| 信息节点 | 新信息被释放（读者或角色获得新认知） | 揭示、发现、被告知 |
| 冲突节点 | 人物间产生正面对抗 | 直接对抗、拒绝、威胁 |
| 转折节点 | 故事方向发生偏转 | 剧情方向改变 |
| 对话节点 | 包含关键信息的对话或潜台词 | 传递新信息或改变关系 |
| 氛围节点 | 环境/氛围的重大转变 | 场景切换、时间跳跃、季节变化 |

### 质量要求

1. **客观白描**：只记录"发生了什么"，禁止主观评价
   - 禁止：「沈暮月心碎地听到了真相」
   - 正确：「沈暮月听到"私生子而已"，手停在半空」
2. **绝对时序**：严格按文本中事件出现顺序排列
3. **情绪标记精确**：每个节点标注情绪类型和强度值
4. **信息保真**：不遗漏改变上下文的关键细节
5. **高度浓缩**：一个节点一句话概括
6. **复合合并**：为同一情绪/叙事目的服务的连续微动作合并为一个节点

### 白描 vs 评价

- 禁止：「通过这段对话展现了她的坚强」
- 正确：「她直视霍庭煜说"我没有义务配合你"」
- 禁止：「气氛变得紧张」
- 正确：「霍庭煜眼神骤冷，周身散发出威压」

## 写作手法分析

### POV 策略

| POV 类型 | 效果 | 适用情绪 |
|----------|------|----------|
| 第一人称 | 亲密感、代入感强、可操控信息 | 虐恋、自省、内心戏 |
| 第三人称限制 | 兼顾代入和观察距离 | 通用 |
| 第三人称全知 | 上帝视角、多线并进 | 群像、悬疑 |
| 第二人称 | 极度沉浸 | 实验性、心理恐怖 |
| 非人视角 | 新奇感、陌生化 | 系统/鬼/动物视角 |

分析要点：
- 选择这个 POV 的好处和代价
- 有没有 POV 内的信息操控（如不可靠叙述者）
- POV 切换（如有）的时机和效果
- **代理人问题**：当POV角色无法直接推动剧情时（如灵魂旁观、昏迷旁观），需依赖"活人代理人"来执行行动。代理人的出场时机和行动力直接影响剧情可信度

### 对话手法

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| 对话占比 | 对话字数 / 全文字数 | 短篇标准 45-65% |
| 潜台词率 | 含潜台词对话条数 / 总对话条数 | 高手 > 60% |
| 审判式对话 | 主角提问→对方自爆的数量 | 至少 2 处 |
| 信息差对话 | 读者知道但角色不知道的对话 | 虐文/反转文核心 |

**对话模式识别**：
- 审判式：主角提问 → 对方被迫自爆
- 压制式：短句碾压 → 对方歇斯底里
- 信息差式：双方信息不对称，读者在中间
- 推拉式：靠近又推开，反复循环
- 称呼变化追踪：不同角色对同一人用不同称呼（如"阿虎"vs"林川"），称呼的变化反映了关系和身份认同的变化。分析时追踪称呼变化轨迹，标注转变的触发事件

### 时间操控

| 手法 | 定义 | 识别特征 |
|------|------|----------|
| 时间跳跃 | 跳过一段时间 | 「三年后」「五年后」 |
| 场景压缩 | 用几句话概括长时间 | 概述性段落 |
| 平行剪辑 | 两条时间线交替 | A线/B线交替出现 |
| 倒叙 | 先说结果再说过程 | 结果先行，回溯原因 |
| 闪回 | 穿插回忆片段 | 突然插入过去场景 |
| 实时展开 | 逐秒展开关键时刻 | 关键场景慢动作 |

### 信息控制

| 时刻 | 读者知道的 | 角色A知道的 | 角色B知道的 | 信息差 |
|------|-----------|-----------|-----------|--------|
| 开头 | | | | |
| 反转前 | | | | |
| 反转后 | | | | |

### 意象/物件追踪

| 物件/意象 | 出现位置 | 每次承载的含义 | 演变轨迹 |
|-----------|---------|---------------|---------|

如无重复物件/意象，注明"无明显意象重复"。

**主题意象群**：当多个物件/意象都指向同一个主题时，不应单独分析每个物件，而应识别它们的关联。如拳套（身份错位）+纹身（被迫标记）+储物间（精神囚禁）都指向"身份囚禁"这个主题。分析时标注：共同主题、各物件的角度、整体效果。

### 留白手法

识别文本刻意留白的内容：
- 省略的背景（留给读者想象）
- 未说出口的话（对话中的停顿/省略号）
- 跳过的过程（直接给结果）
- 留白的结尾（不交代后续）

## 情感线分析

### 情绪类型

| 类型 | 信号词 |
|------|--------|
| 心疼 | 行为细节、隐忍描写 |
| 愤怒 | 反派行为、不公待遇 |
| 震惊 | 反转、真相揭示 |
| 满足 | 爽点释放、恶人受罚 |
| 紧张 | 悬念、危机 |
| 心酸 | 放手、遗憾、成长 |
| 治愈 | 关怀、团聚、被理解 |
| 压抑 | 委曲求全、隐忍 |

### 情绪强度量表

| 值 | 对应 | 描述 |
|----|------|------|
| +9 | 极致爽感 | 打脸、碾压、终极反转、恶人伏法 |
| +7 | 强烈满足 | 关键证据揭露、期待满足 |
| +5 | 明显正面 | 小胜利、温情时刻 |
| +3 | 轻微正面 | 小进展、日常温馨 |
| 0 | 中性 | 过渡、铺垫 |
| -3 | 轻微不适 | 小挫折、小委屈 |
| -5 | 明显负面 | 被羞辱、被误解、被威胁 |
| -7 | 强烈虐感 | 被抛弃、被背叛、孩子受委屈 |
| -9 | 极致虐感 | 生死威胁、灵魂碾压、信仰崩塌 |

> **量表说明**：本量表用于情感曲线，区分方向（虐/爽）和强度，范围 -9~+9（仅使用奇数值：±9, ±7, ±5, ±3 和 0）。开头/结尾分析使用独立的 1-10 绝对强度量表（只看强弱，不看正负）。

### 情感曲线形态

| 形态 | 特征 | 适用题材 | 典型效果 |
|------|------|----------|----------|
| V形 | 先虐后爽 | 打脸/复仇 | 极致反差 |
| 倒V | 先甜后虐 | BE虐文 | 遗憾意难平 |
| 波浪 | 正负交替 | 追妻火葬场 | 情绪拉扯 |
| 阶梯上升 | 逐步升级 | 爽文/打脸 | 节奏感强 |
| 断崖下跌 | 突然反转 | 死人文学/反转 | 震惊 |
| 压缩弹簧 | 长期压抑后一次释放 | 隐忍布局 | 极致释放 |
| 双峰 | 两个高潮 | 复杂短篇 | 层次丰富。两个高潮可以都是虐的（如两次告别），也可以是不同性质的（先解脱后通透） |
| **混合型** | 上述形态的组合 | 视组合而定 | 需标注主要形态和次要形态。甜宠/喜剧类可能全程正値不经历负值，属于阶梯上升的变体 |

## 节奏分析

### 节奏指标

| 维度 | 计算方式 | 标准范围 | 说明 |
|------|----------|----------|------|
| 事件密度 | 节点数 / 全文字数 × 1000 | 3-6 个/千字 | 太低=水，太高=赶 |
| 对话密度 | 对话字数 / 全文字数 × 100% | 45-65% | 短篇偏对话驱动 |
| 冲突密度 | 冲突节点数 / 总节点数 × 100% | 30-50% | 低于20%=太平，高于60%=疲劳 |
| 信息密度 | 信息节点数 / 总节点数 × 100% | 20-35% | 太低=无聊，太高=消化不及 |

### 节奏异常检测

| 异常模式 | 检测条件 | 问题 |
|----------|----------|------|
| 节奏塌陷 | 连续2节冲突密度<15% | 读者可能弃文 |
| 冲突过载 | 连续3节冲突密度>60% | 情绪疲劳 |
| 信息洪峰 | 单节信息节点>50% | 读者消化不及 |
| 对话荒漠 | 连续2节对话<20% | 文本偏干 |
| 对话泛滥 | 单节对话>80% | 碎片化 |
| 反转密度过载 | 连续2节内出现≥3层反转 | 读者消化不及，震惊感递减（每层反转间隔建议≥800字） |

## 人物提取规则

### 提取标准（短篇特化）

短篇人物精简，但必须提取所有有名有姓或有叙事功能的人物。

**必须提取**：
- 有名字（≥2字符）且有台词或推动剧情的人物
- 虽无明确名字但反复出现（≥2节）且承担叙事功能的人物

**不提取**：
- 群体称呼（"旁人""大家"）
- 无名路人（"一个中年人""助理"——除非有关键台词）
- 通用称呼黑名单：同长篇 chapter-extractor 的黑名单（老师、老板、师傅等）

### 人物分类

> 分类标准见 Stage 5 人物分类（二维体系）：叙事角色{主人公/重要配角/功能人物} + 行动角色{主动型/被动型/转变型}。

### 人物内在矛盾提取

短篇人物的核心在于内在矛盾，不是外部遭遇：

| 错误 | 正确 |
|------|------|
| 她被霍庭煜抛弃了（外部遭遇） | 她渴望被爱但学会了在不爱中生存（内在矛盾） |
| 他对她不好（行为描述） | 他习惯了掌控一切却无法掌控自己的感情（内在矛盾） |

### 人物功能评估

短篇每个人物都必须"物尽其用"：

| 指标 | 问题 |
|------|------|
| 出场效率 | 第一次出场就完成定位了吗？ |
| 功能密度 | 承担了多少叙事功能？ |
| 可删除性 | 删掉故事还成立吗？（不成立=好，成立=多余） |
| 对话贡献 | 每次对话都推动剧情或情绪吗？ |

## 共鸣分析

### 共鸣层次（9 层）

| 层次 | 定义 | 分析要点 |
|------|------|----------|
| 情感共鸣 | 读者对角色当下情绪的即时共情（如心疼、愤怒） | 共情点在哪里 |
| 价值观共鸣 | 对故事传达的具体价值观的认同（如"女性不该被定义"） | 传达了什么价值观 |
| 经历共鸣 | 与读者自身经历的相似性 | 哪些经历容易产生代入 |
| 社会现象共鸣 | 与社会热点/普遍现象的关联 | 映射了什么社会现象 |
| 文化共鸣 | 与文化传统的共鸣 | 触发了什么文化记忆 |
| 普世价值共鸣 | 对人类共同追求的宏观认同（如自由、尊严、爱） | 触及了什么普世主题 |
| 哲学思考共鸣 | 引发对人生/存在的深层思考 | 触发了什么思考 |
| 情感深度共鸣 | 情绪体验留下的持久影响（读完几天后还在想） | 情绪是否深入持久 |
| 人物深度共鸣 | 对人物复杂性的理解和认同 | 人物是否有深度 |

每层共鸣标注强度：极强（读者强烈共情，无法自控）/ 强（读者明确感知）/ 中（需要细品）/ 弱（若有若无）/ 无（未触发）。至少 3 层达到"强"或"中"的短篇才具备传播潜力。

> **易混淆区分**：情感共鸣（与角色当下情绪的即时共情）vs 情感深度共鸣（读完几天后还在想的持久影响）；价值观共鸣（对故事传达的具体价值观的认同）vs 普世价值共鸣（对自由、尊严、爱等人类共同追求的宏观认同）。

### 爆点性分析

评估故事的爆点（容易引发传播/讨论的关键点）：
- 核心爆点是什么？
- 爆点的铺垫是否充分？
- 爆点的释放是否到位？
- 爆点是否具有传播性？

### 话题性分析

评估故事的话题性：
- 读者读完会讨论什么？
- 是否有争议点？
- 是否有代入式的自省？
- 是否有"如果是我会怎样"的讨论空间？

## 结构类型速查

| 结构 | 特点 | 适合情绪 |
|------|------|----------|
| 三幕反转式 | 铺垫→升级→反转 | 震惊、意难平 |
| 双线交叉式 | 明线暗线交替推进 | 悬疑、真相揭晓 |
| 时间跳跃式 | 过去/现在穿插 | 遗憾、成长 |
| 套娃反转式 | 多层反转嵌套 | 惊讶、烧脑 |
| 对话驱动式 | 通过对话推进叙事 | 虐恋、人际冲突 |
| 独白式 | 第一人称内心独白 | 治愈、自省 |
| 重生反常式 | 重生后做相反的事→对手自乱 | 爽、解气 |
| 灵魂旁观式 | 死后灵魂看世界→揭露真相 | 意难平、愤怒 |
| 双重生博弈式 | 两人都重生→信息战 | 紧张、痛快 |
| 隐忍布局式 | 表面顺从暗中布局→反攻 | 压抑→极致释放 |
| 非人视角式 | 鬼/系统/灵魂看人间 | 新奇、恐怖、爽 |
| 成长救赎式 | 苦难中被人拉一把→自我救赎→与救赎者约定未来 | 温暖、心酸、治愈 |
| 舆论反转式 | 舆论多次反转→证据反击→最终真相大白 | 爽、解气、震惊 |
| 冷面复仇式 | 冷静主角收集证据→果断出手→全身而退 | 爽、痛快、满足 |
| 倒计时告别式 | 明确时限的告别/离开→倒计时推进→最终诀别 | 心酸、意难平、通透 |

### 匹配分析要点

1. **主结构识别**：主要使用了哪种结构？
2. **混合结构**：大部分优秀短篇都是混合型。标注方式：主结构（主）+ 副结构（副），如"灵魂旁观式（主）+ 压缩弹簧（副）"。主结构决定叙事框架，副结构决定情绪节奏
3. **结构创新**：有没有对标准结构的创新使用？如"AI系统×复仇"的类型嫁接
4. **结构适配度**：这个结构是否最适合这个题材和情绪目标？

## 质量标准

### 拆解完整性检查

| 指标 | 阈值 | 计算 | 不达标处理 |

## references 协作关系

短篇拆书由 SKILL.md + 三个 references 文件协同工作：

- **SKILL.md** = 路由逻辑 + 管道概要（Phase 1 用户交互 + Stage 2-6 管道表）
- **本文件（`material-decomposition.md`）** = 方法论细节（Stage 2-6 操作步骤 + 情节节点 + 写作手法 + 情感线 + 节奏 + 共鸣）+ **质量标准唯一权威**（阈值/数值/计算方式）
- **`output-templates.md`** = 具体输出格式模板（每个 Stage 的 markdown 模板 + 质量门控逐项 checklist，数值引用本文件）
- **`quality-checklist.md`** = 短篇拆书的质量自检清单（评估质量时用，数值引用本文件）

四个文件协同工作：
1. SKILL.md 决定执行哪个 Stage（Phase 1 由 SKILL.md 直接处理）
2. 本文件告诉"怎么分析"（Stage 2-6 的方法论），并定义所有质量阈值数值
3. `output-templates.md` 告诉"怎么输出"（Stage 2-6 的格式模板）
4. `quality-checklist.md` 提供拆书完成后的质量自检清单
