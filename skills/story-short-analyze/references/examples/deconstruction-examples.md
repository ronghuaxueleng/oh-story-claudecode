# 拆文实例（3 个完整案例）

> 拆文时参考结构范例。先用决策路由选相近类型，再对照案例检查自己的拆文。

## 决策路由

| 你在拆什么类型的故事 | 参考案例 | 重点看 |
|---------------|---------|--------|
| 现代虐恋/反转 | 案例 1（盲女保镖 x 霸总） | 假盲设定 + 双关 + 日记揭露 |
| 古代虐恋/身份反转 | 案例 2（庶女 x 将军） | 信物转赠 + 底层翻身 |
| 权谋/多角关系 | 案例 3（长公主 x 四位青梅） | 主动布局 + 信息差设计 |

## 案例 1：现代虐恋（盲女保镖 x 霸总）

**导语**
- 背景：女主是盲人保镖，救过男主，男主承诺照顾她一辈子
- 虐点：女主发现男主和女配出轨，为报恩忍耐
- 反转：女主其实是假盲，她看得见一切背叛
- 钩子：女主能看见但没人知道 | 她只给自己 7 天时间离开

**起**：女主在厨房听到男主打电话约女配 -> 假装不知道端汤出去 -> 男主手机弹出女配消息，女主"碰巧"看到屏幕 -> 虐点：男主在她面前毫无顾忌

**承+转1**：女主发现怀孕想告诉男主 -> 撞见男主给女配庆生 -> 看到项链是当年自己送的定情物 -> 虐点升级：信物被转赠

**承**：女主沉默回家收拾行李 -> 男主回来问眼睛 -> 她说"好多了，什么都能看见了"（双关，男主没听懂）

**承+转2**：男主生日宴会，女配以女伴身份出席，女主作为"保镖"在角落站着 -> 女主当众宣布离婚递协议 -> 爽点：全场震惊

**合**：女主离开后男主发现她留下的日记 -> 日记记录了她"看见"的每一次背叛 -> 男主崩溃但女主已消失 -> 爽点+悬念

## 案例 2：古代虐恋（庶女 x 将军）

**导语**
- 背景：庶女与将军秘密相恋三年，将军承诺战后娶她
- 虐点：将军凯旋带回正妻——是嫡姐
- 反转：庶女发现自己只是嫡姐的"练手工具"
- 钩子：庶女能否从贱籍翻身到龙床 | 将军会后悔到什么程度

**起**：庶女在后院刺绣听到将军回城 -> 满心期待准备嫁衣 -> 门口传来嫡姐声音："夫君，你回来了"

**承+转1**：庶女被迫给嫡姐敬茶 -> 偷听到"不过是拿她练练手，你别介意" -> 虐点：三年感情只是工具

**承**：庶女表面顺从暗中接近太子宴席 -> 用才艺引起太子注意 -> 虐中蓄力

**承+转2**：将军发现庶女被太子纳入府中震惊 -> 庶女当众揭穿将军靠嫡姐上位 -> 爽点：身份反转将军颜面尽失

**合**：庶女成为太子侧妃后成正妃 -> 将军在战场被追杀无人救援 -> 临死前看到庶女画像终于后悔 -> 爽点：迟来的后悔比死更难受

## 案例 3：权谋公主（长公主 x 四位青梅竹马）

**导语**
- 背景：长公主有四位青梅竹马未婚夫，但她心里只有那个不爱她的将军
- 虐点：她选了最没权势的那个，放弃了一直默默守护的将军
- 反转：她选的那个才是害她母后的人
- 钩子：公主放弃真爱会怎样 | 被放弃的人如何反击

**起**：抽签决定婚姻，公主表面顺从 -> 暗中做了手脚让最弱的那个中签 -> 读者知道她在保护将军

**承+转1**：被选中的驸马展露野心 -> 发现驸马与敌国通信 -> 虐点：公主以为在保护人实际养虎为患

**承**：公主被软禁，将军在外征战不知情 -> 公主利用暗线传递消息反击 -> 虐中蓄力

**承+转2**：将军回朝发现公主被囚 -> 公主早已安排好反击计划，不是等救援是在等时机 -> 爽点：公主从来不是弱者

**合**：驸马叛变被镇压公主亲临城楼 -> 将军问为何当年不选自己 -> "因为只有你不在我身边，你才能成为最强的那把刀" -> 爽点+余韵

## 拆文检查清单

1. **标记情绪**：每段标注虐/爽/蓄力，检查分布是否合理
2. **找钩子**：每篇至少 2 个钩子，至少 1 个信息差钩子
3. **查结构**：起承转合是否完整，转折是否合理
4. **看铺垫**：每个反转前是否至少 2 个铺垫线索
5. **评估爽度**：最大爽点是否在全文 70-85% 位置

## 反例对照：什么叫“合规但差”

> 下面不是完整拆文，而是专门示范 3 种常见坏结果。它们往往能把字段写齐，甚至能过部分验收，但对后续写作帮助很弱。

### 反例 1：伪精确判断

**不合格写法**

- 原文整体分数：中低风险
- 内部最高块风险分：中高
- 原文最高风险片段：婚礼桥

问题：

- 没说是否真实跑过检测
- 看起来像脚本结果，实际只是人工阅读印象
- 下游读的人会误以为这些判断可复核

**合格写法**

- 原文整体分数：未测
- 内部最高块风险分：未测
- 原文最高风险片段：人工判断：导语摊牌、遗物碎裂、终局炸场三处亮度最高；依据是这三段承担最大情绪峰值和最高概念反转

为什么这样才对：

- 不把人工判断伪装成检测结果
- 既保留判断，也保留判断依据
- 后续有人真跑检测时，能区分哪些是脚本结论，哪些是阅读判断

### 反例 2：表写得像给脚本看，不像给写手看

**不合格写法**

`导语拆解表`

| 前20字 | 前60字 | 前80字 | 第一句功能 | 第二句加码 | 继续滑动力 |
|--------|--------|--------|------------|------------|------------|
| 已起事 | 已错位 | 已推进 | 起事 | 加码 | 读者继续看 |

问题：

- 字段齐了，但写手拿不到真正可迁移的“钩”
- 去掉表头后，内容几乎无法指导起盘
- 它只是在满足映射，不是在拆导语

**合格写法**

`导语拆解表`

| 原文怎么写 | 钩子内容 | 作用 | 迁移提醒 |
|---|---|---|---|
| 她嫁给我是为了气旧爱 | 错位关系 | 先把婚姻压歪 | 开头第一刀直接给错位，不绕前史 |
| 全网都说我是替身 | 公共命名 | 让羞辱带社会见证 | 让旁人先替读者说出最扎评价 |
| 我快死了却说不爱她 | 生死反告白 | 强制读者追问真相 | 生死信息要和情感反刀绑在一起 |
| 我爱的是已经不在的人 | 真异点 | 立刻改写类型预期 | 异点要尽早亮，不要拖成普通追悔文 |

为什么这样更好：

- 写手一眼就能看出“错位关系 / 公共命名 / 生死反告白 / 真异点”这四个功能钩
- 就算后面再补 `前20 / 60 / 80 字映射`，主表仍然先服务人
- 这才是“可直接仿写层”

### 反例 3：节点数量够了，但后段蓄压被压没

**不合格写法**

- N24 主角拿到证据
- N25 婚礼当天曝光
- N26 反派身败名裂

问题：

- 节点数可能已经够了，但从“埋雷”直接跳到“总炸场”
- 少了 `终局前夜 / 预热 / 硬牌上桌前` 的蓄压层
- 后续写手只会学到“拿到证据 -> 公开曝光”，会写成按钮式终局

**合格写法**

- N24 主角拿到证据，但先不出手
- N25 反派高调准备最高光公开场，主动把自己送进镜头中心
- N26 主角确认公开场条件成熟，收下请帖 / 改直播 / 等仪式开始
- N27 证据在最高光时刻被放出
- N28 反派崩塌，后果继续落地

为什么这样更好：

- 终局爽点不是凭空起爆，而是被“前夜蓄压”抬起来
- 写手能学到“先让人站上高处，再推下去”的场面顺序
- 这类节点对终局写法的价值，往往比再补一个前段小节点更高

### 反例 4：把主角磨平成纯受害者

**不合格写法**

- 主角：隐忍、深情、受伤、反击

问题：

- 这样写虽然安全，但会把复杂主角写扁
- 原文本来如果有“借影 / 借势 / 借局”，这一层会被抹掉
- 后续写手容易把人物写成单线委屈脸

**合格写法**

- 主角：表面隐忍，实际上也在借这段错位关系安放自己的旧情或完成自己的局
- 主角不是无辜到纯白，而是比别人更清醒、更晚出手

为什么这样更好：

- 这能保住人物的反咬感和清算感
- 写手后续迁移时，人物不会只剩“惨”
- 复杂人物通常比纯受害者更耐写

## 反例使用法

拆书时，如果你发现自己的结果同时满足下面两条，就先停下来返查：

1. 字段都齐了
2. 但你自己都不想拿这份东西去写

高概率说明你已经滑进“合规但差”的区间。

优先检查这四件事：

- 有没有把人工判断写成伪实测
- 有没有把导语、钩子、顺序表写成机械壳
- 有没有把终局前夜的蓄压节点压没
- 有没有把主角磨平成纯受害者
