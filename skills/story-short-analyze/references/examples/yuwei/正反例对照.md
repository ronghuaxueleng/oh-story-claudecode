# 《幼薇》正反例对照

用途：

- 给 `story-short-analyze` 一个真实的厚拆样本
- 明确什么叫“保住中段承重桥和原子细节”
- 明确什么叫“表面合规，但人读资产变薄”

---

## 1. 原文细节库：动作细节库

### 正例

来自《幼薇》较厚的拆法：

```md
## 改密码

- 具体发生了什么：宋远回家时发现密码被改成唐月轩的生日，这比一句“她偏向旧爱”更直接。
- 这个细节为什么有用：它是一个硬动作结果，说明家庭秩序已经不需要经过解释。
- 它压的是谁、压在哪：压的是宋远的居住权和主位，位置在家门第一步。
- 后续能迁到什么新桥段：能迁到删指纹、换门锁、搬床位；承重件是“控制权限被改写”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远、唐月轩、周逢雅；情绪是狼狈；反转是丈夫成了外来者。

## 抡包砸人

- 具体发生了什么：唐月轩抢戒指时被宋远挥开，立刻抡包砸过去，金属边刮破额头见血。
- 这个细节为什么有用：一句挑衅马上接一个实伤动作，冲突一下从嘴硬升级成可见伤口。
- 它压的是谁、压在哪：压的是宋远的身体和名分，位置在办公室公开空间。
- 后续能迁到什么新桥段：能迁到扯领带、泼水、甩文件；承重件是“物件争夺要有身体后果”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远、唐月轩；情绪是刺痛；反转是装乖的人一秒变狠。

## 翻垃圾堆

- 具体发生了什么：周逢雅陪着宋远在垃圾站整夜翻找碎玉，天亮才找回残片。
- 这个细节为什么有用：人物要真下去翻，才能把“心疼”和“愧疚”落到动作层，而不是口头心疼。
- 它压的是谁、压在哪：压的是周逢雅的洁癖和高位体面，位置在最脏的现实处境里。
- 后续能迁到什么新桥段：能迁到雨里捡信、废墟找物、医院翻箱；承重件是“角色为了旧物真的降到最低处”。
- 它对应的角色 / 情绪 / 反转是什么：对应周逢雅、宋远；情绪是疲惫中的迟来补偿；反转是她终于做了点事，但还是来晚了。
```

为什么这是正例：

- 不是只列动作名，而是把动作替掉了哪句解释、压坏了谁的位置写清。
- 三张卡分别覆盖“权限改写 / 身体后果 / 脏场补救”，不是同一种动作重复三遍。
- 这类写法能直接给后续仿写提供不同桥段里的动作承重件。

### 反例

来自《幼薇》一版偏薄的拆法：

```md
## 1. 收手
- 具体发生了什么：宋远拉周逢雅袖子让她住手，被冷脸问“轮得到你来管我”，他轻轻松开手。
- 这个细节为什么有用：一个“松手”比一长段委屈独白更真。
- 这条细节真正压坏了什么位置：压的是说话资格。
- 后续能迁到什么新桥段：能迁去饭局、会议、病房，前提是保留“主角试图干预后意识到自己无权”。
- 这条细节自己的角色站位：宋远那时还处在忍的位置。

## 2. 拽领带
- 具体发生了什么：谈离婚条件时，宋远伸手拽住周逢雅领带，把她扯近。
- 这个细节为什么有用：动作一出来，谈判主导权就变了。
- 这条细节真正压坏了什么位置：压的是周逢雅的高位姿态。
- 后续能迁到什么新桥段：能迁去审问、摊牌、分手，前提是保留“主角突然抢近身距离”。
- 这条细节自己的角色站位：宋远后期不是只会退，他会抢局。
```

为什么这是反例：

- 两条本身不假，但只剩 2 张卡，动作资产并集明显不够。
- 漏掉了“改密码”“抡包砸人”“翻垃圾堆”这类中段承重动作，后续写作就拿不到最值钱的桥件。
- 这种结果容易“读起来也没错”，但已经变成过检式压缩，不算厚拆完成态。

## 2. 拆文报告：题面拆解

### 正例

```md
### 题面拆解

| 维度 | 分析 |
|------|------|
| 表面题材 | 替身婚姻 / 绝症离婚 / 旧爱回潮 / 校园旧案复仇 |
| 流派判断 | 婚恋清算流 + 死者回影流 + 公开审判流 |
| 主梗 | 影后把男主当气初恋的婚姻工具，男主却拿这场婚姻当姐姐替身，最后用婚礼直播清算旧案 |
| 副梗 | 旧爱归来挤掉丈夫、玉牌旧伤回流、绝症手术离婚、校园杀人旧案曝光 |
| 组合公式 | 替身婚姻 + 旧爱回潮 + 母亲遗物 + 婚礼直播复仇 |
| 梗位分工 | 替身婚姻负责开头钩子；玉牌和手术负责中段压情绪；婚礼直播负责终局炸场 |
| 真正特别的地方 | 男主并不真的要抢回周逢雅，而是借她完成对周逢瑶和唐月轩的迟来清算 |
| 最先把人拽进去的事 | “因为，我不爱你。我爱的人，是她死去的姐姐。” |
| 中段又加了什么事 | 母亲遗物被扔、房子换主、垃圾站翻碎玉，把私人旧伤从婚恋线拉到原生伤 |
| 读者最先以为这是什么文 | 以为是标准替身老公绝症离婚追悔文 |
| 后面哪一步改写了这个预期 | 手术梦境和周逢瑶真相把主情感对象换成了死去的姐姐 |
| 题面最扎人的那一下 | 男主和女主互相把对方当替身或工具，真正被爱的人已经死了 |
| 导语先亮了什么后果 | 先亮“确诊癌症还说不爱她”，把读者拽去追问真正爱的人是谁 |
```

为什么这是正例：

- 它不只总结题材，还写清“读者先误判什么”“哪一步翻预期”。
- 这能直接反向服务导语、梗概、题面重写和开篇卖点提炼。

### 反例

```md
### 题面拆解
- 标题状态：未验证（来自文件名）
- 表面题材：替身婚姻 / 娱乐圈狗血 / 校园旧案复仇
- 深层流派：旧伤物件复仇型虐爽文
- 题面承诺：看似写“影后出轨替身老公”，实际主打“真爱亡姐+物件旧伤+直播清算”。
```

为什么这是反例：

- 方向不算错，但只有摘要，没有预期翻转链。
- 它回答了“这本是什么”，却没回答“为什么读者会被先骗进去，又是从哪一步被翻掉”。
- 这类写法适合做梗概前的索引，不够格当厚拆题面分析。

## 3. 原文细节库：场面细节库

### 正例

```md
## 热搜夜的双场面

- 具体发生了什么：一边是唐月轩微博发十指相扣，一边是宋远在家里翻照片想替周逢雅澄清，最后却被她本人转发拆台。
- 这个细节为什么有用：同一晚上的两个场面互相咬，能迅速把婚姻关系从私人尴尬推成公开羞辱。
- 它压的是谁、压在哪：压的是宋远的体面，位置在网络与家中同时失守的双场景。
- 后续能迁到什么新桥段：能迁到朋友圈官宣、群聊曝光、直播连线；承重件是“主角刚补台，另一边立刻拆台”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远、周逢雅、唐月轩；情绪是窒息；反转是澄清刚发出就被当事人亲手打烂。

## 垃圾站黎明场

- 具体发生了什么：两人在垃圾堆找了一整夜，天快亮时终于找回碎玉，太阳升起反而把无助照得更明显。
- 这个细节为什么有用：不是夜里黑暗才悲，偏偏在天亮时找到碎片，形成“看见了却救不回”的反差。
- 它压的是谁、压在哪：压的是宋远最后一点希望，位置在从黑夜转到黎明的时间缝里。
- 后续能迁到什么新桥段：能迁到天亮前手术、暴雨停后现场、火熄后废墟；承重件是“找到了，但只能看到损毁”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远、周逢雅；情绪是绝望；反转是辛苦有了结果，但结果本身坏透了。

## 婚礼停仪式场

- 具体发生了什么：牧师证婚到最庄严处，现场有人举手机给周逢雅看，唐月轩伸手去拦却越拦越露馅，仪式被硬生生叫停。
- 这个细节为什么有用：停仪式比打断讲话更狠，因为它直接让“永远忠诚”的誓词变笑话。
- 它压的是谁、压在哪：压的是唐月轩和周逢雅的合法化仪式，位置在最正式的婚礼节点。
- 后续能迁到什么新桥段：能迁到签约中断、颁奖叫停、开庭爆料；承重件是“在最不可中断的流程节点强制叫停”。
- 它对应的角色 / 情绪 / 反转是什么：对应唐月轩、周逢雅；情绪是震惊和崩坏；反转是他们最想被确认的一刻成了被否定的一刻。
```

为什么这是正例：

- 它不是笼统写“大场面很炸”，而是拆成三种完全不同的场面机制。
- 既保了中段双场面互咬，也保了终局仪式停摆，不会只剩头尾亮桥。

### 反例

```md
## 1. 垃圾站天亮
- 具体发生了什么：周逢雅和宋远在垃圾堆里翻到天亮，老夫妻也来帮忙，最后只找到碎掉的玉牌。
- 这个细节为什么有用：场面既脏又静，把补救无效写得特别扎实。
- 这条细节真正压坏了什么位置：压的是体面和最后希望。
- 后续能迁到什么新桥段：能迁去废墟找物、雨夜翻档案、雪地挖盒子，前提是保留“找到时已经坏了”。
- 这条细节自己的角色站位：周逢雅终于下场，但仍赶不上后果。

## 2. 婚礼直播
- 具体发生了什么：铺满白玫瑰的婚礼上，唐月轩刚宣誓，录像就让仪式变成葬礼。
- 这个细节为什么有用：最美的场面给最丑的真相做托底，视觉反差极强。
- 这条细节真正压坏了什么位置：压的是公开体面和爱情神话。
- 后续能迁到什么新桥段：能迁去发布会、颁奖礼、升职礼，前提是保留“最高光时刻被硬牌拦腰截断”。
- 这条细节自己的角色站位：场面站在真相一边，不再站在新人一边。
```

为什么这是反例：

- 单条本身不差，但只剩两张卡，漏掉了“热搜夜的双场面”这种很值钱的中段场面资产。
- 这会让场面库退化成“脏场 + 终局”，中段公开羞辱链断掉。

## 4. 原文细节库：对白细节库

### 正例

```md
## “轮得到你来管我？”

- 具体发生了什么：周逢雅打完人、宋远想劝她停手时，她转头就丢出这句，把丈夫的位置当场踢没。
- 这个细节为什么有用：一句话同时完成站边和排除，比长篇解释“我只是心疼他”更有效。
- 它压的是谁、压在哪：压的是宋远的发言权，位置在公开场中心。
- 后续能迁到什么新桥段：能迁到“关你什么事”“我用你教我吗”；承重件是“用一句话直接取消主角资格”。
- 它对应的角色 / 情绪 / 反转是什么：对应周逢雅、宋远；情绪是冷厉；反转是丈夫成了多余的旁观者。

## “你不必知道。”

- 具体发生了什么：周逢雅追问病情时，宋远没有借势卖惨，只是说她不必知道。
- 这个细节为什么有用：拒绝解释会让关系更冷，也让病情线不沦为索爱工具。
- 它压的是谁、压在哪：压的是周逢雅迟来的关心，位置在她终于想追问的节点。
- 后续能迁到什么新桥段：能迁到伤情、债务、秘密计划；承重件是“最有资格解释时反而拒绝解释”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远；情绪是抽离；反转是卖惨期待被直接掐灭。

## “因为，我不爱你。”

- 具体发生了什么：离婚谈判里，宋远看着周逢雅，说自己从来不爱她。
- 这个细节为什么有用：这句不是孤零零的狠话，它前面压着病情、玉牌、旧案、离婚协议，所以才像最后的盖棺。
- 它压的是谁、压在哪：压的是周逢雅的自我认知，位置在她以为还能靠“爱上他”翻盘的瞬间。
- 后续能迁到什么新桥段：能迁到求复合、求复婚、病床告白；承重件是“前账够满后再落最核心真话”。
- 它对应的角色 / 情绪 / 反转是什么：对应宋远、周逢雅；情绪是决绝；反转是全篇看似追悔的对象根本不是目标。
```

为什么这是正例：

- 三条对白分别承担“取消资格 / 拒绝解释 / 最终盖棺”，功能层次很清楚。
- 它不只是摘金句，而是把对白在桥段里的工作拆出来了。

### 反例

```md
## 1. 软刀感谢
- 具体发生了什么：唐月轩笑着谢谢宋远“替我照顾逢雅这么多年”。
- 这个细节为什么有用：表面是礼貌，实际把婚姻改成代养关系。
- 这条细节真正压坏了什么位置：压的是名分和主体资格。
- 后续能迁到什么新桥段：能迁去前任、白月光、继承争夺，前提是保留“谢谢你替我”。
- 这条细节自己的角色站位：唐月轩靠甜口抢主位。

## 2. 轻声翻刀
- 具体发生了什么：宋远平静地说“我爱的人，从来不是你”。
- 这个细节为什么有用：最狠的话说得最轻，人物控制力就出来了。
- 这条细节真正压坏了什么位置：压的是周逢雅自认的独一无二。
- 后续能迁到什么新桥段：能迁去摊牌、离婚、决裂、拜别，前提是保留“轻声说真相”。
- 这条细节自己的角色站位：宋远终于拿回最后定义权。
```

为什么这是反例：

- 两条都成立，但覆盖不够，漏掉了“取消资格”和“拒绝解释”两种极值对白功能。
- 只保留甜口抢位和终局翻刀，会把对白层误读成“只看金句”，不看桥内功能。

## 5. 情绪同构仿写资产

### 正例

```md
## BID｜热搜夜双场互咬

| 拍位 | 原文触发 | 关系位置变化 | 读者体感 | 烈度 | 迁移要求 |
|---|---|---|---|---:|---|
| 情绪进入点 | 丈夫还在替妻子找照片、准备澄清 | 他暂时还把自己放在婚姻补台者的位置 | 读者产生一点“也许还能救”的希望 | 6 | 新稿也要先让主角做出真实补台动作 |
| 受辱/刺痛 | 旧爱公开十指相扣 | 婚姻被第三人抢先定义 | 替主角发冷、丢脸 | 8 | 必须是公开可见的抢位，不得降成私下误会 |
| 短暂希望或反抗 | 主角把澄清材料整理好 | 主角试图抢回解释权 | 读者短暂期待他能守住体面 | 7 | 反抗必须有实际动作，不只写“他决定相信她” |
| 反刀 | 妻子本人转发旧爱的内容 | 妻子亲手拆掉丈夫替她补的台 | 希望被当事人亲手打烂 | 9 | 反刀必须来自最该维护主角的人，且发生在同一拍位 |
| 场末余痛 | 澄清材料失去意义 | 丈夫从补台者变成被公开排除的人 | 窒息、替他难堪 | 8 | 场末不能立刻讲透或反击，要把余痛带进下一场 |
```

为什么这是正例：

- 它保留的不只是“热搜公开羞辱”功能，而是 `先补台 -> 看见抢位 -> 试图挽回 -> 被妻子亲手反刀 -> 余痛未消` 的完整情绪流程。
- 后续换成直播、群聊或发布会时，人物和职业可以变，但五拍顺序、反刀对象和逐拍烈度不能缩水。

### 反例

```md
- 原文功能：公开掉位。
- 目标写法：男主在会议上撤销女主权限。
- 情绪烈度：8。
```

为什么这是反例：

- 只有功能和总分，没有逐拍触发、希望落空、反刀位置和场末余痛。
- “撤销权限”可能只是职业决定，不能证明读者会获得原文那种被最亲近者亲手拆台的羞辱。

## 5. 高敏桥段识别

### 正例

```md
- 桥段名：旧爱归来同学会抢位
...
- 桥段名：办公室戒指冲突
...
- 桥段名：玉牌翻垃圾站
...
- 桥段名：婚礼直播公开审判
...
```

为什么这是正例：

- 头部桥、终局桥、中段物件桥、中段场面桥都在。
- 尤其保住了“办公室戒指冲突”这种容易在压缩时先被删掉的中段承压桥。

### 反例

```md
- BID-01 桥段名：回国聚会公开掉位
  - 原文：周逢雅先护“老公”面子，后当众为唐月轩动手，再把宋远丢下。
- BID-02 桥段名：私域被替换与玉牌失踪
  - 原文：密码改成唐月轩生日、婚照换掉、书房改钢琴室、玉牌被扔。

```

为什么这是反例：

- 这版并非错误，但桥卡被压得只剩极简摘要，少了“为什么不像加工稿 / 新稿最容易写假的点 / 可学层 / 禁学层 / 后续调用方式”。

## 6. 桥段施工卡

### 正例

```md
## BID-03 垃圾站找回碎玉后被电话截断
- 桥段名：垃圾站找回碎玉后被电话截断
- 一句人话抓手：她陪你翻了一夜垃圾，天亮时还是被别人一通电话叫走。
- 桥段角色：现实后果压顶
- 原文位置：L239-L269
- 原文现象证据：周逢雅满手污泥陪找玉牌，刚让人觉得她终于站对边，电话一响就走，紧接着就是手术短信。
- 原文为什么能过：补救和失去贴得极近，情绪还没回暖就被抽空。
- 为什么不像加工稿：不是纯虐，而是给一点点希望再马上现实落锤。
- 新稿最容易写假的点：只写陪找很感人，不补“今天是我预约手术的日子”。
- 必须保留的承重件：脏场景、找到时已碎、对方电话、现实日期短信。
- 不能丢的顺序：陪找 -> 找到碎玉 -> 短暂安抚 -> 电话叫走 -> 手术短信。
- 为什么这个顺序不能乱：必须先把补救做实，后面的离开才像迟到心疼而不是普通转身。
- 后续调用方式：适合写失物、病历、录音等旧伤桥转入现实伤桥。
```

为什么这是正例：

- 这是标准的“能直接拿去施工”的桥卡，不是报告摘要。
- 尤其把“为什么能过 / 为什么不像加工稿 / 最容易写假 / 顺序为什么不能乱”全保住了。

### 反例

```md
## 垃圾站找玉
- 桥段名：垃圾站找玉
- 原文位置：L239-L269
- 原文：两人在垃圾堆里找玉，后来她走了，主角独自去手术。
- 后续调用方式：适合写虐点桥。
```

为什么这是反例：

- 这种写法不是错，而是严重不够施工。
- 它缺少承重件、顺序、易假点、为什么能过，后续写手拿到也没法直接照着搭。

## 7. 交流承压拆解 / 冲突载体清单

### 正例

```md
## 交流承压拆解｜办公室戒指冲突

- 原文位置：L100-L145
- 场面锚点：`替我照顾逢雅这么多年` / `你戴着我的结婚戒指，该还给我了`
- 谁先施压：唐月轩
- 压力载体：物件 + 身份 + 身体
- 具体施压动作：他先用感谢把宋远改写成代管者，再直接伸手抢婚戒；抢不到后抡包砸出伤口。
- 对方被迫改了什么：宋远从称谓反切转入防御，最后主动摘戒，撤回身份物。
- 删掉台词是否仍成立：是。只看“伸手抢戒 -> 见血 -> 摘戒 -> 戒指被嫌脏丢弃”，关系争夺仍成立。
- 为什么不是作者替人物接招：现场由戒指归属、额头伤口和垃圾桶完成接招，不靠作者解释“他很受伤”。
- 仿写最容易写假的点：只保留“谢谢你替我照顾她”的软刀，不写抢戒和见血，交流会退化成金句互刺。

## 冲突载体清单｜办公室戒指冲突

- 表层冲突：旧爱向丈夫讨戒指。
- 第二层争夺权：婚姻名分定义权 + 身份物处置权。
- 主载体：dialogue / body / object
- 谁当场失位：宋远，合法丈夫的位置被改写成“替别人照顾她的人”。
- 越界后果：额头见血、病情半句话被打断、宋远摘戒离场。
- 如果只保留对白会丢掉什么：会丢掉“身份物真的离手”和“旧人会用身体后果抢叙述权”的恶感。
```

为什么这是正例：

- 它没有把“交流”写成眼神、停顿和漂亮对白，而是写清物件和身体如何改变现场。
- 它能直接服务新稿里的 `交流承压拆解.md` 和 `冲突载体清单.md`。

### 反例

```md
## 办公室对话

- 原文位置：L100-L145
- 谁先施压：唐月轩
- 压力载体：对白
- 具体施压动作：他说宋远只是替他照顾周逢雅。
- 对方反应：宋远觉得难受。
- 冲突类型：情敌争夺。
```

为什么这是反例：

- 它只剩话术概括，没有戒指、抢夺、见血、摘戒、丢弃这条完整压力链。
- 写作侧拿到这种材料，只会写出“你来我往”的台词，不会写出现实夺权。

## 8. 使用方式

正式跑《短篇拆文》时，默认这样消费这组样本：

1. 先读《幼薇》原文，确认正例资产确实能在原文找到。
2. 写原文细节库时，对照“动作细节库”正反例，优先检查是否漏掉中段承重动作。
3. 再对照“场面细节库 / 对白细节库”正反例，检查有没有只剩头尾亮点、漏掉中段双场面或功能对白。
4. 写 `题面拆解` 时，对照正反例，强制补“读者先误判什么 / 从哪一步翻”。
5. 写 `高敏桥段识别 / 桥段施工卡` 时，对照正反例，确认不是只剩桥名摘要。
6. 写 `交流承压拆解 / 冲突载体清单` 时，对照“办公室戒指冲突”，确认不是只剩眼神、停顿和对白。
7. 如果结果更接近反例，默认不要直接 finalize，而是回到候选池和细节库重扫。
