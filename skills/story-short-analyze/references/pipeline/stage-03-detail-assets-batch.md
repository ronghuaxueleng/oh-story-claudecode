# Stage 3：细节库与写作资产

这一阶段最怕“文件齐了，但都只剩标签清单”。

## 目标

把主报告里已经确认的高价值桥、人物脸、关系掉位、场面压迫，扩成真正能给写手调用的资产层。

## 并发边界

- 只有 foundation 预检返回 `ready-for-asset-lanes` 后才能启动。
- `source_details / regular_assets / sensitive_assets` 三条 lane 写入路径互不重叠，可与 Stage 2 两条表格 lane 并发。
- 12000 字以内固定复用 3 个 agent：`agent-core` 连续写结构动作表与高敏资产，`agent-craft` 连续写对白关系表与常规资产，`agent-discovery` 从候选发现直接续写 `source_details`。
- 第二波 `asset_shared_reads` 固定为空。每个 agent 直接使用第一波会话中已加载的原文、样本锚点与分析契约，只允许按 `delta_reads` 补尚未见过的字典或候选池；不得按 `preferred_reads` 重新打开主报告、节点、手法或共享台账。
- `asset_shared_reads` 不含原文全文；只有现有锚点不足以支持某条资产时，才按 `source_on_demand` 读取对应精确行段。
- 主线程等待三次粗粒度派发汇合后，统一处理候选核销、动态字典回扫、BID 贯通、`profile_source.md` 和 finalize。
- 宿主不支持原生并发时，按 `_parallel_plan.json.worker_sequences` 推进现有会话；禁止回退成每条 lane 新起一个 agent。

## 原文细节库

固定 8 类。

默认一次连续落盘 8 份。完成后统一检查文件齐全、输出截断、按篇幅分档的证据卡数量和五项回答；失败时只二分重跑责任文件，二分仍失败才降级为双文件模式。

每张细节卡第一次落盘时就必须逐卡显式填写以下五个字段，不能只在文件开头声明一次，也不能改成旧版近义字段：

- `具体发生了什么`
- `这个细节为什么有用`
- `它压的是谁、压在哪`
- `后续能迁到什么新桥段`
- `它对应的角色 / 情绪 / 反转是什么`

每张卡使用独一的二级或三级标题。8000 字以上每文件至少 5 张，5000-7999 字至少 4 张，5000 字以下至少 3 张；原文确实没有时显式写 `原文未发现`。

每条细节都要落到具体：

- 事件
- 物件
- 动作
- 对白
- 关系掉位

不要写成：

- “这一类都可迁移”
- “原文里有这种感觉”
- “适合做某类桥段”

## 写作资产

默认分两大批：

- 常规资产批：`母结构_故事走法 / 主冲突_副升级器 / 异物清单 / 第二层冲突清单 / 角色口气模板 / 关系重组方式 / 公开场_关键硬牌_后果 / 平台适配提醒 / 情绪母线 / 新状态清单 / 虐点对照细节`
- 高敏资产批：`样本分级与可学层 / 高敏桥段识别 / 作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则 / 桥段施工卡`

每批连续落盘后做齐全、截断、有效字符和专项标题快检。不得为了让一个大批塞进输出而缩写；发现压缩只重跑该批。

当前阶段最值得保厚的文件：

- `高敏桥段识别.md`
- `作者DNA指纹.md`
- `仿写约束_禁写清单.md`
- `同桥段过检规则.md`
- `角色口气模板.md`
- `关系重组方式.md`
- `情绪母线.md`
- `桥段施工卡.md`

## 写作资产首写格式

以下常规资产从首写起必须包含唯一的 `## 原文证据层`，给出可定位的原文现象或短锚点：

- `母结构_故事走法.md`
- `主冲突_副升级器.md`
- `角色口气模板.md`
- `关系重组方式.md`
- `平台适配提醒.md`
- `情绪母线.md`
- `第二层冲突清单.md`

高敏资产按以下固定字段首写：

- `样本分级与可学层.md`：逐项写 `structure_grade / performance_grade / sentence_grade / terminal_consequence_grade: A/B/C`，并写 `正向DNA层 / 仅骨架层 / 反面规则层`；四层不一致时标明 `分层样本`
- `高敏桥段识别.md`：每卡用 `- 桥段名：` 起卡，并逐卡写 `桥段角色 / 原文证据 / 高敏点 / 可学层 / 禁学层`
- `桥段施工卡.md`：每卡使用独一标题 `## BID-xx 桥段名`，BID 只能取自 `_analysis_brief.md`；逐卡写 `桥段名 / 一句人话抓手 / 桥段角色 / 原文位置 / 原文现象证据 / 原文为什么能过 / 为什么不像加工稿 / 新稿最容易写假的点 / 必须保留的承重件 / 不能丢的顺序 / 为什么这个顺序不能乱 / 后续调用方式`

同一 Markdown 文件内所有标题必须唯一。卡片的重复字段必须写成项目符号，禁止把 `高敏点`、`原文证据` 等字段反复写成同名二级或三级标题。

## 特别提醒

当前 skill 最容易掉厚度的，就是这几份：

- `同桥段过检规则.md`
- `仿写约束_禁写清单.md`
- `情绪母线.md`

如果它们开始只剩 3-5 条摘要句，默认视为压缩化，要回炉。

## 人类可用性优先

这一阶段要优先保证：

- 写手拿到文件能直接理解怎么写
- 不是只有脚本能读懂
- 每条禁写法都最好带“为什么假”
- 每座桥最好都能回答“为什么原文能过”

## 本阶段通过条件

- 细节库不是模板壳
- 写作资产不是标签桶
- `桥段施工卡.md` 明显比 `profile_source.md` 更厚
- `同桥段过检规则.md` 和 `仿写约束_禁写清单.md` 不是纯摘要版
