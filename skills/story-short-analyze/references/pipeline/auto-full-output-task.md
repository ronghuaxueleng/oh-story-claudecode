# 短篇拆文全量自动产出任务单

这份文件不是方法论，而是执行单。

用途：

- 当 `story-short-analyze` 进入正式拆书后，强制按批次自动产出整套文件
- 防止模型只写 `拆文报告.md / 情节节点.md / 写作手法.md` 就停
- 防止 16 张表、原文细节库、写作资产被遗漏

硬口径：

- 不允许用户不提，模型就默认少拆
- 不允许先只写“核心三件套”，剩余文件等后补
- 不允许验收前跳过任何一批

## 总顺序

固定按 6 批执行：

1. 主报告批
2. 16 张可直接仿写表批
3. 原文细节库批
4. 写作资产批
5. `profile_source + book.profile` 批
6. 全量验收批

每一批都要满足：

1. 文件全部落盘
2. 每个文件非空
3. 本批关键骨架标题齐
4. 通过本批自检后，才能进入下一批

## 第 1 批：主报告批

### 必产文件

- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`
- `_meta.json`

### 最低要求

- `拆文报告.md` 必须包含：
  - `样本分级与可学层判断`
  - `高敏桥段识别`
  - `题面拆解`
  - `故事梗概`
  - `结构划分`
- `情节节点.md` 必须覆盖全文主要节点
- `写作手法.md` 必须按固定 9 节骨架落盘
- `_meta.json` 必须有 `structure_counts`

## 第 2 批：16 张可直接仿写表批

### 必产文件

- `可直接仿写_导语拆解表.md`
- `可直接仿写_顺序事件表.md`
- `可直接仿写_物件表.md`
- `可直接仿写_动作表.md`
- `可直接仿写_对白功能表.md`
- `可直接仿写_对话衔接表.md`
- `可直接仿写_误判表.md`
- `可直接仿写_钩子表.md`
- `可直接仿写_微动作表.md`
- `可直接仿写_安静压迫场表.md`
- `可直接仿写_人物偏手表.md`
- `可直接仿写_失控说话表.md`
- `可直接仿写_烂关系漏出表.md`
- `可直接仿写_外部秩序表.md`
- `可直接仿写_公开炸场表.md`
- `可直接仿写_后果链表.md`

### 每张表都必须补的 3 段

1. `可直接借的承重结构`
2. `迁移顺序提醒`
3. `为什么这个顺序不能乱`

额外硬要求：

- 三段都必须引用本表自己的具体条目
- `迁移顺序提醒` 必须写出该表专属的先后顺序
- `为什么这个顺序不能乱` 必须写清错序后的假点
- 不允许 16 张表共用统一提醒句

## 第 3 批：原文细节库批

### 必产目录

- `原文细节库/`

### 必产文件

- `场景细节库.md`
- `关系细节库.md`
- `情绪细节库.md`
- `对白细节库.md`
- `翻车细节库.md`
- `旧伤细节库.md`
- `动作细节库.md`
- `场面细节库.md`

### 每份细节库至少回答

1. 具体发生了什么
2. 这个细节为什么有用
3. 它压的是谁、压在哪
4. 后续能迁到什么新桥段
5. 它对应的角色 / 情绪 / 反转是什么

额外硬要求：

- 每个 `##` 小节都必须是具体事件，不准写成“某类场面”
- 同一文件不同小节不能大量复用同一句答案
- 必须能看出具体角色、具体动作或具体物件

### 必产目录

- `写作资产/`

### 必产文件

- `母结构_故事走法.md`
- `主冲突_副升级器.md`
- `异物清单.md`
- `第二层冲突清单.md`
- `角色口气模板.md`
- `关系重组方式.md`
- `公开场_关键硬牌_后果.md`
- `平台适配提醒.md`
- `情绪母线.md`
- `新状态清单.md`
- `虐点对照细节.md`
- `样本分级与可学层.md`
- `高敏桥段识别.md`
- `作者DNA指纹.md`
- `仿写约束_禁写清单.md`
- `同桥段过检规则.md`

### 高敏桥最低额外要求

- `重大证据前隔开的现实后果`
- `尾声入口给了谁 / 为什么不给另一条线`
- `人物不同脸证据 / 谁先解释谁先压场 / 动作权限差`

## 第 5 批：`profile_source + book.profile` 批

### 必产文件

- `写作资产/profile_source.md`
- `book.profile.json`

### 顺序

1. 先产 `profile_source.md`
2. 再运行 `generate_story_profile.py`
3. 再检查 `book.profile.json`

## 第 6 批：全量验收批

### 固定命令

```bash
python3 "$CODEX_HOME/skills/story-short-analyze/scripts/run_short_analyze_finalize.py" "拆文库/{书名}"
```

### 通过后才能说

- `ready-for-write`
- `可调用 story-short-write`
- `资料包已备齐`

### 最短执行话术

1. 按 6 批顺序自动落盘，不要跳批。
2. 凡是 skill 定义过的文件都要产出，不能省略。
3. 每批产完先自检，再继续下一批。
4. 最后一律运行 `run_short_analyze_finalize.py`，不过线不算完成。
