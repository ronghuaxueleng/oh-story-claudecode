# 短篇拆文全量自动产出任务单

这份文件不是方法论，而是执行单。

用途：

- 当 `story-short-analyze` 进入正式拆书后，强制按批次自动产出整套文件
- 防止模型只写 `拆文报告.md / 情节节点.md / 写作手法.md` 就停
- 防止 16 张表、原文细节库、写作资产被遗漏

硬口径：

- 不允许用户不提，模型就默认少拆
- 不允许先只写“核心三件套”，剩余文件等后补
- 不允许验收前跳过任何一批

## 总顺序

固定按 6 批执行：

1. 主报告批
2. 16 张可直接仿写表批
3. 原文细节库批
4. 写作资产批
5. `profile_source + book.profile` 批
6. 全量验收批

每一批都要满足：

1. 文件全部落盘
2. 每个文件非空
3. 本批关键骨架标题齐
4. 通过本批自检后，才能进入下一批

## 第 1 批：主报告批

### 必产文件

- `事实与推断台账.md`
- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`
- `_meta.json`

### 最低要求

- `事实与推断台账.md` 至少覆盖 `主体边界 / 时间边界 / 证据来源`，每条都有真实行号与锚点
- `拆文报告.md` 必须包含：
  - `样本分级与可学层判断`
  - `高敏桥段识别`
  - `题面拆解`
  - `故事梗概`
  - `结构划分`
- `情节节点.md` 必须覆盖全文主要节点，并逐节点写 `类型 / 情绪 / 涉及 / 状态变化 / 因果`
- `写作手法.md` 必须按固定 9 节骨架落盘
- `_meta.json` 必须有 `structure_counts`

## 第 2 批：16 张可直接仿写表批

### 必产文件

- `可直接仿写_导语拆解表.md`
- `可直接仿写_顺序事件表.md`
- `可直接仿写_物件表.md`
- `可直接仿写_动作表.md`
- `可直接仿写_对白功能表.md`
- `可直接仿写_对话衔接表.md`
- `可直接仿写_误判表.md`
- `可直接仿写_钩子表.md`
- `可直接仿写_微动作表.md`
- `可直接仿写_安静压迫场表.md`
- `可直接仿写_人物偏手表.md`
- `可直接仿写_失控说话表.md`
- `可直接仿写_烂关系漏出表.md`
- `可直接仿写_外部秩序表.md`
- `可直接仿写_公开炸场表.md`
- `可直接仿写_后果链表.md`

### 每张表都必须补的 3 段

1. `可直接借的承重结构`
2. `迁移顺序提醒`
3. `为什么这个顺序不能乱`

额外硬要求：

- 每张表至少 4 条有效资产，并含 `原文现象 / 原文证据 / 原文位置 / 原文例子 / 原文功能 / 原文怎么写 / 动作本体` 中至少 1 类证据列；允许按表类型使用自然列名
- `人物偏手表` 必须显式包含 `角色 / 稳定偏手`
- `误判表` 必须显式包含 `先误判了什么 / 从哪开始翻`
- `安静压迫场表` 必须显式包含 `场面压力来源` 或 `环境音`
- `烂关系漏出表` 必须显式包含 `具体漏出件`
- 三段都必须引用本表自己的具体条目
- 三段每段至少 2 条可施工说明，单句概括不放行
- `迁移顺序提醒` 必须写出该表专属的先后顺序
- `为什么这个顺序不能乱` 必须写清错序后的假点
- 不允许 16 张表共用统一提醒句

## 第 3 批：原文细节库批

### 必产目录

- `原文细节库/`

### 必产文件

- `场景细节库.md`
- `关系细节库.md`
- `情绪细节库.md`
- `对白细节库.md`
- `翻车细节库.md`
- `旧伤细节库.md`
- `动作细节库.md`
- `场面细节库.md`

### 每份细节库至少回答

1. 具体发生了什么
2. 这个细节为什么有用
3. 它压的是谁、压在哪
4. 后续能迁到什么新桥段
5. 它对应的角色 / 情绪 / 反转是什么

额外硬要求：

- 每个 `##` 小节都必须是具体事件，不准写成“某类场面”
- 同一文件不同小节不能大量复用同一句答案
- 必须能看出具体角色、具体动作或具体物件

### 必产文件

- `主冲突_副升级器.md`

- `平台适配提醒.md`
- `情绪母线.md`
- `新状态清单.md`
- `虐点对照细节.md`
- `样本分级与可学层.md`
- `高敏桥段识别.md`
- `作者DNA指纹.md`
- `仿写约束_禁写清单.md`
- `同桥段过检规则.md`

### 高敏桥最低额外要求

- `重大证据前隔开的现实后果`
- `尾声入口给了谁 / 为什么不给另一条线`
- `人物不同脸证据 / 谁先解释谁先压场 / 动作权限差`
- `高敏桥段识别.md`、`作者DNA指纹.md`、`同桥段过检规则.md` 都要显式写 `原文：` 证据行
- 如果原文同时出现 `公司/办公室 + 戒指 + 见血/受伤`，该桥必须贯通主报告、节点、顺序事件表、高敏桥、公开场资产和 profile_source

## 第 5 批：`profile_source + book.profile` 批

### 必产文件

- `写作资产/profile_source.md`
- `book.profile.json`

### 顺序

1. 先产 `profile_source.md`
2. 再运行 `generate_story_profile.py`
3. 再检查 `book.profile.json`

### 本批最低要求

- `profile_source.md` 必须补齐 `## 0-11`
- `## 7. 禁句 / 禁写法` 至少 2 条独立的 `- 为什么假：`
- `## 8. 场面资产` 至少补 `公开场硬件 / 外部秩序件 / 后果链`
- `## 9. 后果链` 至少补 `感情伤抬升到现实伤的节点 / 秩序回正节点 / 长尾惩罚节点 / 离场 / 换图节点`
- `## 10. 作者站位高危句` 至少补 `容易写成作者判词的句型 / 容易写成主题总结的句型 / 容易写成整齐揭露的句型`
- `## 11. style_assets 原始材料` 必须逐行提供 10 类短语资产，不写解释句
- 每个 `桥段：` 小节都要补齐 `原文怎么起手 / 承重件 / 不能丢的顺序 / 为什么这个顺序不能乱 / 最容易写假的点 / 原文为什么能过`
- 生成后的 `book.profile.json` 不能只有壳，至少确认：
  - 非空 `bridge_rules[*].must_keep`
  - 非空 `scene_assets`
  - 非空 `banned_phrases`
  - 非空 `author_stance_patterns`
  - 非空 `story_guardrails`
  - 10 类 `style_assets` 全部非空
  - `character_face_split` 三类拆脸字段全部非空
  - `consequence_structure` 四类后果字段全部非空

## 第 6 批：全量验收批

### 固定命令

```bash
python3 "$CODEX_HOME/skills/story-short-analyze/scripts/run_short_analyze_finalize.py" "拆文库/{书名}"
```

### 洁净硬闸

- 正式 Markdown 不得有重复标题、空字段、`桥1 / 桥段卡1 / 待补 / 占位` 标题
- 初始化脚本只建目录和元数据，不预填正式资产
- 通过后 `_progress.md` 必须全量勾选，`_meta.json.last_stage_in_progress` 必须为空

### 通过后才能说

- `ready-for-write`
- `可调用 story-short-write`
- `资料包已备齐`

### 最短执行话术

1. 按 6 批顺序自动落盘，不要跳批。
2. 凡是 skill 定义过的文件都要产出，不能省略。
3. 每批产完先自检，再继续下一批。
4. 最后一律运行 `run_short_analyze_finalize.py`，不过线不算完成。
