# 短篇拆文全量自动产出任务单

这份文件不是方法论，而是执行单。

用途：

- 当 `story-short-analyze` 进入正式拆书后，按快速厚拆批次自动产出整套文件
- 防止模型只写 `拆文报告.md / 情节节点.md / 写作手法.md` 就停
- 防止 16 张表、原文细节库、写作资产被遗漏

硬口径：

- 不允许用户不提，模型就默认少拆
- 不允许先只写“核心三件套”，剩余文件等后补
- 不允许验收前跳过任何阶段
- 主报告必须串行保厚；规则同构的资产允许大批量连续生成
- 读完原文后先落 `事实与推断台账.md`，资产阶段不得重复吞全文和样本

## 总顺序

固定按 11 个步骤执行：

1. 事实台账
2. 拆文报告
3. 情节节点
4. 写作手法 + `_meta.json`
5. 动态字典 + 候选池
6. 16 张表的两个 8 表大批
7. 8 份细节库整批
8. 常规写作资产整批
9. 高敏资产整批
10. `profile_source`，随后脚本单独生成 `book.profile`
11. 全量验收

每个批次都要满足：

1. 当前批次全部文件落盘
2. 每个文件非空
3. 当前文件关键骨架标题与原文证据齐
4. 批末统一做文件齐全、截断和厚度快检
5. 失败批次先二分重跑，二分仍失败才对责任子批降级为双文件模式
6. 通过当前批次后立即进入下一批

禁止：

- 同时规划 40 多个文件后再开始写
- 把主报告四件套合并成一次生成
- 把 16 张表或整个 `写作资产/` 压成统一壳
- 为追求批量输出主动降低表行、细节卡、有效字符或专项解释厚度
- 每个批次都运行全量 validator；全量 finalize 只在最后运行一次

## 第 1 组：主报告串行批

### 必产文件

- `事实与推断台账.md`
- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`
- `_meta.json`

固定拆成四次落盘：事实台账；拆文报告；情节节点；写作手法与 `_meta.json`。不得合并。

### 最低要求

- `事实与推断台账.md` 至少覆盖 `主体边界 / 时间边界 / 证据来源`，每条都有真实行号与锚点
- `拆文报告.md` 必须包含：
  - `样本分级与可学层判断`
  - `高敏桥段识别`
  - `题面拆解`
  - `故事梗概`
  - `结构划分`
- `拆文报告.md` 还必须保住主报告解释层：
  - `题面拆解 / 结构划分 / 人物分析 / 情感曲线 / 反转分析` 至少各补 1 组 `原文现象证据 -> 为什么成立 -> 新稿最容易写假的点`
  - 中段承重桥如果存在，必须先在主报告里解释它如何把表层冲突推到更深后果
  - 不允许把“为什么原文能过”的解释全部后推给 `profile_source.md / 桥段施工卡.md / 原文细节库`
  - 如果主报告明显比资产层更像“摘要”，默认返回第 1 批重写，不进入后续批次
- `情节节点.md` 必须覆盖全文主要节点，并逐节点写 `类型 / 情绪 / 涉及 / 状态变化 / 因果`
- `写作手法.md` 必须按固定 9 节骨架落盘
- `_meta.json` 必须有 `structure_counts`

## 第 2 组：16 张可直接仿写表快速批

### 前置必产文件

- `写作资产/原文资产候选池.md`
- `写作资产/本书动态信号字典.json`

先按全部 Chunk 建立本书动态信号字典，再做第二遍分类回扫和候选池。表格完成后先回补字典，再重扫全文并补登记遗漏。

### 必产文件

- `可直接仿写_导语拆解表.md`
- `可直接仿写_顺序事件表.md`
- `可直接仿写_物件表.md`
- `可直接仿写_动作表.md`
- `可直接仿写_对白功能表.md`
- `可直接仿写_对话衔接表.md`
- `可直接仿写_误判表.md`
- `可直接仿写_钩子表.md`
- `可直接仿写_微动作表.md`
- `可直接仿写_安静压迫场表.md`
- `可直接仿写_人物偏手表.md`
- `可直接仿写_失控说话表.md`
- `可直接仿写_烂关系漏出表.md`
- `可直接仿写_外部秩序表.md`
- `可直接仿写_公开炸场表.md`
- `可直接仿写_后果链表.md`

### 每张表都必须补的 3 段

1. `可直接借的承重结构`
2. `迁移顺序提醒`
3. `为什么这个顺序不能乱`

默认只分两批：

1. 结构/物件/动作批：`导语+顺序+物件+动作+误判+钩子+微动作+安静压迫`
2. 对白/人物/关系/秩序批：`对白功能+对话衔接+人物偏手+失控说话+烂关系漏出+外部秩序+公开炸场+后果链`

每批结束后统一检查 8 张表是否齐全、是否截断、最低资产行是否达标、三段施工层是否存在。失败只二分当前批，不重跑已通过批次。

额外硬要求：

- 候选池识别出的独立资产必须全部落表或写具体不收录理由；无对应资产的表明确写“原文未发现”
- 不按篇幅强制每表条数；必须含原文证据列与逐行迁移列，允许按表类型使用自然列名
- 候选池全部 `已收录` 项必须能在目标表搜到资产名或原文锚点
- 候选池必须覆盖全部 Chunk，并完成 `物件替换对 / 微动作角色覆盖 / 对白侵占与假道歉 / 安静等待与未归 / 未来公开事件钩子` 五项专项回扫
- 动态字典必须完成 `首次全文发现 / 表后回扫`，新增词关联候选或写明仅索引理由，最终 `stabilized=true`
- 逐行迁移列使用 `迁移提醒 / 可迁移写法 / 可替换功能件 / 后续写法提醒` 等自然列名，每行必须非空
- `人物偏手表` 使用原文实际角色，至少与主报告人物分析存在交集；先写角色总偏手，再补事件偏手
- `人物偏手表` 必须显式包含 `角色 / 稳定偏手`
- `误判表` 必须显式包含 `先误判了什么 / 从哪开始翻`
- `安静压迫场表` 必须显式包含 `场面压力来源` 或 `环境音`
- `烂关系漏出表` 必须显式包含 `具体漏出件`
- 三段都必须引用本表自己的具体条目
- 三段每段至少 2 条可施工说明，单句概括不放行
- `迁移顺序提醒` 必须写出该表专属的先后顺序
- `为什么这个顺序不能乱` 必须写清错序后的假点
- 不允许 16 张表共用统一提醒句

## 第 3 组：原文细节库快速批

### 必产目录

- `原文细节库/`

### 必产文件

- `场景细节库.md`
- `关系细节库.md`
- `情绪细节库.md`
- `对白细节库.md`
- `翻车细节库.md`
- `旧伤细节库.md`
- `动作细节库.md`
- `场面细节库.md`

默认 8 份一次连续落盘。若批末快检发现缺件、截断或证据卡不足，先拆成 `场景+关系+情绪+对白` 与 `翻车+旧伤+动作+场面` 两批；仍失败才降级为双文件。

### 每份细节库至少回答

1. 具体发生了什么
2. 这个细节为什么有用
3. 它压的是谁、压在哪
4. 后续能迁到什么新桥段
5. 它对应的角色 / 情绪 / 反转是什么

额外硬要求：

- 证据卡数量按原文字数分档：8000 字以上每类至少 5 张、5000-7999 字每类至少 4 张、5000 字以下每类至少 3 张
- 数量是资产保全下限；发现新的独立事件卡时追加，不准为了凑分类把已识别资产覆盖掉
- 每个 `##` 小节都必须是具体事件，不准写成“某类场面”
- 同一文件不同小节不能大量复用同一句答案
- 必须能看出具体角色、具体动作或具体物件

## 第 4 组：写作资产快速批

### 必产目录

- `写作资产/`

### 必产文件

- `母结构_故事走法.md`
- `主冲突_副升级器.md`
- `异物清单.md`
- `第二层冲突清单.md`
- `角色口气模板.md`
- `关系重组方式.md`
- `公开场_关键硬牌_后果.md`
- `平台适配提醒.md`
- `情绪母线.md`
- `新状态清单.md`
- `虐点对照细节.md`
- `样本分级与可学层.md`
- `高敏桥段识别.md`
- `作者DNA指纹.md`
- `仿写约束_禁写清单.md`
- `同桥段过检规则.md`

默认分两批连续落盘：

1. 常规资产：前 11 份结构、人物、关系与情绪资产
2. 高敏资产：`样本分级与可学层 / 高敏桥段识别 / 作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则 / 桥段施工卡`

每份仍独立使用原文证据，不允许批量复用同一抽象句。批末快检失败时只二分责任批，二分仍失败才降级为双文件。

### 高敏桥最低额外要求

- 每张卡显式写 `桥段角色`
- `桥段角色` 从本书实际功能动态归纳，不预设所有题材都有掉位、私域旧伤或公开炸场
- `重大证据前隔开的现实后果`
- `尾声入口给了谁 / 为什么不给另一条线`
- `人物不同脸证据 / 谁先解释谁先压场 / 动作权限差`
- `高敏桥段识别.md`、`作者DNA指纹.md`、`同桥段过检规则.md` 都要显式写 `原文：` 证据行
- 候选池与事实台账共同确认的核心承重桥，必须贯通主报告、节点、对应仿写表、高敏桥资产和 profile_source

## 第 5 组：`profile_source + book.profile` 微批

### 必产文件

- `写作资产/profile_source.md`
- `book.profile.json`

### 顺序

1. 单独产 `profile_source.md` 并局部验收
2. 单独运行 `generate_story_profile.py`
3. 检查 `book.profile.json`

### 本批最低要求

- `profile_source.md` 必须补齐 `## 0-12`
- `## 7. 禁句 / 禁写法` 至少 2 条独立的 `- 为什么假：`
- `## 8. 场面资产` 必须逐行使用 `scene_assets.public_explosion / scene_assets.external_order / scene_assets.consequence_chain`
- 三类场面资产数量按原文字数分档：8000 字以上至少 `4 / 4 / 6`，5000-7999 字至少 `3 / 3 / 4`，5000 字以下至少 `2 / 2 / 3`
- `public_explosion / external_order` 每类至少 2 条是事件级短语，写出动作或后果，不能全是地点、手机、戒指、公司等硬件名
- `## 9. 后果链` 至少补 `感情伤抬升到现实伤的节点 / 秩序回正节点 / 长尾惩罚节点 / 离场 / 换图节点`
- `## 10. 作者站位高危句` 至少补 `容易写成作者判词的句型 / 容易写成主题总结的句型 / 容易写成整齐揭露的句型`
- `## 11. style_assets 原始材料` 必须提供 10 类固定字段；原文无对应资产时写“原文未发现”，不写解释句或换壳词
- `## 12. 迁移替换资产` 必须逐行提供 5 类换壳资产，与原文 style assets 分开；每类至少 `4 / 3 / 2`
- 每个 `桥段：` 小节都要补齐 `原文怎么起手 / 承重件 / 不能丢的顺序 / 为什么这个顺序不能乱 / 最容易写假的点 / 原文为什么能过`
- 每个 `桥段：` 小节还要补 `桥段角色`，并与 `桥段施工卡.md / 高敏桥段识别.md` 使用同一角色口径
- 生成后的 `book.profile.json` 不能只有壳，至少确认：
  - 非空 `bridge_rules[*].must_keep`
  - 非空 `scene_assets`
  - 非空 `banned_phrases`
  - 非空 `author_stance_patterns`
  - 非空 `story_guardrails`
  - 10 类 `style_assets` 键齐全，原文有资产的类别已完整抽取，无资产的类别保持空数组
  - 5 类 `migration_assets` 每类唯一短语数达到当前字数档的 `4 / 3 / 2`
  - `character_face_split` 三类拆脸字段全部非空
  - `consequence_structure` 四类后果字段全部非空

## 第 6 组：全量验收

### 固定命令

```bash
python3 "$CODEX_HOME/skills/story-short-analyze/scripts/run_short_analyze_finalize.py" "拆文库/{书名}"
```

### 洁净硬闸

- 正式 Markdown 不得有重复标题、空字段、`桥1 / 桥段卡1 / 待补 / 占位` 标题
- 初始化脚本只建目录和元数据，不预填正式资产
- 通过后 `_progress.md` 必须全量勾选，`_meta.json.last_stage_in_progress` 必须为空

### 通过后才能说

- `ready-for-write`
- `可调用 story-short-write`
- `资料包已备齐`

### 最短执行话术

1. 按快速厚拆批次自动落盘，不要跳批。
2. 凡是 skill 定义过的文件都要产出，不能省略。
3. 主报告串行；16 表按 8+8、细节库按 8、资产按常规+高敏批量生成，失败才逐级降级。
4. 最后一律运行 `run_short_analyze_finalize.py`；不过线只定点返修责任批次。
