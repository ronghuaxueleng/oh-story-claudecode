
# 网文质量检查清单

> **定位**：story-short-analyze（短篇拆书）的质量自检清单，评估拆解对象的成文质量时逐项核对。
> 拆书管道本身的质量门控见 `output-templates.md`「质量门控必填字段」；所有数值阈值的唯一权威定义见 `material-decomposition.md`「质量标准」与「节奏分析」，本清单不自定义数值。

## 一、通用检查

### 章节结构
- [ ] 开头有钩子（不是天气/风景/日常开场）
- [ ] 中段有推进（有事件发生）
- [ ] 局势有变化（读完这章，世界跟之前不一样了）
- [ ] 结尾落在变化上（不是总结）

### 开篇检查（前 300-500 字）
- [ ] 有钩子，能抓住注意力
- [ ] 不从天气/风景/日常开始
- [ ] 主角快速出场
- [ ] 卖点或危机可见
- [ ] 前三句各自有功能，不是三句一个意思
- [ ] 第一屏不只是交代处境，还给了结果期待或危险预期

### 章节推进
- [ ] 有核心事件
- [ ] 局势有变化
- [ ] 不是水字数（删掉这章会影响理解吗？不会 = 水了）
- [ ] 推进了主线/关系/设定中的至少一项
- [ ] 中段不是只重复受气，而是又加了一层事，或者人物进了新状态

### 信息传递
- [ ] 没有大段设定说明文
- [ ] 信息跟着冲突走（通过事件传递设定）
- [ ] 设定量可控（一章不超 3 个新概念）

### 场景检查
- [ ] 场景有目标（人物要什么）
- [ ] 场景有阻碍（什么挡着）
- [ ] 场景有变化（结束后跟之前不同）
- [ ] 人物在做事情，不是在感觉事情
- [ ] 没有可删除的段落
- [ ] 主角不是纯承受，至少出现过关键非标反应或主动动作

### 章尾
- [ ] 结尾落在变化上
- [ ] 有危机/决定/发现/反转中的至少一个
- [ ] 不是总结式结尾
- [ ] 拉住读者翻下一页
- [ ] 结尾不是只把结果说完，而是让位置、关系或认知发生了变化

### 语言
- [ ] 没有空洞的抒情段落
- [ ] 没有连续多段同一情绪
- [ ] 对话符合人物身份（不同人说话方式不同）
- [ ] 情绪通过动作落地（不是直接说"他很难过"）
- [ ] 强情绪不只靠抽象词，能看见眼神、身体、说话或动作变化

### 连载连续性
- [ ] 没有遗忘之前的承诺/伏笔
- [ ] 没有突然塞入大量新设定
- [ ] 伏笔有推进
- [ ] 故事引擎还在运转

### 水（filler）检测
以下信号出现 = 可能水了：
- 全章对话没有任何新信息
- 同一个情绪写了 3 段以上
- 场景描写超过 500 字但不推进剧情
- 角色回忆之前发生的事但没有任何新视角
- 连续 2 章以上没有冲突
- 连续几段只是同类羞辱换说法，没有第二层升级

## 二、长篇专项

### 黄金三章检查
- [ ] 第一章前 500 字有钩子？
- [ ] 主角第一章就出场？
- [ ] 第一章有事件（不是纯铺垫）
- [ ] 第二章有升级（矛盾加深）
- [ ] 第三章有追读理由（给出继续看的动力）
- [ ] 前三章有至少 2 个爽点？
- [ ] 世界观没有大段说明文？
- [ ] 每章结尾有悬念？

### 节奏检查
- [ ] 最近 5 章是否有明确进展？
- [ ] 爽点间隔是否超过 5000 字？
- [ ] 有没有连续 2 章以上没有冲突？

### 人物检查
- [ ] 主角行为符合人设？
- [ ] 配角是否有存在感？
- [ ] 反派逼格是否匹配当前阶段？

### 五维评分标准

每个维度 0-100 分，根据评分结果选择精修策略。

### 维度 1：核心一致度
检查：关键冲突、关键行动、人物动机是否前后一致。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 人物动机突然改变无铺垫 | critical | 补充动机转变的触发事件 |
| 核心冲突前后不一致 | high | 回溯修改冲突设定 |
| 关键行动与人物性格矛盾 | high | 调整行动或补充解释 |
| 次要矛盾遗忘 | medium | 回收或弱化 |

### 维度 2：表层重写度
检查：句式与措辞是否足够原创，避免套路化表达。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 照搬原句导致语气不自然 | medium | 只在确有 AI 腔时改写，正常原句可保留 |
| 大量使用 AI 标志词 | high | 替换为具体描述 |
| 同一句式重复出现 | medium | 变换表达方式 |
| 描写过于文学化 | medium | 改为口语化/动作化 |

### 维度 3：格式一致度
检查：段落结构、字数分配、开头结尾格式是否统一。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 段落长度差异过大 | medium | 调整段落划分 |
| 章节字数偏离目标 | **high** | 写作/大纲修复时先回到细纲补足计划内情节点，再展开；去AI味已有正文时不得新增剧情 |
| 格式混乱（对话/描写不统一）| low | 统一格式 |

### 维度 4：可读性
检查：是否有啰嗦、AI 腔、空泛总结、套路修辞。

| AI 腔特征 | 如何识别 | 如何修复 |
|-----------|----------|----------|
| 空泛总结 | 「他终于明白了」「一切尽在不言中」 | 删掉，用行动代替 |
| 套路修辞 | 「命运仿佛在和他开玩笑」 | 删掉或换成具体描述 |
| 情绪标签 | 「他感到一阵悲伤」 | 改为行为表现 |
| 大段心理描写 | 连续 3 句以上内心独白 | 压缩为 1 句或转为对话 |

### 维度 5：逻辑连贯
检查：句间/段间是否通顺，有无设定冲突。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 设定前后矛盾 | critical | 查找并统一设定 |
| 时间线错误 | high | 标记时间线并修正 |
| 角色信息不一致 | high | 建立角色档案对照 |
| 因果链断裂 | medium | 补充过渡 |

### 精修策略

根据五维评分结果，选择精修策略：

| 主要问题 | 策略 | 说明 |
|----------|------|------|
| 核心一致度低 | rewrite | 围绕核心冲突重写相关段落 |
| 字数超标 | compress | 删减不推动剧情的内容 |
| AI 腔重 | de_ai | 替换禁用词、改写句式 |
| 小问题多 | polish | 打磨语言细节 |

## 三、短篇专项

### 仿写 / 融合可用度检查

如果这篇拆文后面不是只做欣赏分析，而是要继续拿去：

- 仿写
- 融合
- 去原作化后再写

那就不能只看“故事核、情绪线、反转机制有没有拆出来”，还要看有没有拆到能直接拿去写。

### 全局成文形状检查

- [ ] 是否判断整本章节弧线是否同构、是否像大纲逐章展开
- [ ] 是否逐项检查章尾收束是否反复使用金句、象征物或总结句
- [ ] 是否判断主角有没有持续做最优决策，以及题材允许的迟疑、误判、失控或代价
- [ ] 是否区分真正改变判断/权限/风险/资源/后果的专业细节，与只营造真实感的术语
- [ ] 是否检查全文对白是否反复走“提问—回答—确认—解释”回路
- [ ] 五项判断是否都有原文行号或可核验短句，并区分可学层、禁学层和迁移提醒
- [ ] 脚本检查与模型人工裁决是否分别留痕，不能用词频或正则结果替代语义判断

这组检查通过后，才算达到 `可直接仿写层 = ready-for-write`。如果没过，不要把这篇拆文判成“已备料完成”。

新增硬口径：

- 不只检查“有没有几份关键资产”，而是检查 skill 定义过的整套文件是否都已自动落盘
- 少任意一个定义文件，默认仍是 `blocked-on-assets`
- 最后必须跑 `run_short_analyze_finalize.py`，收口脚本不过线，默认还没拆完
- 主会话必须按 `short-analyze-execution-prompt.md` 的 6 批顺序执行，不能自由跳批
- 正式拆书前必须先看 `references/examples/INDEX.md`，按失败类型选 1-2 本样本，不允许无差别吞全部样本
- 必须落 `_sample_comparison.md`，记录每本样本实际读取文件、正反例锚点和主报告后复核
- 禁止使用 bak、上一本文档、旧 profile 或其他拆书目录替代内置样本
- finalize 禁止补写、扩写或修复 Markdown；前后 SHA1 必须一致
- 细节库如果还是“原文里出现了某类场面”“可迁到同题材桥段”这种模板壳，也默认不过线
- `book.profile.json` 如果抽出空桶、碎片词、空 `must_keep`，也默认不过线
- 16 张 `可直接仿写_*.md` 如果施工层不引用本表具体条目，也默认不过线
- 16 张 `可直接仿写_*.md` 如果多张共用统一提醒句，也默认整批不过线
- 16 张表如果没有逐行迁移列，或逐行迁移内容为空，也默认不过线
- `人物偏手表` 如果只列事件、不覆盖角色级稳定偏手，也默认不过线
- `book.profile.json` 如果没有独立 `migration_assets`，或把迁移替换件混进原文 `style_assets`，也默认不过线

必查项：
- [ ] 已明确这篇属于什么流，不只是题材名，而是成文流法（如追妻情绪流/全员背叛流/婚恋清算流）
- [ ] 已按 `references/examples/INDEX.md` 选定 1-2 本内置样本，并知道本次是拿它们防什么失败
- [ ] `_sample_comparison.md` 已记录每本样本的 `README / 原文 / 正反例对照` 路径
- [ ] `_sample_comparison.md` 已记录具体正例锚点、反例锚点、本书风险和受影响文件
- [ ] 主报告完成后已回看样本反例区，并记录 `未滑入反例 / 需要回炉` 裁决
- [ ] 没有读取 bak、上一本文档、旧 profile 或其他拆书目录充当样本
- [ ] 已拆出主梗、1-3 个副梗，以及它们各自负责开头/中段/高潮哪一层功能
- [ ] 已能用一句组合公式概括这篇（如“全员背叛 + 装穷 + 女兄弟梗”）
- [ ] 已有 `导语拆解表`，知道前 20 / 60 / 80 字分别完成了什么
- [ ] 已有 `顺序事件表`，能顺着说清原文怎么一步步往前推
- [ ] 已有 `物件表`，知道关键物件伤的是体面、旧伤、亲情、身份，还是关系位置
- [ ] 如果原文出现 `副驾驶 / 家属栏 / 花束 / 黄玫瑰 / 离婚协议 / 礼物` 这类主位或证据物件，`物件表` 已优先保住，不是只散落在别的表里

- [ ] 已有 `对白功能表`，知道不同角色的话是在挑衅、压人、找补、装无辜，还是切割关系

- [ ] 已有 `安静压迫场表`，知道不靠大吵也能怎么把场压住
- [ ] 已有 `样本分级与可学层`，知道这本是 `A类正样本 / B类骨架样本 / C类负样本`
- [ ] `原文细节库/` 8 类细节库已全部自动落盘，不是只补了其中 2-3 类
- [ ] 每类细节库的独立证据卡达到原文字数分档：8000 字以上 5 张、5000-7999 字 4 张、5000 字以下 3 张
- [ ] `原文细节库/` 不是模板壳，至少拆出具体场面、具体物件或具体动作，不是统一泛化句
- [ ] 每份 `原文细节库/*.md` 的多个 `##` 小节没有复用同一句答案只换标题

### 虐爽节奏检查

短篇特有的检查项——虐点和爽点的分布是否合理：

```
理想分布：虐1 → 虐2 → 虐3 → 爽1(小) → 虐4 → 爽2(大)
禁忌分布：虐1 → 虐2 → 虐3 → 虐4 → 虐5（无爽点，读者流失）
禁忌分布：爽1 → 爽2 → 爽3 → 爽4（无铺垫，爽感疲劳）
```

检查规则：
- 每 300 字至少 1 个小冲突
- 虐点间隔不超过全文的 30%
- 最大爽点必须出现在全文的 70-85% 位置
- 结尾必须有情绪落点
- 爽点不能只靠处刑，最好带位置变化、迟到惩罚或价值抬高

### 反套路检查

- [ ] 表面题材明确，但真正抓人的地方不是只靠题材本身
- [ ] 有真正特别的地方，不是常规桥段直推到底
- [ ] 中段又加了一层事，不是单线打脸
- [ ] 主角至少有 1 处非标反应
- [ ] 高潮不只是兑现，还能再往上抬一下，或者让人回头重看前文
- [ ] 结尾不只是处刑，还把整篇文往上抬了一下
- [ ] 题面和导语先扎中了关系错位或最想看的后果，不是平铺背景
- [ ] 如果目标是后续仿写，已经能从拆文里直接抽到“事件 / 物件 / 动作 / 口气 / 误判 / 钩子”，而不是只能抽概括结论
