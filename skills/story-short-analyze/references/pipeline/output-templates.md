# 短篇拆文输出模板

> 拆文输出时加载。先看决策路由选 Stage，再按模板填写。方法论细节见 `material-decomposition.md`。

## 决策路由

## 全量自动产出顺序

进入正式拆书后，不按“想到什么写什么”，固定按下面顺序自动落盘：

1. `拆文报告.md`
2. `情节节点.md`
3. `写作手法.md`
4. `_meta.json`
5. 16 张 `可直接仿写_*.md`
6. `原文细节库/` 8 类
7. `写作资产/` 全包
8. `写作资产/profile_source.md`
9. `写作资产/桥段施工卡.md`
10. `book.profile.json`
11. `run_short_analyze_finalize.py` 收口验收

硬口径：

- 这不是建议顺序，是固定顺序
- 后一批不能在前一批没落盘时先产
- 任意一批缺件，默认不能继续下一批
- `人类写作可用性` 高于 `格式合规表象`；格式合规只是不被挡在门外，不代表已经拆好
- 如果某种写法“更好过 validator，但更难给写手直接拿去用”，默认选后者，再补最少量结构锚点
- 禁止把人工阅读判断伪装成脚本检测结果；没跑检测时，风险、分段、最高块、整体分数一律写 `未测 / 未知 / 人工判断：...`
- 正式执行时主会话同步加载 `short-analyze-execution-prompt.md`
- 16 张 `可直接仿写_*.md` 的施工层不允许复用统一通用句
- 16 张表必须逐行写迁移字段；表后 3 段不能代替逐行迁移提醒
- 原文资产与换壳替代件必须分栏，禁止把迁移词混进原文证据列或 profile 的 style assets
- `迁移顺序提醒 / 为什么这个顺序不能乱` 必须点名引用本表里的具体条目，不能只写“先保留功能顺序”
- `可直接借的承重结构 / 迁移顺序提醒 / 为什么这个顺序不能乱` 三段里，优先直接复用表格条目原名；如果只写抽象概括、不写表内资产名，默认该表未完成
- 对 `顺序事件表 / 动作表 / 对话衔接表 / 微动作表` 这类顺序承重更强的表，施工层最好直接复用整条表格原句、原链条或完整条目，不要只提炼成关键词
- 对 `导语拆解表 / 误判表` 这类看似概括表的文件，施工层也必须直接复用表格里的栏目值或完整单元格文本，不能把 `前20字 / 前60字 / 前100字`、`只是... / 实际... / 翻点...` 再抽象成二级概念
- `为什么这个顺序不能乱` 里引用顺序关系时，也必须复用表格里的原条目名，不接受把表里条目缩写成“婚嫁主位 / 后半清算 / 公开翻盘 / 大坏招”之类抽象代称
- `原文细节库/` 每个细节条目必须落到具体事件、物件、动作、角色或关系掉位，不能写成“这一类场面都可以迁移”
- 只要出现“原文里出现了这类场面”“可迁到同题材桥段”“对应三角关系”这类泛化壳句，默认该文件未完成
- 细节库第 4 条标题虽然固定是 `后续能迁到什么新桥段`，正文里也尽量不要反复写 `可迁到...`，优先写成 `能迁去...，前提是保留...`
- `写作手法.md` 不能写成薄概括，每个核心章节至少要能看见：`人物嘴型/口气差`、`章法失效测试`、`物件铰链或意象物件`、`迁移风险`
- `高敏桥段识别` 不能只挑最亮桥，至少保留 1 条中段关键过桥桥
- `拆文报告.md` 的 `题面拆解` 不能只写题材、流派和主梗；至少显式回答 `读者最先以为什么文` 与 `后面哪一步改写了这个预期`
- `拆文报告.md` 的 `题面拆解 / 结构划分 / 人物分析 / 情感曲线 / 反转分析` 至少各补 1 组 `原文现象证据 -> 为什么成立 -> 新稿最容易写假的点`
- `可直接仿写_顺序事件表.md` 不能只剩开头链和终局链，默认至少 4 条链，且至少包含 1 条中段承重链
- `写作资产/profile_source.md` 不能只剩结构桶；在保证可抽 json 的同时，还要补 `句长切法 / 口气差 / 动作替代 / 旧伤触发器 / 反面句型` 这类人类可读层
- `写作资产/profile_source.md` 仍以抽取稳定性优先；更厚的“为什么原文能过 / 为什么不像加工稿 / 新稿最容易写假的点 / 顺序为什么不能乱”默认写进 `写作资产/桥段施工卡.md`
- 如需 few-shot 参照，先读 `references/examples/INDEX.md` 再按失败类型选样本，不要每次都默认只看第一本
- 如需 few-shot 参照，优先对照 `references/examples/yuwei/正反例对照.md`：那份样本专门展示“细节库缩水”和“题面拆解失去预期翻转”这两类常见失败
- 如需“公权场 / 私域越界 / 证据链清算”类参照，补看 `references/examples/saohuang/正反例对照.md`，避免把中段桥误压成普通抓奸吵架
- 如需“偏心剥夺链 / 换亲秩序 / 掉马反压 / 回门清算”类参照，补看 `references/examples/guiyue/正反例对照.md`，避免把古言向只拆成换亲和打脸标签
- `写作资产/桥段施工卡.md` 不能只写桥名和结论；每张卡都必须是可直接指导写作的施工卡
- `拆文报告.md` 不只保留基础结构层；如果目标是后续仿写/融合，还要补出 `情感曲线 / 爆点分析 / 反转分析 / 人物分析 / 开头分析 / 结尾分析 / 五维评分`
- `拆文报告.md` 的关键判断优先写“原文现象证据 -> 为什么成立 -> 新稿最容易写假的点”，不能只停留在题材结论
- 原文确实存在终局预热、已宣布事件或硬牌上桌前等待时，`情节节点.md` 必须保留；没有时不得强造
- `人物分析` 默认要回答：主角是否也在借关系、借影、借局或反向利用原关系；如果有，不能抹平成纯受害者脸

## Stage 2 全篇结构

### Stage 2A 事实与推断台账

正式写 `拆文报告.md` 前先产 `事实与推断台账.md`：

```text
F01 | L起-L止 | 锚点：原文短语 | 类别：主体边界 | 主体：角色 | 动作：原文动作 | 结果：原文结果 | 叙述时点：正文当前位置 | 故事时点：事件实际发生位置 | 时间依据：回指词或顺序证据 | 口径：原文明确 | 禁止越界：不能把“选择利用”写成“推动形成”
```

最低覆盖：

- `主体边界`：谁真正做了什么，谁只是接受、利用或等待
- `时间边界`：原文只说“那天晚上”时，禁止擅自写成“婚礼前一天”
- `双时间轴`：正文晚讲不等于事件晚发生；遇到“那次同学聚会结束后”一类回指，必须定位它指向的早期事件
- `证据来源`：匿名寄来、偶然发现、主动搜集必须分开

正式产物出现高主动性因果词时，句末必须标 `【原文明确 Fxx】` 或 `【人工推断 Fxx】`。

字数{X}字 | 节数{N} | 平台{平台} | 类型{题材/情绪类型} | 结局{HE/BE/开放式} | POV{第一/第三人称}

- 原文总行数：{与 `_source_manifest.json.line_count` 一致}
- 已读取至：{必须等于原文总行数}
- 识别章节数：{与 `_source_manifest.json.chapter_count` 一致；无章节标记填 0}
- 最后事件：{原文实际最后发生的事件，不是自拟“终局前夜”}

- 本节缺失时不得进入 Stage 2
- 尾部锚点必须能在原文最后 10% 找到
- 禁止只依据单次截断输出判断章节数、结局或最后事件

输出格式硬要求：

- 每一桥都必须从 `- 桥段名：` 起一整张独立卡片，不准先吐空骨架再在后面重写一遍
- 不允许出现 `桥1 / 桥2` 这种未填写占位标题后，又重复输出正式版本
- 整个小节最终只保留一套有效卡片；发现草稿壳、半成品壳、重复卡片，视为未完成
- 每张卡片都必须填满全部字段，不能只写 `原文：...` 两三句概述代替
- 如果原文高敏桥多于 4 条，可以多写；如果少于 4 条，也要保证每条都写成完整卡片
- 卡片数量服从原文真实桥段，不按字数凑数；桥段角色使用本书动态标签

硬口径补充：

- 不能只选“开头最亮”和“结局最炸”两头桥段
- 至少补 1 条中段承重桥，优先选那种负责“从公开掉位推到私域重伤”或“从情绪冲突推到现实后果”的过桥桥
- 如果原文真正稳的不是大炸桥，而是中段慢压桥，宁可少写亮桥，也不能漏掉这类承重桥
- 如果出现“办公室冲突 / 公开见血 / 病情暗示 / 搬出家门”这类中段承压桥，优先纳入，不要被导语桥和终局桥挤掉
- 如果高敏桥名额有限，中段承压桥优先级高于“额外再补一条导语桥”或“额外再补一条终局桥”
- 如果原文里存在 `办公室冲突 / 物件争位 / 见血 / 递进式私域驱逐` 这类可迁移价值很高的中段场面桥，优先级默认高于“再补一条更亮但更难迁的导语桥”

### `profile_source.md` 抽取硬口径

`profile_source.md` 不是题材摘要，它还是后续 `book.profile.json` 的结构化来源。

因此正式落盘时，必须额外保证：

- 每个 `桥段：...` 小节都显式包含：
  - `原文怎么起手`
  - `承重件`
  - `不能丢的顺序`
  - `为什么这个顺序不能乱`
  - `最容易写假的点`
  - `原文为什么能过`
  - `情绪进入点 / 刺痛或受辱拍 / 短暂希望或反抗 / 反刀拍 / 峰值拍 / 场末余痛`
- 六拍中的每一拍统一写成 `{具体动作或处境} | 烈度：1-10 | 原文证据：Lx-Ly 可核验短句`
- `作者DNA` 段至少显式补：
  - `人物不同脸证据`
  - `谁先解释谁先压场`
  - `不同角色的动作权限差`
- 在 `作者DNA` 段或平行的人类说明层里，还要显式补：
  - `句长切法`
  - `口气差`
  - `动作替代`
  - `旧伤触发器`
  - `反面句型`
- `后果链` 段至少显式补：
  - `重大证据前隔开的现实后果`
  - `后果回灌方式`
  - `尾声入口归属`
  - `不给另一条线的原因`

同时必须兼顾两层用途：

- 对脚本：字段名稳定、标签稳定、能抽成 `book.profile.json`
- 对人：读完就能直接知道“原文怎么做才成立、换成什么就会写假”

如果只剩抽取桶、没有人类可读层，常见后果是：

- 虽然能过 validator，但写手拿去不好用
- 容易产出“合规但压缩化”的拆书结果
- 下游仿写时会只拿到标签，拿不到真正的成文手感

如果缺这些标签，`book.profile.json` 常见后果是：

- `bridge_rules[*].must_keep` 为空
- `story_guardrails` 缺失
- 最终 `run_short_analyze_finalize.py` 卡在验收阶段

进一步硬口径：

- `book.profile.json` 不应只剩“能过 validator 的最小 JSON”
- 默认至少保住 3 条桥规则，且每条都有：
  - `opening_pattern`
  - `must_keep`
  - `must_avoid`
  - `fake_signals`
  - `recommended_sequence`
  - `why_order_matters`
  - `why_original_passes`
  - `emotion_sequence`，固定包含六拍的 `beat / content / intensity / source_evidence`
- 如果 `profile_source.md` 已经写出这些层，而 `book.profile.json` 只抽出 `must_keep`，视为抽取链退化，不算合格厚度

## Stage 2B 情节节点提取

### 节点模板


## 16 张可直接仿写表施工层硬规则

每张 `可直接仿写_*.md` 在表格后补的 3 段，不是装饰说明，而是施工指令。

### 一律必须做到

- 不按篇幅强制每表条数；候选池资产全部核销，无资产的类别明确写“原文未发现”
- 每张表至少含 1 个逐行迁移列：`迁移提醒 / 可迁移写法 / 可替换功能件 / 后续写法提醒`
- 逐行迁移列每一行都必须非空，并写这条资产自己的换壳方法，不能五行共用同一句
- 表格字段必须服从表型，不得把 16 张表机械统一成 `资产 / 原文位置 / 原文证据 / 原文功能`
- `可直接借的承重结构`：必须点名本表至少 2 个具体条目，写清哪几个是骨钉，不能只写“大概顺序”
- `迁移顺序提醒`：必须写成“先 A，再 B，最后 C”的本表专属顺序；A/B/C 都要能在表格里找到对应资产
- `为什么这个顺序不能乱`：必须解释“如果把 B 抢到 A 前面，会假在哪里”；不能只写“乱了像功能按钮”
- 施工层必须像“只对这张表成立”的施工说明：去掉表名后，读者也应该能从资产名判断它只属于这一张表
- `为什么这个顺序不能乱` 最好直接写成“如果先放 X，再少了 Y，会丢掉什么 / 会假成什么”，不要写抽象判断句
- `顺序事件表` 还要显式保留 1 条中段承重链，不能只写起手链和收尾链
- `人物偏手表` 先覆盖角色级稳定偏手，再补事件级证据；8000 字以上默认至少 4 个不同角色
- 导语前 20 行存在群体定性时，`导语拆解表` 必须单独保留公共命名信号
- 终局公开身份场存在证据载体时，`物件表` 必须保留该终局硬牌
- 中段承重链优先写成完整链条，例如“先公开掉位 -> 再现实后果压身 -> 最后私域再受伤”，不要缩成“中段升级”
- 默认至少写 4 条事件链：`起手链 / 中段承压链 / 私域或旧伤深化链 / 终局清算链`
- 8000 字以上样本里，`物件表 / 动作表 / 对白功能表 / 钩子表 / 微动作表 / 失控说话表` 默认至少写 5 条独立资产；`公开炸场表` 默认至少写 4 条
- 如果这些表只写到 3 条左右，默认先回候选池复扫：通常不是原文没有，而是把次级承重件压进了解释段
- 如果全文天然不足 4 条大链，也要明确说明“为何不足 4 条”，不能静默压缩成 2-3 条
- 中段承压链必须写成至少 3 段箭头链，例如“先办公室流血 -> 再病情暗示 -> 再搬出家门”，不能拆成三个彼此无因果的散点名词
- 如果写成“病情暗示 / 公开掉位 / 搬出家门”这种并列短语、看不出先后与承压递进，默认这条链未完成

### 一律禁止出现

- `迁移时先保留功能顺序，再替换人物、物件和场景`
- `因为原文先让现实后果站住，再让人物意识跟上`
- `顺序乱了就会像功能按钮`
- 任何 2 张以上表可直接互换的万能提醒句

### 表格施工层落笔模板

不是让你照抄句子，而是要求你按这个粒度写：

- `可直接借的承重结构`
  - 要写成：`这张表真正承重的是 A、B、C。A 负责……，B 负责……，C 负责……`
- `迁移顺序提醒`
  - 要写成：`先放 A，再接 B，最后翻到 C`
- `为什么这个顺序不能乱`
  - 要写成：`如果先把 C 提前，A 的误导 / B 的承压就没了；如果 B 不接在 A 后面，会假成……`

### 原文细节库硬模板

每个 `##` 小节不是写“概念解释”，而是写一张可迁移的证据卡。固定回答 5 条：

1. `具体发生了什么`
2. `这个细节为什么有用`
3. `它压的是谁、压在哪`
4. `后续能迁到什么新桥段`
5. `它对应的角色 / 情绪 / 反转是什么`

每类文件的独立证据卡数量按原文字数分档：8000 字以上至少 5 张、5000-7999 字至少 4 张、5000 字以下至少 3 张。先保全事件资产并集，再做分类；不允许用新卡替换另一张不同事件的旧卡。

这条默认进入 validator 硬校验：别的文件再厚，也不能拿来替代细节库自己的卡数与事件并集覆盖。

### 桥段施工卡硬模板


### `写作手法.md` 硬模板补充

`写作手法.md` 不是一句一节的“手法总评”，而是要让人知道这本书到底怎么写出来的。

至少要拆到：

- `人物嘴型/口气差`：不同角色怎么开口、怎么断句、怎么暴露关系位置
- `章法失效测试`：如果改成顺写、播全、收满、解释满，会假在哪里
- `物件铰链或意象物件`：哪些物件承担转场、翻旧账、压旧伤、抬后果
- `迁移风险`：哪些看起来好学，实际上照搬最容易把新稿写假
- `句子级成文资产`：至少各写 1 条 `活词 / 句法模板 / 段落节拍 / 反面仿写句`

写法要求：

- 每一节都要有原文证据，不接受纯概括
- 每一节都要说明“原文为什么成立”
- 每一节都要补“误学后会假成什么”

### `拆文报告.md` 加厚档

如果这份拆文后续要进入仿写、融合、回修、去味，而不只是做欣赏分析，`拆文报告.md` 除了基础结构层，还要额外补出：

- `情感曲线`
- `爆点分析`
- `反转分析`
- `人物分析`
- `开头分析`
- `结尾分析`
- `五维评分`

加厚要求：

- 这些小节不是可选彩蛋，而是让写手判断“为什么这本能写成这样”的主报告补层
- 如果只剩基础梗概和结构划分，默认仍偏“合规摘要”
- 如果是强样本，优先保留这 7 节，不要只顾压短
- 如果资产层已经很厚，但 `拆文报告.md` 仍像摘要，默认说明厚度被错误分流到 `profile_source / 细节库 / 桥段卡`
- `拆文报告.md` 默认先回答“为什么这本能过 / 为什么不像加工稿 / 为什么会误学成模板”，再让资产层去细拆可迁移件
- `人物分析` 不只做人设总结，至少点破 1 条“角色怎么靠动作/口气/选择顺序把自己和别人拉开脸”
- `情感曲线 / 爆点分析 / 反转分析` 至少各补 1 条中段承重桥解释，避免只剩开头亮刀和终局炸场

### 细节库落笔要求

- 第 1 条必须带出 `谁 + 在哪 + 做了什么`
- 第 2 条不能空谈“有助于情绪落地”，要写出它为什么比一句总结更有效
- 第 3 条要区分压的是 `脸面 / 名分 / 私域 / 旧伤 / 身体 / 公开身份 / 发言权`
- 第 4 条要写成 1-2 个具体桥段名或具体迁移场面，且说明迁移时真正要保留的承重件
- 第 5 条要落在这一条细节自己的角色站位和情绪，不准整份文件共用一句总评

### 细节库生成顺序

每个 `##` 小节动笔前，先在心里做 2 次判断，再下笔：

1. 这条证据核心更偏哪一类：
   - `场景压脸`
   - `物件承重`
   - `动作漏态度`
   - `对白抢解释权`
   - `关系掉位`
2. 这条真正压的是哪一种后果：
   - `体面`
   - `名分`
   - `私域`
   - `旧伤`
   - `身体`
   - `秩序`

判断不出来，就说明这一条还没拆透，不能往下写。

### 细节库禁止句型

- `原文里出现了……这一类场面`
- `可迁到……同题材桥段`
- `对应……关系里的偏心、羞辱或认知反转`
- `这个细节让情绪不靠总结句，而靠具体证据落地`
- 任何删掉标题后还能原样放进别的小节的句子

- `迁移时先保留功能顺序，再替换人物、物件和场景`
- `先让现实后果站住，再让人物意识跟上`
- `顺序乱了就会像功能按钮`
- `替换掉人物物件场景也能用`
- `同题材都可以这样迁移`

### 合格示例口径

- 合格：`先放“旧爱回国”让关系错位成立，再接“热搜打脸”把私密失衡拖进公开场，最后才让“回家取佛牌”把私域主权整个掀开；如果先写回家取佛牌，前面的公开塌陷没有站住，后段会像为了虐而虐。`
- 不合格：`先保留功能顺序，再替换人物场景。`

### 施工层写法骨架

生成 3 段时，优先按下面骨架写，不要自由发挥成空话：

#### 骨架 A：事件顺序类表

适用：

- `导语拆解表`
- `顺序事件表`
- `钩子表`
- `后果链表`

写法：

- `可直接借的承重结构`：`这张表真正承重的不是“都很刺激”，而是“{条目A} -> {条目B} -> {条目C}”。其中 {条目A} 负责起事，{条目B} 负责抬后果，{条目C} 负责把关系/处境彻底改掉。`
- `迁移顺序提醒`：`先写 {条目A}，让 {某种关系/后果} 先站住；再写 {条目B}，把压力从私下拖进公开场/现实层；最后写 {条目C}，把前面压出来的后果真正落地。`
- `为什么这个顺序不能乱`：`如果把 {条目C} 抢到 {条目A} 前面，读者还没看到前因，后果就会像硬塞的功能按钮。`

##### `导语拆解表` 专属写法

- 如果原文更适合按 `错位关系 / 公共命名 / 生死反告白 / 真异点` 这类功能钩拆，优先按功能钩组织；`前20 / 60 / 80字` 只作为字段映射，不要为了凑字段把整张表写机械
- `可直接借的承重结构`：优先写“第一句先扔什么结果/越界，第二句怎么加码，80 字内第二推进点是什么”
- `迁移顺序提醒`：优先写“先亮结果或越界，再补错位/危险，最后给第二推进点”
- `为什么这个顺序不能乱`：优先写“如果先交代人物关系和背景，再起事，导语会怎么变钝”

##### `顺序事件表` 专属写法

- `可直接借的承重结构`：优先写“哪一拍负责起事，哪一拍负责抬压，哪一拍负责翻刀”
- `迁移顺序提醒`：优先写“先把掉位站住，再把后果升级，最后让关系或处境彻底变化”
- `为什么这个顺序不能乱`：优先写“如果先翻刀、后补压迫，为什么会像现成桥段包”

##### `钩子表` 专属写法

- `可直接借的承重结构`：优先写“哪种钩子先吊结果，哪种钩子后回收”
- `迁移顺序提醒`：优先写“先留读者最想确认的问题，再安排最近的兑现点”
- `为什么这个顺序不能乱`：优先写“如果留的钩子离兑现太远，为什么会掉滑动力”

##### `后果链表` 专属写法

- `可直接借的承重结构`：优先写“实锤后第一现实动作、第一误判反应、外部秩序继续压”
- `迁移顺序提醒`：优先写“先让现实动作改变局势，再让误判追错点，最后让外部秩序封口”
- `为什么这个顺序不能乱`：优先写“如果先写追悔、后写现实动作，为什么会失重”

#### 骨架 B：物件动作类表

适用：

- `物件表`
- `动作表`
- `微动作表`

写法：

- `可直接借的承重结构`：`这张表最值钱的是 {条目A} 和 {条目B}。{条目A} 不是普通物件/动作，它负责替掉“{某句解释句}”；{条目B} 负责把人物态度变成看得见的证据。`
- `迁移顺序提醒`：`先放 {条目A} 让读者先看见证据，再接 {条目B} 让态度漏出来；不要一上来先讲感受。`
- `为什么这个顺序不能乱`：`先写感受、后补 {条目A} / {条目B}，会变成解释先行；原文成立，是因为证据先到，情绪后到。`

##### `物件表` 专属写法

- `可直接借的承重结构`：优先写“哪件物件先当证据，哪件物件后回收成翻刀”
- `迁移顺序提醒`：优先写“先亮物件，再让人围着物件失态/争夺/回避”
- `为什么这个顺序不能乱`：优先写“如果先说伤心，再补物件，为什么会像解释补丁”
- 如果原文出现 `副驾驶 / 主座 / 家属栏 / 领证座位 / 饭桌主位` 这类主位证词，优先单列进 `物件表`，不要只放进静压场或关系漏出层

##### `动作表` 专属写法

- `可直接借的承重结构`：优先写“哪个动作负责压脸，哪个动作负责改判”
- `迁移顺序提醒`：优先写“先出偏手动作，再让别人的反应把动作意义抬高”
- `为什么这个顺序不能乱`：优先写“如果先写结论、后补动作，动作就只剩配图功能”

##### `微动作表` 专属写法

- `可直接借的承重结构`：优先写“哪个微动作替掉了‘她很难过/他心冷了’这种解释句”
- `迁移顺序提醒`：优先写“先给微动作，再给一句短反应，不要先整段内心”
- `为什么这个顺序不能乱`：优先写“如果微动作放在解释后面，为什么会失去真人感”

#### 骨架 C：对白口气类表

适用：

- `对白功能表`
- `对话衔接表`
- `失控说话表`

写法：

- `可直接借的承重结构`：`这张表真正承重的是 {条目A} 的口气功能和 {条目B} 的接话方式。不是谁说话更狠，而是谁先抢解释权、谁把场压住。`
- `迁移顺序提醒`：`先让 {条目A} 出来定场，再用 {条目B} 接成关系变化；中间如果有动作垫句，也要先垫再说。`
- `为什么这个顺序不能乱`：`如果所有人一上来都把话说满，{条目B} 的压迫感就没了，人物会变成同一张会推进剧情的功能脸。`

##### `对白功能表` 专属写法

- `可直接借的承重结构`：优先写“哪类话先定位置，哪类话后偷换位置”
- `迁移顺序提醒`：优先写“先让压场口气出现，再让装无辜/找补/切割接上”
- `为什么这个顺序不能乱`：优先写“如果每个人都高质量输出完整观点，为什么会变同脸”

##### `对话衔接表` 专属写法

- `可直接借的承重结构`：优先写“上句逼什么，下句怎么接回去”
- `迁移顺序提醒`：优先写“先逼问/压话，再冷接/装傻/转移；有动作垫句就先垫”
- `为什么这个顺序不能乱`：优先写“如果少了动作垫句，接话为什么会太直太功能”

##### `失控说话表` 专属写法

- `可直接借的承重结构`：优先写“哪种失控方式最能暴露优先级和心虚”
- `迁移顺序提醒`：优先写“先让人被逼到失控边缘，再让失控说话自己漏馅”
- `为什么这个顺序不能乱`：优先写“如果一开口就失控，没有前压层，为什么像故意演戏”

#### 骨架 D：关系秩序类表

适用：

- `安静压迫场表`
- `人物偏手表`
- `烂关系漏出表`
- `外部秩序表`
- `公开炸场表`

写法：

- `可直接借的承重结构`：`这张表真正值钱的是 {条目A} 的秩序压脸和 {条目B} 的关系漏出。原文不是靠狠话，而是靠谁能先动、谁能先站位、谁默认有权。`
- `迁移顺序提醒`：`先让 {条目A} 这种外部秩序/偏手动作站住，再让 {条目B} 把真实关系漏出来；炸场或翻盘一定放后面。`
- `为什么这个顺序不能乱`：`如果先炸后压，场面会像现成桥段包；原文像真人，是因为关系和权限先自己漏，再轮到场面翻。`

## 原文细节库硬规则

`原文细节库/` 8 类文件不是分类词典，而是原文里的可回收细节证据。

### 每个 `##` 细节小节都必须回答

1. `具体发生了什么`
   - 必须写成具体事件，不准写“原文里出现了某类场面”
2. `这个细节为什么有用`
   - 必须写到它具体替掉了什么解释句或情绪句
3. `它压的是谁、压在哪`
   - 必须点名角色和掉位位置
4. `后续能迁到什么新桥段`
   - 必须写 1-2 个具体可迁桥段，不准泛写“同题材桥段”
5. `它对应的角色 / 情绪 / 反转是什么`
   - 必须落到这条细节自身，不准整段复用同一句

### 一律禁止出现

- `原文里出现了……这一类场面或关系后果`
- `可迁到……同题材桥段`
- `对应……三角关系里的偏心、羞辱或认知反转`
- 多个小节五条回答几乎一字不变，只换标题

### 合格示例口径

- 合格：`角色A接电话时先宣布新的私人安排，再把门锁权限改到自己名下；这个细节不靠作者解释偏心，而是让空间权限变化直接呈现关系换位。`
- 不合格：`原文里出现了家门口与卧室这一类场面。`

### 细节小节写法骨架

每个 `##` 小节都尽量按下面骨架落笔：

```md
## {细节名}

- 具体发生了什么：{角色A} 在 {场景} 做了 {动作/说话}，直接导致 {角色B} 在 {位置/关系} 上掉位。
- 这个细节为什么有用：它替掉了“{原本很容易直接解释的句子}”，改成让读者自己看见 {证据/态度/偏心/冷意}。
- 它压的是谁、压在哪：压 {角色B}，压在 {名分/体面/旧伤/主权/亲情/公开形象} 上。
- 后续能迁到什么新桥段：可迁到 {桥段1} 或 {桥段2}，前提是保留 {这条细节的功能件}。
- 它对应的角色 / 情绪 / 反转是什么：对应 {角色A/角色B} 的 {情绪或态度}，并服务 {哪一拍反转或掉位}。
```

### 8 类细节库各自更该写什么

- `场景细节库`：重点写场景怎么天然压脸，不要只写地点名。
- `关系细节库`：重点写关系是怎么自己漏出来的，不要只写“他们关系不好”。
- `情绪细节库`：重点写情绪落在什么动作/停顿/反应上，不要写抽象情绪词堆叠。
- `对白细节库`：重点写谁的话在抢解释权、谁的话在装无辜，不要只摘金句。
- `翻车细节库`：重点写哪一步本来想圆，结果反而更坏。
- `旧伤细节库`：重点写旧伤是被什么具体物件/场面勾出来的。
- `动作细节库`：重点写动作承担了什么说明功能，不要只列动作名。
- `场面细节库`：重点写“谁站着、谁看着、谁默认有权”，不要只写大场面概括。

#### `场景细节库` 专属落笔模板

- 具体发生了什么：`{角色A}` 在 `{场景}` 里被迫面对 `{具体布置/公开环境/空间变化}`，这让 `{角色B}` 当场掉位。
- 这个细节为什么有用：它不用解释“{某人被排除/被羞辱/被替换}”，场景本身就把这件事亮出来了。
- 它压的是谁、压在哪：压 `{角色B}`，压在 `{主权/名分/体面}`。
- 后续能迁到什么新桥段：可迁到 `{公开错位桥}`、`{回家见异物桥}` 之类，但要保留 `{场景的天然压脸功能}`。
- 它对应的角色 / 情绪 / 反转是什么：对应 `{角色B}` 的 `{发冷/心凉/窒息}`，并服务 `{哪一拍掉位或翻刀}`。

#### `关系细节库` 专属落笔模板

- 具体发生了什么：`{角色A}` 对 `{角色B}` 的 `{称呼/站位/默认权限}` 出现偏差，真实关系自己漏出来了。
- 这个细节为什么有用：它替掉“他们关系其实早就坏了/偏了”这种总结句。
- 它压的是谁、压在哪：压 `{角色B}`，压在 `{关系位置/家属位/优先级}`。
- 后续能迁到什么新桥段：可迁到 `{偏心桥}`、`{正宫失位桥}`，前提是保留 `{关系漏出的证据件}`。
- 它对应的角色 / 情绪 / 反转是什么：对应 `{谁在漏，谁在受}` 的 `{委屈/发冷/羞耻}`，并服务 `{关系翻面}`。

#### `情绪细节库` 专属落笔模板

- 具体发生了什么：`{角色A}` 没直接说情绪，而是先出现 `{动作/停顿/呼吸/视线/站位}`。
- 这个细节为什么有用：它替掉“{他很难过/她心冷了}”这种直给情绪句。
- 它压的是谁、压在哪：压 `{角色A}`，压在 `{心口/体面/旧伤}`。
- 后续能迁到什么新桥段：可迁到 `{隐忍桥}`、`{心冷离场桥}`，前提是保留 `{动作先于解释}`。
- 它对应的角色 / 情绪 / 反转是什么：对应 `{具体情绪}`，并服务 `{忍到爆 / 冷到走 / 装平静}` 这种转向。

#### `对白细节库` 专属落笔模板

- 具体发生了什么：`{角色A}` 说了一句 `{口气类型}` 的话，把 `{解释权/道德位/受害者位}` 抢走了。
- 这个细节为什么有用：它替掉“{他在装无辜/她在压人}”这种作者说明。
- 它压的是谁、压在哪：压 `{角色B}`，压在 `{公开形象/辩解机会/位置}`。
- 后续能迁到什么新桥段：可迁到 `{装无辜翻盘桥}`、`{公开压场桥}`，前提是保留 `{这句对白的功能}`。
- 它对应的角色 / 情绪 / 反转是什么：对应 `{角色A}` 的 `{心虚/压场/偷换}`，并服务 `{哪一拍改判或翻面}`。

## Stage 3 情感线+爆点

### A. 情感曲线

至少 max(5, 节数) 个节点，含字数位置，每个节点标注钩子类型：

| 位置 | 字数 | 节点序号 | 情绪 | 强度与方向 | 触发事件 | 钩子类型 |
|------|------|----------|------|------------|----------|----------|
| 开头 | {X} | N{N} | {好奇/心酸/震惊} | {虐-9~爽+9} | {事件} | {悬念/冲突/反差/代入/信息差/无} |
| 低谷 | {X} | N{N} | {心疼/绝望/愤怒} | {虐-9~爽+9} | {事件} | {钩子类型} |
| 反转点 | {X} | N{N} | {震惊/心疼} | {虐-9~爽+9} | {事件} | {钩子类型} |
| 高潮 | {X} | N{N} | {情绪} | {虐-9~爽+9} | {事件} | {钩子类型} |
| 结尾 | {X} | N{N} | {满足/意难平/治愈} | {虐-9~爽+9} | {事件} | {钩子类型} |

钩子类型速查：悬念（想知道后续）、冲突（矛盾激化）、反差（认知颠覆）、代入（感同身受）、信息差（读者知道角色不知道）

曲线特征：起始{...} | 走向{上行/下行/波浪/V形/倒V/阶梯/断崖/压缩弹簧} | 极值{最高%X 最低%Y} | 方向差{从X到Y} | 翻转次数{N}

### A2. 情绪拆法小表

> 不只记“这里虐”，要把剧情、扎人的内核和情绪结果拆开。

| 节点 | 带情绪的剧情 | 扎人的内核 | 先砸中的情绪 | 情绪浓度 |
|------|--------------|------------|--------------|----------|
| N{N} | {具体发生了什么} | {为什么扎人} | {委屈/心寒/羞辱/发冷/掌控感等} | {一般/中等/强烈} |

### A2.5 情绪为什么爽

> 这一段必须单独回答，不要把“爽”混在爆点或共鸣里带过去。

| 项目 | 分析 |
|------|------|
| 前半段读者先替谁憋 | {主角当前处境先让读者替她憋什么：委屈/心寒/发冷/丢脸/不甘} |

验收口径：

- 只要还停在概括层，就不算“可直接仿写”
- 至少要让后续写作能直接抽导语起法、抽事件、抽物件、抽动作、抽口气、抽衔接、抽误判、抽钩子、抽微动作、抽安静压迫场，以及高情绪题材必需的偏手、失控说话、漏出、外部秩序、炸场、后果链
- 如果基础 10 张表缺 2 张以上，默认视为拆文还没拆透
- 如果缺 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 里的任意 1 份，默认还没到可直接安全仿写层
- 如果高情绪题材缺后面 6 张里的任意 3 张以上，默认视为“结构有了，但成文活层没拆出来”
- 如果没有 `同桥段过检规则`，默认还不能直接拿去做同桥段融合、换皮或去AI味回修
- 如果这 3 份文件只是“有结论没证据”，也默认还没到可直接安全仿写层
- `作者DNA指纹` 至少要回答：句长切句、停顿、口气差、动作替代、旧伤触发器、反面句型
- `仿写约束_禁写清单` 至少要回答：禁写句式、为什么假、容易把文写坏在哪
- `同桥段过检规则` 至少要回答：桥段起手件、原文承重件、新稿高风险假点、绝不能丢的保留件
- 高敏桥默认再补：`重大证据前隔开的现实后果 / 尾声入口给了谁 / 为什么不给另一条线 / 人物不同脸证据`
- 如果不能从这 3 份文件里直接提取“该学什么 / 该禁什么 / 同桥怎么过”，就不能标 `ready-for-write`
- `导语拆解表` 至少要能回答“前 20 / 60 / 80 字分别做了什么”
- `对话衔接表` 至少要能回答“对白的压力为什么不只是靠狠词”
- `微动作表` 至少要能回答“哪些解释句其实被动作替掉了”
- `安静压迫场表` 至少要能回答“为什么这场不吵反而更压人”
