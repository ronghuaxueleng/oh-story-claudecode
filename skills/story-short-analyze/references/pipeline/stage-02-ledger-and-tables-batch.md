# Stage 2：字典、候选池、16 张表

这一阶段的目标不是“把 16 个文件凑出来”，而是把原文证据稳定分发到可调用表格里。

## 固定顺序

1. 第一波 `discovery_index` lane 完成 `本书动态信号字典.json + 原文资产候选池.md`
2. foundation 预检通过并冻结上述两个共享文件
3. 16 张表分成两个 8 表 asset lane；宿主支持时并发，否则连续落盘
4. 全部 asset lane 汇合后由主线程回扫动态字典
5. 主线程回扫候选池
6. 同一模型回合内完成两轮不同扫描顺序的独立漏项审计

## 快速厚拆分批

- 批次 2A，结构/物件/动作 8 表：`导语拆解 / 顺序事件 / 物件 / 动作 / 误判 / 钩子 / 微动作 / 安静压迫场`
- 批次 2B，对白/人物/关系/秩序 8 表：`对白功能 / 对话衔接 / 人物偏手 / 失控说话 / 烂关系漏出 / 外部秩序 / 公开炸场 / 后果链`
- 两个表格 lane 直接继承第一波同一 agent 会话；`asset_shared_reads` 固定为空，只读取各自 `delta_reads` 中尚未见过的字典与候选池。`preferred_reads` 只表达证据依赖，不再作为重读指令。
- 每个 agent 的表格 lane 与后续资产 lane 合并为一次派发、批内连续落盘，避免表格完成后再次等待和重新装载上下文。
- 主线程可同时调度 Stage 3 的独立 lane，但必须等待所有 lane 完成后再做统一核销。
- 每批一次性连续写完目标文件，再做文件齐全、截断、最低表行和三段施工层快检。
- 快检失败只二分当前 8 表批次；二分仍发生截断或明显压缩，才对失败子批降级为双文件模式。
- 不因批量生成复用统一壳；每张表仍必须消费自己的候选、证据和施工顺序。

## 候选池原则

每条必须写：

- `C编号`
- `L起-L止`
- `锚点`
- `类别`
- `资产名`
- `去向`
- `状态`
- `理由`

只要标成 `已收录`，目标表里就必须能找到对应条目。

## 16 张表原则

不要把表写成统一壳。

第二波 worker 首写前必须读取本 lane 的 `_parallel_plan.json.asset_lanes[].first_write_contract`，严格按其中 `columns / min_rows / trailing_sections` 建表。禁止沿用旧表头先写完，再等待 validator 报错后追加修补。

每张表都要先判断自己属于哪类：

- 事件顺序类
- 物件动作类
- 对白口气类
- 关系秩序类

然后再按对应表型写施工层。

## 16 张表首写列名

以下列名从第一次落盘就固定使用；可增加确有必要的专属列，但不得删除、改成 validator 无法识别的近义自定义列：

| 文件 | 首写固定列 |
|---|---|
| 导语拆解表 | `层级 / 钩子内容 / 第一句功能 / 原文证据 / 迁移提醒` |
| 顺序事件表 | `层级 / 事件顺序 / 谁做了什么 / 这一拍功能 / 原文位置 / 迁移提醒` |
| 物件表 | `层级 / 物件 / 伤害层 / 原文证据 / 迁移提醒` |
| 动作表 | `层级 / 动作本体 / 谁做 / 原文位置 / 迁移提醒` |
| 误判表 | `层级 / 先误判了什么 / 从哪开始翻 / 原文证据 / 迁移提醒` |
| 钩子表 | `层级 / 钩子内容 / 钩子类型 / 回收位置 / 原文证据 / 迁移提醒` |
| 微动作表 | `层级 / 动作本体 / 对应情绪 / 替代的解释句 / 原文位置 / 迁移提醒` |
| 安静压迫场表 | `层级 / 场面压力来源 / 谁没说话 / 环境音 / 未说破结果 / 原文位置 / 迁移提醒` |
| 对白功能表 | `层级 / 角色 / 典型说法类型 / 这类话负责什么 / 口吻特征 / 原文证据 / 迁移提醒` |
| 对话衔接表 | `层级 / 上句功能 / 下句接法 / 原文证据 / 迁移提醒` |
| 人物偏手表 | `层级 / 角色 / 稳定偏手 / 原文证据 / 迁移提醒` |
| 失控说话表 | `层级 / 角色 / 失控类型 / 触发点 / 暴露 / 原文证据 / 迁移提醒` |
| 烂关系漏出表 | `层级 / 具体漏出件 / 关系伤害 / 原文证据 / 迁移提醒` |
| 外部秩序表 | `层级 / 秩序来源 / 谁掌控秩序 / 后果 / 原文证据 / 迁移提醒` |
| 公开炸场表 | `层级 / 场面 / 关键硬牌 / 谁出的牌 / 后果 / 原文位置 / 迁移提醒` |
| 后果链表 | `层级 / 起点实锤 / 后果链节点 / 最终新状态 / 原文证据 / 迁移提醒` |

`层级` 列从首写起保留，核心资产写 `核心`，其余写 `次级索引`。这样候选池回扫后即使表格超过 5 行，也不需要返工表头。

- `公开炸场表`

## 8000 字以上样本的默认承重下限

不是让你机械凑数，而是要求你先排除“我已经解释清楚，所以表格不用写满”的偷懒倾向。

- `物件表 / 动作表 / 对白功能表 / 钩子表 / 微动作表 / 失控说话表`：默认至少保 `5` 条独立资产行
- `公开炸场表`：默认至少保 `4` 条独立资产行
- `顺序事件表 / 对话衔接表 / 误判表 / 安静压迫场表 / 人物偏手表 / 烂关系漏出表 / 后果链表`：默认至少保 `4` 条独立资产行
- `导语拆解表 / 外部秩序表`：默认至少保 `3` 条独立资产行

如果原文天然不足，必须在本表或候选池里写明为什么不足，不能静默停在 2-3 行。

## 七类表的候选发现前置提醒

以下几类不要等表快写完了才想起来补：

- `物件表`：优先找 `制度文件 / 私域权限件 / 证据载体 / 假性道歉礼物 / 公开身份件 / 座位主位件`
- `动作表`：优先找 `拽 / 护 / 扔 / 挡 / 砸 / 走 / 留证 / 抢手机 / 开门 / 关门`
- `对白功能表`：优先找 `护短辩词 / 装无辜 / 规则问句 / 旧情反咬 / 关系切割句`
- `钩子表`：优先找 `第三人代接 / 门锁失效 / 未来公开事件 / 程序闸门 / 尾声回头`
- `微动作表`：优先找 `咬 / 捏 / 站起 / 僵住 / 深呼吸 / 红眼 / 探头 / 低头`
- `失控说话表`：优先找 `披道德外壳的站错队 / 跳楼式抢代价位 / 反咬 / 程序切刀`
- `公开炸场表`：优先找 `正式场掉位 / 私域主位被抢 / 单位门口或网络公开清算`

## 表后 3 段硬规则

`可直接借的承重结构 / 迁移顺序提醒 / 为什么这个顺序不能乱`

必须使用以下三个二级标题，且在单个文件中各出现、只出现一次：

- `## 可直接借的承重结构`
- `## 迁移顺序提醒`
- `## 为什么这个顺序不能乱`

每段至少 2 条，直接点名本表至少 2 个具体资产，不接受抽象总结句替代。禁止在批末再次追加同名标题。
