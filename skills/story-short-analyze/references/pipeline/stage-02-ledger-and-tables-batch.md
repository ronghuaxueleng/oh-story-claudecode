# Stage 2：字典、候选池、16 张表

这一阶段的目标不是“把 16 个文件凑出来”，而是把原文证据稳定分发到可调用表格里。

## 固定顺序

1. `写作资产/本书动态信号字典.json`
2. `写作资产/原文资产候选池.md`
3. 16 张表，固定双文件微批
4. 表后回扫动态字典
5. 表后回扫候选池
6. 两轮独立漏项审计

## 候选池原则

每条必须写：

- `C编号`
- `L起-L止`
- `锚点`
- `类别`
- `资产名`
- `去向`
- `状态`
- `理由`

只要标成 `已收录`，目标表里就必须能找到对应条目。

## 16 张表原则

不要把表写成统一壳。

每张表都要先判断自己属于哪类：

- 事件顺序类
- 物件动作类
- 对白口气类
- 关系秩序类

然后再按对应表型写施工层。

## 最容易出薄的地方

下面这些最容易被压成空话，默认优先复核：

- `顺序事件表`
- `动作表`
- `对话衔接表`
- `微动作表`
- `人物偏手表`
- `安静压迫场表`
- `物件表`
- `对白功能表`
- `失控说话表`
- `钩子表`
- `公开炸场表`

## 8000 字以上样本的默认承重下限

不是让你机械凑数，而是要求你先排除“我已经解释清楚，所以表格不用写满”的偷懒倾向。

- `物件表 / 动作表 / 对白功能表 / 钩子表 / 微动作表 / 失控说话表`：默认至少保 `5` 条独立资产行
- `公开炸场表`：默认至少保 `4` 条独立资产行
- `顺序事件表 / 对话衔接表 / 误判表 / 安静压迫场表 / 人物偏手表 / 烂关系漏出表 / 后果链表`：默认至少保 `4` 条独立资产行
- `导语拆解表 / 外部秩序表`：默认至少保 `3` 条独立资产行

如果原文天然不足，必须在本表或候选池里写明为什么不足，不能静默停在 2-3 行。

## 七类表的候选发现前置提醒

以下几类不要等表快写完了才想起来补：

- `物件表`：优先找 `制度文件 / 私域权限件 / 证据载体 / 假性道歉礼物 / 公开身份件 / 座位主位件`
- `动作表`：优先找 `拽 / 护 / 扔 / 挡 / 砸 / 走 / 留证 / 抢手机 / 开门 / 关门`
- `对白功能表`：优先找 `护短辩词 / 装无辜 / 规则问句 / 旧情反咬 / 关系切割句`
- `钩子表`：优先找 `第三人代接 / 门锁失效 / 未来公开事件 / 程序闸门 / 尾声回头`
- `微动作表`：优先找 `咬 / 捏 / 站起 / 僵住 / 深呼吸 / 红眼 / 探头 / 低头`
- `失控说话表`：优先找 `披道德外壳的站错队 / 跳楼式抢代价位 / 反咬 / 程序切刀`
- `公开炸场表`：优先找 `正式场掉位 / 私域主位被抢 / 单位门口或网络公开清算`

## 表后 3 段硬规则

`可直接借的承重结构 / 迁移顺序提醒 / 为什么这个顺序不能乱`

必须直接点名引用本表条目，不接受抽象总结句替代。
