# 短篇拆文执行骨架

这份文件只回答 6 件事：

1. 拆前必须判什么
2. 样本分级怎么落
3. 高敏同桥专项为什么必须单列
4. 产物最少要落哪些文件
5. 可直接仿写层怎样才算拆透
6. 什么情况下不准放行到写作侧

---

## 一、拆前必须判的 4 件事

进入正式拆解前，必须先写进 `拆文报告.md`：

1. `高敏层级判断`
2. `本书更像哪一型`
3. `原文检测输入污染检查`
4. `桥安全误判提醒`

类型只允许：

- `讲法型`
- `桥段链型`
- `混合型`

硬口径：

- 不拿原文低分直接等于桥安全
- 不跳过 OCR / 水印 / 杂符号 / 搬运广告等输入污染检查
- 不只拆“用了什么桥”，必须拆“为什么不像加工稿”

---

## 二、样本分级

原文不能默认当正样本。必须先给单书分级：

- `A类 正样本`
- `B类 骨架样本`
- `C类 负样本`

判定固定分三层：

1. `脚本硬筛`
2. `规则拆层`
3. `模型解释`

还必须补 `原文分数快照`：

- 原文整体分数
- 原文是否被切块
- 最高风险块落在哪类桥段
- 这份分数在后续写作里怎么使用

用法限制：

- `A类`：可提句法 DNA、桥段承重件、场面资产
- `B类`：只提骨架、承重件、后果链、场面秩序
- `C类`：只做反面规则与禁写清单

---

## 三、高敏同桥专项

如果用户后续目标包含：

- 仿写
- 原情节实验
- 同桥高敏检测
- 去AI味回修
- 外部分块审计长期卡高

必须额外挂：

- `high-sensitivity-bridge-imitation.md`
- `章法手法硬拆清单.md`
- `../../story/references/high-risk-rewrite-governance.md`

最低要求：

- `高敏桥段识别.md` 说明原文为什么不像加工稿、哪层能学、哪层不能学
- `同桥段过检规则.md` 说明原文为什么能过、仿稿为什么会高
- `作者DNA指纹.md` 补角色反应语法、偏手先后、外部秩序压人方式
- `写作手法.md` 必须补开场入口、时间操控、回忆职责、断场点、尾声入口、收口方式
- `高敏桥段识别.md` 还必须补：重大证据前隔开的现实后果、后果回灌方式、同桥不同脸依据

重点不是桥名，而是：

- 信息怎么漏
- 后果怎么飘
- 为什么普通事件在原文里不像成品包
- 为什么这些桥没有被章法写成完整功能包
- 为什么重大证据前隔的是现实后果，不是纯时间空档
- 为什么尾声入口给了这条线，不给另一条线
- 为什么几位关键人物在同桥里不是同一张脸

---

## 四、标准产物

最低标准产物已经不再是“核心几件套”，而是默认全量自动产出包。

模型进入正式拆书后，必须一次性自动落盘：

- `原文/`
- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`
- 16 张 `可直接仿写_*.md`
- `原文细节库/`
- `写作资产/样本分级与可学层.md`
- `写作资产/高敏桥段识别.md`
- `写作资产/作者DNA指纹.md`
- `写作资产/仿写约束_禁写清单.md`
- `写作资产/同桥段过检规则.md`
- `写作资产/profile_source.md`
- `book.profile.json`
- `写作资产/母结构_故事走法.md`
- `写作资产/主冲突_副升级器.md`
- `写作资产/异物清单.md`
- `写作资产/第二层冲突清单.md`
- `写作资产/角色口气模板.md`
- `写作资产/关系重组方式.md`
- `写作资产/公开场_关键硬牌_后果.md`
- `写作资产/平台适配提醒.md`
- `写作资产/情绪母线.md`
- `写作资产/新状态清单.md`
- `写作资产/虐点对照细节.md`

不允许只产核心三件套再说“后续需要时再补”。

`拆文报告.md` 固定必须补：

- `高敏层级判断`
- `原文检测输入污染检查`
- `桥安全误判提醒`
- `更适合仿讲法还是换骨架`
- `样本分级与可学层判断`

`写作手法.md` 固定执行口径：

1. 先加载 `output-templates.md` 里的 `写作手法.md 固定骨架`
2. 严格按 9 节顺序落盘
3. `章法硬拆` 必须补 7 个子项
4. `章法失效测试` 必须补 3 个子项
5. 缺任意一节，不算 Stage 4 完成

自动产出完成后，固定再跑一次全量验收脚本：

```bash
python3 "$CODEX_HOME/skills/story-short-analyze/scripts/run_short_analyze_finalize.py" "拆文库/{书名}"
```

这个脚本不是只做缺件检查，还会先补：

1. `写作资产/profile_source.md` 是否存在
2. `book.profile.json` 自动生成
3. `validate_short_analyze_outputs.py` 作为收口脚本内部的全量验收
4. 最终状态只返回 `ready-for-write / blocked-on-assets`

固定执行任务单：

- `auto-full-output-task.md`
- `short-analyze-execution-prompt.md`

执行顺序：

1. 先产 `拆文报告.md / 情节节点.md / 写作手法.md / _meta.json`
2. 再产 16 张 `可直接仿写_*.md`
3. 再产 `原文细节库/`
4. 再产 `写作资产/` 全包
5. 再产 `profile_source.md`
6. 再生 `book.profile.json`
7. 最后跑 `run_short_analyze_finalize.py`
8. 若脚本返回 `blocked-on-assets`，回到对应批次补件再重跑

第 2 批额外硬闸：

1. 每张 `可直接仿写_*.md` 动笔前，先判断它属于：
   - `事件顺序类`
   - `物件动作类`
   - `对白口气类`
   - `关系秩序类`
2. 判断完类型后，再按 `output-templates.md` 里该类骨架写 3 段施工层
3. 如果 3 段看起来能通用于别的表，默认类型判断失败，退回重写

第 3 批额外硬闸：

1. 每个 `##` 细节小节动笔前，先判断这一小节的证据核心更偏：
   - `场景压脸`
   - `物件承重`
   - `动作漏态度`
   - `对白抢解释权`
   - `关系掉位`
2. 判断完再写五条回答
3. 如果五条去掉标题后还能直接套给别的小节，默认这一小节仍是模板壳
4. 第 1 条必须落成 `谁 + 在哪 + 做了什么`
5. 第 2 条必须回答“为什么这条证据比一句总结更有用”
6. 第 3 条必须写清压的是 `脸面 / 名分 / 私域 / 旧伤 / 身体 / 秩序`
7. 第 4 条除了桥段名，还要补一句“迁移时真正要保留的承重件”
8. 第 5 条必须是这条细节自己的角色站位，不准整份文件共用一句三角关系总评

第 3 批回放验证结论：

- 细节库真正能过线，不是因为写得更长，而是因为每条都落到了证据层
- 最有效的写法是：
  - 先写具体事件
  - 再写为什么这件证据比一句总结更有效
  - 再写它实际压坏的是哪种位置
  - 最后再谈迁移桥段和角色反转
- 只要出现“这类场面”“同题材桥段”“三角关系偏心”这种总括壳句，默认还没拆到可写层

主会话正式执行时，固定再加载：

- `short-analyze-execution-prompt.md`

口径：

- `analyze-execution-core.md` 负责定义规则与放行条件
- `short-analyze-execution-prompt.md` 负责把 6 批执行顺序推进到主会话执行层
- 两者缺一不可

## 五、可直接仿写层的完成标准

最少要有 16 张表：

1. `顺序事件表`
2. `物件表`
3. `动作表`
4. `对白功能表`
5. `对话衔接表`
6. `误判表`
7. `钩子表`
8. `导语拆解表`
9. `微动作表`
10. `安静压迫场表`
11. `人物偏手表`
12. `失控说话表`
13. `烂关系漏出表`
14. `外部秩序表`
15. `公开炸场表`
16. `后果链表`

每张表正文后还必须补 3 段：

- `可直接借的承重结构`
- `迁移顺序提醒`
- `为什么这个顺序不能乱`

缺任意一段，这张表都还只是分析稿，不算施工稿。
没先判表类型，或施工段与表类型不匹配，这张表也还只是分析稿。

## 六、三份核心写作资产

### 1. 作者DNA指纹

不能只写“文风细腻 / 情绪克制”这类空标签，至少要拆到：

- 句长习惯
- 停顿方式
- 视角贴脸距离
- 情绪落点
- 旧伤触发器
- 角色口气差
- 场面脏度
- 负面样本
- 同桥不同脸的证据
- 权限差 / 动作差 / 节拍差

必须给原文证据。

### 2. 仿写约束_禁写清单

至少列 10 条以上禁写句型或禁写推进法，每条都要补一句“为什么假”。

至少覆盖：

- 空情绪
- 假口语
- 统一过渡句
- 补字废话

### 3. 同桥段过检规则

至少列 3-5 个高频桥段，每个桥段都要写：

- 原文起手件
- 原文真正承重件
- 新稿最容易写假的点
- 推荐迁移顺序
- 为什么这个顺序不能乱
- 原文为什么能过
- 重大证据前隔开的现实后果
- 尾声入口为什么不给次线

并且要能映射进 `profile_source / book.profile.json`。

---

## 七、profile 落盘顺序

固定顺序：

1. 拆完单书
2. 检查 16 张表、原文细节库、写作资产全包是否齐
3. 由模型补 `写作资产/profile_source.md`
4. 再生成 `book.profile.json`

禁止：

- 跳过单书 `profile`
- 直接在写前临时拼题材规则
- 只留 Markdown 结论，不做结构化落盘

---

## 八、不放行条件

任一命中都不放行：

- 没有原文证据，只有抽象评价
- 只有桥段名，没有承重件
- 只有正面规则，没有反面拦截
- 没写输入污染检查
- 没写桥安全误判提醒
- 缺 16 张表
- 缺 `原文细节库/`
- 缺 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则 / 高敏桥段识别`
- 缺 `母结构 / 主冲突副升级器 / 异物 / 第二层冲突 / 角色口气模板 / 关系重组方式 / 公开场后果 / 平台适配提醒 / 情绪母线 / 新状态清单 / 虐点对照细节`
- 缺 `profile_source.md`
- 缺 `book.profile.json`
- 验收脚本未通过

通用高风险任务边界见：

- `../../story/references/high-risk-rewrite-governance.md`

当前 analyze 侧的额外硬口径：

- 结构拆到了，不等于能安全仿写
- 报告写全了，不等于成文活层拆到了
- 原文好看，不等于整本都能提 DNA
