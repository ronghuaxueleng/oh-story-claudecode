# 原文资产候选池

`写作资产/原文资产候选池.md` 是 16 张表的上游核销账，不是额外分析报告。

## 固定流程

1. 主报告完成后，先按 `dynamic-signal-dictionary.md` 建立本书动态信号字典。
2. 按 `_source_manifest.json.chunks` 对全文做第二遍读取；逐块只找资产，不重写梗概。
3. 先形成候选池，再写 16 张表。
4. 16 张表完成后，先回补动态字典，再按全部 Chunk 回扫一次，把首次遗漏的候选追加进池。
5. 每条候选必须标明进入哪张表；确实不收录时写具体排除理由。
6. finalize 前增加 `## 反向漏项审计`，脱离已有分类，从原文重新寻找至少 5 个可能漏项，并逐条写明 `收录 / 不收录 + 理由`。

## 七类易薄表的反向漏项专扫

当这些表还没写满默认承重行数时，回扫不要泛找“更多素材”，而要按下列口径逐类补：

- `物件表`：再扫一次 `礼物 / 文件 / 证据 / 门锁 / 交通工具 / 仪式件 / 座位主位件`
- `动作表`：再扫一次 `拽 / 扔 / 护 / 挡 / 砸 / 冲 / 躲 / 递 / 收`
- `对白功能表`：再扫一次 `护短 / 找补 / 装无辜 / 反咬 / 切割 / 刺痛`
- `钩子表`：再扫一次 `代接电话 / 开门前卡住 / 协议或开庭 / 尾声回头`
- `微动作表`：再扫一次 `嘴唇 / 手 / 眼 / 呼吸 / 站起 / 头 / 肩 / 表情骤停`
- `失控说话表`：再扫一次 `你怎么 / 你凭什么 / 我没有 / 她只是 / 我们法庭见`
- `公开炸场表`：再扫一次 `正式场 / 私域门口 / 单位门口 / 网络 / 学校`

命中后优先入候选池；只有确认它不承担独立迁移功能，才允许写进反向漏项审计排除。

候选数量只作非阻断复核提示，不是完成条件。已经识别出的独立资产必须全部保留，也不得为了达到固定条数拆碎、凑数或编造资产。

## 候选格式

每条一行：

```text
C001 | L起-L止 | 锚点：原文短语 | 类别：物件 | 资产名：制度文件替换私人信物 | 去向：可直接仿写_物件表.md | 状态：已收录 | 理由：同一关系位置发生物件替换
```

固定字段：

- `L起-L止`：最多 80 行
- `锚点`：必须真实存在于范围内
- `类别`：使用 16 张表的标准类别名
- `资产名`：短而具体，必须能在目标表中找到
- `去向`：目标表文件名
- `状态`：只允许 `已收录 / 不收录`
- `理由`：写该资产为何值钱，或为何确实不进入直接仿写表

不允许：

- `待处理 / 待补 / 以后再看`
- 用“价值一般”“重复”排除却不说明与哪条重复
- 多个不同事件合并成一条大候选
- 只登记开头和高潮，漏掉中段或尾声

## 16 类标准类别

- `导语`
- `顺序事件`
- `物件`
- `动作`
- `对白功能`
- `对话衔接`
- `误判`
- `钩子`
- `微动作`
- `安静压迫场`
- `人物偏手`
- `失控说话`
- `烂关系漏出`
- `外部秩序`
- `公开炸场`
- `后果链`

## 类别覆盖确认

16 类必须逐类完成确认，但不要求每类都有资产：

- 有候选：以对应 `C` 记录为准，不设固定数量
- 无候选：写 `- 类别名：已扫，原文未发现`

禁止为了让每类非空而把普通叙述硬塞进微动作、安静压迫、烂关系或公开炸场。

## 五项专项回扫

候选池末尾固定写 `## 专项回扫确认`，逐项回答：

- `物件替换对`：旧物/新物、被夺/补偿、损毁/替代、私人信物/制度文件、证据载体是否都扫过
- `微动作角色覆盖`：核心人物在示弱、心虚、恐慌、强撑、抽离时的身体反应是否都扫过
- `对白侵占与假道歉`：软语侵占私域、假道歉后反客为主、称呼夺位、解释抢权是否都扫过
- `安静等待与未归`：无人回应、整夜未归、设备噪音、门锁提示、流程等待、话后停顿是否都扫过
- `未来公开事件钩子`：婚礼、直播、发布会、签约、审判、颁奖等已宣布但尚未发生的公开事件是否进入钩子表

这些是通用功能检查，不是固定要求原文必须出现某个名词。没有对应现象时，也要写“已扫，原文未发现”，不能留空。

## 分块回扫确认

固定写 `## 分块回扫确认`，每个 manifest Chunk 一行：

```text
- Chunk 1：L1-L120 | 状态：已回扫 | 新增候选：C001-C012 | 空缺复核：已检查物件、动作、对白、微动作、安静场与钩子
```

某个 Chunk 确实没有独立资产时允许写：

```text
- Chunk 2：L121-L240 | 状态：已回扫 | 新增候选：无 | 空缺复核：本块仅承接前一事件，没有新增物件、动作、对白功能或结构钩子
```

全部 Chunk 未确认前不允许进入 finalize；空 Chunk 可以存在，但必须给出具体复核结论。

## 反向漏项审计

固定格式：

```text
- A001 | L起-L止 | 原文：原文短语 | 判定：收录 | 去向：C023 / 可直接仿写_对白功能表.md | 理由：此前只保留了功能归纳，遗漏人物原声
- A002 | L起-L止 | 原文：原文短语 | 判定：不收录 | 去向：无 | 理由：与 C011 是同一事件的重复表述，不构成独立资产
```

硬规则：

- 至少审计 5 项，不能全部写“不收录”且使用同一个理由。
- 判定为收录的项目必须新增候选并进入目标表或 profile。
- 提示词命中原文但未出现在候选池或目标表时，必须逐项核对；确实只是通用载体或无功能动作，可在本节写 `判定：不收录` 和具体理由。
- validator 会读取不收录审计，不再强迫把 `视频 / 低下头 / 攥紧拳头 / 联系不上` 等通用词机械塞进 16 张表。
- 未进入候选、目标表，也没有对应不收录审计的提示词，仍属于阻断错误。
