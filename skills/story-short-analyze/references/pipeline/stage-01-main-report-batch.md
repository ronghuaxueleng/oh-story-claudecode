# Stage 1：事实底座与第一波并发

这一阶段是全流程第一优先级。

如果 token、时间、上下文有压力，宁可后面的表和资产暂时不写，也不能把下面四份写薄：

- `事实与推断台账.md`
- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`

## 固定顺序

前置条件：`_sample_comparison.md` 已完成首轮记录。

1. `事实与推断台账.md`
2. `_analysis_brief.md`
3. 第一波并发：
   - `main_report`：`拆文报告.md`
   - `chronology_craft`：`情节节点.md + 写作手法.md`
   - `discovery_index`：`本书动态信号字典.json + 原文资产候选池.md`
4. 汇合后回看样本反例区并更新 `_sample_comparison.md`
5. 主线程补 `_meta.json.structure_counts`
6. 运行 `validate_short_analyze_foundation.py`

12000 字以内启动 `agent-core`、`agent-craft`、`agent-discovery` 三个子 agent：分别写主报告、节点与手法、候选池与动态字典。三个会话都必须保留到第二波，后续直接复用已加载的原文和分析契约；主线程只做台账、预检与汇合。更长文本按 `_parallel_plan.json.executor` 分配，但仍不得每条 lane 单独冷启动。

## `_analysis_brief.md`

这是并发一致性契约，不是新的分析报告。必须短、准、可核验，至少包含：

- `故事核`
- `主角`
- `核心关系`
- `时间边界`
- `固定称谓`
- `BID 注册表`

BID 固定格式：

```text
BID-01 | L起-L止 | 锚点：原文连续短语 | 桥段角色：本桥承担什么升级
```

第一波所有 worker 只能消费这份注册表，不能自行改 BID 名称、编号或边界。

## 第一波首写格式硬闸

派发第一波 lane 时，必须把对应
`_parallel_plan.json.foundation_lanes[].first_write_contract`
完整放入 worker prompt。worker 第一次写文件前先按契约固定格式，禁止先沿用旧模板再等待 foundation validator 返修。

- `main_report`：固定标题必须逐字使用且各出现一次；结构表字段、题面预期翻转和主角能动性三层标签首写齐全
- `chronology_craft`：每条节点从首写起使用单行施工格式；承重桥 BID 直接写在对应 `Nxx` 行；写作手法九个固定章节和四类句子级资产首写齐全
- `discovery_index`：动态字典首次落盘就是合法 JSON；候选池从第一条起保持固定字段顺序，并保留反向漏项审计章节
- 三条 lane 落盘前都要检查 Markdown 标题唯一、无占位标题、无空标签字段
- 契约与 worker 旧模板冲突时，以 `first_write_contract` 为准

禁止：

- 读完前 1-2 个 Chunk 就先写主报告
- 借用上一本文档或旧 profile 的结构和内容
- 只为 validator 命中原句而机械改写施工层
- 样本复核裁决仍为“需要回炉”时继续第二波
- 第一波三个 lane 各自重读整本原文或重排共享规则前缀
- foundation 预检未通过就启动 16 表和写作资产
- 只把 `first_write_contract` 路径告诉 worker，却不把当前 lane 的完整对象放进 prompt

## 台账硬规则

- 每条写成：

```text
F序号 | L起-L止 | 锚点：... | 类别：... | 主体：... | 动作：... | 结果：... | 叙述时点：... | 故事时点：... | 时间依据：... | 口径：... | 禁止越界：...
```

- 至少覆盖：
  - 主体边界
  - 时间边界
  - 证据来源
- 遇到回指词，先定位它指向的更早正文，再写故事时点
- 高主动性判断必须回指 `【原文明确 Fxx】` 或 `【人工推断 Fxx】`

## 主报告厚度要求

`拆文报告.md` 不能只剩合规骨架，固定还要保：

- `题面拆解`
- `读者最先以为什么文`
- `后面哪一步改写了这个预期`
- `叙事时间线`
- `结构划分`
- `主角能动性三层判断`
- `人物分析`
- `情感曲线 / 爆点分析 / 反转分析 / 开头分析 / 结尾分析 / 五维评分`

额外硬要求：

- `拆文报告.md` 不只负责“讲结构”，还要先承担“为什么这本能写成这样”的第一层解释
- `题面拆解 / 结构划分 / 人物分析 / 情感曲线 / 反转分析` 至少各补 1 组：
  - `原文现象证据`
  - `为什么成立`
  - `新稿最容易写假的点`
- 中段承重桥如果真实存在，主报告里必须先讲它怎么把故事从“公开掉位”推到“私域受伤 / 旧伤爆体 / 现实后果”，不能只在细节库、施工卡或 profile 里解释
- 不允许把“为什么原文成立 / 为什么不像加工稿 / 为什么顺序不能乱”全部后置到 `profile_source.md`、`桥段施工卡.md` 或 16 张表；主报告自己必须先保住一轮人类可读解释
- 如果主报告明显比资产层更像“摘要”，默认说明厚度分流过头，当前批不能算完成

## 节点要求

`情节节点.md` 固定按信息释放顺序列出，但每个节点都要写：

- `L起-L止`
- `锚点：原文短语`
- `类型`
- `情绪`
- `涉及`
- `状态变化`
- `因果`
- `故事时序`

额外要求：

- 覆盖全部自动 Chunk
- 有章节标记时覆盖每章
- 最后有效节点进入原文最后 10%
- 原文存在独立承重桥时，必须把 `BID-xx` 直接写在对应 `Nxx` 节点行；写在文件说明、Chunk 核对区或下游资产里不算节点已标注
- 推荐格式：`类型：BID-01 中段承重桥`；也接受独立字段 `BID：BID-01`
- 同一个节点最多挂 1 个 BID，且该 BID 必须继续贯通主报告、目标表、高敏桥、施工卡、profile_source 和 book.profile

## 写作手法要求

每个核心节至少回答三件事：

1. 谁怎么说话 / 怎么动作
2. 为什么原文这样成立
3. 换成什么会发假

尤其 `对话手法` 必须落到人物嘴型或口气差，不能只写“对话推进快”。

## 第一波通过条件

- 主报告厚度不下降
- 节点覆盖与 BID 显式标注通过
- 写作手法保住原文证据、活词、句法和反面仿写层
- 候选池与动态字典完成全文发现
- 候选池达到篇幅最低值，反向漏项审计至少 5 项；不足时不能用“原文密度低”放行
- `_sample_comparison.md` 已完成主报告后复核
- `validate_short_analyze_foundation.py` 返回 `ready-for-asset-lanes`
