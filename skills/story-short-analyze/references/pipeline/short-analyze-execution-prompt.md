# 短篇拆书主执行提示词

这份文件不是方法论说明，而是主会话执行模板。

用途：

- 让 `story-short-analyze` 在正式拆书时固定按一套话术和顺序执行
- 防止模型漏拆、跳批、先写结论后补文件
- 把“全量自动落盘 + 分批自检 + 最终验收”推进到主执行层

---

## 1. 使用方式

进入正式拆书后，主会话应直接套用下面骨架执行，不要自由发挥流程顺序。

硬口径：

- 不允许只写分析回复，不落盘文件
- 不允许用户没点名，就默认少产某些文件
- 不允许先说“核心文件完成”，再把剩余文件留到后续
- 不允许跳过批次自检
- 不允许不跑最终验收脚本就宣告完成
- 默认进入 `厚拆模式`：目标不是做“刚好能过验收的压缩摘要”，而是优先保留最值钱的中段承重桥、人物口气差、动作权限差、物件回流、旧伤触发器、章法失效测试
- 厚拆不等于平均灌长所有文件；它的意思是高价值层默认宁可多保留，不要先压掉
- 只有原文确实没有对应资产时，才允许写薄；不能因为想省篇幅，主动把关键中段桥和人物不同脸压没
- 不允许用“先把所有文件都铺出来，后面再回头补厚”的方式规避主报告深拆；`主报告批` 必须先过厚拆闸门，后续批次才允许开始
- 如果 token、时间或篇幅出现压力，优先保住 `拆文报告.md / 情节节点.md / 写作手法.md / profile_source.md` 的厚拆层，不允许为了平均铺满 16 张表和细节库，把主报告写成压缩版

---

## 2. 主会话固定执行骨架

把下面内容当作执行提示词使用：

> 你现在执行 `story-short-analyze` 正式拆书。目标不是只写分析结论，而是自动产出 skill 定义过的整套拆书资料包，并在结束前完成全量验收。
>
> 默认按 `厚拆模式` 执行：不要满足于“文件齐了、脚本过了”，而要优先保留后续仿写最值钱的层，尤其是中段承压桥、人物口气差、动作权限差、物件铰链、旧伤触发器、章法失效测试。
>
> 先读取并遵守：
>
> - `references/pipeline/analyze-execution-core.md`
> - `references/pipeline/output-contract.md`
> - `references/pipeline/output-templates.md`
> - `references/pipeline/quality-checklist.md`
> - `references/pipeline/auto-full-output-task.md`
>
> 再固定按 6 批顺序执行：
>
> 1. 主报告批
> 2. 16 张可直接仿写表批
> 3. 原文细节库批
> 4. 写作资产批
> 5. `profile_source + book.profile` 批
> 6. 全量验收批
>
> 每一批都必须：
>
> - 先写完本批全部文件
> - 再做本批自检
> - 自检通过后才能进下一批
>
> 如果任意一批缺文件、缺骨架标题、缺关键段落，立即回到该批补齐，不准继续往下走。

---

## 3. 第 0 步：原文读取与事实边界闸门

正式进入第 1 批前，先确认：

1. 已拿到原文全文，不是摘要
2. 已读取 `_source_manifest.json` 和 `_source_reading_plan.md`
3. 已按读取计划逐块读到 manifest 记录的最后一行，不是只看一次受限输出
4. 已核对最后一节、最后事件、最后关系状态和尾部锚点
5. 已确定输出目录 `拆文库/{书名}/`
6. 已完成拆前 4 判断，并准备写入 `拆文报告.md`
7. 已明确本次任务不是“精简版拆书”

如果任意一项不满足，不能进入正式拆书。

进入第 1 批时固定执行：

- 先落 `事实与推断台账.md`，再写主报告；台账至少覆盖 `主体边界 / 时间边界 / 证据来源`
- 台账格式固定为 `F序号 | L起-L止 | 锚点：... | 类别：... | 主体：... | 动作：... | 结果：... | 口径：... | 禁止越界：...`
- 任何高主动性判断必须句末回指 `【原文明确 Fxx】` 或 `【人工推断 Fxx】`
- `拆文报告.md` 先写 `### 原文覆盖确认`
- `情节节点.md` 每个节点写成 `N序号 | L起-L止 | 锚点：原文短语 | 类型：... | 情绪：... | 涉及：... | 状态变化：... | 因果：...`
- 锚点必须能在对应行范围内找到，不能写概括词代替原文锚点
- 节点必须覆盖全部自动 Chunk；存在章节标记时，每章至少 1 个有效节点
- 最后有效节点必须位于原文最后 10%

## 4. 六批执行模板

## 第 1 批：主报告批

### 必产文件

- `拆文报告.md`
- `情节节点.md`
- `写作手法.md`
- `_meta.json`

### 必做动作

1. 先按 `output-templates.md` 生成三份主报告
2. `写作手法.md` 必须严格按固定骨架落盘
3. `_meta.json` 必须补齐结构计数和文件计数基础字段

### 本批自检

- `拆文报告.md` 是否包含：
  - `原文覆盖确认`
  - `样本分级与可学层判断`
  - `高敏桥段识别`
  - `题面拆解`
  - `故事梗概`
  - `结构划分`
- `拆文报告.md` 是否额外补齐：
  - `#### 1. 脚本硬筛`
  - `#### 2. 规则拆层判断`
  - `#### 4. 可学层 / 禁学层`
  - `#### 5. 后续调用方式`
  - `### 叙事时间线`
  - `### 故事核`
- `拆文报告.md` 的 `结构划分` 是否显式写了 `字数范围 / 占比 / 功能 / 对应节`，而不是只写 4-6 条概括式分段
- `情节节点.md` 是否覆盖全文主要节点
- `情节节点.md` 是否每个节点都有 `L起-L止 + 锚点`
- `情节节点.md` 是否每个节点都有 `类型 / 情绪 / 涉及 / 状态变化 / 因果`
- 节点锚点是否覆盖 `_source_manifest.json.chunks` 的全部分块
- 有章节标记时，是否每章至少存在 1 个有效节点
- 最后有效节点是否进入原文最后 10%，而不是靠“终局前夜”字样冒充尾部覆盖
- `情节节点.md` 是否拆到可施工颗粒度：
  - 最低节点数取分档阈值与 `ceil(原文字数 / 400)` 的较大值
  - 8000 字以上基础阈值 20，5000-7999 字基础阈值 16，5000 字以下基础阈值 12
  - 10840 字文本最低应拆到 28 节点
- `情节节点.md` 是否显式保留至少 1 条中段承重桥节点，而不是只把导语钩子和终局翻盘写清
- `写作手法.md` 是否按固定 9 节骨架完整落盘
- `写作手法.md` 是否不是每节只写 1 句总括，而是至少拆到：`人物嘴型/口气差`、`章法失效测试`、`物件铰链或意象物件`、`迁移风险`
- `写作手法.md` 是否写清“原文怎么做才成立”，而不是只写“用了倒叙/留白/铺垫”
- `写作手法.md` 的 `对话手法` 是否显式落到至少 3 类角色嘴型或口气差，而不是只写“对话有反差/推进快”
- `拆文报告.md` 是否不只停在基础结构层；如果目标是后续写作，是否已补 `情感曲线 / 爆点分析 / 反转分析 / 人物分析 / 开头分析 / 结尾分析 / 五维评分`
- `_meta.json` 是否存在 `structure_counts`

只要有一项缺失，本批不通过。

## 第 2 批：16 张可直接仿写表批

### 必产文件

- `可直接仿写_导语拆解表.md`
- `可直接仿写_顺序事件表.md`
- `可直接仿写_物件表.md`
- `可直接仿写_动作表.md`
- `可直接仿写_对白功能表.md`
- `可直接仿写_对话衔接表.md`
- `可直接仿写_误判表.md`
- `可直接仿写_钩子表.md`
- `可直接仿写_微动作表.md`
- `可直接仿写_安静压迫场表.md`
- `可直接仿写_人物偏手表.md`
- `可直接仿写_失控说话表.md`
- `可直接仿写_烂关系漏出表.md`
- `可直接仿写_外部秩序表.md`
- `可直接仿写_公开炸场表.md`
- `可直接仿写_后果链表.md`

### 每张表必须补的 3 段

1. `可直接借的承重结构`
2. `迁移顺序提醒`
3. `为什么这个顺序不能乱`

### 写法硬约束

- 先把表格行写够，再写 3 段施工层，不能先写空说明后补表
- 每张表默认至少写 4 条有效资产，并显式包含 `原文现象 / 原文证据 / 原文位置 / 原文例子 / 原文功能 / 原文怎么写 / 动作本体` 中至少 1 类证据列；按表类型选自然列名，不要机械统一
- 表名对应的语义列不能省：`人物偏手表=角色+稳定偏手`，`误判表=先误判了什么+从哪开始翻`，`安静压迫场表=场面压力来源/环境音`，`烂关系漏出表=具体漏出件`
- 3 段施工层每段至少写 2 条可施工说明；只有一句总括视为压缩化
- 3 段施工层都必须点名表格里的具体条目，至少 2 个
- 点名时优先直接复用表格第一列或核心资产列里的原条目名，不要只写抽象概括；如果施工层里看不见表内原名，默认这一张表没写实
- 如果这一张表属于 `顺序事件表 / 动作表 / 对话衔接表 / 微动作表` 这类顺序或动作承重表，施工层优先直接复用整条表格原句、原链条或完整条目文本，不要只摘两个关键词
- `顺序事件表` 默认至少写 4 条事件链，不能只保留开头链和终局链；其中必须额外保留至少 1 条中段承重链，优先保留那种“公开掉位/外部压力 -> 身体或生活掉位 -> 私域再受伤”的过桥链
- 如果原文存在“办公室冲突 / 公开见血 / 病情暗示 / 搬出家门”这类中段承压桥，优先写进 `顺序事件表` 和 `高敏桥段识别`，不要被额外的导语桥或终局桥挤掉
- 如果这一张表属于 `导语拆解表 / 误判表`，施工层也要直接点进表格单元格里的栏目值，例如 `前20字 / 前60字 / 前100字`、`只是偏向闺蜜 / 实际已把女主资源也挪走 / 共同账户买车`，不要只写“开头母题”“误判翻转”这类概括词
- `迁移顺序提醒` 必须写成“先 A，再 B，最后 C”；A/B/C 都要能在表格里找到
- `为什么这个顺序不能乱` 必须写“如果把 B 抢到 A 前面，会假在哪里”
- `为什么这个顺序不能乱` 里提到的 A/B/C，也必须直接写成表格里看得到的原条目名；不要把“及笄择婿时兄长说她嫁谁都一样”缩成“婚嫁主位”，也不要把“先错轿救人 -> 再程家破局 -> 再三朝回门清算”缩成“后半清算”
- 16 张表不允许共用同一句提醒句；每张表都要像只对这张表有效
- 写完后再问一句：`把表名遮住，这 3 段还能不能只属于这一张表？`
- 如果答案是“放到别的表也差不多能用”，立即重写，不准带着万能句进入下一张表

### 生成时自问

- 这 3 段里，我有没有直接点到表里写出来的资产名？
- 如果把表名遮住，这 3 段还能看出它只属于这一张表吗？
- 如果不能，说明还在写通用空话，必须重写
- 这张表更像“事件顺序类 / 物件动作类 / 对白口气类 / 关系秩序类”中的哪一种？
- 我写出来的 3 段，是否符合这一类表该有的施工重点？
- 我有没有明确写出“哪一步先来，哪一步后翻，抢跑后具体假在哪里”？

### 本批自检

- 16 张表是否齐全
- 每张表是否非空
- 每张表是否都含固定 3 段
- 每张表的 `迁移顺序提醒` 是否至少点名 2 个本表条目
- 每张表的 `为什么这个顺序不能乱` 是否明确写了“哪一步抢跑会假”
- 每张表的 `为什么这个顺序不能乱` 是否用的是表内原条目名，而不是“婚嫁主位 / 后半清算 / 前账 / 大翻盘”这类抽象代称
- `顺序事件表` 是否默认写满至少 4 条链；若不足 4 条，是否明确说明原因
- `顺序事件表` 是否明确保留了至少 1 条中段承重链，而不是只剩导语链和终局链
- `顺序事件表` 的中段承压链是否写成至少 3 段箭头链，而不是几个散点短语
- 是否出现跨文件复用的统一提醒句；只要 2 张以上雷同，整批退回

缺任意一张或缺任意一段，都回到本批补齐。

## 第 3 批：原文细节库批

### 必产目录与文件

- `原文细节库/场景细节库.md`
- `原文细节库/关系细节库.md`
- `原文细节库/情绪细节库.md`
- `原文细节库/对白细节库.md`
- `原文细节库/翻车细节库.md`
- `原文细节库/旧伤细节库.md`
- `原文细节库/动作细节库.md`
- `原文细节库/场面细节库.md`

### 本批自检

每一份细节库至少回答：

1. 具体发生了什么
2. 这个细节为什么有用
3. 它压的是谁、压在哪
4. 后续能迁到什么新桥段
5. 它对应的角色 / 情绪 / 反转是什么

### 写法硬约束

- 每个 `##` 小节都要先确定“这一小节对应的是原文哪一件具体事”，再写五条
- 第一条必须写具体事件，不许写“原文里出现了某类场面”
- 第四条必须写 1-2 个具体可迁桥段，不许写“同题材桥段”
- 不同 `##` 小节的五条答案不能整段复用，只换标题
- 如果五条里有三条几乎放到别的小节也通用，说明还没拆到细
- 第 2 条必须说明“为什么这件证据比一句总结更有效”，不能只写“让情绪落地”
- 第 3 条必须写清压的是 `脸面 / 名分 / 私域 / 旧伤 / 身体 / 秩序` 中哪一种，不能只写“压主角”
- 第 4 条除了桥段名，还要补一句“迁移时真正不能丢的承重件是什么”；正文里少用 `可迁到`，统一优先写成 `能迁去...，前提是保留...`
- 第 5 条必须落回这一小节自己的角色站位，不能拿整本三角关系总评来糊

### 生成时自问

- 读这一小节的人，能不能直接看出是谁在什么地方做了什么？
- 这五条答案如果把标题删掉，还能不能分辨它和别的小节不同？
- 如果不能，说明还是模板壳，必须重写
- 这一小节真正的证据件是场景、物件、动作、对白，还是关系掉位？我有没有写出来？
- 这条细节真正压的是体面、名分、私域、旧伤、身体还是秩序？我有没有写出来？
- 这一条如果只剩“抽象评价”，删掉具体事件后还成立吗？如果成立，说明还没落到证据层

另外强制检查：

- 不允许出现“原文里出现了这一类场面”“同题材桥段”这类泛化壳句
- 不允许在多个小节反复写 `可迁到...` 这种模板句壳；第 4 条优先改成 `能迁去...，前提是保留...`
- 同一份细节库里，不允许多个 `##` 小节五条答案只换标题不换内容
- 每个 `##` 小节至少要有 1 个具体事件锚点，能看出是谁在什么地方做了什么
- 每个 `##` 小节都应该像一张“证据卡”，不是一个题材词解释

- 如果原文存在“办公室冲突 / 公开见血 / 病情暗示 / 搬出家门”这类中段承压桥，`高敏桥段识别.md` 是否已优先纳入，而不是被多余的导语桥/终局桥替换
- `高敏桥段识别.md` 是否只保留了一套有效卡片，而不是前面留空壳、后面再重写一遍
- `高敏桥段识别.md` 是否不存在 `桥1 / 桥2` 这类空占位标题、半成品草稿壳、重复卡片
- `作者DNA指纹.md` 是否不是空标签，且 3 个核心段都显式写了 `原文：`
- `仿写约束_禁写清单.md` 是否每条都回答“为什么假”
- `同桥段过检规则.md` 是否显式写了 `原文：`，并写清起手件、承重件、假点、顺序原因
- 上面 3 组高敏桥强制补项是否真的写进了文件正文，而不是只在脑子里判断过

## 第 5 批：`profile_source + book.profile` 批

### 必产文件

- `写作资产/profile_source.md`
- `book.profile.json`

### 写法硬约束

- `profile_source.md` 不是只服务脚本抽取，还要服务人直接拿去写；不能只剩结构桶和 json 友好短句
- 在保留结构化小节的同时，必须额外补出明确的人类可读层：`句长切法`、`口气差`、`动作替代`、`旧伤触发器`、`反面句型`
- 这些人类可读层必须写成“原文怎么做 / 为什么这样才像它 / 换成什么就不像”，不能只写标签词
- `桥段：...` 小节不只写亮桥，也要把中段承重桥写进去，避免 profile 只剩导语桥和终局桥

### 本批自检

- `profile_source.md` 是否同时存在结构化抽取层和人类可读解释层
- `profile_source.md` 是否显式包含：`句长切法`、`口气差`、`动作替代`、`旧伤触发器`、`反面句型`
- 上述 5 类是否都不是空标签，而是写清原文证据、起效原因、误学风险
- `book.profile.json` 是否仍能从 `profile_source.md` 正常抽出非空结构字段，而不是被人类说明层冲散

### 固定顺序

1. 先由模型补完 `profile_source.md`
2. 再生成 `book.profile.json`
3. 再检查 JSON 关键字段是否齐

## 第 6 批：全量验收批

### 洁净与贯通硬闸
