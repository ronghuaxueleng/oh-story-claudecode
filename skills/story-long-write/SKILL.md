# story-long-write：长篇网文写作

你负责长篇网文写作，从选题、搭大纲到正文落地都在这个 skill 里处理。

---

## 规则怎么分

本 skill 里的规则分三层：必停、质量、提醒。别混着用。

### 1. 先停层

- 定义：只要撞上就得停，这关不过就不能继续
- 适用：写前检查、写后检查、这一章先留不留、失败码、脚本扫描、追踪同步
- 怎么用：这些都得写进项目文件、都得复检、都得和 `F0-F5` 对上

### 2. 质量层

- 定义：用来看正文到底扎不扎实，能触发 `F1-F3/F5`，但有些地方还得人工帮着看
- 适用：人物出场分量、章尾力度、书名主打卖点兑现、信息场比例、主角声线、高价值角色出场分量
- 怎么用：把这些先写进写前闸门和写后检查，再让审查或复扫重过一遍

### 3. 提醒层

- 定义：写作思路、参考路径、风格提醒
- 适用：情绪方法、对标方法、模块召回、写作技巧、创作提醒
- 怎么用：这一层是帮你把事情理顺的，别单独拿它来直接判死

先记这几件事：

- 必停层 > 质量层 > 提醒层
- 只有高频翻车、能客观识别、也能绑到失败后果的问题，才允许升到必停层
- 别把所有创作建议都硬往必停层上塞，不然流程会越来越重，正文也会越写越僵
- 凡是已经升到必停层的规则，都得做到“撞上以后这一步真停住”

## 快速索引

### 必停层速查

- `F0-F5` 失败码
- 写前该查什么
- 章节生产常走的路
- 细纲准入必查项
- 题材识别 + 对应专项检查
- `scene_lint.py`
- 正文纯净度检查
- 章尾没通过就别收进主正文
- 情报兑现时限
- 追踪/角色卡同步
- 独立交叉复查
- 事实层 / 预案层隔离
- 连着回修时要用的检查单 + 执行计划
- 回修怎么分（补丁式 / 一次性整段重写）

### 质量层速查

- 人物出场分量最低配额
- 主角声线边界
- 配角工具嘴
- 后半章信息场比例
- 高价值角色能不能让人记住、还想继续看
- 书名主打卖点兑现强度
- 外显战果是否扎实
- 章尾是否只是低空擦过
- 功能场景有没有写成只是在把规矩往外念
- 小说类型怎么分
- 爽点怎么落
- 章节名策略
- 解释句密度阈值
- 高潮双段兑现

### 提醒层速查

- 先抓情绪的写法
- 对标/拆文/模块召回
- 商业爽文方法论
- 写作技巧提醒
- 情绪弧线与反转参考

### 细纲必查项速查

进正文前，先拿这张短清单卡一遍细纲：

- [ ] 当前连续章组是否已有 `章组收益卡`，并且已经写清这组结束后到底是谁被逼得把话收回去 / 谁往前挤了半步或退了半步 / 谁手里的活被改了 / 谁把原来的位置丢了
- [ ] 当前章组是否做了 `前移收益检查`：所有前置承接位都要各自承担 `1` 次看得见的现实收益；别把所有硬兑现都压到章组最后一下
- [ ] 本章最先要拿到的收益，到底落在谁手里
- [ ] 本章是否能直接答出“现实改了什么”；若只能答“查到什么 / 知道什么 / 怀疑什么”，就先别进正文
- [ ] 第一爆点之后，第二轮收益到底是什么
- [ ] 本章阻力是谁，怎么拦，主角怎么破
- [ ] 本章会有谁失态、露底、变脸、把话缩回去、被逼着选边
- [ ] 本章物证 / 名分 / 规矩变化到底落在哪一拍
- [ ] 章尾到底停在动作 / 物证 / 失态 / 压制 / 翻面里的哪一拍
- [ ] 情节点序列是否写到了“谁做了什么，造成什么变化”，而不是只写提纲词
- [ ] 本章最先要拿到的 `1-2` 个场面，是否已经拆成 `场面卡 / 动作拍序`
- [ ] 本章世界信息到底落在哪个点上，靠哪个场面里的实物、规矩或人物落地，而不是准备在正文里顺嘴解释
- [ ] 细纲和写前闸门是否只有一个版本，没有收益归属和章尾落点冲突
- [ ] 本章回答的是“查到什么”还是“改掉什么”；若只有前者没有后者，本章是否应该退回章组层重做
- [ ] 关键配角这几件事是否已经交代到位：`岗位 / 怕什么 / 想要什么 / 本章会冒什么风险`；若配角只承担送证据、传话、背书、跑腿功能，先回细纲补人物冲突卡
- [ ] 公开冲突里的对手，是否已经交代清楚 `当场怎么压回来`：能动谁、能压谁、能记谁、能罚谁、能拖哪条线；若对手只剩张嘴骂两句，先回细纲补对抗手段
- [ ] 本章第三方是否真的参与改局：围观兵、书吏、中立者、现场执行人里，得有一人承担 `把事情补圆 / 卡物证 / 传话 / 逼人选边` 的现实作用；若第三方只会震惊围观，先回细纲补世界压力位

上面只要有一项说不实，这章就还是虚的，先别进正文。

## 开工前先认这几条

本 skill 是拿来真干活的，不是给你上课的。

### 书籍级铁律读取必查项

进入 `story-long-write` 的任一长篇任务前，先定位“当前真实书目录”，再显式读取该目录下的 `写作执行铁律.md`。

就这么做：
- 若项目存在 `.active-book`，就把它指向的目录当当前书目录。
- 若不存在 `.active-book`，就以包含 `追踪/` 的真实书目录作为当前书目录；若同时冒出多个可用目录，先明确当前要操作哪一本，再继续。
- 就按这个次序读：`<当前书目录>/写作执行铁律.md -> <当前书目录>/追踪/上下文.md -> <当前书目录>/追踪/时间线.md -> <当前书目录>/追踪/角色状态.md -> <当前书目录>/追踪/伏笔.md`；涉及情报流时，再读 `追踪/情报记录.md`。
- 若 `写作执行铁律.md` 缺失，而任务属于长篇开书后的正文生产、续写、回炉、审查衔接，直接按 `F5` 停在这里；把书籍级铁律先补出来，别只靠项目根 `CLAUDE.md` 或聊天上下文代替。
- 若当前任务是“仅部署宿主环境、尚未创建书籍目录”，就别伪造书目录或预建铁律来凑这条；把书籍创建好，再进长篇写作流程。

这条比普通“上下文恢复提示”更靠前。只要还没读完书籍级铁律，就别进写前检查、细纲准入、正文生产或写后检查。

一旦撞上失败项，就按“失败 -> 回退 -> 重做 -> 再查一遍”来，别再乱顶着往下写：

- 先写完再审查
- 嘴上说“注意一下”后照样继续写
- 只删词不补现场
- 不把闸门和检查结果写进项目文件

所有写作、改章、回炉动作，都要先落失败码，后面留不留、停不停，都按它来：

| 失败码 | 含义 | 接下来要做什么 |
|------|------|---------|
| `F0` | 没撞上失败项 | 可进入下一步 |
| `F1` | 句子层失败：空效果句/解释腔/局部AI味 | 当场回修，复扫过前先别结束本章 |
| `F2` | 章节层失败：章尾没通过/场面不足/战果偏虚 | 回退到本章细纲或本章正文重写，禁止进入下一章 |
| `F3` | 整体写法跑偏：书名主打开始写偏/连续多章只查不爆/副线反过来压主线 | 停止顺写，回退到章节组细纲或卷结构 |
| `F4` | 污染失败：工具残片/补丁标记/执行元话语/非故事语料 | 先清污染，再全文复扫 |
| `F5` | 流程失败：角色卡/追踪/写前闸门/检查单缺失 | 把缺的文件先补全，补完前这轮先别继续 |

后面凡是写“这章留不留 / 这关过没过去 / 要不要停写”，都得对到上面的失败码，别只写含糊话。

### 人工接管怎么走

`人工接管` 不是给人选的开关，而是自动返修。撞上条件就直接进来，别先问“要不要人工接管”。

自动切入条件：
- 正文出现污染残片、半成品中断、运行时异常
- 同一文件已经明显滑出书名主打，或者整套写法开始跑偏
- 细纲能看出主线打架、章组重复、书名主打卖点落空
- 子代理 / 脚本 / 审查那边已经证明“继续自动顺写只会越写越偏”

进来后，先看当前任务卡在哪，再标记成下面三种里的一个：

- `H1-A 纯测试记录态`：适合用户明确要求“纯 skill 冷启动试跑 / 基准测试 / 再跑一遍看看 / 不要人工干预”的情况。这里最多只清污染残片、修文件截断、清明显非故事语料、修运行时异常导致的不可读段；别去改正文结构、收益落点、场面主拍、句子质感。正文要是还是卡在这里，就只把失败记录、失败码、触发规则、原样样本路径写进项目文件，别把失败正文直接修成成品。
- `H1-B 生产自动返修`：适合用户没明确要求纯测试，或者目标就是把作品写到能用。自动接手 `细纲 / 正文 / 追踪 / 章组` 里对应的失败范围，按 `F1-F5` 回退重做，直到重新检查后回到 `F0`，或者明确写出为什么现在还收不住。
- `H1-C 一起写一起修`：适合用户允许“人工一起做 + skill 协同生产”的情况。目标不是保留纯 skill 原样样本，而是边产出边停下边修，尽量把问题灭在首稿和写后检查之间。skill 负责主要推进，人工可以在首稿后、写后检查前、交叉复查前直接插手，修 `细纲 / 正文 / 追踪 / 章组` 的局部或整段问题，再回到同一条路重新检查。要是生产过程中已经认出明确错误（如一级禁用词、明显病句、解释腔、工具残片、事实内容写错、追踪项目错填），别攒着等审查，直接当场修掉，再继续当前步。

三种状态都得做到：
- 要把接管范围写明：`细纲 / 正文 / 追踪 / 章组`
- 要把接管原因写明：对应 `F1-F5`
- 接管后还是得走同一套写前闸门、写后检查、追踪同步，别因为“自动接管”就把检查口跳过去
- 别把 `H1-A` 的失败样本装成 `H1-B` 的修复成果；也别把 `H1-B` 的人工修复装成纯 skill 原样产物

`H1-C 一起写一起修` 还要补这几条：
- 一旦发现明确错误，就在当前生产步当场修，别先挂成“待后修”，也别拖到后面一起看时再一起处理。
- 这里的“明确错误”包括但不限于：
  - 一级禁用词、复发污染词、工具残片、非故事语料
  - 明显作者代判句、解释腔、口语失真、生硬对白
  - 事实层内容写错、追踪项目错填、情报记录漏记
  - 已经能看出根因的 `scene_lint.py` 报错或高危预警
- 当场修完后，把修正后的版本继续送回当前这条路，别留着“错误原稿 + 另外记一份改法”拖到后面。
- 人工可以在下面这些时候直接进来，不用等后面那轮再一起看：
  - 正文首稿刚写完后
  - 首稿污染自检报问题后
  - `scene_lint.py` 报 `error/warn` 且已经看出根因后
  - 写后检查或交叉复查发现正文 / 细纲 / 追踪前后写得不一致后
- 人工可以直接进来处理的问题包括：
  - 一级禁用词、复发污染词、工具残片、非故事语料
  - 明显作者代判句、解释腔、口语失真、生硬对白
  - 章尾塌陷、动作链断、场面空转、对话不改现场
- 细纲没把该写的事写进去、允许新增清单缺失、追踪内容写错
- 协同介入后，要回到原来的顺序继续走：`当前失败点回修 -> 纯净度检查 -> scene_lint -> 写后检查 -> 追踪/角色卡同步 -> 交叉复查`
- 只要人工改动了正文结构、收益落点、场面主拍、角色说话方式或追踪事实记录，最后都要明写“本轮为协同产物，不是纯 skill 原样产物”

### 人工回炉模拟怎么走

`人工回炉模拟` 的意思是：人工不逐句接手时，skill 就按“像编辑那样一轮一轮往回修”的办法自己返修；但它学的是这套改法，不是装成自己已经有稳定的人来收最后一眼。

触发条件：
- 用户明确要求“让 skill 自己回炉 / 模拟人工回炉 / 自动改到更像人写”
- 或当前任务已进入 `H1-B 生产自动返修`
- 或 `scene_lint.py / scene_narrowness_lint.py / 写后检查 / 交叉复查` 已证明问题主要集中在 `对白 / 人物温度 / 声线混同 / 过程写窄 / 作者代判`

硬要求：
1. 别让同一轮同时负责 `诊断 + 重写 + 自己看自己有没有过`。先拆成三步：
  - `critic`：只诊断，不负责润稿
  - `rewrite`：只按诊断单回修，别自己扩题
  - `recheck`：独立复检，别直接沿用 `rewrite` 的自评
2. 每轮回修只能有一个主问题，别一轮同时追打 `结构 + 情绪 + 声线 + 章尾 + 追踪`
3. 若某轮只是在换近义词、缩句、换语气词，而没有改掉 `动作 / 对象 / 代价 / 场面后果 / 人物怕处` 之一，就记为“无实质提升”
4. 连续两轮“几乎没真变好”，就自动退出这套自改办法，回到 `H1-C` 或人工接手；别没完没了自己改

#### 模拟人工回炉怎么走

每轮都按这个次序来：
- `诊断单 -> 单轮回炉 -> 独立复检 -> 记录是否实质提升`

诊断单先回答：
- 当前主失败项是什么：`对白提纲腔 / 人物发冷 / 声线混同 / 过程写窄 / 作者代判 / 章尾翻译`
- 这个问题最集中落在哪些段落 / 哪几句 / 哪个角色
- 本轮只允许改什么
- 本轮明确不改什么
- 本轮到底卡在哪一条，把这一条先处理掉再继续

诊断单要是答不出上面四项，就先别进 rewrite。

#### 分轮回炉先后顺序

回修照下面这个次序来：
1. `结构 / 动作后果`
2. `对白去提纲腔`
3. `角色声线分化`
4. `人物温度 / 场边人热度`
5. `章尾 / 收束`

这几条别乱：
- 若 `scene_narrowness_lint.py` 报 `warn/error`，就先处理 `结构 / 动作后果`，别急着润对白
- 若 `scene_lint.py` 主要报的是 `对白` 类问题，再进 `对白去提纲腔`
- 若对白已经基本像人说话，但角色互相像同一个人，再进 `角色声线分化`
- 若角色说话已有区分，但读感仍冷、疼感不足，再进 `人物温度 / 场边人热度`

禁止：
- 结构没稳就先做“金句优化”
- 角色未分化就先做“台词更狠”
- 过程写窄未解就先靠情绪补热

#### 角色抽线回炉

凡是撞上 `声线混同 / 人物发冷 / 高价值角色只像工具人 / 对白读起来谁都像一个人`，先做一次 `角色抽线回炉`：

先这样做：
- 把当前章节中每个关键角色的全部台词单独抽看
- 每个角色先答四问：
  - 他现在最怕什么
  - 他最想护什么
  - 他不敢明说什么
  - 他急了以后，句子会怎么变短、变狠、变脏、变虚
- 若答不出，就别直接改正文狠话；先回角色卡 / 人物冲突卡补最小声线说明

角色抽线后的回修方向：
- `宁执` 这类推进口角色：先往 `短 / 扎 / 直戳怕处` 改
- `韩柏` 这类保位口角色：先往 `怕担责 / 撑脸 / 先甩锅` 改
- `鲁成` 这类守旧口角色：先往 `护旧账 / 护旧册 / 越拦越虚` 改

若回修后仍能随意互换角色名而句子基本还通，就记成 `声线分化失败`，先按 `F1`

#### 对白回炉模板

撞上 `对白提纲腔 / 功能摘要台词 / 口语失真` 时，别只丢一句“改成人话”。照下面顺序拆句：
- 这句承担什么功能
- 这个人想护什么
- 这个人怕什么被戳穿
- 他会拿什么话先挡
- 若被逼急，他会怎么缩短、怎么变狠、怎么失态

若只改成“更顺口”却没补出 `护什么 / 怕什么 / 怎么挡`，就当成假回炉。

#### 独立复检与停机规则

`rewrite` 做完后，`recheck` 得单独回答：
- 主失败项是否真的下降
- 是否出现了新的问题
- 本轮有没有实质提升

`实质提升` 就看有没有碰到下面任一项：
- 结构上新增了真实动作后果链
- 对白从功能摘要改成带对象、带代价的人话
- 角色之间的声线明显更可区分
- 场边人从背景板变成真实施压或抢命力量

若 `recheck` 只能说“更顺了一点 / 更自然了一点 / 语气更像了”，而答不出上面任一项，就记为“无实质提升”。

连续两轮无实质提升时：
- 自动停止 skill 自改
- 明确标记 `建议转人工接管`
- 别继续无限循环“再润一版”

限制：
- 若当前任务目标是“纯 skill 冷启动试跑 / 基准测试 / 再跑一遍看看”，就用 `H1-A`；别直接拿来把失败正文修成漂亮成品，也别借机改写正文结构去“顺便优化”
- 若接管已改变正文结构、收益落点或场面主拍，最终要明确说明“本轮不是纯 skill 原样产物”

### 剧情怎么生产

后面别老把剧情全丢给模型自由乱编。若任务目标是连续正文、冷启动开书、参考情节库注入，或者用户已经明确说“模型自己写的剧情很傻 / 都差不多 / 不要自由发挥”，就切到下面这套写法：

1. 先定 `章组功能链`，再定正文，别直接从题材词或一句梗概硬扩正文。
2. 先定 `现实卡法从哪来`，再定冲突靠什么东西打起来，别只从别的小说里抄场景壳。
3. 先定 `动作 -> 后果 -> 升级` 链，再定句子，别先想“这一章大概很爽”再边写边补。
4. 先定 `场景目标 -> 当场阻碍 -> 被迫改手 -> 现场变化`，再写场面，别跳过这层直接把剧情梗概摊成正文。

这条比“自由联想式创作建议”更靠前。后面若还直接让模型自由发明大段剧情，通常按 `F3` 看：这不是句子问题，是整套起剧情的办法跑了。

### 冷启动现在大概是这样

按多题材真实冷启动样本回头看，现在大致能分成两块：

1. 现在相对稳的部分
- `opening-design.md`
- `genre-writing-formulas.md`
- `style-commercial-theory.md`
- `hooks-chapter.md`
- `dialogue-mastery.md`
- `character-design-methods.md`

这批资料眼下最好用的地方，不是“直接把正文写漂亮”，而是：
- 把开场直接压进冲突
- 把 `第一爆点 / 第二轮收益 / 两轮外部反应 / 章尾停新结果` 的主线拉对
- 把主角、明敌、高价值角色、记录口最起码写到让人记得住

2. 现在最麻烦的短板
- 不再主要是 `解释腔`
- 也不再主要是 `模板挂名`
- 已经收缩为：`正文厚度不足`
- 但当前任务若是“测试模板和情节是否正确使用”，那 `厚度/字数` 先往后放，不当这轮先看的问题；先看 `模板是否进入正文`、`情节是否改现实`、`收益停在哪一层是否判对`、`审查 是否抓到模板漏用`

现在看下来，大致就是这样：
- 情节方向大多没歪
- 现实改局、把原来的话换掉、把先后次序改掉，这些也常能在正文里看见
- 但中后段经常还是偏薄，只能算“主线写对了”，还没到“场面真的热闹起来了”
- 也就是说，这套 skill 现在更像是“先把主线掰正”，还没到“随手就把场面写厚”

这里的边界也先记清：
- `scene-thickening-method.md` 确实能把新增动作拍写进正文，不只是停在检查表上
- 但它眼下更擅长的，还是把主线写实，不是直接把整章写厚
- 所以 `3-5` 拍扩拍卡，更像第一轮加厚工具，还不是最后拿来救场的办法
- 如果样本里已经写出 `差役改线 / 苦主争先 / 第三方抢第二手失败` 这类新增拍，脚本还是报 `字数.error`，别急着说“资料没用上”；更常见的问题是拍子还不够深，或者拍和拍之间连得还不够长
- `chain-reaction-scene-thickening.md` 也已经证明，能把并列拍拉成连锁拍，让事情传得更清楚
- 但它眼下更擅长的，同样还是把事情写清，不是一下就把厚度顶上去
- 如果样本已经写出清楚的 `起手动作 -> 第一被动改线 -> 第二被动改线`，脚本还是报 `字数.error`，也别再误判成“连锁动作拍还不够多”；更像是整章有效长度还不够，连锁也还没自然长成更大的场面

所以后面先盯这几件事：
- 先研究怎么把已经对的情节主线，扩成够厚的动作拍、场面拍、争先拍
- 别再先往上堆总规则、总必停项、总说明
- 新规则如果不能直接把 `新增动作 / 新增阻力 / 新增外部改线` 写进正文，就先当低价值补丁看
- 新规则如果只能把主线讲得更明白，却抬不起厚度，就把它降回辅助工具，别继续拿它硬当厚度主要办法

### 参考文风怎么用

这轮要是明确用了参考情节库、连续剧情库、文风参数卡、语录化用库，一般就按下面做：

#### TXT 和情节库怎么分工

- 若参考源同时包含：
  - `原始TXT正文 / 全本TXT / rag正文`
  - `本地TXT情节库 / 提取资料库 / 连续剧情库 / 场景母模板库`
- 那就先分清谁管什么，别混成一锅一起喂。

分工先认清：
- `情节库 / 提取资料库` 只负责：
  - 本章借哪种往前推的办法
  - 本章改哪处现实局面
  - 本章收益停在哪一层
  - 对手怎么当场压回来
  - 第三方为什么迟疑，后来又为什么往前站了一步、往后退了半步
  - 章尾得落下什么新后果
- `原始TXT正文` 只负责：
  - 开局起爆速度
  - 主角怎么把场面拿下来
  - 对白咬人程度
  - 群像反应密度
  - 章尾停拍方式

反过来这么用就不对：
- 别用 `TXT正文` 决定：
  - 本章具体发生什么事件
  - 本章冲突对象是谁
  - 本章收益物是什么
  - 本章翻盘顺序如何照搬
- 别用 `情节库 / 提取资料库` 直接生成整章正文话术
- 别边看参考书同章原文边直接落本章正文；如果确实要读 TXT，先把用途缩到：
  - `开头怎么更快进事`
  - `对白怎么更压人`
  - `章尾怎么停在现场`
  这类表演问题，再继续正文

次序别乱：
1. 先用 `情节库 / 提取资料库` 填完 `参考输入卡`
2. 记下这些：
   - 本章借什么往前推的办法
   - 本章禁什么表皮
   - 本章改哪处现实局面
   - 本章章尾停在哪个新后果
3. 再允许抽看 `TXT正文` 的开头 / 对白 / 章尾
4. 最后才允许落正文

一票否决：
- 若出现“参考书这章后面就是这么接的，我也这么接”这一类使用方式，直接按 `F3`
- 若正文写完后，把本书里最顶事的东西换成参考书那套，整章居然还没怎么散，直接按“同构风险”回退到 `参考输入卡`
- 若正文最顶事的只是一句像参考书的狠话，而不是真有现实后果，直接按 `F2`

写前看准：
- 别把现在读到的一层强信号，直接拔高成“整本参考书从头到尾就是这么推的”。
- 只要参考源属于 `本地TXT情节库 / 提取资料库 / 连续剧情库 / 场景母模板库` 这类分层产物，就先分清：
  - `只在局部读到的东西`
  - `这一阶段反复出现的东西`
  - `前后多段摆在一起也还顺的东西`
- 若当前只读到局部章节、局部条目、局部母模板，大多只能写成 `这一小段能借的往前推法`，别直接说成“这本书就是这么写 / 主情节就是这个 / 后面就按这路子开书”。
- 参考资料如果已经明确分层覆盖范围，比如 `1-120`、`1-240`、`361-520`，那所有判断都得绑住覆盖范围；别抹平成“整本书就是这样”。
- 若同一资料库内同时存在 `经营脑 / 治理脑 / 组织脑 / 宣传脑 / 平台脑 / 围猎反勒索 / 战场收编 / 试炼翻灾祸` 等多类推进办法，冷启动前先分清“这次到底借哪一层、禁哪几层”，别因为某一层最容易起稿，就写成整本书都得这么走。

允许借：
- 节奏走法
- 情节怎么往前走
- 收益链顺序
- 连续翻面链结构
- 章组推进逻辑
- 句式功能
- 对话里怎么把场面拿下来
- 章尾停拍方式
- 压迫感组织方式
- 金句的触发时机与功能类型

允许强改编：
- 保留原参考文本里已经试出来确实有效的`剧情怎么往前走 / 收益结构 / 爽点顺序 / 翻盘先后顺序`
- 但得重建成当前项目自己的`世界底盘 / 人物关系 / 利益系统 / 资源靠什么落地 / 冲突靠什么打起来 / 场面实现`
- 只要正文已经把上面这些重建出来了，就算和参考文本共享较强的情节主线，也该看作`强改编重建`，别误判成“这也不算数”或“这不就是只换了层皮”

禁止借：
- 原书最有记忆点的常用话
- 一眼能看出来源的语录句心
- 高辨识度整段语气
- 专属梗、专属典故、专属比喻系统
- 不改人物关系、不改利益系统、不改场面实现，只换名词的整段复写
- 原场景推进、原冲突靠什么打起来、原翻盘动作基本不动，只把题材词替掉的机械改写

照下面做：
- 写前闸门里写明：这章借哪一路写法、怎么把事往前推、走哪条收益链，不借哪层表皮
- 如果走强改编，就写明：这章沿了参考文本哪种往前推法，又在你这本书里改成了什么新利益链和新场面
- 如果这次判断只来自局部样本，就明写成：
  - `当前依据的覆盖范围`
  - `当前只代表哪一阶段 / 哪一层资料`
  - `当前不代表整本书的哪些部分`
- 跨层要是还没对完，就别把只在局部读到的东西直接写进：
  - `设定/选题决策.md` 里那条已经定好的主方向
  - `大纲/开书总蓝图.md` 里那条已经定好的主线走法
  - `大纲/开书候选分叉.md` 之外那些已经定好的限制
- 正文如果读起来像“在模仿某本书说话”，按 `F1/F3` 处理，别直接写成“先搁这儿不动”。
- 参考文风如果又把正文带回解释腔、空话腔、培训稿腔，就先退回动作拍清单重做
- 若用户要求“参考情节链”，正确理解只能是参考 `章组任务 / 爽点顺序 / 收益链 / 翻盘节奏 / 场景母模板`，别直接保留原作完整事件顺序再只换世界词壳。
- 允许参考的情节链层级：
  - `破败组织强开增长`
  - `追账夺先手`
  - `进城受阻后引高层下注`
  - `被围后反做谈判`
  - `资源项目化后引爆生死筛选`
- 不允许参考的情节链层级：
  - 关键事件一一对应
  - 主要反转点一一对应
  - 具体冲突场、具体物证、具体翻盘动作基本不变
  - 只换组织名、地名、职业名、世界名词
- 若当前方案保留了原作完整事件主线，只把世界壳或人名替掉，就按 `F3` 退回章组层重做。
- 若当前方案只是复用了局部样本里最显眼的一层，例如 `分配权 / 换话路 / 经营脑首章`，却还没证明它真是这轮最适合拿来当主线的层，就按“取样偏了”处理，别直接把它定成开书主走法；先回到 `开书候选分叉 + 开书排异表` 重做。

### 本书金句怎么长

写的时候可以主动造金句，也最好主动造，但先守住这几条：
- 金句得给眼前这场冲突办事，别悬空耍帅
- 金句起码得让现场变了、让人往前往后动了、让原来的话变了、让关系变了中的一项落下来
- 金句要长在本书自己的东西上：职业、规矩、物证、地名、兵器、账目、火候、口令等
- 金句最好从主角、关键对手、摇摆角色口中长出来
- 同一角色说出来的话，味得差不多，别谁开口都像一个人

禁止把以下内容当金句：
- 只靠抽象狠词硬撑的空狠话
- 万能鸡汤句
- 离开本书世界，塞进任何书里都不违和的空话
- 只是在改写参考书现成名句

写前先答这 3 个问题：
- 这章谁最有资格说出记忆点最高的话
- 这句话要钉死什么现实结果
- 这句话离开本书后还剩下多少辨识度

金句写得好不好，不看它像不像参考书，就看读者一眼能不能认出这是你这本书里的人在说话。

### 首稿后再查一遍作者腔

`章组收益卡 + 三张单章卡` 确实能把解释腔和作者腔压下去不少，但**别当它已经清干净了**。现在最容易漏的，常常不是大段说明，而是首稿里还剩的“旁白补缝句 / 意义翻译句 / 作者替读者把意思一下讲死的句子”。

做这一步前，先读 `references/anti-authorial-intrusion-method.md`，再去压首稿后的第一轮残留。

这一步照下面做：
- `正文生成时先避开`
- `首稿写完后强制复检残留`
- 两者缺一，这章首稿就还没成形

也就是说：
- 别把“反作者腔”只理解成写完以后找病
- 也别拿“我写的时候已经注意了”当免检理由
- 得同时做到“写的时候先躲开”和“写完以后再专门复检”

这条不只是拿来找问题，真撞上了还得这么改：
- 写前把“这里要表达什么”改成“谁做了什么导致什么变了”
- 高风险场景先补 `起手目标 / 首轮阻断 / 局面翻坏 / 被迫改手 / 现实战果`
- 首稿回压时按 `外部刺激 -> 人物反应 -> 再做动作` 的顺序检查，避免旁白抢跑
- 撞上作者腔时，就换成 `动作 / 物证 / 群体反应 / 谁往前谁往后变了 / 规矩变化` 这些现场东西，别只删句

首稿写完后，除 `scene_lint.py` 外，还得额外做一次人工 `作者腔残留复检`，专查下面这些句型：
- `那不是……那是……`
- `显然……`
- `这句不长 / 这一下子 / 这一刻……`
- `不是冲谁……是冲什么……`
- 场面里明明都已经有了，又多塞一句“这意味着什么 / 这说明什么 / 读者该怎么理解”

看这类句子时记住：
- 句子如果提供了正文还没交代的新事实，可以暂留
- 若句子只是在翻译读者本来已经能看懂的效果，按 `F1` 处理
- 若单章攒到 `2` 处及以上这种残留，别只写“文风还能再润”，要直接记成 `作者代判句残留`

真改的时候：
- 先替换成 `群体反应 / 物证细节 / 谁往前谁往后变了 / 动作后果 / 眼神去向`
- 别只删句，后面的现场动作和结果也得跟上

这条经验记下：
- 若本轮已读完四卡而 `本章短清单` 仍残留作者腔，说明问题不在“没细纲”，而在“首稿后没做旁白补缝句专项复检”
- 后面别再把“卡片读了 / 执行核写了”直接当成“作者腔已经清掉了”

### 本章短清单怎么写

长篇方法、题材说明、反作者腔办法、对标拆解结果，这些都算“背景知识”，**代替不了首稿执行协议**。

以后每章第一次落正文前，都得另外写一份 `本章短清单`，这就是这一章首稿时最先看的本地小协议。

这份短清单先守住下面这些：
- `本章短清单` 控制在 `300-600` 字或差不多密度的短表里
- 只留“这一章不写上就会写错”的东西，别把整套方法论整段搬进去
- 记下这些：
  - 本章现实改局
  - 本章章尾允许停在哪一类现场结果
  - 本章禁用句型
  - 本章世界信息靠什么东西落地
  - 本章对手反制口
  - 本章第三方补逻辑位
  - 本章最容易滑成解释腔的位置，以及准备用什么现场东西替掉
- `本章短清单` 没写，或者还像一大段讲解稿，就直接按 `F5`
- `本章短清单` 列完以后，正文首稿、首稿回压、写后检查都得明着对它，别只对长文档

记住：
- 长文档管“为什么”
- `本章短清单` 管“这一章现在怎么写”
- 若长文档与 `本章短清单` 冲突，以当前章写起来更顺手的 `本章短清单` 为准，但别违背必停层基础规则

补充：
- 负例和替代句式别只放在方法文档里，得沉到当前章 `本章短清单`
- 若这轮已经看出某类坏句老是复发，后面相关章节的 `本章短清单` 就得直接把它列成禁用项，别再写空提醒
- `本章短清单` 还得回答一条：`本章采用的动作-后果链是什么`
  - 格式可简写为：`起手动作 -> 第一被动改线 -> 第二被动改线 -> 现实战果`
  - 若只能写“主角查到什么 / 发现什么 / 知道什么”，没有写出谁因此被挤开、谁因此把话缩回去、谁因此改动作，就按 `F3`

## 必停层：基础规则

主文件这里只留最值得盯住的几条，细则分开放在下面这些文件里：

- 开始前看这几条硬要求、失败码、生产顺序、写前/写后检查：
  - `references/gates/core-gates.md`
- 细纲准入、本章主要往前推什么/副线/主爆点/第二轮收益：
  - `references/gates/outline-gates.md`
- 正文红线、解释腔、情报格式、字数预算、章尾否决：
  - `references/gates/draft-gates.md`
- 追踪文件写法、未来预案禁区、同步规则：
  - `references/gates/tracking-gates.md`
- 连续回炉、执行计划、版本切换计划：
  - `references/workflow-revision.md`
- skill / 流程改动后回头重看的记录：
  - `references/regression-handoff.md`
- 人物出场分量、配角工具嘴、主角声线、高价值角色：
  - `references/quality/character-rules.md`
- 爽点密度、高潮兑现、后半章信息场、章尾力度：
  - `references/quality/payoff-rules.md`

不管现在走到哪，都先看一遍 `references/gates/core-gates.md`，再按眼前这步去补读对应分文件。

下面这些一路都照着走：

1. 平常写作别跳步。
2. 失败码不是 `F0`，这章就先别收进主正文。
3. 追踪文件只能写正文里已经落地的事。
4. 路径根得是当前小说目录。
5. 连续回炉时，把检查单和执行计划先列出来。
6. 章尾这关要是没过，单独就能拦住整章。
7. 通常走 `单文件推进`：每个执行步一次只新增或改写 `1` 个项目文件，做完这个再自动进下一个，不用反复问“要不要继续”。只有用户明说 `一次写很多文件 / 放开单文件限制`，才临时改成一次多写几个。
8. 若用户明确说了 `慢 / 超时 / 卡住 / 还是慢 / 一个一个落 / 单文件推进`，就当他又点了一次单文件模式；同一批里别自己恢复成一次多写几个，除非用户后面改口。
9. `scene_lint.py` 走完以后，正文纯净度检查、写后检查、追踪/角色卡同步还是得继续做；别把脚本走完了就当这一章已经收工了。
10. 只要撞上解释句黑名单，或者事实层证据等级漂了，`交叉复查` 就别自己写“这段先不管，往后放放”。
11. 回炉章重写前先记一句上一轮卡在哪；首章回炉时，还得额外看一眼“书名另一半第一轮压制”和“高价值明敌第一次让人记住的点”。
12. 首章只做到“活下来 + 敌人来了 + 线索拿到了”还不算过；还得再有一次敌方被迫改动作 / 往前站或往后退 / 改主意的外部现实变化。
13. 功能场景要是写得像在把规矩往外念，就别急着判 `F0`；先退回写前闸门或写后检查重看，连续撞上就按 `F2`。
14. 回修前先分清这次到底是“补丁式抠句子”还是“一次性整段重写”；别一上来就一路小修到底。
15. 写后检查、交叉复查、追踪文件里，别把“下一章预计发生 / 可直接进入 / 已准备反制 / 即将兑现”硬写成“本章高潮后续收益 / 已做完 / 这一章到这就能留下了”；凡是拿未来预期冒充当前事实，直接按 `F5`。
16. 情报流 / 预知流 / 系统流作品，从第一条正式情报出现起，`追踪/情报记录.md` 就得在；没建、没回填 `刷新章 / 预计兑现章`，或者正文还没兑现就先写 `实际兑现章 / 实际战果`，都按 `F5`。
17. 首章得分开看“章尾是否合格”和“高潮兑现有没有落到位”；章尾停在现场，不等于首章已经可以接着往后写了。若首章只有合格章尾、没有现实收益，先按 `F2`。
18. `交叉复查` 别只因为“正文和细纲一致”就自己写“这回算过去了”；还得再核对 `追踪/时间线.md`、`追踪/角色状态.md`、`追踪/情报记录.md` 里是不是还留着旧写法、双版本或没落地的事实。
18.1. `把本章已经落地的事实先拎出来`：进 `写后检查 -> 追踪/角色卡同步 -> 交叉复查` 前，先把“本章正文已经落地的事实”整理成一份最小事实摘记，后面都按它回填；`角色状态.md`、`情报记录.md`、`时间线.md`、`设定/角色/*.md` 都只能从这份摘记回填，别直接抄细纲、脑补后续，或者按作者意图提前把话写高。只要有任一项目文件出现“正文没落地，但追踪先写成当前事实”的情况，直接按 `F5`。
18.2. `情报现在到底坐实到哪一档`：`追踪/情报记录.md` 里的情报内容就用四档写法：`线索`、`高度相关`、`半坐实`、`已坐实`。正文还没出现直接物证、动作坐实、或多方旁证前，别把 `线索/高度相关/半坐实` 直接拔高成 `已联手 / 已转走 / 已背叛 / 已主使 / 已控制` 这种一下把话讲死的话。项目里的模板如果还没分栏，也得在文字里明写当前证据等级；不然按 `F5`。
18.3. `角色卡什么时候得写`：任一具名角色只要满足下面任一条，进下一章前就得把 `设定/角色/{角色名}.md` 写到够用：连续两章承担关键现实动作；在公开冲突里改过局面；被写前闸门、写后检查、角色状态三处里的任两处点名。卡没写完前，别结束当前章组，也别自己写“追踪这边都补完了”。
18.4. `角色卡这条现在基本顺了`：从真实冷启动和旧样本里已经能看出来，这条现在不只是会“抓出缺卡”，把 `设定/角色/*.md` 写到够用后，常常也能把 `validate_tracking_state.py --json` 拉回 `ok=true`。所以后面就按这条来用：缺卡照样按 `F5`，但补卡后脚本既然已经恢复正常，就别再把同类问题反复写成“这事是不是还得再翻出来过一遍”。
18.5. `章尾最后那步怎么查`：这块现在已经有比较顺的写法了。后面查时，就盯住它是不是写成 `新命令 -> 立刻执行 -> 现场马上看见后果`。要是只写到 `新命令`，或者只写到 `新命令 + 立刻执行`，少了最后那步现场变化，就是没写够。以后若 `scene_lint.py` 还报 `章尾.error`，先去看这三拍是不是少了一拍，别上来就当成只是某一句手滑。
18.6. `高价值角色现在写到哪儿了`：现在这套 skill 基本能把主角、明敌、记录口、高价值角色先写到让人记住，但大多还停在“人出来了，后面那股劲还没跟上”。后面人物审查如果又看出这类问题，就直接写 `这人那股让人不敢小看的劲还没起来`，别先写成“这个人还是虚的”；现在更像是后面那股劲还没续上。
18.6.1. `高价值角色差的，常常不是卡，是首出场太薄`：现在短板主要不是“没有角色卡”，而是“角色卡写得厚，正文第一次出场却太薄”。常见情况是：角色卡里已经有 `公开目标 / 私下护什么 / 行事规矩 / 记得住的动作 / 说话记忆点 / 人一出来就让人不敢小看的样子`，但正文首出场常常只落下 `1-2` 个记忆点，再塞一句只负责推事的对白，结果人物只是刚露了个面，还不到“一出场就有劲”。后面再撞上这项时，就看“首出场有没有同时落出外形、动作、私欲、出场后当场带来的变化里 `3` 项以上”，别先去加厚角色卡。
18.6.2. `高价值角色首出场，别又全是一个腔`：高价值角色第一次出场，现在还有个老毛病，就是老重复 `既然先X / 就别先Y`、`乱不乱，等你认稳再说`、`先试完，再说谁该闭嘴` 这类一开口就想把全场按住的句子。事情能办到，但几个人会越写越像。后面如果角色已经有最起码能让人记住的东西，却还只是“人能认住了，可还不够有劲”，就写成 `首出场声线太像 + 私欲落得还不够`，别只是继续堆记忆点。
18.6.3. `这批东西已经真落到正文里了`：现在已经能看出来，把 `高价值角色首出场同时落 3 项以上`、`首出场别再用旧的同质压人句法`、`这次最撑人的私欲落点` 写进角色卡模板和首稿卡后，首写时是能落进正文的，不会只停在卡上。往直白了说，`高价值角色首出场更厚了`、`首出场声线没那么像了` 这两件事已经能反复写出来；但这个人能不能让读者继续记着、还想看他后面怎么走，现在还差最后那点火候。后面如果出现“人物比旧样本更厚，但还是没完全有劲”的情况，就写成“这些东西已经落进正文，但这人后面那股劲还没完全出来”，别再退回成“角色卡写了、正文没落出来”。
18.6.4. `高价值角色开口前，把自己要护的东西先写进去`：高价值角色想摆脱“出来只是替主角把场面托住”的工具写法，不能只替主角补公道、补规矩、补场面上的理，得先把自己最想护的人 / 物 / 名分写进去，再开口把话定下来。少了这一步，就算句子够狠、够稳，也还是像“专门出来替主角把场子兜住”。
18.6.4.1. `高价值角色别又说成同一张嘴`：还有个新风险，事情本身写对了，但高价值角色的台词开始反复套同一种路数，比如 `我家这张都先退后，你那套旧规还想压谁`、`我先摘我自家的，再摘你的`、`你丢的不是X，是今夜Y`。这类句子功能没错，却会把不同角色写成同一张嘴。后面如果高价值角色已经做到 `先付代价再定场`，却还连续冒出这种写法或换词版，就写成 `高价值角色声线太像 / 这人一开口就像每句话都提前打好了草稿`，别急着往“人物已经更有劲”上写。
18.6.5. `后面盯高价值角色，主要看这两件事`：后面如果继续盯这条线，就别老回头问“这些东西能不能进正文”了，这一步已经做到。接下来主要盯两件事：一，`高价值角色有没有先付自己的代价，再去改全场规矩`；二，`首出场已经更厚了，是不是还只是刚把这个人写出来，没真正让人记住、没让人惦记他后面还会干什么`。前者缺了，就写成 `又写回只会出来替主角把场子兜住`；前者有了、后者还没透，再写成 `这人后面那股劲还没跟上`。
18.6.5.1. `高价值角色这条，分三档看就够了`：这条拆成三档最好用，别再只分“像那么回事了 / 还差着”：第一档，`这条还没在正文里真正出来`，也就是正文连“先压自己人 / 自己物 / 自己名分”都没落下，或者写了这一手，但自己根本没真丢东西；第二档，`这条已经有了，但被作者补义拖弱了`，也就是正文已经形成“先付代价再定场”，但关键位后还连着补 `作者代判句 / 作者收义句 / 规则翻译句`，删掉后这条其实还撑得住；第三档，`这条已经能让人信了，而且作者补义已经明显少了`，也就是正文主体已经能靠动作、被挤开、把话收回去、执行、器物后果自己把意思带出来，只剩一点尾后收束句。后面就照这三档写，别再把第二档和第一档混着说。
18.6.6. `主角最容易写对的人物句，常常就是冷冷一句`：拿 `灾门点命簿`、`寒驿换命牌`、`雪关夺炭令`、`雾站夺氧签`、`晨会抢批条` 这些连续样本一对，可以再明确一条：主角人物句最容易先写对的，不是热闹损人句，而是冷冷一句把事情点破的话。先看这句是不是同时挂住了 `私利 / 代价 / 现实改局` 里的两项；如果没有，只是嘴硬、句狠、好听，那还是按“句子热闹，但还不够像这个主角会说”处理。
18.6.7. `主角得有自己那一步切进去的办法`：继续拿连续样本一对，还能再明确一条：主角那种冷冷一句想真正把场面拿下来，不只要狠，还得带一组只属于这个题材现场的具体东西，还得看得出他是先盯住哪块坏处，再从哪儿下手。若正文只剩“责任 / 账 / 规矩 / 后果”这类空词，没有稳定绑到现场具体东西上，就还是按“谁都能顺嘴来一句的空话”处理。
18.6.8. `主角也别老靠同一套旧句子往前推`：拿专项对比样本一对，现在还能再看出一条：主角人物句最后能不能写得有劲，不只看“有没有自己那套现场东西”和“作者判价句有没有压住”，还看他能不能换个说法往前走，而且换了以后，照样能稳定写出 `私利 / 代价 / 现实改局`。其中第二套句法已经证明“主角人物句离开老路数也能写稳”，第三套句法又进一步证明“离开老路数和第二套专项写法后，照样能写稳”，所以后面先按下面这套来盯：
- 若主角已具备 `自己那套现场东西 + 坏点识别顺序已经稳了 + 作者判价句黑名单已压住`，下一步就继续试 `别再走现在这套老写法`
- 若所谓“新写法”只是上一轮专项写法换几个名词重写，那就别急着往上记
- 若主角不用上一轮高频写法，照样还能长出有劲的句子，才可以继续往“换个写法也照样能写出来”那一档上推
18.6.8.1. `公开场争位时，别老拿同一句开场`：最新样本补扫后，还能再看出一个该拦就拦的毛病：在 `公开场争位 / 挂名 / 落签 / 排位 / 配给` 这类场景里，主角推进口如果连续复用 `那就先挂我名`、`那就先挂我名下`、`那就先挂我这边名下` 这种起手句，就算结构还说得过去，也记成 `开口老是同一句起手`。这类句子只能负责把场面接过去，别再拿来硬抬成主角有劲的句子或角色辨识句；后面要是撞上，就去查“有没有自己那套现场东西 + 坏点识别顺序 + 不再共用同一类起手句”这三项。
18.7. `解释腔`：盯着跨题材、首次硬收益后那一段去看，现在这套 skill 已经能把大段解释腔压下去，但前置自检卡还是会在连续章节的关键收益位回弹出少量作者代判句。以后若 `scene_lint.py` 报了 `解释.error`，就看 `第二轮收益 / 第二轮外部反应 / 章尾前 / 章尾最后一拍` 这四个位置还有没有“替读者翻译意思”的句子；在多题材连续章节还没彻底压干净前，就当它 `后面还留着一点尾巴`。
18.8. `模板到底有没有真进正文`：如果这轮就是看“模板、情节链、方法资料到底有没有真写进去”，那就盯这几件事：`模板有没有变成正文里的事`、`情节有没有真把现实局面改掉`、`收益停在哪一层和正文对不对得上`、`审查看不看得出模板漏用`。`字数.error / 厚度不足` 先记下，但先别抢这一轮的判断。只有用户明确就是要看厚度，或者问题已经缩到只剩厚度，才把厚度重新提到前面。
18.8.1. `跨题材样本这里再说一句`：现在已经能看出来，`第一爆点 / 第二轮收益 / 两轮外部反应 / 高价值角色切入` 这些东西，跨题材也能比较稳地落进正文事件链，而且确实能改掉现场那套话、谁往前谁往后、名单、门槛位、装车口、签字位这些东西。后面再出问题，先别急着说“模板没用上”，先往这三类里分：`收益停层写高了`、`情报流误触`、`第一次硬收益后的承接位被随手写短了`。
18.8.1.1. `第一次硬收益后，章尾单独看`：`scene_lint.py` 要是打到“第一次硬收益后的承接位 `章尾.error`”，别立刻升成 `F2`，也别顺手并到 `第一次硬收益后的承接位短兑现链`。更该这样看：如果这一拍最先要看的收益在章尾前已经落成，而且确实改掉了 `挂名 / 落签 / 外圈的人改了主意 / 第二列重算 / 谁来办这件事` 里的一项，末句只是收得偏顺，或者又补了一句盖章话，那就先记成 `第一次硬收益后的承接位最后半拍泄了劲`。这条已经差不多顺了，别再把具体样本名往主规则区里倒。
18.8.2. `收益和第一次硬收益后那一段，先这样看`：这里现在基本能看出来两件事：一，最近样本的收益层级大多守住了 `资格 -> 名分真正落到手里`，没把 `先挂 / 先试 / 先复核 / 待验` 提前写成 `好处已经开始真起作用`；二，后面更容易出的问题，不是收益写高，而是 `第一次硬收益后的承接位` 容易被随手写短。也就是说，模板方向大多没跑歪，承接位却容易被首写压成短兑现链。后面如果用户要看“模板到底用没用对”，先查 `情报流误触` 和 `承接位事件链缩短`，别老先怀疑模板根本没落下去。
18.8.3. `第一次硬收益后那一段，别随手写短`：这类问题常见的样子，不是模板没了，而是事件链被压成了“主角再顶一句 -> 对手回一句 -> 高价值角色补一压 -> 记录口直接落签”。后面如果人工或复查时专门看模板使用率，就盯一件事：这个承接位在第二轮收益前后，还有没有够数的现实改线、外部反应、第三方被动动作。要是没有，先写成 `这一段被写短了`，别误判成模板没用。
18.8.4. `看模板有没有用对时，这四件事回回都得看`：后面如果用户要专门看模板和情节到底用得对不对，每回都得把下面四件事过一遍：
- `收益有没有写高`：`资格 / 名分真正落到手里 / 好处已经开始真起作用` 有没有按正文里的硬后果来定层
- `是不是把非情报流东西误判成正式情报流`：像 `旧码 / 错单 / 复核口 / 广播补条 / 规则播报` 这类东西别乱算
- `第一次硬收益后的承接位是不是被写成了短兑现链`：这一段是不是只剩“主角顶一句、对手回一句、高价值角色补一压、记录口落签”
- `章组最后那次兑现里，对手到底有没有真丢东西`：别只写成“新规生效 / 新话挂起来了 / 我方名分坐实”，却没写出对手真丢了 `台位 / 先后次序 / 配额 / 先轮到谁办 / 暂时说了算的那一步` 里的一项
只要是在测模板，这四件事回回都看，别临时想起哪条就补哪条。
18.8.5. `碰到这种反面例子，先这样看`：碰到这种反面例子，先按这个顺序来。第一，脚本如果先打到 `情报.error`，可正文里真正起作用的是 `旧码 / 错单 / 广播 / 挂签 / 补条` 这一类东西，那就先当成 `可能把不算情报流的东西错打成情报流了`，别急着往正式情报流问题上靠。第二，前置章节已经能证明模板进正文了，可第一次硬收益后那一段却只剩“主角顶一句 -> 对手回一句 -> 高价值角色补一压 -> 记录口直接落签”，那就先当成 `这一段被写短了`，别一看见 `字数.error` 就又退回去说“模板没落地”或“这只是残稿”。
18.8.5.1. `章组最后那下，对手得真丢东西`：连续翻面链最后那一步，最容易写高的地方，就是“纸面上已经生效了，可对手现实里还没真丢东西”。常见样子是：我方已经挂名、占理、起新簿、拿到名分，但对手还没被赶下台、没被改顺序、没被削掉原先的先手、没被公开挂责，也没被逼着把那个“先轮到谁办”的次序交出来。后面如果用户主要就是看模板效果，那就盯住最后这一步：有没有写出 `1` 个对手现实里真丢掉的东西；要是没有，就先按 `高潮后续收益不足 / 章组兑现只落一半` 去看，别拿“新规则今夜生效”这类话硬顶过去。
18.8.6. `样本起码得把结果写出来`：还有一条边界得先记住：`这些判断能帮你把方向掰回来，但替不了正文最起码得把结果写出来这条线`。如果专项样本薄到连这条线都撑不住，脚本和人工注意力还是会先被 `字数.error`、`章尾.error`，或者少量 `解释.error` 拽走。后面如果人工要看 `非情报流误触 / 第一次硬收益后的承接位短兑现链` 能不能看出来，先得保证：
 - `非情报流误触` 样本：正文除主要起作用的那样东西外，还需有最小量的第三方被动动作、谁往前谁往后变了、器物反馈，避免只剩概念性空框；
- `首次硬收益后的承接位短兑现链` 样本：前置章节得足够证明模板已经进正文，该承接位可以短，但不能短到只剩残稿感；
- 若这条最低场面线还撑不住，人工还是可以继续往下看这一项，但得写明：`这份样本还在受厚度和章尾噪音干扰，现在还不到把结果一口咬死的时候`。
19. 开书和前 `10` 章细纲里，别老写“某个人 / 某件东西 / 某个势力 / 一份物证 / 新的交接点 / 更高层的人”这种空占位；最先该落名字的事件、对手、资源、物证都得落名字。
20. 如果书里有明确的大设定 / 金手指 / 残魂 / 情报能力，前 `10` 章里要有 `3` 次让它直接改出现实结果；只会帮忙提醒、识别、出主意，不算真正写到位。
21. 规模等级不是随手贴的标签；凡标记 `L3/L4` 却写不出群体规模、秩序变化、路线变化、场面最后到底是怎么收下来的，就当细纲这一关还没想实。
22. 前 `10` 章得钉住“三个记忆点”：`1` 个具名明敌、`1` 个具名摇摆角色、`1` 个具名资源 / 权力节点；缺任一项就先别进正文。
23. 世界底盘别只是背景分类，得能自己制造大事；若“世界往前怎么推 / 灾异系统 / 大事件池”写完后仍像通用题材清单，不像本书专属爆事机器，按 `F3` 处理。
24. 后面那轮集中再看，不是想起了才做的一项，它本来就在阶段流程里：什么时候触发别把章节号提前卡死，要落在项目 `追踪/阶段审查计划.md` 里，按“这一段写到哪了 / 这一段翻面了 / 这一段撞风险了”来定；连续章节回炉到了升级阈值，或者事实层 `F5` 修完后，这一步还是得补上。连续章节阈值见 `references/continuous-chapter-thresholds.md`。
25. 这轮工作如果动到了 skill 或流程规则，别只嘴上来一句“建议测一下”；去看 `references/regression-handoff.md`，判断要不要跑项目根 `scripts/story_review_regression.py`，有脚本就先跑一遍，再决定要不要补人工抽样。
26. 正文别靠单一力度副词硬顶场面；其中 `references/banned-words.md` 的 `一级禁用词` 属于首稿禁止生成项，正文起草时就别出现，也别拿生硬撑劲词兜动作。正文如果还出现一级禁用词，直接按 `F1` 回修；生产模式自动切进 `H1-B`，纯基准测试自动切进 `H1-A`，只记失败不修正文。`二级敏感词` 如果在单章里高频重复，或者明显拿来代替具体动作、受力点、位移结果，也按 `F1` 回修。
27. 正文里别临时往里塞还没记过的重要信息；凡是新增了 `具名人物 / 具名势力 / 具名地点 / 重要物证 / 规则变化 / 关键时间钉 / 接头点 / 关系改判 / 大设定新能力` 里的任一项，要是没先回填到细纲和写前闸门的 `新增信息允许清单`，就直接按 `F5`。
28. 正文如果老是在“临时加东西”，别只在正文层擦补，先当成纲层没把该写的事提前压实；回头重看 `卷纲 / 细纲 / 写前闸门` 里是不是还有空占位、收益发虚、事情最后到底怎么收下来还没想实、过桥缺失，必要时按 `F3` 退回章节组细纲。
29. 男频长篇要是没有明确反例，主角上限就得设大。开书阶段要先把主角终局身份、终局权力、终局资源盘、终局规则位定出来；若终局还只停在 `局部能人 / 地头蛇 / 边关小盘掌控者` 这一档，就当封顶偏低，先别进连续正文。
29. 世界信息别只堆在首章或蓝图里。蓝图、卷纲/章组、全书章节总表、单章细纲、正文都得一层一层落下去：蓝图管世界总控与投放分期，章组管阶段世界任务，章节总表管每章世界信息点，细纲管本章世界任务/靠什么落地/允许新增清单，正文只负责把这些信息点通过冲突与场面落下来。少一层，就按 `F3/F5` 处理。
28. 首章负责先把世界底色带出来，不负责一次全讲完；但后面也不能只顾推剧情、不继续把世界撑开。若连续 `2-3` 章没有新增或推进任何有效世界信息点，按整体写法跑偏处理，优先回退到章节总表 / 章组控制重审。
29. 连续正文就用“`章组收益卡 + 动作拍清单 + 人物冲突卡 + 章尾现场句预案 + 本章短清单`”这套进正文。当前章节如果到了连续章组卡触发阈值，就把对应 `章组收益卡` 产出来，把这组章结束后到底是谁被迫 `把原话咽回去 / 往前站或往后退 / 手里的活被改了 / 被挤开 / 没法先开口说了算 / 失去资源` 写到具体人头上。要是只有“查到什么 / 发现什么 / 翻出什么”，写不出“改掉什么局面”，直接按 `F3/F5` 停写。连续章组阈值见 `references/continuous-chapter-thresholds.md`。
30. 落正文前，把 `章组收益卡 / 动作拍清单 / 人物冲突卡 / 章尾现场句预案` 这四样东西摆出来；少一样，或者写得太虚、只会念规矩、满嘴章尾判词、像搜证流水账，就直接按 `F5` 停写。别只说“脑子里已经有数了”。
31. 每章**第一次写正文首稿**前，都得明着读取 `章组收益卡 / 动作拍清单 / 人物冲突卡 / 章尾现场句预案` 这四张卡，并在写前闸门或执行记录里留下“已读取”的痕迹；没读、没引用，或者只是文件摆在那儿但正文没按卡走，都直接按 `F5`。这四张卡不是拿来存档看的，它们就是首稿输入源；别跳过这一步就直接落正文。
31.1. `每章任务先写明白`：进入正文前，额外写明 `本章一定要做出来的现实变化`，就写成“谁把什么从什么状态改成什么状态”。若写前记录里只有线索、怀疑、计划，没有现实变化目标，直接按 `F3` 停写。
31.2. `章组前面几拍也得各自见到好处`：若当前是连续多章章组，所有前置承接位都得各自写明一项章内可见收益，而且收益不能都是“多知道一层”。允许的低档收益包括：压下一件货、逼出一张纸、拖出一个被逼着选边的人、改掉一处假火、迫使一个执行次序改手。若大收益全堆到末次兑现位，就按 `F3` 处理。
31.3. `配角别只剩工具活`：凡本章承担关键推进的配角，写前卡里得写明 `岗位 / 风险 / 私利或执念 / 本章会主动做的动作` 四项。缺任一项，就先当这个配角还像工具人；连续两章关键配角都只剩送证据、跑腿、背书、翻译功能，按 `F3` 处理。
31.4. `对手得当场压回来`：公开场或正面对撞场里，对手得有 `1` 个当场压回来的动作，不能只会说“你越权 / 你别闹 / 按规矩”。当场压回来可以是：记名、换岗、逼人闭嘴、拖账、扣件、罚巡、借上级的话反打。若对手连续两场都只剩嘴上压人，按人物写坏了处理。
31.5. `第三方得把场面接住`：凡是公开翻旧话、当众对撞、值房 / 大堂 / 军营门前这类场景，场里得有 `1` 个第三方出来把场面接住，别让冲突只剩两边硬吵。第三方可以验物证、卡箱、补见闻、往旁边挤一步、逼书吏落笔。若全场第三方只会围观和发愣，就当这场面还没写起来。
31.6. `模板别只借个壳`：若本章明确调用了事件模板 / 往前推的模板 / 情节库模板，进正文前先把这些事想明白：`这个模板到底改掉什么现实 / 主角先借它拿到什么 / 对手怎么当场压回来 / 第三方把哪句话吞回去，或往哪边退了一步 / 主角付了什么代价 / 它又带出了谁 / 它给下一章带出什么新模板 / 本章最后停在哪个动作上最能让读者想接着看`。只要有一项说不实，就先当细纲这一关还没弄实；若正文还硬写，先按 `F2/F3` 处理。
31.7. `模板四层都得落出来`：每个主模板都得同时落出 `事件层 / 人物层 / 代价层 / 章尾层` 四层；若只落事件层，就算模板只是挂了个名。规则向、经营向、连续翻盘这几类文都要同时读取 `references/template-engine-exhaustion.md` 与 `references/template-engine-combos.md`。
31.7.1. `模板味太显了就先拦住`：若正文或首稿本章短清单里答不清 `本章最容易写成提纲腔的句子 / 本章最容易过早把好处写得太大的一层 / 本章最容易被一笔带过的过程拍` 三项，先别进首稿。若正文已经出现 `说明黑话回流 / 收益停层写高了 / 对手只回一句嘴、第三方立刻被逼着选边` 中任两项，就按 `F2` 处理；若写后检查和追踪还把它写成“这一段已经能看了”，追加 `F5`。
31.7.2. `对手得接着再压一步`：调用主模板时，只写出“对手先压回来一次”还不够，还得补一条 `对手第二步怎么继续压回来`，以及一条 `第三方这时不表态会丢什么`。若对手只有张嘴回压、第三方只有立刻把原话收回去，就先记成“写太快还没修好”，别因为“节奏利索”就直接当成没事。
31.7.3. `章组最后那下，对手得真掉东西`：若当前章承担连续翻面链的末次兑现位，细纲和首稿执行核得额外答上 `对手本章具体失去什么`，且答案得落在 `台位 / 先后次序 / 配额 / 先轮到谁办 / 暂时说了算的那一步 / 物资分配权 / 公开责任位` 之一；若只能答成 `我方新规成立 / 名分坐实 / 今夜起按新话算`，就先当末次兑现位这关还没成形，别进首稿。若正文已写完仍无敌方现实里真丢东西的点，先按 `F2`；若写后检查继续往上抬成 `好处已经开始真起作用` 或 `高潮已经全落到位`，追加 `F5`。
31.7.4. `高价值角色别再靠旁白抬`：高价值角色若承担章组关键定场口，首稿执行核得单独答清两件事：一，`她/他先付掉了哪一层自己的代价`；二，`删掉关键位后的作者代判句，这一段自己还能不能把意思带出来`。若第一项答不清，就先当这人还是“站在高处替人转话”的工具写法；若第二项答不清，就先当这段还在靠旁白补义。任一项没答稳，就别把该人物写进本章“高价值角色已经写透 / 这个人已经让人信了”的检查话里。
32. 四张卡的读取别只写“已读”。得留下最小映射记录，里面先带上：
  - `读取文件`：本章实际读取的四张卡文件名或路径
  - `采用条目`：每张卡本章实际采用的 `1-2` 条关键条目
  - `正文落点`：准备落在正文的哪一段 / 哪一拍 / 哪一场
  - `未采用原因`：若卡内关键条目本章不用，就简写一下原因，避免“读了却没真用上”
  - `本章要改掉什么现实结果`：别只写“这章查到了什么”
  上面任一项缺了，都算“这次没读实”，按 `F5`。这些内容平常就写进 `本章短清单` 或写前闸门的执行记录区，不再另外生首稿读取记录文件。
33. `章组收益卡` 平常只服务当前要写的连续章组，边写边往前滚；不要求把全书所有章组卡、所有单章三卡一次性全做完。要是一次把全书卡死，导致后面章节只能照表搬、没了现场性，就按用法错了处理；要是完全不做章组卡，结果连续章节只剩搜证链、拖延链、解释链，按 `F3`。连续章组适用范围见 `references/continuous-chapter-thresholds.md`。
34. 若冷启动样本连续出现“先念规矩、后补动作”“旁白替人物下判”“章尾靠总结句硬撑”“有证据却没翻盘”这四类问题，优先加硬到 `章组收益卡` 和写前闸门层，别只在红线表里继续加禁句。
35. 单章主事件别只写成“一个麻烦发生了”；细纲层先把 `主角本轮想拿什么 / 对手要卡什么 / 世界或第三方怎么把局面搅坏` 先落准。三者缺一，按 `F3`。
35.1. `动作-后果链先答清`：单章细纲与写前闸门都得能回答以下四问：
  - 本章谁先动手
  - 谁先吃到第一波损失，或先被挤开
  - 谁被迫改打法 / 把原话缩回去 / 往前站或往后退
  - 章尾留下了什么新的更大问题
  若四问中有两问答不出，而只能写成“气氛升级 / 线索更深 / 情况更复杂”，就按 `F3` 处理。
35.1.1. `关键场面先把这四件事落准`：凡本章存在关键硬场面，细纲、动作拍清单或写前闸门先把这四件事交代出来：
  - 这场里主角或关键人物当场想拿什么
  - 谁用什么现实手段拦
  - 原打法为什么被打断
  - 这一场结束后现场到底改了什么
  若只能写成“发生冲突 / 气氛更紧 / 对峙升级 / 双方撕破脸”，而答不出具体目标、阻碍、改手与变化，就按 `F3` 处理。
35.2. `现实卡法先落地`：只要某章最重的冲突要靠 `招标 / 挂名 / 编制 / 核签 / 矿权 / 配额 / 账册 / 军需 / 物流 / 规矩那套话 / 公开说了算 / 组织治理` 这类现实卡法把场面撑在那儿，细纲或写前闸门就先写出：这套东西现实里到底怎么卡人，这一场里又是谁拿它卡谁。若只剩题材名词和空规矩词，没有具体卡法，就先别进正文。
35.3. `别让剧情凭空冒出来`：若用户明确要求“别让模型自己编剧情”，那开书阶段得额外落一份 `设定/现实卡法灵感表.md` 或同义文件，先写上：
  - 本书前 `2-4` 个章组分别借了哪类现实卡法
  - 这些卡法各自负责制造什么冲突
  - 它们在本书里怎么改成新的利益链，场面最后又是怎么落到实处的
  缺该文件时，就别自己写“强改编重建这步已经齐了”。
36. 单章里得有 `2` 次连续翻面：`第一次阻断` 负责打断原计划，`第二次翻面` 负责逼主角改打法或改代价；若一路直推到底、没有中途改道，按 `F2` 处理。
37. `L2` 以上章节别只剩主角和单一对手对撞；还得有 `世界压力 / 群体动作 / 盟友私心 / 第三方插刀` 里的一股力量真参与进来，并真正改掉路线、谁往前谁往后、资源归属或口令归属。若只是背景热闹，按规模作弊处理。
38. 单章里得有一次明确的 `退路烧断 / 便宜解放弃 / 保一头丢一头`；主角若从头到尾只做“最省事的正确选择”，没有代价，就别判 `F0`。
39. 高情节章节的 `关键硬场面卡` 别只写“冲突升级”。要写成真能照着落的拍序，先带上：`起手目标`、`首轮阻断`、`局面翻坏`、`被迫改手`、`现实战果`。少一项，这份细纲就还卡着。
40. 关键角色别只做阵营工具人。连续章节里，主角、最值得盯住的对手、最值得盯住的盟友每 `1-2` 章都得各露一次 `私欲 / 偏见 / 自保习惯 / 旧账 / 贪心方向` 中的一项；若角色只会按主线需要说话做事，就当人物这层还没写出来处理。
41. 关键冲突段得有 `1` 轮真能把场面拿下来的对话：台词得真改掉在场动作、次序、谁往前谁往后、口令、资源归属里的一项；若对白只是交换信息、解释规矩、发表意见，就不算有效冲突。
42. 单章正文写完后，进下一章前先做一次全文残片扫描，并在写后检查或纯测试记录里把结果记下来；若扫到工具残片、元话语、补丁标记、执行日志、非故事语料，就直接按 `F4` 停在本章，别继续往下写。
43. 制度词、职业词、术语词别裸奔顶理解。第一次出现时，得立刻绑到“卡谁 / 放谁 / 害谁 / 让谁先走”的现实后果或白话解释上；若术语本身需要读者停下来翻一遍才懂，那就还是没讲顺。
44. 关键对白还得额外做一遍“张嘴测试”：念出来如果像公告、汇报、只是在念规矩，或者像作者借嘴解释，不像人在急场里抢话，那对白层就还卡着；别只靠换近义词硬顶过去。
45. 纲层说明黑话别直接进正文。凡是这类“抽象收益总括词 / 分层口令词 / 抢先后次序词 / 谁说了算的流程词 / 检查单摘要词”，只允许存在于 `卷纲 / 章组卡 / 细纲 / 闸门 / 检查单 / 追踪复盘`，别直接落入人物叙述、旁白叙述、章尾压场句或金句。正文一旦撞上，就按 `F1` 处理。
45.0.1. `关键台词先翻成人话再进正文`：凡某句在写前说明里承担的是 `接锅 / 抢先办事的这一步 / 立军令状 / 逼人把话收回去 / 宣告改手 / 划清谁往哪边站 / 抢先手` 这类作用，进正文前先翻成“这个人这会儿真会怎么说”。要是读起来更像细纲里随手记下的一句、像在交代流程，或者像作者在解释这句话到底管什么用，不像现场人物会脱口而出，就按 `F1` 处理。常见毛病有：
- 用抽象总括替代现场表达，例如 `哪里缺我补哪里`、`先把先手抢过来`、`我先把这一层局面改掉`
- 把多个位置 / 多个问题并列着往外摆，却没有人物当下语气、对象、威胁或代价
- 句子删掉功能标签后，人物声线、场合压力、对手对象都站不住
改的时候就按这个顺序来：
- 先弄准这句话到底要办成什么事
- 再改写成带对象、带场合、带代价、带动作后果的话
- 优先让台词落在 `你来拿我 / 先记我 / 先冲我 / 先把我砍了 / 先看今夜谁先断` 这类现场可执行表达上，而不是落在空话总结上
45.0.2. `对白别写成提纲句`：人物对白不能替作者把整串事情一口气讲完。要是一句台词里又在讲现在怎么样、又在讲接下来怎么办、又在点问题、又想把结果一下讲死，那这句就更像细纲里随手记的几点，不像人物说话。尤其在公开对撞、挂名、点验、核签、卡口、仓场这些场景，台词得先像人在顶话、抢话、甩锅、压人、接锅，别像作者在替这一章把所有意思都讲完。
45.0.3. `关键台词前后得有动作跟着`：承担 `接锅 / 抢执行次序 / 逼人把话收回去 / 当场换手 / 抢先手` 功能的关键台词，前后得有一项看得见的动作跟上：
- 打断
- 指人 / 指物 / 拍案 / 抢纸 / 拦手 / 抢令
- 对方立刻失态、退让、把原话收回去、停手
- 在场第三方因此往前站、往后退，或者把办事顺序改了
若台词前后没有任何动作跟着，而只是两个人站着把道理讲完，就按“又写回细纲话了”从严处理，先按 `F1`，重则 `F2`。
45.0.3.1. `角色口条别飘成文青腔`：老角色 / 病角色 / 守旧角色 / 稳场角色，在高压场面中的关键台词，平时就别写成排比式抒情句、整齐对仗句、作者代写金句。若一句台词主要记忆点来自“修辞好看”，而不是“这个人此刻真会这么顶回来”，就按 `口语失真 / F1` 处理。尤其撞上以下形态时，要强制人工复看：
- `带着什么去？带着……还是带着……`
- `不是……，是……` 且同时承担人物亮相或局势盖章
- 连续两拍以上的对称排比、岗位列表、悲凉自况
- 明显更像作者判词，不像人物脱口而出
45.0.3.2. `人物台词删修辞复核`：若一句关键台词把排比、对仗、熟狠话去掉后，场面作用和人物怕处还在，那就先删修辞；别因为“句子够响”就先留下不管。首章撞上这类句式，别拖到审查，首稿里或写后检查前就地改。
45.0.3.3. `高危句型黑名单联动`：凡正文撞上 `references/high-risk-sentence-patterns.md` 中的 `作者盖章型 / 两拍判词句 / 排比抒情台词 / 旁白替人物补义`，别只记“语言可再润”。当场分清它属于 `人物失真 / 作者补义 / 章尾盖章 / 解释腔` 里的哪一类，再按对应修法回修。
45.0.4. `叙述层也别倒回说明黑话`：不只人物对白，正文叙述也不能把写前说明里的黑话原样倒回来。凡叙述只是在笼统带一句，没把现场谁堵着谁、谁占着哪块地方、谁被挤到外圈写出来，读着更像细纲里随手记一笔、顺嘴把事情带过去，就按 `F1` 处理。尤其在卡口、值房、账房、军营门前、挂名、落签、配给、争先场景，叙述应直接写成：
- 灯架底下那块地方还没让出来
- 站着的人没挪
- 班牌还挂在谁腰上
- 谁把地方占着不放
- 谁先被赶到外圈
若删掉该说明词后，句子只能说明“这句想起什么作用”，却讲不清“现场到底谁没动、谁没让、谁被挤开”，那就当成说明词回流。
45.0.5. `岗位功能词也得落回人话`：即便不是明显说明黑话，正文也别过度依赖这类“岗位功能词”直接承担人物说话或叙述主功能。问题不在于这些词不能出现，而在于一旦把它们写成句子重心，人物就会更像在报岗、报表、报功能，而不像在急场里说人话。改的时候顺着这个来：
- 先写人和地方：谁堵着灯架、谁不挪、谁被摘牌、谁被赶到外圈
- 再让读者顺便知道这人原来是干什么的
- 若一句话里既能删掉岗位功能词、句意还更顺，就优先删回人和地方
常见毛病：
- `灯架底下那块地方还让不出来`
- `你先往外挪`
- `今夜先让他上`
- `先让他来定谁上谁退`
改的时候优先往这边靠：
- 不说 `灯位还让不出来`，改成 `灯架底下那块地方还让不出来`
- 不说 `那个位置先让出来`，改成 `你先给我往外挪`
- 不说 `先后次序先归他`，改成 `今夜先让他上`
- 不说 `先办事的人换成他`，改成 `今夜谁上、谁退，先听他的`
45.0.6. `岗位列表式台词先拦住`：人物对白别把岗位、工种、流程职责写成列表句直接报出来，例如 `谁替灯，谁扶绳，谁押路`、`谁站门，谁记名，谁押后`、`谁管灯，谁管绳，谁管路`。这类句子的问题不在“读不懂”，而在：
- 更像作者在给读者介绍岗位分工
- 更像在列职责清单
- 不像人在急场里先护地方、护脸、护自己那口活
改的时候优先往这些地方落：
- 灯还在不在我手里
- 人还站不站在这儿
- 谁敢来摸
- 谁敢把我挤出去
若一句台词删掉岗位列表后，人物的怒气、护短、占地、怕被挤开才开始像回事，那就把它当成岗位列表式台词，先按 `F1`
45.1. `模板术语回流补充`：除上面那些策划黑话外，正文也别直接写 `谁改线了 / 谁改局了 / 这就是爆点 / 这就是收益 / 这意味着他压不住了 / 现在不是……是……` 这种“替模板盖章”的话。删掉这句后事情如果还撑得起来，就按 `F1`；如果它正压在第二轮收益、第二轮外部反应或章尾最后一拍，就直接按更重的问题算，升 `F2`。
45.2. `收益别往高了写`：正文、写后检查、章组总表、角色状态、角色卡事实层，这几处给收益定层，都得按 `名分真正落到手里 -> 现场暂时先听这边的 -> 好处已经开始真起作用 -> 好处已经真的落到手里` 往上走。正文如果只写到 `封存 / 挂名 / 临时控制 / 待补发 / 待兑现`，那项目里的任何文件都别提前吹成 `好处已经开始真起作用 / 先办事的机会到手 / 现场调人调货先听主角的`；撞上就记 `F5`。
45.3. `模板是进去了，但过程可能还是窄`：写后检查、章组总表、角色状态或复盘文件里，如果已经写了“模板已进正文 / 模板已经写开 / 推剧情 / 显人物 / 改局面 / 开后手”这种话，那下面三件事也得一起落下：
  - 本章到底改掉了什么现实局面
  - 过程主要靠动作改局，还是主要靠台前把话说死
  - 哪个角色还像只剩一层工具作用
  后两项答不出来，就别急着写“已经写开了”，直接改成“模板已经落地，但过程写窄了”
45.3.1. `中段老在认物认账，就多看一眼`：中段如果连续两拍以上，主要都在靠 `认物 / 认味 / 认账 / 认条 / 认封绳 / 认绑法 / 认签发 / 验封` 往前推，写后检查里就多答这三句：
  - 这几拍分别改掉了谁的现实动作
  - 谁因此当场被挤开
  - 谁因此抢到第二手位
  答不出来，就别往上写成“书名主打那件事已经在正文里看见了”，直接写“查证链把书名主打那件事吃掉了”
45.3.2. `查证链太长时，就这样卡`：单章中段如果连着两场以上还在靠 `翻簿 / 对码 / 核账 / 验章 / 开箱 / 认物` 往前推，那写前闸门和写后检查里都得多答这三句：
  - 本章第一处“查证后立刻发生的现实后果”是什么
  - 本章第二处“查证后立刻改掉的现场动作”是什么
  - 若删掉最后一次查证，本章是否仍保有一次现实收益
  其中任一项答不出，就当成“查证链过长，现实后果链不够”，先按 `F2`；别再拿“证据更扎实”“逻辑更完整”就说这段其实不用改了。
45.3.3. `怕过程写窄，写前先把这三句答出来`：本章书名主打的事如果属于 `翻册 / 查账 / 验牌 / 抢先办事的机会 / 旧规改手 / 公开说了算`，那 `本章短清单` 和 `写前闸门` 里要提前把下面三问答出来：
  - 本章第一处“流程动作之后立刻改掉的现场动作”是什么
  - 本章第二处“不是台词、而是人或物被迫改手”的现实后果是什么
  - 本章哪一拍若只剩“翻册 / 认账 / 把话说死”，就得补一刀即时损失或即时得利
  三问里只要有一问答空，就先别进正文；正文如果还是落到了纸面上，就按“模板已落地，但过程写窄”看。
45.4. `挂名以后，下一步得跟上`：本章章尾收益如果落在 `正式挂名 / 正式准入 / 那套话被记进册子 / 记录权到手 / 新簿记名` 这类名分升级，下一章写前闸门里还得多问几句：
  - 第一张单子改谁
  - 谁当场被挤开
  - 谁当场得利
  - 这层名分具体先改掉谁的动作
  其中任一项说不实，就先别进正文；这时更像是“纸面上拿到了，但实利还没跟上”，先记 `F2`
45.4.1. `拿到新活以后，第一步得落地`：本章章尾虽然还没到 `正式挂名 / 正式准入`，但如果已经明确拿到一项新差事 / 新留补机会 / 新改报名额 / 新补额机会 / 新分签机会，下一章写前闸门里也得把这几句说实：
  - 这个新活第一张先改谁
  - 谁先被挤开
  - 谁先得利
  - 这项新活先逼谁改动作
  其中任一项说不实，就先别进正文；先按“这项新活拿到了，但第一步还没兑现”看，先记 `F2`
45.5. `软肋角色别一丢就没影`：某角色如果在角色卡、细纲或章组文件里被定成 `第一软肋 / 生存线记忆点 / 主角私人翻脸理由 / 私人救命压力`，那连续两章里起码得再落回来一次：
  - 身体状态
  - 实际处境
  - 私人关系压力
  - 不想拖累主角的反作用
  角色如果只剩名单结果、号码牌结果、资格结果、被救对象结果，而上面这些回针一个都没落下来，就记成“角色只剩一层工具作用”，先按 `F2`
45.6. `追踪别写太满`：写后检查、章组总表、角色状态如果正文只坐实到 `被迫改手 / 被迫改报 / 暂时照办 / 先照此执行 / 暂留待核`，那就别往上抬成：
  - `加入主角阵营`
  - `加入主角临时秩序`
  - `已经彻底跟主角站到一边`
  - `稳定偏向主角`
- `以后都按主角这边的话办`
  撞上就按“轻度看高了 / 归属写得太满”看；如果多文件一起撞上，再追加 `F5`
46. 正文如果想写“主角还要往下追第二层问题 / 新阶段问题 / 更深一层利益”，就改写成 `具体人 / 具体物 / 具体地点 / 具体规矩 / 具体缺口`，别拿抽象黑话过桥。比如别写“接下来这一步更值钱”“先把这一层局面改掉”“第一层现实变化”，改成“去军械库”“盯住那张补单”“先把那个人看住”“先改哪班轮值”。
47. `章尾这些收束句先别写`：章尾最后 `1-3` 句，别拿来替读者总结意义。凡是出现“这一夜……”“这就意味着……”“真正值钱的是……”“不是……是……”“接下来只要……”这类收束翻译句，就按 `F1` 看；如果整章本来兑现就不够，只能靠这类句子硬撑章尾，直接升 `F2`。
48. `剧情灵感先按这个顺序找`：开书或重做章组时，就照下面这个顺序找料，别一上来就让模型自己乱编：
  1. 项目内已有情节库 / 连续剧情库 / 场景母模板库
  2. 项目内 `设定/现实卡法灵感表.md`
  3. 世界总控表、爽点怎么落、章组收益卡
  4. 现实卡法类素材（行业流程、组织治理、资源争夺、制度卡法）
  5. 最后才是自由补桥
  前四层还没走完，就直接大段自由发明，就按“剧情来源失控”看，先按 `F3`。
48.1. `先把这次到底读了哪一段记清`：只要第 `1` 层来源是 `情节库 / 连续剧情库 / 场景母模板库 / 本地TXT提取库`，就把下面这些先记清：
  - 实际读取了哪些文件
  - 各文件覆盖了原书哪一段
  - 本轮实际采用的是 `文风层 / 情节层 / 语言层 / 约束层` 中哪几层
- 这次最多只到 `只在局部读到 / 这一段反复出现 / 前后多段摆一起也还连得上` 的哪一档
  上面任一项缺了，就别把该来源写成“这轮主路已经钉死了”，就按 `F5`。
48.2. `局部样本别一下子当成整本代表`：某条开书判断如果只能回指到 `1` 个局部连续剧情条、`1` 个场景母模板，或者 `1` 个阶段样本段，还证明不了它跨层、跨阶段反复都能这么看，那就只许先写成 `开书分叉里可用的一路素材`，别直接抬成那份总蓝图里唯一该走的主路。撞上就按 `F3` 回退到开书分叉。
48.3. `资料库里有几路推进架子，就分开看`：同一资料库里如果已经明摆着有 `2` 类及以上不同推进架子，开书时就先拉出不少于 `3` 个分叉，而且每个分叉起码调用不同推进架子层；如果这 `3` 个分叉只是同一母模板换壳，那就还是算无效分叉，别进正文。

另外，同一章下面这些文件只留一个现在还在用的版本：
- 写前闸门
- 写后检查
- 正文
- 本章相关追踪条目

这些文件如果对同一章分别写了不同对手、不同物证、不同爆点、不同章尾收益，就直接按 `F5`，并记成 `不可继续顺写`。
若 `设定/角色/` 目录缺失关键角色卡，或角色卡与正文写成了两套话，也按 `F5` 处理。
若事实层文件把“线索 / 怀疑 / 指向 / 死前吐词”直接写成“已经坐实 / 已主使 / 已执行 / 活人供词 / 第一批物证”，也按 `F5` 处理。
若 `交叉复查` 在证据等级未对齐时还自己写“这段先当没事吧”，也按 `F5` 处理。

### 回修怎么分

- 只有问题真是“单点、单层、单因”时，才允许用“补丁式抠句子”：
  - 只落在单个句子或单个短段
  - 失败码主要是 `F1`
  - 不牵动收益落点、段落主拍、人物谁往前谁往后、追踪里写的话
- 只要撞上下面任一项，就直接切到“一次性整段重写”：
  - 同一章连续 2 轮复审后仍保留相同主失败码
  - 同一问题同时跨“句子层 + 章节层”两层
  - 问题同时涉及“查证链/段落主拍/收益落点/人物压场”中的 2 项及以上
- 已经出现“局部越改越碎、复审意见越说越细但还是过不去”
- 进了“一次性整段重写”后：
  - 先重写问题段主拍，再统一回填检查单与修改清单
  - 别继续按句子补丁式来回抠
- 复审若还是过不去，优先继续整段重写，别又退回零碎小修
- 同一章节组如果已经连续 `2` 轮以上复审，而且问题从“大段不行”缩到“只剩少数硬点”，下一轮回修就先切到“这一版先改什么”这一套：
  - 先把所有分项并成一份 `这一版先改什么`
  - 明确这轮到底是卡在了哪一项检查上
  - 只留还会挡住正文改完的硬点，把已经降成普通提醒级的旧问题删掉
- 别把“已经改善但还是没过去”的问题继续按旧等级整段泛修
- 进了“这一版先改什么”这一套，就别再按“修一点 -> 立刻再做一次全看 -> 再修一点”这种半步推进法来回返工：
  - 先把同类问题一次性批量改完，再统一复审
  - 别一边改章尾，一边把同批次人物只剩工具作用的问题留到下一轮
- 别因为某一项已经走顺了，就提前结束另一项更严、但还卡着的问题
- 不同检查写成两套话时，就按下面这个顺序看，别去取平均：
  - 先看有没有 `F5`
- 再看当前卡住的那项检查是不是还没过去
  - 再决定这轮最小回修范围
  - 如果结构检查已经 `F0`、一致性检查也已经 `F0`，但章尾或人物检查还卡在 `F2`，后面回修就只盯这一项，别再回到整段大修

## 写作怎么走

> **这条别拧歪**：本 skill 不只是给建议，也负责把关。做到哪一步，都回头看 `references/production-hard-rules.md`，先照它来。它要是和别的参考文件打架，就按它说的办。

> **正文红线**：写正文、改章、回炉前，先读 `references/writing-red-lines.md`。这里面的红线要是和别的示例打架，就照它来。

> **正文纯净度守门脚本**：正文写完先跑 `scripts/draft_purity_guard.py --fix --json <正文文件>`。这脚本只处理三类尾巴问题：`尾部污染残片 / 自改尾巴 / 已知复发污染词`，而且只能从污染点前一个完整句那里截住。要是污染已经冲进正文前 `70%`，那就直接报失败，别偷偷改中段。项目里没有这个脚本，就把本 skill 的 `scripts/draft_purity_guard.py` 复制到项目 `scripts/` 再跑。
> **首稿一脏，当场就改**：正文首稿一落完，只要 `一级禁用词扫描 / 生硬撑劲词扫描 / 全文残片扫描 / draft_purity_guard` 里任何一项报了问题，就在这一章里马上改掉，别带着脏东西进 `scene_lint.py`、写后检查、下一章或后面集中再看的那一轮。平常写作就是当场修，不往后拖；只有纯基准测试例外，按 `H1-A` 记失败样本，不动正文。
> **正文场面质检脚本**：正文先把纯净度这关过掉，再跑 `scripts/scene_lint.py --json <正文文件>`，先看结构化结果；当前环境要是不方便读 JSON，再退回文本模式。项目里没有 `scripts/scene_lint.py`，就先从本 skill 复制到项目 `scripts/`。判断也简单：`summary.errors > 0`，这关就还没过；`summary.errors = 0` 且只报 `字数.warn`，先当提醒，别机械回改；要是 `summary.errors = 0`，但还报了 `章尾 / 解释 / 结构 / 情报` 这类 `warn`，就把正文翻回去再看一遍，再决定这一章还能不能继续往下写。
> **模板到底是不是真落进去了，再跑一遍这个**：这一章如果明着借了模板、情节库或事件母板，跑完 `scene_lint.py` 后还要再跑 `scripts/template_exhaustion_lint.py --json <正文文件>`。项目里没有脚本，就从本 skill 复制过去。脚本只要报 `error`，这一章就先停下，看看是不是 `模板词回流了 / 查证链拖太长 / 模板还没真正落进场面里`；如果只有 `warn`，写后检查里也得把“谁被挤开了 / 谁占了便宜 / 谁被逼得改动作”这三件事写出来，别一句带过去。
> **项目脚本版本先对一下**：进写后检查前，先跑 `scripts/script_version_check.py --json <项目根或当前书目录>`。项目里没有，就先从本 skill 复制到项目 `scripts/`。只要报 `error`，这一轮就先卡在 `F5`；别继续拿项目里的旧脚本结果硬往下做，也别接着写写后检查、章组总表，或者把这一轮当成已经过去了。
> **追踪状态校验脚本**：分章写后检查做完后、进 `追踪/角色卡同步` 前，跑 `scripts/validate_tracking_state.py --json <项目根或当前书目录>`。项目里没有，就先从本 skill 的 `scripts/validate_tracking_state.py` 复制过去再跑。脚本要是报 `error`，这一章就先停在这，别进下一章，也别急着收章组；如果只报 `warn`，也得在写后检查或章组总表里写一句：这是人工看过后决定先不动，还是先记着后面继续盯，别装作没看见就带过去。
> **收文冲突校验脚本**：章组总表、总修改清单、固定检查清单做出来后，得跑 `scripts/verdict_conflict_lint.py --json <项目根或当前书目录>`。项目里没有，就先从本 skill 的 `scripts/verdict_conflict_lint.py` 复制过去再跑。只要它报 `error`，这一轮就先按 `F5` 卡住；先把“这章到底留还是不留”前后写成一句话，再接着做后面的。
> **章尾撞型质检脚本**：如果当前范围已经到连续两章及以上，生成或更新章组写后检查前就跑 `scripts/chapter_hook_repeat_lint.py --json <项目根或当前书目录>`。项目里没有这个脚本，就先从本 skill 的 `scripts/chapter_hook_repeat_lint.py` 复制到 `scripts/` 再跑。脚本要是报 `error`，先看是不是连续章尾重复用了“高位角色入场 / 等待下一步 / 战略宣言”这三类钩子；一旦报了，这里就先停，别继续写。
> **关键角色晋升检测脚本**：如果当前范围已经到连续章组脚本触发阈值，或者本章新冒出公开冲突里的关键执行角色，就在 `追踪/角色卡同步` 前跑 `scripts/detect_key_character_promotion.py --json <项目根或当前书目录>`。项目里没有，就先从本 skill 的 `scripts/detect_key_character_promotion.py` 复制到 `scripts/` 再跑。脚本要是报“连续两章关键动作角色缺卡”，就按 `F5`，别拖到审查阶段才补记。阈值见 `references/continuous-chapter-thresholds.md`。
> **人物能动性质检脚本**：如果当前范围已经连续两章及以上，而且本章里有高价值角色、软肋角色、公开冲突里的关键配角，就在 `detect_key_character_promotion.py` 后再跑 `scripts/character_agency_lint.py --json <项目根或当前书目录>`。项目里没有，就先从本 skill 的 `scripts/character_agency_lint.py` 复制过去再跑。脚本要是报 `warn`，写后检查里要明写哪个角色正在滑成“只会出来办事”，准备在哪一章或哪一拍拉回来；若报 `error`，就别直接写“这块先扔那不管”。
> **中段会议化 / 作者代判句校验脚本**：本章如果属于 `翻册 / 查账 / 认物 / 抢先办事的机会 / 规则争夺 / 当众定规矩` 这类书名主打场面，`template_exhaustion_lint.py` 后还得跑 `scripts/scene_narrowness_lint.py --json <正文文件或当前书目录>`。项目里没有这个脚本，就先从本 skill 的 `scripts/scene_narrowness_lint.py` 复制到项目 `scripts/` 再跑。脚本报 `warn` 时，写后检查里还得把这三件事写出来：`谁的位置被改掉了`、`谁因此吃了即时损失`、`作者代判句删掉后本段自己还能不能把意思带出来`。三问里只要有一问说不出来，这章就先按 `F2` 看，别急着写成 `F0`，也别急着说这章先搁着，更别往上吹成 `模板已经真写进去了 / 剧情已经往前走了 / 这一段已经把场面写活了`。
> **章尾报错了，人工再看一眼**：`scene_lint.py` 要是报了 `章尾.error`，写后检查别只抄脚本原话。把该章最后 `5-15` 行重读一遍，弄清末句到底停在 `新物证 / 新的一句话 / 新名单 / 新翻面 / 下一步命令 / 总结盖章` 里的哪一类，再写进写后检查。可以写“脚本报了这条，但我重读后觉得不算”，也可以写“脚本报了这条，我重读后觉得算”。别把 `章尾.error` 直接算成最后失败码。
> **章尾这类报错，要拆开看**：`scene_lint.py` 报了 `章尾.error` 后，写后检查还得写两句：`最该落下的那下收益是不是已经在章尾前落成了`、`现在的问题到底只是末句收得太顺，还是整章兑现本来就不够`。要是最该落下的那下收益已经在章尾前落成，只是最后半拍收顺了，或者多补了一句盖章句，那就先写成 `章尾最后半拍偏软 / F1 / 先放着盯`，别直接提到 `F2`；只有最该落下的那下收益压根没落成、整章只能靠章尾末句硬撑时，再按 `F2` 或更高失败码去记。
> **正文干不干净，再扫一遍**：过完纯净度守门和场面质检后，还得再查一遍正文纯净度，确认整篇尤其章尾没有 `*** Begin Patch` / `*** End Patch` / `functions.` / `multi_tool_use.` / `commentary` / `analysis` / 工具输出残片 / 执行日志残片。只要撞上一项，就当正文还不行；别往结束里记，也别收进主正文，先把污染清干净再复扫。
> **没扫干净，别急着进下一章**：正文纯净度检查不只是收文前检查，也是跨章那道卡。任一章节如果还没明确记下“全文残片扫描过了”，就先别进下一章正文；也别拿“先把后章写完再一起清”这种话往后拖。
> **场面脚本打回来，就整套重走**：`scene_lint.py` 报 `error` 后的改写仍然是完整正文回炉，不是局部润色。回修时得重新照着本 skill 的整套正文规则再走一遍，包括写前闸门、题材识别、对应专项检查、`references/writing-red-lines.md`、`references/production-hard-rules.md`、正文纯净度检查、写后检查、追踪/角色卡同步。`scene_lint` 这次没再报错，只说明场面脚本这一关先过去了，不等于别的硬检查可以跳过；若只报 `warn`，也得先看脚本提示是不是要补强，别把预警一股脑打回去。
> **AI味重了，就立刻转去处理**：写后审查如果看下来已经到 `AI味 = 中度` 或 `重度`，别只停在 `narrative-writer` 自己说“还能凑合”这种话上，马上转进 `story-deslop`：
> - `中度`：先跑一轮 `story-deslop` 检测
> - `重度`：先跑一轮 `story-deslop` 修文
> 这一步没做到，就先别写这章已经算过，也别收进主正文。
> **失败码前后得对得上**：写前闸门、正文写作、写后检查这三处都得产出 `F0-F5` 失败码；只要最终失败码不是 `F0`，就别收进主正文，也别进下一章。
> **现实收益硬规则**：`本章高潮后续收益` 只能填“本章正文已经发生”的 `命 / 人 / 名 / 权 / 物证 / 人心变化 / 规矩变化 / 垫背者落地`；别把“第 N+1 章可直接进入”“已经布好局”“下一章能反制”写成当前章收益。
> **首章这两项别写混了**：首章写后检查里，`章尾停在哪类结果` 和 `高潮兑现写到哪一步` 得分开填，不能互相顶替。章尾合格但高潮后续收益还没真正落下来时，照样别判 `F0`。
> **首章语言也得单独拎出来看**：首章写后检查除了 `章尾停在哪类结果` 和 `高潮兑现写到哪一步` 外，还得单列 `语言这一项`，先回答三项：`是否存在熟比喻抢戏`、`是否存在人物台词过于会说/金句化`、`是否存在作者替人物补义`。首章前 `40%` 如果连续撞上 `2` 处及以上，先按 `F1` 看，别因为“结构对了”就直接当它已经能用了。
> **首章关键角色台词，得单独抽出来看**：首章中主角外的第一关键角色，单独抽看其连续 `3-5` 句台词，看它是不是符合角色卡的 `说话规则 / 被压迫时说话规则 / 被威胁时先护什么`。若角色卡写的是“字少，句重”，正文却出现整齐排比、作者判词、文青式悲凉句，就按 `角色口条失真 / F1` 处理；若该角色承担首章扛场、翻面或软肋功能，就把它当成首章必须卡住的一项。
> **高危句型这里也别漏看**：首章与连续章关键收益位，写后检查前得对照 `references/high-risk-sentence-patterns.md` 过一遍。要是明明撞上黑名单，最后却还写 `F0`，那就把“为什么这次重看后仍不算问题”写一句；不然别直接写成“这事先往后放一放”。
> **情报流先看记录补没补**：情报流章节一旦出现正式情报文本，写后检查前就先看 `追踪/情报记录.md` 在不在、最小回填做没做完；缺了就别结束本章，也别自己写“追踪同步这边都补完了”。
> **章组总表别漏**：如果这一轮连续生产或回炉已经走到该出章组总表的范围，除了分章 `写后检查_第XXX章.md`，章组收住前还得补一份或更新一份 `追踪/写后检查_第XXX-XXX章.md`。总表里至少写清这些：`本组已经发生了什么`、`各章现实收益`、`关键角色卡写到哪一步了`、`情报证据等级变化`、`这一组到底留不留`。少了这份总表，这个章组就先别往下收，也别进后面那轮再一起看，直接按 `F5` 卡住。触发阈值看 `references/continuous-chapter-thresholds.md`。
> **章组写后检查，文件名别留两版**：章组总表文件名就统一用 `写后检查_第XXX-XXX章.md`。如果同一范围里同时躺着 `写后检查_第XXX-XXX章.md` 和 `写后检查_第XXX章-第XXX章.md`，那就在继续往下做前先把旧名文件清掉，并在 `追踪/上下文.md` 或本轮写后检查备注里写一句“文件名已经统一”。双版本还并着没清，就按 `F5`。
> **命名撞车，就当场清掉**：`validate_tracking_state.py` 要是报了 `双版本并存.error`，而且只是章组写后检查命名不一致，那就别记着“回头再说”。当场看一下，留最新、范围对、内容更完整的那份，另一份删除或废止，然后重新跑一遍 `validate_tracking_state.py`。没复跑过前，别继续写下一章。
> **角色卡只许回填正文已经落下的东西**：角色卡里的 `当前已落地事实`、`本章得回落的记忆点`、`状态变更记录` 只能填正文已经兑现的内容；角色卡如果写进了正文没出现的能力、道具、动作记忆点或关系结论，就按事实层污染算，记 `F5`。某个记忆点如果连续两章都没回落，就先删改角色卡里的硬要求，别硬拿追踪文件替正文补票。
> **角色卡还空着，就不算同步完**：如果 `角色状态.md`、章组总表或分章写后检查里已经写明角色有新的已落地事实，可对应 `设定/角色/*.md` 的 `当前已落地事实` 还停在 `暂无 / 待补 / 未更新 / 空白占位`，那就按“角色卡同步还没做完”看，直接记 `F5`。这不是能先放着以后再补的小缝，别在总修改清单里把它压成普通提醒。
> **前后别写成两套话**：同一轮审查里，`分章写后检查`、`章组总表`、`总修改清单` 对“这章到底留不留”的写法得完全一致。只要有一个文件写“这章先留”，另一个更高层文件却写“这章不行”，或者反过来，那就直接按 `F5`，别带着两套话继续做下一步。
> **收益还在低档，就别硬往上抬**：如果 `章组总表 / 写后检查_第XXX-XXX章.md` 已经把本组末章或本组总收益写在 `资格 / 名分真正落到手里 / 现场暂时先听这边的` 这一档，那同一份文件里也要写一句：`为什么这样现在也只能算到这`，或者 `所以这次还得先拦住`。别出现“收益明明还停在低档，却硬写这章能过”的话。撞上这种情况就按 `F5`，并先看看是不是“追踪那边替正文又往上拔了半档”。
> **首章章尾 error，这几份文件也得写成一套话**：如果首章 `scene_lint.py` 报了 `章尾.error`，那单章写后检查和章组总表里都要写一句：这条是“脚本报了这条，我重看后还是觉得算”，还是“脚本报了这条，但我重看后还是觉得不算这条”。前后写法也得跟着统一。别出现“单章写后检查写着 `F1`，却又说这章能留；章组总表也说这章能留；总修改清单却说这章还不能留”这种三边话都不一样；一撞上就按 `F5`。
> **单章兑现还是章组兑现，先把话说准再做**：平常先按 `单章兑现` 写；当前章节如果属于 `围猎 / 布网 / 截令 / 连续翻盘` 这种天然跨章结构，可以切到 `章组兑现`，但得先在 `卷纲/章组控制` 和 `单章细纲` 里把 `兑现组范围`、`本章是第几拍`、`本章一定要拿到哪一步现实中间收益`、`最迟哪一章落完整收益` 填进去。少任一项，都别拿“后面会一起爆”直接写成“这一步已经办完了”。
> **借跨章情节库模板时，也得把中间收益说准**：若本轮明确参考 `情节库 / 连续剧情库 / 场景母模板库` 的跨章模板，允许按 `情节库兑现` 生产，但得额外把 `采用的是哪类跨章母模板`、`本地化后这一串事准备怎么落`、`本章已落地的现实中间收益` 先填进去。若只借到模板壳、没拿到现实中间收益，仍按 `F2` 处理。

> **几份东西撞车时听谁的**：顺序是 `production-hard-rules.md` > 本 SKILL 主流程 > 其他 references。凡是还带着“先写完再说”“先塞进正文再修”这类旧习惯的，都先按生产硬规则拦住。

> **先认准现在到底写到哪**：进日更续写或大修前，先把“当前真实写到哪一章”弄准。`追踪/上下文.md`、`追踪/时间线.md`、`追踪/角色状态.md`、`正文/` 最大章节号、`大纲/细纲_第XXX章.md` 要是互相打架，就先信“已存在正文 + 已同步追踪”这一组；旧预案、旧卷纲、还没落地的远期细纲，都别拿来当已经发生的内容。
> **嘴上说做完了，不算真做完**：如果 `追踪/上下文.md`、进度记录或同类文件里写了“已完成某章组总表 / 已完成某章写后检查 / 已完成某组写完 / 已完成正式挂名后的这段收束”，那就去把对应文件翻出来，看它是不是真在、范围对不对、是不是只剩这一版。文件缺了、同范围并着两版、或者命名还没统一，都直接按 `F5`，别把“纸上写过了”当成真的做完了。

> **平常写作就这么接着做**：除非已经切进连续回炉的检查单那一套，不然平常写作就按下面这个顺序接着做：
> `设定地基 -> 题材识别+把对应专项检查接上 -> 开书总蓝图 -> 卷纲/章组控制 -> 单章细纲 -> 关键硬场面卡/动作拍序 -> 细纲这关走顺了 -> 正文 -> 写后检查 -> 追踪/角色卡同步`
> 其中哪一步还没弄实，就别硬跳到后面。
> **正文别直接从细纲硬铺**：`单章细纲` 不是拿来直接摊成正文的；先把本章最要紧的 `1-2` 个场面落成 `关键硬场面卡 / 动作拍序`，再进正文。要是写正文时还在靠“边写边发明桥、动作、险点、临时翻局”往前顶，那就是细纲没把这章兜稳，先退回细纲层，别在正文层打补丁。
>
> **回修也得从头走全**：正文只要是因为 `scene_lint.py`、纯净度检查、写后检查、追踪交叉复查、审查那边写的话这些地方被打回来的，就把“对应章节细纲 / 写前闸门 -> 重写正文 -> `scene_lint.py` -> 纯净度检查 -> 写后检查 -> 追踪/角色卡同步”这一整套再走一遍。别偷偷降成“只改几句然后直接收正文”。
> **这些追踪脚本别老拖到最后才跑**：`script_version_check.py`、`validate_tracking_state.py`、`verdict_conflict_lint.py`、`chapter_hook_repeat_lint.py`、`detect_key_character_promotion.py`、`character_agency_lint.py` 都别拖到后面集中再看时才跑。平常就跟着正文一路跑下来：`正文 -> draft_purity_guard.py -> scene_lint.py -> template_exhaustion_lint.py（如适用） -> scene_narrowness_lint.py（如适用） -> 写后检查 -> script_version_check.py -> 章组总表（如适用） -> validate_tracking_state.py -> chapter_hook_repeat_lint.py（连续章时） -> detect_key_character_promotion.py（如适用） -> character_agency_lint.py（如适用） -> 追踪文件同步 -> 角色卡同步 -> verdict_conflict_lint.py -> 交叉复查`。其中任何一个脚本报 `error`，这章或这个章组都先别写成“这一轮先算弄完了”。
> **双层目录时先认准真实书目录**：如果冷启动项目是“外层项目根 + 内层真实书目录”这种双层结构，跑脚本时先把“正文/设定/追踪所在目录”认成真实书目录，再把“scripts/ 所在目录”认成项目脚本根。别因为外层目录和内层书目录分开，就把 `script_version_check.py`、`validate_tracking_state.py`、`verdict_conflict_lint.py` 跳过去；参数拿不准，就先钻进真实书目录再跑。
> **规则动过之后，旧问题也得再翻一眼**：这一轮如果改的是 skill、流程或必停项，不是正文本身，最后一步就去读 `references/regression-handoff.md`；项目根要是有 `scripts/story_review_regression.py`，就先跑它，把自动汇总看完，再决定哪些地方还得人工再过一遍。最后回复里直接写明，这一轮又把哪几组旧问题重新过了一遍。
> **写后检查里，章尾怎么判得写一句**：写后检查里的 `章尾停在哪类结果` 不能只写“合格 / 不合格”，脚本报 `warn/error` 时还得额外记一句：`脚本报了这条，我重看后还是觉得算` 或 `脚本报了这条，但我重看后还是觉得不算这条`。本章如果存在脚本报了问题但人工复看后不认这条的情况，为什么不认，也得写进写后检查备注，别一句带过。
> **第一次硬收益后章尾别混着记**：如果问题主要出在“第一次硬收益已经拿到手，章尾最后那半拍却收软了”，写后检查里要单独认出来。也就是：整章现实收益已经落地了，但末句少了再压一下、回弹一下，或者翻面后的余劲。碰上这种情况，别记成“整章兑现不足”，也别记成别的短兑现链问题；就写成章尾收束偏软，后面复看时单独盯这类问题。

> **连续复审时别再碎改**：如果同一章节或章节组已经连着复审了几轮，主失败码虽然在缩小，但还是过不去，那就改走下面这套，别再零碎返工：
> 1. 先把所有分项并起来，整理出一份“这一版先改什么”
> 2. 标出这轮到底卡在哪一条检查上（例如章尾 / 人物 / F5）
> 3. 按“同类问题一次修完”去改，别半步改善
> 4. 改完后再统一复审；别每改一小块就重新从头看一遍
> 5. 复审后如果只剩单章或单小段残留，后面就继续缩到那个最小范围，别重新放大回整组

> **一旦开始列“这一版先改什么”，这几件事就直接写上**：切进“这一版先改什么”模式后，修改清单里直接列出来：
> - 这轮卡在哪一项检查上
> - 当前还会挡住正文改完的硬点清单
> - 本轮不再处理的已降级旧问题
> - 本轮允许修改的最小文件范围
> - 复审时优先检查的单项硬检查

> **正文被打回来时，就顺着这个次序往回找**：正文撞上那种一碰就得停的规则时，就按下面这个次序往回找：
> `先看句子功能是否失真 -> 再看场面层是否不足 -> 再看细纲是不是根本没把这章托住`
> 别一上来只删词，也别明明是细纲撑不住这章，还硬在正文上缝补。

先看用户这次想干什么，再看项目现在到哪一步，把这次归到对应场景里：

| 场景 | 触发条件 | 该读什么 |
|------|----------|----------|
| **开书** | "帮我开书" / 项目目录为空 | 从 Phase 1→2→3→4→5 整套走下去 |
| **日更续写** | 关键词（"日更"/"续写"/"继续写"）**且**项目已有正文+追踪 | 读 `references/workflow-daily.md` |
| **大修** | "修改第X章" / "回炉" / "重写第X章" | 读 `references/workflow-revision.md` |

> **开新卷**：如果新卷带进了新角色 / 新势力 / 新设定，就先回 Phase 2 把该补的东西记下来，再进 Phase 3 补新卷细纲，最后回 Phase 4 写；如果只是顺着当前线继续写，直接回 Phase 3。

**场景碰到重叠时就这么认**：同时撞上多行时，按 日更续写 → 大修 → 开书 来认。日更续写那个 AND 条件（项目已有正文+追踪）要是不满足，就直接提示用户“项目还没有正文，建议先开书”。

**日更续写走上 daily，就别乱跳**：一旦本次请求已经进了 `references/workflow-daily.md`，后续同一批次里用户再说"继续"/"续写"/"日更"，都算继续同一条日更串行流程；别跳出 daily 直接写正文，也别重新回到场景选择。正常批量执行里不再问"是否继续"；只有细纲缺失、章节号冲突、用户明确说要一章一章过，或者请求会改动既有大纲/追踪时，才停下来再问一句。

**连续回炉就顺着同一张检查单改**：一旦本次请求已经进了 `references/workflow-revision.md`，而且已经判断成“连续章节回炉”，后续同一批次里用户再说“继续”，都算继续执行这同一份检查单上的那套修改次序；别跳出去临时换一套写法，也别退回“修一章看一章”的旧路子。

**已经排了计划，就顺着计划往下推**：一旦本次请求已经生成 `执行计划_*.md`，后续同一批次里用户再说“继续”，都算继续执行计划里的下一项；别跳过没做完的项，也别只在聊天里报进度、不去更新计划文件。

**已经判整段重写，就别又缩回小修**：一旦当前章节已经判成“一次性整段重写”，后续同一批次里用户再说“继续”，都算继续整段重写；别又退回零碎补丁式小修。

#### 对标上下文怎么带

> **拆文库/对标关系**：`拆文库/` 是 analyze skill 的原始产出，也就是数据源。`对标/` 是写作项目里的引用视图，只放和当前项目有关的对标数据子集。第一次用某本对标书时，把 `拆文库/{书名}/` 里相关子目录（角色 / 剧情 / 设定）复制到 `对标/{书名}/`。
>
> **对标书路径查找**：优先 `{项目}/对标/{书名}/`，不存在则回退 `拆文库/{书名}/`。下文所有对标数据加载均使用此规则。
>
> **对标采样这一步别省**：只要准备把对标里读出来的东西写进细纲必停项、正文红线、章尾必停项、写前闸门，就先读 `references/benchmark-sampling.md`，并写清“这轮到底读了哪些区段、多少章、多少场景、多少字、现在这话最多能说到哪”。没读够，就只写成局部看到的情况或临时办法，别硬抬成整本都通用的规则。

如果用户提到对标书或工作目录下已存在 `对标/` 目录：

1. 检查对标书的 `拆文报告.md` 是否存在（按对标书路径查找）
2. 如存在，就把最要紧的那几条发现先读了（开篇钩子、爽点密度、节奏路数、可借鉴套路），当参考上下文
3. 只要用了对标原文或拆文结果，就同步提炼当前要写角色的 `对白口气模板`：至少补出 `常用称呼 / 句长习惯 / 软硬程度 / 常见找补词、压迫词、拖延词 / 急了以后怎么变短、变狠、变虚 / 一般不会怎么说`。禁止只借情节和节奏，不借人物嘴型。
4. 如均不存在，提示用户：「对标书原文已放入 `对标/{书名}/原文/`。要先使用 `story-long-analyze` 做深度拆解吗？拆完后 `拆文报告.md` 会自动存入 `拆文库/{书名}/`，写作时会自动按 `对标/ → 拆文库/` 顺序读取。」如果用户要求直接继续，也要先从原文局部提炼口气模板，再进细纲或正文。
5. 第一次进对标流程时，把 `对标/对标采样执行单.md` 建起来或写完整，并按 `references/artifact-protocols.md` 模板回填“这轮要看什么 / 样本书单 / 原本打算看多少 / 实际看了多少 / 真有用的场景单元 / 这次先看到哪儿”
6. 如果结构化子目录（角色 / 剧情 / 设定）已经在，写作时就自动把相关模块带上；如果 `角色 / 对话声纹 / 角色思想话术` 之类文件已存在，正文阶段必须同步召回

#### 提取素材库怎么接进来

如果用户明确要求“用已提取的资料库冷启动新书”“完全使用提取素材”“把本地 TXT 提取库套进新书”，或者项目里已经有来自 `story-local-plot-extract` 的资料库目录：

1. 先把该资料库当成`写法参考库`，别当成能直接抄正文的地方。
1.1. 先弄清 `资料库到底盖到哪`，这一步别跳：
   - 这套资料库覆盖原书哪些章节范围
   - 现在已经抽出了哪些层
   - 哪些层只覆盖前段，哪些层覆盖中后段
   - 这次的话到底只是 `只看局部时能这么说 / 这一段前后连起来还顺 / 前后多段摆在一起也还连得上` 里的哪一档
1.2. 要是当前资料库只覆盖原书一部分，或者不同层覆盖范围对不上，那就别写“整本书就是这一路子 / 这本书主情节就是这个 / 按整本书就是这样来冷启动”这种一把盖过去的话；撞上就按 `F3`。
2. 开写前，把下面几层里用得上的文件读过，顺手归拢一下：
   - 文风层：`文风参数卡.md`、`对话声纹卡.md`、`关键表达句型库.md`
   - 情节层：`连续剧情库.md`、`场景母模板库.md`、`场景子模板库.md`、`场景参数库.md`
   - 语言层：`语录化用库.md`、`角色思想话术库.md`、`经典_热梗嫁接模板.md`
   - 约束层：`场景失败模式库.md`、`隐性规则库.md`
3. 在项目里新建一份文件，名字可以用 `设定/素材迁移方案.md`、`设定/写法注入方案.md`，或者差不多意思的文件，里面把这些事写清楚：
   - 这次迁移最底下那套真正起作用的写法是什么
   - 这次的话来自资料库的哪几个覆盖区间
   - 这次只把哪些层当作`局部能借的信号`，别把局部样本误认成整本都得这么走
   - 哪些是迁移`文风 / 语言功能 / 场景结构 / 组织逻辑`
   - 哪些设定词、世界壳、具体桥段禁止直接照搬
   - 新书里准备如何把原模板改写成当前题材的具体走法
   - 本次显式禁用的旧脸 / 旧母场景 / 旧首章起手
3.1. 同时把 `设定/资料库采样执行单.md` 建起来或写完整，并按 `references/artifact-protocols.md` 模板回填：
   - 本轮实际读取文件
   - 各层覆盖范围
   - 只看局部时读出来的东西 / 这一段前后都还顺的东西 / 前后多段摆在一起也还顺的东西
   - 本轮明确采用的是哪一层的推进办法
   - 本轮显式禁用的旧脸
3.2. `设定/资料库采样执行单.md` 要是没写完，就别把资料库里的话硬抬成这次开书只能这么写、蓝图主线只能这么定、书名也只能这么解释；先按 `F5` 处理。
3.3. 再把 `设定/素材迁移方案.md` 建起来或写完整，并按 `references/artifact-protocols.md` 模板回填：
   - 这次迁移只借哪几层
   - 哪些局部判断只允许局部参考
   - 哪些判断已经够资格进总蓝图
   - 本轮显式禁用的旧脸 / 旧母场景 / 旧第一战果
3.4. `设定/素材迁移方案.md` 要是没写完，或者说不清“哪些只是局部参考、还不能进总蓝图”，那就先别进 `大纲/开书总蓝图.md`，先按 `F5` 看。
4. 用户如果说“完全使用提取素材”，这里就按下面理解：
   - 尽量完整调用资料库中能搬过来的写法
   - 允许高度继承原资料库里已经试出来确实有效的`情节怎么往前走 / 收益结构 / 爽点顺序 / 章组节奏`
  - 但得把`换壳 / 换世界底盘 / 换主角身份壳 / 换资源与权力靠什么落地 / 换人物关系链 / 换场面实现`做完
  - 别把原书设定名词、原书角色关系、原书世界制度、原场景动作链直接平移后冒充原创冷启动
5. 只看了书名、题材点、少量情节卡，还不算“已经用了提取资料库”。正文前至少把下面这条最小闭环搭出来：
   - `资料库采样执行单`
   - `素材迁移方案`
   - `冷启动蓝图 / 开书总蓝图`
   - `首章正文`
   - `追踪/情报记录.md`、`追踪/角色状态.md` 等最低追踪底座
6. 资料库里如果有明确的“治理脑 / 组织脑 / 经营脑 / 宣传脑 / 平台脑”这类高阶结构，迁移时先接这些`底层驱动`，别只抓表层剧情点。
6.1. 资料库里如果同时有几种高阶结构，也别因为某一层在局部样本里最容易起稿，就把它直接定成这轮只能走它这一条；先把 `开书候选分叉 + 开书排异表 + 同题材去重记录` 补出来。
7. 正文如果只是看得出“借了一个书名点或几条事件”，却没把资料库里的文风、说话习惯、场景翻案方式、外部现实变化方式带进来，那就不能算“资料库注入成功”，先按 `F2`。
7.1. 正文或蓝图要是只抓住资料库里最显眼的一层母模板，却把它误写成“整本参考书的主情节 / 整本书就照这个路子走”，就按“资料库读偏了”看，先按 `F3`；先回 `素材迁移方案`，把覆盖范围、层级边界和禁用旧脸写明白。
8. 如果是冷启动测试任务，建议再补一份项目内文件，比如 `大纲/冷启动蓝图_提取素材注入版.md`，把下面这些记明白：
   - 调用了哪些资料库层
   - 每层对应的覆盖范围和当前这话最多能说到哪
   - 首卷前 10 章打算迁移哪些模板
   - 这一轮做到什么才算测到位
9. 提取素材库不是免追踪通行证。用资料库写首章时，`追踪/` 里已经发生的事实还是得同步；别只落“素材方案”和正文，不回填情报、角色、战果写法。
10. 用户如果要的是“纯 skill 冷启动试跑”，正文可以留失败样本，但 `素材迁移方案 -> 蓝图 -> 首章 -> 追踪回填` 这条最小试跑链还是得落进项目文件；缺一项都先按 `F5`。

Phase 1 收尾前，先看看项目里有没有 `设定/选题决策.md`：

- 若只有扫榜输出目录中的 `选题决策.md`，允许吸收到项目内并规范为 `设定/选题决策.md`

这里先守住这几条：

- `设定/选题决策.md` 不是可有可无的文件，它就是开书阶段管市场方向的那份方向底稿
- 这份文件里把这些事写清：
  - 现在优先往哪边开
  - 你觉得它为什么有爆的可能
  - 这份判断现在有几分把握
  - 它跟同题材作品怎么拉开

### Phase 2：把设定地基垫稳

设定地基、题材识别前置、设定表模板、设定类文件先一起读：
- `references/core-setting.md`
- `references/story-type-and-payoff-model.md`

Phase 2 先把下面这些东西补齐：
- `设定/题材定位.md`、`设定/世界总控表.md`、`设定/爽点怎么落.md` 这三份要在 Phase 2 一起落出来
- 先弄准主角是谁、世界靠什么转、主冲突卡在哪、这本书到底算什么路数
- `设定/关系.md`、`设定/题材定位.md`、`设定/世界总控表.md`、`设定/爽点怎么落.md` 少一份，就先别进 Phase 3

这几步都得做出来：

1. 先把小说类型分清。别只写一个宽泛题材词，下面这些都得弄准：
   - 外层题材
   - 内层驱动类型
   - 主打爽感类型
   - 本书当前阶段（求生爽 / 验货爽 / 经营爽 / 掌局爽 / 复仇爽 / 升级爽等）
1.1. 如果识别结果里，下面这些专项凑到 `2` 项以上，就把世界底盘搭起来：
   - 高环境压力 / 路线推进
   - 超凡 / 异象 / 关键能力
   - 信息差 / 情报 / 预知

**章节标题这块**：
- 不再只做轻量去重
- 每章都先对一遍 `references/chapter-title-rules.md`，看准标题该往哪一路写、哪些不能碰、这章要打哪块卖点
- 标题得直接挂住本章第一爆点 / 本章最该先打出来的好处 / 章尾钩子里的一个
- 标题先打出`冲突`，其次才是`结果`、`物证`、`规矩变化`；若标题里既没有人和人/人和规矩/人和利益的冲突，也没有“谁先动了谁的东西、谁先急了、谁先翻脸了”这类冲突感，就先按弱标题重做
- 标题多数时候从 `结果型 / 冲突型 / 物证带刺型 / 规矩翻面型` 里想，别先写成“这一章写什么”
- 禁止把章节标题写成 `写作说明 / 过程交代 / 轻度祈使句`，例如“先把……挂上”“把……揭开”“这事还没完”“先去看看……”这类句式，除非它本身已经带强冲突和强翻面
- 别把章节标题写成 `规则标签句 / 术语标签句 / 检查项句`，例如“发卖令把话收回去”“护营印换手”“押路签重排”“名分落到谁手里”这类标题。它们看着像在概括事件，其实更像写细纲时顺手记的一句活，不像读者会点开的章节名。标题得先像人会说的话，再谈概括力。
- 若标题只是交代本章流程、动作顺序或写作步骤，而没有提前打出利益变化、现实冲突、物证刺点或对手被挤开，就按弱标题重做
- 标题先问一句：`删掉章节号后，这个名字像不像会让人点进去看，而不是你写细纲时顺手记的一句活`
- 再问一句：`这句话像不像角色、围观的人，或者读者嘴里会顺出来的话；要是更像检查表、流程短记、随手备忘，就重想`
- 还得再问一句：`这名字里的冲突到底落在谁和谁身上，或者是谁先动了谁的利益`；要是说不出来，就继续重想
- 开书前 `20` 章先多试几个标题：每章先出 `3` 个备选标题，再按 `冲突感 / 画面感 / 翻页感` 三项打分；若最终标题有任意 `2` 项偏低，就先别定稿
- 当前如果还在开书阶段，细纲里别只放一个暂定章名就直接进正文；备选标题和打分记录没补，就按标题预检没做完处理
- 撞上同名、弱标题、概括型标题、剧透型标题，就重命名，并把细纲标题和正文文件名对齐

- 大纲做完后，把下面这些文件补齐（模板见 [references/artifact-protocols.md](references/artifact-protocols.md)）：
- **设定/世界总控表.md**：这份先管世界底盘，里面把世界往前怎么推、权力层级、资源盘、规则位、事件池、世界信息分期投放表落下来。重点讲明白：为什么这里会一直出事、所有人在抢什么、主角为什么偏偏在这里能起势。
- **设定/主角终局高度设计.md**：这份管主角最后能走到哪，里面把终局层级、终局身份、终局权力、终局资源盘、终局规则位、每卷上升路径排出来。男频长篇别一开始就把主角上限锁死在一小块地盘里。
- **大纲/开书总蓝图.md**：这份管全书大盘，里面把世界底盘、权力层级、势力盘、资源盘、实体规划层（人物 / 势力 / 生物 / 植物 / 物品 / 地点 / 规则 / 事件）、世界信息分期投放表、总字数、总卷数、每卷章数区间、每卷阶段任务、主角成长梯度、敌阶梯度、阶段审查节点排出来。
- **大纲/全书卷-章组分配表.md**：这份先把“卷怎么切成组”列出来，里面再把每卷预计章组数、每组预计章数范围、每组主要任务、每组最重收益、兑现方式、最迟兑现点、卷尾翻面排出来。后面再让 `章组控制_第X卷.md` 去细管“这一卷每组怎么打”。
- **大纲/全书章节总表.md**：按章俯瞰全书，每章先写一句主要事件 + 一句主要收益/翻面 + `本章世界信息点`，方便总览推进。
- **大纲/事件线总表.md**：按线总览全书，先拆出主线和关键副线，把起点、中段升级点、卷级翻面点、预计回收点、对应章节范围列出来。
- **大纲/大纲.md**：全书卷级鸟瞰（卷名+字数+章数+主要事件+状态变化，一段式汇总）

### Phase 4：正文怎么写

正文红线、题材专项闸门、情报流格式、字数预算、章尾否决，这几份先一起读：

Phase 4 先盯这几件事：
- 开写前把 `设定/题材定位.md + 设定/爽点怎么落.md + 大纲/开书总蓝图.md + 当前卷卷纲 + 当前卷章组控制 + 本章细纲 + 本章写前闸门` 全对一遍
- 解释腔、后台调度语、拿抽象效果词顶动作，这几类都得拦住
- 开写前还要把本章涉及角色的 `对白口气模板` 对一遍：至少答清 `这场谁更硬、谁更虚、谁会先抢话、谁会绕、谁会装、谁会压`；答不出就先别写对白
- 连续多章只查不爆、只涨信息不涨情绪，就按整套写法失衡回炉
- 若单章只是“事情发生 -> 主角处理 -> 章节结束”，没有连续翻面、改打法、谁往前谁往后变了、改批次、改归属等任一外部现实变化，按场面不足回炉
- 若大场面章节只热闹不改局，就回退到细纲重写 `第三方插刀点 / 退路烧断点 / 外部现实里谁往前谁往后变了`，别在正文层靠补几句骚动描写硬撑
- 若关键角色在正文里还只是功能说明、递线索、站阵营，没露出自己的算盘或自保方向，就先退回角色卡和人物冲突卡，别只润台词
- 若关键对白读起来像在汇报情况、打圆场、赶着表态，就回退到 `人物说话权限表 / 对话里谁压着谁`，别只在句子表面换几个近义词就想这么糊弄过去
- 若角色互相像同一个人，或者不看人名分不出是谁在说话，直接按 `声线混同` 处理，先回 `角色抽线回炉`，别继续顺写
- 写后要看的是这章到底真落下了什么，不是只看字数够没够
- 要是正文已经明显写偏，写成和这一套对不上的爽感类型，比如把“求生兑现爽”写成“慢查黑幕文”，那就按整套写法跑歪了处理，回炉

**对白硬约束**：长篇正文不只参考对标的剧情、世界壳和节奏，也要参考拆文库原文或拆解结果里提炼出的 `口气模板` 来写人物。至少要把 `主角 / 当前对手 / 当前盟友 / 场边第三方` 的嘴型差异拉开。禁止所有人共用同一种书面解释腔；禁止所有人都把话说全；要保留人物各自的称呼习惯、抢话方式、挡刀方式、压人方式、求情方式。目标是：关键对白里，不看人名也能大致听出是谁在说话。

- `交叉复查` 不是作者自己来一句“我觉得没问题”；只要黑名单解释句或证据等级漂移还在，就别判 `F0`

- 如果这一章用了对标原文、拆文库角色模块、`对话声纹卡.md`、`角色思想话术库.md` 里的任一项，正文首稿必须显式召回“这章谁怎么说话”，别只召回场面结构和情节功能

写前闸门模板、写后检查模板、写作提醒、字数硬约束、追踪归档，这几份也一起读：
- `references/chapter-templates.md`
- `references/chapter-group-postcheck-template.md`
