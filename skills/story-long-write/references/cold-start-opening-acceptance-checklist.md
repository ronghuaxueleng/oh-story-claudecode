# 冷启动首章组怎么看写没写成

> 这张表专门看新书冷启动的首章组（一般是第 `001-003` 章）。它不是替代常规章节检查，而是多盯一层：开书前两三章到底有没有真把书名主打、主要写法、分量、章尾接上。

---

## 一、适用范围

- 新书第一次从零开写
- 纯 skill 冷启动试写
- 用更新后的 skill 回头试一遍
- 用户明确要求“看看这套路子是不是真的写进了细纲 / 正文 / 追踪 / 写后检查”

碰上以上任一项，就把本清单挂上。

---

## 二、冷启动首章组最低得过到哪

首章组想算“这轮冷启动写成了”，至少得同时满足：

1. 细纲先得能直接拿来写
   - 已写出 `第一爆点`
   - 已写出 `第二轮收益`
   - 已写出 `群体反应位`
   - 已写出 `第二轮外部反应位`
   - 已写出 `章尾前一拍抬升位`
2. 正文得真把事写出来
  - `第一爆点` 已落成现场动作
  - `第二轮收益` 已落成现实改局
  - 至少 `2` 处外部反应已落地，且承担的作用不重复
  - `章尾前一拍` 不是预告，是现场里的新压力 / 新动作 / 新的先后变化
3. 厚度记录
   - 若无题材特殊说明，首章组正文一般应达到细纲目标
   - 冷启动首章组常用最低目标：单章 `3400+`
   - 若低于目标，只做观察记录，并额外说明本轮已经补入：
     - `谁被当场挤开`
     - `谁把第二步先手抢走了`
     - `哪道名分 / 名册 / 名单开始起现场作用`
- 首写阶段别只因为厚度不足，就把整轮都记成没写成
4. 前后接上
   - `追踪/时间线.md` 已同步
   - `追踪/角色状态.md` 已同步
   - `设定/角色/` 已补齐本章组关键角色最小人物卡，且角色卡事实层只写正文已落地内容
   - 若 `角色状态.md` 已写角色新事实，而 `设定/角色/*.md` 的 `当前已落地事实` 还停在 `暂无 / 待补 / 空白`，那就直接记成没接上
   - 若碰上情报流，`追踪/情报记录.md` 已同步
   - `追踪/写后检查_第XXX章-第XXX章.md` 已生成
- 若 `追踪/上下文.md` 或进度文件写着“某章组已完成 / 某组已写完 / 某组汇总已完成”，那对应文件就得真实存在而且只有这一版；若双版本并着或文件缺失，就直接记成没接上
- 若 `分章写后检查`、`章组汇总`、`总安排表` 的收口说法互相打架，也直接记成没接上
5. 脚本检查没出硬岔子
   - `validate_tracking_state.py --json` 为 `ok=true`
   - `scene_lint.py --json` 中以下项目先只作观察，不单独否掉首写这轮：
     - `字数.error / warn`
     - `解释.error`
6. 可读性 / 留存关键项过关
   - 人物出场分量最低配额成立
   - 主角、明敌、重要角色不能只剩办事作用
   - 若存在“第一软肋 / 生存线记忆点 / 私人救命压力”角色，连续两章内至少有 `1` 次再碰到这一面，别只剩名单结果、号码结果、资格结果
   - 若碰上重要角色，至少 `1` 位能让读者顺手想出这人的样子
   - 高潮兑现不是只靠主角宣布 / 纸面背书撑住
- 提纲术语没倒回正文
- 若本章存在新纸面说法，已明确它停在 `名字先挂上去 / 现场先得按这边来 / 这份好处已经当场起作用 / 好处已经真的落到手里` 哪一层
- 至少出现 `1` 个自然长出来的硬句触发位；若没出现，就记成首写弱项，但别单独拿它否掉整轮
- 该句要能答出“为什么只能由这个人说”；若任意角色都能换着说，一般就不算关键台词
7. 方法资料有没有真正用上
  - 本轮已显式记录实际调用的写作方法论资料
  - 每份资料都能回答“解决了哪一个当前风险”
  - 至少有 `1-2` 份资料能在正文里看到明确落地效果，而不是只存在于读取记录
  - 别把 `读了多少 / 用上多少 / 起了多少作用` 一次性全写成“都够了”；要保留：
    - 哪些只是读过
    - 哪些写进去了但还没完全落稳
    - 哪些仍待复审确认

这里只要有一项答不稳，这份冷启动样本就别先算写成了。

补充判断：

- 若首章组已经能证明 `第一爆点 + 第二轮收益 + 章组末次兑现 + 外部现实变化` 都进了正文，就别再把主说法写成“这套路子没用上”
- 此时若仍不好看，优先检查：
  - `路子已经落地，但过程写窄了`
  - `软肋角色这一面回得太少`
  - `重要角色这块代价写空了`
  - `连续章会议化`
- 若首章组章尾收益停在 `正式记名 / 正式准入 / 记录权到手 / 先把说法记到这边名下`，则下一章写前要先明确：
  - 第一张单子改谁
  - 谁当场被挤开
  - 谁当场得利
  不然这轮冷启动就别写得太满，别直接写成“以后已经能顺着写了”

## 三、首章组细纲先这么卡

进正文前，首章组细纲还得额外答这些：

- 书名主打这一章真实改了什么
- 第一爆点之后，谁被逼把原话缩回去 / 往后退 / 让出先后 / 改利益
- 两口外部反应分别落在哪一拍
- 章尾前最后一拍，具体是谁又做了什么
- 如果删掉解释句，这一章剩下的动作链还成不成立
- 如果删掉主角内心判断，第三方是否仍在现场真实改局

下面这些情况先别急着说够了：

- 只有路子名字，没有落点
- 只有“外部反应”四个字，没有写是谁、怎么改
- 第二轮收益只是“大家更震惊了”
- 章尾前一拍只是“更危险了 / 更麻烦了 / 下一章更大”
- 正文直接出现 `第二层反压`、`第三方迟疑成本`、`直接把这套路子的步骤喊出来`、`一章里把这套路子的效果全写死`、`刚起手就立刻全生效` 这类写作说明词

## 四、首章组正文再这么看

### 1. 第一爆点怎么才算立住

- 要是现场动作引爆，不认概念说明
- 要让在场至少一方立即改动作

### 2. 第二轮收益怎么才算到位

- 要回答：
  - 谁被迫把原话缩回去
  - 谁被迫往前挤或往后退
  - 谁被迫让出先后
  - 谁被迫改归属
  以上至少碰上一项
- 若只做到“原来的说法松了 / 大家重新讨论 / 强者开始入场”，一般只算 `半口收益`
- 若要把它往“冷启动有效收益”上提，还得再碰上下面一项：
  - 主角拿到公开试手资格
  - 主角先咬住一个先后机会
  - 主角先占住一个新名单 / 新名分 / 新说法
  - 对手失去独占先开口定事的资格，且已被写入公开记录

收益到了哪一步的补充：

- 若只落到 `新名单 / 新先后 / 新挂号 / 新名分`，判为 `名字先挂上去`
- 若名分已经逼得现场某个具体对象 `停手 / 把话缩回去 / 让道 / 让出先后`，但好处还未稳定兑现，判为 `现场先得按这边来`
- 若名分已落地，且这层名分立刻在现场起作用，判为 `这份好处已经当场起作用`
- 若已经落到 `先拿到 / 先拿货 / 先兑现名额 / 先吃到第一下真好处`，就写成 `好处已经真的落到手里`

要往上抬档时，再追问这几句：

- 若想从 `名字先挂上去` 升到 `这份好处已经当场起作用`，还要继续答出：
  - 哪个现实对象当场被这层名分 `拦住 / 挡住 / 压住 / 改道`
  - 谁因此被动往前往后挪、把话收回去、改先后、改资格
- 若只能答到“对象被拦住 / 被挡住 / 被压住 / 被改道”，但还答不出稳定好处，则只升到 `现场先得按这边来`
- 若只能答出“纸面已写上 / 说法已改 / 话已经放出来”，那就别往上抬
- 若新说法主要靠主角自己宣布才成立，而不是靠现场被动承认成立，也别往上抬

冷启动复盘时，要写清当前章到底停在哪一层，别把不同层混写成“都算真到手了”。

### 3. 两次外部反应怎么才算真落下

- 第一次通常落在中段：
  - 群体改线
  - 协作方把原话收短
  - 现场执行顺序让步
  - 第三方补第二下压力
- 第二次通常落在后段 / 章尾前：
  - 往前挤一步
  - 抢资格
  - 抢先后
  - 抢归属
  - 新一轮现场挤压

若题材属于 `纯个人升级 / 个人突围 / 单人登阶 / 单人抢机缘`，第二轮外部反应还得多看一条：

- 别只看“强者有没有下场”
- 还得看“强者这一步有没有切走某人的现实利益 / 观望权 / 先手权”

若只有“强者走近了、开口了、全场更重视了”，一般就按 `半生效` 看。

下面这些别算外部反应：

- 围观者只说“原来如此”
- 只是脸色变了
- 只是旁白说“大家都明白了”
- 同一类功能换句重复两次

### 4. 章尾前那一拍怎么才算写实

- 先得出现一处新的现场动作 / 现场压力 / 谁前谁后变了，再收章尾
- 别直接“结果句 -> 钩子句”
- 若章尾前一拍删掉后，章尾还是完全成立，那就说明这一处没打实
- 章尾最后一拍还要把前一拍兑现成新的现场结果；若最后只剩气氛收束、意义总结、悬念预告，而没有 `新名单 / 新名分 / 新先后 / 新物证 / 新资格落簿 / 新翻面` 之一，一般先别急着说写成了
- 若本章主打“章尾硬异动”，先看下面这组三连：
  - `新命令`
  - `立即执行`
  - `即时可见后果`
  若只写到前两项，没有第三项现场变化，一般还是按“章尾偏轻”看
- 若这三连已经在 `历史 / 灾难 / 末世 / 都市职场` 等不同题材连续复现，就把它当成开书常用写法，别再当成某个题材才单独用的招
- 冷启动样本收章时，先看这些结果：
  - `新名单`
  - `新名分`
  - `新先后`
  - `新物证`
  - `新资格落簿`
- 若只是“场面更紧 / 下一章更危险 / 竞争者更认真”，一般就别往高了写

### 5. 可读性和留存怎么看才算站住

- 主角、明敌、关键协作口至少各有 `1` 个动作 / 说话 / 私心记忆点
- 若本章有重要角色，别只停在“设定上很重要”，得让读者能顺手脑补出来
- 若主角、明敌、重要角色只能回答“谁先压住别人 / 谁落字 / 谁护住说法 / 谁传话”，一般这项就先别急着说够了
- 本章至少该有 `1` 个关键硬句位置：
  - 可以是先把人压住的话
  - 可以是把人往后逼的话
  - 可以是把对方话头截断的话
  - 但得长在本书现场里，别写成写作标签句
- 若该句其实只是在解释“这是第几次收益/第几次反应/最重要的是哪一步”，一般就不算关键台词，只算作者替读者收尾
- 若该句采用 `最重要的不是……而是…… / 先乱的不是……而是…… / 真正重要的是……` 这类旁白直接替现场收话的句式，一般先按作者句处理；除非它同时落成该人物当场说出口并直接改了别人动作，否则别记为人物句
- 若题材允许走冷口判断句，优先检查该句是否同时压住：
  - `人物自己的私利`
  - `人物愿意承担的代价`
  - `现场被迫改变的现实对象`
  三项里至少两项；若都没有，只是口硬或句狠，一般仍不算这个人自己的硬句
- 若该句判为主角人物句，还要继续追问：
  - 删掉角色名后，读者能不能仍认出“这是主角会说的话”
  - 这句有没有只属于主角的 `看事角度 / 张口就会带出来的比喻 / 会抓的东西`
  - 这句是否仍在复用上一轮那套 `你手一X / 我不认A / 不是我算` 旧走法；若是，只能写“这句还管用”，别直接抬成更高一档的人物硬句
  - 若当前任务本身就是“第二套/第三套句法走法单独练习”，还得继续追问：这句是否只是上一轮那套走法换名词重写；若是，仍不能判成新一层
  - 若答不出，只能说明它有效推进了场面，还不足以算这个人自己的硬句
- 若当前主测是“主角自己的味道”，还得继续追问：
  - 这句前面的看事办法是否只属于这个主角，而不是任何能干活的人都能套
  - 这句背后最怕失去的那样东西，是否已经被正文前后文真实托住
  - 这句或这句前后的判断，是否已经给“将坏未坏”的状态起出只属于主角的名字
  - 这个名字是否绑定本题材现场里的具体东西，而不是抽象同义替换
- 若只有“固定顺序 + 旧伤动作”，没有更明确的私人失去物，一般仍只判“没那么像只会办事的人了”，别直接判“主角自己的味道已经出来了”
- 若该句属于重要角色，还得继续追问：
  - 这个角色是否先把自己最想护的人 / 物 / 名分压进了那套说法里
  - 压进去之后，她/他自己到底失去了哪一项实际位置 / 先后机会 / 面子 / 名分
  - 这项失去是否当场可见，而不是只停在角色设定说明
  - 若没有，只是站在高位替主角把话说出来，一般仍按“替人传话的办事台词”处理，不算重要角色人物句
  - 若只写了“先压自己人”这一步，但压完没有真实代价或现场被动改局，一般仍只算“比传话台词厚一点”，别判成 `读者能顺手想出这人的样子`
  - 若已经写到“先伤自己一下，后面连身边人都不再替她/他说话”这层，也要继续追问这些后果是不是只会长在这个人身上；若只是把“丢权 + 身边人开始不替她/他说话”硬套到任意高位角色身上，一般记为“事是对的，但这个人还是不够像活人”
  - 若已经写到这层，还要继续追问她/他真正没丢掉的那样东西是否已经露头；若正文答不出 `秩序 / 手感 / 她/他真正还能压住别人的本事 / 她/他手里真正还能拍板的东西` 中任一项，一般仍只算“写对了”，别往上吹
- 若角色状态 / 写后检查只能回答“谁先压住别人 / 谁落字 / 谁护住说法 / 谁传话”，答不出人物自己的私心、习惯动作或风险代价，一般人物出场分量先别急着说够了
- 若高潮兑现主要靠：
  - 主角宣布
  - 旁白总结
  - 记录口背书
  - 新说法真的写进了记录里
  则还得继续追问是否已有“别人被迫承认的那一下”；没有就别写成质量很高
- 若正文直接出现这些词，一般本项直接不过，别用“测试样本”豁免
- `解释腔 / 作者下结论句` 仍需记录，但一般转入后置补看，不作为首写阶段主否决项

### 6. 方法资料到底有没有真用上

- 冷启动测试别只证明“资料库存在”
- 要显式记录本轮实际调用的资料，例如：
  - `genre-writing-formulas.md`
  - `style-commercial-theory.md`
  - `hooks-chapter.md`
  - `hooks-suspense.md`
  - `dialogue-mastery.md`
  - `character-design-methods.md`
- 每份资料至少回答：
  - 用来补哪一个风险
  - 最终落成了正文里的哪一个效果
- 若答不出“落成了哪一个效果”，一般只算读过，不算真正用上
- 若所有资料都被写成“已落地且有效”，但正文和复审仍只能证明其中 `1-2` 项，一般记为“方法资料写得太过头”
- 若首写连续多本都出现“读得多、真正写进去的少、起作用的更少”，一般说明当前 skill 还没把方法资料转成能直接写的东西

### 7. 提纲里的词别直接倒回正文

- 以下词汇只允许停留在写作说明、复核记录、追踪文件里，别进正文原句：
  - `第二层反压`
  - `第三方迟疑成本`
  - `直接把模板步骤喊出来`
  - `一章里把模板效果全写死`
  - `模板刚起手就立刻全生效`
  - `容易露馅的句子`
  - `容易露馅的一拍`
- 若正文出现上面任一词汇，这份冷启动样本就直接算“首稿还没改成人话”，别写成冷启动写成了。
- 这条主要看“这些词有没有真的翻成故事动作”，不是看“有没有照着写法做”。

## 五、首章组厚度不够时先这么看

冷启动首章组碰上以下任一项，一般视为 `厚度不足预警`：

- 单章低于 `3400`
- 低于细纲目标 `120` 字以上
- 虽过脚本，但正文只有 `1` 口外部反应
- 虽过脚本，但中段没有争先 / 争位 / 争顺序
- 虽过脚本，但章尾前没有最后一拍抬升

优先按这个顺序补：

1. `第二轮收益`
2. `第二轮外部反应`
3. `中段争先 / 争位 / 争顺序`
4. `章尾前最后一拍`
5. `关键角色亮相动作`
6. `有人被挤开 / 第二步先手被抢走 / 名分开始起作用`

下面这些补法别用：

- 加解释
- 加世界观说明
- 加主角总结
- 重复第一爆点换句说

边界题材补厚度实答制：

- 若题材属于以下任一类，首章组厚度回补后还要额外实答：
  - `女频强情绪`
  - `历史争霸 / 王朝权谋`
  - `恐怖惊悚 / 民俗悬疑`
  - `无限流 / 规则副本流`
- 要明确回答：
  - 哪一项实际位置被切走
  - 哪一步的第二步先手被抢走
  - 哪一项名分 / 名册 / 名单开始起作用
- 若只能回答“气氛更强 / 大家更急 / 局势更乱”，一般视为无效补厚度
- 但这一项一般只作为后续补写提示，不作为首写阶段主否决项

## 六、三本样本回头一对，最容易看出来的是

对着三个已经跑过的冷启动样本回头一看，首章组最稳的规律是：

1. 这套路子不能只进细纲，要进正文事件链
2. 第一爆点后要立刻有第二轮收益
3. 至少两次外部反应，且一次在中段改线，一次在章尾前往前挤一步
4. 冷启动首章组如果只写到 `3100-3200`，大概率仍然偏薄
5. 追踪文件和章组汇总对不上，就别说 skill 这一轮已经顺了
6. 冷启动首章组若漏了 `设定/角色/` 关键人物卡，即使正文还能读，也最多只算“过了一半”

## 七、冷启动首章组收口记录格式

- 冷启动范围：`第001-00X章`
- 细纲能不能直接拿来写：`是 / 否`
- 第一爆点是否落地：`是 / 否`
- 第二轮收益是否落地：`是 / 否`
- 两口外部反应是否落地：`是 / 否`
- 章尾前一拍是否落地：`是 / 否`
- 正文字数够不够：`是 / 否`
- `scene_lint.py --json`：`ok / fail`
- `validate_tracking_state.py --json`：`ok / fail`
- 追踪文件是否对上：`是 / 否`
- 章组汇总是否对上：`是 / 否`
- 最后怎么记：`冷启动写成了 / 只成了一半 / 还没成`

## 八、回头看旧样本时再补这一层

如果用户不是要继续写正文，而是要“拿旧冷启动样本回头看写法规律、回灌 skill、回头试一遍”，那首章组这一步还得再拆成两层：

### A. 这些东西到底有没有写出来

只检查有没有写出这些内容：

- `本章主写法`
- `第一爆点`
- `第二轮收益`
- `群体反应位`
- `第二轮外部反应位`
- `章尾前一拍抬升位`

### B. 写出来以后到底有没有真起作用

还要继续检查这些内容有没有真的在正文里起作用：

- `第二轮收益` 是否真的改掉现实局面
- 两口外部反应承担的事情是否不同，而不是同义重复
- `章尾前一拍` 是否能单独成立，不是直接给章尾垫解释
- 本章收益到底停在 `开门 / 拿到资格 / 名字先挂上去 / 这份好处已经当场起作用 / 好处已经真的落到手里` 哪一层

先看清：

- 写到了纸面上，不等于真的起作用了
- 旧样本如果只是“写法名字都列齐了”，但收益到底停在哪一步答不稳，最多只记成“只成了一半”
- 只有当“写出来了 + 正文里真起作用了 + 收益停在哪一步说得清 + 追踪也接上了”同时成立，才算“这轮冷启动真写成了”

## 九、拿冷启动做压力测试时再记这几条

如果当前任务是“拿 skill 直接冷启动不同题材新书，看可读性和留存这套东西能不能首写就落地”，那除了这张清单，还得额外记：

- 哪些坑是首写时主动绕开的
- 哪些坑首写没绕开，但 review 稳定抓到了
- 哪些坑首写没绕开，review 也漏掉了

其中第三类一般就算“skill 这条还没站住”，别急着说这条已经磨顺了。
