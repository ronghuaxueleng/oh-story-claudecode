# 单项基线样本怎么搭

> 回看链已经恢复，但还缺“更纯的单项负例/正例样本”时，就按这套搭法来，免得每次都临时想怎么故意做出某类问题。

---

## 一、当前优先补的单项纯样本

现阶段优先级最高的两类：

1. `非情报流误触纯样本`
2. `本组第二章偏短链纯样本`

原因：

- 这两类已经是“模板方向对了以后”的高频真问题
- 比“模板没用上”更常见
- 比“厚度不足”更该拿来当模板验证态主判项

---

## 二、非情报流误触纯样本

### 目标

专门验证：

- `旧码 / 错单 / 复核口 / 广播补条 / 规则播报 / 挂签`
- 只是信息场里的现场东西
- 不是正式情报流正文

### 建样要求

- 题材优先：`都市商战 / 职场脑洞 / 规则副本 / 悬疑断案`
- 正文必须有信息凭据，但禁止写成独立情报块
- 第001-002章都必须借这件东西改现实说法、动作或谁站哪儿
- 尽量避免：
  - 严重 `字数.error`
  - 明显 `章尾.error`
  - 大量 `解释.error`

### 复审成功标志

- `story-review full` 能明确写出：
  - `非情报流误触候选`
- 主线程得能说清：
  - 这是现场冲突靠着走的具体东西，不是正式情报文本

### 额外执行规则

- 这类样本复审时，不得只看 `scene_lint.py` 是否先报了 `情报.error`
- 若脚本先把正文打到 `情报` 说法，主线程还得继续做语义复核：
  - 它到底是在发情报
  - 还是在借 `旧码 / 错单 / 播报 / 挂签 / 补条` 改现场先后
- 只有确认正文真的存在“独立情报发布 / 情报刷新 / 情报兑现链”时，才可按正式情报流收尾
- 若正文本来就是现场冲突靠着走的具体东西，应优先保留：
  - `非情报流误触候选`

---

## 三、本组第二章偏短链纯样本

### 目标

专门验证：

- 模板主线已经进正文
- 但本组第二章自然压缩成偏短链

### 建样要求

- 第001章必须相对完整，能证明模板已落地
- 本组第二章要故意逼近：
  - 主角再顶一句
  - 对手回一句
  - 重要角色再往下压一句
  - 记录口直接落签
- 本组第二章中间故意少放：
  - 第三方被动动作
  - 外部反应扩散
  - 群体谁前谁后变化
  - 次级收益扩场
- 不要再混入重型情报流、污染、解释腔问题

### 复审成功标志

- `story-review full` 能明确写出：
  - `本组第二章偏短链`
- 同时主线程还得说清：
  - 模板已用
  - 问题在事件链缩断，不在模板缺席

---

## 四、现在建样先按这个来

单项纯样本的关键，不是把问题做大，而是把问题做纯：

- 尽量让一个主问题占据主结论
- 尽量不要让 `字数 / 章尾 / 解释 / 污染` 抢走结果判断
- 若必须带轻度并发问题，优先只保留 `warn`

当前若用户任务属于：

- `模板验证`
- `skill 冷启动测试`
- `story-review full agents 全量检测`

那主线程建样与验样时，就先围绕：

1. `收益写得太满了`
2. `非情报流误触`
3. `本组第二章偏短链`

组织样本，而不是先围着厚度做样本。
