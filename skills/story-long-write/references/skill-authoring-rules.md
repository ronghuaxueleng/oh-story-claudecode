# Skill 编写约束

> 这份只管 `story-long-write` 自己后续怎么维护、怎么扩写，不直接参与小说项目写作流程。

---

## 一、目的

这份主要是防止 `story-long-write` 在后续维护里又长回这些毛病：

- 通用主干夹带现成桥段句胚
- 某一题材的额外检查伪装成默认规则
- 单一本书的案例、人名、地名、事件被误写进通用模板

---

## 二、通用主干约束

凡属于以下任一层，默认视为 `通用主干`：

- `SKILL.md` 中的主流程与总规则
- `references/` 中面向所有题材的共用模板
- 写前闸门 / 写后检查 / 细纲模板 / 标题模板

通用主干必须遵守：

1. 只写功能，不写现成桥段句胚。
2. 只写判断维度，不写可直接照抄的标题句或正文句。
3. 只写动作模板，不写单一本书的场景组合。
4. 只写抽象结构，不写特定书名、人名、地名、势力名。

允许：

- `某条规矩 + 被改写`
- `某个动作 + 被打断`
- `某件关键物 + 公开见光`
- `某个身份判断 + 被改写`

不允许：

- 可直接复制进正文或标题的整句
- 绑定具体道具、地点、组织、职业的桥段胚
- 以单本项目案例冒充通用模板

---

## 三、题材单独规则约束

凡规则只对某些题材、平台、读者口味成立，必须明确标记为 `题材单独规则`，不得伪装成默认主规则。

题材单独规则必须写清：

- 触发条件
- 适用范围
- 不适用时的处理方式

推荐写法：

- `若题材碰上以下额外条件中的任意两项...`
- `若存在超凡 / 异象 / 最主要能力...`
- `若碰上信息差 / 情报 / 预知型结构...`
- `若平台偏好强爽快节奏...`

禁止写法：

- 把单一复合题材直接写成所有书默认都得这么走
- 不写触发条件，直接写“必须如此”
- 只给某题材示例，却不声明这是单独规则

---

## 四、示例使用约束

若确实需要示例，优先级如下：

1. 功能提示
2. 抽象模板
3. 中性示例
4. 题材单独规则示例

禁止：

- 直接用当前在写项目的人名、地名、事件名
- 用一句现成标题或桥段充当规则本体
- 用单独规则示例覆盖通用模板

题材单独规则示例必须额外写明：

- `以下属于单独提醒，不是所有题材都要照抄`

---

## 五、修改前自检

对 `story-long-write` 任一规则文件做新增或重写前，至少先问：

1. 这条规则是通用主干，还是题材单独规则？
2. 如果删掉当前项目语境，这条规则还成立吗？
3. 这段内容是在给判断维度，还是在偷偷提供现成句胚？
4. 这段内容会不会把某一本书的写法推成默认模板？

若以上任一题答不稳，先降一级抽象，再决定是否写入。

---

## 六、收文前检查

新增或修改 `story-long-write` 后，至少检查：

- 是否出现具体书名 / 人名 / 地名 / 势力名
- 是否出现可直接照抄的标题整句
- 是否出现可直接照抄的正文桥段整句
- 是否把题材单独规则写成默认规则
- 是否把平台偏好写成唯一审美

碰上任一项，一般先别收进正文，先回修。
