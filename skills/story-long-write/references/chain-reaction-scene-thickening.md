# 连锁动作拍增厚法

这页就是给 `story-long-write` 用的。
它专门解决“动作拍已经补了，但链条还是太短”这件事。
冷启动首稿、回炉把主线理顺、边界题材回归验证时，都能拿来用。

---

## 一、这份是补什么的

`scene-thickening-method.md` 解决的是：
- 先把拍子补出来
- 让正文不再只剩主线往前赶

这份解决的是：
- 拍子虽然补了
- 但每一拍还是各自独立
- 于是读感变成“多了几拍”，而不是“场面自己往下滚”

一句话判断：

- 单拍增厚：`谁动了一下 -> 谁只好跟着变`
- 连锁增厚：`谁做了什么 -> 谁被迫怎么变 -> 又逼第三方多变一步`

如果一拍写完，场面就停住了，这拍还是偏短。

---

## 二、什么时候该翻这份

碰上下面任一情况，就先翻这份：

- `scene-thickening-method.md` 已经用了，正文动作拍仍显并列发散
- 正文已经写出 `3-5` 拍新增动作，但读感仍像“排着写完”
- 能答出“动到了谁”，却答不出“这一下又逼谁继续变”
- 中后段没有断，但每一拍之间缺少咬合，像并列片段

注意：

- 别顺手把这份当成“厚度兜底工具”
- 若目标是把章节直接抬过字数线，别只靠这份

---

## 三、这页现在主要就是补链，不包厚

真实回归样本已经证明：

- 连锁动作拍能把事件链写清
- 能把并列拍拉成传导拍
- 但不能稳定把章节抬过厚度线

所以这份现在主要解决的，不是“厚度不足”，而是“链条太短、主线太平”。

薄章里常见的链短问题是：

1. 只改主角和对手
- 主角出手
- 对手把刚才那话吞回去
- 然后就结束

2. 只有结果，没有扩散
- 主验签翻过来
- 账本落下去
- 名册改了
- 但第三方、围观人、执行次序没有跟着变

3. 只有争先，没有传导
- 有人吵起来了
- 但这场争执没把顺序、前后位置、出手法子带着变

现在更稳妥的说法是：

- 第一层问题：没有拍
- 第二层问题：有拍但不传导
- 第三层问题：传导有了，但长度还是不够

这份主要解决第二层，第三层能不能顺带补上，不包。

真实有效的连锁，通常至少要连到第二层，最好连到第三层：

- 第一层：主角先碰到对手
- 第二层：对手变化逼第三方也变
- 第三层：第三方往前一挤或往后一退以后，现场规则或章尾结果也跟着变

## 四、连锁动作拍卡

当章节偏薄，且普通扩拍后仍然不够时，别再自由补句，直接写这张卡。

至少补出 `2-3` 条连锁，每条连锁至少写成：

`起手动作 -> 第一层被动变化 -> 第二层被动变化`

推荐格式：

### 1. 本章最该拉长的那一拍

- 原位置：
- 原来只写了什么：
- 为什么这拍不够：

### 2. 连锁补法

- 连锁 1：
  - 起手动作：
- 第一层被动变化：
- 第二层被动变化：
- 连锁 2：
  - 起手动作：
  - 第一被动改线：
  - 第二被动改线：
- 连锁 3：
  - 起手动作：
  - 第一被动改线：
  - 第二被动改线：

### 3. 连锁分别改了什么

- 连锁 1 最终改掉了谁原来的地方 / 先后 / 好处：
- 连锁 2 最终改掉了谁原来的地方 / 先后 / 好处：
- 连锁 3 最终改掉了谁原来的地方 / 先后 / 好处：

### 4. 连锁分别落在哪段

- 前 `1/3`：
- 中段：
- 后 `1/3`：
- 章尾前：

### 5. 先看哪条连锁最重要

- 最主要的连锁：
- 为什么：

若写到第二步就断掉，说明这条连锁还没立住。

## 五、最常见的三种连锁方式

### 1. 争先连锁

一般用在：
- 中段已经有人争先
- 但争完以后什么都没改

补法：
- 有人抢先发言
- 逼另一个人把原话缩回去
- 再逼记账的人 / 执行次序 / 围观的人改掉下一步

要补到的结果：
- 让“吵起来”变成“现场顺序真变了”

### 2. 改线连锁

一般用在：
- 已经有第三方反应
- 但第三方只是表态，没有动手

补法：
- 第三方先换手
- 逼旁边的人往前挤一步 / 往后退一步 / 改顺序
- 再让章尾结果变得能落下去

要补到的结果：
- 让“第三方下场”变成“现实线路被挪了”

### 3. 承压连锁

一般用在：
- 章尾最后一拍本身没问题
- 但前一拍压力不够，章尾显轻

补法：
- 对手先抢第二手
- 第三方先犹豫或卡一下
- 主角或协作方逼出最后那一下落笔 / 落名 / 落令

要补到的结果：
- 让章尾结果不是直接给，而是顶着一串阻力砸下来

## 六、不同题材里，连锁该怎么长

### 1. 升级 / 宗门 / 资格争夺

- 主角抢先手
- 对手被逼得换了前后位置
- 办事次序被改掉
- 谁先谁后开始真挡人

### 2. 古代断案 / 审讯 / 官场说法

- 主角翻出一口真伤 / 真证
- 老资格先把原话收回去或先翻错
- 书吏 / 差役 / 主验口被迫改手
- 名册 / 签簿 / 验法顺序因此改变

### 3. 都市异能 / 事故处置

- 主角先指出异常
- 旧执行次序不敢再按原流程做
- 封锁线 / 调度口 / 记录口被迫改令
- 处置权因此换手

### 4. 情绪拉扯 / 婚恋关系

- 一方先把话咽回去
- 另一方先失态或退半步
- 旁观者 / 家人 / 第三者因此往前站、往后退，位置变了
- 关系名分或相处权力发生现场变化

---

## 七、怎么判这次补的是不是真厚

补完后要能答上来：

1. 每条连锁有没有至少 `3` 步
2. 第二步是不是被第一步逼出来的
3. 第三步是不是又把现场结果往前推了一步
4. 删掉整条连锁后，章尾或中段是否明显发空

第 `2-3` 项要是答不稳，就还不算连锁增厚成了。

---

## 八、这时候怎么用最合适

按现在实测下来的结果，这样分工更顺：

- 先用 `scene-thickening-method.md` 补出基础动作拍
- 若读感仍像并列片段，再用这份把其中最重要的 `2-3` 拍拉成长连锁

别一上来就把全章都写成长连锁。
优先拉长：

- 中段最容易一笔带过的争先拍
- 章尾前最容易直接跳过去的承压拍
- 第三方已经下场但还没有真改线的那一拍

这份方法不是为了“把每章都写厚”，而是为了“把关键那几拍带下去，让事情更清楚”。

## 九、现在先别越过这条线

基于 `裂命观星台` 等真实回归样本，现在能看清：

- 这份已经能让 `压住局面 -> 当场炸开 -> 逼人让路`、`贴上牌子 -> 被挤到边上 -> 名单挂出来` 这类连锁进正文
- 但它没有把章节稳定抬过脚本厚度线
- 所以后面别再把“已经补了连锁动作拍”直接当成“厚度问题已经解决”

后面先按这个理解：

- `scene-thickening-method.md`：用来补基础动作拍
- `chain-reaction-scene-thickening.md`：用来把关键传导写清

两者都不是最终厚度兜底工具。
