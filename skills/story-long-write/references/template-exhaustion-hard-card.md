# 这套写法有没有真的写透

这页就是给 `story-long-write` 用的。
- 它专门解决“这套写法已经进了正文，但还没写透”。
- 首稿前、首稿后自检、写后检查这三处都能用。
- 它不替代 `章组收益卡`，就是盯住“这套写法到底有没有真把现场改掉”。

---

## 怎么用

- 只要本章显式调用了某套写法 / 情节库 / 事件母板，就要填写
- 别写“已经用了”“已经进正文了”这种空话
- 每题都要回答到 `具体人 / 具体物 / 具体动作 / 具体后果`

---

## 必填项

- 本章主写法：
- 本章这一套要改掉的旧结果：
- 本章这一套抢到的新结果：
- 本章第一硬动作：
- 第一硬动作后谁立刻被挤开：
- 第一硬动作后谁立刻得利：
- 第二硬动作：
- 第二硬动作后谁被迫改动作：
- 第三方不是立刻选边，而是先迟疑在哪一步：
- 对手第二手还会怎么压回来：
- 本章最主要的现实变化：
- 若删掉最后一次查证 / 表态 / 认物，本章还剩什么现实结果：
- 本章最容易写成黑话的句子：
- 这句必须改成哪一个现场东西：

---

## 一看就知道还没写透的地方

- 若 `谁被挤开 / 谁得利 / 谁改动作` 三项任一答不出，默认这套写法还没写透
- 若“最主要的现实变化”只能答成 `大家都知道了 / 更重视了 / 更紧张了`，默认这套写法还没写透
- 若删掉最后一次查证后整章就空掉，默认就是“查证链在替这套写法撑场面”
- 若最容易冒出的句子属于：
  - `这就是第二轮收益`
  - `这一下最该落地`
  - `真正重要的是`
  - `现在争的已经不是……而是……`
  默认先记“这套写法里的黑话又冒回来了”

---

## 最后只看一句

- 不看这套写法的名字有没有进正文
- 只看它有没有连续改掉：`谁被挤开 / 谁说不下去 / 谁改动作 / 谁没了退路`
