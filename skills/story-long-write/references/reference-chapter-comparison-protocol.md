# 参考章节对比要怎么做

碰上这些情况就翻这份：
- 用户明确要求“使用原文做参考写一版”
- 用户要求“和现稿对比一下”“看看差在哪”
- 用户要求判断“为什么参考原文味道更足/现稿更稳”

这份主要想办的事：
- 把参考原文的作用卡在“对着写法比一比”，别让参考章直接接管本章事件顺序
- 产出一份单独放着的“参考驱动版”和一份对比单
- 结论得能指导后面怎么吸收，别只给“像/不像”这种空话

## 先把这些读了

开始前先读：
1. 项目内 `参考书使用边界.md` 或同类边界说明
2. 本章 `参考输入卡`
3. 本章 `单章写前卡`
4. 本章现稿
5. 对应参考原文章节或对应段落

先把边界说清：
- 参考原文只用于对着看压住场面的方式、群像抬轿、句式推力、章尾吊法
- 本章事件顺序、书名主打兑现、人物决策链，仍以当前项目自己的大纲和写前卡为准

## 这些别干

- 别直接覆盖现稿
- 别把参考原文章节顺序直接平移成本章正文
- 别因为参考章更强，就把本章原本该扛的书名主打和伏笔活直接跳过去
- 别只输出“更像原文”“不够有感觉”这类发虚的评价
- 别把参考驱动版误当成最终定稿，除非后面已经明确要回炉并重新检查

## 一步一步这么做

1. 先按边界对清：这轮能借的是推进骨架、写法层、局部压场手法；不能借的是设定结论、关键事件顺序、角色命运落点。
2. 读取本章 `参考输入卡` 与 `单章写前卡`，看清这章一定得完成哪些事件、人和章尾上的事。
3. 读取参考原文对应章节，只抽能迁过来的写法：
   - 压住场面的节拍
   - 群像抬轿方式
   - 信息抖落顺序
   - 句式冲击方式
   - 章尾钩子吊法
4. 单独写一份“参考驱动版”：
   - 另存为比对样本，别覆盖现稿
   - 保留本项目自己这章该完成的事和人物活
5. 输出“现稿 vs 参考驱动版”的对比单。
6. 给出吸收建议：结论只落“局部吸收”“局部重写”“整章回炉”这三类，别含糊收尾。

## 对比时主要看这几层

### 1. 情节层
- 现稿是否更清楚地完成本章任务
- 参考驱动版是否只是更热闹，但牺牲了本章事件精度
- 哪些推进动作可以吸收，哪些不能照搬

### 2. 压场层
- 谁在压住场面
- 压住场面这一下是否有铺垫、递进、落点
- 爽点是一次性砸出，还是有前摇、有停顿、有见证人放大

### 3. 群像层
- 配角是否只在报信息，还是在给主角抬轿
- 旁观者、对手、同伴三类视角是否形成共振
- 场内反应是否在持续推高主角存在感

### 4. 语言层
- 句子是否只有信息，没有劲道
- 是否缺少短促压句、重复锤击、递进断句、现场感动作词
- 哪些表达可以吸收成“写法动作”，哪些属于原文高相似风险区

### 5. 章尾层
- 现稿章尾是收死、收平，还是留钩
- 参考驱动版章尾强在哪里：悬念、反差、兑现后再逼人把原话吞回去
- 更建议吸收的是章尾怎么收，还是章尾那股措辞节奏

## 输出就按这个来

输出就按下面三段来：

### 现稿更稳的点
- 写清哪些部分更符合本项目既定大纲、人物链、伏笔链

### 参考驱动版更强的点
- 写清强在压住场面、群像、语言还是章尾，别笼统说“更有味道”

### 处理建议
- 保留什么
- 吸收什么
- 不建议覆盖什么
- 结论得明确写成：`挑着用 / 局部重写 / 整章回炉`

## 最后这么判断

- 如果参考驱动版只是“更像参考书”，但把本章自己的走法带坏了，就别判它更好
- 如果现稿走法是对的，但压场力度、群像、语言明显发虚，就先判“挑着用”
- 只有当现稿在情节、人、章尾这三层都失守时，才建议“整章回炉”
