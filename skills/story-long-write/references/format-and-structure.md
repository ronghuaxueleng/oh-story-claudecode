# 短篇格式怎么排，小节怎么切

> 这里整理的是仓库里当前默认的正文排法；用户或平台另有要求，就按用户或平台来。
>
> **这份一般管到哪**：段落格式（一句一段、≤60字）和对话格式，各题材都能用。小节（beat）怎么切和字数要求只管短篇。长篇字数还是按当前 skill 的硬约束走。

---

## 章节标记

常用格式按常见程度排在下面：

| 格式 | 一般用在哪 | 示例 |
|------|----------|------|
| `###1.` | 黑岩、点众（主流） | `###1.` `###2.` `###3.` |
| `###第一章` | 部分平台 | `###第一章` `###第二章` |
| `1.`（纯数字） | 知乎 | `1.` `2.` `3.`（无 ### 前缀） |

别混着来，全文统一一种。短篇一般用 `###1.` 或纯数字就够了。

---

## 段落怎么排

### 先守这条：一句一段

默认排法就是**一句一段，紧密排**。

- 每段 1-3 句，不超 60 字
- 段与段之间**不加空行**（紧密排列）
- 无缩进（平台渲染器自行处理，不需要 `　　` 或空格）
- 一段只做一个动作：叙事/对话/心理/动作，不混合
- 三维度织入后仍要按镜头断段：新动作、新物件、新信息、新对话另起一段，不拆成动作层/感知层/反应层

### 段落节奏

连续短段（1-2 句）更容易带紧张感。偶尔给一个 3 句段当换气点，但别放出超过 60 字的大段。

---

## 对话怎么排

### 对话标记

先看平台和用户要求；没特别指定时，就用默认格式：

| 顺手程度 | 格式 | 一般用在哪 |
|--------|------|----------|
| 首选 | `"说话内容"` | 黑岩、点众、番茄（主流） |
| 次选 | `「说话内容」` | 部分古言、日式 |
| 平台指定 | `【说话内容】` | 仅知乎盐言故事 |

一般直接用 `""`；只有用户明确说知乎平台时，再换成 `【】`。

### 对话怎么排更顺

1. 对话**独立成行**，不嵌在叙述段落中
2. 用动作代替对话标签：不用「他说」「她道」，改为动作描写 + 对话
3. 两人对话连续出现时，省略标签，靠内容区分说话人

**正确示例**：
```
她把杯子放下。
"你走吧。"
他没有动。
"我说，你走吧。"
```

**错误示例**：
```
她把杯子放下，说道："你走吧。"他没有动，她又说："我说，你走吧。"
```

## 小节（beat）怎么切

### 常见切法

- 用数字编号（`1` `2` `3`）分小节，每个小节都得是一个完整叙事 beat
- 每小节 800-1500 字（爽文可压缩至 500-800 字/节，见 `genre-writing-formulas.md`）
- 8-15 个小节覆盖全文（按目标字数反推；8000÷1000≈8节，12000÷800≈15节）
- 每小节推进一个明确的小情节

### 小节里最好带上的东西

每个小节一般都要带上：

1. **一个主事件** + **3-5 个子事件**（主事件推进当前主要情节，子事件丰富层次；初稿偏短时先回到小节/细纲补足计划内子事件、对话或冲突，再写正文；去AI味已有正文时别新增剧情）
2. **一个情绪变化**（读者的感受如何变化）
3. **一条读者新获知的信息**
4. **3-5 轮对话交锋**（揭示关系/升级冲突，参考 `writing-craft.md` 里对话怎么一来一回顶起来。特定场景如独自发现/翻阅日记可标零，按当前 skill 的细纲/设计任务要求执行）
5. **每个子事件三维度织入**：发生+感知+反应织入同一段正文（参考 `writing-craft.md` 场景写法）


### 小节之间怎么接

- 小节结尾留一个钩子（悬念/未解决的情绪/新问题）
- 下一节开头快速接续，不要重新铺垫
- 情绪跨节递进：每一节的情绪强度 ≥ 上一节。例外：峰值情绪（反转节）后可以维持 1 节不降，但别骤降

## 各平台对话格式对照表

| 平台 | 章节标记 | 对话格式 | 特殊要求 |
|------|----------|----------|----------|
| 黑岩 | `###1.` | `""` | 无 |
| 点众 | `###1.` | `""` | 无 |
| 知乎盐言 | `1.` | `【】` | 导语需单独标注 |
| 番茄 | `###第一章` | `""` | 首段需有吸引力 |
| 红果 | `###1.` | `""` | 无 |

**没特别说明时就这么用**：直接走黑岩/点众格式（`###1.` + `""`）。

## 5 条别犯

下面这几条一直都算数，不因为题材或风格变：

1. **别留大段落**：超过 60 字的段落就拆开
2. **禁止段间空行**：段落紧密排列，不留空行
3. **避免对话标签机械化**：高频或公式化的「他说」「她道」「他笑了笑说」用动作/上下文替代；普通“说”可保留
4. **禁止缩进**：不使用 `　　`（全角空格）或半角空格缩进
5. **禁止正文段落 Markdown 渲染**：除统一的小节/章节标记（如 `###1.`）外，正文段落中不使用加粗 `**`、斜体 `*`、标题 `#`、分隔线 `---` 等 Markdown 语法

## 写完后顺手对一遍

写完后顺手对一遍：

- [ ] 每段 ≤ 60 字？
- [ ] 三维度织入段是否按镜头/信息变化断开，而不是一段到底？
- [ ] 段间无空行？
- [ ] 对话独立成行且用正确引号？
- [ ] 对话标签是否避免高频/公式化，普通“说”是否按语境保留？
- [ ] 无缩进？
- [ ] 每个小节有事件+情绪+信息？
