# 核心门禁

> 本文件收束 `story-long-write` 的最高优先级硬规则。进入任一写作阶段前，先读本文件，再按场景加载其他 workflow / quality / reference 文件。

---

## 一、统一失败码

| 失败码 | 含义 | 强制动作 |
|------|------|---------|
| `F0` | 未命中失败项 | 可进入下一步 |
| `F1` | 句子层失败：空效果句 / 解释腔 / 局部 AI 味 | 当场回修，复扫通过前不得结束本章 |
| `F2` | 章节层失败：章尾不过线 / 场面不足 / 战果偏虚 | 回退到本章细纲或本章正文重写，禁止进入下一章 |
| `F3` | 机制层失败：题面滑轨 / 连续多章只查不爆 / 副驱动反客为主 | 停止顺写，回退到章节组细纲或卷结构 |
| `F4` | 污染失败：工具残片 / 补丁标记 / 执行元话语 / 非故事语料 | 先清污染，再全文复扫 |
| `F5` | 流程失败：角色卡 / 追踪 / 写前闸门 / 验收单缺失或假完成 | 先补流程文件，补完前本次工作不算完成 |

规则：
- 写前闸门、正文写作、写后验收都必须产出 `F0-F5`
- 最终失败码不是 `F0`，不得并入主正文，不得进入下一章
- 不得用“基本可用”“后续再补”替代失败码裁决

---

## 二、生产顺序

正常生产顺序固定为：

`核心设定 -> 题材识别+专项挂载 -> 卷纲 -> 单章细纲 -> 细纲过线 -> 正文 -> 写后验收 -> 追踪/角色卡同步`

规则：
- 细纲不过线，不得写正文
- 正文不过线，不得并入主正文
- 追踪 / 角色卡未同步，视为本次工作未完成
- 不允许先写正文，后补细纲 / 追踪 / 验收

## 四、写前硬闸

进入细纲或正文前，必须先完成：

1. 题材识别结果落盘
2. 专项 checks 挂载完成
3. 本章 / 本组主驱动明确
4. 本章最值钱战果明确
5. 写前闸门落盘

若任一项缺失，按 `F5` 处理。

写前闸门至少必须包含：
- 题材识别结果
- 本章唯一口径
- 本章主驱动
- 本章副驱动
- 本章最值钱战果
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 本章章尾钩子类型
- 本章专项失败风险
- 预判失败码

---

## 五、写后硬闸

正文落稿后，至少完成以下动作才允许判定完成：

1. 运行 `scripts/scene_lint.py <正文文件>`
2. 做正文纯净度检查
3. 做章尾人工回看
4. 生成写后验收
5. 同步追踪与角色卡

写后验收至少必须包含：
- 场面数
- 外显战果
- 章尾判型
- `scene_lint` 结果
- 纯净度结果
- AI味分级
- 角色卡同步结果
- 追踪同步结果
- 口径反校验结果
- 最终失败码
- `可并入主正文 / 不可并入主正文`

---

## 六、章节完成裁决

章节完成裁决只能写二选一：
- `可并入主正文`
- `不可并入主正文`

只要出现以下任一情况，默认 `不可并入主正文`：
- `scene_lint.py` 未通过
- 正文纯净度未通过
- 章尾未过线
- 外显战果未成立
- 场面兑现未成立
- AI味达到中度以上且未完成衔接
- 追踪同步未完成
- 角色卡同步未完成
- 口径反校验未通过

---

## 七、回炉与执行计划

命中连续回炉 / 施工单 / 验收单时，必须先：
- 生成固定验收单或执行计划
- 明确处理范围
- 明确施工顺序
- 明确完成定义

未生成执行计划前，不得直接改正文或改追踪。
