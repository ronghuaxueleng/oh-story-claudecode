# 最先拦这些

> 这一页汇总 `story-long-write` 里最先要拦住的那些问题。进入任何写作阶段前，可先看这页，再按场景补读别的文件。

---

## 一、失败码先统一

| 失败码 | 含义 | 强制动作 |
|------|------|---------|
| `F0` | 没碰上失败项 | 可进入下一步 |
| `F1` | 句子层失败：空效果句 / 解释腔 / 局部 AI 味 | 当场回修，复扫通过前不得结束本章 |
| `F2` | 章节层失败：章尾没写出来 / 场面不足 / 战果偏虚 | 回退到本章细纲或本章正文重写，禁止进入下一章 |
| `F3` | 整层问题：书名主打开始写偏 / 连续多章只查不爆 / 副线反客为主 | 停止顺写，回退到章节组细纲或卷结构 |
| `F4` | 污染失败：工具残片 / 补丁标记 / 执行元话语 / 非故事语料 | 先清污染，再全文复扫 |
| `F5` | 流程失败：角色卡 / 追踪 / 写前闸门 / 那张检查单缺失或假完成 | 先补流程文件，补完前本次工作不算完成 |

先按这几条来：
- 写前闸门、正文写作、写后检查三处都得产出 `F0-F5`
- 最终失败码不是 `F0`，这章就先别算完成，也别往下写
- 不能拿“基本可用”“后面再补”顶替失败码判断
- 失败码先写进对应文件，再继续；没写进去就直接算 `F5`
- 若当前任务属于 `冷启动 / 回归验证 / 首章组测试`，并撞上下面任一项，就按 `F2` 当场停下，别再拿“先挂个预警、后面再补”放过去：
  - 正文首稿低于 `scene_lint` 下限
  - “现实挤压补充卡” 未答满
  - `解释腔前置自检卡` 未通过
  - `末尾反删测试` 未通过

## 一点五、先后顺序别乱

长篇写作都按这个顺序往下走：

`S0 资料未齐 -> S1 纲层可写 -> S2 细纲过关 -> S3 写前闸门过关 -> S4 正文可写 -> S5 写后检查过关 -> S6 追踪同步过关 -> S7 允许进入下一文件`

先按这几句走：
- 别跳步骤
- 任一步没有明确写进文件的证据，就还算停在上一步
- 当前这一步的失败码只要不是 `F0`，就别往下走
- 用户说“继续 / 不要停 / 一直往下推”，也不算默认允许跳步
- 若模型说不清自己现在走到哪一步，就按 `F5` 处理，把对应文件补齐
- 若当前范围属于连续 `2-4` 章同一章组，`S6 追踪同步通过` 还得有 `追踪/写后检查_第XXX-XXX章.md` 章组汇总；缺了就先别进 `S7`

再补一句人工接管：

- 若碰上污染残片、文件截断、结构漂移、书名主打开始写偏、章节组打架，直接切入 `H1` 人工接管
- `H1` 不是可通过状态，而是返修状态，碰上后也不要再问是否接管：
  - 先写明接管原因
  - 得写明接管文件范围
  - 得在接管完成后重新回到 `S2/S3/S4` 之一继续走流程
- `H1` 也不能绕过 `F0-F5`
- 若当前任务声明为“纯 skill 冷启动验证”，就用 `H1-A`；`H1-B` 只给生产自动返修态用

## 二、正常生产顺序

正常生产顺序如下：

`主要设定 -> 题材识别+补上对应单独检查 -> 开书总蓝图 -> 卷纲/章组控制 -> 单章细纲 -> 细纲通过 -> 正文 -> 写后检查 -> 追踪/角色卡更新`

照下面拦：
- `大纲/开书总蓝图.md` 缺失，就别进卷纲与细纲
- `大纲/全书章节一览.md` 缺失，就别当成已经有全书总览
- `大纲/事件线一览.md` 缺失，就别当成已经有全局事件线规划
- `大纲/章组控制_第X卷.md` 缺失，就别批量写该卷单章细纲
- `大纲/开书总蓝图.md` 若没有建立关键实体层（人物 / 势力 / 生物 / 植物 / 物品 / 地点 / 规矩 / 事件）的最小资产表，也别进卷纲与细纲
- 细纲没过，正文先别写
- 正文没过，就别收进主正文
- 追踪 / 角色卡未同步，视为本次工作未完成
- 别先写正文，再回头补细纲 / 追踪 / 检查
- 开书后凡作品碰上 `情报流 / 系统流 / 预知流 / 任务流` 任一项，`追踪/情报记录.md` 缺失时，生产顺序就停在 `S0/S1`，首章正文先别进

## 四、下笔前先拦一遍

进细纲或正文前，先把这些都补齐：

1. 题材识别结果写进文件
2. 对应单独检查已补齐
3. `大纲/开书总蓝图.md` 已存在且当前有效
4. 当前卷 `卷纲 + 章组控制` 已存在且当前有效
5. 本章 / 本组主要往前推什么明确
6. 本章最主要收获明确
7. 写前闸门写进文件
8. 若碰上 `情报流 / 系统流 / 预知流 / 任务流`，`追踪/情报记录.md` 已存在且本章对应条目已预填
9. 若本章属于某个连续章节组（默认最近 `2-3` 章），已完成 `章节组闸门`
10. 本章关键角色出场分量检查已完成，至少写清：
   - 本章谁是 `主角 / 主要对手 / 主要协作方 / 重要角色`
   - 这些角色各自当前想要什么
   - 各自当前怕失去什么
   - 各自进场前 `5` 分钟在做什么
   - 各自本章至少 `1` 个独有记忆点怎么落下
11. 若本章存在 `2` 人及以上连续共演，已完成 `删名可分辨预检`
12. 若本章碰上 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，先把 `别把场面写成讲规矩的说明文` 这一步做完：
   - 规矩信息压在哪个在场动作里
   - 本章最重的现实变化是哪一步
   - 哪一段最容易滑成讲解话，以及如何压回场景
13. 本章 `本章短清单` 已生成，且不是泛化提醒：
   - 已写清本章现场到底改了什么
   - 已写清本章章尾允许类型
   - 已写清本章禁用句型
   - 已写清本章世界信息靠什么落地
   - 已写清本章对手反制口
   - 已写清本章第三方把前后补顺的那一下
   - 已写清本章最容易滑成解释腔的位置，以及准备用什么现场东西替掉
14. 若当前任务属于 `边界题材冷启动 / 补丁后回归验证`，已完成“现实挤压补充卡”：
   - 至少答出 `谁被挤掉实际位置 / 谁把第二步先手抢走了 / 谁被迫往前往后挪或改先后 / 哪道名分开始当场起作用` 四项中的三项
   - 任一项若只写成“更乱了 / 更紧张了 / 大家更急了”，视为未答
15. 若当前任务属于 `边界题材冷启动 / 补丁后回归验证`，已完成 `章尾末句预演`：
   - 最后一句必须是新增现场结果
   - 若自然落成压话句、止语句、预告句、总结句，正文就先别写

若任一项缺失，就按 `F5` 处理。

再补一句情报记录拦停：

- 下面任一项一旦撞上，直接 `F5`，正文先别写：
  - 已写过正式情报正文，但项目不存在 `追踪/情报记录.md`
  - 本章会刷新新情报，但记录文件未先写 `刷新章 / 预计兑现章 / 预计收益类型`
  - 上一条情报尚未兑现，记录文件没有解释为何允许跨章延后
  - 记录文件里存在“尚未进入正文的正式情报本体”
- `追踪/情报记录.md` 别放到正文后面补；先建、先填、后写
- 一旦碰上“未来正式情报先写进记录文件”，当前状态就得回退到 `S0/S1` 先把事实层清干净，后面的写后检查、交叉复查和下一章都先别走

再补一句纲层只能留一个说法：

- 写任一单章细纲前，先比对以下文件里的本章 / 近三章升级说法，是不是只剩一个当前有效版本：
  - `设定/题材定位.md`
  - `大纲/开书总蓝图.md`
  - `大纲/全书章节一览.md`
  - `大纲/卷纲_第X卷.md`
  - `大纲/章组控制_第X卷.md`
- 至少核对以下几项：
  - 当前章规模等级
  - 当前章主场景层级
  - 当前章主收益到了哪一步
  - 当前章是否该抬新局面 / 新路线 / 新资源去向
- 若任一文件仍保留旧说法，或这些几项彼此冲突，就按 `F3` 回退到纲层统一说法，本章细纲先别进
- 别一边写新细纲，一边默认旧蓝图 / 旧一览 / 旧题材定位“后面再补”

再补一句章节组得先回头看：

- 写第 `N` 章前，若 `N >= 3`，默认先对最近 `2-3` 章做一次 `章节组闸门`
- `章节组闸门` 至少判断：
  - 最近 `2-3` 章主收益走法是否重复
  - 最近 `2-3` 章是否都在复用同一翻盘路子
  - 上一章章尾是否提前做空本章主任务
  - 最近 `2-3` 章是否真的把书名主打卖点往前推了一步，而不是只换场面
- 以上任一项答不稳，就按 `F3` 回退到 `章组控制 / 细纲`
- `章节组闸门` 没写进文件，第 `N` 章正文就先别写

再补一句信息别乱越权：

- 若正文阶段出现未在细纲 / 写前闸门备案的重要新增信息，别一边写一边默认吞进去
- 若正文阶段反复依赖蓝图里从未备案的重要实体救场，也别一边写一边默认吞进去
- 先停下，补写细纲与写前闸门，再继续正文
- 所谓 `重要新增信息`，至少包括：
  - 新具名人物 / 新具名势力 / 新具名地点
  - 新具名生物 / 新具名植物
  - 新主要物证 / 新主要资源 / 新接头点
  - 新规矩变化 / 新关键时间钉 / 新关系变化
  - 主要设定 / 金手指 / 残魂的新能力说法
- 若新增信息只在正文出现、未反写纲层，按 `F5`
- 若连续两章都靠正文临时补设定才能成立，按 `F3`

再补一句首章世界底盘：

- 若本章是首章，且作品存在足以改写读者默认现实认知的世界卡法，则写前闸门先把 `首章世界总盘交代` 写清
- `世界卡法` 至少包括以下任一类：
  - 超凡 / 灾异 / 异兽 / 修行 / 末世 / 黑科技 / 异能 / 异界 / 禁地
  - 身份制度、社会秩序、职业体系、交通方式、资源争夺方式与普通现实显著不同
  - 主线冲突强依赖该世界独有规矩才能成立
- `首章世界总盘交代` 不是名词表，至少要回答三件事：
  - 这到底是个什么世界
  - 这个世界里的制度 / 日常 / 生存方式为什么这样运转
  - 主角当前开局死局为什么正好由这套世界规矩制造
- 若首章只写局部奇观、局部边地、局部行业特殊，而没有先让读者知道“自己在读什么世界”，默认至少按 `F2`

再补一句书名主打到底兑现到哪一步：

- 若书名主打属于 `经营 / 配额 / 分配 / 执牌 / 拍卖 / 夺权 / 定顺序 / 掌规矩` 任一类，细纲、写前闸门、写后检查都必须明确本章处于哪一层：
  - `D1 活下来`
  - `D2 拿到干活资格 / 自保资格`
  - `D3 拿到自保口`
  - `D4 拿到分别人命的口`
  - `D5 拿到一段公共秩序 / 资源分配权`
- 正文若只落到 `D1-D2`，写后检查里就别写成“已定顺序 / 已把书名主打兑现出来”
- 对于书名或书名主打直接售卖“分配权 / 生火权 / 名额权 / 车权 / 放票权”的作品：
  - 首章最低别低于 `D3`
  - 前 `2-3` 章最少得有 `1` 章到 `D4`
  - 阶段一结束前得出现 `D5` 的雏形或等价现实结果
- 层级没达标却被写后检查判 `F0`，就直接按 `F5`

再补一句人物出场别只剩功能：

- 关键角色不得只以“身份标签 + 功能动作”进入正文
- 进入正文前，细纲或写前闸门必须至少回答：
  - 这个角色这场最想拿到什么
  - 这个角色最怕丢掉什么
  - 这个角色会用什么动作 / 说话节奏 / 观察偏好暴露自己
- 若答不出，默认该角色还只剩办事作用，正文先别写
- 若本章关键翻面必须依赖某配角认出、确认、失口、转述、解释才能成立，而该角色没有前置先后关系、私心或动作记忆点，就直接按 `F2`
- 多人对话场景若删去人名后仍明显混同，至少按 `F2`
- 重要角色若只有“设定上值得看”，正文没有能让读者自己补画面的记忆点，至少按 `F2`

补充全量审查要求：

- 项目里得先有 `追踪/阶段审查计划.md`
- 日更或续写前，看一下当前有没有撞上该文件里的“阶段审查触发条件”
- 若已经撞上，却还直接继续写下一章，默认按流程失守处理
- 若发生以下任一情况，也得追加一次 `story-review full`：
  - 连续 `3` 章以上回炉 / 大修刚完成
  - 主要推进方向或章节组方向发生改写
  - 最近一次问题碰上过 `F5`，且涉及事实层 / 追踪层 / 交叉复查层
  - 当前卷结束，且阶段计划未写明可跳过

写前闸门至少必须包含：
- 题材识别结果
- 本章唯一说法
- 本章主要往前推什么
- 本章副驱动
- 本章最主要收获
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 兑现类型（单章兑现 / 章组兑现 / 情节库兑现）
- 本章章尾钩子类型
- 本章单独检查失败风险
- 预判失败码

补充本章短清单要求：

- `本章短清单` 是当前章正文首稿阶段最先要盯住的短清单，别省
- `本章短清单` 要短，推荐 `300-600` 字；写成长篇说明默认就算失效
- `本章短清单` 里至少要有 `2-4` 条本章禁用句型，而且得来自当前真实风险，别只写泛泛提醒
- 若本章已知高频复发 `解释腔 / 作者直接替读者收尾的句子 / 预告腔 / 只剩气势没落地的重话` 之一，`本章短清单` 不写对应禁项，按 `F5`
- 首稿完成后，写后检查要逐条回答：这些禁项是否碰上；若碰上，怎么替换；答不全就不能判 `F0`

## 五、写完后先拦这些

正文落稿后，至少完成以下动作，这章才算做完：

1. 运行 `scripts/scene_lint.py <正文文件>`
2. 做正文纯净度检查
3. 做章尾人工回看
4. 生成写后检查
5. 同步追踪与角色卡

补充首稿立刻停下的要求：
- 若正文首稿低于 `scene_lint` 下限，直接停在 `S4`，按 `F2` 处理：
  - 写后检查先别进
  - 追踪同步先别进
  - 不得把它只记成“厚度预警”
- 若当前任务属于 `冷启动 / 回归验证 / 首章组测试`，正文首稿低于 `2900` 时，一律别往后走
- 若正文首稿碰上 `解释腔前置自检卡` 且仍存在“删掉也成立”的作者直接替读者收尾的句子，直接停在 `S4`，按 `F1/F2` 处理：
  - 写后检查先别进
  - 不得等待 `story-review` 再抓
- 若 `末尾反删测试` 任一题答案为“是”，直接停在 `S4/S5` 之间，按 `F2` 处理：
  - 删掉最后一句后更强
  - 删掉最后两句后章节仍完整成立
  - 最后一句只承担压话 / 止语 / 预告 / 总结功能

补充脚本判断要求：
- 优先运行 `scripts/scene_lint.py --json <正文文件>` 并读取：
  - `summary.errors`
  - `summary.warnings`
  - `summary.categories`
  - `summary.severity_categories`
- 判断顺序照下面来：
  1. 若 `summary.errors > 0`，直接别算过
  2. 若 `summary.errors = 0` 且只报 `字数.warn`，可继续后续检查，不得机械回修
  3. 若 `summary.errors = 0` 但报 `章尾.warn / 解释.warn / 结构.warn / 情报.warn`，必须人工复看后再决定是否算过
- 别把所有 `warn` 都一把抓成失败
- 别只看文本行数判断脚本结果，优先看 JSON 里的结果项

写后检查至少必须包含：
- 场面数
- 外显战果
- 章尾结果类型
- 高潮兑现怎么看
- 本章目标收益到了哪一步
- 本章已落地收益类型
- 本章实际收益到了哪一步
- 本章收益到了这一步的证据句
- 本章为什么还没往上走
- 功能场景有没有写成说明文（如适用）
- `scene_lint` 结果
- 纯净度结果
- AI味分级
- 角色卡更新结果
- 追踪更新结果
- 说法交叉复查结果
- 最终失败码
- `这章能算完成 / 这章先别算完成`

写后检查必须答实的几句：

- 写后检查别只复述“本章书名主打卖点兑现项”，也别照抄写前闸门
- 凡书名主打碰上 `经营 / 配额 / 分配 / 执牌 / 拍卖 / 定顺序 / 掌规矩 / 生火权 / 名额权 / 车权 / 放票权` 任一类，写后检查必须额外实答以下问题：
  - 主角这一章到底决定了谁的什么
  - 这份控制权属于 `D1-D5` 哪一层
  - 若未到 `D3`，为什么仍然还能算过
- 若答不出“谁 / 什么 / 怎么改了”，只写“更接近定顺序 / 已拿到火门 / 已经开始分配”，就按 `F2/F5`
- 若答不出“目标到了哪一步 / 实际到了哪一步 / 证据句是哪句 / 为什么还没再往上走”，就直接按 `F5`
- 若写后检查与正文实答不一致，或正文只有 `D1-D2` 却被写成“书名主打已经兑现”，就按 `F5`
- 若本章碰上 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，写后检查还必须额外实答：
  - 本章哪一段在承担规矩信息
  - 该段是否同时推动了 `试用 / 交易 / 排队 / 讨价 / 受阻 / 改规矩 / 利益变化` 中的至少一项
  - 若删掉该段，当前场面推进是否会明显变弱
- 若上述三问任一答不稳，就不能判 `F0`
- 若写后检查把明显可改写成讲解话的段落判成“信息交代充分、功能场景成立”，就按 `F5`

补充要求：
- `本章已落地收益类型` 只能填写正文已经发生的 `命 / 人 / 名 / 权 / 物证 / 被逼选边 / 规矩变化 / 垫背者落地`
- 禁止把“下一章可直接反制 / 可直接进入 / 已经布好局 / 即将兑现”写成当前章收益
- 若首章 `章尾结果类型 = 对得上` 但 `高潮兑现怎么看 = 没对上`，仍不能判 `F0`
- 情报流正文若已出现正式情报文本，写后检查前一定先核对 `追踪/情报记录.md` 已创建，且至少补写 `刷新章 / 预计兑现章`

## 六、这章现在能不能算写完

这里最后只能写二选一：
- `这章能算完成`
- `这章先别算完成`

只要出现以下任一情况，就直接记“这章先别算完成”：
- `scene_lint.py` 存在任一 `error`
- 正文纯净度没过
- 章尾没过
- 外显战果未成立
- 场面兑现未成立
- 高潮后续收益未成立
- 若标记为 `章组兑现 / 情节库兑现` 但写不清 `兑现组范围 / 本章第几拍 / 本章现实中间收益 / 最迟完整兑现章`，按 `F3`
- 若标记为 `章组兑现 / 情节库兑现` 但本章没有任何现实中间收益，按 `F2`
- AI味达到中度以上且未完成衔接
- 追踪同步未完成
- 角色卡更新未完成
- 说法交叉复查没过
- 情报记录缺失或说法超前

补充一句：
- 若 `scene_lint.py` 仅剩 `字数.warn`，且章尾、纯净度、写后检查、交叉复查都对得上，可写“这章能算完成”
- 若 `scene_lint.py` 的 `warn` 涉及 `章尾 / 解释 / 结构 / 情报` 任一类，默认别因为“脚本退出码为 0”就直接算过
