# 核心门禁

> 本文件收束 `story-long-write` 的最高优先级硬规则。进入任一写作阶段前，先读本文件，再按场景加载其他 workflow / quality / reference 文件。

---

## 一、统一失败码

| 失败码 | 含义 | 强制动作 |
|------|------|---------|
| `F0` | 未命中失败项 | 可进入下一步 |
| `F1` | 句子层失败：空效果句 / 解释腔 / 局部 AI 味 | 当场回修，复扫通过前不得结束本章 |
| `F2` | 章节层失败：章尾不过线 / 场面不足 / 战果偏虚 | 回退到本章细纲或本章正文重写，禁止进入下一章 |
| `F3` | 机制层失败：题面滑轨 / 连续多章只查不爆 / 副驱动反客为主 | 停止顺写，回退到章节组细纲或卷结构 |
| `F4` | 污染失败：工具残片 / 补丁标记 / 执行元话语 / 非故事语料 | 先清污染，再全文复扫 |
| `F5` | 流程失败：角色卡 / 追踪 / 写前闸门 / 验收单缺失或假完成 | 先补流程文件，补完前本次工作不算完成 |

规则：
- 写前闸门、正文写作、写后验收都必须产出 `F0-F5`
- 最终失败码不是 `F0`，不得并入主正文，不得进入下一章
- 不得用“基本可用”“后续再补”替代失败码裁决
- 失败码必须先落盘再继续；未落盘视为 `F5`

## 一点五、执行状态机

所有长篇写作阶段，统一按以下状态推进：

`S0 资料未齐 -> S1 纲层可写 -> S2 细纲过线 -> S3 写前闸门过线 -> S4 正文可写 -> S5 写后验收过线 -> S6 追踪同步过线 -> S7 允许进入下一文件`

硬规则：
- 不得跳状态
- 任一状态未形成明确落盘证据，默认停在上一状态
- 只要当前状态对应失败码不是 `F0`，禁止推进到下一状态
- 用户要求“继续 / 不要停 / 一直往下推”，不构成跳状态授权
- 若模型无法明确自己当前处于哪个状态，按 `F5` 处理，先补状态文件

补充人工接管状态：

- 若命中污染残片、文件截断、结构漂移、题面滑轨、章节组打架，必须自动切入 `H1` 人工接管
- `H1` 不是放行状态，而是返修状态，且命中后不得继续询问是否接管：
  - 必须先写明接管原因
  - 必须写明接管文件范围
  - 必须在接管完成后重新回到 `S2/S3/S4` 之一继续走流程
- `H1` 不得绕过 `F0-F5`
- 若当前任务声明为“纯 skill 冷启动验证”，必须使用 `H1-A`；`H1-B` 仅供生产自动返修态使用

## 二、生产顺序

正常生产顺序固定为：

`核心设定 -> 题材识别+专项挂载 -> 开书总蓝图 -> 卷纲/章组控制 -> 单章细纲 -> 细纲过线 -> 正文 -> 写后验收 -> 追踪/角色卡同步`

规则：
- `大纲/开书总蓝图.md` 缺失，不得进入卷纲与细纲
- `大纲/全书章节总表.md` 缺失，不得视为已具备全书总览
- `大纲/事件线总表.md` 缺失，不得视为已具备全局事件线规划
- `大纲/章组控制_第X卷.md` 缺失，不得批量写该卷单章细纲
- `大纲/开书总蓝图.md` 若没有建立关键实体层（人物 / 势力 / 生物 / 植物 / 物品 / 地点 / 规则 / 事件）的最小资产表，也不得进入卷纲与细纲
- 细纲不过线，不得写正文
- 正文不过线，不得并入主正文
- 追踪 / 角色卡未同步，视为本次工作未完成
- 不允许先写正文，后补细纲 / 追踪 / 验收
- 开书后凡作品命中 `情报流 / 系统流 / 预知流 / 任务流` 任一项，`追踪/情报台账.md` 缺失时，生产顺序直接中断在 `S0/S1`，不得进入首章正文

## 四、写前硬闸

进入细纲或正文前，必须先完成：

1. 题材识别结果落盘
2. 专项 checks 挂载完成
3. `大纲/开书总蓝图.md` 已存在且当前有效
4. 当前卷 `卷纲 + 章组控制` 已存在且当前有效
5. 本章 / 本组主驱动明确
6. 本章最值钱战果明确
7. 写前闸门落盘
8. 若命中 `情报流 / 系统流 / 预知流 / 任务流`，`追踪/情报台账.md` 已存在且本章对应条目已预填
9. 若本章属于某个连续章节组（默认最近 `2-3` 章），已完成 `章节组闸门`

若任一项缺失，按 `F5` 处理。

补充情报台账阻断硬规则：

- 以下任一项命中，直接 `F5`，不得写正文：
  - 已写过正式情报正文，但项目不存在 `追踪/情报台账.md`
  - 本章会刷新新情报，但台账未先写 `刷新章 / 预计兑现章 / 预计收益类型`
  - 上一条情报尚未兑现，台账没有解释为何允许跨章延后
  - 台账里存在“尚未进入正文的正式情报本体”
- `追踪/情报台账.md` 不得在正文后补；必须先建、先填、后写
- 一旦命中“未来正式情报先入表”，当前状态必须回退到 `S0/S1` 先清事实层，不得继续写后验收、反校验或下一章

补充纲层单一版本比对硬规则：

- 写任一单章细纲前，必须先比对以下文件中的本章 / 近三章升级口径是否只有一个当前有效版本：
  - `设定/题材定位.md`
  - `大纲/开书总蓝图.md`
  - `大纲/全书章节总表.md`
  - `大纲/卷纲_第X卷.md`
  - `大纲/章组控制_第X卷.md`
- 至少核对以下字段：
  - 当前章规模等级
  - 当前章主场景层级
  - 当前章主收益档位
  - 当前章是否该抬新盘面 / 新路线 / 新资源口
- 若任一文件仍保留旧口径，或这些字段彼此冲突，直接按 `F3` 回退到纲层统一口径，不得进入本章细纲
- 不得一边写新细纲、一边默认旧蓝图 / 旧总表 / 旧题材定位“后面再补”

补充章节组闸门硬规则：

- 写第 `N` 章前，若 `N >= 3`，默认必须先对最近 `2-3` 章做一次 `章节组闸门`
- `章节组闸门` 至少判断：
  - 最近 `2-3` 章主收益骨架是否重复
  - 最近 `2-3` 章是否都在复用同一翻盘模板
  - 上一章章尾是否提前做空本章主任务
  - 最近 `2-3` 章是否真的把题面卖点往前推了一步，而不是只换场面
- 以上任一项答不稳，直接按 `F3` 回退到 `章组控制 / 细纲`
- `章节组闸门` 未落盘，不得写第 `N` 章正文

补充信息越权硬规则：

- 若正文阶段出现未在细纲 / 写前闸门备案的核心新增信息，不得一边写一边默认吞入
- 若正文阶段反复依赖蓝图里从未备案的核心实体救场，也不得一边写一边默认吞入
- 必须先停下，回填细纲与写前闸门，再继续正文
- 所谓 `核心新增信息`，至少包括：
  - 新具名人物 / 新具名势力 / 新具名地点
  - 新具名生物 / 新具名植物
  - 新核心物证 / 新核心资源 / 新交接口
  - 新规则变化 / 新关键时间钉 / 新关系改判
  - 核心设定 / 金手指 / 残魂的新能力口径
- 若新增信息只在正文出现、未反写纲层，按 `F5`
- 若连续两章都靠正文临时补设定才能成立，按 `F3`

补充首章世界总盘硬规则：

- 若本章是首章，且作品存在足以改写读者默认现实认知的世界机制，则写前闸门必须先写清 `首章世界总盘交代`
- `世界机制` 至少包括以下任一类：
  - 超凡 / 灾异 / 异兽 / 修行 / 末世 / 黑科技 / 异能 / 异界 / 禁地
  - 身份制度、社会秩序、职业体系、交通方式、资源争夺方式与普通现实显著不同
  - 主线冲突强依赖该世界独有规则才能成立
- `首章世界总盘交代` 不是名词表，至少要回答三件事：
  - 这到底是个什么世界
  - 这个世界里的制度 / 日常 / 生存方式为什么这样运转
  - 主角当前开局死局为什么正好由这套世界规则制造
- 若首章只写局部奇观、局部边地、局部行业特殊，而没有先让读者知道“自己在读什么世界”，默认至少按 `F2`

补充题面兑现档位硬规则：

- 若题面属于 `经营 / 配额 / 分配 / 执牌 / 拍卖 / 夺权 / 掌口 / 掌规矩` 任一类，细纲、写前闸门、写后验收都必须明确本章处于哪一档：
  - `D1 活下来`
  - `D2 拿到干活资格 / 自保资格`
  - `D3 拿到自保口`
  - `D4 拿到分别人命的口`
  - `D5 拿到一段公共秩序 / 资源分配权`
- 正文若只落到 `D1-D2`，不得在写后验收里写成“已掌口 / 已题面兑现”
- 对于书名或题面直接售卖“分配权 / 生火权 / 名额权 / 车权 / 放票权”的作品：
  - 首章最低不得低于 `D3`
  - 前 `2-3` 章至少 `1` 章必须到 `D4`
  - 阶段一结束前必须出现 `D5` 的雏形或等价现实结果
- 档位未达标却被写后验收判 `F0`，直接按 `F5`

补充全量审查硬规则：

- 项目必须存在 `追踪/阶段审查计划.md`
- 日更或续写前，必须先判断当前是否命中该文件中的“阶段审查触发条件”
- 若已命中，却仍直接继续写下一章，默认按流程失守处理
- 若发生以下任一情况，也必须追加一次 `story-review full`：
  - 连续 `3` 章以上回炉 / 大修刚完成
  - 主驱动或章节组方向发生改写
  - 最近一次问题命中过 `F5`，且涉及事实层 / 追踪层 / 反校验层
  - 当前卷结束，且阶段计划未写明可跳过

写前闸门至少必须包含：
- 题材识别结果
- 本章唯一口径
- 本章主驱动
- 本章副驱动
- 本章最值钱战果
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 本章章尾钩子类型
- 本章专项失败风险
- 预判失败码

## 五、写后硬闸

正文落稿后，至少完成以下动作才允许判定完成：

1. 运行 `scripts/scene_lint.py <正文文件>`
2. 做正文纯净度检查
3. 做章尾人工回看
4. 生成写后验收
5. 同步追踪与角色卡

补充脚本裁决硬规则：
- 优先运行 `scripts/scene_lint.py --json <正文文件>` 并读取：
  - `summary.errors`
  - `summary.warnings`
  - `summary.categories`
  - `summary.severity_categories`
- 裁决顺序固定为：
  1. 若 `summary.errors > 0`，直接不过线
  2. 若 `summary.errors = 0` 且只命中 `字数.warn`，可继续后续验收，不得机械回修
  3. 若 `summary.errors = 0` 但命中 `章尾.warn / 解释.warn / 结构.warn / 情报.warn`，必须人工复看后再决定是否放行
- 不得把所有 `warn` 统一视为失败
- 不得只看文本行数判断脚本结果，优先以 JSON 结构字段为准

写后验收至少必须包含：
- 场面数
- 外显战果
- 章尾判型
- 高潮兑现结论
- 本章已落地收益类型
- `scene_lint` 结果
- 纯净度结果
- AI味分级
- 角色卡同步结果
- 追踪同步结果
- 口径反校验结果
- 最终失败码
- `可并入主正文 / 不可并入主正文`

写后验收实答制硬规则：

- 写后验收不得只复述“本章题面卖点兑现项”或照抄写前闸门
- 凡题面命中 `经营 / 配额 / 分配 / 执牌 / 拍卖 / 掌口 / 掌规则 / 生火权 / 名额权 / 车权 / 放票权` 任一类，写后验收必须额外实答以下问题：
  - 主角这一章到底决定了谁的什么
  - 这口控制权属于 `D1-D5` 哪一档
  - 若未到 `D3`，为什么仍可放行
- 若答不出“谁 / 什么 / 怎么改了”，只写“更接近掌口 / 已拿到火门 / 已经开始分配”，直接按 `F2/F5`
- 若写后验收与正文实答不一致，或正文只有 `D1-D2` 却被判“题面兑现通过”，直接按 `F5`

补充硬规则：
- `本章已落地收益类型` 只能填写正文已经发生的 `命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化 / 垫背者落地`
- 禁止把“下一章可直接反制 / 可直接进入 / 已经布好局 / 即将兑现”写成当前章收益
- 若首章 `章尾判型 = 合格` 但 `高潮兑现结论 = 未过线`，仍不得判 `F0`
- 情报流正文若已出现正式情报文本，写后验收前必须核对 `追踪/情报台账.md` 已创建且至少回填 `刷新章 / 预计兑现章`

## 六、章节完成裁决

章节完成裁决只能写二选一：
- `可并入主正文`
- `不可并入主正文`

只要出现以下任一情况，默认 `不可并入主正文`：
- `scene_lint.py` 存在任一 `error`
- 正文纯净度未通过
- 章尾未过线
- 外显战果未成立
- 场面兑现未成立
- 高潮后续收益未成立
- AI味达到中度以上且未完成衔接
- 追踪同步未完成
- 角色卡同步未完成
- 口径反校验未通过
- 情报台账缺失或口径超前

补充放行口径：
- 若 `scene_lint.py` 仅剩 `字数.warn`，且章尾、纯净度、写后验收、反校验均为通过，可判 `可并入主正文`
- 若 `scene_lint.py` 的 `warn` 涉及 `章尾 / 解释 / 结构 / 情报` 任一类，默认不得直接因为“脚本退出码为 0”而放行

## 七、回炉与执行计划

命中连续回炉 / 施工单 / 验收单时，必须先：
- 生成固定验收单或执行计划
- 明确处理范围
- 明确施工顺序
- 明确完成定义

未生成执行计划前，不得直接改正文或改追踪。
