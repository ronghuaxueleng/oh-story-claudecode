# 正文门禁

> 本文件只负责正文阶段的执行清单：写前必须确认什么、写后必须完成什么、情报流/字数/章尾怎么验收。句子与段落级红线统一交给 `references/writing-red-lines.md`。

---

## 一、正文前置要求

开写正文前必须确认：

- 已读取 `设定/题材定位.md`
- 已读取本章细纲
- 已完成写前闸门
- 已写明本章目标字数与上限字数
- 已为正文预留 `字数安全边际`；若脚本下限为 `2900`，首写正文默认目标不得低于 `3100`
- 已明确第一爆点与高潮后续收益
- 已明确本章 `核心对手 / 核心物证或核心资源 / 核心场面载体`
- 已明确本章 `核心母场景 / 主事件来源 / 世界主动施压载体`
- 已确认本章不是对上一章母场景的换皮复用
- 已确认本章残魂 / 金手指不是继续只做“提醒 / 判断增强”
- 若本章是首章且作品存在足以改写默认现实认知的世界机制，已明确 `首章世界总盘交代` 与 `世界底色显影锚点`
- 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，已明确 `世界底色续显点`
- 已确认本章 `主收益骨架` 不与最近 `2-3` 章重复
- 已确认残魂 / 金手指会留下 `外部可感知后果`
- 若本章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，已明确 `跨场景过桥`
- 已明确本章中后段至少 `1` 口 `外部反应 / 群体站位变化 / 系统屏幕变化 / 协作方改口`
- 已确认本章不存在未替换的抽象占位表达
- 已确认不存在“单一力度副词”批量刷句；动作强度必须写在受力点、位移、失衡、破坏结果里
- 已完成 `首稿污染预防`：`references/banned-words.md` 的 `一级禁用词` 本章正文起草时不得出现，不得用口头发力词、空力度词、抽象效果词替代动作链
- 已完成 `禁用词反向污染预防`：不得因为读取了禁用词表，就把表内词面、口头发力词或解释句黑名单词写进正文
- 已读取本章 `新增信息白名单` 与 `正文禁增项`
- 已确认正文若需新增核心信息，必须先回填细纲与写前闸门，不得边写边吞
- 已明确本章 `题面兑现档位`
- 若题面命中 `经营 / 配额 / 分配 / 掌口 / 生火权 / 放票权 / 名额权`，已明确本章正文至少要写出 `谁被主角实际改了火 / 票 / 路 / 名额`

## 二、句子红线入口

正文出现以下问题时，不在本文件展开重复说明，直接按 `references/writing-red-lines.md` 执行：

- 效果词先行
- 后台调度语
- 作者盖章句
- 提纲扩写句
- 只删词不回句子功能
- 单一力度副词反复刷屏
- 未备案核心信息临时入正文

---

## 三、情报流专门要求

- 正文里第一次出现情报时，必须用 `「」` 单独包住
- 情报首次出现必须交代来源或触发机制
- 情报必须在当前章或后 `1` 章内转成现实收益
- “知道更多 / 更危险 / 更复杂”不算兑现

补充硬判：
- 若情报只负责提醒、解释、抬线索价值，但没有推动现实盘面变化，按未兑现处理
- `现实收益` 至少应能落到 `命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化 / 垫背者落地` 之一
- 若首章情报只换来“对手更警惕/主角更清醒”，默认不算过线
- 情报句之后，正文必须在短距内出现具体动作响应；不得出现“情报出现 -> 主角思考很多 -> 下一段再动”
- 若本章细纲已写 `本章情报条目`，正文不得改成“临场观察替代情报”
- 若本章正文关键翻盘依赖名单、车、票、口令、批文、调度等关键信息，而这些信息未在情报条目或新增信息白名单中备案，默认按 `F3/F5`
- 情报流作品前 `3` 章至少 `2` 次出现“情报 -> 动作 -> 外部现实变化”的完整链；不足则默认题面滑轨

## 三点一、题面结果化兑现硬规则

- 正文不得把题面兑现写成“更接近、快要摸到、差一点拿到”
- 每章必须至少回答一个结果化问题：
  - 主角这一章真实掌到哪一口分配权 / 放票权 / 车权 / 名额权 / 调度权
  - 若没有掌到，是否至少逼敌方公开改动作 / 改规则 / 改站位
- 若题面是经营 / 配额 / 分配 / 执牌 / 夺权类，连续 `2-3` 章都只做到“掀黑、截物证、揭脏手”，默认按 `F3`
- 若题面命中 `掌口 / 生火权 / 名额权 / 放票权 / 分火权`，正文中必须尽快出现以下至少一种实答结果：
  - 主角点了谁先进 / 先拿 / 先走
  - 主角卡了谁不能进 / 不能拿 / 不能走
  - 敌方被迫按主角的新顺序执行
- 若整章只能回答“主角拿到一张签 / 一口资格 / 一份自保位”，默认最高只算 `D2`，不得写成“已掌口”

## 三点二、章节组反重复硬规则

- 连续 `2-3` 章不得反复使用同一翻盘模板，例如：
  - 物证见光 -> 群众炸锅 -> 执行口让步
  - 名单见光 -> 群众炸锅 -> 临时改口
  - 关键物见光 -> 对手失态 -> 章尾再抬高层敌意
- 若最近 `2-3` 章已复用同模板，本章正文必须主动换打法：
  - 从抢物证切到抢人 / 抢口 / 抢规则执行权
  - 从围观炸锅切到正式秩序对撞 / 调度对撞 / 群体改线
  - 从掀基层脏手切到咬中层盘面敌人
- 若正文写到中段才发现仍在复用旧模板，立即停写，回退细纲，按 `F3`

## 三点四、字数安全边际硬规则

- `scene_lint.py` 的下限不是目标字数；正文首写必须主动留安全边际
- `scene_lint.py` 的字数结果分为两层：
  - `ERROR`：明显过短 / 过长，硬拦截
  - `WARN`：接近下限 / 偏高，只做预警，不单独拦截
- 若项目脚本下限为 `2900`，则：
  - 首写正文推荐目标 `3300-3800`
  - 不得以 `2900-3000` 作为收稿目标
- 若细纲目标字数已低于脚本下限上浮 `100` 字，写前闸门必须先把目标抬高
- 若最近 `2` 章都命中 `字数.warn`，下一章首写目标至少再上抬 `100`
- 若细纲或写前闸门写了目标下限，正文首稿低于该目标下限 `200` 字以上，直接按 `F2`，不得进入写后验收
- 若正文首稿低于 `scene_lint` 下限，直接按 `F2`，不得用“节奏快 / 可读性强”放行
- 若正文首稿低于 `3100`，即使脚本未报错，也必须人工判定是否存在“正文主体不足”；不得默认 `F0`

默认不过线情形：
- 正文多次停在 `2900` 附近擦线
- 正文连续停在 `3000` 左右、虽能过线却反复触发接近下限告警
- 通过补一两句机械凑字来过脚本
- 正文主体不足，只在章尾补解释或总结凑到下限

## 三点四点一、首稿污染自检硬规则

- 正文首稿写完后，进入写后验收前必须先执行首稿污染自检
- 自检范围：
  - `references/banned-words.md` 的一级禁用词
  - 口头发力词污染
  - 解释句黑名单词面
  - 工具残片 / 补丁标记 / 非故事语料
- 命中任一项：
  - 默认 `F1/F4`
  - 不得进入 `F0` 写后验收
  - 生产模式下自动切入 `H1-B 生产自动返修态`
  - 纯基准测试下自动切入 `H1-A 纯测试记录态`，只记录失败，不修正文
- 禁止把“只是一个词”降级为观察项；一级禁用词属于首稿失败项

## 三点五、具象化正文硬规则

- 正文中的核心位不得落成以下抽象占位：
  - 某个人
  - 某件东西
  - 某个势力
  - 一份物证
  - 更高层的人
  - 新的交接点
  - 外部突然生乱
- 若细纲已具名 `核心对手 / 核心物证 / 核心资源 / 核心场面载体`，正文首轮出场时必须尽快落名，不得重新打回抽象词
- 若正文把细纲中的具名大事件重新写回“匪患 / 混乱 / 追杀 / 证据 / 调令”这类通用词，视为兑现降级

## 三点五点一、正文新增信息越权硬规则

- 正文不得临时新增未备案的核心信息来救场面、救章尾、救收益
- `核心信息` 至少包括：
  - 新具名人物 / 新具名势力 / 新具名地点
  - 新核心物证 / 新核心资源 / 新交接口
  - 新规则变化 / 新关键时间钉 / 新关系改判
  - 新的残魂 / 金手指 / 系统能力口径
- 若写作中发现确需新增：
  - 先停正文
  - 先回填细纲 `新增信息白名单`
  - 再回填写前闸门
  - 必要时同步卷纲 / 伏笔 / 角色卡准入口径
  - 完成后才可继续正文

默认不过线情形：
- 为了让章尾更硬，临时多出一个新接头点 / 新暗号 / 新账簿 / 新路线
- 为了让冲突成立，临时补一个之前没预热的新人物 / 新势力
- 为了让主角翻盘，临时补一个细纲没写过的新能力口径
- 为了补场面，正文突然出现细纲未备案的新规则或新世界设定
- 为了让读者理解“这口资源升级了”，正文直接补一句 `更值钱 / 更硬 / 更实 / 这一口更…… / 比上一口更……`
- 为了让章尾像钩子，正文直接用 `现在真正该抢的是…… / 最该抢的已经不是……`

## 三点六、核心设定出手硬规则

若本章细纲写有 `本章核心设定参与方式`，正文必须让读者看到“出手结果”，不认纯解释。

合格示例方向：
- 借体后完成普通肉身做不到的动作
- 压迫后让对手失神、失口、露底
- 共鸣后拆穿禁印、伪装、假货
- 刺感后抢在群体变化前先动一步

不过线情形：
- 只写“主角忽然觉得不对”
- 只写“残魂提醒了他”
- 只写“他判断出了问题所在”
- 结果仍完全像普通聪明人能做到的事，且没有任何可见反应差

## 三点七、L3/L4 场面落地硬规则

凡细纲标记 `L3/L4` 的章节，正文必须写出：
- 群体规模感
- 被改写的路线 / 秩序 / 营地 / 资源 / 站位
- 至少一种强场面载体

若正文只剩：
- “场面大乱”
- “所有人都慌了”
- “局势一下失控”

而没有具体现实变化，按规模兑现失败处理。

## 三点八、母场景禁复用硬规则

- 正文不得把上一章的 `核心母场景` 只换个地名或道具后重复再写一遍
- 若上一章核心母场景属于以下任一类，正文默认高风险回流：
  - 囚车 / 押解路
  - 尸体 / 验尸
  - 驿站 / 旧棚 / 临时窝点
  - 公验 / 对质 / 逼口 / 翻供
  - 小股灭口 / 小队试探
- 命中高风险回流时，正文必须尽快切出新的主事件载体，例如：
  - 军粮车队
  - 矿场暴乱
  - 山寨并线
  - 黑市洗货
  - 关城封锁
  - 仓储争抢
  - 灾异冲营
  - 禁地异动

默认不过线情形：
- 还是围着车、尸、驿、口供、规矩翻脸打转
- 只是把“验尸”换成“公验”
- 只是把“驿站”换成“黑店 / 破棚 / 偏院”

## 三点九、世界主动施压硬规则

- 本章大冲突必须先由 `世界 / 势力 / 环境 / 资源争夺` 压到主角脸上，再由主角反切
- 不得长期沿用“情报出现 -> 主角去查 -> 对手来灭口 -> 主角翻面”的单一小查线模型
- 正文必须尽快交代：是谁先动、动了哪条路 / 哪批货 / 哪个营 / 哪道关 / 哪群人

默认不过线情形：
- 主角又是唯一发动者
- 敌人只来一波小灭口，没有更大盘面变化
- 情报仍主要服务于“找人 / 验尸 / 套口供”

## 三点十、残魂 / 金手指轮换硬规则

- 若本章存在残魂 / 金手指直接出手，正文必须尽快让读者看见其 `出手类型`
- 连续两章不得都只写成：
  - 提醒危险
  - 判断更准
  - 提前察觉
  - 单纯识别问题
- 若上一章已主要使用 `识破类`，本章优先改用 `压迫类 / 夺势类 / 超常动作类 / 异象共鸣类`
- 若本章仍沿用低效出手类型，必须额外补出一次可见现实变化，不得只让主角“更聪明”
- 残魂 / 金手指出手后，正文必须尽快给出至少 `1` 个外部可感知后果：
  - 对手失神 / 失口 / 误判
  - 器物 / 阵纹 / 骨灯 / 封印 / 环境出现反应差
  - 旁人因为不合常理结果改站位、改口、改动作

默认不过线情形：
- 只有主角自己知道残魂刚才出手了
- 外部人和环境完全无反应

## 三点十一、首章世界总盘与世界底色显影硬规则

- 若本章是首章，且作品存在足以改写默认现实认知的世界机制，正文前 `1/3` 必须同时完成两件事：
  - 让读者知道自己在读什么世界
  - 让读者看见这套世界规则如何现场改动作、改规矩、改生死
- `首章世界总盘交代` 必须至少回答：
  - 这到底是个什么世界
  - 这个世界的制度 / 日常 / 生存方式为什么会这样运转
  - 主角当前死局为什么正好由这套世界规则制造
- `世界底色显影锚点` 必须落到动作、规矩、环境、职业常识、群体反应、制度执行中的至少 `2` 项，不能只术语点名

合格示例方向：
- 王朝存在，但驿路、军路、流放、劳役本来就和异兽、灾潮、修行资源绑定
- 都市存在，但交通、治安、职业、阶层已被异能或黑科技长期改写
- 校园存在，但升学、社交、竞争、惩罚机制本来就依赖超凡规则

默认不过线情形：
- 首章只有局部奇观，没有世界总定义
- 读者前半章仍会误判成普通古代 / 普通都市 / 普通现实
- 世界机制只存在于主角脑内旁白
- 只有边地 / 副本 / 行业特殊，没有交代整个世界为什么能容纳这套特殊
- 首章前 `1/3` 连续使用世界说明句，却没有让至少一条规则通过现场执行先裁到主角脸上

## 三点十二、非首章世界底色续显硬规则

- 若题材命中 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，第 `2-5` 章不得把世界又写回纯世俗层
- 至少每 `2` 章必须出现一次 `世界底色续显`
- 续显必须落到现场规矩、环境余波、器物用法、行业避讳或群体反应

默认不过线情形：
- 第1章有玄异，第2章起只剩现实冲突
- 所有玄异内容都缩回主角脑内

## 三点十三、主收益骨架轮换硬规则

- 连续 `2-3` 章不得反复使用同一套主收益骨架
- 尤其不得长期复用：
  - 情报指向关键物
  - 残魂压一拍逼失口
  - 主角抢下物证 / 账 / 令
  - 章尾再抬更高层敌意
- 正文必须在收益打法上做轮换，例如：
  - 抢人
  - 改规矩
  - 收站队
  - 逼敌人自毁
  - 借灾异翻盘

默认不过线情形：
- 场景换了，收益仍只是“再抢一份更值钱的东西”
- 主角每章都靠同一招数收口

## 三点十四、跨场景过桥硬规则

- 若本章相对上一章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，正文开章前 `100-200` 字内必须补清以下至少 `2-3` 项：
  - 主角为什么来
  - 主角怎么来的
  - 主角靠什么锁定这里
  - 主角为什么正好撞上本章第一爆点
- 若上章章尾只抛出“营里要出事 / 新据点有问题 / 某路会炸”，下章正文不得直接跳成主角已在现场
- 若本章是 `L3/L4` 大场面切入，正文必须先给主角一个短决策或短追踪动作，再起爆，不得纯作者调度式硬切
- 过桥必须是人物行动链，不认作者代说

合格示例方向：
- 上章逼出敌方裂口，下章先写主角看见异常信号、判断对方今夜必内乱，因此先摸到关键入口
- 上章拿到名册，下章先写主角顺名册上的交接口、车辙、运输路线或人物余波赶去新场

默认不过线情形：
- “主角赶到时”直接顶开全章
- “事情果然爆了”代替行动链
- 章首全是大场面，主角为什么在场只能靠读者脑补
- 不是人物追过去，而是作者把人物搬过去

## 三点十五、外部反应配额硬规则

- 正文中后段必须至少出现 `1` 口可见的 `外部反应 / 群体站位变化 / 协作方改口 / 系统口径变化`
- 这口反应不得只来自主角或核心对手，优先落在：
  - 围观者
  - 执行层
  - 协作方
  - 平台 / 屏幕 / 轮播 / 群消息
  - 后勤 / 司机 / 兵丁 / 驿卒 / 船工 等现场承压人
- 目的不是凑热闹，而是把“盘面翻了”写成别人也被迫改动作

默认不过线情形：
- 全章只有主角和对手在互顶
- 后补字数时只能往解释句或心理句里塞
- 明明发生公开争夺，现场旁人却像不存在

---

## 四、字数硬约束

- 默认首章目标：`3100-4500`
- 默认非首章目标：`3100-4000`
- 若细纲另有标注，以细纲为准
- 未达目标，不得把“节奏快”当成放行理由

---

## 五、章尾硬闸

合格章尾只认现场，不认总结：

必须停在以下之一：
- 新危险
- 新反扑
- 新人物 / 新敌意 / 新物证落地
- 失口 / 露底 / 翻面
- 新规矩马上被打破
- 新的不可逆局势

不得停在：
- 预告句
- 总结句
- 信息说明句
- 平收转场句

补充硬判：
- 若章尾继续使用“更大的危险来了 / 更高层的人来了 / 还有新的线索”但未具名，按弱章尾处理
- 若章尾声势很大，但本章最值钱收益仍是抽象判断，按 `F2`
- 章尾最后 `80-150` 字不得集中出现以下盖章词：
  - 终于
  - 总算
  - 最终
  - 这一刻
  - 接下来
  - 更大的风暴
  - 这意味着
  - 这说明
  - 这表明
  - 这证明
- 若章尾需要引下一章，优先用 `新物证 / 新备注 / 新名单 / 新屏幕口径 / 新坐标 / 新封记` 落钩，不得用作者代说式总结

---

## 六、写后必做

正文落盘后，必须：

1. 跑 `scripts/scene_lint.py`
2. 做纯净度检查
3. 人工回看最后 `20-50` 行，并逐项排：
   - 是否出现章尾盖章词
   - 是否把下一章预期写成了本章结论
   - 是否用解释句代替新钩子
4. 若脚本只报 `WARN`、未报 `ERROR`，不得把 `WARN` 直接当失败码；先判断是否真需补强
5. 生成写后验收
6. 同步角色卡与追踪

若任一步未完成，正文默认未验收，不得并入主正文。
