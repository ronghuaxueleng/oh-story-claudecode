# 正文开写前后过这份

> 这份只管正文这一步开写前后要过哪些口，像情报流、字数、章尾这些都在这里看。句子和段落层面的红线，去翻 `references/writing-red-lines.md`。

---

## 一、正文开写前先过

正文下笔前，把下面这些事先顺一遍：

- 已读取 `设定/题材定位.md`
- 已读取本章细纲
- 已完成写前闸门
- 已写明本章目标字数与上限字数
- 已为正文预留 `字数安全边际`；若脚本下限为 `2900`，首写正文目标一般别低于 `3100`
- 若属于 `冷启动首章组`，首写正文目标一般别低于 `3400`
- 已答清第一爆点与高潮后续收益
- 已答清兑现类型（单章兑现 / 章组兑现 / 情节库兑现）
- 若非单章兑现，已答清兑现组范围、本章第几拍、本章现实中间收益、最迟完整兑现章
- 已答清本章 `主要对手 / 主要物证或资源 / 场面靠什么撑住`
- 已答清本章 `主场面 / 主事件来源 / 世界主动施压靠什么落地`
- 已确认本章不是对上一章母场景的换皮复用
- 已确认本章残魂 / 金手指不是继续只做“提醒 / 判断增强”
- 若本章是首章且作品存在足以改写默认现实认知的世界规矩，已明确 `首章世界总盘交代` 与 `世界底色落点`
- 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，已明确 `世界底色续显点`
- 已确认本章 `主收益走法` 不与最近 `2-3` 章重复
- 已确认残魂 / 金手指会留下 `外部可感知后果`
- 若本章发生 `地点 / 势力 / 时间段 / 事件依托` 切换，已明确 `跨场景过桥`
- 已答清本章中后段至少 `1` 处 `外部反应 / 群体往前往后挪 / 系统屏幕变化 / 协作方把话收回去`
- 若属于 `冷启动首章组`，已答清本章至少 `2` 口外部反应，且分别落在 `中段改线` 与 `章尾前抬升`
- 已答清本章 `事件拍序` 至少 `3-5` 拍，且不是单拍结果文
- 已答清本章 `情绪回摆` 至少 `2` 次，且都绑在事件与舍取上
- 已想好章尾最后一句的 `末句功能型`，而且只能是：
  - `新动作`
  - `新物证`
  - `新说法`
  - `新名单`
  - `新翻面`
  - `下一步命令`
- 已确认本章不存在未替换的抽象占位表达
- 已确认不存在“单一力度副词”批量刷句；动作强度得写在受力点、位移、失衡、破坏结果里
- 已完成 `首稿污染预防`：`references/banned-words.md` 的 `一级禁用词` 本章正文起草时别出现，也别用生硬撑劲词、空力度词、抽象效果词去顶动作链
- 已完成 `禁用词反向污染预防`：别因为读了禁用词表，就把表内词面、生硬撑劲词或解释句黑名单词写进正文
- 已读取本章 `新增信息允许清单` 与 `正文禁增项`
- 已确认正文若需新增关键信息，就先补写细纲与写前闸门，别边写边吞
- 已答清本章 `第二轮收益` 是什么：
  - 不是第一爆点重复喊一遍
  - 而是第一爆点之后，现实里谁被迫把话收回去、挪地方、改办事顺序、改利益分配
- 已答清本章 `群体反应位`：
  - 至少有一处围观者 / 协作方 / 执行者 / 系统说法的变化
  - 别整章都只有主角和对手互顶
- 已答清本章 `章尾前一拍抬升位`：
  - 章尾前先落一处新的现实压力或新的执行动作
  - 别直接从结果句硬切到章尾钩子
- 已完成本章 `模板有没有真写进这一章检查`：
  - `本章主模板` 已写
  - `本章第一爆点` 已写
  - `本章第二轮收益` 已写
  - `本章群体反应位` 已写
  - 若属于 `冷启动首章组`，`本章第二轮外部反应位` 已写
  - `本章章尾前一拍抬升位` 已写
- 已完成本章 `模板效果检查`：
  - 已答清模板最终改了哪一步现实
  - 已答清第一爆点之后谁被迫把话收回去 / 挪地方 / 改先后 / 改利益
  - 已答清外部反应怎样改了现场
  - 已答清章尾前一拍到底抬高了哪一步压力
- 已答清本章 `书名主打兑现到了哪一步`
- 若书名主打碰上 `经营 / 配额 / 分配 / 定顺序 / 生火权 / 放票权 / 名额权`，已明确本章正文至少要写出 `谁被主角实际改了火 / 票 / 路 / 名额`
- 若本章碰上 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，已明确：
  - 本章最主要的规则信息压在哪个在场动作里
  - 本章至少哪一步 `试用 / 交易 / 排队 / 受阻 / 讨价 / 改规矩 / 利益变化` 会真正发生
  - 若删掉解释段，哪一拍现实推进仍必须成立
- 已完成本章 `动作拍清单`：
  - 至少 `4-6` 拍
  - 每拍只写 `谁做了什么 -> 谁被迫怎么改`
  - 每拍已补明 `谁被挤掉原来的地方 / 谁立刻吃损失 / 谁不能退回旧说法`
  - 别把“规则说明 / 局势总结 / 作者判词”混入
- 已完成本章 `人物冲突卡`：
  - 主角嘴上报什么、动作先护什么
  - 对手嘴上压什么、动作先藏什么
  - 摇摆位嘴上推什么、动作先避什么
  - 关键角色各自这一场会冒什么风险、准备先弃哪样、站出来立刻失去什么
  - 关键角色各自这一场最想保什么、最愿意先牺牲什么
- 若本章主打主角人物句，已写清该句准备同时压住 `私利 / 代价 / 现实改局` 里至少两项
  - 若本章主打重要角色人物句，已写清她/他先压进去的“自己人 / 自己物 / 自己名分”
  - 若本章主打重要角色人物句，已写清她/他压进去后自己当场失去的 `原来的地方 / 先后机会 / 面子 / 名分`
  - 至少 `1` 个角色会在中段临场把话收回去或挪地方，别所有人一路按原计划说到底
- 已完成本章 `章尾现场句预案`：
  - 只准属于 `新动作 / 新物证 / 新名单 / 新命令 / 新翻面`
  - 别写意义总结、黑幕升级、下一章预告
- 已完成本章 `最后一拍兑现预案`：
  - 章尾前一拍之后，最后一拍还得再砸出 `新动作 / 新物证 / 新说法 / 新名单 / 新翻面 / 下一步命令`
  - 若最后一拍属于 `新命令`，还得继续写清：
    - 谁立刻执行
    - 现场哪一样东西当场变化
    - 读者最后看到的即时后果
  - 别把最后一句收在 `别急 / 别碰 / 别先记 / 今夜还没完 / 事情不对 / 先这样` 这类压话句
  - 若最后一句只是压人、止语、预告、总结，而没有新增现实结果，就说明本章章尾预案还没过
- 已完成本章 `首稿厚度补足预案`：
  - 若首稿低于 `3100`，优先补 `后续实利 / 群体反应 / 章尾前一拍 / 现实代价 / 关键角色出场动作`
  - 别用加解释、加回忆、加设定说明来凑厚度
  - 若属于 `冷启动首章组` 且报了 `字数.warn`，优先再补 `第二层外部反应 / 争先动作 / 章尾前抢地方`
  - 若属于边界题材冷启动，厚度补足一律优先补 `现实挤压`，别优先补背景或心理总结：
    - `女频强情绪`：补 `记名后第二手争夺 / 主桌位变化 / 谁先背锅`
    - `历史争霸 / 王朝权谋`：补 `礼台位 / 军籍位 / 调兵位 / 谁先担叛名`
    - `恐怖惊悚 / 民俗悬疑`：补 `供桌位 / 后门位 / 器物位 / 谁先背祟名`
    - `无限流 / 规则副本流`：补 `一号桌位 / 后门位 / 签册位 / 谁先背死签`
- 已想好本章正文写完后的 `全文残片扫描`：
  - 明确扫描命令或扫描范围
  - 明确若碰上工具残片 / 元话语 / 非故事语料，则本章按 `F4` 停下
- 已确认本章至少有 `1` 段 `对话能压住别人`：
  - 不是交换信息
  - 而是借台词改变口令、次序、谁站哪儿、归属、谁先办中的至少一项
- 已完成本章 `解释腔替代现场结果` 预防复检：
  - 若正文已经靠器物、谁前谁后、名单、门位、倒计时、广播、纸面落字把现实结果写出来，别再补 `这就算真的起作用了 / 现在不是…… / 不是X被解释了，而是Y发生了 / 这一刻才算……`
  - 若一句话的主功能是替现场下判、替读者总结、替模板把意思定死，就回退成动作、反应、物件变化或他人被动承认
- 已确认本章涉及的 `制度口 / 流程口 / 职业术语` 已经落到白话里：
  - 第一次出现时，立刻能让读者知道“它会卡谁、放谁、害谁”
  - 别只丢术语名词，不给口语解释或现场后果
- 已确认本章不是“单事件直线推进”；至少要有：
  - `首轮阻断`
  - `中段改手`
  - `第二次翻面`
  - `现实代价或退路烧断`

## 二、句子红线入口

正文出现以下问题时，这里不再重复展开，直接按 `references/writing-red-lines.md` 处理：

- 效果词先行
- 后台调度语
- 作者下结论句
- 提纲扩写句
- 只删词不回句子功能
- 单一力度副词反复刷屏
- 未备案关键信息临时入正文
- 章尾最后一句承担意义总结 / 作者下结论 / 资源判价功能

## 三、情报流专门要求

- 正文里第一次出现情报时，先用 `「」` 单独包住
- 情报第一次出现时，得带上来源或触发方式
- 情报最好在当前章或后 `1` 章内转成现实收益
- “知道更多 / 更危险 / 更复杂”不算兑现

补充这块怎么卡：
- 若情报只负责提醒、解释、抬线索价值，但没有推动现实局面变化，就按没兑现看
- `现实收益` 至少得落到 `命 / 人 / 名 / 权 / 物证 / 有人被逼选边 / 规矩变化 / 垫背者落地` 之一
- 若首章情报只换来“对手更警惕/主角更清醒”，一般不算过
- 情报句后面，正文很快就得跟上具体动作；别写成“情报先出来 -> 主角想半天 -> 下一段才动”
- 若本章细纲已写 `本章情报条目`，正文别改成“临场观察替代情报”
- 若本章正文关键翻盘依赖名单、车、票、口令、批文、调度等关键信息，而这些信息没在情报条目或新增信息允许清单中备案，一般按 `F3/F5`
- 情报流作品前 `3` 章至少得有 `2` 次“情报 -> 动作 -> 外部现实变化”的完整链；不够就说明书名主打开始写偏了

规则播报分开判断补充：
- `广播播报 / 系统播报 / 轮次提示 / 复核口令 / 倒计时变化 / 规则条目亮起` 不自动等于情报流
- 若它的主功能只是让当前场面改说法、换地方、改顺序、改办事次序，正文就别硬塞情报引号块去凑情报流
- 只有当播报文本本身会变成后面跨章追踪的东西，而且能单独答清 `它到底带来了什么新信息、最迟什么时候兑现、兑现后改掉哪块现实`，才往情报流上提
- 规则副本 / 系统文若只是“规则执行次序在现场改局面”，一般按功能场景处理，别硬按情报流处理

## 三点一、书名主打结果化兑现

- 正文别把书名主打兑现写成“更接近、快要摸到、差一点拿到”
- 每章至少得答一个结果化问题：
  - 主角这一章真实掌到哪一块分配权 / 放票权 / 车权 / 名额权 / 调人调货的权
  - 若没有掌到，是否至少逼敌方公开改动作 / 改规则 / 挪地方
- 若书名主打是经营 / 配额 / 分配 / 执牌 / 夺权类，连续 `2-3` 章都只做到“掀黑、截物证、揭脏手”，一般按 `F3`
- 若书名主打碰上 `定顺序 / 生火权 / 名额权 / 放票权 / 分火权`，正文中得尽快落出以下至少一种实答结果：
  - 主角点了谁先进 / 先拿 / 先走
  - 主角卡了谁不能进 / 不能拿 / 不能走
  - 敌方被迫按主角的新顺序执行
- 若整章只能回答“主角拿到一张签 / 一份资格 / 一份自保位”，一般最高只算 `D2`，别写成“已定顺序”

## 三点二、章节组反重复

- 连续 `2-3` 章别反复使用同一翻盘模板，例如：
  - 物证见光 -> 群众炸锅 -> 现场执行顺序让步
  - 名单见光 -> 群众炸锅 -> 临时把原话收回去
  - 关键物见光 -> 对手失态 -> 章尾再抬高层敌意
- 若最近 `2-3` 章已经复用同模板，本章正文就得主动换打法：
  - 从抢物证切到抢人 / 抢先开口定事 / 抢谁先办这条规矩
  - 从围观炸锅切到正式秩序对撞 / 调度对撞 / 群体改做法
  - 从掀基层脏手切到咬中层局面里的敌人
- 若正文写到中段才发现还在复用旧模板，就立刻停写，回退细纲，按 `F3`

## 三点三、单章事件拍序与情绪回摆

- 正文别写成“单章一个事件名词 + 单拍结果”
- 每章正文至少得有下面这套最小结构：
  - `起目标`
  - `遇受阻 / 被压回来`
  - `变打法 / 舍掉一样`
  - `现实改局`
- 若篇幅或题材确实偏短，最低也得满足：
  - 至少 `1` 次中段受阻
  - 至少 `1` 次中段变招
  - 至少 `1` 次现实变化
- 本章情绪至少 `2` 次回摆，别只有单一路情绪，例如：
  - 一路冷静
  - 一路查线
  - 一路强压
  - 一路愤怒
- 情绪回摆得通过以下现场东西落地，不认纯旁白解释：
  - 动作停顿 / 失真
  - 临场把话收回去
  - 错判后改决断
  - 快拿到又被打断
  - 代价突然落下
  - 人站的地方 / 队形 / 协作关系改变

## 三点二点八、模板落进去后的首稿厚度补足

- 模板落进细纲，不等于正文就算写到位；若只是“细纲里写了模板内容”，正文却没把模板写成看得见的现场结果，一般按未验证看
- 若项目脚本下限为 `2900`，正文首稿低于 `3100` 时，别先写模板已经验证成功
- 若属于 `冷启动首章组`，正文首稿低于 `3400` 时，一般按厚度不足预警处理；除非细纲明确要求短章
- 首稿低于 `3100` 时，固定回补顺序为：
  - 先补 `第二轮收益`
  - 再补 `群体反应 / 谁前谁后变化`
  - 再补 `章尾前一拍`
  - 再补 `现实代价 / 退路烧断`
  - 再补 `关键角色出场动作`
- 若属于 `冷启动首章组` 且正文低于细纲目标 `120` 字以上，额外固定回补：
  - `第二轮外部反应`
  - `中段争先 / 争位 / 争顺序`
  - `章尾前最后一拍抬升`
  - `当前场景里的现实挤压`
- 回补时别拿下面这些假厚度充数：
  - 解释规则
  - 总结局势
  - 回忆补设定
  - 作者判词
  - 重复第一爆点换句说
- 若报了 `章尾.error` 且最后一句只是压话 / 收尾 / 预告，回补优先级高于补氛围：
  - 先补“最后一拍新增结果”
  - 再补“章尾前一拍承压”
  - 最后才考虑补句子质感
- 若首稿补字后还是答不出“谁把原话收回去、谁被迫改做法、谁先后被改、谁到手的好处被改掉”，一般就说明模板只打出第一步，按 `F3`

## 三点二点七、后续实利硬规则

- 每章用到模板后，同章里至少得落出 `1` 处 `后续实利`
- `后续实利` 至少得写到这一步：
  - 第一爆点出现后，有现实人群、办事次序、分配顺序、协作动作、对手说法中的至少 `1` 项被迫变化
- 以下内容不算后续实利：
  - 主角自己觉得更稳
  - 对手只是脸色变了
  - 围观者只说“原来如此”
  - 章尾只抬下章风险，没有改当前局面
- 若章节主打碰上 `经营 / 分配 / 资格 / 放票 / 交易 / 拍卖 / 试炼 / 审批 / 定顺序`，后续实利优先落在：
  - 顺序被改
  - 资格被改
  - 票 / 货 / 路 / 名额归属被改
  - 谁能先动手被改
- 若整章只有第一爆点，没有后续实利，即使章尾有钩子，最高只算 `D2`

## 三点二点五、模板有没有写进去、有没有真起作用

- 模板检查必须拆成两段，不能只看“有没有写模板名”
- `模板有没有真写进这一章检查` 负责验这些内容：
  - `本章主模板`
  - `本章第一爆点`
  - `本章后续实利`
  - `本章群体反应位`
  - 若属于 `冷启动首章组`，`本章第二轮外部反应位`
  - `本章章尾前一拍抬升位`
- 若缺项，或只有名词没有落点，默认按 `F2`
- `模板效果检查` 负责验结果：
  - 模板是否改了现实局面
  - 后续实利是否真实改说法 / 让人挪地方 / 改先后 / 改利益
  - 外部反应是否真实改线，而不是围观喊话
  - 章尾前一拍是否真实抬高当前章压力，而不是预告下一章
- 若这些内容都写了，但效果答不稳，默认按“模板只是挂了个名，还没真正写透”处理，至少 `F2`

## 三点二点六、冷启动样本最低过关线

- 冷启动测试不能只验证“模板内容已经进了细纲”
- 冷启动样本要判定为验证成功，至少同时满足：
  - 细纲里已写出模板内容与落点
  - 正文里已打出第一爆点与后续实利
  - 正文里已打出至少 `2` 处不重复的外部反应
  - 追踪文件已同步
  - 章组写后检查总表已生成或更新
  - `scene_lint.py --json` 没报实质性失败项
- 若只完成细纲和正文，没有追踪闭合或章组总表，默认只算“半验证”
- 若正文只是把模板内容写成提示词式口号，没有落成事件链，默认按验证失败处理

补充边界题材人物薄化预警：

- 若题材碰上 `末世经营 / 规则经营 / 公开名额争夺 / 配给 / 放号 / 能不能进 / 排队进场`，且首章组明显采用“短章高压推进”，必须额外复检：
  - 第三方是否只剩“把话收回去的人 / 记账的人 / 喊价的人 / 抢地方的人”
  - 关键配角是否只有功能，没有 `私心 / 自保 / 风险 / 延迟反应`
  - 群体是否只有“抢地方”这一种作用，没有任何能被记住的个人毛边
- 还必须继续检查：关键配角是否缺少以下任意 `2` 项及以上：
  - 怕点
  - 私利 / 私心
  - 被逼选边的代价
  - 个人控场方式
- 只有在“功能化迹象成立”且“上面四项至少缺 2 项”同时碰上时，才记为 `人物薄化预警`
- `人物薄化预警` 不直接升成硬检查，但别再把“模板效果已通过”误判成“人物出场分量也已通过”
- 边界题材冷启动复盘时，必须单列：
  - `模板是否通过`
  - `人物是否被推进功能挤薄`
  两者不得混写成一项

别急着算过的情形：
- 本章只有“去做一件事，然后做成/做败”
- 中段没有第二次变向，只是在拉长同一动作
- 所谓情绪拉扯只靠“他很怒 / 很痛 / 很纠结 / 很不甘”
- 章中没有“快拿到 -> 被打断”或“被打断 -> 改打法”
- 结尾虽然有结果，但读者几乎没经历过程里的希望、打断、舍取、代价
- 关键角色除了支持主线功能外，没有任何个人算盘、私心取舍、自保动作
- 所谓对白很多，但没人借对白去改现场秩序，只是在轮流把信息说出来
- 本章正文写完后，若还没先做全文残片扫描，就别直接进下一章，也别继续批量写后章

## 三点三点二、章内事件拉扯硬规则

- 单章主事件默认别写成单线跑图；至少要出现以下 `4` 项中的 `3` 项：
  - 第三方插刀
  - 盟友私心或短视坏事
  - 世界压力临时改局
  - 主角主动放弃一条更便宜的解
- 若碰上 `L3/L4`，则上面 `4` 项里至少实装 `3` 项，且必须让其中 `2` 项落到可见动作，不认纯旁白说明
- “大场面”不等于人多；若人多却没有改线、踩踏、争先、抢关键东西、改批次、改归属，默认只是背景噪音
- 章中至少要有一次“原本以为要抢 A，临场改成先保 B”的取舍，不得一路照着最初目标顺推
- 若主角每次都能选到更强、更快、更正确的解，而没有丢掉任何一头，默认拉扯不足

## 三点三点一、三章小循环硬规则

- 连续 `3` 章默认要形成一个小循环，不得三章都写成并列任务章
- 默认位置：
  - 第 `1` 章：`起火`
  - 第 `2` 章：`升压`
  - 第 `3` 章：`兑现`
- 若当前章承担 `升压`：
  - 正文必须扩大阻力、代价、两难、关系拉扯中的至少 `2` 项
  - 不得只重复上一章打法
- 若当前章承担 `兑现`：
  - 正文必须至少改掉一处现实局面
  - 不得只留下新线索、新名单、新猜测

先别急着算过的情形：
- 第 `2` 章只是把第 `1` 章再做一遍
- 第 `3` 章没有兑现，只是再抬更高层黑幕
- 三章都只有局部胜负，没有形成一轮完整情绪释放

## 三点三点零、功能场景别写成讲解稿

- 功能场景不是不能讲规则，而是不能让“讲规则”替代“场景主功能”
- 以下场景默认都要额外做“别写成讲解稿”复检：
  - 卖货 / 看房 / 食堂 / 配套
  - 配额 / 名额 / 资格 / 放票 / 能不能进
  - 审批 / 审查 / 封存 / 试炼 / 考核
  - 流程口 / 制度口 / 规则口 / 服务口
- 若正文某段同时满足以下 `5` 条中的 `4` 条，默认该段写成了讲解稿，至少按 `F2` 回退：
  1. 该段场景目标本身是制度、流程、福利、玩法、资格或规则入口
  2. 连续 `4` 句及以上主要功能是在解释规则，而不是推动在场人物动作
  3. 该段没有明确的 `试用 / 交易 / 排队 / 讨价 / 受阻 / 改规矩 / 利益变化`
  4. 删掉该段后，人物关系和场面推进几乎不受影响
  5. 该段内容能直接改写成产品说明 / 制度公告 / 规则介绍，而意思几乎不变

硬执行：
- 碰上上面判据时，不得仅按“解释腔偏多”降成 `F1`
- 若只是某一句在解释，可按 `F1` 修句
- 若整段主功能已经变成讲解稿，默认按 `F2`
- 若连续 `2` 章以上都把功能场景写成讲解稿，或因此导致书名主打一直只停在“知道规则更多”，默认升级为 `F3`

排误伤规则：
- 规则多，不等于讲解稿
- 若规则信息始终压在 `点单 / 试吃 / 结账 / 排队 / 抢地方 / 争执 / 把话收回去 / 改规矩 / 受阻 / 外部谁前谁后变化` 里，不得仅因信息密度高直接判死
- 若删掉规则信息会直接削弱当前场景的交易、争抢、局面变化，也不得误判成讲解稿

### 功能场景四步生成顺序

- 碰上功能场景时，正文起草顺序照下面来：
  1. `谁想拿什么`
  2. `谁先动手`
  3. `谁被迫改动作 / 把话收回去 / 挪地方`
  4. `规则因此显形`
- 禁止顺序：
  1. 先讲制度
  2. 再讲规则例外
  3. 最后才让人物去执行
- 写前若答不出前 `3` 步，说明该场景还只是讲解提纲，正文别先写
- 正文中若一段先把规矩讲了一大段，却还没出现 `争 / 试 / 卡 / 放 / 改 / 挡 / 抢 / 压` 任一动作，默认该段优先按讲解稿处理
- 功能场景的信息量，至少只能答清眼下这场：
  - 这条规则现在压到谁
  - 谁正借它吃人
  - 谁准备把它扳回来
- 不得顺手把“完整制度全景 / 所有例外情况 / 常规运转全貌”一起讲完

### 规则显形允许写法

- 功能场景里，规则只能优先通过以下现场东西显形：
  - 门
  - 票
  - 名单
  - 夹板
  - 屏幕
  - 广播
  - 印章
  - 回收单
  - 锁闸
  - 谁前谁后变化
- 若规则信息不附着在上述现场东西和在场动作上，而是只靠角色把意思念出来或旁白讲解，一般先别算过

## 三点三点零点一、首稿里别让旁白替读者下判断

- 正文首稿里，旁白别替读者把下面这些判断先说完：
  - 判谁更狠
  - 判这一下意味着什么
  - 判人物真正怕什么
  - 判这句为什么更硬
  - 判规则到底卡在哪一步
- 碰上下面这些句法，一般先记 `F1`；要是扎堆出现在开头 / 中段关键翻面 / 章尾，通常升 `F2`：
  - `这一下比……更……`
  - `这不是……，是……`
  - `从这一刻开始`
  - `真正……的是……`
  - `不是怕A，是怕B`
  - `不是为了A，是因为B`
  - `他不是在……，是在……`
  - `这才算……`
- 正确修法只有一种：
  - 回到前一句
  - 改成 `手 / 眼 / 门 / 票 / 屏 / 钟 / 伤 / 呼吸 / 人站的地方 / 物件状态`
  - 让读者自己得出判断
- 删掉以后段落更顺，就直接删，别再补一版差不多意思的总结句
- 同章如果连续 `2` 句以上都在替同一拍下判断，就说明这一拍根本没写进场里；别只删一半留一半

## 三点三点零点二、对白得像活人在顶

- 高压对话里，多写 `打断、抢词、避答、说漏嘴、硬压、借物件说话`，少写一整套完整论证
- 一段关键对白如果整段都能改写成“情况说明会记录”，那基本就没人味了
- 人物说话得受身份、处境、疼痛、喘息、时间压力拖着走；刚打完、刚摔过、刚被掐住退路的人，不该一直稳稳当当说工整判断句
- 同章里要是好几个人连着说出同一类句式、同一类判断结构、同一种摆出来的话，就按 `F1/F2` 回修
- 关键对白得额外过一次“张嘴测试”：念出来拗口、像宣读制度、像书面汇报、像作者借嘴发言，碰上一条都不算过

## 三点三点零点三、术语得落回人话里

- 正文里可以有题材术语、职业术语、制度术语，但别让术语干巴巴杵在那儿
- 术语第一次出现时，至少得同时绑上下面一项：
  - 谁会因为它先走 / 先拿 / 先活
  - 谁会因为它被卡 / 被罚 / 被挤下去
  - 一个当场能看见的东西
  - 一句更白的话把它翻给读者听
- 如果术语得让读者自己停下来翻译，一般就别急着算过
- 下面这些类型一般优先回看：
  - `轮次 / 批次 / 配额 / 指标 / 权限 / 流程环节 / 执行次序 / 资格环节 / 调度环节` 这类制度摘要词
  - 一句里连续塞 `2` 个及以上术语
  - 角色明明在急场里，却还在稳定使用长串职业名词
- 正确顺序：
  1. 先让读者看懂这东西在卡谁
  2. 再顺手知道这东西叫什么
  3. 术语只负责增厚世界，不负责替代冲突
- 别按这个顺序来：
  1. 先报术语名
  2. 再补制度说明
  3. 最后才告诉读者为什么眼下要命
- 如果术语删掉后读者反而更顺、更快懂当前冲突，就优先删掉或换成白话，别为了“显得专业”硬留着

## 三点四、字数别总贴着线跑

- `scene_lint.py` 的下限不是目标字数；正文首写就得主动多留一点余量
- `scene_lint.py` 的字数结果分为两层：
  - `ERROR`：明显过短 / 过长，直接拦
  - `WARN`：接近下限 / 偏高，只做提醒，不单独拦
- 若项目脚本下限为 `2900`，则：
  - 首写正文推荐目标 `3300-3800`
  - 别把 `2900-3000` 当成交稿目标
- 若细纲目标字数已低于脚本下限上浮 `100` 字，写前闸门把目标先抬高
- 最近 `2` 章如果都报了 `字数.warn`，下一章首写目标至少再往上提 `100`
- 细纲或写前闸门如果写了目标下限，正文首稿低于这个下限 `200` 字以上，就按 `F2`，先别进写后检查
- 正文首稿如果低于 `scene_lint` 下限，就按 `F2`，别再拿“节奏快 / 可读性强”直接算过
- 正文首稿如果低于 `3100`，就算脚本没报错，也得人工看看是不是“正文主体不够”；别默认 `F0`
- 项目脚本下限如果是 `2900`，正文首稿一旦低于 `3100`，就进 `配额回补态`，别随手补几句凑线

`配额回补态` 就这么补：
  - 前 `1/3` 得补进 `1` 处新的受阻 / 截堵 / 压回来，别只是把原动作拖长
- 中段得补进 `1` 处第三方插手 / 协作方把话收回去 / 围观人往前往后挪
- 后 `1/3` 得补进 `1` 处现实代价 / 关系变坏 / 局面改线 / 外部可感后果
- 关键角色至少补进 `1` 处亮相动作：
  - `身体先泄露，台词后补位`
  - `嘴上说A，动作护B`
  - `被问A，绕去答B`
- 回补别落成解释句、总结句、作者替读者收尾的句子
- 可以补细节，但细节只能挂在上面这 `4` 类配额上，别单独起一段只为撑字数
- 细节只允许补：
  - 动作打点
  - 物件磨损 / 状态变化
  - 身体先泄露
  - 环境如何直接压到当前动作
  - 规则如何落到当前职业习惯 / 群体反应
- 不允许把细节写成纯空镜头、纯情绪句、纯气氛句
- 上述 `4` 类配额任一未补满，就别重新统计字数，写后检查也先别进
- 如果当前差字主要在 `200-600` 字，一般先补 `1` 个微事件或 `1` 轮带目标的短对话，别先补总结句
- 如果当前差字主要在 `600-1200` 字，一般先补 `1` 个完整小场面：`受阻 -> 把话收回去/改手 -> 现实变化`
- 所谓“带目标的短对话”，至少得改掉下面一项：
  - 说法
  - 次序
  - 谁站哪儿
  - 归属
  - 谁先办
- 别用下面这些办法补字数：
  - 白话解释主角已经想明白的事
  - 重复上一段结论
- 用抽象压人话、感慨句、作者下结论句顶行数
  - 单独补“他更清楚了 / 他更确定了 / 他知道后面会更难”这类判断句
- 如果补字后删掉那一段，当前章的冲突、兑现、人物或局面变化都不受影响，那就算无效补字，得重新补事件或对话

先别急着算过的情形：
- 正文多次停在 `2900` 附近擦线
- 正文连续停在 `3000` 左右、虽能通过却反复触发接近下限告警
- 通过补一两句机械凑字来过脚本
- 正文主体不足，只在章尾补解释或总结凑到下限

## 三点四点一、首稿先自己扫一遍污染

- 正文首稿写完后，进写后检查前自己先扫一遍污染
- 自检范围：
  - `references/banned-words.md` 的一级禁用词
  - 生硬撑劲词污染
  - 解释句黑名单词面
  - 工具残片 / 补丁标记 / 非故事语料
- 自检结果得按结构记下来：
  - 一级禁用词是否碰上
  - 生硬撑劲词是否碰上
  - 全文残片扫描是否通过
  - 若任一失败，失败码是什么
- 碰上任一项就这么处理：
  - 默认 `F1/F4`
  - `F0` 写后检查先别进
  - 生产模式下得当场回修本章并重跑首稿污染自检，别把问题往 `scene_lint`、写后检查或 `story-review` 后面拖
  - 生产模式下自动切入 `H1-B 生产自动返修态`
  - 纯基准测试下自动切入 `H1-A 纯测试记录态`，只记录失败，不修正文
- 别把“只是一个词”降成观察项；一级禁用词就是首稿失败项

## 三点四点二、章尾最后一句是干活的

- 正文写完后，得单独看一眼最后一句还符不符合写前登记的 `末句功能型`
- 最后一句只干一件事：把章尾停在新的现场结果上
- 下列功能一旦落在最后一句，就按 `F1/F2`：
  - 意义总结
  - 局势翻译
  - 作者判价
  - 资源判价
  - “真正要命的是 / 现在最该抢的是” 这类替读者算账
- 如果最后一句删掉后反而更强，就直接判章尾功能写错了，不算过
- 如果最后两句删掉后本章照样成立，就说明章尾在补缝，不算过
- 如果最后一句主要是在告诉读者“下一章该去哪 / 这件事该怎么理解 / 这一拍为什么重要”，那就按章尾不过来处理
- 下面这些章尾句型一般优先回看：
  - `原来……`
  - `这说明……`
  - `这意味着……`
  - `接下来……`
  - `得去……`
  - 单独拿一个字 / 一个概念 / 一个方向出来替下一章盖义

正确修法只有这一种：
- 若前一拍已成立，删掉错误末句
- 若前一拍未成立，回补前一拍的动作 / 物证 / 外部反应 / 他人失态 / 现实变化
- 别直接把错误末句换成差不多意思的解释句

## 三点四点三、解释句要分清到底坏在哪

- 碰上解释腔 / 下结论句 / 作者替读者收尾的句子后，还得继续往下看根因：
  - `G1 信息抢跑`
  - `G2 现场东西不够`
  - `G3 提纲语言漏出`
  - `G4 人物失声`
- 还没归到 `G1-G4` 之一，就说明检查还没做完
- 修法要跟根因绑死，别混着修：
  - `G1`：把信息后移到决策压力点或现实变化点
  - `G2`：回前一拍补动作、后果、代价、失态、外部变化，别只删句
  - `G3`：删除结构调度语，改为人物当下可见可做内容
  - `G4`：改回人物动作、对白、错判、把话收回去来写
- 同章如果连续出现 `2` 处以上 `G2`，就说明本章场面根本没写进场里，按 `F3` 回退细纲或场面卡

## 三点四点四、对白得说人话

- 对白的第一职责，是让角色彼此攻防，不是替作者向读者讲解
- 写对白前，先答出这句眼下是在干什么，只允许落在下面这些里：
  - 压人
  - 试探
  - 光靠嘴压判断
  - 拖时间
  - 甩锅
  - 自保
  - 逼人让地方
  - 掩饰
  - 挑衅
  - 求活
- 若一句对白的主要作用是：
  - 讲设定
  - 讲规则全貌
  - 讲完整方案
  - 替读者下结论
  - 替人物总结意义
  默认至少 `F1`；高压场景里连续出现 `2` 句及以上，默认升 `F2`

### 对白里这些信号别急着放过

- 一句对白同时完整交代：
  - `事实`
  - `判断`
  - `结论`
  - `方案`
  任意 `3` 项及以上，就默认像在做说明，至少 `F1`
- 高压场景下，人物一句话如果超过 `2` 个并列分句，而且分句都在“解释 / 说明 / 归纳”，一般优先回看
- 角色把本来能靠动作、物证、谁前谁后看出来的东西，又用台词说一遍，就按“对白在替动作补缝”处理
- 反派 / 高位敌人如果主动把自己的脏手、把柄、受贿说法、关键护点讲得太明，一般就先别算过
- 主角在极险、极乱、极快的场景里，如果频繁说出那种像专门摆好的完整短句，一般优先回看
- 高压场景里人物如果频繁说出“制度播报腔 / 公告腔 / 会议纪要腔”，默认至少 `F1`

### 这些句型要重点回看

- 以下句型若在对白中高频出现，一般优先复检：
  - `不是A，是B`
  - `你要的不是A，是B`
  - `给我活路的不是……`
  - `我来的不是……`
  - `真正……的是……`
  - `你一句……就想……`
  - `今夜这件事……先……不……`
  - `谁现在……，谁就……`
  - `不是你……拍板`
- 上面这些句型单句偶发，不直接定罪
- 同章同角色碰上 `2` 次以上，或者连续 `2-3` 章反复拿它们当发力句，就默认对白这一层先别算过

### 高压场景对白写法硬要求

- 碰上以下场景之一：
  - 截堵
  - 抢口
  - 战斗
  - 追逐
  - 谈判翻桌
  - 临场夺权
  - 灾异逃生
  - 众目之下顶牛
  必须优先：
  - 短句
  - 半句
  - 抢话
  - 被打断
  - 动作插入
  - 岔答
- 不得优先写成：
  - 完整论证
  - 漂亮排比
  - 像提前写好再照着念的书面话
  - 规则宣讲
  - 立场总结
- 若冲突里最像“人话”的不是对白，而是旁白在替人物讲话，默认判 `G4 人物失声`

### 对白与动作分工硬规则

- 规则、制度、方案、流程，不得主要靠人物把意思念出来完成
- 若一句对白删掉后，现场通过动作 / 物证 / 谁前谁后仍能成立，优先删对白，不得硬留“解释型台词”
- 若一句对白删掉后，读者看不懂，那先补动作、物证、承压反应，不得先补另一句解释台词
- 角色说话时，至少应有一种同步的现场东西：
  - 手上先护什么
  - 身体先退还是先顶
  - 目光先找谁
  - 被问A时先去碰B
- 说压场的话时先护自己的口袋 / 令牌 / 名单 / 人

### 对白终检三问

- 每段关键对白写完后，得再问一遍：
  - 这句话是说给对手听，还是说给读者听
  - 这句话如果换个人名，是否谁都能说
  - 这句话念出来像吵架 / 顶牛 / 试探，还是像演讲 / 公告 / 作者总结
- 任一答成后者，一般先别算过，写后检查先别进

## 三点五、正文别写成一堆空指代

- 正文里的关键位置，别落成下面这些空指代：
  - 某个人
  - 某件东西
  - 某个势力
  - 一份物证
  - 更高层的人
  - 新的交接点
  - 外部突然生乱
- 如果细纲已经具名 `主要对手 / 物证 / 资源 / 场面靠什么撑住`，正文首轮出场时就得尽快落名，别又打回抽象词
- 如果正文把细纲里的具名大事件又写回“匪患 / 混乱 / 追杀 / 证据 / 调令”这类通用词，就按兑现降级看

## 三点五点一、正文别临时塞新东西救场

- 正文别临时塞进没备案的关键信息去救场面、救章尾、救收益
- `关键信息` 至少包括：
  - 新具名人物 / 新具名势力 / 新具名地点
  - 新关键物证 / 新关键资源 / 新接头点
  - 新规则变化 / 新关键时间钉 / 新关系变化
  - 新的残魂 / 金手指 / 系统能力说法
- 写着写着如果发现确实得加：
  - 先停正文
  - 先补写细纲 `新增信息允许清单`
  - 再补写写前闸门
  - 必要时同步卷纲 / 伏笔 / 角色卡准入说法
  - 完成后才可继续正文

先别急着算过的情形：
- 为了让章尾更硬，临时多出一个新接头点 / 新暗号 / 新账簿 / 新路线
- 为了让冲突成立，临时补一个之前没预热的新人物 / 新势力
- 为了让主角翻盘，临时补一个细纲没写过的新能力说法
- 为了补场面，正文突然出现细纲未备案的新规则或新世界设定
- 为了让读者理解“这项资源升级了”，正文直接补一句 `更要紧 / 更硬 / 更实 / 这一项更…… / 比上一项更……`
- 为了让章尾像钩子，正文直接用 `现在真正该抢的是…… / 最该抢的已经不是……`

## 三点六、关键设定出手得让人看见

如果本章细纲写了 `本章关键设定参与方式`，正文就得让读者看见“出手结果”，别只停在解释上。

可照着这个方向写：
- 借体后完成普通肉身做不到的动作
- 压迫后让对手失神、失口、露底
- 共鸣后拆穿禁印、伪装、假货
- 刺感后抢在群体变化前先动一步

先别算过的情形：
- 只写“主角忽然觉得不对”
- 只写“残魂提醒了他”
- 只写“他判断出了问题所在”
- 结果仍完全像普通聪明人能做到的事，且没有任何可见反应差

## 三点七、L3/L4 场面得真正落地

凡是细纲标了 `L3/L4` 的章节，正文都得写出：
- 群体规模感
- 被改写的路线 / 秩序 / 营地 / 资源 / 人站的地方
- 至少一种能撑住强场面的具体东西

若正文只剩：
- “场面大乱”
- “所有人都慌了”
- “局势一下失控”

但如果没有具体现实变化，就按规模没兑现处理。

## 三点八、别把老场面换个壳再写一遍

- 正文别把上一章的 `主场面` 只换个地名或道具就再写一遍
- 如果上一章主场面属于下面这些类，正文就默认很容易回流：
  - 囚车 / 押解路
  - 尸体 / 验尸
  - 驿站 / 旧棚 / 临时窝点
  - 公验 / 对质 / 逼口 / 翻供
  - 小股灭口 / 小队试探
- 碰上这种高风险回流，正文就得尽快切出新的主事件依托，比如：
  - 军粮车队
  - 矿场暴乱
  - 山寨并线
  - 黑市洗货
  - 关城封锁
  - 仓储争抢
  - 灾异冲营
  - 禁地异动

先别急着算过的情形：
- 还是围着车、尸、驿、口供、规矩翻脸打转
- 只是把“验尸”换成“公验”
- 只是把“驿站”换成“黑店 / 破棚 / 偏院”

## 三点九、得先让世界压到主角脸上

- 本章大冲突先由 `世界 / 势力 / 环境 / 资源争夺` 压到主角脸上，再由主角反切
- 别老沿用“情报出现 -> 主角去查 -> 对手来灭口 -> 主角翻面”这一条小查线
- 正文得尽快交代清楚：是谁先动、动了哪条路 / 哪批货 / 哪个营 / 哪道关 / 哪群人

先别急着算过的情形：
- 主角又是唯一发动者
- 敌人只来一波小灭口，没有更大局面变化
- 情报仍主要服务于“找人 / 验尸 / 套口供”

## 三点十、残魂 / 金手指别老只用一种出手法

- 如果本章有残魂 / 金手指直接出手，正文就得尽快让读者看见它是怎么出的手
- 连续两章别都只写成：
  - 提醒危险
  - 判断更准
  - 提前察觉
  - 单纯识别问题
- 上一章如果主要用了 `识破类`，这一章就优先换成 `压迫类 / 夺势类 / 超常动作类 / 异象共鸣类`
- 这一章如果还沿用低效出手法，就得额外补出一次看得见的现实变化，别只让主角“更聪明”
- 残魂 / 金手指出手后，正文得尽快给出至少 `1` 个外部能感觉到的后果：
  - 对手失神 / 失口 / 误判
  - 器物 / 阵纹 / 骨灯 / 封印 / 环境出现反应差
- 旁人因为不合常理结果往前往后挪、把话收回去、改动作

先别急着算过的情形：
- 只有主角自己知道残魂刚才出手了
- 外部人和环境完全无反应

## 三点十一、首章得先把世界底盘亮出来

- 如果本章是首章，而且作品里有足以改写默认现实认知的世界规矩，那正文前 `1/3` 就得同时做成两件事：
  - 让读者知道自己在读什么世界
  - 让读者看见这套世界规则如何现场改动作、改规矩、改生死
- `首章世界总盘交代` 至少得答清：
  - 这到底是个什么世界
  - 这个世界的制度 / 日常 / 生存方式为什么会这样运转
  - 主角当前死局为什么正好由这套世界规则制造
- `世界底色落点` 得落到动作、规矩、环境、职业常识、群体反应、制度执行里的至少 `2` 项，别只报术语名
- 可以用具体细节把世界底色写出来，但得满足：
  - 细节对象直接属于这套世界的生存规矩
  - 细节结果直接改了排队、争抢、能不能进、躲避、职业动作中的至少 `1` 项
  - 别只写“世界很怪 / 很险 / 很压抑”这种空判断

可照着这个方向写：
- 王朝存在，但驿路、军路、流放、劳役本来就和异兽、灾潮、修行资源绑定
- 都市存在，但交通、治安、职业、阶层已被异能或黑科技长期改写
- 校园存在，但升学、社交、竞争、惩罚规矩本来就依赖超凡规则

先别急着算过的情形：
- 首章只有局部奇观，没有世界总定义
- 读者前半章仍会误判成普通古代 / 普通都市 / 普通现实
- 世界规矩只存在于主角脑内旁白
- 只有边地 / 副本 / 行业特殊，没有交代整个世界为什么能容纳这套特殊
- 首章前 `1/3` 连续使用世界说明句，却没有让至少一条规则通过现场执行先裁到主角脸上

## 三点十二、非首章也别把世界味写没了

- 如果题材碰上 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，第 `2-5` 章别又把世界写回纯世俗层
- 至少每 `2` 章得出现一次 `世界底色续显`
- 续显得落到现场规矩、环境余波、器物用法、行业避讳或群体反应上

先别急着算过的情形：
- 第1章有玄异，第2章起只剩现实冲突
- 所有玄异内容都缩回主角脑内

## 三点十三、主收益走法得轮着换

- 连续 `2-3` 章别反复用同一套主收益走法
- 尤其别长期复用：
  - 情报指向关键物
  - 残魂压一拍逼失口
  - 主角抢下物证 / 账 / 令
  - 章尾再抬更高层敌意
- 正文得在收益打法上轮着换，比如：
  - 抢人
  - 改规矩
  - 收人到自己这边
  - 逼敌人自毁
  - 借灾异翻盘

先别急着算过的情形：
- 场景换了，收益仍只是“再抢一份更要紧的东西”
- 主角每章都靠同一招数收尾

## 三点十四、跨场景得把桥搭出来

- 如果本章相对上一章切了 `地点 / 势力 / 时间段 / 事件依托`，那正文开章前 `100-200` 字里至少得补清下面 `2-3` 项：
  - 主角为什么来
  - 主角怎么来的
  - 主角靠什么锁定这里
  - 主角为什么正好撞上本章第一爆点
- 上章章尾如果只抛出“营里要出事 / 新据点有问题 / 某路会炸”，下章正文别直接跳成主角已经在现场
- 如果本章是 `L3/L4` 大场面切入，正文先给主角一个短决策或短追踪动作，再起爆，别纯靠作者调度硬切
- 过桥得是人物行动链，别拿作者代说顶过去

可照着这个方向写：
- 上章逼出敌方内部有人先绷不住，下章先写主角看见异常信号、判断对方今夜必内乱，因此先摸到关键入口
- 上章拿到名册，下章先写主角顺名册上的接头点、车辙、运输路线或人物余波赶去新场

先别急着算过的情形：
- “主角赶到时”直接顶开全章
- “事情果然爆了”代替行动链
- 章首全是大场面，主角为什么在场只能靠读者脑补
- 不是人物追过去，而是作者把人物搬过去

## 三点十五、中后段得看见外部人也被带着动

- 正文中后段至少得出现 `1` 处看得见的 `外部反应 / 群体往前往后挪 / 协作方把话收回去 / 系统说法变化`
- 这一下反应别只来自主角或主要对手，优先落在：
  - 围观者
  - 执行层
  - 协作方
  - 平台 / 屏幕 / 轮播 / 群消息
  - 后勤 / 司机 / 兵丁 / 驿卒 / 船工 等现场承压人
- 目的不是凑热闹，而是把“局面翻了”写成别人也被迫改动作

先别急着算过的情形：
- 全章只有主角和对手在互顶
- 后补字数时只能往解释句或心理句里塞
- 明明发生公开争夺，现场旁人却像不存在

## 三点十五点一、人物出场别只剩功能

- 正文里的关键角色，别只剩 `身份标签 + 功能动作`
- 每个关键角色这章至少得落下 `1` 枚独有记忆点，优先落在：
  - 外形
  - 动作习惯
  - 说话节奏
  - 观察偏好
  - 压人时的样子
- 关键配角别只负责：
  - 认出
  - 解释
  - 害怕
  - 失口
  - 附和主角判断
- 关键配角如果参与了本章翻面，正文就得让他至少有一次按自己利益做出的动作
- 可以用具体细节补人物亮相，但只能补：
  - 独有动作习惯
  - 护物动作
  - 观察偏好
  - 说话前的身体泄露
  - 与职业/身份强绑定的小习惯
- 别把人物细节写成一串通用反应：
  - 皱眉
  - 发抖
  - 脸白
  - 沉默
  - 握拳
  删掉人名后谁都能套上，就默认不算亮相
- 多人对话场景若删去人名后仍分不出主角 / 对手 / 重要配角，至少按 `F2`
- 重要角色若正文只说“强 / 美 / 危险 / 压人”，没有能让读者自己补画面的细节，至少按 `F2`
- 关键人物的欲望、损失点、私心、他站哪边，别主要靠旁白直译；正文至少得落下：
  - `1` 处身体先泄露
  - `1` 处台词回避 / 岔答 / 装硬
  - `1` 处动作先护住他最不肯撒手的东西
- 同一关键角色若出现 `2` 句及以上“旁白替他讲算盘”的直译句，至少按 `F2`
- 同一关键角色如果在同章里反复出现下面这些句型中的任意 `2` 句及以上，就默认人物出场这一层先别算过：
  - `他不是为了……`
  - `他真正想要的是……`
  - `他想把……打回……`
  - `他其实在护……`
  - `他怕的不是A，是B`
- 如果人物嘴上说的和动作护的完全一样，而且所有欲望都直接说穿了，就默认人物偏扁平
- 如果人物矛盾只写成设定句，不落到现场，至少按 `F1/F2`

先别急着算过的情形：
- 角色只会“脸白、手抖、吓住、愣住”，没有更独有的反应方式
- 对手只有压迫功能，没有自己的急迫目标和损失口
- 协作方只有“听懂了 / 配合了 / 认出来了”，没有自己的算盘
- 角色记忆点只在首次出场给一次，后面马上退回办事作用
- 人物一有算盘，旁白立刻替他解释“他怕的是 / 他想要的是 / 他其实是在……”
- 字数一不够，就开始补“他真正怕的是 / 他真正想要的是 / 他算明白了什么”
- 人物明明有权力差、利益冲突、怕丢脸、怕背锅，却所有台词都直来直去，没有一点回避和遮掩

## 三点十五点二、冲突别只剩一层

- 正文关键场景别只剩单层冲突
- 每个关键场景至少得同时有：
  - 表层冲突：明面上在抢什么
  - 里层冲突：各自真正护什么
- 冲突升级别只靠几句压场的话或对骂，至少得落到下面这些动作之一：
  - 抢
  - 压
  - 藏
  - 删
  - 挡
  - 推
  - 把话收回去
- 场景里至少得有 `1` 个第三方插刀点，优先落在：
  - 协作方
  - 围观者
  - 执行层
  - 规则口
  - 群体反应
- 如果字数不够，先补第二层冲突和第三方插刀，别先补解释句

先别急着算过的情形：
- 只有“主角想做 -> 对手拦一下 -> 主角再赢”
- 所有人都把真心话直接说穿，没有任何护物、回避、甩锅、装硬
- 第三方只站着看，没有任何往前往后挪、把话收回去、拖延、递假东西、先保自己

## 四、字数约束

- 默认首章目标：`3100-4500`
- 默认非首章目标：`3100-4000`
- 若细纲另有标注，以细纲为准
- 没到目标字数，别拿“节奏快”当过关理由

## 五、章尾拦截

能过的章尾只认现场，不认总结：

能停住的章尾，只认下面这些：
- 新危险
- 新反扑
- 新人物 / 新敌意 / 新物证落地
- 失口 / 露底 / 翻面
- 新规矩马上被打破
- 新的不可逆局势

别停在：
- 预告句
- 总结句
- 信息说明句
- 平收转场句

补充判断：
- 章尾如果还在用“更大的危险来了 / 更高层的人来了 / 还有新的线索”这类没具名的说法，就按弱章尾处理
- 章尾声势如果很大，但本章最主要的收益还是抽象判断，就按 `F2`
- 章尾最后 `80-150` 字里别扎堆出现下面这些下结论词：
  - 终于
  - 总算
  - 最终
  - 这一刻
  - 接下来
  - 更大的风暴
  - 这意味着
  - 这说明
  - 这表明
  - 这证明
- 章尾如果要引下一章，优先用 `新物证 / 新备注 / 新名单 / 新屏幕说法 / 新坐标 / 新封记` 落钩，别用作者代说式总结

## 六、写后必做

正文写完后，得做这几步：

1. 跑 `scripts/scene_lint.py`
2. 做纯净度检查
3. 人工回看最后 `20-50` 行，并逐项排：
   - 是否出现章尾下结论词
   - 是否把下一章预期写成了本章结论
   - 是否用解释句代替新钩子
4. 脚本如果只报 `WARN`、没报 `ERROR`，别直接把 `WARN` 当失败码；先看是不是真要补强
5. 生成写后检查
6. 同步角色卡与追踪

上面哪一步没做完，这章就当还没查过，别收进主正文。
