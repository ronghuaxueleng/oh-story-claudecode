# 正文门禁

> 本文件只负责正文阶段的执行清单：写前必须确认什么、写后必须完成什么、情报流/字数/章尾怎么验收。句子与段落级红线统一交给 `references/writing-red-lines.md`。

---

## 一、正文前置要求

开写正文前必须确认：

- 已读取 `设定/题材定位.md`
- 已读取本章细纲
- 已完成写前闸门
- 已写明本章目标字数与上限字数
- 已为正文预留 `字数安全边际`；若脚本下限为 `2900`，首写正文默认目标不得低于 `3100`
- 已明确第一爆点与高潮后续收益
- 已明确本章 `核心对手 / 核心物证或核心资源 / 核心场面载体`
- 已明确本章 `核心母场景 / 主事件来源 / 世界主动施压载体`
- 已确认本章不是对上一章母场景的换皮复用
- 已确认本章残魂 / 金手指不是继续只做“提醒 / 判断增强”
- 若本章是首章且作品存在足以改写默认现实认知的世界机制，已明确 `首章世界总盘交代` 与 `世界底色显影锚点`
- 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，已明确 `世界底色续显点`
- 已确认本章 `主收益骨架` 不与最近 `2-3` 章重复
- 已确认残魂 / 金手指会留下 `外部可感知后果`
- 若本章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，已明确 `跨场景过桥`
- 已明确本章中后段至少 `1` 口 `外部反应 / 群体站位变化 / 系统屏幕变化 / 协作方改口`
- 已明确本章 `事件拍序` 至少 `3-5` 拍，且不是单拍结果文
- 已明确本章 `情绪回摆` 至少 `2` 次，且都绑在事件与舍取上
- 已明确章尾最后一句的 `末句功能型`，且只能是：
  - `新动作`
  - `新物证`
  - `新口径`
  - `新名单`
  - `新翻面`
  - `下一步命令`
- 已确认本章不存在未替换的抽象占位表达
- 已确认不存在“单一力度副词”批量刷句；动作强度必须写在受力点、位移、失衡、破坏结果里
- 已完成 `首稿污染预防`：`references/banned-words.md` 的 `一级禁用词` 本章正文起草时不得出现，不得用口头发力词、空力度词、抽象效果词替代动作链
- 已完成 `禁用词反向污染预防`：不得因为读取了禁用词表，就把表内词面、口头发力词或解释句黑名单词写进正文
- 已读取本章 `新增信息白名单` 与 `正文禁增项`
- 已确认正文若需新增核心信息，必须先回填细纲与写前闸门，不得边写边吞
- 已明确本章 `题面兑现档位`
- 若题面命中 `经营 / 配额 / 分配 / 掌口 / 生火权 / 放票权 / 名额权`，已明确本章正文至少要写出 `谁被主角实际改了火 / 票 / 路 / 名额`
- 若本章命中 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，已明确：
  - 本章最主要的规则信息压在哪个在场动作里
  - 本章至少哪一口 `试用 / 交易 / 排队 / 受阻 / 讨价 / 改规矩 / 利益变化` 会真正发生
  - 若删掉解释段，哪一拍现实推进仍必须成立
- 已完成本章 `动作骨架卡`：
  - 至少 `4-6` 拍
  - 每拍只写 `谁做了什么 -> 谁被迫怎么改`
  - 不得把“规则说明 / 局势总结 / 作者判词”混入
- 已完成本章 `人物冲突卡`：
  - 主角嘴上报什么、动作先护什么
  - 对手嘴上压什么、动作先藏什么
  - 摇摆位嘴上推什么、动作先避什么
  - 关键角色各自这一场最想保什么、最愿意先牺牲什么
  - 至少 `1` 个角色会在中段临场改口或改站位，不得所有人一路按原计划说到底
- 已完成本章 `章尾现场句预案`：
  - 只准属于 `新动作 / 新物证 / 新名单 / 新命令 / 新翻面`
  - 不准写意义总结、黑幕升级、下一章预告
- 已计划本章正文落盘后的 `全文残片扫描`：
  - 明确扫描命令或扫描范围
  - 明确若命中工具残片 / 元话语 / 非故事语料，则本章按 `F4` 截停
- 已确认本章存在至少 `1` 段 `对话抢势`：
  - 不是交换信息
  - 而是借台词改变口令、次序、站位、归属、执行权中的至少一项
- 已确认本章涉及的 `制度口 / 流程口 / 职业术语` 已经过白话落地：
  - 第一次出现时，立刻能让读者知道“它会卡谁、放谁、害谁”
  - 不得只丢术语名词，不给口语解释或现场后果
- 已确认本章不是“单事件直线推进”；至少要有：
  - `首轮阻断`
  - `中段改手`
  - `第二次翻面`
  - `现实代价或退路烧断`

## 二、句子红线入口

正文出现以下问题时，不在本文件展开重复说明，直接按 `references/writing-red-lines.md` 执行：

- 效果词先行
- 后台调度语
- 作者盖章句
- 提纲扩写句
- 只删词不回句子功能
- 单一力度副词反复刷屏
- 未备案核心信息临时入正文
- 章尾最后一句承担意义总结 / 作者盖章 / 资源判价功能

## 三、情报流专门要求

- 正文里第一次出现情报时，必须用 `「」` 单独包住
- 情报首次出现必须交代来源或触发机制
- 情报必须在当前章或后 `1` 章内转成现实收益
- “知道更多 / 更危险 / 更复杂”不算兑现

补充硬判：
- 若情报只负责提醒、解释、抬线索价值，但没有推动现实盘面变化，按未兑现处理
- `现实收益` 至少应能落到 `命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化 / 垫背者落地` 之一
- 若首章情报只换来“对手更警惕/主角更清醒”，默认不算过线
- 情报句之后，正文必须在短距内出现具体动作响应；不得出现“情报出现 -> 主角思考很多 -> 下一段再动”
- 若本章细纲已写 `本章情报条目`，正文不得改成“临场观察替代情报”
- 若本章正文关键翻盘依赖名单、车、票、口令、批文、调度等关键信息，而这些信息未在情报条目或新增信息白名单中备案，默认按 `F3/F5`
- 情报流作品前 `3` 章至少 `2` 次出现“情报 -> 动作 -> 外部现实变化”的完整链；不足则默认题面滑轨

## 三点一、题面结果化兑现硬规则

- 正文不得把题面兑现写成“更接近、快要摸到、差一点拿到”
- 每章必须至少回答一个结果化问题：
  - 主角这一章真实掌到哪一口分配权 / 放票权 / 车权 / 名额权 / 调度权
  - 若没有掌到，是否至少逼敌方公开改动作 / 改规则 / 改站位
- 若题面是经营 / 配额 / 分配 / 执牌 / 夺权类，连续 `2-3` 章都只做到“掀黑、截物证、揭脏手”，默认按 `F3`
- 若题面命中 `掌口 / 生火权 / 名额权 / 放票权 / 分火权`，正文中必须尽快出现以下至少一种实答结果：
  - 主角点了谁先进 / 先拿 / 先走
  - 主角卡了谁不能进 / 不能拿 / 不能走
  - 敌方被迫按主角的新顺序执行
- 若整章只能回答“主角拿到一张签 / 一口资格 / 一份自保位”，默认最高只算 `D2`，不得写成“已掌口”

## 三点二、章节组反重复硬规则

- 连续 `2-3` 章不得反复使用同一翻盘模板，例如：
  - 物证见光 -> 群众炸锅 -> 执行口让步
  - 名单见光 -> 群众炸锅 -> 临时改口
  - 关键物见光 -> 对手失态 -> 章尾再抬高层敌意
- 若最近 `2-3` 章已复用同模板，本章正文必须主动换打法：
  - 从抢物证切到抢人 / 抢口 / 抢规则执行权
  - 从围观炸锅切到正式秩序对撞 / 调度对撞 / 群体改线
  - 从掀基层脏手切到咬中层盘面敌人
- 若正文写到中段才发现仍在复用旧模板，立即停写，回退细纲，按 `F3`

## 三点三、单章事件拍序与情绪回摆硬规则

- 正文不得写成“单章一个事件名词 + 单拍结果”
- 每章正文至少要形成以下最小结构：
  - `起目标`
  - `遇受阻 / 反压`
  - `变打法 / 舍一口`
  - `现实改盘`
- 若篇幅或题材确实偏短，最低也必须满足：
  - 至少 `1` 次中段受阻
  - 至少 `1` 次中段变招
  - 至少 `1` 次现实变化
- 本章情绪至少 `2` 次回摆，不得只有单一路径情绪，例如：
  - 一路冷静
  - 一路查线
  - 一路强压
  - 一路愤怒
- 情绪回摆必须通过以下承载物落地，不认纯旁白解释：
  - 动作停顿 / 失真
  - 临场改口
  - 错判后改决断
  - 快拿到又被打断
  - 代价突然落下
  - 站位 / 队形 / 协作关系改变

默认不过线情形：
- 本章只有“去做一件事，然后做成/做败”
- 中段没有第二次变向，只是在拉长同一动作
- 所谓情绪拉扯只靠“他很怒 / 很痛 / 很纠结 / 很不甘”
- 章中没有“快拿到 -> 被打断”或“被打断 -> 改打法”
- 结尾虽然有结果，但读者几乎没经历过程里的希望、打断、舍取、代价
- 关键角色除了支持主线功能外，没有任何个人算盘、私心取舍、自保动作
- 所谓对白很多，但没人借对白改现场秩序，只是在轮流把信息说出来
- 本章正文落盘后未先做全文残片扫描，就直接进入下一章或继续批量写后章

## 三点三点二、章内事件拉扯硬规则

- 单章主事件默认不得写成单线跑图；至少要出现以下 `4` 项中的 `3` 项：
  - 第三方插刀
  - 盟友私心或短视坏事
  - 世界压力临时改局
  - 主角主动放弃一条更便宜的解
- 若命中 `L3/L4`，则上面 `4` 项里至少实装 `3` 项，且必须让其中 `2` 项落到可见动作，不认纯旁白说明
- “大场面”不等于人多；若人多但没有改线、踩踏、争口、抢载体、改批次、改归属，默认只是背景噪音
- 章中至少要有一次“原本以为要抢 A，临场改成先保 B”的取舍，不得一路照着最初目标顺推
- 若主角每次都能选到更强、更快、更正确的解，而没有丢掉任何一头，默认拉扯不足

## 三点三点一、三章小循环硬规则

- 连续 `3` 章默认要形成一个小循环，不得三章都写成并列任务章
- 默认位置：
  - 第 `1` 章：`起火`
  - 第 `2` 章：`升压`
  - 第 `3` 章：`兑现`
- 若当前章承担 `升压`：
  - 正文必须扩大阻力、代价、两难、关系拉扯中的至少 `2` 项
  - 不得只重复上一章打法
- 若当前章承担 `兑现`：
  - 正文必须至少改掉一口现实盘面
  - 不得只留下新线索、新名单、新猜测

默认不过线情形：
- 第 `2` 章只是把第 `1` 章再做一遍
- 第 `3` 章没有兑现，只是再抬更高层黑幕
- 三章都只有局部胜负，没有形成一轮完整情绪释放

## 三点三点零、功能场景反说明书硬规则

- 功能场景不是不能讲规则，而是不能让“讲规则”替代“场景主功能”
- 以下场景默认都要额外做反说明书复检：
  - 卖货 / 看房 / 食堂 / 配套
  - 配额 / 名额 / 资格 / 放票 / 放行
  - 审批 / 审查 / 封存 / 试炼 / 考核
  - 流程口 / 制度口 / 规则口 / 服务口
- 若正文某段同时满足以下 `5` 条中的 `4` 条，默认该段写成了说明书，至少按 `F2` 回退：
  1. 该段场景目标本身是制度、流程、福利、玩法、资格或规则入口
  2. 连续 `4` 句及以上主要功能是在解释规则，而不是推动在场人物动作
  3. 该段没有明确的 `试用 / 交易 / 排队 / 讨价 / 受阻 / 改规矩 / 利益变化`
  4. 删掉该段后，人物关系和场面推进几乎不受影响
  5. 该段内容能直接改写成产品说明 / 制度公告 / 规则介绍，而意思几乎不变

硬执行：
- 命中上面判据时，不得仅按“解释腔偏多”降成 `F1`
- 若只是某一句在解释，可按 `F1` 修句
- 若整段主功能已经变成说明书，默认按 `F2`
- 若连续 `2` 章以上都把功能场景写成说明书，或因此导致题面一直只停在“知道规则更多”，默认升级为 `F3`

排误伤规则：
- 规则多，不等于说明书
- 若规则信息始终压在 `点单 / 试吃 / 结账 / 排队 / 抢位 / 争执 / 改口 / 改规矩 / 受阻 / 外部站位变化` 里，不得仅因信息密度高判死
- 若删掉规则信息会直接削弱当前场景的交易、争抢、盘面变化，也不得误判为说明书

### 功能场景四步生成顺序

- 命中功能场景时，正文起草顺序固定为：
  1. `谁想拿什么`
  2. `谁先动手`
  3. `谁被迫改动作 / 改口 / 改站位`
  4. `规则因此显形`
- 禁止顺序：
  1. 先讲制度
  2. 再讲规则例外
  3. 最后才让人物去执行
- 写前若答不出前 `3` 步，说明该场景仍是说明书骨架，不得进入正文
- 正文中若一段先完成了“规则全貌解释”，却还没出现 `争 / 试 / 卡 / 放 / 改 / 挡 / 抢 / 压` 任一动作，默认该段优先按说明书处理
- 功能场景的合格信息量只能回答当下这场：
  - 这口规则现在压到谁
  - 谁正借它吃人
  - 谁准备把它扳回来
- 不得顺手把“完整制度全景 / 所有例外情况 / 常规运转全貌”一起讲完

### 规则显形白名单

- 功能场景里，规则只能优先通过以下承载物显形：
  - 门
  - 票
  - 名单
  - 夹板
  - 屏幕
  - 广播
  - 印章
  - 回收单
  - 锁闸
  - 站位变化
- 若规则信息不附着在上述承载物和在场动作上，而是只靠角色口播或旁白讲解，默认不过线

## 三点三点零点一、旁白代判首稿硬闸

- 正文首稿里，旁白不得替读者完成以下工作：
  - 判谁更狠
  - 判这一下意味着什么
  - 判人物真正怕什么
  - 判这句为什么更硬
  - 判规则真正值钱的是哪一口
- 以下句法命中后，默认优先判 `F1`，集中出现在开头 / 中段关键翻面 / 章尾时默认升 `F2`：
  - `这一下比……更……`
  - `这不是……，是……`
  - `从这一刻开始`
  - `真正……的是……`
  - `不是怕A，是怕B`
  - `不是为了A，是因为B`
  - `他不是在……，是在……`
  - `这才算……`
- 正确修法只有一种：
  - 回到前一句
  - 改成 `手 / 眼 / 门 / 票 / 屏 / 钟 / 伤 / 呼吸 / 站位 / 物件状态`
  - 让读者自己得出判断
- 若删掉该句后段落更顺，直接删除，不得换近义总结句
- 若同章连续 `2` 句以上旁白代判同一拍，默认本拍承载失败；不得只删一半保一半

## 三点三点零点二、对白人味硬规则

- 高压对话里，优先写 `打断、抢词、避答、错口、硬压、借物件说话`，少写完整论证
- 一段关键对白若能整体改写成“情况说明会记录”，默认没有人味
- 人物说话必须受身份、处境、疼痛、喘息、时间压力约束；刚打完、刚摔过、刚被掐住退路的人，不应持续稳定输出工整判断句
- 若同章多名角色连续说出相同句式、相同判断结构、相同金句力度，默认按 `F1/F2` 回修
- 关键对白必须额外过一次“张嘴测试”：念出来拗口、像宣读制度、像书面汇报、像作者借嘴发言，任一命中都不得放行

## 三点三点零点三、术语口语化硬规则

- 正文允许有题材术语、职业术语、制度术语，但不允许术语裸奔
- 术语第一次出现时，必须至少同步绑定以下之一：
  - 谁会因为它先走 / 先拿 / 先活
  - 谁会因为它被卡 / 被罚 / 被挤下去
  - 一个当场能看见的承载物
  - 一句更白的话把它翻给读者听
- 若术语需要读者停下来自己翻译，默认不过线
- 以下类型默认优先复检：
  - `轮次 / 批次 / 配额 / 指标 / 权限 / 流程口 / 执行口 / 资格口 / 调度口` 这类制度口词
  - 一句里连续塞 `2` 个及以上术语
  - 角色明明在急场里，却还在稳定使用长串职业名词
- 正确落地顺序：
  1. 先让读者看懂这口东西在卡谁
  2. 再顺手知道这口东西叫什么
  3. 术语只负责增厚世界，不负责替代冲突
- 错误落地顺序：
  1. 先报术语名
  2. 再补制度说明
  3. 最后才告诉读者为什么眼下要命
- 若术语删掉后，读者反而更顺、更快懂当前冲突，优先删或换成白话，不得硬留“显得专业”

## 三点四、字数安全边际硬规则

- `scene_lint.py` 的下限不是目标字数；正文首写必须主动留安全边际
- `scene_lint.py` 的字数结果分为两层：
  - `ERROR`：明显过短 / 过长，硬拦截
  - `WARN`：接近下限 / 偏高，只做预警，不单独拦截
- 若项目脚本下限为 `2900`，则：
  - 首写正文推荐目标 `3300-3800`
  - 不得以 `2900-3000` 作为收稿目标
- 若细纲目标字数已低于脚本下限上浮 `100` 字，写前闸门必须先把目标抬高
- 若最近 `2` 章都命中 `字数.warn`，下一章首写目标至少再上抬 `100`
- 若细纲或写前闸门写了目标下限，正文首稿低于该目标下限 `200` 字以上，直接按 `F2`，不得进入写后验收
- 若正文首稿低于 `scene_lint` 下限，直接按 `F2`，不得用“节奏快 / 可读性强”放行
- 若正文首稿低于 `3100`，即使脚本未报错，也必须人工判定是否存在“正文主体不足”；不得默认 `F0`
- 若项目脚本下限为 `2900`，正文首稿一旦低于 `3100`，必须进入 `配额回补态`，不得自由补几句凑线

`配额回补态` 硬要求：
- 前 `1/3` 必须补进 `1` 口新的受阻 / 截堵 / 反压，不得只是延长原动作
- 中段必须补进 `1` 口第三方插刀 / 协作方改口 / 围观站位变化
- 后 `1/3` 必须补进 `1` 口现实代价 / 关系变坏 / 盘面改线 / 外部可感后果
- 关键角色至少补进 `1` 处显影动作：
  - `身体先泄露，台词后补位`
  - `嘴上说A，动作护B`
  - `被问A，绕去答B`
- 回补不得落成解释句、总结句、作者代判句
- 允许补细节，但细节只能附着在上述 `4` 类配额上，不得单独成段顶字数
- 细节只允许补：
  - 动作打点
  - 物件磨损 / 状态变化
  - 身体先泄露
  - 环境如何直接压到当前动作
  - 规则如何落到当前职业习惯 / 群体反应
- 不允许把细节写成纯空镜头、纯情绪句、纯气氛句
- 上述 `4` 类配额任一未补满，不得重新统计字数，不得进入写后验收

默认不过线情形：
- 正文多次停在 `2900` 附近擦线
- 正文连续停在 `3000` 左右、虽能过线却反复触发接近下限告警
- 通过补一两句机械凑字来过脚本
- 正文主体不足，只在章尾补解释或总结凑到下限

## 三点四点一、首稿污染自检硬规则

- 正文首稿写完后，进入写后验收前必须先执行首稿污染自检
- 自检范围：
  - `references/banned-words.md` 的一级禁用词
  - 口头发力词污染
  - 解释句黑名单词面
  - 工具残片 / 补丁标记 / 非故事语料
- 自检结果必须结构化记录：
  - 一级禁用词是否命中
  - 口头发力词是否命中
  - 全文残片扫描是否通过
  - 若任一失败，失败码是什么
- 命中任一项：
  - 默认 `F1/F4`
  - 不得进入 `F0` 写后验收
  - 生产模式下必须当场回修本章并重跑首稿污染自检，不得把问题后移到 `scene_lint`、写后验收或 `story-review`
  - 生产模式下自动切入 `H1-B 生产自动返修态`
  - 纯基准测试下自动切入 `H1-A 纯测试记录态`，只记录失败，不修正文
- 禁止把“只是一个词”降级为观察项；一级禁用词属于首稿失败项

## 三点四点二、章尾末句功能硬规则

- 正文写完后，必须单独检查最后一句是否仍符合写前登记的 `末句功能型`
- 最后一句只能完成一种动作：把章尾停在新的现场结果上
- 下列功能一旦落在最后一句，直接按 `F1/F2`：
  - 意义总结
  - 局势翻译
  - 作者判价
  - 资源判价
  - “真正要命的是 / 真正值钱的是 / 现在真正该抢的是” 这类替读者算账
- 若最后一句删掉后更强，直接判章尾功能错误，不得放行
- 若最后两句删掉后本章仍成立，说明章尾在补缝，不得放行
- 若最后一句主要作用是告诉读者“下一章该去哪 / 这件事该怎么理解 / 这一拍为什么重要”，直接按章尾不过线处理
- 以下章尾句型默认优先复检：
  - `原来……`
  - `这说明……`
  - `这意味着……`
  - `接下来……`
  - `得去……`
  - 单独拿一个字 / 一个概念 / 一个方向出来替下一章盖义

唯一正确修法：
- 若前一拍已成立，删掉错误末句
- 若前一拍未成立，回补前一拍的动作 / 物证 / 外部反应 / 他人失态 / 现实变化
- 不允许直接把错误末句换成近义解释句

## 三点四点三、解释句根因分流硬规则

- 命中解释腔 / 盖章句 / 作者代判句后，必须继续判根因：
  - `G1 信息抢跑`
  - `G2 现场承载不足`
  - `G3 提纲语言漏出`
  - `G4 人物失声`
- 若未判到 `G1-G4` 之一，默认仍未完成验收
- 修法绑定根因，不得混修：
  - `G1`：把信息后移到决策压力点或现实变化点
  - `G2`：回前一拍补动作、后果、代价、失态、外部变化，不得只删句
  - `G3`：删除结构调度语，改为人物当下可见可做内容
  - `G4`：改回人物动作、对白、错判、改口来承载
- 若同章连续出现 `2` 处以上 `G2`，默认本章场面承载失败，按 `F3` 回退细纲或场面卡

## 三点四点四、对白人话硬规则

- 对白的第一职责是让角色彼此攻防，不是替作者向读者讲解
- 写对白前，必须先答出这句的即时功能，只允许落在以下之一：
  - 压人
  - 试探
  - 套话
  - 拖时间
  - 甩锅
  - 自保
  - 逼站位
  - 掩饰
  - 挑衅
  - 求活
- 若一句对白的主要作用是：
  - 讲设定
  - 讲规则全貌
  - 讲完整方案
  - 替读者下结论
  - 替人物总结意义
  默认至少 `F1`；高压场景中连续出现 `2` 句及以上，默认升 `F2`

### 对白不过线高危信号

- 一句对白同时完整交代：
  - `事实`
  - `判断`
  - `结论`
  - `方案`
  任意 `3` 项及以上，默认不像人话，至少 `F1`
- 高压场景下，人物一句话超过 `2` 个并列分句，且分句功能都在“解释 / 说明 / 归纳”，默认优先复检
- 角色把本可通过动作、物证、站位看出来的东西，再用台词说一遍，默认按“对白在替动作补缝”处理
- 反派 / 高位敌人主动把自己的脏手、把柄、受贿口径、关键护点说得过明，默认不过线
- 主角在极险、极乱、极快场景中频繁说出“金句式完整短句”，默认优先复检
- 高压场景里人物频繁说出“制度播报腔 / 公告腔 / 会议纪要腔”，默认至少 `F1`

### 句型复检黑名单

- 以下句型若在对白中高频出现，默认优先复检：
  - `不是A，是B`
  - `你要的不是A，是B`
  - `给我口子的不是……`
  - `我来的不是……`
  - `真正……的是……`
  - `你一句……就想……`
  - `今夜这口……先……不……`
  - `谁现在……，谁就……`
  - `不是你……说了算`
- 以上句型单句偶发不直接定罪
- 同章同角色命中 `2` 次以上，或连续 `2-3` 章反复拿它们当发力句，默认对白层不过线

### 高压场景对白写法硬要求

- 命中以下场景之一：
  - 截堵
  - 抢口
  - 战斗
  - 追逐
  - 谈判翻桌
  - 临场夺权
  - 灾异逃生
  - 众目之下顶牛
  必须优先：
  - 短句
  - 半句
  - 抢话
  - 被打断
  - 动作插入
  - 岔答
- 不得优先写成：
  - 完整论证
  - 漂亮排比
  - 工整金句
  - 规则宣讲
  - 立场总结
- 若冲突里最像“人话”的不是对白，而是旁白在替人物讲话，默认判 `G4 人物失声`

### 对白与动作承载分工硬规则

- 规则、制度、方案、流程，不得主要靠人物口播完成
- 若一句对白删掉后，现场通过动作 / 物证 / 站位仍能成立，优先删对白，不得硬留“解释型台词”
- 若一句对白删掉后，读者看不懂，那先补动作、物证、承压反应，不得先补另一句解释台词
- 角色说话时，至少应有一种同步承载物：
  - 手上先护什么
  - 身体先退还是先顶
  - 目光先找谁
  - 被问A时先去碰B
  - 说狠话时先护自己的口袋 / 令牌 / 名单 / 人

### 对白终检三问

- 每段关键对白写完后，必须复问：
  - 这句话是说给对手听，还是说给读者听
  - 这句话如果换个人名，是否谁都能说
  - 这句话念出来像吵架 / 顶牛 / 试探，还是像演讲 / 公告 / 作者总结
- 任一答成后者，默认不过线，不得进入写后验收

## 三点五、具象化正文硬规则

- 正文中的核心位不得落成以下抽象占位：
  - 某个人
  - 某件东西
  - 某个势力
  - 一份物证
  - 更高层的人
  - 新的交接点
  - 外部突然生乱
- 若细纲已具名 `核心对手 / 核心物证 / 核心资源 / 核心场面载体`，正文首轮出场时必须尽快落名，不得重新打回抽象词
- 若正文把细纲中的具名大事件重新写回“匪患 / 混乱 / 追杀 / 证据 / 调令”这类通用词，视为兑现降级

## 三点五点一、正文新增信息越权硬规则

- 正文不得临时新增未备案的核心信息来救场面、救章尾、救收益
- `核心信息` 至少包括：
  - 新具名人物 / 新具名势力 / 新具名地点
  - 新核心物证 / 新核心资源 / 新交接口
  - 新规则变化 / 新关键时间钉 / 新关系改判
  - 新的残魂 / 金手指 / 系统能力口径
- 若写作中发现确需新增：
  - 先停正文
  - 先回填细纲 `新增信息白名单`
  - 再回填写前闸门
  - 必要时同步卷纲 / 伏笔 / 角色卡准入口径
  - 完成后才可继续正文

默认不过线情形：
- 为了让章尾更硬，临时多出一个新接头点 / 新暗号 / 新账簿 / 新路线
- 为了让冲突成立，临时补一个之前没预热的新人物 / 新势力
- 为了让主角翻盘，临时补一个细纲没写过的新能力口径
- 为了补场面，正文突然出现细纲未备案的新规则或新世界设定
- 为了让读者理解“这口资源升级了”，正文直接补一句 `更值钱 / 更硬 / 更实 / 这一口更…… / 比上一口更……`
- 为了让章尾像钩子，正文直接用 `现在真正该抢的是…… / 最该抢的已经不是……`

## 三点六、核心设定出手硬规则

若本章细纲写有 `本章核心设定参与方式`，正文必须让读者看到“出手结果”，不认纯解释。

合格示例方向：
- 借体后完成普通肉身做不到的动作
- 压迫后让对手失神、失口、露底
- 共鸣后拆穿禁印、伪装、假货
- 刺感后抢在群体变化前先动一步

不过线情形：
- 只写“主角忽然觉得不对”
- 只写“残魂提醒了他”
- 只写“他判断出了问题所在”
- 结果仍完全像普通聪明人能做到的事，且没有任何可见反应差

## 三点七、L3/L4 场面落地硬规则

凡细纲标记 `L3/L4` 的章节，正文必须写出：
- 群体规模感
- 被改写的路线 / 秩序 / 营地 / 资源 / 站位
- 至少一种强场面载体

若正文只剩：
- “场面大乱”
- “所有人都慌了”
- “局势一下失控”

而没有具体现实变化，按规模兑现失败处理。

## 三点八、母场景禁复用硬规则

- 正文不得把上一章的 `核心母场景` 只换个地名或道具后重复再写一遍
- 若上一章核心母场景属于以下任一类，正文默认高风险回流：
  - 囚车 / 押解路
  - 尸体 / 验尸
  - 驿站 / 旧棚 / 临时窝点
  - 公验 / 对质 / 逼口 / 翻供
  - 小股灭口 / 小队试探
- 命中高风险回流时，正文必须尽快切出新的主事件载体，例如：
  - 军粮车队
  - 矿场暴乱
  - 山寨并线
  - 黑市洗货
  - 关城封锁
  - 仓储争抢
  - 灾异冲营
  - 禁地异动

默认不过线情形：
- 还是围着车、尸、驿、口供、规矩翻脸打转
- 只是把“验尸”换成“公验”
- 只是把“驿站”换成“黑店 / 破棚 / 偏院”

## 三点九、世界主动施压硬规则

- 本章大冲突必须先由 `世界 / 势力 / 环境 / 资源争夺` 压到主角脸上，再由主角反切
- 不得长期沿用“情报出现 -> 主角去查 -> 对手来灭口 -> 主角翻面”的单一小查线模型
- 正文必须尽快交代：是谁先动、动了哪条路 / 哪批货 / 哪个营 / 哪道关 / 哪群人

默认不过线情形：
- 主角又是唯一发动者
- 敌人只来一波小灭口，没有更大盘面变化
- 情报仍主要服务于“找人 / 验尸 / 套口供”

## 三点十、残魂 / 金手指轮换硬规则

- 若本章存在残魂 / 金手指直接出手，正文必须尽快让读者看见其 `出手类型`
- 连续两章不得都只写成：
  - 提醒危险
  - 判断更准
  - 提前察觉
  - 单纯识别问题
- 若上一章已主要使用 `识破类`，本章优先改用 `压迫类 / 夺势类 / 超常动作类 / 异象共鸣类`
- 若本章仍沿用低效出手类型，必须额外补出一次可见现实变化，不得只让主角“更聪明”
- 残魂 / 金手指出手后，正文必须尽快给出至少 `1` 个外部可感知后果：
  - 对手失神 / 失口 / 误判
  - 器物 / 阵纹 / 骨灯 / 封印 / 环境出现反应差
  - 旁人因为不合常理结果改站位、改口、改动作

默认不过线情形：
- 只有主角自己知道残魂刚才出手了
- 外部人和环境完全无反应

## 三点十一、首章世界总盘与世界底色显影硬规则

- 若本章是首章，且作品存在足以改写默认现实认知的世界机制，正文前 `1/3` 必须同时完成两件事：
  - 让读者知道自己在读什么世界
  - 让读者看见这套世界规则如何现场改动作、改规矩、改生死
- `首章世界总盘交代` 必须至少回答：
  - 这到底是个什么世界
  - 这个世界的制度 / 日常 / 生存方式为什么会这样运转
  - 主角当前死局为什么正好由这套世界规则制造
- `世界底色显影锚点` 必须落到动作、规矩、环境、职业常识、群体反应、制度执行中的至少 `2` 项，不能只术语点名
- 允许用具体细节显影世界底色，但必须满足：
  - 细节对象直接属于这套世界的生存规矩
  - 细节结果直接改了排队、争抢、放行、躲避、职业动作中的至少 `1` 项
  - 不得只写“世界很怪 / 很险 / 很压抑”的空判断

合格示例方向：
- 王朝存在，但驿路、军路、流放、劳役本来就和异兽、灾潮、修行资源绑定
- 都市存在，但交通、治安、职业、阶层已被异能或黑科技长期改写
- 校园存在，但升学、社交、竞争、惩罚机制本来就依赖超凡规则

默认不过线情形：
- 首章只有局部奇观，没有世界总定义
- 读者前半章仍会误判成普通古代 / 普通都市 / 普通现实
- 世界机制只存在于主角脑内旁白
- 只有边地 / 副本 / 行业特殊，没有交代整个世界为什么能容纳这套特殊
- 首章前 `1/3` 连续使用世界说明句，却没有让至少一条规则通过现场执行先裁到主角脸上

## 三点十二、非首章世界底色续显硬规则

- 若题材命中 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，第 `2-5` 章不得把世界又写回纯世俗层
- 至少每 `2` 章必须出现一次 `世界底色续显`
- 续显必须落到现场规矩、环境余波、器物用法、行业避讳或群体反应

默认不过线情形：
- 第1章有玄异，第2章起只剩现实冲突
- 所有玄异内容都缩回主角脑内

## 三点十三、主收益骨架轮换硬规则

- 连续 `2-3` 章不得反复使用同一套主收益骨架
- 尤其不得长期复用：
  - 情报指向关键物
  - 残魂压一拍逼失口
  - 主角抢下物证 / 账 / 令
  - 章尾再抬更高层敌意
- 正文必须在收益打法上做轮换，例如：
  - 抢人
  - 改规矩
  - 收站队
  - 逼敌人自毁
  - 借灾异翻盘

默认不过线情形：
- 场景换了，收益仍只是“再抢一份更值钱的东西”
- 主角每章都靠同一招数收口

## 三点十四、跨场景过桥硬规则

- 若本章相对上一章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，正文开章前 `100-200` 字内必须补清以下至少 `2-3` 项：
  - 主角为什么来
  - 主角怎么来的
  - 主角靠什么锁定这里
  - 主角为什么正好撞上本章第一爆点
- 若上章章尾只抛出“营里要出事 / 新据点有问题 / 某路会炸”，下章正文不得直接跳成主角已在现场
- 若本章是 `L3/L4` 大场面切入，正文必须先给主角一个短决策或短追踪动作，再起爆，不得纯作者调度式硬切
- 过桥必须是人物行动链，不认作者代说

合格示例方向：
- 上章逼出敌方裂口，下章先写主角看见异常信号、判断对方今夜必内乱，因此先摸到关键入口
- 上章拿到名册，下章先写主角顺名册上的交接口、车辙、运输路线或人物余波赶去新场

默认不过线情形：
- “主角赶到时”直接顶开全章
- “事情果然爆了”代替行动链
- 章首全是大场面，主角为什么在场只能靠读者脑补
- 不是人物追过去，而是作者把人物搬过去

## 三点十五、外部反应配额硬规则

- 正文中后段必须至少出现 `1` 口可见的 `外部反应 / 群体站位变化 / 协作方改口 / 系统口径变化`
- 这口反应不得只来自主角或核心对手，优先落在：
  - 围观者
  - 执行层
  - 协作方
  - 平台 / 屏幕 / 轮播 / 群消息
  - 后勤 / 司机 / 兵丁 / 驿卒 / 船工 等现场承压人
- 目的不是凑热闹，而是把“盘面翻了”写成别人也被迫改动作

默认不过线情形：
- 全章只有主角和对手在互顶
- 后补字数时只能往解释句或心理句里塞
- 明明发生公开争夺，现场旁人却像不存在

## 三点十五点一、人物显影硬规则

- 正文中的关键角色不得只剩 `身份标签 + 功能动作`
- 每个关键角色本章至少要落下 `1` 枚独有锚点，优先落在：
  - 外形
  - 动作习惯
  - 说话节奏
  - 观察偏好
  - 压场姿态
- 关键配角不得只负责：
  - 认出
  - 解释
  - 害怕
  - 失口
  - 附和主角判断
- 若关键配角参与了本章翻面，正文必须让他至少有一次按自己利益做出的动作
- 允许用具体细节补人物显影，但只能补：
  - 独有动作习惯
  - 护物动作
  - 观察偏好
  - 说话前的身体泄露
  - 与职业/身份强绑定的小习惯
- 不得把人物细节写成通用反应堆砌：
  - 皱眉
  - 发抖
  - 脸白
  - 沉默
  - 握拳
  若删掉人名后谁都能套，默认不算显影
- 多人对话场景若删去人名后仍分不出主角 / 对手 / 核心配角，至少按 `F2`
- 高价值角色若正文只说“强 / 美 / 危险 / 压人”，没有可脑补细节，至少按 `F2`
- 关键人物的欲望、损失口、私心、站位，不得主要靠旁白直译；正文必须至少落下：
  - `1` 处身体先泄露
  - `1` 处台词回避 / 岔答 / 装硬
  - `1` 处动作先护住真正值钱的东西
- 同一关键角色若出现 `2` 句及以上“旁白替他讲算盘”的直译句，至少按 `F2`
- 同一关键角色若在同章反复出现以下句型中的任意 `2` 句及以上，默认人物显影不过线：
  - `他不是为了……`
  - `他真正想要的是……`
  - `他想把……打回……`
  - `他其实在护……`
  - `他怕的不是A，是B`
- 若人物嘴上说的和动作护的完全一致，且所有欲望都直接说穿，默认人物偏扁平
- 若人物矛盾只写成设定句，不落到现场，至少按 `F1/F2`

默认不过线情形：
- 角色只会“脸白、手抖、吓住、愣住”，没有更独有的反应方式
- 对手只有压迫功能，没有自己的急迫目标和损失口
- 协作方只有“听懂了 / 配合了 / 认出来了”，没有自己的算盘
- 角色锚点只在首次出场给一次，后面马上退回功能位
- 人物一有算盘，旁白立刻替他解释“他怕的是 / 他想要的是 / 他其实是在……”
- 字数一不够，就开始补“他真正怕的是 / 他真正想要的是 / 他算明白了什么”
- 人物明明有权力差、利益冲突、怕丢脸、怕背锅，却所有台词都直来直去，没有一点回避和遮掩

## 三点十五点二、冲突分层硬规则

- 正文关键场景不得只有单层冲突
- 每个关键场景至少应同时存在：
  - 表层冲突：明面上在抢什么
  - 里层冲突：各自真正护什么
- 冲突升级不得只靠狠话或对骂，至少要落到以下动作之一：
  - 抢
  - 压
  - 藏
  - 删
  - 挡
  - 推
  - 改口
- 场景里至少要有 `1` 个第三方插刀点，优先落在：
  - 协作方
  - 围观者
  - 执行层
  - 规则口
  - 群体反应
- 若字数不足，优先补第二层冲突和第三方插刀，不得优先补解释句

默认不过线情形：
- 只有“主角想做 -> 对手拦一下 -> 主角再赢”
- 所有人都把真心话直接说穿，没有任何护物、回避、甩锅、装硬
- 第三方只站着看，没有任何改站位、改口、拖延、递假东西、先保自己

## 四、字数硬约束

- 默认首章目标：`3100-4500`
- 默认非首章目标：`3100-4000`
- 若细纲另有标注，以细纲为准
- 未达目标，不得把“节奏快”当成放行理由

---

## 五、章尾硬闸

合格章尾只认现场，不认总结：

必须停在以下之一：
- 新危险
- 新反扑
- 新人物 / 新敌意 / 新物证落地
- 失口 / 露底 / 翻面
- 新规矩马上被打破
- 新的不可逆局势

不得停在：
- 预告句
- 总结句
- 信息说明句
- 平收转场句

补充硬判：
- 若章尾继续使用“更大的危险来了 / 更高层的人来了 / 还有新的线索”但未具名，按弱章尾处理
- 若章尾声势很大，但本章最值钱收益仍是抽象判断，按 `F2`
- 章尾最后 `80-150` 字不得集中出现以下盖章词：
  - 终于
  - 总算
  - 最终
  - 这一刻
  - 接下来
  - 更大的风暴
  - 这意味着
  - 这说明
  - 这表明
  - 这证明
- 若章尾需要引下一章，优先用 `新物证 / 新备注 / 新名单 / 新屏幕口径 / 新坐标 / 新封记` 落钩，不得用作者代说式总结

---

## 六、写后必做

正文落盘后，必须：

1. 跑 `scripts/scene_lint.py`
2. 做纯净度检查
3. 人工回看最后 `20-50` 行，并逐项排：
   - 是否出现章尾盖章词
   - 是否把下一章预期写成了本章结论
   - 是否用解释句代替新钩子
4. 若脚本只报 `WARN`、未报 `ERROR`，不得把 `WARN` 直接当失败码；先判断是否真需补强
5. 生成写后验收
6. 同步角色卡与追踪

若任一步未完成，正文默认未验收，不得并入主正文。
