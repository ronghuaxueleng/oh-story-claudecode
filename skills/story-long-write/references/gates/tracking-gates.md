# 追踪门禁

> 本文件只定义 `追踪/` 主表的写法、同步时机和禁区。凡涉及 `时间线.md`、`角色状态.md`、`情报台账.md`、`伏笔.md` 的创建、更新、回炉，必须加载本文件。

---

## 一、追踪主表的定位

以下文件属于“当前有效事实表”，不是预案池：

- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报台账.md`
- `追踪/伏笔.md`

它们只能记录：
- 正文已经发生的事实
- 当前有效口径
- 已经落地的角色变化 / 事件变化 / 情报变化 / 伏笔状态

不允许记录：
- 未来章节预案
- 远期卷纲事件
- 尚未写出的角色变化
- 计划补、理论成立、准备补强
- 细纲里会发生但正文尚未发生的内容

---

## 二、四张主表各自能写什么

### 1. `追踪/时间线.md`

只允许写：
- 已写出的章节事件
- 已经发生的故事时间
- 已落地的角色参与关系

禁止：
- 提前写第 `N+1` 章及以后事件
- 把远期细纲事件写成当前时序
- 把“下一章大概率会发生”写成已发生
- 把“线索 / 怀疑 / 指向”写成“物证 / 坐实 / 确认”

### 2. `追踪/角色状态.md`

只允许写：
- 当前身份
- 当前能力
- 当前关系状态
- 当前公众形象
- 已发生的状态变更记录

禁止：
- 提前写未来登场人物的当前状态
- 提前写未来关系变化 / 未来立场变化
- 把“后面会变成盟友/敌人”写成当前状态
- 把“高度疑似主使 / 知情”写成“已主使 / 已执行”

### 3. `追踪/情报台账.md`

只允许写：
- 正文里已经出现的情报
- 已经明确的 `刷新章`
- 合法预估的 `预计兑现章`
- 正文已发生时才回填 `实际兑现章 / 实际战果`
- 必须包含 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果`

禁止：
- 尚未在正文出现的情报先入表
- 未出现正式情报文本时记为“已刷新 / 已埋”
- 把主角推断 / 危险征兆 / 章尾悬念直接冒充正式情报
- 把“死前吐词 / 半句失口”写满成“完整口供”
- 把“下一章可直接反制 / 即将兑现 / 已经布好局”写成 `实际战果`
- 正文尚未兑现时，先写 `实际兑现章 / 实际战果`

情报流硬规则：
- 情报流 / 预知流 / 系统流作品，自第一条正式情报出现起，`追踪/情报台账.md` 必须存在
- 写后验收前，至少完成 `刷新章 / 预计兑现章`
- 只有正文真正兑现后，才能回填 `实际兑现章 / 实际战果`
- 缺失台账、提前写满、或把悬念冒充兑现，直接按 `F5`

### 4. `追踪/伏笔.md`

只允许写：
- 正文已埋伏笔
- 已规划但尚未埋设的伏笔，状态必须明确为 `未埋`
- 已回收 / 已过期的伏笔

禁止：
- 尚未正文埋下，就记为 `已埋`
- 把“后面想补”的点伪装成已落地伏笔

---

## 三、合法分流

未来预案、远期计划、未落地细纲，必须放到：
- `大纲/`
- `卷纲/`
- `细纲/`
- `追踪/总施工单_*.md`
- `追踪/执行计划_*.md`

不得混入追踪主表。

`设定/角色/*.md` 中的“当前已落地事实 / 状态变更记录”同样按事实层处理：
- 只允许写正文已落地内容
- 不得提前写未来伤情、未来关系变化、未来身份变化
- 若角色卡其他区块引用了未来口径，必须明确标记为“设定锚点”，不得伪装成当前事实
- 不得把“怀疑 / 指向 / 关联迹象”写成“确认 / 已坐实”

---

## 四、同步时机

正文写完后立即检查是否需要同步：
- `追踪/伏笔.md`
- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报台账.md`

同步原则：
- 正文写到哪，追踪就只记到哪
- 正文没写到，不得先记
- 验收没过线，不得把未稳定口径写入追踪主表
- 同步时永远从正文抽事实，不得从细纲 / 卷纲 / 预案文件直接复制

---

## 五、双版本与假完成

以下情况直接按 `F5` 处理：

- 同一事件在正文和追踪中口径不一致
- 追踪写“已完成”，正文实际未落地
- 新旧版本并存，旧口径未废止
- 写后验收判“追踪同步通过”，但追踪内容和正文不一致
- 角色卡已要求创建，但 `设定/角色/` 目录仍缺关键角色条目
- 角色卡“当前已落地事实”区块写入了正文未发生内容
- 事实层文件把线索口径写满成结论口径
- 情报流章节正文已出现正式情报，但 `追踪/情报台账.md` 缺失
- 写后验收或反校验把“下一章预计发生”写成“本章已兑现 / 已完成”

特别是：
- 追踪先于正文超前
- 验收先于追踪核准
- 台账把悬念写成已兑现

都属于流程失败。

---

## 六、最小核对清单

每次同步前后至少核对：

- 正文里明确拿到什么，就只在追踪里记什么
- 正文里明确写出的角色变化，才回写角色状态
- 正文里明确出现的情报，才进入情报台账
- 正文里明确埋下的伏笔，才改成 `已埋`
- 同一章的正文 / 细纲 / 闸门 / 验收 / 追踪是否只有一个当前有效口径
- `设定/角色/` 是否已同步正文已落地的关键角色
- 角色卡中的伤情 / 关系 / 身份 / 证据口径，是否严格来自正文而不是预案
- 事实层文件的措辞，是否仍停留在正文已能承受的证据等级
