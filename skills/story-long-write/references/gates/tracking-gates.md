# 追踪文件先照这份过

> 这份就管 `追踪/` 这几份文件该怎么写、什么时候同步、哪些内容别提前写。凡是碰到 `时间线.md`、`角色状态.md`、`情报记录.md`、`伏笔.md` 的创建、更新、回修，都先照这份来。

---

## 一、追踪文件是干什么的

以下文件只记“正文里已经发生的事”，不是拿来存后面打算怎么写的：

- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报记录.md`
- `追踪/伏笔.md`

它们只能记录：
- 正文已经发生的事实
- 当前已经落地、正文能撑住的说法
- 已经落地的角色变化 / 事件变化 / 情报变化 / 伏笔状态

不允许记录：
- 未来章节预案
- 远期卷纲事件
- 尚未写出的角色变化
- 计划补、理论成立、准备补强
- 细纲里会发生但正文尚未发生的内容

---

## 二、这四份追踪文件各自能写什么

### 1. `追踪/时间线.md`

只允许写：
- 已写出的章节事件
- 已经发生的故事时间
- 已落地的角色参与关系

禁止：
- 提前写第 `N+1` 章及以后事件
- 把远期细纲事件写成当前时序
- 把“下一章大概率会发生”写成已发生
- 把“线索 / 怀疑 / 指向”写成“物证 / 坐实 / 确认”

### 2. `追踪/角色状态.md`

只允许写：
- 当前身份
- 当前能力
- 当前关系状态
- 当前公众形象
- 已发生的状态变更记录

禁止：
- 提前写未来登场人物的当前状态
- 提前写未来关系变化 / 未来立场变化
- 把“后面会变成盟友/敌人”写成当前状态
- 把“高度疑似主使 / 知情”写成“已主使 / 已执行”
- 把正文仍处于 `待查 / 半认 / 先记名 / 待补 / 待定 / 先记上再看` 的收益，直接概括成“这件事已经彻底轮到他拍板 / 大家已经都按他说的办 / 好处已经彻底落到手里”

收益到了哪一步的约束补充：

- `角色状态.md` 的“当前状态”不是总结口号，必须服从正文已落地到哪一步
- 若正文当前只做到：
  - 公开把原话收回去
  - 先拿到试办资格
  - 名字先写进纸面 / 名单 / 第一排位置
  - 先记名 / 先排上号 / 名单先落下去
  角色状态最多只可写成：
  - `已把XX先挂上去试办`
  - `已逼得对面先把名字写上去`
  - `已让这件事先按他的名字落纸`
  - `已先拿到记名 / 资格 / 先试一次的机会`
- 只有当正文已经明确落出“别人不能再按旧说法办事，而且主角这一步已经把人手、货路、名额、谁先拍板里的至少一项真的拿走”时，才允许在角色状态里升级成“这件事现在先按他这边办 / 对面的旧做法已经压不住场”
- 若第一步人 / 货 / 路 / 名额 / 资源尚未真实走成，不得在角色状态里写成“已拿到实利”

### 3. `追踪/情报记录.md`

只允许写：
- 正文里已经出现的情报
- 已经明确的 `刷新章`
- 合法预估的 `预计兑现章`
- 正文已发生时才补写 `实际兑现章 / 实际战果`
- 必须包含 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果`

禁止：
- 尚未在正文出现的情报先入表
- 未出现正式情报文本时记为“已刷新 / 已埋”
- 把主角推断 / 危险征兆 / 章尾悬念直接冒充正式情报
- 把“死前吐词 / 半句失口”直接写死成“完整口供”
- 把“下一章可直接反制 / 即将兑现 / 已经布好局”写成 `实际战果`
- 正文尚未兑现时，先写 `实际兑现章 / 实际战果`

情报流硬规则：
- 情报流 / 预知流 / 系统流作品，只要第一条正式情报已经出现，`追踪/情报记录.md` 就必须存在
- 写后检查前，至少先把 `刷新章 / 预计兑现章` 填完
- 只有正文真正兑现后，才准补写 `实际兑现章 / 实际战果`
- 缺这份情报记录、提前写死、或把悬念冒充兑现，直接按 `F5`
- 若这份情报记录里出现“正文里还没正式出现的情报原文”，就得立刻回退清掉；后面的写后检查、交叉复查和下一章都先别走
- 若正文已把该条情报兑现成现实改局，而 `情报记录.md` 仍只停留在 `预计兑现章 / 预计收益类型`，默认 `追踪同步 = 未完成`，直接按 `F5`
- 若正文里没有正式出现该条情报文本或怎么被触发，只是主角顺着物证 / 观察 / 现场判断推进，就不得先把它写成“已刷新正式情报”

非情报流分开判断补充：
- 若当前题材主要推进方向不是 `正式情报刷新 / 预知播报 / 系统文本 / 规则条文 / 播报式新信息`，默认不要创建 `追踪/情报记录.md`
- 以下题材若当前章组只是靠 `当场把原话收回去 / 抢先后次序 / 物证贴脸 / 公开试错 / 项目试跑 / 记名先后变化` 推进，而没有正式情报文本，同样不得硬建情报记录：
  - `职场 / 商战 / 创业`
  - `纯个人升级 / 个人突围`
  - `末世经营 / 规则经营`
  - `古代断案 / 悬疑追索` 中仅靠现场尸证推进、尚无正式口供 / 文书情报时
  - `脑洞玄幻高概念流` 中仅靠现场规则临时变更推进、尚无正式播报文本时
  - `历史争霸 / 王朝权谋` 中仅靠礼台试印、旧簿试主、军籍改挂、调兵先后变化推进时
  - `女频强情绪 / 婚恋拉扯` 中仅靠记名先后变化、主桌把话收回去、关系先后争夺推进时
  - `恐怖惊悚 / 民俗悬疑` 中仅靠器物异常、旧簿试错、供桌换位、后门争先推进时
  - `无限流 / 规则副本流` 中仅靠桌次、签册、旧签、倒计时、广播轮次、复核口令推进时
- 若误建了 `追踪/情报记录.md`，则必须二选一：
  - 删回非情报流分支，不再保留这份情报记录
  - 或把正文和这份情报记录补齐到完整情报流规格
- 只要 `追踪/情报记录.md` 已存在，默认就按情报流脚本规格验：
  - 正文需有独立情报引号块
  - 记录文件需有 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果`
  - 缺任一项，直接按 `F5`

规则播报与情报流分开判断补充：
- `广播补条 / 系统播报 / 轮次提示 / 复核口令 / 倒计时变化 / 桌面异常提示 / 旧签异常提示` 这类内容，若主功能只是：
  - 当场改先后次序
  - 当场改谁往前谁往后
  - 当场改执行先后
  - 当场改谁先背锅
  默认按 `规则执行次序` 处理，不自动升为情报流
- 只有同时满足以下条件时，才允许把规则播报升级为正式情报条目并写入 `追踪/情报记录.md`：
  - 正文中存在独立、可引用的正式文本本体
  - 该文本不只是当场执行次序口令，而是能跨章持续兑现的新信息
  - 写后检查能明确答出它的 `刷新章 / 预计兑现章 / 兑现收益类型`
- 若正文只是用广播或规则口令推动当前场面改局，却没有独立情报条目，不得为了迎合脚本误建情报记录
- 若规则副本 / 系统文 / 播报式题材被误打到情报流，先判“这只是现场执行顺序，还是能跨章起作用的正式情报文本”，再决定是否建这份情报记录；不得一律按情报流补表

### 4. `追踪/伏笔.md`

只允许写：
- 正文已埋伏笔
- 已规划但尚未埋设的伏笔，状态必须明确为 `未埋`
- 已回收 / 已过期的伏笔

禁止：
- 尚未正文埋下，就记为 `已埋`
- 把“后面想补”的点伪装成已落地伏笔

## 三、该放哪边

未来预案、远期计划、未落地细纲，必须放到：
- `大纲/`
- `卷纲/`
- `细纲/`
- `追踪/总安排表_*.md`
- `追踪/执行计划_*.md`

不得混进这几份追踪文件。

`设定/角色/*.md` 中的“当前已落地事实 / 状态变更记录”同样按事实层处理：
- 只允许写正文已落地内容
- 不得提前写未来伤情、未来关系变化、未来身份变化
- 若角色卡其他区块引用了未来说法，必须明确标记为“设定说明”，不得伪装成当前事实
- 不得把“怀疑 / 指向 / 关联迹象”写成“确认 / 已坐实”

## 四、同步时机

正文写完后立即检查是否需要同步：
- `追踪/伏笔.md`
- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报记录.md`

同步时按这个来：
- 正文写到哪，追踪就只记到哪
- 正文没写到，不得先往里写
- 检查没过，不得把还没坐实的说法写进追踪文件
- 同步时永远从正文抽事实，不得从细纲 / 卷纲 / 预案文件直接复制
- 追踪文件里若还留着未来章节的说法，默认当前章 `追踪同步 = 未完成`
- 若当前工作范围属于连续 `2-4` 章同一章组，分章检查完成后，得先生成或更新 `追踪/写后检查_第XXX-XXX章.md`，把它当成这组已经写实内容的总汇；没生成前，不得判 `追踪同步 = 通过`
- 章组总表中的每章收益与总收益，都必须同时实填：`目标到了哪一步 / 实际到了哪一步 / 证据句 / 为什么还没再往上走`
- 若当前任务属于冷启动 / 首章组测试，`设定/角色/` 中必须补齐本章组关键角色最小人物卡；缺卡时，即使 `时间线 / 角色状态 / 情报记录` 已更新，`追踪同步` 仍不得判通过
- 若当前章组涉及情报推进，章组总表中必须显式记录每条关键情报的当前证据等级：`线索 / 高度相关 / 半坐实 / 已坐实`；未标证据等级，不得补写 `情报记录.md`
- 每章写完后，这几份追踪文件至少必须能实答以下 `4` 个问题：
  - 主角这一章手里多了什么
  - 敌方这一章少了什么 / 被迫把什么收回去 / 被迫让出了什么
  - 哪个配角这一章被逼着换边、让路，或者先把话吞回去了
  - 哪条旧线 / 旧物证 / 旧名单这一章往前推进了一步
- 任一题答不上来，说明追踪只搭了个空架子、没跟上正文事实，直接按 `F5`

## 五、双版本与假完成

以下情况直接按 `F5` 处理：

- 同一事件在正文和追踪中说法不一致
- 追踪写“已完成”，正文实际未落地
- 新旧版本并存，旧说法未废止
- 写后检查判“追踪同步通过”，但追踪内容和正文不一致
- 章组总表只有“当前收益到了哪一步”，没有 `目标到了哪一步 / 实际到了哪一步 / 证据句 / 为什么还没再往上走`
- 角色卡已要求创建，但 `设定/角色/` 目录仍缺关键角色条目
- 冷启动样本的 `设定/角色/` 虽已创建，但关键角色卡事实层仍引用了未进入正文的未来说法
- 连续 `2` 章及以上同一章组已完成分章检查，却缺 `追踪/写后检查_第XXX-XXX章.md`
- 角色卡“当前已落地事实”区块写入了正文未发生内容
- 事实层文件把线索说满成结论
- 情报流章节正文已出现正式情报，但 `追踪/情报记录.md` 缺失
- 写后检查或交叉复查把“下一章预计发生”写成“本章已兑现 / 已完成”

特别是：
- 追踪先于正文超前
- 检查先于追踪核准
- 情报记录把悬念写成已兑现

- 角色状态提前写入下一章 / 后续章的立场、去向、交接人、主战场
- 角色状态用“这事已经彻底轮到他做主 / 大家已经都照他的话办”这类高概括词，覆盖正文仍只是 `待查 / 先记名 / 半认 / 待补 / 待定`
- 交叉复查明知事实层超前仍判“通过”

都属于流程失败。

## 六、最小核对清单

每次同步前后至少核对：

- 正文里明确拿到什么，就只在追踪里记什么
- 正文里明确写出的角色变化，才回写角色状态
- 正文里明确出现的情报，才进入情报记录
- 正文里明确埋下的伏笔，才改成 `已埋`
- 同一章的正文 / 细纲 / 闸门 / 检查 / 追踪是否只有一种当前有效说法
- `设定/角色/` 是否已同步正文已落地的关键角色
- 若本轮属于连续 `2-4` 章章组，`追踪/写后检查_第XXX-XXX章.md` 是否已存在，且已被用作这组已写事实的总汇
- 角色卡中的伤情 / 关系 / 身份 / 证据说法，是否严格来自正文而不是预案
- 事实层文件的措辞，是否仍停留在正文已能承受的证据等级
- `时间线.md`、`角色状态.md`、`伏笔.md` 是否已经写出本章新增的关键承接物证 / 名单 / 谁往前谁往后变了，而不是只保留抽象摘要
