# 细纲门禁

> 本文件负责单章细纲、章节组细纲、细纲与写前闸门反校验的准入规则。

---

## 一、细纲准入硬闸

进入正文前，细纲必须同时满足：

1. 写清 `本章主驱动`
2. 写清 `本章题面兑现`
3. 写清 `本章副驱动`
4. 写清 `题材识别结果`
5. 写清 `本章自动挂载专项 checks`
6. 写清 `本章专项风险`
7. 至少有一口外显战果
8. 至少有一个硬场面
9. 第一爆点明确
10. 高潮后续收益明确
11. 章尾停点明确
12. 对应 `设定/爽点模型说明书.md` 的爽点类型明确
13. 明确写清本章禁止滑入的错误写法
14. 章节标题类型、承载卖点、标题禁区明确
15. 本章事件规模等级明确
16. 本章核心设定 / 金手指 / 身份差是否直接参与翻盘明确
17. 本章核心对手 / 核心物证 / 核心资源 / 核心场面载体至少具名其三
18. 本章是否命中抽象占位拦截明确
19. 本章主事件是否来自 `设定/爽点模型说明书.md` 的事件池明确
20. 本章核心母场景明确
21. 与上一章相比，本章是否通过 `母场景避重复` 明确
22. 本章是否属于 `世界主动施压` 而非“小场景查线自转”明确
23. 本章残魂 / 金手指 `出手类型` 明确，且不是连续复用同一低效类型
24. 若本章是首章且作品存在足以改写默认现实认知的世界机制，必须明确 `首章世界总盘交代` 与 `首章世界底色显影锚点`
25. 本章 `主收益骨架` 明确，且与最近 `2-3` 章不重复
26. 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，必须明确 `世界底色续显点`
27. 本章残魂 / 金手指的 `外部可感知后果` 明确
28. 若本章相对上一章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，必须明确 `跨场景过桥`
29. 本章中后段至少 `1` 口 `外部反应 / 群体站位变化 / 协作方改口 / 系统口径变化` 明确
30. 本章目标字数与 `字数安全边际` 明确；若脚本下限为 `2900`，默认不得低于 `3100`
31. 本章 `新增信息白名单` 明确
32. 本章 `正文禁增项` 明确
33. 若本章依赖新地点 / 新人物 / 新物证 / 新规则起爆，必须写清其 `来源、用途、为何此章落地`
34. 若本章承接上一章章尾钩子，必须写清 `哪些信息已在上一章落地，哪些只能在本章新增`
35. 本章所属 `卷 / 章组 / 阶段任务` 明确
36. 本章服务的 `卷级任务 / 章组必拿战果` 明确
37. 本章调用的 `蓝图资产` 明确
38. 本章新落地的 `蓝图资产` 明确
39. 本章禁止越权新增的 `资产类型` 明确
40. 本章最值钱的 `1-2` 个硬场面，必须额外写成 `场面卡 / 动作拍序`
41. 本章 `世界显影点` 明确，且不是抽象的“继续展开世界观”
42. 本章 `世界显影载体` 明确，必须说明靠哪个场景 / 规则执行 / 资源争夺 / 群体反应落地
43. 若不是首章，必须明确本章是在“延续哪一块旧世界信息”或“新抬哪一块世界信息”，不得默认正文自己想起什么写什么
44. 若本章命中 `情报流 / 系统流 / 预知流 / 任务流`，必须明确 `本章情报条目`
45. 若本章命中上条，必须明确 `本章情报如何直接改现实`
46. 若本章是连续章节组中的第 `2-3` 章，必须明确 `最近三章反重复检查`
47. 若本章是连续章节组中的第 `2-3` 章，必须明确 `上一章章尾是否做空本章主任务`
47.1. 若本章是连续章节组中的第 `2-3` 章，必须明确 `本章翻盘发动机`
47.2. 若本章是连续章节组中的第 `2-3` 章，必须明确 `相对上一章换掉了哪两环`
48. 本章 `题面结果化兑现` 明确，不得只写“更接近题面卖点”
49. 本章 `题面兑现档位` 明确（`D1-D5`）
50. 写本章前，已完成 `纲层单一版本比对`，并写清冲突结果
51. 本章 `事件拍序` 明确，至少拆成 `3-5` 拍，不能只有“目标 -> 执行 -> 结果”
52. 本章 `情绪回摆` 明确，至少写清 `2` 次情绪摆动，不得只有单一路径情绪
53. 若本章属于连续章节组，必须明确它在 `三章小循环` 中属于 `起火 / 升压 / 兑现` 哪一位
53.1. 本章 `非预期障碍` 明确：
   - 至少写清 `1` 口不是主角原计划内的障碍
   - 障碍来源至少属于以下之一：
     - 第三方插刀
     - 世界环境改压
     - 对手临场换招
     - 协作方失手或摇摆
     - 资源 / 路线 / 时间窗口突然收口
53.2. 本章 `灾后反应 -> 两难 -> 决断过桥` 明确：
   - 至少写清哪一拍先让主角或关键角色被打断
   - 被打断后必须出现一次舍取、误判修正或临场改决断
   - 不得直接从受阻跳到结果
54. 本章关键角色 `进场三问` 明确，至少覆盖主角 + 核心对手 + 核心协作方 / 高价值角色中的在场者：
   - 现在想要什么
   - 现在怕失去什么
   - 进场前 `5` 分钟在做什么
55. 本章关键角色各自 `独有锚点` 明确，至少落到以下之一：
   - 外形
   - 动作习惯
   - 说话节奏
   - 观察偏好
   - 压场姿态
56. 本章关键配角 `独立动作` 明确：
   - 不得只负责认出 / 解释 / 害怕 / 失口
   - 必须至少有一次按自己利益做出的动作
57. 若本章存在 `2` 人及以上连续共演，必须明确 `删名可分辨依据`
58. 本章关键场景 `冲突分层` 明确，至少写清：
   - 表层冲突：这场明面上在抢什么
   - 里层冲突：这场暗地里各自真正护什么
59. 本章关键场景 `第三方插刀点` 明确：
   - 协作方 / 围观者 / 执行层 / 规则口 / 群体反应，至少有 `1` 口会改站位
60. 本章关键场景 `冲突升级动作` 明确：
   - 不得只靠对骂
   - 至少写清谁会抢、压、藏、删、挡、推、改口中的哪一类动作
61. 本章 `对白对标拆解结果` 明确：
   - 若项目已启用对标流程，必须先存在并回填 `对标/对标采样执行单.md`
   - 必须先按 `references/benchmark-sampling.md` 写清本轮采样档位
   - 必须写清本轮实际读了哪些区段、多少章、多少场景、约多少字、当前结论稳定度
   - 至少写清本章参考了哪 `1-3` 个对标样本
   - 必须拆成以下五项，而不是泛泛写“风格参考”：
     - 主角说话功能
     - 对手护点话术
     - 摇摆位露底方式
     - 围观位发声方式
     - 章尾收口方式
62. 本章 `对白禁学项` 明确：
   - 不得直接模仿对标书的成品金句、固定口头禅、具体桥段台词
   - 必须写清“这次借的是机制，不是成品声腔”
63. 本章 `叙述对标拆解结果` 明确：
   - 必须沿用与对白拆解一致的采样口径，不得一处记实、一处写虚
   - 至少写清本章参考了哪 `1-3` 个对标样本的叙述收力方式
   - 必须拆成以下四项：
     - 开场如何压现场，不靠旁白金句
     - 中段如何翻冲突，不靠作者翻译句
     - 人物站位如何通过动作和器物显出来，不靠旁白判词
     - 章尾如何停在新结果，不停在盖义句
64. 本章 `叙述禁学项` 明确：
   - 不得直接模仿对标书的成品开场句、成品收尾句、成品旁白金句
   - 必须写清“这次借的是叙述承载方式，不是现成句型”
65. 本章 `对标采样达标结论` 明确：
   - 当前采样是只够支持 `本章局部借法`，还是已经够支持 `正文通用规则`
   - 若未达 `references/benchmark-sampling.md` 中对应档位，不得把结论写成“通用规律 / 硬规则 / 一定如此”
66. 若本章命中 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，必须额外写清 `反说明书判定`：
   - 本章靠哪 `1-2` 个在场动作把规则压进场景
   - 本章哪一口 `试用 / 交易 / 排队 / 受阻 / 讨价 / 改规矩 / 利益变化` 才是真正主功能
   - 哪一段最容易滑成说明书，以及准备用什么人物反应或现实变化压住
67. 若本章命中上条，必须额外写清 `禁止写法`：
  - 不得先整段讲制度怎么运转，再让人物补动作
  - 不得把删掉后仍不影响场面推进的规则说明，当作正文主段
  - 不得让角色只剩“来问一句，好让作者解释规则”

开书级额外硬闸：
68. 已存在 `大纲/开书候选分叉.md`，且至少完成 `3` 组有效候选对比
69. 已存在 `追踪/开书排异表.md`，且已对首章母场景 / 首章翻盘模板 / 主角壳 / 第一明敌 / 第一现实战果完成排异
70. 已存在 `追踪/同题材去重记录.md`，且已对比项目内可见的同题材旧样本
71. 若同一开书题面与历史样本在 `主角壳 / 首章事件 / 第一母场景 / 第一明敌 / 第一战果` 中撞 `2` 项及以上，必须先改候选分叉与排异结果，不得进入蓝图
72. `开书候选分叉.md` 的最终入选方案，必须与 `开书总蓝图` 当前口径一致；若两处口径冲突，按 `F5`

缺任一项，不得进入正文。

## 三、细纲不通过的典型情况

- 只写“体现 / 展现 / 推进”，没写“谁做了什么”
- 主战果只是“知道更多”
- 没有硬场面
- 连续两章只查信息推进
- 题面卖点连续失频
- 主副驱动倒挂
- 连续 `2-3` 章没有任何新的世界显影点，只剩剧情推进
- 细纲写“会顺手交代世界 / 体现世界残酷 / 暗示更大盘面”，却说不清靠哪场冲突落地
- 新地点 / 新机构 / 新资源口已经进章，却没有把它写成明确的世界显影任务，只准备正文里临时解释
- 第一爆点之后没有第二口现实收益
- 章尾只剩“下一章继续”
- 与 `设定/爽点模型说明书.md` 的当前阶段爽法冲突
- 明明该写求生兑现 / 验货 / 经营 / 掌局，却滑成慢查黑幕或纯悬疑挂钩
- 标题只是物件名 / 线索名 / 抽象概括，没有冲突和翻页力
- 连续多章都只使用低规模事件推进，例如连续困在：
  - 小房间对话
  - 小范围搜证
  - 小股试探
  - 小层级口风互咬
- 作品有明确核心设定 / 金手指 / 身份差，但连续多章只负责“解释存在”，不直接参与翻盘
- 核心事件写成“某个人 / 某件东西 / 某个势力 / 一份证据 / 外部发生混乱”这种抽象占位
- 明明标了 `L3/L4`，却写不出群体规模、秩序变化、路线变化、场面载体
- 前 `10` 章没有具名明敌、具名摇摆角色、具名资源 / 权力节点
- 残魂 / 系统 / 情报只是“帮助判断”，没有造成可见现实结果
- 连续两章复用同类母场景，例如反复缩在：
  - 囚车 / 押解车 / 车板
  - 尸体 / 验尸 / 收尸
  - 驿站 / 旧棚 / 后院 / 临时窝点
  - 公验 / 对质 / 口供 / 翻口
  - 靠“规矩翻脸”代替真正的大事件推进
- 细纲名义上有事件池，实际本章主事件仍不是从事件池中挑出，而只是给旧小场景套一层大词
- 细纲主事件虽然具名，但实际仍只是：
  - 小卡口改口
  - 小证据翻面
  - 小范围拦人
  - 小层级互咬
  没有抬到 `群体失控 / 多方争夺 / 路线改写 / 会死人会断路` 的高情节档
- 本章冲突不是世界 / 势力 / 环境先压下来，而是主角又去顺着一条小线索慢慢摸
- 残魂 / 金手指连续两章都只承担“预警 / 判断增强 / 识别风险”，没有换成新的出手方式
- 首章仍让读者误以为这是纯世俗王朝流放文，到中后段才突然冒出仙侠玄异层
- 连续 `2-3` 章主收益骨架都一样，例如反复使用：
  - 情报指向关键物
  - 残魂压一拍逼失口
  - 主角抢下物证 / 账 / 令
  - 章尾再抬更高层敌意
- 非首章开始，玄异底色又掉回纯世俗冲突，只剩主角脑内还有残魂
- 残魂出手只有主角自己知道，外部人物、环境、器物、阵纹都没有出现可见反应差
- 上一章刚抛出新场钩子，下一章却没写主角为什么会去、怎么会到、为什么正好撞上
- 章首直接写“主角赶到时 / 主角来到 / 事情果然爆了”，但没有锁定路径和到场动机
- 场景、势力、时间段都切了，细纲却没写过桥，默认正文高概率跳脱

### 高情节事件硬规则

若题材天然适配：

- 情报流
- 复仇流

### 事件升级硬规则

章节组细纲必须单独回答：

- 这一组里，哪一章负责把事件量级抬高
- 抬高的维度是哪一个：
  - 死伤
  - 势力数量
  - 改盘范围
  - 控制权层级
  - 主角代价
- 这一组里，哪一章专门负责把 `关系 / 站位 / 协作链` 拉裂
- 这一组里，哪一章专门负责让主角在 `快拿到 -> 被打断 -> 改决断` 里做一次难选
- 若上一章已经是 `L2`，本章准备如何避免只是“同型 L2 换地点”

若答不出，默认这一章仍在低水平换皮，不得进入正文。

### 单章事件拍序硬规则

每章细纲必须单独写 `本章事件拍序`。

最低要求：
- 不得少于 `3` 拍，不宜少于 `4` 拍
- 推荐写成以下结构中的一种：
  - `目标 -> 受阻 -> 变招 -> 改盘`
  - `目标 -> 受阻 -> 假顺 -> 反压 -> 改盘`
  - `入场起火 -> 首次翻面 -> 中段反压 -> 代价 -> 章尾新驱动`

硬规则：
- 任一拍都必须写“谁做了什么，造成什么变化”
- 不得把 `情报出现 / 发现线索 / 到达现场 / 开始行动` 这种准备动作单独充当整拍主收益
- 若 `本章事件拍序` 只能写成 `想做什么 -> 做了 -> 成了/没成`，细纲不过线

### 情绪回摆硬规则

每章细纲必须单独写 `本章情绪回摆`，至少 `2` 次。

允许的回摆方向包括：
- `起势 -> 被打断 -> 再起`
- `压住 -> 破功 -> 强压`
- `想拿 -> 快拿到 -> 必须舍一口`
- `想信 / 想杀 / 想走 -> 不能这么做 -> 改决断`
- `局势看似顺 -> 代价落下 -> 反而更狠`

硬规则：
- 情绪回摆必须绑现实事件，不得只写心理活动
- 每次回摆至少要落到以下之一：
  - 动作停顿 / 变形
  - 临场改口
  - 错误选择或被迫舍取
  - 对手或盟友的站位变化
  - 代价突然落地
- 若情绪只靠旁白解释“他很怒 / 很痛 / 很纠结”，默认按承载失败处理
- 若情绪回摆里没有一次真正导致 `换打法 / 舍一口 / 改站位 / 改协作关系`，默认回摆偏假，不得进入正文

### 三章小循环硬规则

若本章处于连续章节组中，细纲必须单独写 `本章三章小循环位置`。

默认小循环：
- 第 `1` 章：`起火`
- 第 `2` 章：`升压`
- 第 `3` 章：`兑现`

三者定义：
- `起火`：抛出本轮要抢的具体目标，并让局面先失衡
- `升压`：把代价、阻力、两难、错判、关系拉扯抬高，不能直接平推
- `兑现`：至少改掉一口现实盘面，不能只换来“知道更多”

硬规则：
- 连续三章不得都写成并列单元任务
- 若第 `2` 章没有明显升压，只是重复执行前章打法，按机制层滑轨
- 若第 `3` 章没有现实兑现，只留下新线索、新危险、新猜测，按机制层滑轨

### 题面兑现档位硬规则

每章细纲必须单独写 `本章题面兑现档位`，用于防止“收益名称很大，真实档位很低”。

统一档位：
- `D1`：活下来
- `D2`：拿到干活资格 / 自保资格
- `D3`：拿到自保口
- `D4`：拿到分别人命的口
- `D5`：拿到一段公共秩序 / 资源分配权

硬规则：
- 若题面属于 `经营 / 配额 / 分配 / 夺权 / 拍卖 / 执牌 / 掌口 / 生火权 / 放票权 / 名额权`，细纲不得只写抽象收益名，必须明确档位
- 首章若只到 `D1-D2`，不得写成“题面已兑现”
- 若本章计划档位低于作品当前阶段最低要求，细纲不过线

### 纲层单一版本比对硬规则

每章细纲必须单独写 `纲层单一版本比对结果`。

至少比对：
- `设定/题材定位.md`
- `大纲/开书总蓝图.md`
- `大纲/全书章节总表.md`
- `大纲/卷纲_第X卷.md`
- `大纲/章组控制_第X卷.md`

至少回答：
- 本章规模等级是否一致
- 本章主场景层级是否一致
- 本章是继续低层掌口，还是应抬到更大盘面
- 本章主收益档位是否一致

输出格式至少包括：
- `一致 / 不一致`
- 若不一致，冲突文件有哪些
- 本章采用哪一个当前有效版本

只要不是 `一致`，不得进入正文。

### 新增信息白名单硬规则

每章细纲必须单独写 `本章新增信息白名单`，用于限制正文临时乱补。

白名单只允许收以下两类：
- 本章确实需要首次落地的新信息
- 已在卷纲 / 上章章尾 / 世界规则中预热，本章正式具名或正式实锤的信息

每条白名单至少写清：
- 新增项名称
- 类型：人物 / 势力 / 地点 / 物证 / 规则 / 时间钉 / 交接口 / 关系改判 / 设定能力
- 来源：为什么这章能合理出现
- 用途：它服务本章哪一口收益
- 落地方式：对话 / 动作 / 实物 / 场面 / 章尾哪一拍
- 是否需要同步追踪 / 角色卡 / 伏笔 / 情报台账

硬规则：
- 白名单为空，不代表正文可自由新增；代表本章禁止新增核心信息
- 若正文新增项不在白名单内，默认越权
- 若白名单超过 `3-5` 条，默认本章装载过量，需重审是否拆章

### 正文禁增项硬规则

每章细纲必须单独写 `本章正文禁增项`，至少明确：
- 哪些高风险信息这章绝不能临时发明
- 哪些关系 / 设定 / 规则只能保留为怀疑，不能直接坐实
- 哪些伏笔只能预热，不能抢跑兑现

通用默认禁增：
- 新上线直接具名且直接定性为真凶 / 主使
- 没有预热的新核心能力
- 没有过桥的新场面地点
- 没有来源的新关键物证
- 为了章尾好看临时补出的新暗号、新簿册、新路线、新交接点

### 纲层承载自审硬规则

若担心正文会偷加信息，先审细纲是不是在逼正文乱补。至少逐条自问：

- 不新增任何核心信息，这章还能不能完整跑通
- 第一爆点、第二口收益、章尾翻面，是否都已有现成载体
- 主事件如果去掉“临时冒出来的一样东西 / 一个人 / 一个地点”，是否立刻塌掉
- 细纲有没有用抽象词占坑，正文只能靠临场补名词来填
- 本章冲突和章尾是否已经写出足够硬的现实变化，而不是靠“再补个新料”撑住

任一项答不稳，先回细纲层，不得直接写正文。

### 关键硬场面卡硬规则

为尽量做到“正文施工时不再临时补桥、补动作、补险点”，每章细纲必须至少为本章最值钱的 `1` 个硬场面补写 `本章关键硬场面卡`；若本章存在多个高动作冲突点，默认写 `1-2` 个，不建议超过 `2` 个。

高优先级场面类型包括但不限于：

- 追逐 / 截堵 / 反绕 / 逃生
- 战斗 / 反杀 / 围杀 / 群战
- 争抢 / 验货 / 抢签 / 抢令 / 抢资源口
- 谈判翻桌 / 当众逼口 / 公开打脸 / 规则改写
- 群体改道 / 营地改序 / 阵线切换 / 灾异压场

每张 `场面卡 / 动作拍序` 至少写清：

- 场面名称：这一场到底在打什么
- 进入原因：为什么这场会在本章爆
- 现场载体：桥、巷、矿道、擂台、仓口、厅堂、车队、山路、阵眼、街区等
- 先手方：谁先发现 / 谁先动 / 谁先压
- 截堵点或爆点：冲突真正卡在哪里
- 主角预判依据 / 误判风险：主角凭什么敢这么做，做错会死什么
- 动作拍序：至少 `4-8` 拍，按“谁做了什么 -> 造成什么变化”写
- 险点：本场最容易死人 / 掉队 / 失手 / 翻车的一拍
- 脱身或翻盘结果：这一场打完，现实局势具体改了什么
- 禁止正文临时补的关键件：例如新暗号、新接头点、新地道、新证物、新关系改判

动作拍序示例格式：

- `A` 发现主角改向，点 `B/C` 两人从侧坡截下
- 主角借 `X` 提前看到护坡 / 埋伏 / 坍塌 / 空档，临时改走 `Y` 路线
- 跟随者里 `D` 在 `Z` 点差点掉队 / 暴露，逼主角当场补一手
- 对手扑空 / 陷入障碍 / 被迫改口 / 改站位，主角抢到入口 / 名分 / 时间差

不过线情形：

- 只有“双方交手、主角取胜、对手吃亏”这种结果句
- 只有情绪判断，没有现场载体、阻力点、险点
- 只有起点和终点，中间关键动作靠正文临场发明
- 场面能成立，全靠正文新增一个未备案的人、地点、物证或规则

### 事件规模等级说明

为避免开篇和前中期长期缩在低规模推进里，每章细纲必须标记事件规模等级：

- `L1 小规模`：单点对话、单房间冲突、两三人试探、局部搜证
- `L2 中规模`：多人围观、多人对峙、公开验货、局部规则翻面、单处硬冲突
- `L3 大规模`：群体混乱、路线变更、多人势力撞局、环境事件、公共秩序改写
- `L4 超常规模`：世界规则异动、灾异爆发、核心设定压场、大场面翻盘

通用硬规则：
- 前 `3` 章中，至少 `1` 章达到 `L3`
- 前 `10` 章中，至少 `2` 章达到 `L3`，至少 `1` 章达到 `L4` 或“同等强度的核心设定大翻盘”
- 不允许连续 `3` 章都停在 `L1`
- 若题材天然偏日常 / 言情 /校园，也必须用该题材对应的“大事件”替代 `L3/L4`，例如：
  - 家庭伦理：公开撕裂、重大秘密曝光、社会关系重排
  - 都市职场：公开事故、群体围观、舆论爆点、资源大战
  - 校园恋爱：集体活动翻盘、公开身份/关系冲击、群体评价改写
  - 奇幻仙侠：灾异、战斗、异象、势力撞局
- 凡标记 `L3/L4`，细纲中必须同时写出：
  - 至少一种群体规模描述
  - 至少一种被改写的公共秩序 / 路线 / 资源 / 营地 / 站位
  - 至少一种场面载体
- 缺任一项，规模等级自动降一档处理；若因此跌破前 `3/10` 章规模硬规则，细纲不过线

### 母场景避重复硬规则

每章细纲必须单独写明 `本章核心母场景`，用于防止正文长期缩在同一类低成本场景里打转。

核心母场景指：
- 本章最主要的冲突承载空间
- 本章最主要的动作载体
- 本章最主要的盘面翻动方式

默认高风险母场景：
- 囚车 / 押解路 / 临时看押点
- 尸体 / 验尸 / 收尸点
- 驿站 / 旧棚 / 暂驻黑店
- 公验 / 对质 / 翻供 / 逼口
- 小范围灭口 / 小股试探 / 小队夜袭

硬规则：
- 连续 `2` 章不得使用同类核心母场景
- 连续 `3` 章内，至少 `1` 章必须切到 `军 / 市 / 寨 / 矿 / 关 / 仓 / 城 / 灾异区 / 禁地` 之一
- 若上一章核心载体是 `尸 / 车 / 驿 / 验 / 规矩翻脸`，下一章必须优先从 `多人势力撞局 / 群体秩序变化 / 路线变化 / 资源争夺 / 环境爆事` 中选主事件
- 只换地名、不换冲突骨架，不算通过避重复

不过线情形：
- 囚车换成马车，仍是同一押解母场景
- 驿站换成破棚，仍是同一暂驻母场景
- 验尸换成公验，仍是同一口供 / 尸证母场景
- 把“更大势力来了”包在章尾，但本章主体还是同一组小场景

### 世界主动施压硬规则

对命中以下专项中的复合题材，默认要求本章冲突优先由世界或势力主动压来，而不是主角独自去找线索：
- 高环境压力 / 路线推进
- 超凡 / 异象 / 核心能力
- 信息差 / 情报 / 预知
- 权力博弈 / 阵营争夺 / 规则控制

细纲必须写清：
- 是哪一股 `势力 / 组织 / 群体 / 环境机制 / 资源竞争方` 先动
- 这股力量改写了什么 `路线 / 营地 / 粮路 / 货路 / 站位 / 规矩 / 资源`
- 主角不是“开始查”，而是“被压进局里后，借自身优势切进去改结果”

默认不过线情形：
- 先出一条线索，再让主角顺线摸人
- 先查尸证，再查口供，再查旧账
- 只有敌人来灭口，没有更大的外部盘面变化

### 首章世界总盘与世界底色显影硬规则

若首章命中以下任一类：
- 仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地
- 科幻 / 赛博 / 末世 / 异界 / 异能 / 超级职业体系
- 制度、交通、资源、身份运行方式明显不同于读者默认现实

则首章细纲必须同时写出：
- `首章世界总盘交代`
- `首章世界底色显影锚点`

`首章世界总盘交代` 至少要写清：
- 这到底是个什么世界，不是什么世界
- 这个世界的制度 / 日常 / 权力 / 生存方式为什么会这样运转
- 主角当前开局死局为什么正好由这套世界规则制造

`首章世界底色显影锚点` 则负责把上述规则现场化，让读者在前 `1/3` 真正看见它如何改动作、改规矩、改生死。

硬规则：
- 总盘交代不认纯设定表，必须能落到现场规矩、职业常识、群体反应、资源使用、制度执行中的至少 `2` 项
- 不得只写“局部边地特殊 / 局部行业特殊 / 局部副本特殊”，却不交代整个世界为什么容纳这套特殊
- 不得让读者先把作品误读成普通古代 / 普通都市 / 普通校园，再在中后段突然发现世界不是那个世界

不过线情形：
- 首章只有奇观，没有世界总定义
- 只有名词，没有制度和日常常识
- 只有边地 / 副本 / 门派 / 校园一角的特殊，却没有整世界逻辑
- 主角死局看起来像普通冲突，后面才知道其实依赖另一套世界规则

### 非首章世界底色续显硬规则

若题材命中 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，则第 `2-5` 章不得把世界重新写回纯世俗冲突。

硬规则：
- 至少每 `2` 章必须出现 `1` 次世界底色续显
- 续显必须落在 `环境反应 / 行业规矩 / 群体避讳 / 器物用途 / 旧阵旧印 / 灾异余波` 之一
- 不认“核心设定在脑子里说了一句”代替世界续显

合格示例方向：
- 不同行当、不同地点的人都默认遵守某种避煞、避潮、镇炁、压铃、封印规矩
- 旧阵、骨灯、镇物、异潮余波继续影响不同场所的秩序
- 不同行当的人对同一灾异 / 禁地 / 阵痕有共同反应

不过线情形：
- 第1章有玄异底色，第2章开始完全像现实黑幕文
- 所有玄异内容都缩回主角独享视角

### 主收益骨架轮换硬规则

每章细纲必须写清 `本章主收益骨架`，并与最近 `2-3` 章做对比，防止场景变了、爽法却没变。

主收益骨架指：
- 本章最值钱的胜利是怎么赢到的
- 读者最记得住的收益链条是什么

默认高风险重复骨架：
- 情报指关键物
- 残魂压一拍逼失口
- 主角抢下物证 / 账 / 令
- 章尾再抬更高层敌意

轮换方向至少选一项：
- 抢人而不是抢物
- 改规矩而不是拿证
- 收站队而不是逼失口
- 公开处刑 / 公开站队重排，而不是继续抢账
- 借环境灾异翻盘，而不是继续抢令

### 跨场景过桥硬规则

若本章相对上一章发生以下任一切换：
- 主场面地点切换
- 主势力 / 主对手切换
- 白天 / 夜里 / 次日等时间段切换
- 主事件载体切换

则细纲必须单列 `本章跨场景过桥`，至少写清：
- 主角为什么必须转去这个新场
- 主角靠什么信息 / 余波 / 人物动作 / 现场异常锁定这个新场
- 主角怎么到场，或为什么能提前卡位
- 主角到场后第一眼先看什么，为什么会正好撞上本章第一爆点

合格示例方向：
- 上章逼出敌方势力下场，本章细纲写明主角看见异常信号、判断当夜必出事，因此提前卡到关键入口
- 上章拿到名册或账本，本章细纲写明路线、票号、交接口或运输余波把主角直接引到新据点

不过线情形：
- 只写“章尾有钩子，下一章自然接”
- 只写“主角赶到时”而不写怎么锁到现场
- 只顾新场更炸，不顾主角行动链
- 让敌方主动犯错自毁，而不是主角总是抢到关键物

不过线情形：
- 连续 `2-3` 章只是把“账 / 令 / 牌 / 册 / 钥”换不同名词
- 主角每次都靠同样的“压一拍 -> 抢一物 -> 抬更高层”完成本章

### 核心设定参与硬规则

若作品存在以下任一“核心设定”：
- 金手指
- 系统
- 情报
- 残魂
- 特殊血脉
- 强身份反差
- 独特世界规则权限

则细纲必须明确写出：
- 本章该设定怎么直接参与翻盘
- 是不是只在解释、提醒、铺垫

默认不过线情形：
- 设定只负责说明背景
- 设定只负责告诉主角一条信息
- 设定只在章尾提示下一章
- 读者知道设定存在，但看不到它怎么直接改写现实
- 设定参与方式只写“帮助判断 / 帮助识别 / 提醒风险”，但没有任何可见现实结果

残魂 / 金手指出手类型至少从下列中明确选一类：
- `压迫类`：逼得对手失神、失口、露怯、露底
- `识破类`：拆穿伪装、禁印、假令、假货、假阵
- `夺势类`：提前一步抢人、抢位、抢令、抢路
- `超常动作类`：短时完成普通肉身做不到的动作
- `异象共鸣类`：借环境 / 灾异 / 旧物 / 旧阵放大压场

轮换硬规则：
- 连续 `2` 章不得只复用 `预警 / 判断增强` 型低效出手
- 前 `10` 章内，上述类型至少落地 `3` 类
- 若上一章已用 `识破类` 完成翻盘，下一章优先换成 `压迫类 / 夺势类 / 超常动作类`

外部可感知后果硬规则：
- 残魂 / 金手指出手后，至少要有 `1` 个外部层面的可见反应
- 可见反应可以落在：
  - 对手失神、失口、误判
  - 旁人看见不合常理动作后改站位
  - 器物、骨灯、阵纹、封印、灾异出现异常反馈
  - 环境秩序因此被改写一拍

不过线情形：
- 只有主角自己觉得更准
- 只有内心独白知道残魂出手
- 外部世界对残魂出手没有任何反应差

### 抽象占位拦截

以下表达不得作为细纲核心位的最终写法：

- 某个人
- 某件东西
- 某个势力
- 新的交接点
- 一份物证
- 更高层的人
- 外部发生混乱
- 某个活口

适用范围：
- `本章主驱动`
- `本章题面兑现`
- `核心事件`
- `情节点序列`
- `本章最值钱战果`

允许条件：
- 只可用于临时占位
- 且必须在同段内补出具名替代项

若核心位到落盘时仍保留上述抽象占位，直接按细纲不过线处理

---

## 四、情节点要求

每个情节点至少交代：

- 谁在动
- 动作落在哪
- 造成什么可见变化
- 局势往哪边偏了一寸

禁止只写：
- 发现线索
- 双方对质
- 主角翻盘
- 拿到物证
- 敌人震惊
- 留下悬念

---

## 五、情节点密度

- 前 `1/3` 尽快入局
- 中段必须有阻力变化
- 后 `1/3` 必须再拿一口现实收益

若连续 `3` 个情节点都只是“知道更多”，细纲不过线。

---

## 六、口径反校验

正文开写前，细纲和写前闸门必须逐项一致：

- 章节标题
- 核心对手
- 核心物证 / 核心收益
- 本章主驱动
- 本章最值钱战果
- 本章第一爆点
- 本章高潮后续收益
- 本章外显代价类型
- 本章硬场面
- 本章章尾钩子
- 本章目标字数

若同一章出现双版本口径，直接按 `F5` 处理。

额外核对：

- 具名明敌是否已经在前 `10` 章某处落位
- 具名摇摆角色是否已经在前 `10` 章某处落位
- 具名资源 / 权力节点是否已经在前 `10` 章某处落位

若三锚点缺任一项，前 `10` 章细纲组不得判通过。
