# 细纲门禁

> 本文件负责单章细纲、章节组细纲、细纲与写前闸门反校验的准入规则。

---

## 一、细纲准入硬闸

进入正文前，细纲必须同时满足：

1. 写清 `本章主驱动`
2. 写清 `本章题面兑现`
3. 写清 `本章副驱动`
4. 写清 `题材识别结果`
5. 写清 `本章自动挂载专项 checks`
6. 写清 `本章专项风险`
7. 至少有一口外显战果
8. 至少有一个硬场面
9. 第一爆点明确
10. 高潮后续收益明确
11. 章尾停点明确
12. 对应 `设定/爽点模型说明书.md` 的爽点类型明确
13. 明确写清本章禁止滑入的错误写法
14. 章节标题类型、承载卖点、标题禁区明确
15. 本章事件规模等级明确
16. 本章核心设定 / 金手指 / 身份差是否直接参与翻盘明确
17. 本章核心对手 / 核心物证 / 核心资源 / 核心场面载体至少具名其三
18. 本章是否命中抽象占位拦截明确
19. 本章主事件是否来自 `设定/爽点模型说明书.md` 的事件池明确
20. 本章核心母场景明确
21. 与上一章相比，本章是否通过 `母场景避重复` 明确
22. 本章是否属于 `世界主动施压` 而非“小场景查线自转”明确
23. 本章残魂 / 金手指 `出手类型` 明确，且不是连续复用同一低效类型
24. 若本章是首章且作品存在足以改写默认现实认知的世界机制，必须明确 `首章世界总盘交代` 与 `首章世界底色显影锚点`
25. 本章 `主收益骨架` 明确，且与最近 `2-3` 章不重复
26. 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，必须明确 `世界底色续显点`
27. 本章残魂 / 金手指的 `外部可感知后果` 明确
28. 若本章相对上一章发生 `地点 / 势力 / 时间段 / 事件载体` 切换，必须明确 `跨场景过桥`
29. 本章中后段至少 `1` 口 `外部反应 / 群体站位变化 / 协作方改口 / 系统口径变化` 明确
30. 本章目标字数与 `字数安全边际` 明确；若脚本下限为 `2900`，默认不得低于 `3100`
31. 本章 `新增信息白名单` 明确
32. 本章 `正文禁增项` 明确
33. 若本章依赖新地点 / 新人物 / 新物证 / 新规则起爆，必须写清其 `来源、用途、为何此章落地`
34. 若本章承接上一章章尾钩子，必须写清 `哪些信息已在上一章落地，哪些只能在本章新增`

缺任一项，不得进入正文。

## 三、细纲不通过的典型情况

- 只写“体现 / 展现 / 推进”，没写“谁做了什么”
- 主战果只是“知道更多”
- 没有硬场面
- 连续两章只查信息推进
- 题面卖点连续失频
- 主副驱动倒挂
- 第一爆点之后没有第二口现实收益
- 章尾只剩“下一章继续”
- 与 `设定/爽点模型说明书.md` 的当前阶段爽法冲突
- 明明该写求生兑现 / 验货 / 经营 / 掌局，却滑成慢查黑幕或纯悬疑挂钩
- 标题只是物件名 / 线索名 / 抽象概括，没有冲突和翻页力
- 连续多章都只使用低规模事件推进，例如连续困在：
  - 小房间对话
  - 小范围搜证
  - 小股试探
  - 小层级口风互咬
- 作品有明确核心设定 / 金手指 / 身份差，但连续多章只负责“解释存在”，不直接参与翻盘
- 核心事件写成“某个人 / 某件东西 / 某个势力 / 一份证据 / 外部发生混乱”这种抽象占位
- 明明标了 `L3/L4`，却写不出群体规模、秩序变化、路线变化、场面载体
- 前 `10` 章没有具名明敌、具名摇摆角色、具名资源 / 权力节点
- 残魂 / 系统 / 情报只是“帮助判断”，没有造成可见现实结果
- 连续两章复用同类母场景，例如反复缩在：
  - 囚车 / 押解车 / 车板
  - 尸体 / 验尸 / 收尸
  - 驿站 / 旧棚 / 后院 / 临时窝点
  - 公验 / 对质 / 口供 / 翻口
  - 靠“规矩翻脸”代替真正的大事件推进
- 细纲名义上有事件池，实际本章主事件仍不是从事件池中挑出，而只是给旧小场景套一层大词
- 本章冲突不是世界 / 势力 / 环境先压下来，而是主角又去顺着一条小线索慢慢摸
- 残魂 / 金手指连续两章都只承担“预警 / 判断增强 / 识别风险”，没有换成新的出手方式
- 首章仍让读者误以为这是纯世俗王朝流放文，到中后段才突然冒出仙侠玄异层
- 连续 `2-3` 章主收益骨架都一样，例如反复使用：
  - 情报指向关键物
  - 残魂压一拍逼失口
  - 主角抢下物证 / 账 / 令
  - 章尾再抬更高层敌意
- 非首章开始，玄异底色又掉回纯世俗冲突，只剩主角脑内还有残魂
- 残魂出手只有主角自己知道，外部人物、环境、器物、阵纹都没有出现可见反应差
- 上一章刚抛出新场钩子，下一章却没写主角为什么会去、怎么会到、为什么正好撞上
- 章首直接写“主角赶到时 / 主角来到 / 事情果然爆了”，但没有锁定路径和到场动机
- 场景、势力、时间段都切了，细纲却没写过桥，默认正文高概率跳脱
- 细纲目标字数直接贴脚本下限，没有任何安全边际
- 细纲目标字数仍习惯性卡在 `3000` 左右，导致正文虽能过脚本却反复触发接近下限告警
- 细纲只写主角与对手互顶，没有预留中后段外部反应位，导致正文后半只能补解释或心理句
- 细纲靠“到正文再补”才能成立，没有把新地点 / 新人物 / 新物证 / 新规则提前备案
- 细纲主事件要靠一个正文里临时发明的接头点、暗号、物证或关系变化才能跑起来
- 细纲没有区分“已落地事实”和“本章准入新增”，导致正文写着写着把预案直接写成既成事实
- 细纲把核心推进写得太空，正文只能临场发明新料来补场面、补章尾、补收益
- 细纲没有写清 `正文禁增项`，导致模型默认拥有无限补设定权

### 新增信息白名单硬规则

每章细纲必须单独写 `本章新增信息白名单`，用于限制正文临时乱补。

白名单只允许收以下两类：
- 本章确实需要首次落地的新信息
- 已在卷纲 / 上章章尾 / 世界规则中预热，本章正式具名或正式实锤的信息

每条白名单至少写清：
- 新增项名称
- 类型：人物 / 势力 / 地点 / 物证 / 规则 / 时间钉 / 交接口 / 关系改判 / 设定能力
- 来源：为什么这章能合理出现
- 用途：它服务本章哪一口收益
- 落地方式：对话 / 动作 / 实物 / 场面 / 章尾哪一拍
- 是否需要同步追踪 / 角色卡 / 伏笔 / 情报台账

硬规则：
- 白名单为空，不代表正文可自由新增；代表本章禁止新增核心信息
- 若正文新增项不在白名单内，默认越权
- 若白名单超过 `3-5` 条，默认本章装载过量，需重审是否拆章

### 正文禁增项硬规则

每章细纲必须单独写 `本章正文禁增项`，至少明确：
- 哪些高风险信息这章绝不能临时发明
- 哪些关系 / 设定 / 规则只能保留为怀疑，不能直接坐实
- 哪些伏笔只能预热，不能抢跑兑现

通用默认禁增：
- 新上线直接具名且直接定性为真凶 / 主使
- 没有预热的新核心能力
- 没有过桥的新场面地点
- 没有来源的新关键物证
- 为了章尾好看临时补出的新暗号、新簿册、新路线、新交接点

### 纲层承载自审硬规则

若担心正文会偷加信息，先审细纲是不是在逼正文乱补。至少逐条自问：

- 不新增任何核心信息，这章还能不能完整跑通
- 第一爆点、第二口收益、章尾翻面，是否都已有现成载体
- 主事件如果去掉“临时冒出来的一样东西 / 一个人 / 一个地点”，是否立刻塌掉
- 细纲有没有用抽象词占坑，正文只能靠临场补名词来填
- 本章冲突和章尾是否已经写出足够硬的现实变化，而不是靠“再补个新料”撑住

任一项答不稳，先回细纲层，不得直接写正文。

### 事件规模等级说明

为避免开篇和前中期长期缩在低规模推进里，每章细纲必须标记事件规模等级：

- `L1 小规模`：单点对话、单房间冲突、两三人试探、局部搜证
- `L2 中规模`：多人围观、多人对峙、公开验货、局部规则翻面、单处硬冲突
- `L3 大规模`：群体混乱、路线变更、多人势力撞局、环境事件、公共秩序改写
- `L4 超常规模`：世界规则异动、灾异爆发、核心设定压场、大场面翻盘

通用硬规则：
- 前 `3` 章中，至少 `1` 章达到 `L3`
- 前 `10` 章中，至少 `2` 章达到 `L3`，至少 `1` 章达到 `L4` 或“同等强度的核心设定大翻盘”
- 不允许连续 `3` 章都停在 `L1`
- 若题材天然偏日常 / 言情 /校园，也必须用该题材对应的“大事件”替代 `L3/L4`，例如：
  - 家庭伦理：公开撕裂、重大秘密曝光、社会关系重排
  - 都市职场：公开事故、群体围观、舆论爆点、资源大战
  - 校园恋爱：集体活动翻盘、公开身份/关系冲击、群体评价改写
  - 奇幻仙侠：灾异、战斗、异象、势力撞局
- 凡标记 `L3/L4`，细纲中必须同时写出：
  - 至少一种群体规模描述
  - 至少一种被改写的公共秩序 / 路线 / 资源 / 营地 / 站位
  - 至少一种场面载体
- 缺任一项，规模等级自动降一档处理；若因此跌破前 `3/10` 章规模硬规则，细纲不过线

### 母场景避重复硬规则

每章细纲必须单独写明 `本章核心母场景`，用于防止正文长期缩在同一类低成本场景里打转。

核心母场景指：
- 本章最主要的冲突承载空间
- 本章最主要的动作载体
- 本章最主要的盘面翻动方式

默认高风险母场景：
- 囚车 / 押解路 / 临时看押点
- 尸体 / 验尸 / 收尸点
- 驿站 / 旧棚 / 暂驻黑店
- 公验 / 对质 / 翻供 / 逼口
- 小范围灭口 / 小股试探 / 小队夜袭

硬规则：
- 连续 `2` 章不得使用同类核心母场景
- 连续 `3` 章内，至少 `1` 章必须切到 `军 / 市 / 寨 / 矿 / 关 / 仓 / 城 / 灾异区 / 禁地` 之一
- 若上一章核心载体是 `尸 / 车 / 驿 / 验 / 规矩翻脸`，下一章必须优先从 `多人势力撞局 / 群体秩序变化 / 路线变化 / 资源争夺 / 环境爆事` 中选主事件
- 只换地名、不换冲突骨架，不算通过避重复

不过线情形：
- 囚车换成马车，仍是同一押解母场景
- 驿站换成破棚，仍是同一暂驻母场景
- 验尸换成公验，仍是同一口供 / 尸证母场景
- 把“更大势力来了”包在章尾，但本章主体还是同一组小场景

### 世界主动施压硬规则

对命中以下专项中的复合题材，默认要求本章冲突优先由世界或势力主动压来，而不是主角独自去找线索：
- 高环境压力 / 路线推进
- 超凡 / 异象 / 核心能力
- 信息差 / 情报 / 预知
- 权力博弈 / 阵营争夺 / 规则控制

细纲必须写清：
- 是哪一股 `势力 / 组织 / 群体 / 环境机制 / 资源竞争方` 先动
- 这股力量改写了什么 `路线 / 营地 / 粮路 / 货路 / 站位 / 规矩 / 资源`
- 主角不是“开始查”，而是“被压进局里后，借自身优势切进去改结果”

默认不过线情形：
- 先出一条线索，再让主角顺线摸人
- 先查尸证，再查口供，再查旧账
- 只有敌人来灭口，没有更大的外部盘面变化

### 首章世界总盘与世界底色显影硬规则

若首章命中以下任一类：
- 仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地
- 科幻 / 赛博 / 末世 / 异界 / 异能 / 超级职业体系
- 制度、交通、资源、身份运行方式明显不同于读者默认现实

则首章细纲必须同时写出：
- `首章世界总盘交代`
- `首章世界底色显影锚点`

`首章世界总盘交代` 至少要写清：
- 这到底是个什么世界，不是什么世界
- 这个世界的制度 / 日常 / 权力 / 生存方式为什么会这样运转
- 主角当前开局死局为什么正好由这套世界规则制造

`首章世界底色显影锚点` 则负责把上述规则现场化，让读者在前 `1/3` 真正看见它如何改动作、改规矩、改生死。

硬规则：
- 总盘交代不认纯设定表，必须能落到现场规矩、职业常识、群体反应、资源使用、制度执行中的至少 `2` 项
- 不得只写“局部边地特殊 / 局部行业特殊 / 局部副本特殊”，却不交代整个世界为什么容纳这套特殊
- 不得让读者先把作品误读成普通古代 / 普通都市 / 普通校园，再在中后段突然发现世界不是那个世界

不过线情形：
- 首章只有奇观，没有世界总定义
- 只有名词，没有制度和日常常识
- 只有边地 / 副本 / 门派 / 校园一角的特殊，却没有整世界逻辑
- 主角死局看起来像普通冲突，后面才知道其实依赖另一套世界规则

### 非首章世界底色续显硬规则

若题材命中 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，则第 `2-5` 章不得把世界重新写回纯世俗冲突。

硬规则：
- 至少每 `2` 章必须出现 `1` 次世界底色续显
- 续显必须落在 `环境反应 / 行业规矩 / 群体避讳 / 器物用途 / 旧阵旧印 / 灾异余波` 之一
- 不认“核心设定在脑子里说了一句”代替世界续显

合格示例方向：
- 不同行当、不同地点的人都默认遵守某种避煞、避潮、镇炁、压铃、封印规矩
- 旧阵、骨灯、镇物、异潮余波继续影响不同场所的秩序
- 不同行当的人对同一灾异 / 禁地 / 阵痕有共同反应

不过线情形：
- 第1章有玄异底色，第2章开始完全像现实黑幕文
- 所有玄异内容都缩回主角独享视角

### 主收益骨架轮换硬规则

每章细纲必须写清 `本章主收益骨架`，并与最近 `2-3` 章做对比，防止场景变了、爽法却没变。

主收益骨架指：
- 本章最值钱的胜利是怎么赢到的
- 读者最记得住的收益链条是什么

默认高风险重复骨架：
- 情报指关键物
- 残魂压一拍逼失口
- 主角抢下物证 / 账 / 令
- 章尾再抬更高层敌意

轮换方向至少选一项：
- 抢人而不是抢物
- 改规矩而不是拿证
- 收站队而不是逼失口
- 公开处刑 / 公开站队重排，而不是继续抢账
- 借环境灾异翻盘，而不是继续抢令

### 跨场景过桥硬规则

若本章相对上一章发生以下任一切换：
- 主场面地点切换
- 主势力 / 主对手切换
- 白天 / 夜里 / 次日等时间段切换
- 主事件载体切换

则细纲必须单列 `本章跨场景过桥`，至少写清：
- 主角为什么必须转去这个新场
- 主角靠什么信息 / 余波 / 人物动作 / 现场异常锁定这个新场
- 主角怎么到场，或为什么能提前卡位
- 主角到场后第一眼先看什么，为什么会正好撞上本章第一爆点

合格示例方向：
- 上章逼出敌方势力下场，本章细纲写明主角看见异常信号、判断当夜必出事，因此提前卡到关键入口
- 上章拿到名册或账本，本章细纲写明路线、票号、交接口或运输余波把主角直接引到新据点

不过线情形：
- 只写“章尾有钩子，下一章自然接”
- 只写“主角赶到时”而不写怎么锁到现场
- 只顾新场更炸，不顾主角行动链
- 让敌方主动犯错自毁，而不是主角总是抢到关键物

不过线情形：
- 连续 `2-3` 章只是把“账 / 令 / 牌 / 册 / 钥”换不同名词
- 主角每次都靠同样的“压一拍 -> 抢一物 -> 抬更高层”完成本章

### 核心设定参与硬规则

若作品存在以下任一“核心设定”：
- 金手指
- 系统
- 情报
- 残魂
- 特殊血脉
- 强身份反差
- 独特世界规则权限

则细纲必须明确写出：
- 本章该设定怎么直接参与翻盘
- 是不是只在解释、提醒、铺垫

默认不过线情形：
- 设定只负责说明背景
- 设定只负责告诉主角一条信息
- 设定只在章尾提示下一章
- 读者知道设定存在，但看不到它怎么直接改写现实
- 设定参与方式只写“帮助判断 / 帮助识别 / 提醒风险”，但没有任何可见现实结果

残魂 / 金手指出手类型至少从下列中明确选一类：
- `压迫类`：逼得对手失神、失口、露怯、露底
- `识破类`：拆穿伪装、禁印、假令、假货、假阵
- `夺势类`：提前一步抢人、抢位、抢令、抢路
- `超常动作类`：短时完成普通肉身做不到的动作
- `异象共鸣类`：借环境 / 灾异 / 旧物 / 旧阵放大压场

轮换硬规则：
- 连续 `2` 章不得只复用 `预警 / 判断增强` 型低效出手
- 前 `10` 章内，上述类型至少落地 `3` 类
- 若上一章已用 `识破类` 完成翻盘，下一章优先换成 `压迫类 / 夺势类 / 超常动作类`

外部可感知后果硬规则：
- 残魂 / 金手指出手后，至少要有 `1` 个外部层面的可见反应
- 可见反应可以落在：
  - 对手失神、失口、误判
  - 旁人看见不合常理动作后改站位
  - 器物、骨灯、阵纹、封印、灾异出现异常反馈
  - 环境秩序因此被改写一拍

不过线情形：
- 只有主角自己觉得更准
- 只有内心独白知道残魂出手
- 外部世界对残魂出手没有任何反应差

### 抽象占位拦截

以下表达不得作为细纲核心位的最终写法：

- 某个人
- 某件东西
- 某个势力
- 新的交接点
- 一份物证
- 更高层的人
- 外部发生混乱
- 某个活口

适用范围：
- `本章主驱动`
- `本章题面兑现`
- `核心事件`
- `情节点序列`
- `本章最值钱战果`

允许条件：
- 只可用于临时占位
- 且必须在同段内补出具名替代项

若核心位到落盘时仍保留上述抽象占位，直接按细纲不过线处理

---

## 四、情节点要求

每个情节点至少交代：

- 谁在动
- 动作落在哪
- 造成什么可见变化
- 局势往哪边偏了一寸

禁止只写：
- 发现线索
- 双方对质
- 主角翻盘
- 拿到物证
- 敌人震惊
- 留下悬念

---

## 五、情节点密度

- 前 `1/3` 尽快入局
- 中段必须有阻力变化
- 后 `1/3` 必须再拿一口现实收益

若连续 `3` 个情节点都只是“知道更多”，细纲不过线。

---

## 六、口径反校验

正文开写前，细纲和写前闸门必须逐项一致：

- 章节标题
- 核心对手
- 核心物证 / 核心收益
- 本章主驱动
- 本章最值钱战果
- 本章第一爆点
- 本章高潮后续收益
- 本章外显代价类型
- 本章硬场面
- 本章章尾钩子
- 本章目标字数

若同一章出现双版本口径，直接按 `F5` 处理。

额外核对：

- 具名明敌是否已经在前 `10` 章某处落位
- 具名摇摆角色是否已经在前 `10` 章某处落位
- 具名资源 / 权力节点是否已经在前 `10` 章某处落位

若三锚点缺任一项，前 `10` 章细纲组不得判通过。
