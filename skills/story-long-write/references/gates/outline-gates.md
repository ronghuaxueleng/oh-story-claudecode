# 细纲这道硬检查

> 这一页管的是单章细纲、章节组细纲，还有细纲和写前闸门怎么对。这里没过，就别进正文。

---

## 一、细纲进正文前先过这道坎

进正文前，细纲得把下面这些事先答出来：

1. 答清 `本章主要往前推什么`
2. 答清 `本章书名主打卖点兑现`
3. 答清 `本章副线推进`
4. 答清 `题材识别结果`
5. 答清 `本章要额外盯的单独检查`
6. 答清 `本章单独风险`
7. 至少有一项外显战果
8. 至少有一个硬场面
9. 第一爆点答得清
10. 高潮后续收益答得清
11. 章尾停点答得清
12. 对应 `设定/爽点怎么落.md` 的爽点类型答得清
13. 答清本章最容易滑进去的错误写法
14. 章节标题类型、主卖点、标题禁区答得清
15. 本章事件规模等级答得清
16. 本章最主要的设定 / 金手指 / 身份差是否直接参与翻盘答得清
17. 本章主要对手 / 物证 / 资源 / 场面靠什么撑住，至少具名其三
18. 本章是否碰上抽象占位拦截答得清
19. 本章主事件是否来自 `设定/爽点怎么落.md` 的事件池答得清
20. 本章主场面答得清
21. 与上一章相比，本章是否通过 `母场景避重复` 答得清
22. 本章是否属于 `世界主动施压` 而非“小场景查线自转”答得清
23. 本章残魂 / 金手指 `出手类型` 答得清，且不是连续复用同一低效类型
24. 若本章是首章且作品存在足以改写默认现实认知的世界规矩，得答清 `首章世界总盘交代` 与 `首章世界底色落点`
25. 本章 `主收益走法` 答得清，而且和最近 `2-3` 章不重复
26. 若本章不是首章且题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，得答清 `世界底色续显点`
27. 本章残魂 / 金手指的 `外部可感知后果` 答得清
28. 若本章相对上一章发生 `地点 / 势力 / 时间段 / 事件靠什么撑住` 切换，得答清 `跨场景过桥`
29. 本章中后段至少 `1` 口 `外部反应 / 群体里谁往前谁往后变了 / 协作方把原话收回去 / 系统给出的说法变了` 答得清
30. 本章目标字数与 `字数安全边际` 答得清；若脚本下限为 `2900`，一般别低于 `3100`
31. 本章 `新增信息允许清单` 答得清
32. 本章 `正文禁增项` 答得清
33. 若本章依赖新地点 / 新人物 / 新物证 / 新规则起爆，得写清它的 `来源、用途、为何此章落地`
34. 若本章承接上一章章尾钩子，得写清 `哪些信息已在上一章落地，哪些只能在本章新增`
35. 本章所属 `卷 / 章组 / 阶段任务` 答得清
36. 本章服务的 `卷级任务 / 章组必拿战果` 答得清
37. 本章调用的 `蓝图资产` 答得清
38. 本章新落地的 `蓝图资产` 答得清
39. 本章禁止越权新增的 `资产类型` 答得清
40. 本章最该先盯的 `1-2` 个硬场面，得额外写成 `场面卡 / 动作拍序`
41. 本章 `世界信息点` 答得清，而且不是抽象的“继续展开世界观”
42. 本章 `世界信息靠什么落地` 答得清，得说明靠哪个场景 / 规则执行 / 资源争夺 / 群体反应落地
43. 若不是首章，得答清本章是在“延续哪一块旧世界信息”还是“新抬哪一块世界信息”，别指望正文自己想起什么写什么
44. 若本章碰上 `情报流 / 系统流 / 预知流 / 任务流`，得答清 `本章情报条目`
45. 若本章碰上上条，得答清 `本章情报如何直接改现实`
46. 若本章是连续章节组中的第 `2-3` 章，得答清 `最近三章反重复检查`
47. 若本章是连续章节组中的第 `2-3` 章，得答清 `上一章章尾是否做空本章主任务`
47.1. 若本章是连续章节组中的第 `2-3` 章，得答清 `本章翻盘推进骨架`
47.2. 若本章是连续章节组中的第 `2-3` 章，得答清 `相对上一章换掉了哪两环`
48. 本章 `书名主打结果化兑现` 答得清，别只写“更接近书名卖点”
49. 本章 `书名主打兑现到了哪一步` 答得清（`D1-D5`）
50. 写本章前，把 `纲层单一版本比对` 先做完，并写清冲突结果
51. 本章 `事件拍序` 答得清，至少拆成 `3-5` 拍，不能只有“目标 -> 执行 -> 结果”
52. 本章 `情绪回摆` 答得清，至少写清 `2` 次情绪摆动，别只剩一条直线情绪
53. 若本章属于连续章节组，得答清它在 `三章小循环` 中属于 `起火 / 升压 / 兑现` 哪一位
53.1. 本章 `非预期障碍` 答得清：
   - 至少写清 `1` 口不是主角原计划内的障碍
   - 障碍来源至少得落在下面之一：
     - 第三方插刀
     - 世界环境改压
     - 对手临场换招
     - 协作方失手或摇摆
     - 资源 / 路线 / 时间窗口突然卡死
53.2. 本章 `灾后反应 -> 两难 -> 决断过桥` 答得清：
   - 至少写清哪一拍先让主角或关键角色被打断
   - 被打断后得出现一次舍取、误判修正或临场改决断
   - 别直接从受阻跳到结果
54. 本章关键角色 `进场三问` 答得清，至少覆盖主角 + 主要对手 + 主要协作方 / 重要角色中的在场者：
   - 现在想要什么
   - 现在怕失去什么
   - 进场前 `5` 分钟在做什么
55. 本章关键角色各自 `独有记忆点` 答得清，至少落到以下之一：
   - 外形
   - 动作习惯
   - 说话节奏
   - 观察偏好
  - 压人时的样子
56. 本章关键配角 `独立动作` 答得清：
   - 别只负责认出 / 解释 / 害怕 / 失口
   - 至少得有一次按自己利益做出的动作
57. 若本章存在 `2` 人及以上连续共演，得答清 `删名可分辨依据`
58. 本章关键场景 `冲突分层` 答得清，至少写清：
   - 表层冲突：这场明面上在抢什么
   - 里层冲突：这场暗地里各自真正护什么
59. 本章关键场景 `第三方插刀点` 答得清：
   - 协作方 / 围观者 / 执行层 / 规则提示 / 群体反应，至少得有 `1` 口会让人前后次序变掉，或者逼谁先退半步
60. 本章关键场景 `冲突升级动作` 答得清：
   - 别只靠对骂
   - 至少写清谁会抢、压、藏、删、挡、推、把原话吞回去里的哪一类动作
61. 本章 `对白对标拆解结果` 要答清：
- 若项目已启用对标流程，把 `对标/对标采样执行单.md` 先建好并补写
   - 先按 `references/benchmark-sampling.md` 答清本轮采样深度
   - 答清本轮实际读了哪些区段、多少章、多少场景、约多少字、这次判断稳到哪一步
   - 至少答清本章参考了哪 `1-3` 个对标样本
   - 得拆成下面这五项，别泛泛写“风格参考”：
     - 主角说话功能
     - 对手护点话术
     - 摇摆的人怎么露底
     - 围观的人怎么开口
     - 章尾落法
62. 本章 `对白禁学项` 要答清：
   - 别直接模仿对标书里几乎能直接搬过来的句子、固定口头禅、具体桥段台词
   - 得写清“这次借的是路数，不是成品声腔”
63. 本章 `叙述对标拆解结果` 要答清：
   - 得沿用与对白拆解一致的采样说法，别一处记实、一处写虚
   - 至少写清本章参考了哪 `1-3` 个对标样本的叙述收力方式
   - 得拆成下面这四项：
     - 开场如何压现场，不靠把参考书旁白稍微改改就能塞进来的句子
     - 中段如何翻冲突，不靠作者翻译句
     - 人物谁往前谁往后，如何通过动作和器物显出来，不靠旁白判词
     - 章尾如何停在新结果，不停在盖义句
64. 本章 `叙述禁学项` 要答清：
   - 别直接模仿对标书的成品开场句、成品结尾句、把参考书旁白稍微改改就能塞进来的句子
   - 得写清“这次借的是叙述怎么落地，不是现成句型”
65. 本章 `对标采样现在够写到哪一步` 要答清：
   - 当前采样只是够支撑 `本章局部借法`，还是已经够支撑 `正文通用规则`
   - 若未达 `references/benchmark-sampling.md` 里对应的采样深度，别把结论写成“通用规律 / 硬规则 / 一定如此”
66. 若本章碰上 `经营 / 配额 / 分配 / 卖货 / 流程口 / 制度口 / 试炼口 / 审批口 / 封存口 / 看房配套 / 资格口 / 交易口` 任一功能场景，得额外答清 `别写成讲解稿判定`：
   - 本章靠哪 `1-2` 个在场动作把规则压进场景
   - 本章哪一步 `试用 / 交易 / 排队 / 受阻 / 讨价 / 改规矩 / 利益变化` 才是真正主功能
   - 哪一段最容易滑成讲解稿，以及准备用什么人物反应或现实变化压住
67. 若本章碰上上条，得额外答清 `禁止写法`：
  - 别先整段讲制度怎么运转，再让人物补动作
  - 别把删掉后仍不影响场面推进的规则说明，当作正文主段
  - 别让角色只剩“来问一句，好让作者解释规则”
68. 若本章明确调用事件模板 / 推进骨架模板 / 情节库模板，得额外答清 `模板榨干必答项`：
  - 本章主模板是哪 `1-3` 个
  - 每个模板各自改什么现实状态
  - 每个模板里对手是怎么当场压回来的
  - 每个模板里第三方到底又改了什么
  - 每个模板的主角代价是什么
  - 每个模板章尾停在哪个最该卡住的动作
  - 模板之间如何串联
69. 若当前题材属于 `规则流 / 经营流 / 脑洞玄幻 / 连续翻盘文`，细纲阶段一般同时读取：
  - `references/template-motherboards.md`
  - `references/template-engine-combos.md`
  - `references/template-engine-exhaustion.md`
70. 若只写模板名，不写现实变化与前后怎么接，就当细纲没过准入线，一般至少 `F3`

开书时还得额外拦这几下：
68. 已存在 `大纲/开书候选分叉.md`，而且至少完成 `3` 组有效候选对比
69. 已存在 `追踪/开书排异表.md`，而且已对首章母场景 / 首章翻盘模板 / 主角壳 / 第一明敌 / 第一现实战果完成排异
70. 已存在 `追踪/同题材去重记录.md`，而且已对比项目内可见的同题材旧样本
71. 若同一开书主打和历史样本在 `主角壳 / 首章事件 / 第一母场景 / 第一明敌 / 第一战果` 中撞 `2` 项及以上，就先改候选分叉与排异结果，别进蓝图
72. `开书候选分叉.md` 的最终入选方案，得和 `开书总蓝图` 当前说法一致；若两处说法冲突，按 `F5`

少一项，就别进正文。

## 三、细纲不通过的典型情况

- 只写“体现 / 展现 / 推进”，没写“谁做了什么”
- 主战果只是“知道更多”
- 没有硬场面
- 连续两章只查信息推进
- 书名主打卖点连续几章都没真正出来
- 主副驱动倒挂
- 连续 `2-3` 章没有任何新的世界信息点，只剩剧情推进
- 细纲写“会顺手交代世界 / 体现世界残酷 / 暗示更大局面”，却说不清靠哪场冲突落地
- 新地点 / 新机构 / 新资源去向已经进章，却没有把它写成明确的世界信息任务，只准备正文里临时解释
- 第一爆点之后没有第二步现实收益
- 章尾只剩“下一章继续”
- 和 `设定/爽点怎么落.md` 里定好的这一段爽法对不上
- 明明该写求生兑现 / 验货 / 经营 / 掌局，却滑成慢查黑幕或纯悬疑挂钩
- 标题只是物件名 / 线索名 / 抽象概括，没有冲突和翻页力
- 连续多章都只使用低规模事件推进，例如连续困在：
  - 小房间对话
  - 小范围搜证
  - 小股试探
  - 小圈子里来回顶嘴、拆台
- 作品有明确主要设定 / 金手指 / 身份差，但连续多章只负责“解释存在”，不直接参与翻盘
- 最主要那件事写成“某个人 / 某件东西 / 某个势力 / 一份证据 / 外部发生混乱”这种抽象占位
- 明明标了 `L3/L4`，却写不出群体规模、秩序变化、路线变化、场面靠什么撑住
- 前 `10` 章没有具名明敌、具名摇摆角色、具名资源 / 权力节点
- 残魂 / 系统 / 情报只是“帮助判断”，没有造成可见现实结果
- 连续两章复用同类母场景，例如反复缩在：
  - 囚车 / 押解车 / 车板
  - 尸体 / 验尸 / 收尸
  - 驿站 / 旧棚 / 后院 / 临时窝点
  - 公验 / 对质 / 口供 / 逼人把原话吞回去
  - 靠“规矩翻脸”代替真正的大事件推进
- 细纲名义上有事件池，实际本章主事件仍不是从事件池中挑出，而只是给旧小场景套一层大词
- 细纲主事件虽然具名，但实际仍只是：

- 直接抄对标书的成品口气，导致本书人物没有自己的语言指纹

- 对标记录没有写实际读了哪些区段、多少章、多少场景、多少字，却直接下“整体风格 / 通用规律 / 已精读”式结论
- 当前采样明明只够支持本章局部借法，却在细纲里直接上升成正文通用规则或 skill 级硬规则
- 场景只有单层冲突，只有“主角想做 -> 对手拦一下 -> 主角再抢”

- 场景里只有主角和对手互顶，没有第三方借机让谁前谁后变掉、把话吞回去、顺手插刀

- 上一章章尾已经把本章主要目标做完 / 做空 / 做废，本章仍照旧展开
- 书名主打兑现写成“更接近分东西的权 / 更接近拍板的位置 / 更接近掌规则”，而不是本章现实里真的抢到哪一步实权

- `题材定位 / 章节总表 / 章组控制 / 本章细纲` 对同一章的规模等级、主场景层级、升级说法不一致
- 本章只有单一事件结果，没有拆出 `受阻 -> 变招 -> 代价 / 被压回来 -> 现场到底改了什么`
- 本章目标情绪只有一种直线输出，例如全章只稳、只查、只强、只怒，没有情绪回摆
- 本章所谓“情绪拉扯”只靠心理词或解释句，没有落到动作失真、把话收回去、停顿、舍取、失态
- 本章没有任何非预期障碍，所有阻力都在预料内，导致场面像照表执行
- 本章受阻后直接跳结果，没有 `灾后反应 -> 两难 -> 决断` 过桥，导致情绪不挂肉
- 连续三章都像并列任务章，没有形成 `起火 -> 升压 -> 兑现` 的小循环

### 高情节事件硬规则

若题材天然适配：

- 情报流
- 复仇流
- 夺权流
- 灾异流
- 边荒 / 流放 / 逃亡 / 关城 / 帮派 / 权谋 / 升级 / 仙侠争霸

那前 `10` 章细纲组还得额外满足：

- 至少 `2-3` 章主事件得来自 `高情节事件池`
- 至少 `1` 章写到：
  - 抢门
  - 截道
  - 焚档
  - 押运
  - 围杀
  - 暴动
  - 兽潮
  - 封断
  这类能把现场真改掉的事件
- 至少 `1` 次让第三方不是“围观起哄”，而是带着自己利益真的插手改局
- 至少 `1` 次让主角不是只抢到物证，而是抢到：
  - 名单该谁定
  - 最后过不过按谁的话定
  - 调人调货的权力
  - 能定人生死的那一下
  - 资源分配先按谁的话来

先别急着算过的情形：

- 前 `5-10` 章一直靠“翻小证据 + 对手把话收回去 + 群体炸锅”这一套反复推进
- 所谓大事件只是章尾传闻，下一章又缩回小场景解释
- 细纲写了高情节事件名，但答不出谁会死、哪条路会断、哪批人会乱、哪口权会换手
- 主角每次都只是更接近真相，没有借事件真正把新的入口、人手、名额或拍板机会抓到手里

### 事件升级硬规则

章节组细纲得单独回答：

- 这一组里，哪一章负责把事件量级抬高
- 抬高的维度是哪一个：
  - 死伤
  - 势力数量
  - 现场到底改了多大
  - 控制权层级
  - 主角代价
- 这一组里，哪一章专门负责把 `关系 / 谁前谁后 / 协作链` 拉裂
- 这一组里，哪一章专门负责让主角在 `快拿到 -> 被打断 -> 改决断` 里做一次难选
- 若上一章已经是 `L2`，本章准备如何避免只是“同型 L2 换地点”

若答不出，就说明这一章还在低水平换皮，正文别写。

### 单章事件拍序硬规则

每章细纲都得单独写 `本章事件拍序`。

最低要求：
- 别少于 `3` 拍，最好别少于 `4` 拍
- 推荐写成以下结构中的一种：
  - `目标 -> 受阻 -> 变招 -> 现场被改掉`
  - `目标 -> 受阻 -> 假顺 -> 被压回来 -> 现场被改掉`
  - `入场起火 -> 首次翻面 -> 中段被压回来 -> 代价 -> 章尾新驱动`

这里照着拦：
- 任一拍都得写“谁做了什么，造成什么变化”
- 别把 `情报出现 / 发现线索 / 到达现场 / 开始行动` 这种准备动作单独当成整拍主收益
- 若 `本章事件拍序` 只能写成 `想做什么 -> 做了 -> 成了/没成`，细纲就先别算过

### 情绪回摆硬规则

每章细纲都得单独写 `本章情绪回摆`，至少 `2` 次。

允许的回摆方向包括：
- `起势 -> 被打断 -> 再起`
- `压住 -> 破功 -> 强压`
- `想拿 -> 快拿到 -> 必须舍掉一样`
- `想信 / 想杀 / 想走 -> 不能这么做 -> 改决断`
- `局势看似顺 -> 代价落下 -> 反而更狠`

硬规则：
- 情绪回摆得绑现实事件，别只写心理活动
- 每次回摆至少要落到下面之一：
  - 动作停顿 / 变形
  - 临场把原话收回去
  - 错误选择或被迫舍取
  - 对手或盟友谁前谁后变了
  - 代价突然落地
- 若情绪只靠旁白解释“他很怒 / 很痛 / 很纠结”，就当没写进场里
- 若情绪回摆里没有一次真正导致 `换打法 / 舍掉一样 / 让谁前谁后变掉 / 改协作关系`，那多半是假的，正文先别写

### 三章小循环这块怎么卡

若本章处于连续章节组中，细纲得单独写 `本章三章小循环位置`。

一般就按这个小循环看：
- 第 `1` 章：`起火`
- 第 `2` 章：`升压`
- 第 `3` 章：`兑现`

三者定义：
- `起火`：抛出本轮要抢的具体目标，并让局面先失衡
- `升压`：把代价、阻力、两难、错判、关系拉扯抬高，不能直接平推
- `兑现`：至少改掉一处现实局面，不能只换来“知道更多”

这里照着拦：
- 连续三章别都写成并列单元任务
- 若第 `2` 章没有明显升压，只是重复执行前章打法，就说明主线开始跑偏
- 若第 `3` 章没有现实兑现，只留下新线索、新危险、新猜测，也说明主线开始跑偏

### 书名主打兑现到了哪一步

每章细纲都得单独写 `本章书名主打兑现到了哪一步`，免得“收益名称很大，实际走得很低”。

这块统一按下面分：
- `D1`：活下来
- `D2`：拿到干活资格 / 自保资格
- `D3`：拿到自保的机会
- `D4`：拿到能决定别人去留生死的那一下
- `D5`：拿到一段公共秩序 / 资源该怎么分最后按谁的话定

这里照着拦：
- 若书名主打属于 `经营 / 配额 / 分配 / 夺权 / 拍卖 / 执牌 / 定序 / 生火权 / 放票权 / 名额权`，细纲别只写抽象收益名，得答清走到了哪一步
- 首章若只到 `D1-D2`，别写成“书名主打已兑现”
- 若本章计划走得比作品当前阶段最低要求还低，细纲就先别算过

### 纲层单一版本比对

每章细纲都得单独写 `纲层单一版本比对结果`。

至少对这几份：
- `设定/题材定位.md`
- `大纲/开书总蓝图.md`
- `大纲/全书章节总表.md`
- `大纲/卷纲_第X卷.md`
- `大纲/章组控制_第X卷.md`

至少答这几句：
- 本章规模等级是否一致
- 本章主场景层级是否一致
- 本章是继续低层定序，还是应抬到更大局面
- 本章主收益到了哪一步是否一致

输出至少带上：
- `一致 / 不一致`
- 若不一致，冲突文件有哪些
- 本章采用哪一个当前有效版本

只要不是 `一致`，正文就别写。

### 新增信息允许清单

每章细纲都得单独写 `本章新增信息允许清单`，免得正文临时乱补。

允许清单只收下面两类：
- 本章确实需要首次落地的新信息
- 已在卷纲 / 上章章尾 / 世界规则中预热，本章正式具名或正式实锤的信息

每条允许清单至少答清：
- 新增项名称
- 类型：人物 / 势力 / 地点 / 物证 / 规则 / 时间钉 / 接头地点或接头用的东西 / 关系变化 / 设定能力
- 来源：为什么这章能合理出现
- 它是给本章哪一层现实收益服务的
- 落地方式：对话 / 动作 / 实物 / 场面 / 章尾哪一拍
- 是否需要同步追踪 / 角色卡 / 伏笔 / 情报记录

这里照着拦：
- 允许清单为空，不代表正文可自由新增；代表本章禁止新增关键信息
- 若正文新增项不在允许清单内，就当越权
- 若允许清单超过 `3-5` 条，就说明本章装载过量，得重审是否拆章

### 正文禁增项

每章细纲都要单独写 `本章正文禁增项`，至少答清：
- 哪些高风险信息这章绝不能临时发明
- 哪些关系 / 设定 / 规则只能保留为怀疑，不能直接坐实
- 哪些伏笔只能预热，不能抢跑兑现

一般默认别加这些：
- 新上线直接具名且直接定性为真凶 / 主使
- 没有预热的新关键能力
- 没有过桥的新场面地点
- 没有来源的新关键物证
- 为了章尾好看临时补出的新暗号、新簿册、新路线、新交接点

### 纲层自审

若担心正文会偷加信息，先看看是不是细纲自己把正文逼到乱补。至少逐条自问：

- 不新增任何关键信息，这章还能不能完整跑通
- 第一爆点、第二轮收益、章尾翻面，是否都已有现成撑场面的东西
- 主事件如果去掉“临时冒出来的一样东西 / 一个人 / 一个地点”，是否立刻塌掉
- 细纲有没有用抽象词占坑，正文只能靠临场补名词来填
- 本章冲突和章尾是否已经写出足够硬的现实变化，而不是靠“再补个新料”撑住

任一项答得发虚，就先回细纲层，别直接写正文。

### 关键场面卡

为尽量做到“正文落笔时不再临时补桥、补动作、补险点”，每章细纲都得至少给本章最重的 `1` 个场面补写 `本章关键场面卡`；若本章有多个高动作冲突点，一般写 `1-2` 个就够了，别贪多。

高优先级场面类型包括但不限于：

- 追逐 / 截堵 / 反绕 / 逃生
- 战斗 / 反杀 / 围杀 / 群战
- 争抢 / 验货 / 抢签 / 抢令 / 抢资源去向
- 谈判翻桌 / 当众逼口 / 公开打脸 / 规则改写
- 群体改道 / 营地改序 / 阵线切换 / 灾异压人

每张 `场面卡 / 动作拍序` 至少答清：

- 场面名称：这一场到底在打什么
- 进入原因：为什么这场会在本章爆
- 现场地点或依托物：桥、巷、矿道、擂台、仓口、厅堂、车队、山路、阵眼、街区等
- 先手方：谁先发现 / 谁先动 / 谁先压
- 截堵点或爆点：冲突真正卡在哪里
- 主角预判依据 / 误判风险：主角凭什么敢这么做，做错会死什么
- 动作拍序：至少 `4-8` 拍，按“谁做了什么 -> 造成什么变化”写
- 险点：本场最容易死人 / 掉队 / 失手 / 翻车的一拍
- 脱身或翻盘结果：这一场打完，现实局势具体改了什么
- 禁止正文临时补的关键件：例如新暗号、新接头点、新地道、新证物、新关系变化

动作拍序可以照这个格式写：

- `A` 发现主角改向，点 `B/C` 两人从侧坡截下
- 主角借 `X` 提前看到护坡 / 埋伏 / 坍塌 / 空档，临时改走 `Y` 路线
- 跟随者里 `D` 在 `Z` 点差点掉队 / 暴露，逼主角当场补一手
- 对手扑空 / 陷入障碍 / 被迫把原话收回去 / 被逼着后退半步，主角抢到入口 / 名分 / 时间差

下面这些情况先别算过：

- 只有“双方交手、主角取胜、对手吃亏”这种结果句
- 只有情绪判断，没有现场地点、阻力点、险点
- 只有起点和终点，中间关键动作靠正文临场发明
- 场面能成立，全靠正文新增一个未备案的人、地点、物证或规则

### 事件规模等级说明

为避免开篇和前中期长期缩在低规模推进里，每章细纲都得标记事件规模等级：

- `L1 小规模`：单点对话、单房间冲突、两三人试探、局部搜证
- `L2 中规模`：多人围观、多人对峙、公开验货、局部规则翻面、单处硬冲突
- `L3 大规模`：群体混乱、路线变更、多人势力撞局、环境事件、公共秩序改写
- `L4 超常规模`：世界规则异动、灾异爆发、关键设定压人、大场面翻盘

通用拦法：
- 前 `3` 章中，至少 `1` 章达到 `L3`
- 前 `10` 章中，至少 `2` 章达到 `L3`，至少 `1` 章达到 `L4` 或“同等强度的关键设定大翻盘”
- 不允许连续 `3` 章都停在 `L1`
- 若题材天然偏日常 / 言情 /校园，也得用该题材对应的“大事件”替代 `L3/L4`，例如：
  - 家庭伦理：公开撕裂、重大秘密曝光、社会关系重排
  - 都市职场：公开事故、群体围观、舆论爆点、资源大战
  - 校园恋爱：集体活动翻盘、公开身份/关系冲击、群体评价改写
  - 奇幻仙侠：灾异、战斗、异象、势力撞局
- 凡标记 `L3/L4`，细纲里得同时答出：
  - 至少一种群体规模描述
  - 至少一种被改写的公共秩序 / 路线 / 资源 / 营地 / 谁前谁后
  - 至少一种场面靠什么撑住的具体东西
- 少任一项，规模等级就往下掉一档；若因此跌破前 `3/10` 章规模硬规则，细纲就先别算过

### 母场景避重复

每章细纲都得单独写明 `本章主场面`，免得正文老缩在同一类低成本场景里打转。

主场面指：
- 本章最主要的冲突发生在哪
- 本章最主要的动作依托
- 本章最主要的局面翻动方式

这些母场景风险偏高：
- 囚车 / 押解路 / 临时看押点
- 尸体 / 验尸 / 收尸点
- 驿站 / 旧棚 / 暂驻黑店
- 公验 / 对质 / 翻供 / 逼口
- 小范围灭口 / 小股试探 / 小队夜袭

这里卡紧一点：
- 连续 `2` 章别再用同类主场面
- 连续 `3` 章里，最好有 `1` 章切到 `军 / 市 / 寨 / 矿 / 关 / 仓 / 城 / 灾异区 / 禁地` 之一
- 若上一章最主要还是靠 `尸 / 车 / 驿 / 验 / 规矩翻脸` 这类东西撑着，下一章就优先从 `多人势力撞局 / 群体秩序变化 / 路线变化 / 资源争夺 / 环境爆事` 里挑主事件
- 只换地名、不换冲突走法，不算通过避重复

先别算过的情形：
- 囚车换成马车，仍是同一押解母场景
- 驿站换成破棚，仍是同一暂驻母场景
- 验尸换成公验，仍是同一套口供 / 尸证母场景
- 把“更大势力来了”包在章尾，但本章主体还是同一组小场景

### 世界主动施压

碰上以下单独条件的复合题材，这章冲突最好先由世界或势力主动压过来，别老是主角一个人出去找线索：
- 高环境压力 / 路线推进
- 超凡 / 异象 / 关键能力
- 信息差 / 情报 / 预知
- 谁压谁 / 阵营争夺 / 规则控制

细纲得答清：
- 是哪一股 `势力 / 组织 / 群体 / 环境规矩 / 资源竞争方` 先动
- 这股力量改写了什么 `路线 / 营地 / 粮路 / 货路 / 谁前谁后 / 规矩 / 资源`
- 主角不是“开始查”，而是“被压进局里后，借自身优势切进去改结果”

先别急着算过的情形：
- 先出一条线索，再让主角顺线摸人
- 先查尸证，再查口供，再查旧账
- 只有敌人来灭口，没有更大的外部局面变化

### 首章世界总盘与世界底色落点

若首章碰上以下任一类：
- 仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地
- 科幻 / 赛博 / 末世 / 异界 / 异能 / 超级职业体系
- 制度、交通、资源、身份运行方式明显不同于读者默认现实

那首章细纲就得同时写出：
- `首章世界总盘交代`
- `首章世界底色落点`

`首章世界总盘交代` 至少得答清：
- 这到底是个什么世界，不是什么世界
- 这个世界的制度 / 日常 / 权力 / 生存方式为什么会这样运转
- 主角当前开局死局为什么正好由这套世界规则制造

`首章世界底色落点` 则负责把上述规则现场化，让读者在前 `1/3` 真正看见它如何改动作、改规矩、改生死。

这里也一样：
- 总盘交代别只剩设定表，至少得落到现场规矩、职业常识、群体反应、资源使用、制度执行里的 `2` 项
- 别只写“局部边地特殊 / 局部行业特殊 / 局部副本特殊”，却不交代整个世界为什么容得下这套特殊
- 别让读者先把作品当成普通古代 / 普通都市 / 普通校园，到了中后段才突然发现世界根本不是那个世界

先别算过的情形：
- 首章只有奇观，没有世界总定义
- 只有名词，没有制度和日常常识
- 只有边地 / 副本 / 门派 / 校园一角的特殊，却没有整世界前后说得通的那套东西
- 主角死局看起来像普通冲突，后面才知道其实依赖另一套世界规则

### 非首章世界底色续显

若题材碰上 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，那第 `2-5` 章别又把世界写回纯世俗冲突。

这里照着卡：
- 至少每 `2` 章得再露 `1` 次世界底色
- 续显得落在 `环境反应 / 行业规矩 / 群体避讳 / 器物用途 / 旧阵旧印 / 灾异余波` 之一
- 别拿“主要设定在脑子里说了一句”顶替世界续显

可以朝这个方向写：
- 不同行当、不同地点的人，一般都会照着某种避煞、避潮、镇炁、压铃、封印规矩行事
- 旧阵、骨灯、镇物、异潮余波继续影响不同场所的秩序
- 不同行当的人对同一灾异 / 禁地 / 阵痕有共同反应

先别算过的情形：
- 第1章有玄异底色，第2章开始完全像现实黑幕文
- 所有玄异内容都缩回主角独享视角

### 主收益走法轮换

每章细纲都得写清 `本章主收益走法`，再和最近 `2-3` 章对一下，免得场景变了，爽法却没变。

主收益走法指的是：
- 本章最主要的胜利是怎么赢到的
- 读者最记得住的收益链条是什么

这些走法重复风险高：
- 情报指关键物
- 残魂压一拍逼失口
- 主角抢下物证 / 账 / 令
- 章尾再抬更高层敌意

轮换方向至少挑一项：
- 抢人而不是抢物
- 改规矩而不是拿证
- 逼人重新选边，而不是只逼出一句失口
- 公开处刑 / 当众逼人重新选边，而不是继续抢账
- 借环境灾异翻盘，而不是继续抢令

### 跨场景过桥

若本章相对上一章发生以下任一切换：
- 主场面地点切换
- 主势力 / 主对手切换
- 白天 / 夜里 / 次日等时间段切换
- 主事件依托切换

那细纲就得单列 `本章跨场景过桥`，至少答清：
- 主角为什么得转去这个新场
- 主角靠什么信息 / 余波 / 人物动作 / 现场异常锁定这个新场
- 主角怎么到场，或为什么能提前卡位
- 主角到场后第一眼先看什么，为什么会正好撞上本章第一爆点

可以朝这个方向写：
- 上章逼出敌方势力下场，本章细纲写明主角看见异常信号、判断当夜必出事，因此提前卡到关键入口
- 上章拿到名册或账本，本章细纲写明路线、票号、接头地点或运输余波把主角直接引到新据点

先别算过的情形：
- 只写“章尾有钩子，下一章自然接”
- 只写“主角赶到时”而不写怎么锁到现场
- 只顾新场更炸，不顾主角行动链
- 让敌方主动犯错自毁，而不是主角总是抢到关键物

先别算过的情形：
- 连续 `2-3` 章只是把“账 / 令 / 牌 / 册 / 钥”换不同名词
- 主角每次都靠同样的“压一拍 -> 抢一物 -> 抬更高层”完成本章

### 主要设定参与

若作品存在以下任一“主要设定”：
- 金手指
- 系统
- 情报
- 残魂
- 特殊血脉
- 强身份反差
- 独特世界规则权限

那细纲就得明确写出：
- 本章该设定怎么直接参与翻盘
- 是不是只在解释、提醒、铺垫

先别急着算过的情形：
- 设定只负责说明背景
- 设定只负责告诉主角一条信息
- 设定只在章尾提示下一章
- 读者知道设定存在，但看不到它怎么直接改写现实
- 设定参与方式只写“帮助判断 / 帮助识别 / 提醒风险”，但没有任何可见现实结果

残魂 / 金手指出手类型至少从下列中挑明一类：
- `压迫类`：逼得对手失神、失口、露怯、露底
- `识破类`：拆穿伪装、禁印、假令、假货、假阵
- `夺势类`：提前一步抢人、抢先后、抢令、抢路
- `超常动作类`：短时完成普通肉身做不到的动作
- `异象共鸣类`：借环境 / 灾异 / 旧物 / 旧阵放大压迫感

轮换这块照着卡：
- 连续 `2` 章别只复用 `预警 / 判断增强` 这种偏虚的出手
- 前 `10` 章里，上述类型最好真落地 `3` 类
- 若上一章已用 `识破类` 完成翻盘，下一章优先换成 `压迫类 / 夺势类 / 超常动作类`

外部可感知后果这块照着卡：
- 残魂 / 金手指出手后，至少要带出 `1` 个外部层面的可见反应
- 可见反应可以落在：
  - 对手失神、失口、误判
  - 旁人看见不合常理动作后，立刻换边、让路，或者把先后次序改掉
  - 器物、骨灯、阵纹、封印、灾异出现异常反馈
  - 环境秩序因此被改写一拍

先别算过的情形：
- 只有主角自己觉得更准
- 只有内心独白知道残魂出手
- 外部世界对残魂出手没有任何反应差

### 抽象占位拦截

下面这些说法别当成细纲关键位置的最终写法：

- 某个人
- 某件东西
- 某个势力
- 新的交接点
- 一份物证
- 更高层的人
- 外部发生混乱
- 某个还活着的证人

主要卡这些地方：
- `本章主要往前推什么`
- `本章书名主打卖点兑现`
- `这章最该先抓的那件事`
- `情节点序列`
- `本章最主要战果`

允许条件：
- 只可先拿来临时占位
- 而且得在同段里补出具名替代项

若关键位置到真正开写时还留着上述抽象占位，细纲就先别算过

## 四、情节点要求

每个情节点至少交代：

- 谁在动
- 动作落在哪
- 造成什么可见变化
- 局势往哪边偏了一寸

别只写：
- 发现线索
- 双方对质
- 主角翻盘
- 拿到物证
- 敌人震惊
- 留下悬念

---

## 五、情节点密度

- 前 `1/3` 尽快入局
- 中段得有阻力变化
- 后 `1/3` 得再拿一项现实收益

若连续 `3` 个情节点都只是“知道更多”，细纲就别先算过。

---

## 六、说法交叉复查

正文开写前，细纲和写前闸门得逐项对上：

- 章节标题
- 主要对手
- 主要物证 / 收益
- 本章主要往前推什么
- 本章最主要战果
- 本章第一爆点
- 本章高潮后续收益
- 本章外显代价类型
- 本章硬场面
- 本章章尾钩子
- 本章目标字数

若同一章冒出双版本说法，就直接按 `F5` 记。

再顺手核对：

- 具名明敌是否已经在前 `10` 章某处落位
- 具名摇摆角色是否已经在前 `10` 章某处落位
- 具名资源 / 权力节点是否已经在前 `10` 章某处落位

若三个记忆点少任一项，前 `10` 章细纲组就先别判通过。
