## 小纲四步法

细纲与正文比例控制在 **1:2.5 ~ 1:3**。

一般就按这四步走：

1. **分段判断** — 把大纲按剧情节点分段
2. **标注目的和效果** — 每段标注，不展开情节
3. **标注详写/略写** — 明确哪些段展开、哪些段带过
4. **快速定位** — 让后面一眼就知道这段到底要写成什么结果

别忘了：细纲只关注目的和效果，不展开情节。

## 高潮逆推法与AB粗纲

### 先看这一条

从高潮反推前面需要铺垫的人物和情节，再用AB交替法填充。

### AB粗纲法

| 标记 | 是什么意思 | 怎么写 |
|------|------|------|
| A | 压情绪/铺垫/伏笔 | 铺设困难、对手强势、悬念埋线 |
| B | 抬情绪/擦边/小收获 | 小反转、小进步、读者爽一下 |

写的时候就按这个走：确定高潮 → 反推所需铺垫 → ABABAB 排列 → 写作时只盯当前 AB 段。

这种走法更适合：节奏快、高潮节点很明确的剧情。

## 高潮构建公式

### 五步公式

按顺序执行：**蓄能 → 假胜 → 崩解 → 交叉死磕 → 悬置收尾**

1. **蓄能**：牺牲+焦灼打底，让观众从"看客"变"参与者"
2. **假胜**：先给希望再击碎（情绪落差 = 反转冲击力）
3. **崩解**：所有伏笔一次引爆 + 推入单人绝境（帮手全失）
4. **交叉死磕**：对抗线+绝境线来回切换（每切一次紧张感+1）
5. **悬置收尾**：余劲不散，胜负不立刻揭晓

关键就在假胜。得先让读者松一口气，觉得“这把稳了”，再当场打碎；没这一下，高潮就少一层情绪落差。

## 噱头分类与开篇流程

### 噱头大概三种

| 噱头是哪种 | 长什么样 | 写的时候记住什么 |
|----------|------|----------|
| 事件噱头 | 集中在开篇，约5章 | 一上来就进入事件，不铺垫穿越/金手指 |
| 金手指噱头 | 分布全文，前期多后期少 | 先写主角和困境，再引出金手指 |
| 人设噱头 | 人设直接影响剧情构建 | 只有当人设本身能持续制造戏剧性时使用 |

记住：事件写法和金手指写法别混着用。

### 两种常用开篇流程

**事件开篇**：事件切入（5章）→ 嫁接主线 → 拆分目标 → 阶段性爽点循环

**主线开篇**：描写主角现状 → 营造代入感 → 描写社会环境 → 设立主角目标 → 把目标拆成几道关 → 绑定金手指 → 获得第一次提升 → 情绪拉扯2-3次 → 完成 → 引出下一个目标

### 噱头吸量策略

| 策略 | 做法 |
|------|------|
| 噱头延伸型 | 后续都围着开头噱头来长，继续找同类噱头往下搭 |
| 噱头引流+常规型 | 开头噱头只负责吸量，后续走常规题材内容 |

开头噱头主要就是把点击和追读先拉起来。目的到了，就得马上嫁接主线。

开书前评估：噱头能不能延伸出后续更多字数？不能延伸就用"噱头引流+常规"策略。

## 主线的正确定义

**主线不等于升级**。主线是一件事，升级是主角达成目标的行动。

| 概念 | 定义 | 示例 |
|------|------|------|
| 目标（主线） | 主角要完成的一件事 | 斗破苍穹：上云岚宗复仇 |
| 行动 | 主角为达成目标做的事 | 努力修炼、不断提升实力 |

检查主线是否符合以下特征：
- 主线是一件事，不是一个元素
- 主线完成后，要么先铺垫再开第二条主线，要么就完结
- 记忆点选错会导致后续所有剧情偏差

## 卡文对策与剧情循环设计

### 这套怎么转

题材 + 金手指 + 主角身份 = 循环模式。这三样最好拧在一起。

### 6种经典循环模式

| 模式 | 一直怎么转 | 靠什么一直转 |
|------|---------|---------|
| 案件串循环 | 案件→解谜→部分真相→更大谜团→新案件 | 信息差+推理 |
| 扮猪吃虎循环 | 默默发育→挑衅→碾压→震惊→继续发育 | 读者-角色信息差 |
| 资源积累循环 | 资源→技能→实力→新地图→新资源 | 螺旋上升 |
| 戏剧性反转循环 | 亏钱→反转赚更多→拿更多钱去亏→又赚 | 不依赖数值膨胀 |
| 组织枢纽循环 | 各自冒险→信息汇聚→衍生新剧情 | 信息交换+多线 |
| 公路片循环 | 走一段路→遇一个人→又走→又遇 | 人物塑造力 |

### 新手村四股人得先摆齐

新手村最好先把四股人摆齐，这样资源流转才接得起来：

1. **学校/武馆** — 学技能、提升实力
2. **商贩/药行** — 卖出收获、获取资源
3. **山贼/敌人** — 展现学习成果的靶子
4. **官府/管理机构** — 更高的上升通道

### 地位和环境要一起抬

地位往上抬时，环境危险度也得一起往上抬。两边不同步，读者就会觉得无聊。

### 换地图三策略

1. **新旧地图联动**（新势力是旧势力的上级）
2. **带人走**（把重要人物带到新地图）
3. **提前铺垫吸引力**（让读者主动盼着去）

### 特殊题材怎么拐

- **天才流**：大幅增加等级数量，防止数值几十章就崩
- **无敌流**：一直靠信息差转（来一个秒一个，每次都让人重新认一遍）

## 日常文大纲怎么搭

### 就按这几步写

一般就按这几步走：

1. 从阅读中发现有趣的设定/关系/背景
2. 围绕灵感确定事业线+感情线
3. 选择节奏最快、情绪最足的节点切入
4. 对每一段拆开看信息差、人际关系、情绪
5. 按时间顺序排列事件
6. 写作前勾勒章纲

### 大纲推演法

以"实现房东租客关系"为例：

1. 目标：实现男女主房东租客关系
2. 拆分：主角家中要有空房，距离高中近
3. 推演：有空房=家庭条件好 → 最好是贷款买，有压力
4. 逆转理由：父母上世投资失败 → 这世主角逆转 → 买学区房

### 大纲节点格式

```
事件名称：
信息差：1. 主角知道什么/配角不知道什么 2. 读者视角 vs 角色视角
人际关系与情绪：1. 主角情绪 2. 女主/配角反应 3. 负面情绪提供者
```

### 这里得抓住

- 大纲服务正文生成，不是枷锁
- 日常文不要有太多猛增好感的大事件，用小事件串联缓慢增长
- 事业线要和女主自身及家庭串联，达到日常和事业互相糅合

## 大剧情拉期待法

### 从上往下怎么推

按顺序执行：

1. **先说清这段要爽在哪** — 先把这段大剧情最主要的爽点钉住
2. **确定篇幅目标**
3. **分段把剧情排出来** — 围着爽点排若干小剧情，前一个给后一个垫路
4. **技巧杂糅填充**

### 案例演示：参加好歌曲演唱青花瓷

**主要爽点**：主角登台演唱青花瓷，引起震撼

一段一段这样排：
1. 收邀请+公园哼唱《送别》→ 震惊评委（前置暗示）
2. 彩排现场挑衅 → 制造压力
3. 节目前夜给邓紫棋写《泡沫》→ 叠加实力展示
4. 现场PK → 设备故障（加压）→ 演唱青花瓷 → 震惊
5. 评委扣分 → 分数持平（反转压制）
6. 歌手演唱主角作品 → 曝光主角是创作者（二级震惊）
7. 设备故障说明 → 得分逆袭（反转翻倍）
8. 评委要求唱《送别》→ 持续拉期待

记住：先有大爽点，再一段一段去够它；局部看节奏很快，整体看其实只是在讲一件事。

## 连续性怎么盯，节奏怎么稳

### 热度状态

| 状态 | 定义 |
|------|------|
| hot | 当前最能顶着冲突往前跑的那条线 |
| warm | 近期活跃的元素 |
| cold | 超过安全线未触及，有被遗忘风险 |
| archived | 已完结/有意关闭的元素 |

### 什么才算真碰到了

以下算有效触碰：直接推进该线索、施加压力、改变关系状态、产生实际后果、交代合理的休眠原因。

不算有效触碰：纯粹提个名字、空头回调、随机提及。

### 回顾阈值

| 这一类东西 | 多久得碰一下 |
|----------|----------|
| 关键角色 | 3-5 章 |
| 主要支线 | 4-6 章 |
| 活跃伏笔 | 2 次错过机会 |
| 不稳定关系 | 2 次出场 |

### 每章必做自检

1. 当前 hot 的元素是什么？
2. 有没有 cold 了但该 warm 的？
3. 哪些可以合理保持休眠？
4. 哪条线索的回归能加深压力？

### 失败信号（出现就要修正）

- 读者问"那个谁去哪了？"
- 重要的线索到结尾才突然冒出来
- cold 的铺垫突然变成 hot 的回报（没有预热）

### 最重那条冲突怎么护住

1. 非大结局章节禁止提前解决最重那条冲突
2. 章末 200 字得带着悬念走
3. 局部赢一把时，也得顺手挂上新的代价或风险

### 事件后的冷却章节数

| 事件是哪种 | 过几章再拿它当主拍 |
|----------|-----------|
| conflict_thrill（大冲突/打斗） | 2 |
| bond_deepening（关系深化） | 1 |
| faction_building（建立势力） | 2 |
| world_painting（世界观展开） | 3 |
| tension_escalation（压力升级） | 2 |

记住：冷却期里，这一类别拿来当主拍；`conflict_thrill` 最多连着跑 2 章；每 5 章最好带上 `bond_deepening` 或 `world_painting`。

### 过渡章怎么拿捏

- 单元故事结束前先把下一个目标拉出来
- 过渡章最少得留住一条还在动的期待线

### 换地图期待感延续

| 延续方式 | 做法 |
|----------|------|
| 复仇线 | 未完成的目标跨地图持续 |
| 旧日关系线 | 老角色在新地图出现 |
| 信息差 | 某方以为某事，实际不是 |
| 提前铺垫 | 换地图前让新地图角色/传说与主角接触 |

### 金手指四步怎么往上抬（基础→发展→成熟→升华）

| 走到哪一步 | 这一段主要抓什么 |
|------|---------|
| 基础 | 先写清主要作用，让读者记住 |
| 发展 | 增加新的使用方式，但主要作用别变 |
| 成熟 | 与世界观深度结合，可与其他系统联动 |
| 升华 | 跳出原来的写法，局面一下抬高，前面得有足够伏笔撑住 |

记住：金手指重心可以慢慢转，但前面得先垫够；可以稍微淡一点，别一下扔没了；写出来的分量也得一段比一段重。

### 矛盾网怎么织

- 同一时刻保持2-3条矛盾线同时运行
- 矛盾线之间要有关联（因果、利益冲突、信息差）
- 每解决一个矛盾，最好顺手把另一个矛盾点燃或压深

| 这一层 | 管多大一块 | 大概是干什么的 |
|------|------|------|
| 章级 | 2-3章 | 小冲突，服务于当前单元 |
| 卷级 | 一卷 | 本卷最重的矛盾，卷末解决 |
| 书级 | 全书 | 终极矛盾，大结局解决 |

### “两长一短”期待安排

- 1个短期期待：当前单元的明确目标（只能有一个）
- 1-2个长期期待：远期目标预告/悬念/组织/人物

### 期待怎么一直往前拽

1. 在"基底期待"上添加细节，注入新的具体化需求缺口
2. 设置多条最能吊人的线交替运行（装逼线A + 解密线B + 感情线C）
3. 顺手埋一点世界层面的“秘密”和“阴谋”

### 升级前后得拉开哪些差

| 看哪一面 | 主要看什么 |
|------|------|
| 能力差异化 | 每个等级有质变性的新能力 |
| 待遇差异化 | 不同等级获得不同的社会待遇 |
| 人际关系网差异化 | 不同等级面对不同层次的人际关系 |

如果升级前后这三面拉不开差，读者就吃不到升级快感。

### 单元故事嵌套

- 在第一个单元故事高潮前，插入第二个故事的期待线
- 完成当前目标前，提前给出下一个目标的线索伏笔
- 完成当前目标后，迅速给出反转或变故，营造新期待

### 长线节奏怎么排

- 一卷 = 一个完整的中套娃，有自己的起承转合
- 每卷开头是代入期（5-10章），中间是发展期，结尾是高潮+收束
- 信息密度：高密度（情绪强烈、推进快）与低密度（铺垫积蓄）交替，不能一直高或一直低
- 每个低密度章节最少埋一个让读者想知道后续的点

### 期待感怎么同时挂着

三层期待同时运行：
- 短期期待（下一章会发生什么）
- 中期期待（这个剧情单元会怎么收）
- 长期期待（主角最终能不能达成目标）

## 剧情过渡与衔接

### 从一个剧情点到下一个

常见问题是：主角实力提升了，但下一步干什么不清楚。解决办法：每次提升后立刻拉出新的挑战或目标。

### 剧情怎么一层套一层

- 大套娃：整本书的主线
- 中套娃：每个卷、每一段的阶段性目标
- 小套娃：每个剧情单元的即时目标
- 三层套娃同时运行，读者始终有事可看

### 场景过渡

- 过渡场景该跳过就跳过，不拖泥带水
- 过渡不是填充，没有信息量就删掉

### 情绪衔接

- 上一场景的情绪要自然过渡到下一场景
- 不能前一个场景还在热血，下一个场景突然平淡

### 信息差衔接

- 前一个场景埋下的信息差在后一个场景回收
- 读者带着疑问进入下一场景，衔接自然有效

## 把剧情拆成几道关，往后拉开

### 这几道关到底是什么

这不是随便放个敌人挡路，而是把目标前头摆上几道非过不可的关：主角想拿到结果，就得一条条过。

### 具体怎么拆这几道关

| 关卡是哪种 | 例子 |
|----------|------|
| 资源型 | 系统升级需要1000灵石，主角身上没有 |
| 成就型 | 考顶级院校需要气血值达标+独自灭杀XX级怪物+比赛前五名 |
| 多条件型 | 把一个大目标拆成几个一段一段的小目标 |
| 动态关卡 | 主角资源超标时，就把后面的要求抬高 |
| 收集型 | 突破境界需要多件宝物 |

### 这里记住三句

- 这些关得围着本书最主要的看点来，和金手指、脑洞没关系的拦法基本没用
- 这些关要一批一批往外放，别一次全甩给读者
- 每过完一道关，就立刻把下一道关摆出来

### 循环从设定中诞生

- 最主要的看点要靠设定撑起来，设定一立住，后面的反应就会自己往外冒
- 设定多一条，循环元素就多一层，可写的内容就越多
- 写着写着把设定写丢了 = 把卖点写丢了

---

## 场景转换技巧

- 过渡场景没有信息量就直接跳过
- 前一个场景留下悬念，后一个场景回应悬念
- 前一个场景热血收尾，下一个场景开头要有余韵
- 切换视角时在悬念点切出，不要在平淡处切出
- 每条线在切换前都要留下一个"钩子"
- 多条线的读者关注度不同，主线占比要最大

### 换地图往深里看怎么搭

- 新地图 = 新环境 + 新角色 + 新规则 + 新目标 + 新冲突
- 换地图前：旧地图最重那条冲突，最少得先收掉一截
- 换地图后：前 5 章得赶紧把新的代入感和期待感立起来

别这么干：旧角色一刀切全扔掉、新设定一次全倒出来、新地图和旧地图一点关系都没有。最少留一条贯穿主线。

每换一次地图，循环都得往上抬：盘子更大、要过的关更难、对手更狠。

### 换地图时要细到什么程度

- 每张地图包含三种势力：训练场（武馆/学校）、地头蛇（豪绅/地方势力）、破坏者（山贼/麻匪）
- 日常路线扩展关系网：上学/归还装备/逛集市作为新故事弧线的桥梁

---

## 悬念与伏笔技巧

- 小剧情伏笔：先确定结局再倒推线索，从终点往回铺，前后要接得上
- 整本书伏笔：大纲那一步先定关键情节，书里再不断埋细节去照应，长线伏笔记下来，别写着写着忘了
- 谜语人vs伏笔：谜语人是故意不说明（容易惹烦）；伏笔是巧妙融入剧情、自然不刻意
- 烧脑剧情：视角只跟主角走，不写反派视角，避免破坏悬念张力

---

## 信息团概念

- 信息团 = 一段剧情最主要的信息块
- 每个信息团最好都能一句话说清“这一段到底在讲什么”
- 几组信息之间要一层层往前推
- 节奏快 = 信息团密集；节奏慢 = 信息团稀疏；水章节 = 信息团为零或与主线无关

## 看自己是不是写嗨了

### 先把这三个问题答清

1. 我这书写给谁看？（最少得说清目标读者的年龄段、职业、性别、常用平台、人生处境、普遍渴望）
2. 我的目标读者群体在看网文时希望看到什么内容？
3. 我的这本书有哪些内容是目标读者群体想看的？

判断就看这个：三问有一问答不好 = 自嗨。立刻停下修正。

### 发现写嗨了就这么拉回来

- 分析同类书的读者评论，提取高频正面关键词
- 对比同类书中高互动与低互动段落的差异特征
- 用同类书的目标读者画像反向校验本书的情节选择

## 质量检查清单

每次搞完一段剧情设计或一卷大纲后，一项一项对：

### 基础完整性
- [ ] 主线是否明确为"一件事"（不是元素，不是升级）
- [ ] 循环模式是否与题材+金手指+主角身份统一
- [ ] 三层套娃（大/中/小）是否同时运行

### 节奏与期待
- [ ] 三层期待（短/中/长）是否同时在线
- [ ] 章末200字是否包含悬念元素
- [ ] 信息密度是否有高低交替（不能一直高或一直低）
- [ ] 冷却期那几条有没有守住

### 连续性
- [ ] hot/warm/cold 状态是否正常（无该 warm 却 cold 的元素）
- [ ] 关键角色 3-5 章、主要支线 4-6 章内是否有有效触碰
- [ ] 换地图、换阶段后，最少有没有留住一条贯穿主线

### 高潮与过渡
- [ ] 高潮是否包含假胜（先给希望再击碎）
- [ ] 每次胜利是否伴随新的代价或风险
- [ ] 过渡场景是否删掉了无信息量的部分
- [ ] 场景切换是否在悬念点切出

### 避坑检查
- [ ] 这些关是不是都围着本书最主要的看点设计（不是白拦路）
- [ ] 金手指是不是按四步往上走（没乱跳、没半路扔掉）
- [ ] 自嗨三问是否全部能回答清楚
- [ ] 信息团是否每段都能一句话概括"在讲什么"
