# 单章怎么一路写完

> 这一页把单章生产里最先要看的东西收成一条主线。写单章、回炉单章、续写单章时，都先看这页；其他 references 负责补解释，不再当主入口。

---

## 一、单章固定顺序

所有章节都按下面这个顺序走，别乱插：

1. `细纲`
2. `写前闸门`
3. `正文`
4. `写后检查`
5. `追踪/角色卡更新`
6. `交叉复查`
7. `能不能收进主正文`

先照这几条走：
- 前一步没过，后面就先别往下走
- `交叉复查` 别并进写后检查一句话带过，单独写进文件
- `能不能收进主正文` 要等 `写后检查 + 追踪/角色卡更新 + 交叉复查` 全做完后再给
- 若当前任务处于 `H1-C 协同生产态`，可以在 `正文`、`写后检查`、`交叉复查` 三步之间插入“主线程即时回修”，但插入后要回到当前失败点，把后面步骤重新走完，别直接并过去
- 冷启动 / 首章组任务里，`追踪/角色卡更新` 还要顺手看：`设定/角色/` 关键角色卡有没有建好、已落地事实有没有写超前；关键角色卡少了，就先按 `F5`

---

## 二、下笔前这些事先答出来

下面这些项目只要有一项答不出来，就直接按 `F5`：

- 题材识别结果
- 本章要额外盯的单独检查
- 本章主要往前推什么
- 本章副线推进
- 本章最主要的收获
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 本章外部现实变化
- 本章书名主打卖点兑现项
- 本章后 `1/3` 新的现实翻面
- 本章章尾钩子类型
- 本章动作拍清单
- 本章人物冲突卡
- 本章章尾现场句预案
- 本章主写法
- 本章这套写法改掉了什么现实
- 本章对手这一下怎么压回来
- 本章这套写法里第三方怎么改局
- 本章这套写法里主角付了什么代价
- 本章这套写法最该落下的章尾动作
- 本章单独检查失败风险
- 预判失败码

再补几句：
- `书名主打卖点兑现项` 只能写本章主兑现项，别写“都有”
- `外部现实变化` 别写“知道更多 / 更危险 / 更复杂”这类空话
- `高潮后续收益` 与 `外部现实变化` 不能互相代替
- `动作拍清单` 最少写出 `4-6` 拍现场动作，别只剩“主角反制 / 对手失手 / 场面翻转”这种提纲话
- `动作拍清单` 每一拍除 `谁做了什么 -> 谁被迫怎么改` 外，还要再补实：
  - 谁被改掉了哪一处 `实际位置 / 先后次序 / 原本守着的位置 / 办事次序 / 先开口定事的资格`
  - 谁因此立刻吃了哪一种 `损失 / 丢脸 / 失手 / 失口 / 担责 / 延误`
  - 哪个人从这一拍起已经 `不能退回旧说法 / 不能继续观望 / 不能装没看见`
  - 若任一拍只能写成“场面更乱了 / 压力更大了 / 大家更急了”，就说明这拍还没写实
- `人物冲突卡` 最少写出主角 / 对手 / 摇摆位各自的 `嘴上说什么 + 实际护什么 + 本章会冒什么风险 + 一旦站出来先失去什么`
- `章尾现场句预案` 只能停在 `动作 / 物证 / 名单 / 命令 / 新翻面`，别又写成总结句
- `本章主写法` 最少写 `1` 个，最多写 `3` 个；若为规则流 / 经营流 / 连续翻盘文，一般同时读取 `references/template-motherboards.md`、`references/template-engine-combos.md`、`references/template-engine-exhaustion.md`
- `本章这套写法改掉了什么现实` 要写成“谁把什么从什么状态改成什么状态”，别只写这套写法的名字
- `本章对手这一下怎么压回来 / 第三方现场又改了什么 / 主角代价 / 章尾最该落下的动作` 这四项少一项，就说明这套写法只是摆在那儿，还没真写进去
- `对手第二手怎么继续压回来 / 第三方这时不表态会吃什么亏 / 本章最容易写得像在交代思路的句子 / 本章最容易过早把好处写得太大的一步 / 本章最容易被一笔带过的过程拍` 只留在写前准备里就行；若打算把这些术语原词写进正文，就先别开写

### 正文前再拦一遍

进正文前，再把下面这几项过一遍：

1. `动作拍清单`
   - 删掉解释句后，动作链还得能自己跑通，才算过关
   - 若主顺序还是“先讲规则，再让人物执行”，一般先记 `F2`
   - 若答不出任一拍到底改掉了谁的实际位置、谁立刻吃了损失、谁从此退不回旧说法，一般先记 `F2`
   - 若关键中段连续两拍以上只有 `翻册 / 查账 / 认条 / 验封 / 表态 / 落笔`，却没有同时写出 `谁被挤开 / 谁吃亏 / 已经退不回去`，一般就按“过程写窄，前面没站稳”看，先记 `F2`
2. `人物冲突卡`
   - 若角色只剩功能，不剩私心、回避、下意识护点，一般先记 `F2`
   - 若主角 / 对手 / 摇摆位的台词功能可以互换，一般先记 `F2`
- 若关键角色答不出“本章会冒什么风险、先弃哪一步、站出来立刻失去什么”，一般就按“人物张嘴像在替作者讲情况”看，先记 `F2`
3. `章尾现场句预案`
   - 要能独立成立为新的现场结果
   - 若自然写成“真正要命的是 / 更大的还在后头 / 现在才只是开始”，一般先记 `F2`
4. `这套路子那八句先答清`
   - 要能答清：
     - 这套路子改什么现实
     - 主角借它抢什么
     - 对手怎么当场压回来
     - 第三方怎么换说法 / 往前挤或往后退 / 改资源流向
     - 主角付什么代价
     - 这套路子还显了谁的人物
     - 它给下一章接出什么新东西
     - 本章最后停在哪个动作上最合适
   - 若只能答出“发生了什么”，一般先记 `F2`
4.3. `现实挤压补位卡`
   - 若当前任务是 `边界题材冷启动 / 补丁后回归验证`，进入正文前还要额外答清：
     - 谁被挤掉实际位置
     - 谁拿到第二个能插手的位置
     - 谁被迫往前挤或往后退 / 改顺序 / 改守的位置
     - 哪道名分 / 名册 / 名单开始当场起作用
   - 上述 `4` 项最少答出 `3` 项；少于 `3` 项直接按 `F2`
   - 若答案停留在“气氛更紧 / 局势更乱 / 竞争更激烈”，就算没答到，先别进正文
4.1. `这套路子是不是真的写进了这一章`
   - 写前要逐项看下面这些内容是不是已经写实了，而不是只写个名字：
     - `本章主写法`
     - `本章第一爆点`
     - `本章后续实利`
     - `本章群体反应落点`
     - 若还是冷启动首章组：`本章第二次外部反应落在哪`
     - `本章章尾前一拍抬升位`
   - 若上述项目缺项，或只有名字没有写清会落在哪一拍，一般先记 `F2`
4.2. `这套路子到底打出了什么`
   - 写前要能回答：
     - 这些路子最终改了哪一样现实
     - 第一爆点之后，谁被迫把原话缩回去 / 往前挤或往后退 / 改先后 / 改利益
     - 前后两次外部反应分别如何改变现场
     - 章尾前最后一拍到底抬高了哪一处压力
   - 若只能回答“本章用了哪些路子”，答不出“这套路子打出了什么效果”，一般先记 `F2`
4.4. `提纲术语有没有翻成人话`
   - 写前要逐项看这些术语是不是已经翻成故事里的说法了，别准备原词往正文里塞：
     - `对手第二手怎么继续压回来`
     - `第三方这会儿不表态会吃什么亏`
     - `直接把这套路子的步骤喊出来`
     - `一章里把这套路子的效果全写死`
     - `这套路子刚起手就立刻全生效`
     - `一读就悬空的句子`
     - `一看就落不到现场的那一拍`
   - 若任何一项还打算把术语原词直接塞进正文，一般至少记 `F2`，先别往正文里写
4.5. `有没有把这套路子直接说破`
   - 写前要逐项看正文会不会直接拿下面这些句式替这套路子下结论：
     - `这是第一下群体改线`
     - `这是第一下外部反应`
     - `第二次外部反应也来了`
     - `这就算真的起作用了`
     - `这才是……真正重要的地方`
     - `这不是帮腔`
     - `这不是只下场`
     - `问，就是要写`
   - 若删掉该句后，动作链、谁前谁后变了、器物结果、落字结果仍然成立，说明该句只是作者判题句，一般至少记 `F2`

5. `冷启动首章组先过那张清单`
   - 若当前任务是新书冷启动、skill 回归验证、或首章组测试，还要额外读取 `references/cold-start-opening-acceptance-checklist.md`
   - 没通过该清单时，别拿“普通章节已经能过”顶替冷启动能过
   - 若细纲答不出 `第二次外部反应落在哪` 或 `章尾前一拍会落在哪`，一般至少记 `F2`
5.1. `方法资料到底有没有真用上`
   - 若当前任务是新书冷启动、skill 回归验证、或首章组测试，写前要明写这轮实际调用了哪些方法论资料，别只写一句“背景知识已加载”
   - 至少从以下资料中按需选取并记录：
     - `references/genre-writing-formulas.md`
     - `references/style-commercial-theory.md`
     - `references/style-craft.md`
     - `references/outline-structure-theory.md`
     - `references/opening-design.md`
     - `references/hooks-chapter.md`
     - `references/hooks-suspense.md`
     - `references/hooks-paragraph.md`
     - `references/emotional-arc-design.md`
     - `references/emotional-methods.md`
     - `references/dialogue-mastery.md`
     - `references/character-design-methods.md`
   - 记录时至少答：
     - 本轮实际读取了哪几份资料
     - 每份资料用来解决哪一个当前风险
     - 准备把它落实到细纲 / 动作拍清单 / 对话 / 章尾 / 人物出场中的哪一处
   - 若只能写“已参考方法论 / 已按资料执行”，但答不出具体要落在哪，算没真用上
   - 冷启动测试里若连续出现“资料已读，但正文看不出实际用处”，一般记为 `资料读了，但文里没用上`
6. `纯个人升级流再补这一层`
   - 若当前题材是 `纯个人升级 / 个人突围 / 单人登阶 / 单人抢机缘`，还要额外读取 `references/personal-upgrade-template-variant.md`
   - 若是该题材，`第二次外部反应` 别继续顺手写成“大群体一起争前后”，要明确写成：
     - `竞争者往前挤或往后退`
     - `资格官把原话收短`
     - `强者提前封线`
     - `主角多轮试探后的换手`
   - 若正文仍主要靠围观惊呼、主角判断、强弱说明撑住外部反应，一般至少记 `F2`

## 三、写完后这些问题都要答

写后检查别只写个结论，下面这些都得一条条答：

1. 本章第一爆点是否成立
2. 本章高潮后续收益是否成立
3. 本章是否拿到外部现实变化
4. 本章后 `1/3` 是否滑成信息场
5. 本章书名主打卖点是否真正主推动
6. 本章章尾最后停在了哪一种结果上
7. 本章解释句密度是否超阈值
8. 本章人物出场分量最低配额是否达标
9. 追踪同步是否只写已发生事实
10. 角色卡是否只写已落地说法
11. 本章主要明敌 / 盟友首次强出场是否落够 `2` 项稳定记忆点
12. 若为首章，本章是否把“书名另一半的第一波压制”真正打出来
13. 若还是冷启动首章组，本章是否已满足 `冷启动首章组那张清单`
14. 本章这套路子到底有没有真的写进正文
15. 本章这套路子到底有没有打出效果
16. 若还是冷启动首章组，本章方法资料到底有没有真正用上
17. 本章哪些方法资料真的在正文里起了作用
18. 若本章存在 `章尾前一拍`，最后一拍是否已把它兑现成新的现场结果，而不是只做气氛收束 / 预告收束
19. 若本章报过 `字数.error / 字数.warn`，是否已实答：
   - 谁被挤掉实际位置
   - 谁抢到第二个能插手的位置
   - 哪道名分 / 名册 / 名单开始起作用
20. `解释句前置自检卡` 是否通过
21. `末尾反删测试` 是否通过
22. 本章是否出现把写作说明里的词直接写进正文
23. 本章好处停在哪一步，是否误把 `名字先挂上去 / 现场先得按这边来` 写得太满，直接写成 `这一下好处已经当场起作用 / 好处已经真的落到手里`
24. 本章写前想拿到的是哪一步（来自哪份写前文件）
25. 本章实际只写到了哪一步
26. 支撑这一步的正文证据句是什么
27. 若写前想要的高于实际写到的，拦住本章继续往上走的硬缺口是什么
28. 如果这一章写到重要角色的关键台词，她/他先压进去的“自己人 / 自己物 / 自己名分”是什么
29. 如果这一章写到重要角色的关键台词，压进去后她/他自己当场失去了什么 `实际位置 / 先后机会 / 面子 / 名分`
30. 这一下失去是否立刻逼出别人把原话缩回去 / 往前挤或往后退 / 让出先后 / 不敢再硬顶

写后这里再补几句：
- 第 `22` 条只要答成“是”，本章一般就别判 `F0`。
- 第 `23` 条若答得发虚，`写后检查`、`章组汇总`、`角色状态`、`角色卡已落地事实`、`情报记录` 就一起降回最保守那一步，别各写各的。
- 第 `24-27` 条里只要有一条答不出，就别把写前想拿到的那一步硬补成写后事实；若还写得太满，直接按 `F5`。
- 第 `28-30` 条若只答出“先压自己人”，却答不出角色自己失去了什么，一般还是按“比普通办事人厚一点”看，别直接写成 `读者能顺手想出这人的样子`
- 若正文出现 `把这套路子说破的句子检查` 中任一句式，而且该句正压在关键收益位、外部反应落点或章尾最后一拍，别只当“文风问题”往下放。

只要任一项答得发虚，一般就先别判过。

## 四、解释句密度阈值

下面这些情况一般至少记 `F2`：

- 开头 `500` 字内解释句 `>= 2`
- 任意连续 `300` 字内解释句 `>= 2`
- 单章累计解释句 `>= 3`
- 章尾最后 `1-2` 句承担意义总结 / 局势翻译 / 作者直接替读者收尾的功能

解释句一冒出来，就继续往下找根因：
- `G1 信息抢跑`
- `G2 现场东西不够，撑不起这一章`
- `G3 提纲语言漏出`
- `G4 人物失声`

别只写一句“解释腔偏多”就当这事过去了。

### 这些解释句一出现就先复检

下面这些句式在章节生产里一般都太像作者在替人物解释，别轻易判过：

- `他只知道一件事`
- `他这才明白`
- `今夜若……他会死`
- `最后留下来的，是……`
- `情报一个字没错`
- `就是现在`
- `就这一瞬，够了`
- `但这份……撑不了多久`
- 任意“先下结论，再补动作”的短句

就记这句：
- 撞上这份清单别只删词，要回到动作 / 身体反应 / 即时念头这一步重写
- 若这些高危句子集中在开头 `500` 字内，一般优先判 `G1/G3`
- 若关键收益位出现 `这就算真的起作用了 / 现在不是…… / 这就不是只…… / 这才是……` 一类句子，且删掉后现场仍成立，一般至少记 `F2`

### 解释句前置自检没改顺就停

- 首稿写完后，若 `解释句前置自检卡` 任一关键位答成：
  - 这句删掉后现场仍成立
  - 但还没改回动作 / 谁前谁后变了 / 物件 / 他人被动承认
  就直接按 `F1/F2` 停下
- 这类句子先别放进写后检查，也别等 `story-review` 再抓
- 若它出现在：
  - 第一爆点后
  - 后续实利
  - 章尾前一拍
  - 章尾最后一拍
  一般优先按 `F2`

## 五、人物出场分量这块卡严一点

下面这些情况一般至少记 `F2`：

- 主角 / 主要反派 / 盟友 / 重要角色首次强出场少于 `2` 项记忆点
- 连续登场章节未复现 `1` 项稳定记忆点
- 删除角色名后，主角组与重要配角台词仍明显混同

再补几句：
- 重要明敌 / 重要盟友首次强出场若只剩“名字重要 / 身份重要 / 下章会出手”，一般至少记 `F2`
- 若角色卡已写外形 / 动作 / 说话 / 气场记忆点，正文却没回落至少 `2` 项，一般至少记 `F2`

---

## 六、高潮得分两段来看

所有章节都得拆成两段来看：

- `第一爆点`
- `高潮后续收益`

如果只有前者成立，而后者只剩：
- 对方变脸
- 黑幕更深
- 事情不简单
- 下一章再处理

一般至少记 `F2`。

### 首章书名另一半也得打出来

若为第 `1` 章，且书名主打除“活下来”外还承诺了：
- 复仇
- 夺权
- 打脸
- 上位
- 反咬局势

那首章除第一爆点外，还得额外答一下：
- 第一批该正面反咬回去的人是否已被钉出来
- 第一波压制是否已落成“敌方不能再按原规矩处理主角”

若两项都答不稳，即使“活下来了”，一般至少记 `F2`。

### 冷启动厚度不够就先停

- 若当前任务是 `冷启动 / 回头试写 / 首章组测试`，正文首稿低于 `scene_lint` 下限，直接按 `F2`
- 先别往下跑写后检查
- 别把它只记成“厚度不足预警”
- 别用“节奏快 / 可读性强 / 该写的已经写进正文”就硬说没问题
- 若首稿低于 `2900`，但写后检查还想往 `F0` 或“能收进主正文”上靠，直接按 `F5`

### 章尾最后两句先删着试试

- 写后检查前先做 `末尾反删测试`：
  - 删掉最后一句后是否更强
  - 删掉最后两句后，本章是否仍完整成立
- 只要任一答案是“是”，直接按 `F2`
- 若最后一句主要功能是：
  - 压话
  - 止语
  - 预告
  - 总结
  - 翻译前一拍意义
  直接按 `F2`

## 七、已经发生的和后面打算写的分开记

### 1. 后面打算写的

未来内容就往这些地方写：
- `大纲/`
- `卷纲/`
- `细纲/`
- `追踪/总安排表_*.md`
- `追踪/执行计划_*.md`

### 2. 已经落地的事实

已落地内容就往这些地方写：
- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报记录.md`
- `追踪/伏笔.md`
- `设定/角色/*.md` 中的“当前已落地事实”区块

这块照着卡：
- 追踪同步时，永远从正文抽事实，别从细纲复制
- 角色卡别提前写还没发生的伤情、关系变化、身份变化
- 若已落地事实里抄进了后续预案内容，就直接按 `F5`

### 已落地事实先往保守里写

已落地事实文件别把“线索 / 强指向 / 高度疑似 / 死前失口”直接写死成“已确认 / 已主使 / 已执行 / 已坐实”。

一般可以这么写：
- `获得指向 X 的线索`
- `高度疑似`
- `死前吐词 / 临死失口`
- `与 X 有关联迹象`
- `怀疑 X 方向仍有埋杀链`

一般别这么写：
- `确认 X 仍有埋杀局`
- `X 已派人灭口`
- `X 是灭口执行者`
- `第一批物证外露`（若正文仍只是线索）
- `活人供词`（若正文并无活人作证）

若已落地事实用了上面这些禁写说法，而正文又没把同等级事实坐实，就直接按 `F5`。

### 回写前先降一档看看

正文写完后、更新 `追踪/` 与 `设定/角色/` 前，先逐条过一下：
- 这条内容在正文里是 `线索 / 怀疑 / 指向 / 死前失口`，还是 `已确认 / 已坐实`
- 若删掉角色主观判断，这条内容还剩什么客观事实
- 是否正在把“作者知道后面会这样发展”提前写进已落地事实

只要任一项答不稳，就先别回写已落地事实。

### 已落地事实这块再补几句

- 只有“情报/线索/强指向”不等于“现实战果”
- “赵三迟疑”“敌人更警惕”“幕后更深”不算高潮后续收益
- 若首章只完成“活下来 + 敌人来了 + 线索拿到了”，但没有让敌方改动作 / 往前往后挪 / 把原话收回去，一般至少记 `F2`
- 若本章后 `1/3` 主要在抬线索、抬黑幕、抬危险而没有新的现实翻面，一般至少记 `F2`
- 若“知道更多”被写进 `外部现实变化` 或 `高潮后续收益`，视为说法错误，至少 `F2`

## 八、交叉复查至少查这四组

`交叉复查_第XXX章.md` 至少看这几组：

1. 正文 vs 细纲
2. 正文 vs 写前闸门
3. 正文 vs 追踪文件
4. 正文 vs 角色卡

每组就写这几项：
- `对上了`
- `还得改`
- 冲突项
- 冲突归属文件

若任一组还是 `还得改`，最终失败码就先别往 `F0` 上写。

### 哪些情况下别急着写“对上了”

`交叉复查_第XXX章.md` 别在以下情况下急着写“对上了”：

- 细纲仍保留旧证据来源
- 写前闸门仍保留旧战果说法
- 追踪文件存在把线索直接写成结论
- 角色卡“当前已落地事实”仍使用正文未坐实的判断

只要上面任一项出现：
- 各分项可写 `还得改`
- `最终交叉复查结果` 只能写 `还得改`

### 交叉复查再补几句

- 正文 vs 追踪文件：只看已落地事实，不看“预计会发生”
- 正文 vs 角色卡：只看正文已坐实的状态，不看角色卡预设
- 若追踪文件把“怀疑/指向/线索”写成“已确认/已执行/已主使”，就算正文方向没错，这里也得记 `要改`

这里别这么干：
- 先写 `对上了`，再等 review 打回来
- 把“尚可理解”当成“已对齐”

## 九、首章额外高压要求

第 `1` 章在通用协议上再多压几下：

- 前 `300` 字内要亮出主卖点死局
- 只活命不够，还要显出书名另一半的第一波压制
- 后 `1/3` 别主要靠线索收纳和意义总结撑住
- 开头 `500` 字内这类容易悬空的解释句别超过 `1`
- 章尾前至少要出现一处“敌方被迫改动作 / 往前往后挪 / 把原话收回去”的外部现实变化

若首章只做到：
- 主角活下来了
- 敌人来了
- 线索拿到了

但没有更硬的外部局面改写，一般仍至少记 `F2`。

### 首章判定补强

- 书名另一半第一波压制，要表现成“主角的情报/行动，逼得敌方当章改动作、往前往后挪、把原话收回去、改处置”
- 仅仅“发现危险”“看见敌人”“知道更多”不算第一波压制
- 若开头 300 字内亮出主卖点死局，但到章尾还是没有外部局面变化，那首章这关就没过去

答不上来，一般至少记 `F2`。

## 十、回炉前先把上一轮栽在哪说清

若同一章已经因为 `F2 / F5` 回炉过一次，再次动笔前先补一份最小复盘：

- 上一轮主要栽在哪
- 是哪一步漏掉了：`细纲 / 写前闸门 / 正文 / 追踪同步 / 交叉复查`
- 这轮最先别再犯的老毛病是什么
- 哪些项目文件得和正文一起改

下面这些先别做：
- 不复盘就直接重写正文
- 只修正文，不修细纲 / 角色卡 / 追踪 / 交叉复查旧说法
