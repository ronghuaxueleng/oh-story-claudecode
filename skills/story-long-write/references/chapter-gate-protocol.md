# 章节生产协议

> 本文件把单章生产的最低硬规则压成统一协议。执行任意单章写作、回炉、续写时，优先读取本文件；其他 references 负责解释，不再承担主执行入口。

---

## 一、单章固定流水线

所有章节固定按以下顺序执行，不得跳步：

1. `细纲`
2. `写前闸门`
3. `正文`
4. `写后验收`
5. `追踪/角色卡同步`
6. `反校验`
7. `并入裁决`

规则：
- 任一步失败，后续步骤不算完成
- `反校验` 不得并入写后验收一句话带过，必须单列落盘
- `并入裁决` 只能在 `写后验收 + 追踪/角色卡同步 + 反校验` 全部完成后给出

---

## 二、写前闸门必填字段

以下字段缺任一项，直接按 `F5`：

- 题材识别结果
- 已挂载专项 checks
- 本章主驱动
- 本章副驱动
- 本章最值钱战果
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 本章外部现实变化
- 本章题面卖点兑现项
- 本章后 `1/3` 新的现实翻面
- 本章章尾钩子类型
- 本章专项失败风险
- 预判失败码

补充要求：
- `题面卖点兑现项` 只能写本章主兑现项，不得写“都有”
- `外部现实变化` 不得写“知道更多 / 更危险 / 更复杂”
- `高潮后续收益` 与 `外部现实变化` 不能互相代替

---

## 三、写后验收必答问题

写后验收不得只写结果，必须逐条回答：

1. 本章第一爆点是否成立
2. 本章高潮后续收益是否成立
3. 本章是否拿到外部现实变化
4. 本章后 `1/3` 是否滑成信息场
5. 本章题面卖点是否真正主驱动
6. 本章章尾判型是什么
7. 本章解释句密度是否超阈值
8. 本章人物显影最低配额是否过线
9. 追踪同步是否只写已发生事实
10. 角色卡是否只写已落地口径
11. 本章高价值明敌 / 高价值盟友首次强出场是否落够 `2` 项稳定锚点
12. 若为首章，本章是否把“题面另一半的第一口势”真正打出来

只要任一项答不稳，默认不过线。

---

## 四、解释句密度阈值

以下情况默认至少 `F2`：

- 开头 `500` 字内解释句 `>= 2`
- 任意连续 `300` 字内解释句 `>= 2`
- 单章累计解释句 `>= 3`
- 章尾最后 `1-2` 句承担意义总结 / 局势翻译 / 作者代判功能

解释句命中后，必须继续判根因：
- `G1 信息抢跑`
- `G2 现场承载不足`
- `G3 提纲语言漏出`
- `G4 人物失声`

不得只写“解释腔偏多”。

### 解释句黑名单（命中即优先复检）

以下句式在章节生产中默认视为高危解释句，不得轻易放行：

- `他只知道一件事`
- `他这才明白`
- `今夜若……他会死`
- `最后留下来的，是……`
- `情报一个字没错`
- `就是现在`
- `就这一瞬，够了`
- `但这份……撑不了多久`
- 任意“先下结论，再补动作”的短句

规则：
- 命中黑名单不得只删词，必须回到动作 / 身体反应 / 即时念头层重写
- 若黑名单句式集中在开头 `500` 字内，默认优先判 `G1/G3`

---

## 五、人物显影硬闸

以下情况默认至少 `F2`：

- 主角 / 核心反派 / 核心盟友 / 高价值角色首次强出场少于 `2` 项锚点
- 连续登场章节未复现 `1` 项稳定锚点
- 删除角色名后，主角组与核心配角台词仍明显混同

补充硬规则：
- 高价值明敌 / 高价值盟友首次强出场若只剩“名字重要 / 身份重要 / 下章会出手”，默认至少 `F2`
- 若角色卡已写外形 / 动作 / 说话 / 气场锚点，正文却未回落至少 `2` 项，默认至少 `F2`

---

## 六、高潮双段兑现硬闸

所有章节都必须拆成两段验收：

- `第一爆点`
- `高潮后续收益`

如果只有前者成立，而后者只剩：
- 对方变脸
- 黑幕更深
- 事情不简单
- 下一章再处理

默认至少 `F2`。

### 首章题面另一半硬闸

若为第 `1` 章，且题面除“活下来”外还承诺了：
- 复仇
- 夺权
- 打脸
- 上位
- 反咬局势

则首章除第一爆点外，还必须额外回答：
- 第一批该狠狠干回去的人是否已被钉出来
- 第一口势是否已落成“敌方不能再按原规矩处理主角”

若两项都答不稳，即使“活下来了”，默认至少 `F2`。

---

## 七、事实层与预案层隔离

### 1. 预案层

只允许写未来内容：
- `大纲/`
- `卷纲/`
- `细纲/`
- `追踪/总施工单_*.md`
- `追踪/执行计划_*.md`

### 2. 事实层

只允许写已落地内容：
- `追踪/时间线.md`
- `追踪/角色状态.md`
- `追踪/情报台账.md`
- `追踪/伏笔.md`
- `设定/角色/*.md` 中的“当前已落地事实”区块

硬规则：
- 追踪同步时，永远从正文抽事实，不得从细纲复制
- 角色卡不得提前写未发生伤情、关系变化、身份变化
- 若事实层抄入预案层内容，直接按 `F5`

### 事实层措辞降级规则

事实层文件不得把“线索 / 强指向 / 高度疑似 / 死前失口”写满成“已确认 / 已主使 / 已执行 / 已坐实”。

默认允许的表述：
- `获得指向 X 的线索`
- `高度疑似`
- `死前吐词 / 临死失口`
- `与 X 有关联迹象`
- `怀疑 X 方向仍有埋杀链`

默认禁止的表述：
- `确认 X 仍有埋杀局`
- `X 已派人灭口`
- `X 是灭口执行者`
- `第一批物证外露`（若正文仍只是线索）
- `活口口供`（若正文并无活口）

若事实层使用了禁止表述，而正文未完成同等级坐实，直接按 `F5`。

### 回写前降级检查

正文写完后、更新 `追踪/` 与 `设定/角色/` 前，必须先逐条回答：
- 这条内容在正文里是 `线索 / 怀疑 / 指向 / 死前失口`，还是 `已确认 / 已坐实`
- 若删掉角色主观判断，这条内容还剩什么客观事实
- 是否正在把“作者知道后面会这样发展”提前写进事实层

只要任一项答不稳，不得回写事实层。

### 事实层硬判补充

- 只有“情报/线索/强指向”不等于“现实战果”
- “赵三迟疑”“敌人更警惕”“幕后更深”不算高潮后续收益
- 若首章只完成“活下来 + 敌人来了 + 线索拿到了”，但没有让敌方改动作 / 改站位 / 改判断，默认至少 `F2`
- 若本章后 `1/3` 主要在抬线索、抬黑幕、抬危险而没有新的现实翻面，默认至少 `F2`
- 若“知道更多”被写进 `外部现实变化` 或 `高潮后续收益`，视为口径错误，至少 `F2`

---

## 八、反校验必查四组

`反校验_第XXX章.md` 必须至少检查：

1. 正文 vs 细纲
2. 正文 vs 写前闸门
3. 正文 vs 追踪主表
4. 正文 vs 角色卡

每组只能写：
- `通过`
- `未通过`
- 冲突字段
- 冲突归属文件

若任一组未通过，最终失败码不得为 `F0`。

### 反校验权限限制

`反校验_第XXX章.md` 不得在以下情况下自报“通过”：

- 细纲仍保留旧证据来源
- 写前闸门仍保留旧战果口径
- 追踪主表存在把线索写满成结论
- 角色卡“当前已落地事实”仍使用正文未坐实的判断

命中任一项时：
- 各分项可写 `待回修`
- `最终反校验结果` 只能写 `未通过`

### 反校验落点补充

- 正文 vs 追踪主表：只看已落地事实，不看“预计会发生”
- 正文 vs 角色卡：只看正文已坐实的状态，不看角色卡预设
- 若追踪文件把“怀疑/指向/线索”写成“已确认/已执行/已主使”，即使正文方向正确，也必须判 `未通过`

禁止：
- 先写 `通过`，再等 review 打回来
- 把“尚可理解”当成“已对齐”

---

## 九、首章额外高压标准

第 `1` 章在通用协议上再加：

- 前 `300` 字内必须亮出主卖点死局
- 只活命不够，必须显出题面另一半的第一口势
- 后 `1/3` 不得主要靠线索收纳和意义总结撑住
- 开头 `500` 字内黑名单解释句不得超过 `1`
- 章尾前必须至少出现一口“敌方被迫改动作 / 改站位 / 改判断”的外部现实变化

若首章只做到：
- 主角活下来了
- 敌人来了
- 线索拿到了

但没有更硬的外部盘面改写，默认仍至少 `F2`。

### 首章判定补强

- 题面另一半第一口势，必须表现为“主角的情报/行动，逼得敌方当章改动作、改站位、改判断、改处置”
- 仅仅“发现危险”“看见敌人”“知道更多”不算第一口势
- 若开头 300 字内亮出主卖点死局，但到章尾仍没有外部盘面变化，按首章硬闸不通过

答不上来，默认至少 `F2`。

---

## 十、回炉复盘硬规则

若同一章已经因为 `F2 / F5` 回炉过一次，再次动笔前必须先补一份最小复盘：

- 上一轮主失败码是什么
- 是哪一步漏掉了：`细纲 / 写前闸门 / 正文 / 追踪同步 / 反校验`
- 本轮优先防止复发的旧问题是什么
- 哪些项目文件必须和正文同步改

禁止：
- 不复盘直接重写正文
- 只修正文，不修细纲 / 角色卡 / 追踪 / 反校验旧口径
