# 单章模板

> 本文件集中存放写前闸门、写后验收、字数约束和写作提醒模板，避免主文件重复。

## 二、写后验收模板

```markdown
## 本章写后验收
- 场面数：
- 外显战果：
- 本章第一爆点：
- 本章高潮后续收益：
- 本章已落地收益类型：
- 下章预期收益（不得冒充本章收益）：
- 本章外部现实变化：
- 外显代价类型：
- 本章后 1/3 是否滑成信息场：否 / 是
- 本章题面卖点兑现项：
- 章尾判型：合格 / 预告型 / 总结型 / 信息型 / 平收型 / 污染型
- 高潮兑现结论：过线 / 未过线
- 解释句密度：合格 / 超阈值
- 解释句根因：G1 / G2 / G3 / G4 / 无
- 是否命中解释句黑名单：否 / 是
- draft_purity_guard 结果：通过 / 已自动截尾 / 未通过
- 一级禁用词扫描：通过 / 未通过
- 口头发力词扫描：通过 / 未通过
- scene_lint 结果：通过 / 未通过
- 全文残片扫描：通过 / 未通过
- 正文纯净度：通过 / 未通过
- AI味分级：轻度 / 中度 / 重度
- 人物显影最低配额：通过 / 未通过
- 角色卡同步：通过 / 未通过
- 追踪同步：通过 / 未通过
- 口径反校验结果：通过 / 未通过
- 最终失败码：F0 / F1 / F2 / F3 / F4 / F5
- 并入裁决：可并入主正文 / 不可并入主正文
```

写后验收填写口径补充：
- `本章高潮后续收益` 只能写 `命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化 / 垫背者落地`
- `本章已落地收益类型` 必须与上项一致，且必须是本章正文已经发生的事实
- `下章预期收益` 只能写预期，不得挪到 `本章高潮后续收益` 或 `本章已落地收益类型`
- `本章外部现实变化` 必须是敌我盘面可见变化，不得写 `知道更多 / 更危险 / 更复杂`
- `本章后 1/3 是否滑成信息场` 若主要在抬线索、抬黑幕、抬危险，填 `是`
- 若首章仍只做到“活下来 + 敌人来了 + 线索拿到了”，写后验收不得判 `可并入主正文`
- 若首章 `章尾判型 = 合格` 但 `高潮兑现结论 = 未过线`，写后验收不得判 `F0`

## 三、反校验模板

```markdown
## 本章反校验
- 正文 vs 细纲：通过 / 未通过
  - 冲突字段：
  - 相关文件：
  - 证据等级是否一致：通过 / 未通过
- 正文 vs 写前闸门：通过 / 未通过
  - 冲突字段：
  - 相关文件：
  - 证据等级是否一致：通过 / 未通过
- 正文 vs 追踪主表：通过 / 未通过
  - 冲突字段：
  - 相关文件：
  - 证据等级是否一致：通过 / 未通过
- 正文 vs 角色卡：通过 / 未通过
  - 冲突字段：
  - 相关文件：
  - 证据等级是否一致：通过 / 未通过
- 正文 vs 情报台账：通过 / 未通过
  - 冲突字段：
  - 相关文件：
  - 证据等级是否一致：通过 / 未通过
- 最终反校验结果：通过 / 未通过
```

反校验填写口径补充：
- 只要追踪主表或角色卡把“线索 / 指向 / 怀疑”写成“已确认 / 已执行 / 已主使”，正文即便方向正确，也不得判 `通过`
- `正文 vs 追踪主表`、`正文 vs 角色卡` 必须先对齐证据等级，再谈文字表述
- `正文 vs 情报台账` 必须核对 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果` 是否存在提前写满或缺失
- 反校验不是作者自评，不能用“后面会补”代替当前证据
- 只要写后验收把“下一章预期”写成“本章收益”，`反校验` 不得判 `通过`

## 四、写作技巧提醒

| 场景 | 技巧 |
|------|------|
| 开篇 500 字 | 必须有钩子，不能从天气/风景开始（除非反差极大） |
| 对话 | 推进剧情或揭示性格，不能只为了凑字数 |
| 打斗 | 不要流水账，写策略和反转，不写“你一拳我一脚” |
| 日常 | 要有人物互动和伏笔，不能只是“吃饭睡觉” |
| 爽点释放 | 铺垫要充分，释放要干脆 |
| 爽点密度 | 每 3000-5000 字至少一处情绪节点 |
| 章尾 | 每章结尾都要有翻页动力 |

## 五、字数硬约束

| 节奏 | 最低字数 | 说明 |
|------|----------|------|
| 高速推进 | ≥ 2000 | 每章一个明确事件 |
| 正常节奏 | ≥ 3000 | 主线 + 少量副线 |
| 舒缓铺垫 | ≥ 3000 | 人物互动 + 伏笔 |
| 高潮爆发 | ≥ 2000 | 集中释放，不拖沓 |

默认最低字数：`3000`。

## 六、追踪文件归档

每完成 `50` 章或一个卷结束时：
- `追踪/上下文.md` 保留最近 `5` 章详记
- 更早内容压缩到 `追踪/归档/第XXX-YYY章.md`
- 活跃伏笔、时间线、角色状态仍保留在当前文件
