# 单章反复会用到的这几张表

> 这一页集中放单章写作会反复用到的几张表：写前检查、写后检查、字数下限和写作提醒，省得主文件来回重复。

---

```markdown

- 这一段主要靠什么爽：
- 本章标题是哪一路：
- 本章标题主卖点：
- 本章标题禁区：
- 这章还得额外盯哪一项：
- 本章采用的章组走法：
- 本章现实卡法借的是哪一套：
- 本章动作-后果链：
- 本章关键场景目标-阻碍-变化：
- 本章主要往前推什么：
- 本章副线推进：
- 本章最猛收获：
- 本章硬场面：
- 本章第一爆点：
```

写前这张表再补几句：
- `本章对手第二手怎么压回来`、`第三方这时不表态会吃什么亏`、`本章最容易写得像在交代思路的句子`、`本章最容易提前写过头的收益到了哪一步`、`本章最容易被一笔带过的过程拍` 这些话只留在提纲里，别把原词直接塞回正文。
- 若预计正文会直接出现 `对面又压回来一次`、`第三方这会儿不表态会吃亏`、`直接把模板步骤喊出来`、`一章里把模板效果全写死`、`模板刚起手就立刻全生效` 这类术语，写前检查就直接记 `要改`，先翻成故事里的动作、关系变化或现实代价。
- 若预计正文会直接出现 `这是第一波群体改线 / 这是第一波外部反应 / 第二次外部反应也来了 / 这就算真的起作用了 / 这才是……真正重要的地方 / 这不是帮腔 / 这不是只下场 / 问，就是要写` 这类“把模板名直接说出来的句子 / 旁白先替读者把轻重说死的句子”，写前检查同样直接记 `要改`。
- `本章采用的章组走法` 这里只写走法，例如 `强开增长 / 追账夺先手 / 进城受阻 / 被围反勒 / 开场先把局面撑起来`，别写成原书具体事件顺序。
- `本章现实卡法借的是哪一套` 得写清本章冲突借的是哪套现实卡法，例如 `空额制度 / 核签权 / 卡口护送权 / 配额分发 / 名字先记到谁头上才算数`；答不上来，一般就先别进正文。
- `本章动作-后果链` 最少写成 `起手动作 -> 第一被动改线 -> 第二被动改线 -> 现实战果`；若只能写情绪和线索，就先别进正文。
- `本章关键场景目标-阻碍-变化` 最少要为本章最重的 `1-2` 个场面写成 `谁想拿什么 -> 谁用什么拦 -> 为什么得换手 -> 现场最后改了什么`；若这里只能写“冲突升级 / 对峙更强 / 张力更高”，一般就别进正文。
- `本章高风险办事台词` 要填本章最可能写假的 `1-3` 句，尤其是扛着 `接锅 / 抢先办事 / 立军令状 / 逼人把原话吞回去 / 当场宣布换人来办` 作用的台词。
- `本章高风险办事台词的人话改写结果` 要把这些句子的“要办成什么事”翻成现场人物真会说的话；如果这里只能写成事情总结、写不成人话，一般就别进正文。
- `本章拟触发的人工回炉模拟主问题` 只写一个主问题，例如 `对白像在复述思路 / 人物发冷 / 声线混同 / 过程写窄 / 章尾翻译`；别一次并列多个问题一起改。
- `本章角色抽线回写结果` 只在出现 `声线混同 / 人物发冷 / 重要角色像是在走流程` 时填写，最少写清这章那几个主要角色各自 `最怕什么 / 最护什么 / 不敢明说什么 / 急了以后句子怎么变`。
- `本章角色声线指纹卡` 最少为本章 `2-4` 个主要角色各写一条最稳定的声音特征，格式照着写：`这个人急了先骂什么 / 先护什么 / 句子会变短还是变绕 / 最不会说哪类话`。
- `本章遮名测试预判` 得先答一句：`如果遮住人物名，本章最关键的 2-3 句台词还能不能一眼分出是谁说的`；若答“不稳”，一般就别进正文。
- `本章提纲内容禁回灌检查` 只看一件事：提纲内容有没有全翻成故事表达；如果还是术语原话，就别进正文。

## 二、写完后用这张表再看一遍

```markdown
- 场面数：
- 外显战果：
- 本章第一爆点：
- 本章高潮后续收益：
- 本章已经真正拿到的好处类型：
- 本章目标收益到了哪一步（来自写前闸门/细纲）：
- 本章收益现在停在哪一步：开门 / 拿到资格 / 名字先挂上去 / 现场先得按这边来 / 这一下好处已经当场起作用 / 好处已经真的落到手里
- 本章收益停在这一步的证据句（正文原句或动作结果）：
- 本章为什么还上不到更高一步：
- 下章预期收益（别冒充本章收益）：
- 本章外部现实变化：
- 摆在台面上的代价是哪一路：

- 生硬撑劲词扫描：对得上 / 要改
- scene_lint 结果：对得上 / 要改
- scene_narrowness_lint 结果：对得上 / warn / 未跑 / 要改
- 全文残片扫描：对得上 / 要改
- 正文纯净度：对得上 / 要改
- 写作说明词回流扫描：对得上 / 要改
- 关键台词说出来顺不顺：对得上 / 要改
- 关键台词像不像在复述思路：对得上 / 要改
- 关键对白动作记忆点：对得上 / 要改
- 遮名测试：对得上 / 要改
- 清单分工式台词回流：对得上 / 要改
- 人工回炉模拟本轮是否有实质提升：有 / 无
- 若无实质提升，是否转主线程人工接管：是 / 否
- AI味分级：轻度 / 中度 / 重度

- 如果这一章写到重要角色那句关键台词，先压进去的“自己人 / 自己物 / 自己名分”：

- 如果这一章写到重要角色那句关键台词，这一下失去逼出的外部变化：
- 角色卡更新：对得上 / 要改
- 追踪同步：对得上 / 要改
- 说法前后对下来：对得上 / 要改

- 能不能收进主正文：能 / 不能
```

写后这张表再补几句：
- `本章高潮后续收益` 只能写 `命 / 人 / 名 / 权 / 物证 / 选边变化 / 规矩变化 / 垫背者落地`
- `本章已经真正拿到的好处类型` 得和上项一致，而且得是本章正文已经发生的事实
- `本章目标收益到了哪一步` 只能来自 `写前闸门 / 细纲 / 章组收益卡`，别拿作者脑补或下一章计划来顶。
- `本章收益到了哪一步` 就按 `开门 -> 拿到资格 -> 名字先挂上去 -> 现场先得按这边来 -> 这一下好处已经当场起作用 -> 好处已经真的落到手里` 一步一步判，别跳着写太满。
- `本章收益停在这一步的证据句` 要填能直接回指正文的动作、落字、让位、被挤开、交接、收走、把话收回去这些结果；别写“更接近了 / 基本成立 / 大家都懂了”这种空话。
- `本章为什么还上不到更高一步` 要答实：是缺 `被动承认的一击`、缺 `敌方现实里真被挤开`、缺 `稳定办成事的链条`，还是还停在 `名分/纸面/临时占住`。
- `现场先得按这边来` 只适用于“主角或其一方已经在现场把某个真人、真东西、真流程按住了，但还没真正拿到稳定好处”的状态；若只是纸面改说法、名分改写，仍停在 `名字先挂上去`。
- `下章预期收益` 只能写预期，别挪到 `本章高潮后续收益` 或 `本章已落地收益类型`
- `本章外部现实变化` 得是敌我局面看得见的变化，别写 `知道更多 / 更危险 / 更复杂`
- `本章后 1/3 是否滑成信息场` 若主要在抬线索、抬黑幕、抬危险，填 `是`
- 若 `scene_narrowness_lint 结果 = warn`，写后检查还得多答三项：
  - `谁原本占着的位置被改掉`
  - `谁因此吃了即时损失`
- `本章哪一拍删掉作者直接替读者收尾的句子 / 流程说明句后，现场还能成立`
- 若 `scene_narrowness_lint 结果 = warn` 且上面三项有一项答不清，就别判 `最终失败码 = F0`，也别写“能收进主正文”。
- 若 `scene_narrowness_lint 结果 = warn`，`高潮兑现结论` 别单独拿来当“这章没问题”的理由；还得同时证明正文里真有足够的 `现实变化 / 谁往前谁往后变了 / 即时损失 / 执行后果`。
- 若首章仍只做到“活下来 + 敌人来了 + 线索拿到了”，写后检查别写成 `能收进主正文`
- 若首章 `章尾结果类型 = 过关` 但 `高潮兑现结论 = 未通过`，写后检查别判 `F0`
- 若 `本章目标收益到了哪一步` 高于 `本章收益到了哪一步`，就保留更低那一步，别为了贴合目标硬把事实往上抬。
- 若答不出 `本章收益到了这一步的证据句`，一般别判 `F0`，最少回退为 `F2/F5` 复核。
- 如果这一章写到重要角色，却只写了“先压自己人 / 自己捞到什么”和代价，却答不出“她/他自己到底失去了什么”，一般还是别把人物算到 `读者能顺手想出这人的样子`
- `写作说明词回流扫描` 只要正文出现 `对面又压回来一次`、`第三方这会儿不表态会吃亏`、`直接把模板步骤喊出来`、`一章里把模板效果全写死`、`模板刚起手就立刻全生效`、`一看就很假` 这类句子或桥段，就记 `要改`，最少判 `F1`；别拿“只是测试项”写成没问题。
- `关键台词说出来顺不顺` 重点检查承担 `接锅 / 抢先办事 / 立军令状 / 逼人把原话吞回去 / 当场宣布换人来办` 的句子是否像真人出口；若更像把事情总结一遍、像在复述思路、像作者替人物收尾，一般记 `要改`。
- `关键台词像不像在复述思路` 重点检查人物台词是否在一句里同时承担“概括问题 + 列清缺口 + 宣告解法 + 总结立场”四种作用；若出现，一般最少 `F1`，别把它判成关键台词。
- `关键对白动作记忆点` 重点检查关键办事台词前后是否跟上了打断、抢物、拦手、拍案、失态、把原话吞回去、往前顶或往后退等现场记忆点；若只有嘴上讲这句要办什么，没有现场改动，一般记 `要改`。
- `遮名测试` 重点检查遮住人物名后，那几句主要台词是否还能一眼分出是谁在说；若三个人互换名字后基本都能成立，一般记 `要改`。
- `清单分工式台词回流` 重点检查是否出现 `谁替灯，谁扶绳，谁押路`、`谁站门，谁记名，谁押后` 这类分工清单式对白；若出现，一般最少 `F1`。
- `人工回炉模拟本轮是否有实质提升` 不看“更顺一点”“更像一点”这类虚话，最少得答出一项：后果更实、办事台词改成人话、角色声线更分化、场边人从背景板变成现实压力。
- 若连续两轮 `人工回炉模拟本轮是否有实质提升 = 无`，`若无实质提升，是否转主线程人工接管` 就一般填 `是`，别继续没完没了地自改。
- 若正文出现 `这是第一下…… / 第二次外部反应也来了 / 这就算真的起作用了 / 这才是……真正重要的地方 / 这不是帮腔 / 这不是只下场 / 问，就是要写` 一类句子，且删掉后现场仍成立，一般同样按 `要改` 处理；若正压在后续实利、外部反应或章尾最后一拍，直接按 `F2`。
- 若正文检测已经提示“中段偏 `翻册 / 认账 / 拍板` 推进”，写后检查就别还把这章写成“模板已经全写出来 / 推剧情 / 能收进主正文”，却不补过程写窄那三问；要是还把结果判太高，一般按 `F5`。

```markdown
  - 冲突项：
  - 相关文件：
  - 证据等级是否对得上：对得上 / 要改
- 正文 vs 追踪文件：对得上 / 要改
  - 冲突项：
  - 相关文件：
  - 证据等级是否对得上：对得上 / 要改
- 正文 vs 角色卡：对得上 / 要改
  - 冲突项：
  - 相关文件：
  - 证据等级是否对得上：对得上 / 要改
- 正文 vs 情报记录：对得上 / 要改
  - 冲突项：

  - 证据等级是否对得上：对得上 / 要改
```

这张交叉对照表再补几句：
- 只要追踪文件或角色卡把“线索 / 指向 / 怀疑”写成“已确认 / 已执行 / 已主使”，就算正文大方向没错，也别写成 `对得上`
- `正文 vs 追踪文件`、`正文 vs 角色卡` 先对齐证据等级，再谈文字表述
- `正文 vs 情报记录` 要核对 `刷新章 / 预计兑现章 / 实际兑现章 / 实际战果` 有没有提前写死或漏掉
- 交叉复查不是作者自评，别拿“后面会补”顶当前证据
- 只要写后检查把“下一章预期”写成“本章收益”，`交叉复查` 就别写成 `对得上`

## 四、写作提醒

| 场景 | 技巧 |
|------|------|
| 开篇 500 字 | 得有钩子，别从天气/风景开始（除非反差极大） |
| 对话 | 推进剧情或揭示性格，不能只为了凑字数 |
| 打斗 | 不要流水账，写策略和反转，不写“你一拳我一脚” |
| 日常 | 要有人物互动和伏笔，不能只是“吃饭睡觉” |
| 爽点释放 | 铺垫要充分，释放要干脆 |
| 爽点密度 | 每 3000-5000 字最少一处情绪节点 |
| 章尾 | 每章结尾都要有翻页动力 |

---

## 五、字数硬约束

| 节奏 | 最低字数 | 说明 |
|------|----------|------|
| 高速推进 | ≥ 2000 | 每章一个明确事件 |
| 正常节奏 | ≥ 3000 | 主线 + 少量副线 |
| 舒缓铺垫 | ≥ 3000 | 人物互动 + 伏笔 |
| 高潮爆发 | ≥ 2000 | 集中释放，不拖沓 |

一般最低字数：`3000`。

## 六、追踪文件归档

每完成 `50` 章或一个卷结束时：
- `追踪/上下文.md` 保留最近 `5` 章详记
- 更早内容压缩到 `追踪/归档/第XXX-YYY章.md`
- 活跃伏笔、时间线、角色状态仍保留在当前文件
