# 章组写完后这张汇总表怎么写

这张表用在：**连续 `2-4` 章同一章组写完，或者回炉快收住前**

这张表主要干三件事：
- 把分章 `写后检查_第XXX章.md` 收成一份章组汇总表
- 为 `追踪/角色状态.md`、`追踪/情报记录.md`、`追踪/时间线.md`、`设定/角色/*.md` 提供这一组统一可回填的事实摘记
- 防止“单章各自看着都能过，章组一合起来就出现事实漂移、角色卡漏补、收益全集中末章”

---

## 怎么用这张表

- 这张表不是可选总结，而是连续 `2-4` 章章组快收住前要补的一张汇总表。
- 只要当前工作范围属于同一翻面链、同一兑现链、同一围猎链、同一公开逼人把话吞回去的链条，就填。
- 本表要在分章写后检查都做完后再填，而且排在 `追踪/角色卡更新` 前面。
- 本表里的“统一事实摘记”只写正文已经落地的事实，别抄细纲预案、远期安排、作者意图。
- 若本表答不清“这组章到底改掉了什么现实局面”，就说明章组还没真正收住，别判 `F0`。

## 说法补充

- `现场先得按这边来` 用于章组中段已经真的压住现场对象、拦住对手动作、逼得第三方绕路，但好处还没稳稳到手的状态。它比 `名字先挂上去` 更进一步，但还没到 `这份好处已经当场起作用`。
- 若章组只能证明“新名分写上了、公开说法改了、名单挂出来了”，就别直接写成 `现场已经得按这边来`。
- 每章和章组汇总表都要同时写清 `目标到了哪一步 / 实际到了哪一步 / 证据句 / 为什么还没再往上走`，别只留“这一章现在算哪一步”一栏。
- 若分章目标高于实际，章组汇总表要保留更低那一步，并把差额原因写实；别为了贴合章组目标把单章事实抬高。
- 若章组答不出 `总收益到了这一步的证据句`，就别判 `F0`，最少回退为 `F2/F5` 再看一遍。
- `把写作说明里的词直接写进正文` 指正文直接出现说明层词汇，例如 `第二层反压`、`第三方迟疑成本`、`直接把模板步骤喊出来`、`一章里把模板效果全写死`、`模板刚起手就立刻全生效`、`容易露馅的句子`、`容易露馅的一拍` 等。只要出现，就说明首稿还没改成人话，最少按 `F1` 处理。
- `该写的到底有没有真写进去 / 写得太满` 指写后检查直接把分章写成 `全写 / 全进正文 / 全都对 / 全是`，却没有保留“哪一章只到资格、哪一章才把名字写实、哪一章才逼得现场真改”的分层。出现这种写法就按 `F5` 处理。
- `方法资料写得太满` 指 `读了多少` 全写成够了、每份资料都说成已经起作用，却没有保留 `只是看过 / 还没完全写进去 / 还得再看` 这种收着说的话。出现这种写法就按 `F5` 处理。
- 若任一分章出现 `scene_narrowness_lint.warn`，章组汇总表别把该章直接写成“全都没问题 / 都能收进主正文”；最少先写清该章的过程写窄三问，不然就按 `F5`。
