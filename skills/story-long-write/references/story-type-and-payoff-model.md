# 小说类型怎么判，爽点怎么定

> 这一页就是在卷纲、细纲、正文之前，把“这本到底是什么书、读者到底爽在哪、最容易写歪到哪儿”先说清，再落成一份后面随时能翻的 `设定/爽点怎么落.md`。

---

## 一、使用时机

以下这些场景，最好先翻这份：

1. 开书
2. 改题材定位
3. 前 10 章细纲搭建
4. 连续回炉 3 章及以上
5. 用户明确抱怨“不够爽 / 节奏慢 / 只查不爆 / 不像书名主打”

---

## 二、先把类型判清，别直接下笔

禁止只写：

- 历史
- 都市
- 玄幻
- 情报文
- 种田文

至少先判四层：

1. 外层题材
2. 内层结构
3. 最顶上的爽感
4. 这一段主要靠什么爽

通用示例：

- 外层题材：高压生存 / 都市竞争 / 校园关系 / 仙侠升级
- 内层结构：信息差破局 / 权力链挤压 / 资源争夺 / 身份反转
- 最顶上的爽感：提前半步反咬 / 当众把局势改掉 / 资源到手 / 身份重估
- 这一段主要靠什么爽：求生爽 + 验货爽 / 经营爽 + 掌局爽 / 升级爽 + 身份反转爽

---

## 三、常见爽法类型速查

| 爽法类型 | 最该先答的问题 | 典型收益 | 最容易写偏成 |
|------|------|---------|-------------|
| 求生爽 | 主角怎么不死，还反咬一口 | 命、规矩变化、暂时安全 | 悬疑查案 |
| 验货爽 | 别人怎么被迫承认主角有分量 | 名、被逼着选边、资格 | 主角自吹 |
| 经营爽 | 资源怎么变成筹码 | 钱、人、地盘、渠道 | 日常流水账 |
| 掌局爽 | 主角怎么让别人按他节奏走 | 权、名分、最后听谁的 | 高位空谈 |
| 复仇爽 | 主角怎么让仇人失态丢脸 | 打脸、名声、清算 | 情绪哭诉 |
| 升级爽 | 主角怎么快速变强，还让人真看见强在哪 | 实力、排名、战果 | 纯数值播报 |
| 身份反转爽 | 别人什么时候知道他不是废物 | 名、被逼着重新选边、资源重估 | 延迟过久不爆 |
| 信息差爽 | 主角比别人早知道半步并兑现 | 物证、先手、压制 | 只知道不兑现 |

一本书可以同时有 2-3 类，但必须先定清：

- 主爽法
- 次爽法
- 前 30 章主要靠什么爽

---

## 四、`设定/爽点怎么落.md` 至少要写到什么程度

最好把下面这些区块都写进去：

### 1. 这本书到底是哪一挂

- 外层题材：
- 内层结构：
- 最顶上的爽感：
- 主爽法：
- 次爽法：
- 前 30 章主要靠什么爽：

### 2. 这本书不是什么

这里要写清最容易滑错的方向，例如：

- 不是慢查真相文
- 不是单纯悬疑驿站文
- 不是普通经营种田文

### 3. 本书最该稳定出现的 3-5 类爽点

每类最好都写：

- 爽点名
- 读者在爽什么
- 这类爽点一般是怎么立起来的
  - 欲望
  - 压制
  - 反差
  - 事件
  - 兑现
  - 围观后果
- 触发条件
- 兑现方式
- 这类收益才算打中
- 这种写法不行
- 适配事件 / 场面靠什么撑住

再补几条要盯住的：
- 别只写“求生爽 / 验货爽 / 复仇爽”这类空标签
- 每类爽点都得写成“为什么爽 + 靠什么场面落地 + 最低怎么才算打中”
- 至少 `2` 类爽点得明确绑到“大事件 / 大环境 / 大场面”，例如：
  - 资源争夺
  - 公共事故
  - 群体冲突
  - 舆论爆点
  - 势力火并
  - 灾害 / 异动 / 封锁 / 赛事 / 集体活动翻车
- 至少 `1` 类爽点得明确绑到“高代价结果”的翻盘用途，例如：
  - 肉体代价 / 生存代价 / 资源代价
  - 关系断裂 / 名声炸裂 / 资格剥夺
  - 公开失控 / 集体误判 / 关系翻面
- 至少 `1` 类爽点得明确绑到“外部局面变化”，例如：
  - 群体看法被改掉
  - 规则改写
  - 谁往前谁往后变了
  - 资源流向变化
- 若同一爽点同时有“保守实现”和“激进实现”两种方案，别无脑默认选最稳妥版本；得结合题材、平台和眼下这一段到底该怎么爆，判断哪种兑现更对味。
- 优先比较的维度包括：
  - 代价是否更扎实
  - 场面是否更可感
  - 外部影响是否更明确
  - 群体或关系变化是否更成立
  - 是否更符合眼下题材与平台能吃下的力度

### 4. 前 30 章的收益怎么分

至少说明：

- 哪几章优先拿命
- 哪几章优先拿名
- 哪几章优先拿人
- 哪几章优先拿权 / 规矩变化

再加几条要卡住的：
- 别只按“命 / 名 / 人 / 权”做静态配比
- 还得额外写出前 `30` 章的爽点阶段排布，例如：
  - `001-003`：高压入局期
  - `004-010`：首次翻盘期
  - `011-020`：局面放大期
  - `021-030`：主线逼近期
- 每个阶段至少得写清：
  - 主导爽法
  - 必须出现的事件类型
  - 必须出现的现实收益

### 5. 假推进清单

以下都要按项目实际情况写具体版：

- 只知道更多
- 更接近真相
- 黑幕更深
- 别人脸色变了
- 下一章再说

### 6. 什么样的章尾才算停得住

只认：

- 动作未完
- 新的勾人点落地
- 对手失态
- 外部谁往前谁往后变了
- 规矩或关系改写
- 新压制当场压下

再加几条要卡住的：
- 别把“危险快来了 / 下一章更刺激 / 主角准备反制”写成有效章尾
- 章尾只是停得住，不等于整章就过了；这份文档里得单独写清：
  - `章尾停住了` 不能替代 `高潮兑现成立`

### 7. 连着往下写最容易歪到哪

这里要写清这本书最容易滑成什么，比如：

- 连续查证据
- 连续解释局势
- 情报只做钩子，不做兑现
- 主角聪明但始终没吃到肉

再补几条要盯住的：
- 至少再写清下面三类跑偏：
  - 事件模型跑偏：一直困在小场景、小搜证、小判断，缺大环境和大事件
  - 场面靠什么撑住跑偏：只有对话和分析，没有可感动作、可感代价、可感外部变化
  - 爽点兑现跑偏：一直查得深、逼得近，但真正翻盘动作太浅

### 8. 能反复拿来用的事件池

这份文档里最好额外补出“这本书适合反复拿来用的事件池”，至少 `8-12` 个，按类型分组：

- 环境事件
- 人为冲突事件
- 改局事件
- 夺势事件
- 失控事件
- 追杀 / 截堵事件
- 群体被带着动起来的事件
- 公开翻面事件

每个事件最好都写：
- 事件名
- 适合带出哪类爽点
- 最容易产出什么现实收益
- 最容易写废成什么
- 主角最容易先拿到哪一步控制资格
- 最容易死人 / 失控 / 断路的点在哪

再补几条要盯住的：
- 事件池得按“规模等级”分组或标注：
  - `L1 小规模`
  - `L2 中规模`
  - `L3 大规模`
  - `L4 超常规模`
- 前 `10` 章规划时，别只从 `L1-L2` 池中取材
- 若作品存在主要设定 / 金手指 / 身份差，还得额外写：
  - 该设定最适合在哪个规模等级下最有冲击力
  - 哪些事件能逼它不只是解释，而是直接出手
- 事件池别只由通用标签构成，至少 `6` 个事件名得具名到“这本书独有的叫法 / 资源 / 地点 / 卡法”这一级
- 事件池里的“环境事件”别只是天气或背景，得能主动改写路线、秩序、营地、资源，或者让一群人的前后位置变掉
- 若事件名可无损替换成“某次混乱 / 某份账册 / 某个还活着的证人 / 某个势力”，那就说明事件池还没具象化

### 8.1 还得补一组更猛的事件池

事件池不能只有“查证、逼人把原话吞回去、截物证、改一次规矩”这种低烈度事件，还得额外补一组 `高情节事件池`，至少 `6-10` 个。

优先类型：

- 劫道 / 劫狱 / 劫法场 / 劫船
- 山贼 / 流寇 / 私兵 / 帮派围杀
- 兽潮 / 灾异 / 火灾 / 塌关 / 封城 / 断路
- 押运 / 抢粮 / 烧仓 / 焚档 / 抢矿 / 夺炉
- 夜袭 / 追杀 / 反围猎 / 反截堵
- 暴动 / 哗变 / 抢门 / 踩踏 / 群体逃亡
- 公开审判 / 擂台翻面 / 城门点杀 / 当众夺印
- 政令突改 / 名册重排 / 临时征发 / 强制清洗

每个高情节事件最好顺手写清：

- 主角为何必须卷进去，而不是可以旁观
- 这一炸会改掉哪一项现实局面：
  - 路线
  - 秩序
  - 资源
  - 名单
  - 生死
  - 一群人谁往前谁往后变了
- 这一炸最容易逼出哪类人物本性：
  - 贪
  - 怕
  - 急
  - 护短
  - 甩锅
  - 反咬
- 主角能借机掌到哪一步新的控制权

下面这些情况一般都别算过：

- 所谓大事件只是“听说外面乱了 / 某处出事了”
- 事件名很大，正文实际还是两三个人在小场景里解释
- 事件只负责抬危险，不负责直接把现场改掉
- 事件不会死人、不会断路、不会逼得谁让位、不会改名单，只是背景风声

### 8.2 事件升级怎么排，先定住

这份文档还得把前中期的事件升级怎么排写清，别长期赖在低烈度：

- 前 `3` 章：
  - 至少 `1` 章碰上 `L3`
  - 别 `3` 章全是 `L1-L2` 小卡口、小搜证、小逼人把话吞回去
- 前 `10` 章：
  - 至少 `2-3` 次碰上 `L3`
  - 至少 `1` 次碰上“群体失控 / 路线改写 / 名单重排 / 强制封断”之一
- 前 `30` 章：
  - 至少得出现 `1` 次 `L4` 或题材等效的大翻面事件

升级不能只看规模标签，下面这些里至少也得抬起一项：

- 死伤量级
- 牵涉多少拨人
- 控制权层级
- 局面被掀翻到多大
- 主角代价

若连续 `2-3` 章都只是：

- 小场景查线
- 小层级互咬
- 小物证翻面
- 小规矩被人当场翻掉旧说法

而且没有把事件抬到更大环境 / 多方争夺 / 失控现场，一般就按 `事件升级失手` 看。

### 9. 每章最低得拿到什么

这份文档还得把“单章最低爽点要求”写清，至少包括：

- 每章至少一种现实收益
- 每章至少一次外部局面变化
- 前 `10` 章的大多数章节都该带硬场面
- 别只剩“知道更多 / 更接近 / 更危险”
- 前 `10` 章别连续 `3` 章都停在 `L1`
- 前 `3` 章至少 `1` 章达到 `L3`
- 前 `10` 章至少 `1` 次主要设定 / 金手指 / 身份差直接改写现实，而不是只给说明和提示
- 前 `10` 章至少 `3` 次主要设定直接造成可见现实结果；“帮助判断 / 提醒 / 识别”不算

### 10. 哪些空词别老拿来糊

这份文档还得把本书写作时禁止高频乱用的抽象占位表达写清，例如：

- 某个人
- 某件东西
- 某个势力
- 新的交接点
- 一份物证
- 更高层的人
- 外部发生混乱

再加几条要卡住的：
- 卷纲与细纲里的主要事件、主要资源、主要对手、主要物证，别拿上面这些抽象占位直接顶替
- 若出现抽象占位，就得在同段里补出具名替代项，不然就算还没具象化

### 11. 这本书自己的记忆点在哪

这份文档还得补写：

- 这本书最不可替代的 `3` 个世界记忆点
- 这本书前 `10` 章最该被记住的 `3` 个事件记忆点
- 这本书前 `10` 章最主要的 `3` 个角色 / 资源记忆点

再加几条要卡住的：
- 如果删掉书名和主角名后，这份文档还能几乎原样套到另一本书上，那就说明它还不够像这本书自己的东西

## 五、卷纲和细纲怎么用这份文档

1. 卷纲先把这几件事钉住：
   - 这一卷主导爽法是什么
   - 哪一路爽法别写过头
   - 哪一路收益是这一卷最值钱的硬货

2. 细纲先把这几件事答出来：
   - 本章在打这份文档里的哪一路爽点
   - 本章绝不能滑成哪一路写法
   - 本章收益落在哪
   - 本章如果只剩“知道更多”，是不是就该先别往下算过

3. 正文要能答出来：
   - 这一章的爽点有没有真落到现场里
   - 有没有偷偷滑成这份文档明写不能走的那一路

---

## 六、哪些信号一出来就说明写歪了

下面这些里只要冒出一项，就说明已经写偏：

1. 连续两章主要收益都是“拿到下一条线”
2. 连续两章主要动作都是盘问、验货、查账、对口供
3. 连续两章都没有命 / 人 / 名 / 权 / 物证 / 选边变化 / 规矩变化的现实收益
4. 情报连续出现，但连续不兑现
5. 读者只能感觉主角聪明，感觉不到主角变强、变有分量、变难动
6. 卷纲或细纲里大量出现“某个 / 某次 / 某件 / 更高层 / 新的交接点 / 一份物证”等抽象占位
7. 世界设定只像背景板，不像会自己制造大事的机器

---

## 七、最后别把话写空了

写这份文档时，别只讲方向，不落地：

- 禁止“本书要爽”
- 禁止“本书节奏要快”
- 禁止“加强冲突”

要写成真能拿去写的说法：

- 前 `10` 章的主导爽法必须写清，不许连续两章只查不爆
- 若碰上情报流 / 信息差流，该类章节的第二步收益必须是命 / 名 / 选边变化 / 规矩变化 / 资源变化之一
- 如果一章章尾只剩“更接近黑幕 / 更靠近真相 / 下一步再说”，那就直接算弱收
- 必须继续具体到“场面靠什么撑住”，例如：
  - 前 `10` 章至少混入 `1-2` 个能改写局面的事件
  - 高代价结果不能只当背景，必须翻局
  - 硬冲突 / 资源争夺 / 外部局势变化 / 关系失衡 至少轮流承担爽点落点
