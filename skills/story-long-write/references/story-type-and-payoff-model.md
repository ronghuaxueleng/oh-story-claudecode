# 小说类型判定与爽点模型说明书

> 本文件负责在卷纲、细纲、正文之前，先判定“这本到底是什么书、靠什么爽、最容易写偏成什么”，并生成可复用的 `设定/爽点模型说明书.md`。

---

## 一、使用时机

以下任一场景必须先读取本文件：

1. 开书
2. 改题材定位
3. 前 10 章细纲搭建
4. 连续回炉 3 章及以上
5. 用户明确抱怨“不够爽 / 节奏慢 / 只查不爆 / 不像题面”

---

## 二、先判类型，不要直接写

禁止只写：

- 历史
- 都市
- 玄幻
- 情报文
- 种田文

必须至少判四层：

1. 外层题材
2. 内层结构
3. 核心爽感
4. 当前阶段爽法

通用示例：

- 外层题材：高压生存 / 都市竞争 / 校园关系 / 仙侠升级
- 内层结构：信息差破局 / 权力链挤压 / 资源争夺 / 身份反转
- 核心爽感：提前半步反咬 / 当众改判 / 资源到手 / 身份重估
- 当前阶段爽法：求生爽 + 验货爽 / 经营爽 + 掌局爽 / 升级爽 + 身份反转爽

---

## 三、常见爽法类型速查

| 爽法类型 | 核心问题 | 典型收益 | 最容易写偏成 |
|------|------|---------|-------------|
| 求生爽 | 主角怎么不死，还反咬一口 | 命、规矩变化、暂时安全 | 悬疑查案 |
| 验货爽 | 别人怎么被迫承认主角值钱 | 名、站队、资格 | 主角自吹 |
| 经营爽 | 资源怎么变成筹码 | 钱、人、地盘、渠道 | 日常流水账 |
| 掌局爽 | 主角怎么让别人按他节奏走 | 权、名分、话语权 | 高位空谈 |
| 复仇爽 | 主角怎么让仇人失态丢脸 | 打脸、名声、清算 | 情绪哭诉 |
| 升级爽 | 主角怎么快速变强并验证强 | 实力、排名、战果 | 纯数值播报 |
| 身份反转爽 | 别人什么时候知道他不是废物 | 名、站队、资源重估 | 延迟过久不爆 |
| 信息差爽 | 主角比别人早知道半步并兑现 | 物证、先手、压制 | 只知道不兑现 |

一本书可以同时有 2-3 类，但必须钉死：

- 主爽法
- 次爽法
- 前 30 章主导爽法

---

## 四、生成 `设定/爽点模型说明书.md` 的最小结构

必须包含以下区块：

### 1. 本书类型判定

- 外层题材：
- 内层结构：
- 核心爽感：
- 主爽法：
- 次爽法：
- 前 30 章当前阶段爽法：

### 2. 这本书不是什么

必须写清最容易滑错的方向，例如：

- 不是慢查真相文
- 不是单纯悬疑驿站文
- 不是普通经营种田文

### 3. 本书最该稳定出现的 3-5 类爽点

每类要写：

- 爽点名
- 读者在爽什么
- 标准构成
  - 欲望
  - 压制
  - 反差
  - 事件
  - 兑现
  - 围观后果
- 触发条件
- 兑现方式
- 合格收益
- 不合格写法
- 适配事件 / 场面载体

补充硬规则：
- 不得只写“求生爽 / 验货爽 / 复仇爽”这类空标签
- 必须把每类爽点写成“为什么爽 + 靠什么场面落地 + 最低怎么才算打中”
- 至少 `2` 类爽点必须明确绑定“大事件 / 大环境 / 大场面”，例如：
  - 资源争夺
  - 公共事故
  - 群体冲突
  - 舆论爆点
  - 势力火并
  - 灾害 / 异动 / 封锁 / 赛事 / 集体活动翻车
- 至少 `1` 类爽点必须明确绑定“高代价结果”的翻盘用途，例如：
  - 肉体代价 / 生存代价 / 资源代价
  - 关系断裂 / 名声炸裂 / 资格剥夺
  - 公开失控 / 集体误判 / 关系翻面
- 至少 `1` 类爽点必须明确绑定“外部盘面变化”，例如：
  - 群体改判
  - 规则改写
  - 站位变化
  - 资源流向变化
- 若同一爽点同时存在“保守实现”和“激进实现”两种方案，不得无脑默认选最稳妥版本；必须结合题材、平台和当前阶段判断哪种兑现更对味。
- 优先比较的维度包括：
  - 代价是否更扎实
  - 场面是否更可感
  - 外部影响是否更明确
  - 群体或关系改判是否更成立
  - 是否更符合当前题材与平台的接受阈值

### 4. 前 30 章收益配比

至少说明：

- 哪几章优先拿命
- 哪几章优先拿名
- 哪几章优先拿人
- 哪几章优先拿权 / 规矩变化

补充硬规则：
- 不得只按“命 / 名 / 人 / 权”做静态配比
- 必须额外写出前 `30` 章的爽点阶段排布，例如：
  - `001-003`：高压入局期
  - `004-010`：首次翻盘期
  - `011-020`：盘面放大期
  - `021-030`：主线逼近期
- 每个阶段至少写清：
  - 主导爽法
  - 必须出现的事件类型
  - 必须出现的现实收益

### 5. 假推进清单

以下都要按项目实际情况写具体版：

- 只知道更多
- 更接近真相
- 黑幕更深
- 别人脸色变了
- 下一章再说

### 6. 章尾有效类型

只认：

- 动作未完
- 新抓手落地
- 对手失态
- 外部站位变化
- 规则或关系改写
- 新压制当场压下

补充硬规则：
- 不得把“危险快来了 / 下一章更刺激 / 主角准备反制”写成有效章尾
- 章尾只是合格停点，不等于整章过线；说明书里必须单独写清：
  - `章尾合格` 不能替代 `高潮兑现成立`

### 7. 连续顺写滑轨风险

必须写清这本书最容易滑成什么，比如：

- 连续查证据
- 连续解释局势
- 情报只做钩子，不做兑现
- 主角聪明但始终没吃到肉

补充硬规则：
- 至少额外写清以下三类滑轨：
  - 事件模型滑轨：一直困在小场景、小搜证、小判断，缺大环境和大事件
  - 场面载体滑轨：只有对话和分析，没有可感动作、可感代价、可感外部变化
  - 爽点兑现滑轨：一直查得深、逼得近，但真正翻盘动作太浅

### 8. 适配事件池

说明书必须额外产出“本书适合反复调用的事件池”，至少 `8-12` 个，按类型分组：

- 环境事件
- 人为冲突事件
- 改判事件
- 夺势事件
- 失控事件
- 追杀 / 截堵事件
- 群体改线事件
- 公开翻面事件

每个事件至少写：
- 事件名
- 适合承载哪类爽点
- 最容易产出什么现实收益
- 最容易写废成什么
- 主角最容易抢到哪一口控制权
- 最容易死人 / 失控 / 断路的点在哪

补充硬规则：
- 事件池必须按“规模等级”分组或标注：
  - `L1 小规模`
  - `L2 中规模`
  - `L3 大规模`
  - `L4 超常规模`
- 前 `10` 章规划时，不得只从 `L1-L2` 池中取材
- 若作品存在核心设定 / 金手指 / 身份差，还必须额外写：
  - 该设定最适合在哪个规模等级下最有冲击力
  - 哪些事件能逼它不只是解释，而是直接出手
- 事件池不得只由通用标签构成，至少 `6` 个事件名必须具名到“这本书独有的叫法 / 资源 / 地点 / 机制”级别
- 事件池中的“环境事件”不得只是天气或背景，必须能主动改写路线、秩序、营地、资源或群体站位
- 若事件名可无损替换成“某次混乱 / 某份账册 / 某个活口 / 某个势力”，视为事件池未具象化

### 8.1 高情节事件池硬补丁

事件池不能只有“查证、翻口、截物证、改一次规矩”这种低烈度事件，还必须额外补一组 `高情节事件池`，至少 `6-10` 个。

优先类型：

- 劫道 / 劫狱 / 劫法场 / 劫船
- 山贼 / 流寇 / 私兵 / 帮派围杀
- 兽潮 / 灾异 / 火灾 / 塌关 / 封城 / 断路
- 押运 / 抢粮 / 烧仓 / 焚档 / 抢矿 / 夺炉
- 夜袭 / 追杀 / 反围猎 / 反截堵
- 暴动 / 哗变 / 抢门 / 踩踏 / 群体逃亡
- 公开审判 / 擂台翻面 / 城门点杀 / 当众夺印
- 政令突改 / 名册重排 / 临时征发 / 强制清洗

每个高情节事件必须额外写清：

- 主角为何必须卷进去，而不是可以旁观
- 这一炸会改掉哪一项现实盘面：
  - 路线
  - 秩序
  - 资源
  - 名单
  - 生死
  - 群体站位
- 这一炸最容易逼出哪类人物本性：
  - 贪
  - 怕
  - 急
  - 护短
  - 甩锅
  - 反咬
- 主角能借机掌到哪一口新的控制权

默认不过线情形：

- 所谓大事件只是“听说外面乱了 / 某处出事了”
- 事件名很大，正文实际还是两三个人在小场景里解释
- 事件只负责抬危险，不负责改盘
- 事件不会死人、不会断路、不会改站位、不会改名单，只是背景风声

### 8.2 事件升级分布硬规则

说明书必须额外写清前中期的事件升级分布，不得长期停在低烈度：

- 前 `3` 章：
  - 至少 `1` 章命中 `L3`
  - 不得 `3` 章全是 `L1-L2` 小卡口、小搜证、小翻口
- 前 `10` 章：
  - 至少 `2-3` 次命中 `L3`
  - 至少 `1` 次命中“群体失控 / 路线改写 / 名单重排 / 强制封断”之一
- 前 `30` 章：
  - 必须至少出现 `1` 次 `L4` 或题材等效的大翻面事件

升级不是只看规模标签，还必须至少升级以下一项：

- 死伤量级
- 牵涉势力数量
- 控制权层级
- 改盘范围
- 主角代价

若连续 `2-3` 章都只是：

- 小场景查线
- 小层级互咬
- 小物证翻面
- 小规则改口

且没有把事件抬到更大环境 / 多方争夺 / 失控现场，默认按 `事件升级失手` 处理。

### 9. 每章最低交付

说明书必须额外写清“单章最低爽点交付线”，至少包括：

- 每章至少一种现实收益
- 每章至少一次外部盘面变化
- 前 `10` 章的大多数章节都应带硬场面
- 不能只剩“知道更多 / 更接近 / 更危险”
- 前 `10` 章不得连续 `3` 章都停在 `L1`
- 前 `3` 章至少 `1` 章达到 `L3`
- 前 `10` 章至少 `1` 次核心设定 / 金手指 / 身份差直接改写现实，而不是只给说明和提示
- 前 `10` 章至少 `3` 次核心设定直接造成可见现实结果；“帮助判断 / 提醒 / 识别”不计入

### 10. 反抽象占位清单

说明书必须额外写清本书生成时禁止高频使用的抽象占位表达，例如：

- 某个人
- 某件东西
- 某个势力
- 新的交接点
- 一份物证
- 更高层的人
- 外部发生混乱

补充硬规则：
- 卷纲与细纲中的核心事件、核心资源、核心对手、核心物证，不得使用上述抽象占位直接顶替
- 若出现抽象占位，必须在同段内补出具名替代项，否则视为未完成具象化

### 11. 本书不可替代性

说明书必须额外写：

- 这本书最不可替代的 `3` 个世界锚点
- 这本书前 `10` 章最该被记住的 `3` 个事件锚点
- 这本书前 `10` 章最值钱的 `3` 个角色 / 资源锚点

补充硬规则：
- 若删掉书名与主角名后，这份说明书还能几乎无损套到另一本文上，视为不可替代性不足

## 五、卷纲与细纲如何使用这份说明书

1. 卷纲要回答：
   - 这一卷主导爽法是什么
   - 哪一类爽法不能过量
   - 哪一类收益是本卷的硬通货

2. 细纲要回答：
   - 本章属于说明书里的哪一类爽点
   - 本章不属于哪一类写法
   - 本章收益落在哪
   - 本章如果只剩“知道更多”，是否直接不过线

3. 正文要回答：
   - 有没有把该章爽点写成现场
   - 有没有偷换成说明书明确禁止的写法

## 六、判偏信号

出现以下任一项，说明已经写偏：

1. 连续两章主要收益都是“拿到下一条线”
2. 连续两章主要动作都是盘问、验货、查账、对口供
3. 连续两章都没有命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化的现实收益
4. 情报连续出现，但连续不兑现
5. 读者只能感觉主角聪明，感觉不到主角变强、变值钱、变难动
6. 卷纲或细纲里大量出现“某个 / 某次 / 某件 / 更高层 / 新的交接点 / 一份物证”等抽象占位
7. 世界设定只像背景板，不像会自己制造大事的机器

---

## 七、输出口径

生成说明书时，禁止空话：

- 禁止“本书要爽”
- 禁止“本书节奏要快”
- 禁止“加强冲突”

必须写成可执行表达：

- 前 `10` 章的主导爽法必须写清，不许连续两章只查不爆
- 若命中情报流 / 信息差流，该类章节的第二口收益必须是命 / 名 / 站队 / 规矩变化 / 资源变化之一
- 若一章章尾只是“更接近黑幕 / 更靠近真相 / 下一步再说”，直接判弱收
- 必须继续具体到“场面载体”，例如：
  - 前 `10` 章至少混入 `1-2` 个能改写盘面的事件
  - 高代价结果不能只当背景，必须翻局
  - 硬冲突 / 资源争夺 / 外部改判 / 关系失衡 至少轮流承担爽点载体
