# 小说类型判定与爽点模型说明书

> 本文件负责在卷纲、细纲、正文之前，先判定“这本到底是什么书、靠什么爽、最容易写偏成什么”，并生成可复用的 `设定/爽点模型说明书.md`。

---

## 一、使用时机

以下任一场景必须先读取本文件：

1. 开书
2. 改题材定位
3. 前 10 章细纲搭建
4. 连续回炉 3 章及以上
5. 用户明确抱怨“不够爽 / 节奏慢 / 只查不爆 / 不像题面”

---

## 二、先判类型，不要直接写

禁止只写：

- 历史
- 都市
- 玄幻
- 情报文
- 种田文

必须至少判四层：

1. 外层题材
2. 内层结构
3. 核心爽感
4. 当前阶段爽法

示例：

- 外层题材：历史流放
- 内层结构：情报破局 + 权力链灭口
- 核心爽感：提前半步求生反咬
- 当前阶段爽法：求生爽 + 验货爽

---

## 三、常见爽法类型速查

| 爽法类型 | 核心问题 | 典型收益 | 最容易写偏成 |
|------|------|---------|-------------|
| 求生爽 | 主角怎么不死，还反咬一口 | 命、规矩变化、暂时安全 | 悬疑查案 |
| 验货爽 | 别人怎么被迫承认主角值钱 | 名、站队、资格 | 主角自吹 |
| 经营爽 | 资源怎么变成筹码 | 钱、人、地盘、渠道 | 日常流水账 |
| 掌局爽 | 主角怎么让别人按他节奏走 | 权、名分、话语权 | 高位空谈 |
| 复仇爽 | 主角怎么让仇人失态丢脸 | 打脸、名声、清算 | 情绪哭诉 |
| 升级爽 | 主角怎么快速变强并验证强 | 实力、排名、战果 | 纯数值播报 |
| 身份反转爽 | 别人什么时候知道他不是废物 | 名、站队、资源重估 | 延迟过久不爆 |
| 信息差爽 | 主角比别人早知道半步并兑现 | 物证、先手、压制 | 只知道不兑现 |

一本书可以同时有 2-3 类，但必须钉死：

- 主爽法
- 次爽法
- 前 30 章主导爽法

---

## 四、生成 `设定/爽点模型说明书.md` 的最小结构

必须包含以下区块：

### 1. 本书类型判定

- 外层题材：
- 内层结构：
- 核心爽感：
- 主爽法：
- 次爽法：
- 前 30 章当前阶段爽法：

### 2. 这本书不是什么

必须写清最容易滑错的方向，例如：

- 不是慢查真相文
- 不是单纯悬疑驿站文
- 不是普通经营种田文

### 3. 本书最该稳定出现的 3-5 类爽点

每类要写：

- 爽点名
- 读者在爽什么
- 标准构成
  - 欲望
  - 压制
  - 反差
  - 事件
  - 兑现
  - 围观后果
- 触发条件
- 兑现方式
- 合格收益
- 不合格写法
- 适配事件 / 场面载体

补充硬规则：
- 不得只写“求生爽 / 验货爽 / 复仇爽”这类空标签
- 必须把每类爽点写成“为什么爽 + 靠什么场面落地 + 最低怎么才算打中”
- 至少 `2` 类爽点必须明确绑定“大事件 / 大环境 / 大场面”，例如：
  - 劫囚
  - 山贼
  - 乱兵
  - 流民暴动
  - 黑驿卖囚
  - 山崩 / 暴雪 / 冻河 / 火并
- 至少 `1` 类爽点必须明确绑定“死人 / 活口 / 见血 / 翻案”的翻盘用途，而不是只当气氛
- 至少 `1` 类爽点必须明确绑定“围观改判 / 规矩翻面 / 官面改口”

### 4. 前 30 章收益配比

至少说明：

- 哪几章优先拿命
- 哪几章优先拿名
- 哪几章优先拿人
- 哪几章优先拿权 / 规矩变化

补充硬规则：
- 不得只按“命 / 名 / 人 / 权”做静态配比
- 必须额外写出前 `30` 章的爽点阶段排布，例如：
  - `001-003`：死局见血期
  - `004-010`：死人翻案期
  - `011-020`：乱局控盘期
  - `021-030`：仇线逼近期
- 每个阶段至少写清：
  - 主导爽法
  - 必须出现的事件类型
  - 必须出现的现实收益

### 5. 假推进清单

以下都要按项目实际情况写具体版：

- 只知道更多
- 更接近真相
- 黑幕更深
- 别人脸色变了
- 下一章再说

### 6. 章尾有效类型

只认：

- 动作未完
- 新物证落地
- 对手失态
- 站队改口
- 规矩翻面
- 更大压制当场压下

补充硬规则：
- 不得把“危险快来了 / 下一章更刺激 / 林渊准备反制”写成有效章尾
- 章尾只是合格停点，不等于整章过线；说明书里必须单独写清：
  - `章尾合格` 不能替代 `高潮兑现成立`

### 7. 连续顺写滑轨风险

必须写清这本书最容易滑成什么，比如：

- 连续查证据
- 连续解释局势
- 情报只做钩子，不做兑现
- 主角聪明但始终没吃到肉

补充硬规则：
- 至少额外写清以下三类滑轨：
  - 事件模型滑轨：一直困在小场景、小搜证、小判断，缺大环境和大事件
  - 场面载体滑轨：只有对话和分析，没有打斗、见血、死人、翻案、群体围观
  - 爽点兑现滑轨：一直查得深、逼得近，但狠狠干得浅

### 8. 适配事件池

说明书必须额外产出“本书适合反复调用的事件池”，至少 `8-12` 个，按类型分组：

- 大环境事件
- 人祸事件
- 翻案事件
- 夺势事件

每个事件至少写：
- 事件名
- 适合承载哪类爽点
- 最容易产出什么现实收益
- 最容易写废成什么

### 9. 每章最低交付

说明书必须额外写清“单章最低爽点交付线”，至少包括：

- 每章至少一种现实收益
- 每章至少一次外部盘面变化
- 前 `10` 章的大多数章节都应带硬场面
- 不能只剩“知道更多 / 更接近 / 更危险”

---

## 五、卷纲与细纲如何使用这份说明书

1. 卷纲要回答：
   - 这一卷主导爽法是什么
   - 哪一类爽法不能过量
   - 哪一类收益是本卷的硬通货

2. 细纲要回答：
   - 本章属于说明书里的哪一类爽点
   - 本章不属于哪一类写法
   - 本章收益落在哪
   - 本章如果只剩“知道更多”，是否直接不过线

3. 正文要回答：
   - 有没有把该章爽点写成现场
   - 有没有偷换成说明书明确禁止的写法

---

## 六、判偏信号

出现以下任一项，说明已经写偏：

1. 连续两章主要收益都是“拿到下一条线”
2. 连续两章主要动作都是盘问、验货、查账、对口供
3. 连续两章都没有命 / 人 / 名 / 权 / 物证 / 站队 / 规矩变化的现实收益
4. 情报连续出现，但连续不兑现
5. 读者只能感觉主角聪明，感觉不到主角变强、变值钱、变难动

---

## 七、输出口径

生成说明书时，禁止空话：

- 禁止“本书要爽”
- 禁止“本书节奏要快”
- 禁止“加强冲突”

必须写成可执行表达：

- 前 10 章以求生爽为主，不许连续两章只查不爆
- 情报流章节的第二口收益必须是命 / 名 / 站队 / 规矩变化之一
- 若一章章尾只是“更接近旧驿黑幕”，直接判弱收
- 必须继续具体到“场面载体”，例如：
  - 前 10 章至少混入 `1-2` 个大环境事件
  - 死人不能只当背景，必须翻局
  - 打斗 / 见血 / 活口争夺 / 围观改口 至少轮流承担爽点载体
