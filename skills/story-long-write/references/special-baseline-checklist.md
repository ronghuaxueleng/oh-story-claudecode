# 单项样本 Checklist

> 冷启动单项样本时，直接逐条勾。长版看 `special-baseline-writing-protocol.md`。

---

## 一、开样前

- [ ] 先定唯一第一主问题
- [ ] 写清“不该抢主结论”的噪音项
- [ ] 题材是否适合这个单项问题
- [ ] 若本轮要拿这份单项样本验证 `story-long-write` 全流程，而不只是正文单项回归，是否已建最小追踪文件：
  - [ ] `追踪/时间线.md`
  - [ ] `追踪/角色状态.md`
  - [ ] 若碰上情报流：`追踪/情报记录.md`
- [ ] 是否已建最小项目结构：
  - `设定/题材定位.md`
  - `设定/世界总控表.md`
  - `设定/角色/*.md`
  - `大纲/开书总蓝图.md`
  - `大纲/细纲_第001章_*.md`
  - `大纲/细纲_第002章_*.md`
  - `正文/第001章_*.md`
  - `正文/第002章_*.md`
  - `追踪/写后检查_第001章-第002章.md`
  - `追踪/时间线.md`
  - `追踪/角色状态.md`

再补几句：
- 若缺 `时间线.md / 角色状态.md`，这份样本仍可做“正文单项回归”，但不得宣称“story-long-write 全流程回归通过”。
- 若本轮目标是测 `双版本并存 / 收益写得太满了 / 写后检查 -> 追踪同步` 这类流程问题，缺最小追踪文件时应先补结构，再测这一项。

---

## 二、`非情报流误触` Checklist

- [ ] 现场具体东西是 `旧码 / 错单 / 复核口 / 广播补条 / 口令 / 挂签 / 规则播报`
- [ ] 主功能是把现场谁先谁后、谁担责、谁记名当场改掉，不是发正式情报
- [ ] 第001章先写“旧说法还在照常执行”
- [ ] 主角用信息凭据当场卡住旧次序
- [ ] 本组第二章继续围绕同一件东西改现实
- [ ] 没写独立情报块
- [ ] 没写 `情报刷新`
- [ ] 至少有 `1` 个第三方被动动作
- [ ] 至少有 `1` 次谁往前挤、谁往后退的变化
- [ ] 至少有 `1` 个器物/屏幕/灯牌反馈

复核目标：

- [ ] 即使脚本先报 `情报.error`，主线程也优先判 `非情报流误触候选`

---

## 三、`本组第二章偏短链` Checklist

- [ ] 第001章已证明模板主线进正文
- [ ] 第001章至少有：第一爆点、第二轮收益、外部反应、重要角色出场
- [ ] 本组第二章主体仍是四拍：
  - [ ] 主角顶一句
  - [ ] 对手回一句
  - [ ] 重要角色再往下压一句
  - [ ] 记录口直接落签
- [ ] 本组第二章没有扩成新支线
- [ ] 本组第二章至少有少量现场反应
- [ ] 本组第二章至少有 `1` 个器物/签架/牌架/记录口反馈
- [ ] 本组第二章没有短到只剩残稿感

复核目标：

- [ ] 即使脚本先报 `字数.error`，主线程也优先判 `本组第二章偏短链`

---

## 四、`本组第二章最后半拍回弹不足` Checklist

- [ ] 第001章已证明模板主线进正文
- [ ] 本组第二章中前段已先落成现实收益
- [ ] 本组第二章至少有 `1` 次外部反应
- [ ] 本组第二章至少有 `1` 个器物/牌架/记录口反馈
- [ ] 本组第二章最后 `1-3` 句故意收弱
- [ ] 问题集中在末半拍，而不是整章都薄
- [ ] 章尾前已经能看见 `记名 / 落签 / 把先后次序改掉 / 外面的人改了看法`

复核目标：

- [ ] 即使脚本先报 `章尾.error`，主线程也先判 `本组第二章最后半拍回弹不足`
- [ ] 已明确写出 `最主要收益已在章尾前落成`
- [ ] 未把它误判成 `本组第二章偏短链` 或 `F2`

---

## 五、`中段直接把意思说破的句子拦截` Checklist

- [ ] 重要角色已先压 `自己人 / 自己物 / 自己名分`
- [ ] 她/他自己当场真实丢掉了 `眼下的位置 / 先后次序 / 面子 / 名分` 里的至少一项
- [ ] 对手 / 记录的人 / 执行的人已因此把做法改了
- [ ] 中段是否出现：
  - [ ] `我若先把自己人……谁还认……`
  - [ ] `我若先护自己人……谁还认……`
  - [ ] `我若先把自己人塞进……谁还认……`
- [ ] 若出现，删掉后规则是否仍成立
- [ ] 若仍成立，是否已改回：
  - [ ] 动作
  - [ ] 被挤开
  - [ ] 落字
  - [ ] 把话收回去
  - [ ] 执行后果

复核目标：

- [ ] 若这类句子删掉后规则仍成立，主线程优先判 `规则成立，但被中段直接把意思说破的句子卡在中间这档`

---

## 六、脚本后复核

- [ ] 已跑 `scene_lint.py --json`
- [ ] `scene_lint.py` 是按逐章文件调用，不是把 `正文/` 目录直接传给脚本
- [ ] 已跑 `validate_tracking_state.py --json --legacy-compat`
- [ ] 已区分：
  - [ ] `脚本候选`
  - [ ] `主线程第一眼怎么判`
  - [ ] `现在主要噪音`
- [ ] 未把脚本标签直接当终局判断
- [ ] 若样本目录里混有嵌套子样本副本，已显式锁定唯一主样本根位置，未混读子副本文件

---

## 七、噪音判断

- [ ] 当前是否被 `字数.error` 抢走主结论
- [ ] 当前是否被 `章尾.error` 抢走主结论
- [ ] 当前是否被 `解释.error` 抢走主结论
- [ ] 若被抢走，是否说明样本还没过“最低能看清结果的场面线”
- [ ] 若碰上 `章尾.error`，是否已先区分：
  - [ ] `最后半拍回弹不足`
  - [ ] `本组第二章偏短链`
  - [ ] `整章兑现不足`

---

## 八、收尾输出

- [ ] 第一眼该怎么判
- [ ] 为什么不是另一个相邻结论
- [ ] 现在主要噪音是什么
- [ ] 现在是不是已经到了“最纯验证态”

补充基线：
- `skill回归_收益写得太满了`
  - 目标：只测 `写后检查 / 章组总表 / 角色状态` 把收益写得太满
  - 算过要满足：
    - 脚本主要报 `收益写得太满了`
    - 不应混入 `双版本并存`
    - 不应混入 `缺时间线 / 缺角色状态 / 缺情报记录`

- `skill回归_双版本并存`
  - 目标：只测 `同范围重复写后检查文件并存`
  - 算过要满足：
    - 脚本主要报 `双版本并存`
    - 不应混入 `收益写得太满了`
    - 不应混入 `缺时间线 / 缺角色状态 / 缺情报记录`

- `skill回归_非情报流误触`
  - 目标：验证脚本先报 `情报.error` 时，主线程仍能判成 `非情报流误触候选`
  - 算过要满足：
    - `scene_lint.py` 主要报 `情报.error`
    - `validate_tracking_state.py` 保持 `0 error / 0 warn`
    - 允许存在轻度 `字数.warn`
    - 不应混入 `收益写得太满了`
    - 不应混入 `双版本并存`
    - 不应混入 `缺时间线 / 缺角色状态 / 缺情报记录`

- `skill回归_本组第二章偏短链`
  - 目标：验证第001章模板已落地时，主线程不会把第002章的 `字数.error` 误判成“残稿”或“模板未落地”
  - 算过要满足：
    - 第001章至少接近完整开场，能证明模板已进正文
    - 第002章主要报 `字数.error`
    - `validate_tracking_state.py` 保持 `0 error / 0 warn`
    - 允许第001章保留轻度 `字数.warn`
    - 不应混入 `收益写得太满了`
    - 不应混入 `双版本并存`
    - 不应混入 `情报.error / 解释.error / 章尾.error`

- `skill回归_本组第二章最后半拍回弹不足`
  - 目标：验证第002章最主要收益已在章尾前落成时，主线程不会把末半拍偏顺误判成 `第二章写得太短` 或 `整章兑现不足`
  - 当前使用说法：
    - 第001章应保持模板落地开场
    - 第002章中前段必须先落成：
      - 停签/落签/把先后次序改掉等现实收益
      - 至少 `1` 次外部反应
      - 至少 `1` 个器物/牌架/记录口反馈
    - 最后 `1-3` 句故意收弱
  - 现在先这样判：
    - `validate_tracking_state.py` 保持 `0 error / 0 warn`
    - `scene_lint.py` 仍可能先报 `字数.error`
    - 所以这组样本现在主要拿来练“主线程人工分开判断”，不是给脚本自动纯判准备的样本

补充写法约束：
- 若样本目标不是测 `收益写得太满了`，追踪文案里不要出现偏高一档的否定式说法，例如：
  - `还没轮到他先开口拍板`
  - `现在还没轮到他拍板`
  - `还没排到他先来`
- 这类句子当前也会被 `收益写得太满了` 视为候选，应改写成更低一层的事实描述：
  - `仍停在还要再核一遍这一层`
  - `只写成先挂上去的事实`
  - `只到暂停直接执行`
  - `还没有改成按他这边的说法办`

- 若样本目标是测 `非情报流误触`：
  - 正文里可以出现 `情报` 词面
  - 但不要写独立情报引号块
  - 必须确保这件东西的主功能仍是：
    - 把现场谁先谁后改掉
    - 卡住旧说法
    - 逼执行次序转入待复核
  - 追踪链要保持干净，避免被流程错误抢走主结论

- 若样本目标是测 `本组第二章偏短链`：
  - 第001章必须先证明模板主线已进正文
  - 第002章可以短，但不能短成只剩四五句主线摘要
  - 第002章至少还要保留：
    - 四拍主链本身
    - 少量现场反应
    - 至少一个器物/牌架/记录口反馈
  - 追踪链要保持干净，避免被流程错误抢走主结论

- 若样本目标是测 `本组第二章最后半拍回弹不足`：
  - 第002章不能只剩偏短链
  - 必须先证明最主要收益已在章尾前落成
  - 当前若脚本仍先报 `字数.error`，主线程必须额外人工确认：
    - 末句问题是不是只在最后半拍
    - 章尾前是否已经落成停签/记名/把先后次序改掉/外部改看
  - 这类样本当前重点不是追求脚本自动纯判，而是把人工分开判断的说法先写明白

---

## 九、当前硬规则

- [ ] `判断顺序能纠偏，但不能替代最低能看清结果的正文厚度`
- [ ] 样本太薄时，先补到“够看清结果”，再判断规则是否失效
