# 项目目录和文件别放乱

> 这份主要说 `story-long-write` 的书内文件该放哪、各自管什么。开书、写正文、回炉、追踪同步前都可以看一眼。

---

## 一、书里的东西必须放进小说目录

若项目内存在明确书名目录，例如：

- `某本书名目录/`

那下面这些东西都得放进这本书自己的目录里：

- `设定/`
- `大纲/`
- `正文/`
- `追踪/`
- `参考资料/`
- `对标/`

禁止写到项目根目录。

若误写到项目根目录，就先挪回去，再继续写作或检查。

---

## 二、目录大致长这样

```text
{书名}/
├── 设定/
├── 大纲/
├── 正文/
├── 追踪/
├── 对标/
└── 参考资料/
```

---

## 三、每个目录各管什么

### 1. `设定/`

负责：
- 主要设定
- 世界观
- 角色卡
- 关系图
- 题材定位
- 选题决策

不负责：
- 未来章节事件推进
- 当前追踪状态

再补几句：
- 角色卡默认放在 `设定/角色/`
- 开书阶段默认必须存在 `设定/选题决策.md`
- `设定/选题决策.md` 至少记录：
  - 优先方向
  - 能爆假设
  - 市场验证证据等级
  - 差异化定位
  - 最易失败点
  - 开写前验证动作
- 前 `3` 章涉及的关键角色卡必须前置建好，不得等正文写完后再补
- 角色卡默认使用：
  - `references/character-card-min-template.md`
- 关键角色卡至少包括：
  - 主角
  - `1` 个具名明敌
  - `1` 个具名摇摆角色 / 主要协作方
- 若关键角色卡缺失，或卡中没有“公开目标 / 私下最怕失去什么 / 身上最别扭的地方 / 被威胁时先护什么 / 删名可分辨依据”，不得进入首章正文
- 若关键角色卡缺失，或卡里连“明面上想要什么 / 私下最怕丢什么 / 身上最别扭的地方 / 一被逼就先护什么 / 去掉名字还能不能认出来”都没写清，就别进首章正文

### 2. `大纲/`

负责：
- 大纲
- 卷纲
- 细纲
- 开书候选分叉
- 未来章节预案

不负责：
- 当前已发生事实追踪

再补几句：
- 开书阶段默认必须存在 `大纲/开书候选分叉.md`
- `开书候选分叉.md` 至少记录：
  - 候选编号
  - 主角身份壳
  - 世界推进主路
  - 首章主事件
  - 第一明敌
  - 第一现实战果
  - 淘汰理由
  - 最终入选方案
- 若从同一开书主打直接跳到唯一蓝图，没有候选分叉，就别说开书去重已经做完了

### 3. `正文/`

负责：
- 当前唯一有效故事文本

不负责：
- 解释规矩
- 写作说明
- 工具残片

### 4. `追踪/`

负责：
- 当前有效事实
- 已落地事件、角色状态、情报状态、伏笔状态
- 上下文与修改单
- 开书排异与同题材去重过程记录

不负责：
- 未来章节预案
- 远期细纲内容

再补几句：
- 开书阶段默认必须存在：
  - `追踪/开书排异表.md`
  - `追踪/同题材去重记录.md`
- `开书排异表.md` 至少记录：
  - 本次禁止复用的母场景
  - 本次禁止复用的首章翻盘写法
  - 本次禁止复用的主角壳
  - 本次禁止复用的第一明敌形态
  - 本次禁止复用的第一现实战果形态
  - 本次最终采用方案如何避开上述重复
- `同题材去重记录.md` 至少记录：
  - 对比样本来源
  - 已对比的旧项目 / 旧测试样本
  - 主角身份壳是否撞型
  - 首章事件是否撞型
  - 第一母场景是否撞型
  - 第一明敌是否撞型
  - 第一现实战果是否撞型
  - 若撞型，最终改成了什么

### 5. `对标/`

负责：
- 对标书原文引用视图
- 拆文报告或对标摘要
- 本项目当前正在执行的对标采样记录

不负责：
- 把局部抽样伪装成整本精读
- 用模糊措辞替代实际采样量

再补几句：
- 每个项目只要进了对标流程，就必须有 `对标/对标采样执行单.md`
- `对标采样执行单.md` 不是摆着好看的说明页，而是边读边补的记事页
- 这张记录单至少要写：
  - 本轮目标
  - 样本书单
  - 每本计划采样量
  - 每本实际采样量
  - 有效场景单元
  - 这次看下来稳到哪一步
  - 哪些说法可以往硬限制里提
  - 哪些说法还只是局部观察
- 若项目已经用了对标样本，却没有 `对标/对标采样执行单.md`，那就别急着把这些说法提进细纲硬拦截、正文红线或写前闸门里

---

## 四、追踪文件和预案文件别混写

追踪文件：
- `时间线.md`
- `角色状态.md`
- `情报记录.md`
- `伏笔.md`

预案文件：
- `大纲.md`
- `卷纲_*.md`
- `细纲_*.md`
- `总安排表_*.md`
- `执行计划_*.md`

别混了：
- 未来内容写预案文件
- 已发生事实写追踪文件
- 不得把两类文件混用

---

## 五、默认怎么找目录

后续 skill 执行时，默认以“当前小说目录”为根：

- 写正文时：`{书名}/正文/`
- 补细纲时：`{书名}/大纲/`
- 同步追踪时：`{书名}/追踪/`
- 更新角色卡时：`{书名}/设定/角色/`

若项目根下同时有多个书名目录，必须先明确当前正在处理哪一本，再动文件。
