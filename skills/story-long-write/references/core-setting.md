# 核心设定

> 本文件负责 Phase 2 的最低产出、题材识别前置、核心设定表与设定类 artifact。

---

## 一、Phase 2 最低硬产出

进入 Phase 3 前，至少完成并落盘：

- `设定/核心设定.md` 或同等文件
- `设定/关系.md`
- `设定/题材定位.md`

缺任一项，不得进入卷纲、细纲或正文。

---

## 二、题材识别前置

产出核心设定时，必须同步产出一版当前作品题材识别结果，并写入 `设定/题材定位.md`。

最低字段：
- `主平台标签`
- `主卖点标签`
- `结构标签`
- `识别依据`
- `默认专项 checks`

规则：
- 若用户调整书名、题面、主卖点或核心金手指，必须先更新 `设定/题材定位.md`
- `设定/题材定位.md` 缺失时，不得进入 Phase 3

## 四、设定设计原则

- 从目标情绪倒推题材框架和剧情模式
- 从对标书提取可复用模块，但必须做角色位抽象，不得直接照搬
- 角色至少要先立住主角、核心反派、核心盟友、关键高价值角色
- 设定只保留会影响卷纲、细纲、正文判断的部分

---

## 五、设定类 artifact

完成核心设定后，按 `references/artifact-protocols.md` 创建：

- `设定/关系.md`
  - 角色关系映射
  - 细则参考 `references/character-relations.md`
- `设定/题材定位.md`
  - 题材核心梗三分法
  - 对标分析只保留 `2-3` 行摘要
  - 详细数据留在 `对标/`
  - 强制补入 `题材识别结果 + 默认专项 checks`
