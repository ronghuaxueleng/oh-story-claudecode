# 设定先搭这些

> 这一段就是管你在第 2 阶段先把哪些设定搭出来，别地基还没垫稳就往后写。

---

## 一、第 2 阶段先把这些写出来

进第 3 阶段前，先把下面几份东西落进文件里：

- `设定/主要设定.md` 或同等文件
- `设定/关系.md`
- `设定/题材定位.md`

少任一项，就别进卷纲、细纲或正文。

---

## 二、先把题材看准

搭设定时，顺手把这本书现在到底算什么题材写进 `设定/题材定位.md`。

最少要答清：
- `主平台标签`
- `主卖点标签`
- `结构标签`
- `识别依据`
- `默认额外检查`

别漏掉：
- 若用户调整书名、书名主打、主卖点或最顶上的金手指，先更新 `设定/题材定位.md`
- `设定/题材定位.md` 缺失时，就别进第 3 阶段

### 题材一旦定了，这些世界东西也得补上

题材看准以后，不能只写几个标签就算完，还得把这个题材天生该有的世界东西补出来。

只有碰上这些题材，才补对应内容；没碰上，就别硬塞进主路数里。

#### 1. `流放流 / 公路流 / 边荒流`

先补：
- 边荒为什么乱
- 流放路线为什么危险
- 边荒最主要的资源是什么
- 流放路上天然会爆哪些大事件
- 边荒规则系统是谁定的

最少写出：
- `世界总矛盾`
- `边荒乱源`
- `大环境事件池`
- `押送 / 流放事件模型`

#### 2. `仙侠 / 修仙 / 残魂 / 魂穿`

先补：
- 残魂 / 魂穿和世界的接入口
- 主角保留了哪些超出凡人的能力
- 能力的边界和代价
- 世界里哪些东西会与该能力产生反应

最少写出：
- `残魂保留能力`
- `能力使用边界`
- `残魂与世界咬合点`

#### 3. `情报流 / 系统流 / 预知流`

先补：
- 情报最适合落在哪些事件里
- 情报不能只落在小搜证 / 小判断 / 小试探里
- 情报最该兑现成哪类大场面翻盘

最少写出：
- `情报适配事件池`
- `情报兑现模型`
- `情报最容易滑成什么`

#### 4. `权谋 / 夺权 / 官场 / 掌局`

先补：
- 主要势力系统
- 主要资源系统
- 规矩系统由谁控制
- 主角最短该从哪一下开始抢势

最少写出：
- `势力系统`
- `资源系统`
- `规矩系统`
- `夺势入口`

若同时碰上多个标签，上面这些都要补，别只挑一个。

## 三、设定底稿怎么列

```md
## 设定地基表

### 基本信息
- 书名：{暂定名}
- 题材/类型：{主类型 + 副类型}
- 目标平台：{起点/番茄/晋江/其他}
- 预计字数：{X} 万字
- 目标读者：{画像}

### 一句话梗概
{主角 + 目标 + 阻碍 + 反转}

### 主角设定
- 姓名：{}
- 年龄：{}
- 最突出的特质：{2-3 个关键词}
- 金手指/关键能力：{}
- 弱点/缺陷：{}
- 最重的动机：{}
- 外形记忆点：{}
- 动作记忆点：{}
- 说话记忆点：{}
- 气场记忆点：{}

### 世界观主线
- 时代/背景：{}
- 关键设定：{}
- 力量体系：{}
- 社会结构：{}
- 世界总矛盾：{}
- 世界推进骨架：{}
- 最稀缺的资源：{}
- 视觉记忆点：{}
- 这世界天然会爆什么大事件：{}

若还碰上以下额外情况，再按需追加：
- 存在超凡 / 玄异 / 异象：补 `超凡接入口 / 异象系统 / 外面会怎么起反应`
- 存在权力争夺 / 阵营博弈：补 `势力系统 / 规则控制者 / 夺势入口`
- 存在资源路线 / 地图推进：补 `路线系统 / 区域风险 / 资源分布`
- 存在主要设定 / 金手指：补 `能力接入口 / 使用边界 / 代价 / 外部反应`

### 最重的冲突
- 主线矛盾：{}
- 终极 Boss/终极阻碍：{}
```

## 四、设定怎么搭才不虚

- 从目标情绪倒推题材路数和剧情走法
- 从对标书提取可复用模块，但得先做角色位抽象，别直接照搬
- 角色最少要先写到位主角、主要反派、主要盟友、关键重要角色
- 设定只保留会影响卷纲、细纲、正文判断的部分
- 若题材本身依赖“大环境 / 大事件 / 异象 / 多方争抢”，别把世界观压缩成“阵营名词 / 地图名词 / 空档次名词”；要答出“为什么这个世界会持续爆事”

## 五、设定文件还要补哪几份

把设定底稿搭完后，再按 `references/artifact-protocols.md` 去补这些文件：

- `设定/关系.md`
  - 角色关系映射
  - 细则参考 `references/character-relations.md`
- `设定/题材定位.md`
  - 题材主打路数三分法
  - 对标分析只保留 `2-3` 行摘要
  - 详细数据留在 `对标/`
  - 记得补进 `题材识别结果 + 默认额外检查`
