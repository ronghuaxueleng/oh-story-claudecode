# 核心设定

> 本文件负责 Phase 2 的最低产出、题材识别前置、核心设定表与设定类 artifact。

---

## 一、Phase 2 最低硬产出

进入 Phase 3 前，至少完成并落盘：

- `设定/核心设定.md` 或同等文件
- `设定/关系.md`
- `设定/题材定位.md`

缺任一项，不得进入卷纲、细纲或正文。

---

## 二、题材识别前置

产出核心设定时，必须同步产出一版当前作品题材识别结果，并写入 `设定/题材定位.md`。

最低字段：
- `主平台标签`
- `主卖点标签`
- `结构标签`
- `识别依据`
- `默认专项 checks`

规则：
- 若用户调整书名、题面、主卖点或核心金手指，必须先更新 `设定/题材定位.md`
- `设定/题材定位.md` 缺失时，不得进入 Phase 3

### 题材触发的世界补全硬规则

识别题材后，不得只产出“标签结论”，还必须根据标签自动补完对应世界底盘。

若命中以下标签，再追加对应专项；未命中时，不得反向把这些专项写成默认主路径。

#### 1. `流放流 / 公路流 / 边荒流`

必须补：
- 边荒为什么乱
- 流放路线为什么危险
- 边荒最值钱的资源是什么
- 流放路上天然会爆哪些大事件
- 边荒规则系统是谁定的

至少产出：
- `世界总矛盾`
- `边荒乱源`
- `大环境事件池`
- `押送 / 流放事件模型`

#### 2. `仙侠 / 修仙 / 残魂 / 魂穿`

必须补：
- 残魂 / 魂穿和世界的接口
- 主角保留了哪些超出凡人的能力
- 能力的边界和代价
- 世界里哪些东西会与该能力产生反应

至少产出：
- `残魂保留能力`
- `能力使用边界`
- `残魂与世界咬合点`

#### 3. `情报流 / 系统流 / 预知流`

必须补：
- 情报最适合承载在哪些事件里
- 情报不能只落在小搜证 / 小判断 / 小试探里
- 情报最该兑现成哪类大场面翻盘

至少产出：
- `情报适配事件池`
- `情报兑现模型`
- `情报最容易滑成什么`

#### 4. `权谋 / 夺权 / 官场 / 掌局`

必须补：
- 主要势力系统
- 主要资源系统
- 规矩系统由谁控制
- 主角夺势的最短路径是什么

至少产出：
- `势力系统`
- `资源系统`
- `规矩系统`
- `夺势入口`

若命中多个标签，以上补全部都要做，不得取其一。

## 四、设定设计原则

- 从目标情绪倒推题材框架和剧情模式
- 从对标书提取可复用模块，但必须做角色位抽象，不得直接照搬
- 角色至少要先立住主角、核心反派、核心盟友、关键高价值角色
- 设定只保留会影响卷纲、细纲、正文判断的部分
- 若题材本身依赖“大环境 / 大事件 / 异象 / 多方争抢”，不得把世界观压缩成“阵营名词 / 地图名词 / 空层级”；必须回答“为什么这个世界会持续爆事”
