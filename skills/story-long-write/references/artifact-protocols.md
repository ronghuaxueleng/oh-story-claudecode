# 这些项目文件怎么起

这里放的是各类项目文件怎么起、怎么填。子代理在第 2-3 阶段过渡时按需读这份文件。

**文件列表：**
- 设定/关系.md
- 设定/题材定位.md
- 设定/选题决策.md
- 设定/资料库采样执行单.md
- 设定/素材迁移方案.md
- 设定/世界大盘表.md
- 设定/主角终局高度设计.md
- 大纲/开书候选分叉.md
- 大纲/开书总蓝图.md
- 大纲/全书卷-章组分配表.md
- 大纲/全书章节一览.md
- 大纲/事件线一览.md
- 大纲/卷纲_第X卷.md（含情绪弧线+反转规划）
- 大纲/章组控制_第X卷.md
- 追踪/开书排异表.md
- 追踪/同题材去重记录.md
- 追踪/伏笔.md
- 追踪/时间线.md
- 追踪/角色状态.md
- 对标/对标采样执行单.md
- 对标/{对标书名}/拆文报告.md
- 对标/{对标书名}/原文/第XXX章_{章名}.md

**这些文件大概是干什么的：**
- 开书总蓝图.md = 全书总盘子（世界、权力、字数、卷数、阶段升级）
- 世界大盘表.md = 世界这层的大盘子（世界推进主路、权力层级、资源盘、规矩位、事件池、投放分期）
- 主角终局高度设计.md = 主角最后能爬到哪（终局身份、终局权力、每卷怎么往上抬，别把上限写低）
- 全书卷-章组分配表.md = 全书怎么排（每卷多少章组、每组大致章数、每组主要好处和怎么兑现）
- 全书章节一览.md = 全书按章俯看（每章一句话事件与收益）
- 事件线一览.md = 全书按线俯看（主线/副线/世界线）
- 大纲.md = 全书粗看（每卷一两句话定位）
- 卷纲_第X卷.md = 单卷安排（爽点节奏+情绪+人物+伏笔+反转）
- 章组控制_第X卷.md = 卷纲和单章细纲之间那一层
- 细纲_第XXX章.md = 单章写前底稿（事件+钩子+爽点+悬念，钩子直接写在细纲里）

- 若 `3` 组候选都来自同一局部母写法或同一层最显眼强信号，就算白分叉

## 设定/选题决策.md

这份通常在开书前由 `story-long-scan` 或 `story-long-write` 起。

这份主要是把三件事放到一起：
- 把“想写什么”和“市场上什么有信号”先并到一份方向判断里
- 给后面的开书分叉先划个边，免得一路写偏
- 记下为什么选这条路，开写前还差哪一步

```markdown

  - 趋势：
  - 代表样本：
  - 反例：
- 差异化定位：
- 可行性：高 / 中 / 低
- 最易失败点：
- 开写前还得补哪一步：

## 备选方向

### 备选 1
- 一句话方向：
- 能爆假设：
- 差异化定位：
- 可行性：
- 为什么没选为主方案：

### 备选 2
- 一句话方向：
- 能爆假设：
- 差异化定位：
- 可行性：
- 为什么没选为主方案：

## 与开书分叉的绑定要求
- 开书候选分叉最少要围绕上面这条主方向展开 `3` 组有效分叉
- 若最终入选方案偏离上面这条主方向，就写清改动原因
- 别跳过 `开书排异表` 与 `同题材去重记录`
```

先按这几条走：
- 若只有内置知识，没有实时榜单，`可行性` 先别往 `中` 以上写
- 别只写热词堆砌，要写清“能爆假设 + 差异化定位 + 失败风险”
- 这份文件不能直接拿来顶 `开书总蓝图`
