# Artifact 创建模板

各 artifact 的标准模板与创建指引。子代理在 Phase 2-3 过渡时按需加载此文件。

**模板列表：**
- 设定/关系.md
- 设定/题材定位.md
- 大纲/开书候选分叉.md
- 大纲/开书总蓝图.md
- 大纲/全书章节总表.md
- 大纲/事件线总表.md
- 大纲/卷纲_第X卷.md（含情绪弧线+反转规划）
- 大纲/章组控制_第X卷.md
- 追踪/开书排异表.md
- 追踪/同题材去重记录.md
- 追踪/伏笔.md
- 追踪/时间线.md
- 追踪/角色状态.md
- 对标/对标采样执行单.md
- 对标/{对标书名}/拆文报告.md
- 对标/{对标书名}/原文/第XXX章_{章名}.md

**层级关系：**
- 开书总蓝图.md = 全书总控（世界、权力、字数、卷数、阶段升级）
- 全书章节总表.md = 全书按章鸟瞰（每章一句话事件与收益）
- 事件线总表.md = 全书按线鸟瞰（主线/副线/世界线）
- 大纲.md = 全书鸟瞰（每卷一两句话定位）
- 卷纲_第X卷.md = 单卷规划（爽点节奏+情绪+人物+伏笔+反转）
- 章组控制_第X卷.md = 卷纲和单章细纲之间的中控层
- 细纲_第XXX章.md = 单章蓝图（事件+钩子+爽点+悬念，钩子直接写在细纲里）

## 对标/{对标书名}/拆文报告.md

> **拆文库/对标关系**：`拆文库/` = analyze skill 的原始产出（source of truth）。`对标/` = 写作项目的引用视图。首次引用时从 `拆文库/` 复制到 `对标/`。

此文件由 `story-long-analyze` 拆解管道输出（快速预览报告或完整拆文报告）。write skill 的职责是**读取**，不创建。

如需手动创建简版对标摘要（未使用 analyze skill 时）：

```markdown
# {对标书名} 对标摘要

## 基本信息
- 书名：{}
- 作者：{}
- 题材/类型：{}
- 目标平台：{}

## 核心发现
- 开篇钩子：{类型 + 手法}
- 爽点密度：{约 N 字/次}
- 节奏模式：{描述}
- 可借鉴套路：
  1. {}
  2. {}
  3. {}

## 禁止照搬
- {}（只学结构，不抄桥段）
```

## 对标/对标采样执行单.md

此文件由 `story-long-write` 在命中对标流程时创建并持续回填。

职责：
- 记录本项目当前对标采样到底做到哪一层
- 防止把局部抽读冒充成整体风格结论
- 给细纲、写前闸门、正文规则升级提供证据底账

```markdown
# 对标采样执行单

## 本轮目标
- 目标层级：{学开头 / 学正文 / 本章局部借法 / skill级规则}
- 本轮日期：{YYYY-MM-DD}
- 当前项目：{书名}

## 采样标准来源
- 固定标准：`story-long-write/references/benchmark-sampling.md`
- 当前执行口径：{对应档位摘要}

## 样本书单

| 对标书 | 用途 | 计划采样量 | 实际采样量 | 有效场景单元 | 当前状态 |
|--------|------|-----------|-----------|-------------|---------|
| {书名} | {开头/正文/对白/叙述/skill规则} | {黄金三章/中段2段/前8章...} | {已读区段/章数/约字数} | {N} | {未开始/进行中/阶段够用} |

## 实际采样记录

### {对标书名}
- 本轮抽读：
  - {区段1}
  - {区段2}
- 本轮连续读到：{第X章 / 前X行 / 前X字}
- 本轮有效场景单元：{N}
- 当前观察：
  - {机制1}
  - {机制2}
- 当前结论稳定度：{低 / 中 / 高}
- 暂不得上升的原因：{若有}

## 当前可用结论

### 可用于本章局部借法
- {结论}

### 可升级为正文通用规则
- {结论}

### 暂不得升级
- {结论 + 原因}

## 升级裁决
- 是否允许写入细纲硬闸：{是/否}
- 是否允许写入写前闸门：{是/否}
- 是否允许写入正文红线：{是/否}

```

硬规则：
- 首次引用对标样本时就要创建，不得等写完正文再补
- 每轮补样后必须回填“实际采样量 / 有效场景单元 / 当前结论稳定度”
- 若文件只写了书名和感觉，没有实际读量，视为无效执行单
