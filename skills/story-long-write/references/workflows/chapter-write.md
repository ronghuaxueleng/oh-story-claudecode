# 单章写作流程

> 这份只整理“有细纲后正文该按什么顺序往下写”。
> 开始前读 `references/chapter-gate-protocol.md`，这里主要保留顺序，不把所有规则再抄一遍。

---

## 一、写前

1. 读取本章细纲
2. 读取上一章正文
3. 读取相关角色卡
3.1. 若本章关键角色卡缺失，直接按 `F5` 停写，不得跳过
3.2. 关键角色卡至少包括：
   - 主角
   - 本章主要对手
   - 本章主要协作方 / 摇摆角色
3.3. 读取角色卡时，不得只看身份和外形，必须额外提取：
   - 公开目标
   - 私下最怕失去什么
   - 这章最重的冲突
   - 平时说话规则
   - 被压迫时说话规则
   - 被威胁时先护什么
   - 会下意识碰什么东西
   - 删名可分辨依据
4. 读取追踪文件里当前已经写实的内容
4.1. `追踪/时间线.md`、`追踪/角色状态.md` 必须存在；若作品是 `情报流 / 系统流 / 预知流 / 任务流`，`追踪/情报记录.md` 也必须存在
4.2. 任一必需追踪文件缺失，直接按 `F5` 停写，先补文件，不得用“从正文推断”“先写完再同步”代替
5. 若本章为回炉章，先补上轮问题短记
6. 读取 `设定/爽点模型.md`
7. 先判上一章 `主场面 / 场面靠什么东西撑住 / 事件规模等级 / 残魂出手类型`
8. 从 `设定/爽点模型.md` 的事件池中先选本章 `主事件`
9. 判定本章是不是 `母场景复用`、`低成本场面东西回流`、`连续低压推进`
10. 若是 `世界主动施压` 题材，先写清“是哪股世界事件先压到主角脸上”
11. 再判本章残魂 / 金手指应该走哪一种 `出手类型`
12. 若本章是首章且作品里有足以改掉默认认知的世界规矩，先钉一个 `首章世界总盘交代`
13. 若本章是首章且作品里有足以改掉默认认知的世界规矩，再钉一个 `世界底色落点`
14. 回看最近 `2-3` 章的 `主收益走法`，避免连续复用同一套“抢关键物 + 逼失口 + 章尾再抬一下”
15. 若题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，本章再补一个 `世界底色续显点`
16. 若本章相对上一章发生 `主场面切换 / 地点切换 / 势力切换 / 时间段切换`，先写清 `跨场景过桥`
17. 读取 `references/chapter-title-rules.md` 并确认本章标题
18. 完成状态筛选
19. 完成模块召回
20. 先补 `字数安全边际` 与 `中后段外部反应落点`
20.1. 若还是 `冷启动首章组`（通常为第 `001-003` 章），正文首写目标默认按 `3500-4000` 起，不得低于 `3400`
20.2. 若还是 `冷启动首章组`，本章默认至少预埋 `2` 处外部反应：
   - `1` 处中段群体 / 协作 / 办事顺序起变化
   - `1` 处章尾前先后变化 / 资格变化 / 谁前谁后变了 / 说法变化
21. 若最近 `2` 章内出现过 `字数.warn`，本章目标字数默认再上抬 `150`
22. 生成写前闸门
22.1. 读取 `references/first-draft-execution-kernel-template.md`，生成本章 `本章短清单`
22.2. 读取 `references/authorial-intrusion-negative-examples.md`，至少为本章挑出 `2-4` 条最可能复发的坏句结构，写入 `本章短清单`
22.3. 若 `本章短清单` 还在重复方法论术语，没有写清“本章只准改掉什么 / 禁什么 / 章尾只许停哪”，直接停写，按 `F5`
23. 逐条核对允许新增清单：本章允许新增什么，不允许新增什么
24. 关键角色起草前再问一遍：
   - 这人这场嘴上会说什么
   - 这人心里真正护的是什么
   - 这人先露出来的是哪一个身体 / 动作信号
   - 删掉名字后，读者还能不能认出他
25. 若本章是首章，起草前再问一遍：
   - 读者前 `300-500` 字内能不能知道这是什么世界
   - 读者能不能看懂这个世界里的制度 / 日常 / 生存方式为什么是这样
   - 主角当前死局是不是已经和这套世界规则绑死，而不是后文再补设定
26. 起草正文前先过一次 `首稿污染预防口令`：
   - `一级禁用词` 不得出现在本章首稿里
   - 不得用生硬撑劲词承担动作主功能
   - 动作句必须先写 `受力点 / 位移 / 破坏结果`，再写语气
   - 如果脑中先冒出来的是“狠、炸、压、猛”，先把它翻译成具体动作，再落笔
27. 若项目脚本下限为 `2900`，正文首写目标默认按 `3300-3800` 起，不得拿 `2900-3000` 当收稿区
27.1. 若还是 `冷启动首章组`，正文首写目标默认按 `3500-4000` 起；除非细纲明确要求短章，否则不得低于 `3400`
28. 若首稿统计后低于 `3100`，必须按固定配额回补：
   - 前 `1/3` 补 `1` 处受阻 / 被压回来
   - 中段补 `1` 处第三方插手 / 谁前谁后变了 / 群体反应
   - 后 `1/3` 补 `1` 处后续实利
   - 章尾前补 `1` 轮代价 / 关系变坏 / 局面改线
   - 关键角色补 `1` 处亮相动作
   - 每补一处，必须同时写清“谁因此只好改动作 / 往前挤或往后退 / 让出先后”
   - 若补完后答不出上述变化，默认这一段只是虚胖，不计入回补
   - 必须额外读取 `references/scene-thickening-method.md`，按其中的 `动作拍扩写清单` 至少新增 `3` 拍
28.1. 若本章第一爆点已经打出，但还答不出“爆点之后谁把话咽回去了、谁挪位了、谁把办事顺序改了”，优先视为 `后续实利缺失`，不得用别的段落替代
28.2. 若本章是 `经营 / 分配 / 资格 / 放票 / 交易 / 审批 / 拍卖 / 定顺序` 这类场景，回补顺序优先改为：
   - 先补谁被主角实际改了先后 / 资格 / 归属
   - 再补围观者 / 执行次序 / 协作方的现场变化
   - 最后才补章尾抬升
28.3. 若还是 `冷启动首章组` 且首稿低于细纲目标 `120` 字以上，除上面配额外还必须再补：
   - `1` 口现场细部阻力
   - `1` 处围观者 / 利益相关者争先
   - `1` 处章尾前有人往前挤一步，或者现场又变一下
28.4. 若还是边界题材冷启动，首稿起草前必须先写一张“现实挤压补充卡”，至少答出以下 `4` 项中的 `3` 项；答不出不得进入正文：
   - 哪个实际位置会被抢：`桌位 / 门位 / 主桌位 / 礼台位 / 供桌位 / 后门位 / 名册位 / 册位 / 调兵位`
   - 哪个第三方会因为自保或私利先挪地方
   - 哪个角色会被挤掉一格实际位置
   - 哪一项“先记名 / 先挂号 / 先挂册 / 先挂簿”会立刻改掉谁的动作
28.5. “现实挤压补充卡”的回补顺序就按下面来：
   - 先补 `谁被挤掉实际位置`
   - 再补 `谁趁乱抢到第二个能伸手的位置`
   - 再补 `哪道名分 / 名册 / 名单因此开始起现场作用`
   - 不得先补背景、心理、术语解释
28.6. 首稿进入 `配额回补态` 后，必须再写一张 `厚度回补卡`：
   - 先补哪三口
   - 三口分别落在哪一拍
   - 每一处补完后，谁被逼得改动作 / 往前挤或往后退 / 改先后
   - 哪一处承担中段新增争先
   - 哪一处承担章尾前新增抬升
   - 若只能写“气氛更强 / 大家更急 / 冲突更大”，视为未写卡，不得继续
28.7. `厚度回补卡` 写完后，必须立刻转成 `动作拍扩写清单`：
   - 至少新增 `3` 拍，推荐 `3-5` 拍
   - 每拍固定写成 `谁做了什么 -> 谁被迫怎么改`
   - 至少覆盖：
     - `1` 拍中段争先 / 往前挤 / 改顺序
     - `1` 拍第三方现场变化 / 执行次序改手
     - `1` 拍章尾前承压 / 抢第二手失败 / 新后果开始起作用
   - 若新增拍仍主要由解释句、总结句、心理句承担，视为扩拍失败，不得继续
28.8. 若 `动作拍扩写清单` 写完后仍明显低于细纲目标 `120` 字以上，必须额外读取 `references/chain-reaction-scene-thickening.md`，再写一张 `连锁动作拍卡`：
   - 至少补 `2` 条连锁
   - 每条固定写成 `起手动作 -> 第一波被动变化 -> 第二波被动变化`
   - 至少 `1` 条连锁落在中段，让场面继续滚动
   - 至少 `1` 条连锁落在章尾前，给最后一拍加承压
   - 若只能写到第一波变化，视为连锁失败，不得继续
29. 配额未补满前，不得再次跑字数统计，不得收章，不得宣称模板验证成功
29.1. 正文首稿一写完，进入写后检查前必须先做：
   - `draft_purity_guard.py --fix --json`
   - `一级禁用词扫描`
   - `全文残片扫描`
   - `scene_lint.py --json`
29.2. 其中任一失败：
   - 不得进入下一章
   - 不得把失败正文当成“待会再统一收拾”
   - 若失败项是 `一级禁用词 / 生硬撑劲词 / 已知复发污染词 / 工具残片 / 非故事语料`，必须先在本章即时回修，再重跑当前四项，不得等后续检测或 `story-review`
   - 纯基准测试只记录失败，不修正文
29.3. 正文首稿一写完，进入写后检查前必须再做一次 `解释句前置自检卡`：
   - 抽查 `章首 / 第一爆点后 / 章尾前 / 章尾最后一拍` 四处
   - 逐条回答：
     - 这里有没有 `这就算真的起作用了 / 现在不是…… / 这就不是只…… / 这才是……`
     - 删掉这句后，现场意义是否仍然成立
     - 若成立，是否已改回 `动作 / 谁前谁后变了 / 物件变化 / 他人被动承认`
     - 必须把 `原句 -> 是否可删 -> 替换成什么` 三栏写进 `本章短清单` 或写后检查，不得只写“已自检”
     - 只写“待脚本复检 / 待人工复看 / 后续再压”一律视为还没改实
   - 若关键位已经能答出“可删”，但替换栏还是空白，直接按 `F2`
   - 只要任一处答成“删掉也成立，但还没改”，不得进入写后检查；生产流程按 `F1/F2` 当场回修，纯测试说法记录失败
29.3.1. 若本章写的是 `重要角色先付代价再压住别人`，必须额外做一次 `中段把意思直接说破的句子拦截`：
   - 重点抽查重要角色完成 `先压自己人 / 自己物 / 自己名分` 后、章尾最后一拍前的中段句子
   - 逐条回答：
     - 这里有没有 `我若先把自己人……谁还认…… / 我若先护自己人……谁还认…… / 我若先把自己人塞进……谁还认……` 一类直接把意思说破的句子
     - 删掉这句后，`先付代价再压住别人` 是否仍成立
     - 若仍成立，是否已改回 `动作 / 谁被挤开 / 落字 / 他人把话收回去 / 执行结果`
   - 只要任一处答成“删掉也成立，但还没改”，不得进入写后检查；生产流程按 `F1/F2` 当场回修，纯测试说法记录失败
29.3.2. 若本章写的是 `重要角色先付代价再压住别人`，必须再额外做一次 `旁白提前说破 / 章节硬提示句拦截`：
   - 重点抽查重要角色完成 `先压自己人 / 自己物 / 自己名分` 后，到章尾最后一拍之间的旁白句
   - 逐条回答：
     - 这里有没有 `先替读者把争点说死 / 先替人物点明最怕丢什么 / 先替章节指定停点` 一类句子
     - 这里有没有 `不是X，而是Y` 一类把现场意义直接说破的变体直说句
     - 删掉这句后，现场争夺链 / 人物压力链 / 章尾动作链是否仍成立
     - 若仍成立，是否已改回 `动作 / 争先 / 失手 / 谁前谁后变了 / 新执行结果`
   - 只要任一处答成“删掉也成立，但还没改”，不得进入写后检查；生产流程按 `F1/F2` 当场回修，纯测试说法记录失败
29.3.3. 若本章想把重要角色写到最高那档“先伤自己一下，再把场子彻底压住”，必须额外实答：
   - 她/他本人当场失去的 `实际位置 / 先后机会 / 面子 / 名分` 到底是什么
   - 这一步失去是否已经被现场别人看见并被迫改局
   - 这一下失去之后，她/他立刻又失去的 `后面还能不能稳住的位置 / 再签一次的权力 / 还能不能替这事作保 / 执行次序 / 自己人信任` 是什么
   - 她/他这边的人里，谁先因此不再替她/他说圆场话 / 把话收回去 / 只照命令办事
   - 若只有“当场见血 / 见伤 / 见霜”，但没有继续带出“后面能不能稳住的位置往下掉了”或“自己人开始不再替她/他说圆场话”，默认仍只算“写到一半”或“接近这一步”，不得直接写成已经完全压住了
   - 若答不出，默认只可停在“还差一点”，不得在 `本章短清单`、写后检查、角色状态里写成已经完全压住了
29.4. 正文首稿一写完，进入写后检查前必须再做一次 `模板节点标签句自检`：
   - 抽查 `第一爆点后 / 群体反应落点 / 第二次外部反应落点 / 章尾前 / 章尾最后一拍`
   - 逐条回答：
     - 这里有没有 `这是第一下…… / 第二次外部反应也来了 / 这就算真的起作用了 / 这才是……真正重要的地方 / 这不是帮腔 / 这不是只下场 / 问，就是要写`
     - 删掉这句后，动作链、谁前谁后变了、器物结果、落字结果是否仍然成立
     - 若成立，是否已改回 `动作 / 谁前谁后变了 / 器物结果 / 他人被动承认`
     - 必须把 `原句 -> 是否可删 -> 替换成什么` 三栏写进 `本章短清单` 或写后检查
   - 只要任一关键位答成“可删但还没改”，不得进入写后检查；生产流程按 `F1/F2` 当场回修，纯测试说法记录失败
29.4.1. 进入写后检查时，`方法资料用了多少` 不得一次性写成 `全部已落地且有效`：
   - 每份资料至少分开写 `已落地`、`待复审确认`、`仅读取未落地` 三种之一
   - 每份资料还必须实填三段：`资料名 -> 解决风险 -> 落到正文哪一拍`
   - 若任一资料答不出 `解决风险` 或 `落到正文哪一拍`，默认按 `方法资料没有写实` 处理
   - 若所有资料都被直接写成 `已落地且有效`，默认先判 `追踪说法写得太满风险`
   - 纯测试说法必须显式保留至少 `1` 项 `待复审确认` 或 `仅读取未落地`，除非正文已给出不可争议的独立证据链
30. 若本章写的是功能场景，起草前必须先写 `功能场景四步`：
   - 谁想拿什么
   - 谁先动手
   - 谁被逼得改动作 / 把话咽回去 / 往前挤或往后退
   - 规则因此如何显形
   若自然顺序写成“先讲规则再讲动作”，直接停写
31. 起草前必须写 `动作拍清单`：
   - 至少 `4-6` 拍
   - 每拍只写 `谁做了什么 -> 谁被迫怎么改`
   - 每拍还必须补一行：`谁被改掉了哪一步实际位置 / 谁立刻吃了什么损失 / 谁从此不能退回旧说法`
   - 不得把“规则说明 / 作者判断 / 世界总述”写进动作拍清单
   - 若任一拍只能写成“更乱了 / 更急了 / 更危险了”，视为无效拍，不得进入正文
32. 起草前必须写 `人物冲突卡`：
   - 主角：嘴上报什么，动作先护什么
   - 对手：嘴上压什么，动作先藏什么
   - 摇摆位：嘴上推什么，动作先避什么
   - 关键角色：本章会冒什么风险，先弃哪一样，站出来立刻失去什么
   - 若答不出“这人先护哪一样”，直接停写
   - 若答不出“这人站出来会立刻失去什么”，默认该角色仍只剩出来走个过场，不得进入正文
33. 起草前必须写 `章尾现场句预案`：
   - 只许写一条现场结果句
   - 只许落在 `新动作 / 新物证 / 新名单 / 新命令 / 新翻面`
   - 若自然写成“意义更大了 / 黑幕更深了 / 下一章更危险”，直接停写
34. 起草前必须写 `人物说话权限表`：
   - 主角只许报后果 / 死线 / 漏口，不许完整讲制度
   - 对手只许护自己的那一份、拿规则压人，不许替读者讲全规则
   - 摇摆位只许报自己利益、留半句，不许一下全说尽
   - 围观位只许报自己死活，不许替主角总结局势
35. 起草前必须写 `旁白禁越权`：
   - 旁白不许替读者判谁更狠
   - 旁白不许替读者判这一拍意味着什么
   - 旁白不许替人物翻译“真正怕什么 / 真正想要什么”
   - 旁白不许直接给模板节点贴标签，例如 `这是第一下外部反应 / 这就算真的起作用了 / 这才是……真正重要的地方`
   - 若脑中先冒出总结句，先改成动作句再落笔
35.1. 起草前必须再压缩确认一次 `本章短清单`：
   - 只能保留当前章最关键的 `6-10` 条执行项
   - 每条都必须能回答“漏了会写错什么”
   - 若还有“文风自然一点 / 更有人味 / 多点张力”这类只说个方向的话，删掉重写
35.2. 起草前必须写 `模板兑现卡`：
   - 第一爆点是什么
   - 后续实利是什么
   - 群体反应落在哪一拍
   - 章尾前一拍怎么先抬一下
   - 章尾最后一拍具体新增什么结果
   - 若首稿偏薄，优先回补哪三口
   - 若答不出以上任一项，不得进入正文
36. 起草前必须写 `受阻过桥卡`：
   - 本章至少写清 `1` 次：谁原本准备怎么拿 -> 被什么突然打断 -> 只能舍掉什么 -> 改用什么打法
   - 若写不出这条链，说明本章阻力还是平的，不得进入正文
37. 起草前必须写 `非预期障碍卡`：
   - 至少 `1` 口障碍不能来自主角事先知道的旧阻力
   - 必须写清它是谁或什么造成的、打断了哪一拍、逼谁改动作
   - 若障碍只是“对手继续拦”或“事情更麻烦了”，视为未写实

## 二、正文

写作时至少盯住：
- 当前章 `本章短清单`
- 本章主要往前推什么
- 本章最主要收获
- 本章第一爆点
- 本章高潮后续收益
- 本章章尾停点
- 本章标题是否真的勾住卖点
- 本章收益类型是否符合 `爽点模型`
- 本章是在兑现书名主打那件事，还是偷偷写偏成了别的东西
- 本章是不是先由 `大事件 / 大环境 / 多势力撞局` 起爆，而不是又缩回小场景查线
- 本章是否和上一章复用了同类母场景
- 本章主收益走法是否和最近 `2-3` 章拉开
- 本章主要对手 / 物证 / 资源 / 场面靠什么撑住，是否已经尽快点出来
- 本章关键角色卡里的 `公开目标 / 私下所护 / 说话规则 / 记得住的动作` 是否已经真正回落到正文
- 本章关键角色是否遵守 `人物说话权限表`
- 本章中后段是否已经落出至少 `1` 处旁人 / 协作方 / 系统说法的外部反应
- 若还是 `冷启动首章组`，本章是否已经落出至少 `2` 处外部反应，且承担的作用不重复
- 本章第一爆点之后，是否已经打出 `后续实利`，而不是只剩情绪反应
- 本章是否至少出现 `1` 处群体反应 / 执行次序改线 / 资格先后变化
- 本章章尾前，是否已经先落 `1` 处新的现实动作或新的压力抬升
- 本章最关键的设定有没有真的打出“只有它能弄出来的结果”
- 本章残魂 / 金手指的出手类型是否和上一章拉开
- 若本章发生跨场景切换，前 `100-200` 字内是否补清：
  - 主角为什么来
  - 主角怎么来的
  - 主角为什么正好撞上这场事
- 本章是否存在“章尾只抛钩，下一章直接上大场面”的硬切
- 本章大场面切入前，是否已经给出主角的行动决策或怎么到场
- 若本章是首章，读者是否已经知道“自己在读什么世界”，而不是先误读成普通现实题材
- 若本章是首章，世界总盘是否已经通过制度 / 日常 / 生存方式 / 群体常识至少落出 `2` 项
- 若本章是首章，读者是否已经看见世界规则怎样改动作、改规矩、改生死
- 若本章不是首章，世界底色是否还在通过规矩 / 环境 / 群体动作继续露出
- 残魂 / 金手指的后果，是否已经被外部人物、环境或器物真实感知
- 本章若是 `L3/L4`，有没有把规模写成现实变化，而不是口头放大
- 本章是否偷偷塞进了细纲 / 写前闸门没记过的新东西
- 若出现新增项，它是否在允许新增清单里；若不在，是否已先补写大纲/细纲再继续正文
- 本章关键角色是否仍在靠身份标签说话，而不是按角色卡里的行为规则说话
- 本章是否先用动作把规则逼出来，而不是先讲规则、再让动作补执行
- 本章是否至少有 `1` 次非预期障碍真正打断了行动节奏，而不是按计划受阻
- 本章受阻后，是否真的发生了 `舍取 / 改决断 / 改协作关系`，而不是直接跳到结果
- 若脑中先冒出 `这说明 / 这意味着 / 真正要命的是 / 更大的还在后头`，必须先停笔，改写成 `动作 / 物证 / 群体反应 / 谁前谁后变了 / 规矩变化`

## 三、写后

1. 统计字数
2. 优先跑 `scene_lint.py --json`
3. 做纯净度检查
4. 回看章尾，专查：
   - 章尾下结论词
   - “下一章将会/接下来/更大的”类预告腔
   - 是否真有新钩子实物或新局面
4.1. 逐条回看 `本章短清单` 中的禁用句型，至少回答：
   - 有没有出现
   - 若出现，用了哪种现场东西替换
   - 若没替换完，为什么还不能判 `F0`
5. 若 `scene_lint.py --json` 结果只报 `字数.warn`，先当成可通过预警，不得直接视为失败
5.0. 但若还是 `冷启动首章组`，`字数.warn` 默认视为 `首稿厚度不足预警`
5.1. 但若 `字数.warn` 同时伴随 `后续实利缺失 / 群体反应缺失 / 章尾前一拍缺失` 任一人工复核项，不得直接算过，先回补正文
5.2. 若还是 `冷启动首章组` 且报了 `字数.warn`，必须额外复查：
   - 是否只写出 `1` 处外部反应
   - 是否缺少中段争先 / 争位 / 争顺序
   - 是否章尾前缺少最后一拍现场抬升
   任一项出现，都应先回补再判是否保留
6. 若 `scene_lint.py --json` 结果报出 `章尾.warn / 解释.warn / 结构.warn / 情报.warn`，先人工复看，不得只因退出码为 `0` 就直接算过
6.1. 冷启动样本额外复核：
   - 模板内容是否真的进入正文事件链
   - 是否已同步追踪文件
   - 是否已生成章组写后检查总表
   - 若任一缺失，不得判定本轮 skill 验证成功
7. 生成写后检查
7.1. 若当前工作范围还是连续 `2-4` 章同一章组，且本章为该组最后一章或本轮回炉收尾章，必须先生成或更新 `追踪/写后检查_第XXX-XXX章.md`
7.2. 章组总表至少写清：`这组已写事实总汇 / 分章现实收益 / 每章目标到了哪一步+实际到了哪一步+证据句+为什么还没再往上走 / 关键角色卡补齐状态 / 情报证据轻重变化 / 这组现在能不能算完成`
7.3. 若缺章组总表，不得进入追踪同步，不得把分章写后检查视作章组收尾完成
8. 先做已落地事实回写前的降级检查
9. 更新追踪文件
10. 更新角色卡
11. 生成交叉复查
12. 给出这章现在能不能算完成的判断

- 如果这一章写到重要角色的关键台词：这一下失去逼谁把原话吞回去 / 往前往后挪 / 让出先后 / 不敢再硬顶

- 这章现在能不能算完成

- 写后检查不得只写 `本章收益到了哪一步`，必须同时写 `目标收益到了哪一步 / 实际收益到了哪一步 / 证据句 / 为什么还没再往上走` 四项。
- 若 `目标收益到了哪一步` 与 `实际收益到了哪一步` 不一致，默认保留更低那一步，并把差额原因写入写后检查与章组总表。
- 若答不出 `证据句`，不得进入 `F0` 写后检查，也别收进主正文。

## 六、独立交叉复查

不得把“说法交叉复查通过”只写成写后检查中的一行。

必须单独写 `交叉复查_第XXX章.md`，至少检查：
- 正文 vs 细纲
- 正文 vs 写前闸门
- 正文 vs 追踪文件
- 正文 vs 角色卡

任一组还没改实，就别收进主正文。

## 七、回炉前的最小回看

若本章不是首写，而是回炉 / 重写，动笔前必须单独写出：
- 上轮主失败码
- 上轮主问题是在 `规矩 / 章节 / 句子 / 流程` 里的哪一块
- 本轮最先要防止复发的旧问题
- 需要同步修改的项目文件列表

---

## 八、跨场景过桥最小硬规则

若本章相对上一章发生以下任一变化，默认必须写 `跨场景过桥`：
- 主场面地点变化
- 主对手 / 主势力变化
- 时间段变化（如白天切夜里、夜里切次日）
- 主事件靠什么撑住发生了变化（如公开争抢切祭台对峙、祭台对峙切营地内乱）

写前闸门与细纲中至少写清：
- 主角为什么必须去这个新场
- 主角靠什么信息 / 观察 / 现场余波锁到这个新场
- 主角怎么赶到或提前摸到
- 主角到场时，为什么正好撞上本章第一爆点

正文硬要求：
- 若这一章有跨场景切换，开章前 `100-200` 字内必须补足上述至少 `2-3` 项
- 不得只用“主角赶到时 / 主角来到 / 事情果然爆了”一笔带过
- 不得把“读者能猜到”当成免写理由

默认别算过的情形：
- 上一章章尾只抛某处要出事，下一章直接写主角已在现场观战
- 上一章刚拿到账册，下一章直接切到新据点，没有交代怎么锁住这条线
- 只顾大场面开炸，不补主角行动链
