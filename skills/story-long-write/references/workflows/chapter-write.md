# 单章写作流程

> 本文件负责“有细纲后如何写正文”的最小强制顺序。
> 执行前优先读取 `references/chapter-gate-protocol.md`，本文件只保留顺序，不再重复承载所有规则。

---

## 一、写前

1. 读取本章细纲
2. 读取上一章正文
3. 读取相关角色卡
4. 读取追踪主表中的当前有效事实
5. 若本章为回炉章，先补最小复盘
6. 读取 `设定/爽点模型说明书.md`
7. 先判上一章 `核心母场景 / 核心场面载体 / 事件规模等级 / 残魂出手类型`
8. 从 `设定/爽点模型说明书.md` 的事件池中先选本章 `主事件`
9. 判定本章是否命中 `母场景复用`、`低成本载体回流`、`连续低压推进`
10. 若命中 `世界主动施压` 题材，先写清“是哪股世界事件先压到主角脸上”
11. 再判本章残魂 / 金手指应该走哪一种 `出手类型`
12. 若本章是首章且作品存在足以改写默认现实认知的世界机制，先钉一个 `首章世界总盘交代`
13. 若本章是首章且作品存在足以改写默认现实认知的世界机制，再钉一个 `世界底色显影锚点`
14. 回看最近 `2-3` 章的 `主收益骨架`，避免连续复用同一套“抢关键物 + 逼失口 + 章尾抬高层”
15. 若题材含 `仙侠 / 玄异 / 残魂 / 灾异 / 旧阵 / 禁地`，本章再补一个 `世界底色续显点`
16. 若本章相对上一章发生 `核心场面切换 / 地点切换 / 势力切换 / 时间段切换`，先写清 `跨场景过桥`
17. 读取 `references/chapter-title-rules.md` 并确认本章标题
18. 完成状态筛选
19. 完成模块召回
20. 先补 `字数安全边际` 与 `中后段外部反应位`
21. 若最近 `2` 章内出现过 `字数.warn`，本章目标字数默认再上抬 `100`
22. 生成写前闸门
23. 逐条核对白名单：本章允许新增什么，不允许新增什么
24. 若本章是首章，起草前再问一遍：
   - 读者前 `300-500` 字内能不能知道这是什么世界
   - 读者能不能看懂这个世界里的制度 / 日常 / 生存方式为什么是这样
   - 主角当前死局是不是已经和这套世界规则绑死，而不是后文再补设定
25. 起草正文前先过一次 `首稿污染预防口令`：
   - `一级禁用词` 不得出现在本章首稿里
   - 不得用口头发力词承担动作主功能
   - 动作句必须先写 `受力点 / 位移 / 破坏结果`，再写语气
   - 如果脑中先冒出来的是“狠、炸、压、猛”，先把它翻译成具体动作，再落笔

---

## 二、正文

写作时至少盯住：
- 本章主驱动
- 本章最值钱战果
- 本章第一爆点
- 本章高潮后续收益
- 本章章尾停点
- 本章标题是否真的勾住卖点
- 本章收益类型是否符合爽点模型说明书
- 本章是在兑现题面，还是在偷换成别的写法
- 本章是不是先由 `大事件 / 大环境 / 多势力撞局` 起爆，而不是又缩回小场景查线
- 本章是否和上一章复用了同类母场景
- 本章主收益骨架是否和最近 `2-3` 章拉开
- 本章核心对手 / 核心物证 / 核心资源 / 核心场面载体是否已经尽快落名
- 本章中后段是否已经落出至少 `1` 口旁人 / 协作方 / 系统口径的外部反应
- 本章核心设定有没有真正打出“只有它能造成的结果”
- 本章残魂 / 金手指的出手类型是否和上一章拉开
- 若本章发生跨场景切换，前 `100-200` 字内是否补清：
  - 主角为什么来
  - 主角怎么来的
  - 主角为什么正好撞上这场事
- 本章是否存在“章尾只抛钩，下一章直接上大场面”的硬切
- 本章大场面切入前，是否已经给出主角的行动决策或到场路径
- 若本章是首章，读者是否已经知道“自己在读什么世界”，而不是先误读成普通现实题材
- 若本章是首章，世界总盘是否已经通过制度 / 日常 / 生存方式 / 群体常识至少落出 `2` 项
- 若本章是首章，读者是否已经看见世界规则怎样改动作、改规矩、改生死
- 若本章不是首章，世界底色是否还在通过规矩 / 环境 / 群体动作继续显影
- 残魂 / 金手指的后果，是否已经被外部人物、环境或器物真实感知
- 本章若是 `L3/L4`，有没有把规模写成现实变化，而不是口头放大
- 本章是否偷偷加入了细纲 / 写前闸门未备案的新核心信息
- 若出现新增项，它是否属于白名单；若不属于，是否已先回填纲层再继续正文

---

## 三、写后

1. 统计字数
2. 优先跑 `scene_lint.py --json`
3. 做纯净度检查
4. 回看章尾，专查：
   - 章尾盖章词
   - “下一章将会/接下来/更大的”类预告腔
   - 是否真有新钩子实物或新盘面
5. 若 `scene_lint.py --json` 结果仅命中 `字数.warn`，先记为可放行预警，不得直接视为失败
6. 若 `scene_lint.py --json` 结果命中 `章尾.warn / 解释.warn / 结构.warn / 情报.warn`，先人工复看，不得只因退出码为 `0` 就放行
7. 生成写后验收
8. 先做事实层回写前降级检查
9. 更新追踪主表
10. 更新角色卡
11. 生成反校验
12. 给出并入裁决

---

## 四、写前闸门最小模板

- 题材识别结果
- 本书类型判定
- 当前阶段爽法
- 本章标题类型
- 本章标题承载卖点
- 本章标题禁区
- 已挂载专项 checks
- 上一章核心母场景
- 本章主事件来源
- 本章世界主动施压载体
- 本章母场景避重复结果
- 本章跨场景过桥（如适用）
- 首章世界总盘交代（如适用）
- 首章世界底色显影锚点（如适用）
- 本章世界底色续显点（如适用）
- 最近三章主收益骨架对比结果
- 本章字数安全边际
- 本章中后段外部反应位
- 本章新增信息白名单
- 本章正文禁增项
- 本章主驱动
- 本章副驱动
- 本章最值钱战果
- 本章硬场面
- 本章第一爆点
- 本章高潮后续收益
- 本章外部现实变化
- 本章核心对手
- 本章核心物证或核心资源
- 本章核心场面载体
- 本章核心母场景
- 本章残魂 / 金手指出手类型
- 本章题面卖点兑现项
- 本章后 `1/3` 新的现实翻面
- 本章章尾钩子类型
- 本章专项失败风险
- 预判失败码

---

## 五、写后验收最小模板

- 场面数
- 外显战果
- 章尾判型
- 具象化检查结果
- 核心设定出手结果
- `L3/L4` 规模兑现结果（如适用）
- `scene_lint` 结果
- 纯净度结果
- AI味分级
- 角色卡同步结果
- 追踪同步结果
- 口径反校验结果
- 高价值明敌 / 高价值盟友首次锚点结果
- 首章题面另一半结果（如适用）
- 最终失败码
- 并入裁决

---

## 六、独立反校验

不得把“口径反校验通过”只写成写后验收中的一行。

必须单独落 `反校验_第XXX章.md`，至少检查：
- 正文 vs 细纲
- 正文 vs 写前闸门
- 正文 vs 追踪主表
- 正文 vs 角色卡

任一组未通过，不得并入主正文。

---

## 七、回炉最小复盘

若本章不是首写，而是回炉 / 重写，动笔前必须单独写出：
- 上轮主失败码
- 上轮主问题属于 `机制 / 章节 / 句子 / 流程` 哪一层
- 本轮最先要防止复发的旧问题
- 需要同步修改的项目文件列表

---

## 八、跨场景过桥最小硬规则

若本章相对上一章发生以下任一变化，默认必须写 `跨场景过桥`：
- 主场面地点变化
- 主对手 / 主势力变化
- 时间段变化（如白天切夜里、夜里切次日）
- 主事件载体变化（如公开争抢切祭台对峙、祭台对峙切营地内乱）

写前闸门与细纲中至少写清：
- 主角为什么必须去这个新场
- 主角靠什么信息 / 观察 / 现场余波锁到这个新场
- 主角怎么赶到或提前摸到
- 主角到场时，为什么正好撞上本章第一爆点

正文硬要求：
- 若命中跨场景切换，开章前 `100-200` 字内必须补足上述至少 `2-3` 项
- 不得只用“主角赶到时 / 主角来到 / 事情果然爆了”一笔带过
- 不得把“读者能猜到”当成免写理由

默认不过线情形：
- 上一章章尾只抛某处要出事，下一章直接写主角已在现场观战
- 上一章刚拿到账册，下一章直接切到新据点，没有交代锁定路径
- 只顾大场面开炸，不补主角行动链
