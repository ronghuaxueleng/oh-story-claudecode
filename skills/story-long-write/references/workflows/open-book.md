# 开书流程

> 本文件负责从零开书的最小执行顺序。详细参考仍由各类 reference 承接。

---

## 顺序

1. 确认题材方向
2. 判定小说类型与核心爽法
3. 产出核心设定
3.1. 若题材命中以下任一专项组合，必须先补世界底盘：
  - 高环境压力 / 路线推进
  - 超凡 / 异象 / 核心能力
  - 信息差 / 情报 / 预知
  - 权力博弈 / 阵营争夺 / 规则控制
  - 世界总矛盾
  - 世界发动机
  - 稀缺资源
  - 大环境 / 异象 / 资源系统
  - 势力 / 规则系统
  - 核心设定 / 金手指接口
  - 大事件池
3.1.1. 世界底盘不得只写抽象类别，必须额外补齐最小具象锚点：
  - 至少 `3` 个“会自己制造大事”的世界机制
  - 至少 `6` 个具名事件模板
  - 至少 `3` 个具名高价值资源点 / 权力节点
  - 至少 `2` 个具名势力接口人或组织接口
  - 若只能写出“某条路线 / 某股势力 / 某桩旧案 / 某个活口 / 某份物证”这类通用词，视为世界底盘未完成
3.2. 无论题材是什么，开书阶段都必须额外补一份“事件规模分层”：
  - 本书的 `L1-L4` 各对应什么类型的事件
  - 前 `3` 章准备使用哪些规模等级
  - 前 `10` 章准备如何避免长期停在低规模推进
3.2.1. 规模分层不得只打标签，凡标记 `L3/L4` 的事件，必须写清：
  - 群体规模或势力数量
  - 被改写的是路线 / 秩序 / 营地 / 资源 / 群体站位中的哪一项
  - 场面载体是什么
  - 若只写“发生混乱 / 更大压力到场 / 场面升级”，视为规模作弊
3.3. 若作品存在核心设定 / 金手指 / 身份差，必须先写清：
  - 该设定除了“解释世界 / 提供信息”之外，前 `10` 章将如何直接参与翻盘
3.3.1. 若核心设定是 `外部意识 / 情报能力 / 系统 / 血脉 / 特权身份`，前 `10` 章还必须补：
  - 至少 `3` 次“该设定直接造成的可见现实结果”
  - 不得只写“帮助判断 / 提醒 / 提供信息 / 帮助识别”
  - 必须至少写清 `3` 种可能兑现形态中的一种或多种，例如：
    - 直接改动作
    - 直接改判断
    - 直接改资源归属
    - 直接改关系站位
    - 直接改生死或去留
4. 落题材定位
5. 生成 `设定/爽点模型说明书.md`
6. 搭建卷纲
7. 搭建前 `10` 章细纲
7.1. 前 `10` 章规划前，必须先钉住“三锚点”：
  - `1` 个具名明敌
  - `1` 个具名摇摆角色
  - `1` 个具名资源 / 权力节点
7.2. 若三锚点缺任一项，不得进入前 `10` 章细纲
7.3. 开书阶段必须额外生成 `追踪/阶段审查计划.md`：
  - 不是写死“第几章后审查”，而是按“阶段”定义何时触发 `story-review full`
  - 每个阶段至少写清：
    - 阶段名
    - 阶段目标
    - 阶段开始 / 结束判据
    - 阶段必须完成的现实收益
    - 何时触发 `story-review full`
  - 审查触发条件必须写成“阶段完成 / 阶段翻面 / 阶段性风险出现”这类条件，而不是只写固定章节号
7.4. 若 `追踪/阶段审查计划.md` 缺失，不得进入正式日更顺写
8. 建立追踪主表
9. 进入第 `1` 章写前闸门
10. 写第 `1` 章正文
11. 写后验收
12. 同步追踪与角色卡
