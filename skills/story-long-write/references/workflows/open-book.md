# 开书流程

> 本文件负责从零开书的最小执行顺序。详细参考仍由各类 reference 承接。

---

## 顺序

1. 确认题材方向
2. 判定小说类型与核心爽法
3. 产出核心设定
3.1. 若题材命中以下任一专项组合，必须先补世界底盘：
  - 高环境压力 / 路线推进
  - 超凡 / 异象 / 核心能力
  - 信息差 / 情报 / 预知
  - 权力博弈 / 阵营争夺 / 规则控制
  - 世界总矛盾
  - 世界发动机
  - 稀缺资源
  - 大环境 / 异象 / 资源系统
  - 势力 / 规则系统
  - 核心设定 / 金手指接口
  - 大事件池
3.1.1. 世界底盘不得只写抽象类别，必须额外补齐最小具象锚点：
  - 至少 `3` 个“会自己制造大事”的世界机制
  - 至少 `6` 个具名事件模板
  - 至少 `3` 个具名高价值资源点 / 权力节点
  - 至少 `2` 个具名势力接口人或组织接口
  - 若只能写出“某条路线 / 某股势力 / 某桩旧案 / 某个活口 / 某份物证”这类通用词，视为世界底盘未完成
3.2. 无论题材是什么，开书阶段都必须额外补一份“事件规模分层”：
  - 本书的 `L1-L4` 各对应什么类型的事件
  - 前 `3` 章准备使用哪些规模等级
  - 前 `10` 章准备如何避免长期停在低规模推进
  - 前 `10` 章准备打哪 `2-3` 口高情节事件
  - 哪些章节负责把“小卡口”抬成“群体失控 / 路线改写 / 多方争夺 / 会死人会断路”的现场
3.2.1. 规模分层不得只打标签，凡标记 `L3/L4` 的事件，必须写清：
  - 群体规模或势力数量
  - 被改写的是路线 / 秩序 / 营地 / 资源 / 群体站位中的哪一项
  - 场面载体是什么
  - 若只写“发生混乱 / 更大压力到场 / 场面升级”，视为规模作弊
3.2.2. 开书阶段必须额外补一张 `高情节事件分布表`：
  - 事件名
  - 预计落章范围
  - 对应规模等级
  - 会改掉哪口现实盘面
  - 会逼出哪一类人物本性
  - 主角能抢到哪一口控制权
3.2.2.1. 开书阶段还必须额外补一张 `前10章事件链升级表`：
  - 章节范围
  - 本段共同抢的那口东西到底是什么
  - 第 `1` 章先失衡什么
  - 第 `2` 章新增哪一种非预期障碍
  - 第 `3` 章必须付哪一种代价才能兑现
  - 这一轮结束后，哪一口现实秩序真的被改了
  - 若三章之间只是“换地点继续做同类任务”，视为事件链未成立
3.2.3. 若前 `10` 章的分布表里只有：
  - 查证
  - 翻口
  - 抢单个物证
  - 改一次小规矩
  - 小范围对质
  而没有 `劫 / 烧 / 抢 / 围 / 追 / 逃 / 封 / 断 / 兽潮 / 暴动 / 哗变 / 抢门` 这类高情节事件，视为开书事件层未完成
3.3. 若作品存在核心设定 / 金手指 / 身份差，必须先写清：
  - 该设定除了“解释世界 / 提供信息”之外，前 `10` 章将如何直接参与翻盘
3.3.1. 若核心设定是 `外部意识 / 情报能力 / 系统 / 血脉 / 特权身份`，前 `10` 章还必须补：
  - 至少 `3` 次“该设定直接造成的可见现实结果”
  - 不得只写“帮助判断 / 提醒 / 提供信息 / 帮助识别”
  - 必须至少写清 `3` 种可能兑现形态中的一种或多种，例如：
    - 直接改动作
    - 直接改判断
    - 直接改资源归属
    - 直接改关系站位
    - 直接改生死或去留
4. 落题材定位
5. 生成 `设定/爽点模型说明书.md`
5.1. 生成 `大纲/开书候选分叉.md`（模板见 `references/artifact-protocols.md`）
5.1.1. 候选分叉默认不少于 `3` 组；每组至少拉开以下 `4` 项中的 `3` 项：
  - 主角身份壳
  - 世界发动机
  - 首章主事件
  - 第一明敌形态
  - 第一现实战果
5.1.2. 若 `3` 组候选之间只是换名字、不换骨架，视为候选分叉无效
5.2. 生成 `追踪/开书排异表.md`（模板见 `references/artifact-protocols.md`）
5.2.1. 至少先排异以下高频同构位：
  - 首章母场景
  - 首章翻盘模板
  - 主角职业 / 身份壳
  - 第一明敌出手方式
  - 第一现实战果形态
5.3. 生成 `追踪/同题材去重记录.md`（模板见 `references/artifact-protocols.md`）
5.3.1. 至少先对比项目内可见的同题材旧测试样本或旧开书样本
5.3.2. 若对比后仍与旧样本在 `主角壳 / 首章事件 / 第一母场景 / 第一明敌 / 第一战果` 中撞 `2` 项及以上，不得进入蓝图
6. 生成 `大纲/开书总蓝图.md`
6.1. `开书总蓝图` 无论题材为何，至少必须写清：
  - 这到底是个什么世界
  - 世界如何自己制造大事
  - 权力层级 / 势力盘 / 资源盘如何分布
  - 关键实体层如何分布：人物、势力、生物、植物、物品、地点、规则、事件
  - 世界信息分期投放表：哪些世界模块首章必须立底色，哪些在前 `3-10` 章持续显影，哪些到中后期再抬
  - 主角未来会跨越哪些区域、层级、盘面，而不是永远困在开局场景
  - 全书预计总字数、总卷数、每卷章数区间、每卷阶段任务
  - 主角成长梯度、敌阶梯度、规则翻面梯度
  - 事件升级梯度：前期怎么从小冲突抬到大事件，中期怎么从抢口抬到夺势，后期怎么从夺势抬到改盘
  - 情绪升级梯度：前期靠哪些 `快拿到 -> 被打断 -> 改决断` 的回摆拉人，中期靠哪些 `代价 / 误判 / 关系翻面` 抬压，后期靠哪些 `失控 / 牺牲 / 公开改判` 释放
  - 哪几个阶段必须触发 `story-review full`
6.2. 若蓝图里只有世界观名词、势力名词、卷名和口号，没有“层级如何运转、卷与卷如何升级、主角如何一路抬盘”，视为蓝图未完成
6.2.1. 若蓝图只写抽象世界和卷任务，没有为后续反复出现的实体建立资产表，也视为蓝图未完成
6.3. 若题材存在明显的世界规则改写、阶层跨越、长线经营、复仇夺权、升级打脸、势力争霸任一项，蓝图里还必须补：
  - `主角活动半径梯度`
  - `权力接触梯度`
  - `资源争夺梯度`
  - `敌人反制升级梯度`
6.4. `实体规划层` 至少按以下口径建立：
  - 人物层：主角轴、明敌轴、摇摆轴、盟友轴、接口人轴
  - 势力层：官面 / 军面 / 宗门 / 商会 / 帮派 / 地方豪强 / 地下链条等题材对应力量
  - 生物层：动物 / 异兽 / 坐骑 / 宠兽 / 灾异生物
  - 植物层：药材 / 毒植 / 灵植 / 粮食 / 燃料 / 环境标志物
  - 物品层：证物 / 兵器 / 禁物 / 身份物 / 关键资源 / 交通接口物
  - 地点层：城 / 关 / 矿 / 寨 / 港 / 驿 / 禁地 / 遗迹 / 市场 / 学宫等
  - 规则层：法律 / 禁令 / 修炼或科技规则 / 阶层边界 / 交易规则 / 交通规则
  - 事件层：战事 / 灾异 / 抢粮 / 夺矿 / 清洗 / 商战 / 试炼 / 政变等本书事件池
6.4.1. `事件层` 不能只列类别，至少还要补：
  - `高情节事件池`
  - `会死人 / 会断路 / 会改群体站位` 的事件名单
  - `主角最容易借机掌口 / 抢权 / 借刀` 的事件名单
6.5. 实体规划不是百科全书，只要求规划“会反复影响剧情”的实体：
  - 会反复出现
  - 会推动冲突
  - 会改变关系、资源归属或规则
  - 会承接伏笔或阶段收益
6.6. 一次性路人、一次性背景动物、纯气氛摆件、后续不再影响推进的小物件，不强制建档
7. 搭建卷纲
7.1. 每卷卷纲前，必须先从 `开书总蓝图` 拆出本卷边界：
  - 本卷处理哪一层敌阶
  - 本卷完成哪一级身份跃迁
  - 本卷改写哪一种现实规则
  - 本卷结束后主角会进入哪一个更大盘面
7.2. 若卷纲无法回答这四项，视为卷纲未过线
8. 生成 `大纲/全书章节总表.md`
8.1. 这不是重细纲，而是全书章节鸟瞰表
8.2. 至少覆盖：
  - 当前已规划的全部章节范围
  - 每章一句话核心事件
  - 每章主收益或主翻面
  - 每章 `本章世界显影点`
  - 每章服务哪条卷任务 / 哪个章组
8.2.1. `本章世界显影点` 不是设定抄录，而是“这一章要让读者实际看见世界的哪一块”：
  - 新机构 / 新资源口 / 新执行规则 / 新阶层关系 / 新区域 / 新灾异后果 / 新交通或交易口
  - 必须与本章主冲突绑定，不得写成独立背景说明
8.2.2. 若连续 `2-3` 章的 `本章世界显影点` 为空，或都只是同一块旧信息换词复述，视为章节总表不过线
8.3. 若当前只适合先规划前 `30-60` 章，也必须明确写出“已细排范围”和“后续待扩排范围”
9. 生成 `大纲/事件线总表.md`
9.1. 事件线总表按“线”写，不按“章”写
9.2. 至少拆出：
  - 主线
  - 核心副线（如复仇 / 势力争夺 / 感情 / 经营 / 灾异 / 升级等）
  - 世界事件线或大环境事件线（如适用）
9.3. 每条线至少写清：
  - 起点
  - 中段升级点
  - 卷级翻面点
  - 预计回收点
  - 对应章节范围
9.4. 若只有卷摘要、没有事件线总表，不得视为已具备全书总览能力
10. 为当前卷生成 `大纲/章组控制_第X卷.md`
10.1. 章组控制不是可选项；只要是长篇，默认都要有
10.2. 每个章组至少写清：
  - 章节范围
  - 阶段任务
  - 必拿战果
  - 必须出现的外部变化
  - 阶段世界显影任务：这一组章节必须让读者看见世界的哪几块，以及禁止提前抬哪几块
  - 禁止偏写点
11. 搭建前 `10` 章细纲
11.1. 前 `10` 章规划前，必须先钉住“三锚点”：
  - `1` 个具名明敌
  - `1` 个具名摇摆角色
  - `1` 个具名资源 / 权力节点
11.2. 若三锚点缺任一项，不得进入前 `10` 章细纲
11.2.1. 前 `3` 章涉及的关键角色卡必须先建：
  - 主角
  - 至少 `1` 个具名明敌
  - 至少 `1` 个具名摇摆角色 / 核心协作方
11.2.2. 若上述角色卡缺任一项，不得进入第 `1-3` 章细纲与正文
11.2.3. 关键角色卡不得只写身份简介，至少要写清：
  - 剧情职能
  - 公开目标
  - 私下最怕失去什么
  - 核心矛盾
  - 平时说话规则
  - 被压迫时说话规则
  - 被威胁时先护什么
  - 会下意识碰什么东西
  - 删名可分辨依据
11.2.1. 前 `10` 章细纲组还必须满足“世界显影连贯”：
  - 首章立世界底色与最低层规则入口
  - 第 `2-5` 章持续补“这世界怎么运转”
  - 第 `6-10` 章开始抬更大盘面或更高接口
  - 禁止把全部世界架构都堆进首章，也禁止首章后连续多章完全不扩世界
11.3. 开书阶段必须额外生成 `追踪/阶段审查计划.md`：
  - 不是写死“第几章后审查”，而是按“阶段”定义何时触发 `story-review full`
  - 每个阶段至少写清：
    - 阶段名
    - 阶段目标
    - 阶段开始 / 结束判据
    - 阶段必须完成的现实收益
    - 何时触发 `story-review full`
  - 审查触发条件必须写成“阶段完成 / 阶段翻面 / 阶段性风险出现”这类条件，而不是只写固定章节号
11.4. 若 `追踪/阶段审查计划.md` 缺失，不得进入正式日更顺写
12. 建立追踪主表
12.1. 建立 `设定/角色/` 目录与前 `3` 章关键角色卡
13. 进入第 `1` 章写前闸门
14. 写第 `1` 章正文
15. 写后验收
16. 同步追踪与角色卡
