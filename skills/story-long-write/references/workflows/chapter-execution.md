# 单章执行细则

> 这份主要写单章该按什么顺序读、怎么准备、写完后怎么硬查、中途怎么记情况。
> 章节最低要求的说法统一由 `references/chapter-gate-protocol.md` 定义。

---

## 一、进入正文前

1. 检查本章细纲是否存在
2. 读取上一章正文
3. 读取本章细纲
4. 读取相关追踪文件
5. 读取关键角色卡
6. 读取必要对标材料

---

## 二、准备层

### 1. 状态筛选

从 `追踪/角色状态.md` 与 `追踪/伏笔.md` 中筛出：
- 本章涉及角色当前状态
- 本章需要收回来 / 往前推的伏笔

输出最简记忆包。

### 2. 模块召回

回答三问：
- 本章目标情绪词是什么
- 借鉴哪个参考文件的哪个技法
- 用在哪些段落

### 3. 指令确认

用一句话确认本章节奏与情绪目标。

### 4. 额外单独检查确认

基于 `设定/题材定位.md` + 本章细纲，写清：
- 本章最容易滑向哪种失败
- 本章如何避免
- 碰上失败后应回退到细纲层还是正文层

### 5. 协议确认

进入正文前，还得额外确认：
- 本章书名主打卖点兑现项是什么
- 本章外部现实变化是什么
- 本章后 `1/3` 新的现实翻面是什么
- 本章高潮后续收益里，哪些是“本章已发生”，哪些只是“下章预期”；若分不清，不得开写
- 本章是否已有“事实层 / 预案层”混写风险
- 本章 `新增信息允许清单` 是否足够支撑第一爆点、第二轮收益和章尾；若不够，先回细纲补齐
- 本章 `正文禁增项` 是否已经写明；若没写，默认正文存在临时乱补风险，不得开写
- 若为首章，本章“书名另一半的第一轮压制”具体落在哪一拍
- 本章重要明敌 / 重要盟友首次强出场准备落哪 `2` 项稳定记忆点
- 本章所有写作说明词是否已经改成故事里的说法；若仍是术语本体，不得开写

若当前任务说法为 `H1-C 协同生产态`，还得先确认：
- 本章允许主线程即时介入的范围是什么：`细纲 / 正文 / 追踪 / 章组`
- 什么问题一旦碰上就不后拖：`一级禁用词 / 污染残片 / 作者直接替读者收尾的句子 / 生硬对白 / 章尾塌陷 / 场面断链 / 追踪说法错写`
- 即时介入后回到哪一环复检，避免“修了但没复跑”

---

## 三、首章外化规则

第 `1` 章如果以内心戏、设定认知或独处开场，必须先把内心变化外化成：
- 决定
- 误判
- 对话
- 物件变化
- 外部压力

不得用大段心理独白凑字数。

---

## 四、正文落稿前强制四问

- 本章谁被打疼了，疼在什么动作、什么代价、什么失态上
- 本章哪一步现实被改掉了
- 本章最抓人的场面具体落在哪一段
- 章尾最后一下是危险 / 发现 / 反扑，还是作者总结
- 删掉抽象词后，正文是否仍成立
- 是否偷用了抽象效果词 / 判断词 / 作者定性词代替场面
- 是否碰上本章额外单独检查

答不上来，就先别算过。

---

## 五、写后硬检

至少一条条确认：
- 删掉解释句后，场面还能不能成立
- 删掉抽象效果词后，场面还能不能成立
- 是否连续 `300-500` 字只有判断 / 解释 / 部署
- 是否存在配角替作者翻译局势
- 是否至少落下两层以上的“肉”
- 本章是否真的把这些额外单独检查落实进正文
- 本章解释句密度是否超阈值
- 本章后 `1/3` 是否滑成信息场
- 本章是否拿到了外部现实变化
- 本章是否把 `名字先挂上去 / 现场先得按这边来` 写得太满，硬写成 `这份好处已经当场起作用 / 好处已经真的落到手里`
- 本章是否出现 `第二层反压 / 第三方迟疑成本 / 直接把模板步骤喊出来 / 一章里把模板效果全写死 / 模板刚起手就立刻全生效 / 容易露馅的句子 / 容易露馅的一拍` 这类词
- 本章是否出现允许清单之外的新人物 / 新地点 / 新物证 / 新规则；若出现，是否已先补写纲层再继续
- 写后检查里填写的“高潮后续收益”是否全部来自本章正文，而不是下一章预期
- `章尾过了没有` 是否被误当成 `本章已经全过`
- 若为首章，本章是否只做到“活下来 + 敌人来了 + 线索拿到了”
- 本章重要明敌 / 重要盟友是否真正写够了第一次强出场该有的记忆点

若当前任务说法为 `H1-C 协同生产态`，补充要求：
- 写后硬检不只是负责“判病”，还要决定是不是当场就让主线程介入修本章
- 若问题已经可以在当前章内明确定位，不得默认后拖到 `story-review`
- 主线程即时介入后，得把本章对应脚本和检查重跑一遍，不能只人工读一遍就直接算过

---

## 六、禁用词与句子功能扫描

写后除了词表扫描，还必须检查：
- 程度副词 + 抽象效果词
- 单一力度副词是否在一章内重复顶动作，例如生硬撑劲词、空力度副词
- 先说气氛 / 结果，再补动作
- 作者结论句盖在场面后
- 角色只负责翻译局势

若碰上这些情况，即使禁词没报，也按 `F1/F2` 处理。

---

## 七、高潮兑现闸门

写前和写后都必须回答：
- 本章第一爆点是什么
- 本章高潮后续收益是什么
- 本章外显代价是什么
- 本章收益类型是什么
- 本章最后 `1/3` 是否还有新的现实翻面

补充要求：
- 只回答“活下来了 / 知道更多了”不算写够
- 必须说清书名主打卖点是不是这章真正的主要推进
- `高潮后续收益` 只能写当前章已经发生的收益，禁止填写“下一章可直接进入 / 已完成前置 / 已经布好局”
- 首章必须单独回答：`章尾过了没有` 和 `高潮兑现过了没有`；前者过了，不代表后者也过了

---

## 八、中途情况记录

每连续写完 `3` 章：
- 更新 `追踪/上下文.md`
- 用 `ls -la 正文/` 核对最近 `3` 个章节文件
- 发现异常立即重写入

---

## 九、事实层同步规则

追踪与角色卡更新时，必须按以下顺序：

1. 从正文抽取已发生事实
2. 过滤未发生的细纲/卷纲/预案内容
3. 再写入 `追踪/` 与 `设定/角色/`
4. 回写前逐条降级检查证据等级

禁止：
- 从细纲直接拷贝到追踪文件
- 从预案直接拷贝到角色卡“当前已落地事实”
- 把未来关系变化、未来伤情、未来证据提前写成当前事实
- 把“死前失口 / 高度疑似 / 指向线索”写成“活人供词 / 已确认 / 已主使 / 已执行”
- 把“第 N+1 章会发生什么”写成“第 N 章已经完成什么”
