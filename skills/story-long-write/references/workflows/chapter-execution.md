# 单章执行细则

> 本文件承接单章写作的详细读取顺序、准备层、写后硬检与中途快照。

---

## 一、进入正文前

1. 检查本章细纲是否存在
2. 读取上一章正文
3. 读取本章细纲
4. 读取相关追踪主表
5. 读取关键角色卡
6. 读取必要对标材料

---

## 二、准备层

### 1. 状态筛选

从 `追踪/角色状态.md` 与 `追踪/伏笔.md` 中筛出：
- 本章涉及角色当前状态
- 本章需要回收 / 推进的伏笔

输出最简记忆包。

### 2. 模块召回

回答三问：
- 本章目标情绪词是什么
- 借鉴哪个参考文件的哪个技法
- 用在哪些段落

### 3. 指令确认

用一句话确认本章节奏与情绪目标。

### 4. 专项 checks 确认

基于 `设定/题材定位.md` + 本章细纲，写清：
- 本章最容易滑向哪种失败
- 本章如何避免
- 命中失败后应回退到细纲层还是正文层

---

## 三、首章外化规则

第 `1` 章如果以内心戏、设定认知或独处开场，必须先把内心变化外化成：
- 决定
- 误判
- 对话
- 物件变化
- 外部压力

不得用大段心理独白凑字数。

---

## 四、正文落稿前强制四问

- 本章谁被打疼了，疼在什么动作、什么代价、什么失态上
- 本章哪一口现实被改掉了
- 本章最硬的场面落在哪一段
- 章尾最后一下是危险 / 发现 / 反扑，还是作者总结
- 删掉抽象词后，正文是否仍成立
- 是否偷用了抽象效果词 / 判断词 / 作者定性词代替场面
- 是否命中已挂载专项 checks

答不上来，默认不过线。

---

## 五、写后硬检

至少逐条确认：
- 删掉解释句后，场面还能不能成立
- 删掉抽象效果词后，场面还能不能成立
- 是否连续 `300-500` 字只有判断 / 解释 / 部署
- 是否存在配角替作者翻译局势
- 是否至少落下两层以上的“肉”
- 本章是否真正执行了专项 checks

---

## 六、禁用词与句子功能扫描

写后除了词表扫描，还必须检查：
- 程度副词 + 抽象效果词
- 先说气氛 / 结果，再补动作
- 作者结论句盖在场面后
- 角色只负责翻译局势

若命中，即使禁词未命中，也按 `F1/F2` 处理。

---

## 七、高潮兑现闸门

写前和写后都必须回答：
- 本章第一爆点是什么
- 本章高潮后续收益是什么
- 本章外显代价是什么
- 本章收益类型是什么
- 本章最后 `1/3` 是否还有新的现实翻面

---

## 八、中途快照

每连续写完 `3` 章：
- 更新 `追踪/上下文.md`
- 用 `ls -la 正文/` 核对最近 `3` 个章节文件
- 发现异常立即重写入
