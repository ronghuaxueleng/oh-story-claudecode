# 单项样本怎么写

> 把“单项负例/单项样本”的经验收成一份能直接照着搭的写法。后面再做冷启动单项样本时，不用临时硬想，照着这份往下搭就行。

---

## 一、适用范围

这份先覆盖四类单项样本：

1. `非情报流误触`
2. `本组第二章偏短链`
3. `本组第二章最后半拍回弹不足`
4. `中段直接把意思说破的句子拦截`

这四类样本的共同目标不是“写成成品”，而是：

- 把主问题做纯
- 让 `story-review full` 能稳定抓到正确主结论
- 验证 skill 的默认判断顺序是否真实生效

---

## 二、统一走法

每做一本单项样本，都按下面顺序走：

1. 先定主问题
- 只能有一个第一主问题
- 例如：
  - `非情报流误触`
  - `本组第二章偏短链`
  - `本组第二章最后半拍回弹不足`

2. 再定“别抢主结论”的噪音项
- 常见噪音：
  - `字数.error`
  - `章尾.error`
  - `解释.error`
  - `流程.error`

3. 再搭最小项目主结构
- `设定/题材定位.md`
- `设定/世界总控表.md`
- `设定/角色/*.md`
- `大纲/开书总蓝图.md`
- `大纲/细纲_第001章_*.md`
- `大纲/细纲_第002章_*.md`
- `正文/第001章_*.md`
- `正文/第002章_*.md`
- `追踪/写后检查_第001章-第002章.md`
- `追踪/时间线.md`
- `追踪/角色状态.md`

4. 首写后立刻跑脚本
- `scene_lint.py --json`
- `validate_tracking_state.py --json --legacy-compat`

5. 跑完脚本后主线程还得自己再看一遍
- 别只看脚本标签
- 得判断：
  - `脚本候选`
  - `主线程第一眼怎么判`
  - `现在主要噪音`

6. 最后落一张单项复看单
- 重点只写：
  - 第一眼怎么判
  - 为什么不是别的结论
  - 现在还剩哪些噪音

---

## 三、非情报流误触协议

### 1. 先把主问题钉住

主问题是：

- `旧码 / 错单 / 复核口 / 广播补条 / 口令 / 挂签 / 规则播报`

这些现场东西在正文里的主功能，是：

- 改现场先后
- 改谁先背责
- 改谁先记名
- 改谁先停手

而不是：

- 独立情报发布
- 情报刷新
- 情报兑现链

### 2. 题材优先

- `都市脑洞 / 职场执行 / 港区调度 / 规则副本`

### 3. 首写要求

- 第001章得先把“旧说法正在执行”写出来
- 主角得用“信息凭据”当场卡住旧先后
- 本组第二章得继续围绕同一件东西改现实，别突然转成解释新情报
- 先禁用：
  - 独立情报引号块
  - 明示 `情报刷新`
  - 情报记录必填项前置

### 4. 最少得写到什么程度，人才看得清

不能只剩：

- 一个概念
- 两句争辩
- 一张具体东西的名字

至少要有：

- `1` 个第三方被动动作
- `1` 次谁前谁后改线
- `1` 个器物/屏幕/灯牌反馈

不然就算主问题方向对，脚本和人工也很容易先被“太薄”带走。

### 5. 回头怎么判

若脚本先报 `情报.error`：

- 先当成：`脚本候选`
- 然后继续往下看：
  - 这是正式情报流
  - 还是 `非情报流误触候选`

只要正文主要推进是“借具体东西改现场”，主线程第一眼就先写：

- `非情报流误触候选`

---

## 四、本组第二章偏短链协议

### 1. 先把主问题钉住

主问题是：

- 第001章已经证明模板主线进正文
- 本组第二章却被压缩成：
  - 主角顶一句
  - 对手回一句
  - 那个重要角色再补一下压力
  - 记录口直接落签

### 2. 题材优先

- `古代断案 / 公开场争位 / 都市商战 / 规则争签`

### 3. 首写要求

- 第001章得相对完整，至少证明：
  - 第一爆点
  - 第二步收益
  - 外部反应
  - 重要角色出场
- 本组第二章可以短，但只能短在“四拍主链”，别短成纯残稿

### 4. 最少得写到什么程度，人才看得清

第001章至少得像一个完整开场。

本组第二章至少要有：

- 四拍主链本身
- 少量现场反应
- 至少一个器物/牌架/签架/记录口反馈

不要求把本组第二章扩厚成完整翻面章，但得避免只剩四五句主线摘要。

### 5. 回头怎么判

若脚本报 `字数.error`：

- 先当成噪音项
- 然后继续往下看：
  - 第001章是否已证明模板落地
- 本组第二章是不是明显只剩偏短链

若两者都成立，就先这样判：

- `本组第二章偏短链`

别先回退成：

- `模板未落地`
- `只是残稿`

---

## 五、本组第二章最后半拍回弹不足协议

### 1. 先把主问题钉住

主问题是：

- 第001章已经证明模板主线进正文
- 本组第二章的 `记名 / 落签 / 改先后 / 外部看法改变` 已在章尾前落成
- 但最后 `1-3` 句没有再压一下、再翻一面、再挂一个小钩子

这类问题不是：

- `本组第二章偏短链`
- `整章兑现不足`

而是：

- `章尾收束偏顺`
- `最后半拍回弹不足`

### 2. 题材优先

- `古代断案 / 公开场争位`
- `玄幻公开场夺序`
- `都市规则争签`

### 3. 首写要求

- 第001章得先把模板主线写出来
- 本组第二章中前段得已经写出：
  - `现实收益落地`
  - `至少一次外部反应`
  - `至少一项器物/牌架/记录口反馈`
- 故意只把最后 `1-3` 句收弱，别把整章一起写塌

### 4. 最低能看清结果的场面线

本组第二章至少要有：

- `1` 次明确落签 / 落字 / 记名 / 改先后
- `1` 次第三方被动动作或谁前谁后改线
- `1` 个可见反馈物

如果缺这些基础项，那就不是“最后半拍回弹不足”，而是整章还太薄。

### 5. 复核说法

若脚本报 `章尾.error`：

- 先当成噪音项
- 然后继续往下看：
  - `最主要收益是否已在章尾前落成`
  - 问题到底是 `最后半拍偏顺`
  - 还是 `整章兑现不足`

只要最主要收益已经成立、问题只在最后半拍，就先这样判：

- `本组第二章最后半拍回弹不足`

别先回退成：

- `本组第二章偏短链`
- `F2 级整章兑现不足`

---

## 六、中段直接把意思说破的句子拦截协议

### 1. 主问题定义

主问题是：

- `重要角色先付代价再压住别人` 已经成立
- 但正文中段仍继续补：
  - `我若先把自己人……谁还认……`
  - `我若先护自己人……谁还认……`
  - `我若先把自己人塞进……谁还认……`

这类句子的问题不是“规则没成立”，而是：

- 规则已经能靠动作 / 被挤开 / 把话收回去 / 执行后果自己撑住
- 作者却又把规则意义直接翻出来讲了一遍

### 2. 题材优先

- `校园规则`
- `都市调度`
- `民俗规则`
- `灾夜秩序`

这些题材最容易稳定复发：

- 中段直接把意思说破的句子
- 章尾已经很硬，但人物这条线还只写到中间那档

### 3. 首写要求

- 第001章得先证明：
  - 重要角色已经不是纯粹出来顶场转场的人
  - 已经出现 `先压自己人 / 自己物 / 自己名分`
- 第002章得继续证明：
  - 她/他自己真实失去了一样东西
  - 别人因此被逼着跟着改
- 同时先禁用：
  - `我若先把自己人……谁还认……`
  - `我若先护自己人……谁还认……`
  - `我若先把自己人塞进……谁还认……`

### 4. 最低能看清结果的场面线

别只剩：

- 1 句重要角色关键台词
- 1 次往后挪先后
- 1 个章尾封口

至少得有：

- `1` 次真实后挪 / 后划 / 后排
- `1` 次记录口 / 墙板 / 簿册 / 白板改记
- `1` 次对手被迫松手 / 把话收回去
- `1` 个章尾执行后果

不然就算方向对，也还是不足以证明“删掉那句把意思说破的话后，规则还能自己站住”。

### 5. 复核说法

若正文碰上：

- `我若先把自己人……谁还认……`
- `我若先护自己人……谁还认……`
- `我若先把自己人塞进……谁还认……`

先判清：

1. 删掉它后，规则是否仍成立
2. 若仍成立，它就不是“人物成立关键句”
3. 而是：
- `中段直接把意思说破的句子`
   - 一般先拦在“中间这档”

只要这三步成立，就先这样判：

- `规则成立，但被中段直接把意思说破的句子卡在中间这档`

## 七、统一噪音处理规则

单项样本最常见的三类噪音：

1. `字数.error`
2. `章尾.error`
3. `解释.error`

补充判断：

- 若 `章尾.error` 报的只是本组第二章最后半拍回弹不足，先当成：
  - `章尾收束问题`
  - 别先判成 `本组第二章偏短链`
  - 也别直接升为 `F2`

处理顺序：

- 先判主问题是否已经成立
- 再判噪音有没有抢走主结论
- 最后再决定是否补量

这里先按这个看：

- `判断顺序能纠偏，但不能替代最低能看清结果的正文厚度`
- 样本太薄时，不要误以为是规则没生效
- 先补到“够看清结果”，再看主结论是否稳定

---

## 八、最后输出怎么写

单项样本复核时，统一输出四句：

1. `第一眼该怎么判`
2. `为什么不是另一个相邻结论`
3. `现在主要噪音是什么`
4. `样本是不是已经到了最纯验证态`

示例：

- 第一眼怎么判：`非情报流误触候选`
- 不是正式情报流：因为这件东西的主功能是改现场先后，不是发独立情报
- 当前噪音：`字数.error + 少量解释.error`
- 状态：`方向对，但还没到最纯验证态`

---

## 九、这一阶段怎么看

当前单项样本测试链已经能比较稳地给出三类判断：

1. `规则是否生效`
2. `主问题是否成立`
3. `还差多少才到最纯验证态`

后续若继续做单项样本，一般先读本协议，再决定是否补正文。
