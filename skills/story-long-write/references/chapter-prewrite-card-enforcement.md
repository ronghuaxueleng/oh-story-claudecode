# 单章写前卡先按这几条拦

只要任务碰上下面这些情况：
- 开写正文
- 续写
- 回炉重写
- 用户明确说“不想再写完后回炉”

那当前章下笔前，先得有下面两种里的一个：
- 项目内已有该章 `单章写前卡`
- 或按同等内容先补一份专用写前卡

最少得答出来这些：
- 本章要办的事
- 本章现场到底改了什么
- 场景怎么拆
- 第一爆点怎么起
- 压住别人的动作
- 群像怎么把场面推大
- 主角这章不能用哪种说话味道
- 哪些动作要和关键台词绑在一起
- 章尾停在哪一下
- 黑名单复查结果

最主要的项里少一个，就先别进正文。

## 顺序别弄反

正文前面的顺序按这个来：
1. 把这章要发生什么先定实
2. 再把顺序排实
3. 再表演
4. 后风格
5. 最后黑名单复查

别用这种错误顺序：
- 先语录，再嘴炮，再补情节
- 先看参考原文同章，再边看边改再往正文里塞

## 这种污染先拦住

下面任一条撞上了，最少 `F1`，严重时 `F2`：
- 只有嘴上放话，没有现实局面被改掉
- 只有主角输出，没有群像承接
- 只有概念，没有动作和后果
- 删掉修仙名词后还能一眼看出是参考书主角在说话
- 主角连续三次都用现代经营 / 治理 / 咨询那套说法压人

## 提纲术语先拦住

若写前卡、写前闸门、参考输入卡里把“抽象控制词、流程摘要词、抢先后次序词、谁先开口定事这类总括词”直接拿来当本章主收益词、主爽点词或主任务词，就先回到写前层重写，再进正文：

改写时按这个来：
- 别写抽象控制词，改写成动作结果词
- 先写清：
  - 谁先动手
  - 谁先分货
  - 谁先补灯
  - 谁先放人
  - 谁被迫认账
  - 谁失去先手
  - 谁先吃亏
  - 谁先退
  - 谁先交东西
  - 谁先让地方

反例：
- `本章要办的事：把堵门扣车写成谁先办、谁先退，当场换了人`
- `本章后 1/3 再补一层更重的动作`
- `本章最猛的收获：主角拿到谁先拍板的那一下`

正例：
- `本章主线任务：把堵门扣车写成边军被迫按主角验出来的顺序先补灯、再放路`
- `本章后 1/3：让对手先拆不敢拆、先认不敢认，最后只能照新顺序动手`
