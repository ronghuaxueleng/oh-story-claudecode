# `writing-red-lines.md`：长篇正文红线总表

本文件不是灵感参考。

本文件是执行红线。

命中任一红线时：
- 不得继续顺写
- 不得并入主正文
- 不得只做词语替换式修补
- 不得跳过 `scripts/scene_lint.py` 的写后质检

---

## 一、正文禁止生成顺序

### 红线 1：禁止效果词先行

错误顺序：
- 先想“这一段要狠、要炸、要压、要有气势”
- 再用抽象效果词、判断词、气氛词去顶正文

正确顺序：
1. 谁在动
2. 动作打在何处
3. 造成什么可见后果
4. 对方如何失态、受伤、变形、退缩、反扑

以上四项没落地前，不准写效果判断。

判死标准：
- 删掉效果词后，段落立不住
- 说明动作层、后果层、疼感层没有写出来

### 红线 1.5：禁止用口头发力词起稿

错误顺序：
- 先想“这里要狠一点、猛一点、炸一点”
- 再用口头发力词或空力度副词把句子顶过去

正确顺序：
1. 先写谁冲上去
2. 再写打在什么部位
3. 再写人或物怎么偏、怎么断、怎么翻、怎么退
4. 最后才允许极少量补强语气

硬规则：
- `references/banned-words.md` 中的 `一级禁用词` 属于首稿禁止生成项，不是写后再删项
- 口头发力词若承担动作主功能，正文起草时不得出现
- 二级敏感词若承担了动作主功能，也视为失手词

判死标准：
- 去掉该词后，句子只剩“有人动了”而没有受力点、位移、伤害结果
- 同一段或相邻两段连续靠这类词顶动作
- 该词承担了本句主要力度来源

唯一正确修法：
- 把力度拆回 `动作路径 + 受力部位 + 位移结果 + 失态反应`
- 不允许只换成别的近义力度词

### 红线 2：禁止作者先知道“结构”，再把结构话漏进正文

错误顺序：
- 先想“这里要撕开军权线”“这里进入第二阶段”“这里是第一卷节点”
- 再把这种后台判断漏成正文句子

正文只能承载：
- 人物当下看到的
- 人物当下听到的
- 人物当下感觉到的
- 人物当下做出来的

正文不能承载：
- 卷结构判断
- 章节功能判断
- 线索调度判断
- 写作复盘判断

### 红线 2.3：禁止把正文写成事件记录，不写成主角当下经历

错误顺序：
- 先按作者视角记“发生了什么”
- 再把人物塞进这些事件里走流程

正确顺序：
1. 主角这一瞬先注意到什么
2. 主角因此误判 / 起疑 / 改主意什么
3. 主角先护什么、先舍什么、先做什么
4. 外界再因此发生什么变化

硬规则：
- 正文必须优先按 `主角感知顺序` 落地，不得优先按 `作者说明顺序` 落地
- 若一段删去主角的感知、误判、改招后仍完全成立，说明这段更像事件记录，不像人物正在经历

判死标准：
- 连续 `3` 句及以上都在写“谁做了什么、然后又做了什么”，却没有主角当下的注意力偏移、判断变化、临场取舍
- 段落能回答“发生了什么”，却答不出“主角是先看见、先听见、先怀疑、先决定了什么”
- 读者读完像看值班记录、战报、行程单，而不是像被按进现场

唯一正确修法：
- 把总述拆回主角这一刻最先抓到的东西
- 把动作链拆回“受阻 -> 改招 -> 现实变化”
- 把信息后移到主角真正拿它做决断的那一拍

### 红线 2.4：禁止让制度术语直接顶正文理解

错误顺序：
- 先报一个制度词、职业词、流程词
- 让读者自己去猜它现在压谁、放谁、卡谁

正确顺序：
1. 先让读者看见它正在卡谁或放谁
2. 再顺手知道这口东西叫什么
3. 若需要术语，也只保留最小必要量

判死标准：
- 一个词本身不难，但放在这句里要读者停下来翻译
- 删掉术语换白话后，句子更顺、冲突更清
- 一句里连报多个制度口名词，人物却还在急场里说得很稳

唯一正确修法：
- 把术语拆回 `谁先走 / 谁被卡 / 谁得掏钱 / 谁会死`
- 把制度词挂到具体承载物、次序变化、执行动作上
- 若术语只是显得专业，直接删或换白话

## 二、正文禁止句型

### 红线 3：禁止后台调度语

正文不得出现：
- `这一卷/这一段/到这一步`
- `终于把某条线撕开`
- `真正进入某个阶段`
- `现在的问题是`
- `接下来就是`

这些不是现场，是后台写作语言。

### 红线 4：禁止作者盖章句

正文不得靠这些句子收口或定性：
- `这意味着……`
- `这说明……`
- `表明……`
- `证明……`
- `显示……`
- `从这一刻起……`

如果动作、对白、反应已经成立，就不准再补一句作者结论。

### 红线 4.5：章尾最后一句禁止承担“解释它为什么重要”的功能

章尾最后一句只能停在：
- 新动作
- 新物证
- 新口径
- 新名单
- 新翻面
- 下一步命令

章尾最后一句不得承担：
- 意义总结
- 局势翻译
- 作者判价
- 资源判价
- “这意味着 / 这说明 / 真正要命的是 / 现在真正该抢的是” 这类盖章功能

判死标准：
- 删掉最后一句后，章尾更利索、更有力
- 最后一句主要作用是告诉读者“刚才那一下为什么重要”
- 最后一句没有新增现场结果，只是在总结前一拍

唯一正确修法：
1. 先回看最后一句前，现场是否已经有独立成立的结果
2. 若已成立，直接删掉最后一句
3. 若未成立，不补解释句，回前一拍补动作 / 物证 / 他人失态 / 现实变化

### 红线 5：禁止抽象效果句替代现场

正文不得主要依赖：
- `压迫感很强`
- `杀气逼人`
- `气势惊人`
- `场面可怕`
- `危险至极`
- `爽感炸开`

这些只能极少量辅助，不能承担主体表达功能。

### 红线 5.5：正文叙事句禁止让空效果表达承担主功能

以下表达一旦承担了句子主功能，就判失败：
- 抽象强弱词
- 抽象气势词
- 抽象危险词
- 抽象速度词
- 抽象判断词

判定方式：
- 删掉这些表达后，句子只剩空架子
- 说明动作、后果、失态、物理变化没有写出来

命中后不允许：
- 只删词
- 只换近义词
- 只缩短句子

唯一正确修法：
- 回到动作层
- 回到后果层
- 回到受伤/失态/位移/声响层
- 把偷懒表达替换成可见事件

### 红线 6：禁止提纲扩写句

正文不得出现：
- `真正的……`
- `值钱的是……`
- `不是X，而是Y`
- `原来……`

如果这些句子在一段里承担主要翻面功能，说明正文仍停留在提纲层。

### 红线 6.4：禁止对白和叙述同时落成书面制度口

正文与对白若同时出现以下情况，默认至少 `F1`：
- 叙述句在报制度词
- 对白句也在解释制度词
- 现场动作反而没有先落地

这类写法读起来最像“作者已经懂了，所以让角色帮忙讲给读者听”。

唯一正确修法：
1. 先删最晚出现的解释句
2. 回前一拍补动作、物证、站位、受阻结果
3. 若仍要保留术语，只留一次，并挂到当场代价上

### 红线 6.1：禁止旁白替作者发力

正文叙述句不得承担以下功能：
- 替读者下判断
- 替场面下结论
- 替人物发狠
- 替章节收金句
- 替冲突翻译“这一拍到底爽在哪”

高危表达包括但不限于：
- `先乱的不是A，是B`
- `最值钱的不是A，是B`
- `不是在X，是在Y`
- `不是……，而是……`
- `因为这不是……，是……`
- `这话不是在……，是……`
- `真怕……的，不是……`
- `真正可怕的是……`
- `真正要命的是……`
- `现在该抢的不是……`

说明：
- 以上句型不是一律禁词
- 但只要它们承担了句子主功能，默认就是旁白替作者发力
- 若删掉这句后，现场动作、失态、站位、物证仍成立，说明它本来就在做“作者发言”

判死标准：
- 删掉该句后，段落更利索、更像现场
- 该句没有新增动作、物理变化、利益变化，只是在重新翻译前一拍
- 该句的快感主要来自句型本身，而不是来自现场已经成立的现实变化
- 同章命中 `2` 句及以上同类发力句，默认至少 `F1`
- 若命中位置在章首开场、章中关键对顶、章尾收口，默认优先升格复检

唯一正确修法：
1. 把该句想表达的“判断”，拆回前一拍动作
2. 把该句想表达的“价值”，拆回谁先护什么、谁先退什么
3. 把该句想表达的“爽点”，拆回外部反应、现实改盘、敌方失态
4. 不允许只把句型换成近义句继续发力

优先替代承载物：
- 谁先伸手去护牌、票、账、令
- 谁先往后退半步、卡半步、改口半句
- 围观的人先往哪边挤
- 器物先怎么响、怎么偏、怎么卡住
- 规则先压到了谁、又先放过了谁

### 红线 6.3：细节只能增厚，不能代替事件

用于增厚正文的细节，只能承担以下辅助功能：
- 把人物动作落点写清
- 把物件状态变化写清
- 把环境压力怎样压到当前动作上写清
- 把身体先泄露出来的反应写清
- 把规矩、职业习惯、群体常识写到现场里

这些细节不得承担以下主功能：
- 代替新事件
- 代替新阻力
- 代替新代价
- 代替现实改盘
- 代替人物真正出手

判死标准：
- 本段删掉细节后，事件没有任何损失，只说明原本就在空转
- 本段只有环境、神态、气味、声响，却没有动作推进、站位变化、利益变化
- 所谓“补字数”主要靠环境句、心情句、抒压句，而不是靠冲突链扩展
- 一章里连续出现多段“看起来有画面，实际上没改局”的细节堆叠

唯一正确修法：
1. 先补事件：谁拦、谁抢、谁改口、谁失手、谁被逼站位
2. 再补冲突：明面抢什么，暗里护什么
3. 最后才允许补细节：把动作落点、物件状态、环境压力、身体反应写清

可用的细节承载物：
- 被捏软的票角 / 牌角 / 名册边
- 门轴、风索、锁扣、栏杆、印泥、旧伤、污痕
- 排队姿势、护物动作、看向哪里、退半步还是逼半步
- 与规则直接相关的声响、器物磨损、职业习惯、群体常识

禁用误区：
- 用细节代替解释句之后，又把整段写成空镜头
- 用细节补世界观，却不让这条世界规则改动作、改生死
- 用细节补人物，却只写“皱眉、发抖、沉默、脸白”这类通用反应

### 红线 6.4：禁止平均铺叙，关键一秒不得被总述抹平

正文不得把最值钱的瞬间写成：
- `他看了一眼`
- `众人顿时明白`
- `局势一下变了`
- `双方很快交手`
- `然后事情就翻了`

说明：
- 这些句子不是不能出现
- 但若它们承担关键翻面、关键受阻、关键立威的主功能，默认就是把戏写成了流水账

判死标准：
- 关键翻面只用 `一下 / 顿时 / 很快 / 立刻 / 转眼` 一类时间词带过去
- 本应最有代入感的一拍，反而是段落里最抽象、最短、最像摘要的一拍
- 删掉某个总述句后，发现根本没有补上对应近景动作、器物变化、失态反应

唯一正确修法：
1. 找出本段最值钱的 `1-2` 秒
2. 把它拆成：
   - 谁先碰到什么
   - 什么先响、先偏、先断、先停
   - 谁先失手、先退、先改口
3. 不允许继续用总述句代替

### 红线 6.7：解释句不得只按词面处理，必须继续判根因

命中解释腔 / 盖章句 / 作者代判句时，不得只写“这句太解释”。

必须继续判到以下四类之一：
- `G1 信息抢跑`：正文抢着替读者说明设定、局势、价值
- `G2 现场承载不足`：动作、后果、失态、现实收益没写够，只能靠判断句补缝
- `G3 提纲语言漏出`：章节意图、结构调度、线索调度直接漏进正文
- `G4 人物失声`：本该由人物动作 / 对话承载的内容，被旁白替说了

规则：
- `G1 / G4` 默认至少句子层失败
- `G2` 默认优先判场面承载失败，不得只删句
- `G3` 默认优先判提纲语言漏出，不得只换词

若不能判到 `G1-G4` 之一，说明仍停留在现象层，不得结束回修。

### 红线 6.72：禁止 POV 松掉成“监控视角”

正文若采取有限视角，就不得随手滑出当前人物的当下感知边界。

默认不过线情形：
- 主角背对某人，却先写清对方脸色和完整心思
- 一段里不断切换“这边看到什么、那边同时怎么想、围观者又整体怎么判断”
- 场景描写像俯拍总览，谁都看见了，唯独没有主角自己的注意力顺序

判死标准：
- 删掉人物名字后，整段像无主镜头扫场，不像某个人在现场里承压
- 读者更像站在天花板上看混乱，而不是站在主角肩后吃风、吃压、吃误判

唯一正确修法：
- 锁回当前人物：
  - 先看到什么
  - 先听到什么
  - 先误会什么
  - 先决定什么
- 其他信息只能通过动作、声音、器物、结果倒推出来

### 红线 6.75：旁白发力句不得伪装成“有劲的开场句”或“有钩子的收束句”

以下场景默认优先抽查：
- 章首前 `200` 字
- 第一爆点前后
- 主角与高位敌人第一次正面对顶时
- 章尾最后 `5-15` 行

默认不过线情形：
- 章首靠一句“先乱的不是……是……”硬拉气势
- 章中靠一句“这不是在……，是在……”硬翻译冲突
- 章尾靠一句“真正要命的是 / 真正值钱的是……”硬抬钩子
- 同一章里，旁白比人物更会说、更会下判、更会收句

修法要求：
- 开场要回到现场载体：谁挤、谁卡、谁先掉、谁先喊
- 中段要回到冲突载体：谁护物、谁改口、谁失手、谁被迫换站位
- 章尾要回到结果载体：新动作、新物证、新名单、新口径、新翻面

### 红线 6.8：人物算盘不得主要靠旁白直译

正文不得主要依赖以下写法来说明人物“像活人”：
- `他最怕的是……`
- `他真正想要的是……`
- `他不是为了X，而是为了Y`
- `他不是想帮谁，他只是……`
- `他终于算明白了……`
- `他怕的不是A，是B`

这些句子可以极少量存在，但不得承担人物成立的主功能。

判死标准：
- 删掉这句后，角色就只剩身份和剧情功能
- 该角色的欲望、损失口、私心、站位，没有通过动作 / 台词 / 停顿 / 失手 / 回避自行露出来
- 同一角色在同一章里反复靠这类句子补“他到底在想什么”
- 同一角色在同一章里命中 `2` 句及以上同类直译句，默认至少按人物显影不过线处理

唯一正确修法：
1. 先把人物这场最想拿到什么，转成他正在护的具体东西
2. 再把人物最怕失去什么，转成身体先泄露出来的反应
3. 再把人物真正站位，转成一句带回避、转移、压价、甩锅、装硬的台词
4. 能让读者看懂时，不再补旁白翻译

优先替代承载物：
- 手里反复碰的物件
- 说到关键处突然改口
- 明明想往前，却先退半步
- 嘴上压人，动作却先护住别的东西
- 沉默、岔话、跳答
- 抢先去挡 / 去删 / 去藏 / 去压的那一下
- 被点破后先护住的门卡 / 名册 / 护腕 / 算盘 / 钥匙 / 芯片

根因优先判定：
- 多数命中优先判 `G4 人物失声`
- 若该句同时在替整段补场面，则再加判 `G2 现场承载不足`

### 红线 6.5：禁止资源判价句 / 作者替读者算账句

正文不得靠以下句型来替代“资源升级已经通过现场成立”：

- `更值钱的是……`
- `更硬的是……`
- `更实的是……`
- `这一口更……`
- `比……更……`
- `现在真正该抢的是……`
- `最该抢的已经不是……`
- `他终于明白，真正值钱的……`

尤其当这些句子承担了：
- 资源升级说明
- 爽点翻面说明
- 章节收益总结
- 章尾钩子主功能

时，直接判失败。

原因：
- 这类句子本质上仍是作者在替读者算账
- 它们会把“资源升级 / 规则升级 / 爽点升级”从现场动作重新打回提纲口吻
- 高频出现时，会快速形成明显 AI 味和模板味

唯一正确修法：
1. 先写资源到底怎么换了归属
2. 再写谁因为这口资源改站位、改动作、改口
3. 再写谁因此活下来 / 被压下去 / 被挤出去
4. 不允许再补一句“所以这口更值钱 / 更硬 / 更实”

合格方向示例：
- 不写：`这口收益比夜票更实`
- 改写成：`夜票只能让他挤上车。可刚才这一推一挤，活着站进前厢的人，已经真的换了。`

- 不写：`真正值钱的是下一站谁先下车`
- 改写成：`站外那七个冷黄站位只亮了七道，多一个人挤过去，后面就得有人被压回门里。`

## 三、正文失败后的回退顺序

### 红线 7：禁止只删词，不回句子功能

发现口号腔、解释腔、后台句时，回修顺序固定为：
1. 先看句子功能是不是错了
2. 再看场面层是不是没写够
3. 再看细纲是不是承载失败

不允许：
- 只删单个词
- 只换近义词
- 只把句子缩短

### 红线 7.5：写后必须跑正文场面质检

固定动作：
- 正文落盘
- 运行 `scripts/scene_lint.py <正文文件>`
- 按命中结果整句或整段回修
- 复扫通过后才能并入正文

如果没跑脚本，默认视为未验收。

### 红线 8：句子层、章节层、机制层不能混修

句子层：
- 解释句
- 作者盖章句
- 抽象效果句

章节层：
- 场面不足
- 爽点不落地
- 围观层/阻力层/代价层缺失

机制层：
- 题面滑轨
- 连续多章只查不兑现
- 副驱动反客为主

规则：
- 句子层才允许局改
- 章节层必须补场面和章肉
- 机制层必须回细纲或连续章节组

## 四、执行自问

写正文前，先问：
1. 这段是不是先有动作，再有感受？
2. 这段删掉效果词后还成立吗？
3. 这段有没有作者站出来替读者总结？
4. 这段是不是人物现场，而不是提纲后台？

写正文后，再问：
1. 谁赢了？
2. 谁亏了？
3. 谁被打疼、打丢脸、打失态？
4. 现实盘面改掉了哪一块？
5. 删掉最后一句，章尾会不会更强？
6. 删掉最后两句，章尾现场还成立吗？
7. 最后一句是在新增现场，还是在解释前面的现场？

任一问答不上来，默认不过线。
