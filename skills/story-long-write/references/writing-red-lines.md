# `writing-red-lines.md`：长篇正文这些坑别踩

这份不是拿来找灵感的。

这份是让你先别掉坑里。

碰上任一红线时：
- 先别继续顺写
- 先别并入主正文
- 别只做词语替换式修补
- 别跳过 `scripts/scene_lint.py` 的写后质检

---

## 一、正文别按这类歪顺序写

### 红线 1：别先想效果词，再拿效果词顶正文

错误顺序：
- 先想“这一段要狠、要炸、要压、要有气势”
- 再用抽象效果词、判断词、气氛词去顶正文

正确顺序：
1. 谁在动
2. 动作打在何处
3. 造成什么可见后果
4. 对方如何失态、受伤、变形、退缩、反扑

以上四项没落地前，先别写效果判断。

一看就不对的情况：
- 删掉效果词后，段落立不住
- 说明动作层、后果层、疼感层没有写出来

### 红线 1.5：别用生硬撑劲词起稿

别这么来：
- 先想“这里要狠一点、猛一点、炸一点”
- 再用生硬撑劲词或空力度副词把句子顶过去

稳一点的顺序：
1. 先写谁冲上去
2. 再写打在什么部位
3. 再写人或物怎么偏、怎么断、怎么翻、怎么退
4. 最后才允许极少量补强语气

先看清：
- `references/banned-words.md` 中的 `一级禁用词` 属于首稿先别生成的内容，不是写后再删项
- 生硬撑劲词要是扛了动作主活，首稿里就先别冒出来
- 二级敏感词要是扛了动作主活，也算写失手了

看到这些情况就说明不对：
- 去掉该词后，句子只剩“有人动了”而没有受力点、位移、伤害结果
- 同一段或相邻两段连续靠这类词顶动作
- 这词扛了本句大半力度

更稳的改法：
- 把力度拆回 `动作怎么过去 + 打到哪儿 + 人怎么被带歪 + 当场怎么失态`
- 别只换成别的近义力度词

### 红线 2：别让作者脑子里的走法话漏进正文

错误顺序：
- 先想“这里要撕开军权线”“这里进入第二阶段”“这里是第一卷节点”
- 再把这种后台判断漏成正文句子

正文只能写：
- 人物当下看到的
- 人物当下听到的
- 人物当下感觉到的
- 人物当下做出来的

正文不能写：
- 卷走法判断
- 章节作用判断
- 线索调度判断
- 写作复盘判断

### 红线 2.3：别把正文写成事件记录，要写成主角正在经历

别这么来：
- 先按作者视角记“发生了什么”
- 再把人物塞进这些事件里走流程

稳一点的顺序：
1. 主角这一瞬先注意到什么
2. 主角因此误判 / 起疑 / 改主意什么
3. 主角先护什么、先舍什么、先做什么
4. 外界再因此发生什么变化

先看清：
- 正文先按 `主角感知顺序` 往下落，别先按 `作者说明顺序` 往下摆
- 若一段删去主角的感知、误判、改招后仍完全成立，说明这段更像事件记录，不像人物正在经历

看到这些情况就说明不对：
- 连续 `3` 句及以上都在写“谁做了什么、然后又做了什么”，却没有主角当下的注意力偏移、判断变化、临场取舍
- 段落能回答“发生了什么”，却答不出“主角是先看见、先听见、先怀疑、先决定了什么”
- 读者读完像看值班记录、战报、行程单，而不是像被按进现场

靠谱改法：
- 把总述拆回主角这一刻最先抓到的东西
- 把动作链拆回“受阻 -> 改招 -> 现实变化”
- 把信息后移到主角真正拿它做决断的那一拍

### 红线 2.4：别让制度术语直接顶正文理解

别这么来：
- 先报一个制度词、职业词、流程词
- 让读者自己去猜它现在压谁、放谁、卡谁

稳一点的顺序：
1. 先让读者看见它正在卡谁或放谁
2. 再顺手知道这东西叫什么
3. 若需要术语，也只保留最小必要量

看到这些情况就说明不对：
- 一个词本身不难，但放在这句里要读者停下来翻译
- 删掉术语换白话后，句子更顺、冲突更清
- 一句里连报多个制度口名词，人物却还在急场里说得很稳

更稳的改法：
- 把术语拆回 `谁先走 / 谁被卡 / 谁得掏钱 / 谁会死`
- 把制度词挂到具体东西、次序变化、执行动作上
- 若术语只是显得专业，直接删或换白话

## 二、正文这些句型别往里塞

### 红线 3：别把后台调度语漏进正文

正文别出现：
- `这一卷/这一段/到这一步`
- `终于把某条线撕开`
- `真正进入某个阶段`
- `现在的问题是`
- `接下来就是`

这些不是现场，是后台写作语言。

### 红线 4：别让作者替现场下结论

正文别靠这些句子收尾或定性：
- `这意味着……`
- `这说明……`
- `表明……`
- `证明……`
- `显示……`
- `从这一刻起……`

如果动作、对白、反应已经立住了，就别再补一句作者结论。

### 红线 4.5：章尾最后一句别拿来解释“它为什么重要”

章尾最后一句只能停在：
- 新动作
- 新物证
- 新说法
- 新名单
- 新翻面
- 下一步命令

章尾最后一句别承担：
- 意义总结
- 局势翻译
- 作者判价
- 资源判价
- “这意味着 / 这说明 / 真正要命的是 / 现在真正该抢的是” 这类把意思定死的说法

一看就不对的情况：
- 删掉最后一句后，章尾更利索、更有力
- 最后一句主要作用是告诉读者“刚才那一下为什么重要”
- 最后一句没有新增现场结果，只是在总结前一拍

靠谱改法：
1. 先回看最后一句前，现场是否已经有独立成立的结果
2. 若已成立，直接删掉最后一句
3. 若未成立，不补解释句，回前一拍补动作 / 物证 / 他人失态 / 现实变化

### 红线 5：禁止抽象效果句替代现场

正文别主要依赖：
- `压迫感很强`
- `杀气逼人`
- `气势惊人`
- `场面可怕`
- `危险至极`
- `爽感炸开`

这些顶多少量帮衬一下，别拿来扛主体表达。

### 红线 5.5：正文叙事句别让空效果词扛主活

下面这些表达一旦扛了句子的主活，就算写偏：
- 抽象强弱词
- 抽象气势词
- 抽象危险词
- 抽象速度词
- 抽象判断词

怎么判断：
- 删掉这些表达后，句子就散了
- 说明动作、后果、失态、物理变化没有写出来

碰上后别这么干：
- 只删词
- 只换近义词
- 只缩短句子

靠谱改法：
- 回到动作层
- 回到后果层
- 回到受伤/失态/位移/声响层
- 把偷懒表达换成读者能看见的事

### 红线 6：别把提纲扩写句直接抄进正文

正文里别直接冒出：
- `真正的……`
- `更该先看的其实是……`

### 红线 6.2：别把“这句拿来干什么”的总结话直接塞进人物嘴里

错误顺序：
- 先想清这句要拿来干什么
- 再把这个总结话直接塞进人物嘴里

典型错误：
- `哪边缺，我就先补哪边`
- `先把这事先按谁的法子办拿住`
- `我先把这处局面改掉`
- `这件事我先接下来，后面再慢慢清`

这些句子的问题不是“看不懂”，而是：
- 更像在念思路
- 更像作者摘要
- 更像执行层总结
- 不像人在当场说话

一看就不对的情况：
- 句子里能看见这句想办什么事，看不见人
- 句子里能看见策略摘要，看不见对谁说、为什么此刻这么说
- 把句子里抽象词拆掉后，场合压力、对象关系、现实代价都不成立

顺着这么改：
1. 先写清这句话到底要办什么事：接锅 / 抢先办事 / 立军令状 / 逼人把话收回去 / 抢先手
2. 再补三件事：
   - 对谁说
   - 代价落在哪
   - 今夜或当场会发生什么
3. 最后改成带对象、带压力、带后果的人话

往这边改更顺：
- 不说“我补哪一层”，改成“哪道地方先漏，我先去堵哪道”
- 不说“我先把拍板那一下拿到手”，改成“今夜这事先按我的法子来，出了错先算我”
- 不说“我改局面”，改成“你要按旧例赶人，那今夜断巡断押的锅别往别人头上推”
- `不是X，而是Y`
- `原来……`

如果这些句子在一段里扛着主要翻面作用，说明正文还停在提纲层。

### 红线 6.3：别让写作说明里的词漏进叙述层

错误顺序：
- 先在写作说明里想“这块地方谁占着 / 这条办事次序 / 这道得先过的地方 / 这一下就改了”
- 再把这些提纲词半遮半掩地塞回正文叙述

典型错误：
- `这块地方还没让`
- `今夜这盏灯，你没份了`
- `这股火立刻就炸了`
- `先把这块地方腾出来`

这些句子的问题不是“不通顺”，而是：
- 更像写作备注
- 更像执行层摘要
- 更像作者拿提纲词替代现场
- 不像人真能看见、摸见、听见的东西

只要出现这些样子，就说明还没落回现场：
- 删掉这类提纲腔词以后，句子只剩“要办什么事”，现场东西不成立
- 句子能说明“这下办成了什么”，却说不清“到底谁没让、谁没挪、谁被赶开、谁把灯抢过去了”
- 该句可以轻易替换到别的场景里，说明它不长在当前现场

顺着这么改：
1. 先把说明词删掉
2. 换成现场东西：
   - 灯位
   - 班牌
   - 木架
   - 站口
   - 墙根
   - 外圈
3. 再写清：
   - 谁占着
   - 谁不让
   - 谁被挤开
   - 谁先上去

往这边改更顺：
- 不说 `这块地方还没让`，改成 `灯位还让不出来`
- 不说 `护那一层缺口`，改成 `护地方 / 护灯位 / 护那块站地`
- 不说 `这股火炸了`，改成 `卡口边那几个人一下吵炸了`

### 红线 6.4：别把岗位说明词当成人说话的主心骨

错误顺序：
- 先想清“这个人现在负责哪一摊事”
- 再直接让正文或人物把 `灯位 / 人站的地方 / 先后次序 / 办事次序 / 押路位 / 扶绳位` 当成主要说法

这些词不是完全不能碰，但一旦成了句子里最顶事的那一块，常见结果就是：
- 更像岗位说明单
- 更像一张调度摘要
- 更像作者在替现场做总括
- 不像人在急场里盯着具体人、具体地方、具体东西说话

典型错误：
- `灯位还让不出来`
- `先把地方让出来`
- `现在先轮到他`
- `先让他定这一步`

一看就不对的情况：
- 删掉岗位说明词以后，句子更顺，更像人话
- 句子能告诉你“这句想干嘛”，却说不清“谁堵着、谁不让、谁先上、谁被赶开”
- 同一段里岗位说明词连续出现，压过了具体动作和地方

靠谱改法：
1. 先回到现场东西：
   - 灯架
   - 班牌
   - 木桩
   - 墙根
   - 外圈
   - 那块地方
2. 再回到具体动作：
   - 挤开
   - 摘下来
   - 往外推
   - 塞进手里
   - 拖过去
3. 最后才在最少的量上留一点岗位信息

往这边改更顺：
- 不说 `灯位还让不出来`，改成 `灯架底下那块地方还让不出来`
- 不说 `先把地方让出来`，改成 `先给我挪开`
- 不说 `现在先轮到他`，改成 `今夜先让他上`
- 不说 `执行次序给他`，改成 `今夜谁上谁退，先听他的`

### 红线 6.5：别把岗位分工念成列表式台词

错误顺序：
- 先想清楚这一场里有哪些岗位和分工
- 再让人物一下把这些岗位全报给读者听

典型错误：
- `今夜谁替灯，谁扶绳，谁押路，我还没聋`
- `谁站门，谁记名，谁押后，轮不到你来说`
- `谁看前头，谁守后头，谁搬货，我都知道`

这些句子的问题是：
- 更像在报岗位
- 更像职责说明
- 更像作者在教读者认工种
- 不像人在急场里护地方、护脸、护今晚这份活

一看就不对的情况：
- 句子一读像排班表，不像人在顶嘴
- 人物先列工种，再表达火气
- 删掉岗位列表后，人物说话反而更像活人

靠谱改法：
1. 先删掉岗位列表
2. 改成现场争夺：
   - 灯还挂在谁手里
   - 人还站不站在这儿
   - 谁敢伸手
   - 谁敢把谁挤走
3. 再在最少的量上让读者顺手知道这些人是干什么的

往这边改更顺：
- 不说 `谁替灯，谁扶绳，谁押路`，改成 `灯还挂在这儿，人也还站在这儿，轮不到你们来指手画脚`
- 不说 `谁站门，谁记名，谁押后`，改成 `人还没挪，你先别来抢地方`

### 红线 6.4：别让对白和叙述一起掉进制度说明腔

正文和对白要是同时撞上下面这些情况，一般至少记 `F1`：
- 叙述句在报制度词
- 对白句也在解释制度词
- 现场动作反而没有先落地

这类写法读起来最像“作者自己先想明白了，再借角色的嘴讲给读者听”。

靠谱改法：
1. 先删最晚出现的解释句
2. 回前一拍补动作、物证、谁在前谁在后、被拦下来以后出了什么结果
3. 若仍要保留术语，只留一次，并挂到当场代价上

### 红线 6.1：别让旁白替作者使劲

正文叙述句别去替人干下面这些活：
- 替读者下判断
- 替场面下结论
- 替人物发狠
- 替章节收一句看着利落的话
- 替冲突翻译“这一拍到底爽在哪”

高危表达包括但不限于：
- `先乱的不是A，是B`
- `最重要的不是A，而是B`
- `不是在X，是在Y`
- `不是……，而是……`
- `因为这不是……，是……`
- `这话不是在……，是……`
- `真怕……的，不是……`
- `真正可怕的是……`
- `真正要命的是……`
- `现在该抢的不是……`

先看清：
- 以上句型不是见一个封一个的死词
- 但只要它们扛了句子里最主要的活，一般就算旁白替作者使劲
- 若删掉这句后，现场动作、失态、谁前谁后、物证还都成立，说明它本来就是在替作者说话

一看就不对的情况：
- 删掉该句后，段落更利索、更像现场
- 该句没有新增动作、物理变化、利益变化，只是在重新翻译前一拍
- 这句的劲主要来自句型本身，而不是来自现场已经立住的现实变化
- 同章如果撞上 `2` 句及以上同类发力句，一般至少记 `F1`
- 若这些句子撞在章首开场、章中关键对顶、章尾收束处，一般先往上提一级复检

靠谱改法：
1. 把该句想表达的“判断”，拆回前一拍动作
2. 把该句想表达的“价值”，拆回谁先护什么、谁先退什么
3. 把该句想表达的“爽点”，拆回外部反应、现场被改掉的地方、敌方失态
4. 别只把句型换个近义说法就继续硬顶

更该补的是这些现场东西：
- 谁先伸手去护牌、票、账、令
- 谁先往后退半步、卡半步、把半句话吞回去
- 围观的人先往哪边挤
- 器物先怎么响、怎么偏、怎么卡住
- 规矩先压到了谁、又先放过了谁

### 红线 6.3：细节只能垫厚，别拿来顶事件

拿来垫厚正文的细节，一般只干这些辅助活：
- 把人物动作落点写清
- 把物件状态变化写清
- 把环境压力怎样压到当前动作上写清
- 把身体先泄露出来的反应写清
- 把规矩、职业习惯、群体常识写到现场里

这些细节别去顶下面这些主要作用：
- 代替新事件
- 代替新阻力
- 代替新代价
- 代替现场被改掉的地方
- 代替人物真正出手

一看就不对的情况：
- 本段删掉细节后，事件没有任何损失，只说明原本就在空转
- 本段只有环境、神态、气味、声响，却没有动作推进、谁前谁后变化、利益变化
- 所谓“补字数”主要靠环境句、心情句、抒压句，而不是靠冲突链扩展
- 一章里连续出现多段“看起来有画面，实际上没改局”的细节堆叠

靠谱改法：
1. 先补事件：谁拦、谁抢、谁把话收回去、谁失手、谁被逼着挪地方
2. 再补冲突：明面抢什么，暗里护什么
3. 最后才允许补细节：把动作落点、物件状态、环境压力、身体反应写清

可用的细节落点：
- 被捏软的票角 / 牌角 / 名册边
- 门轴、风索、锁扣、栏杆、印泥、旧伤、污痕
- 排队姿势、护物动作、看向哪里、退半步还是逼半步
- 与规矩直接相关的声响、器物磨损、职业习惯、群体常识

禁用误区：
- 用细节代替解释句之后，又把整段写成空镜头
- 用细节补世界观，却不让这条世界规矩改动作、改生死
- 用细节补人物，却只写“皱眉、发抖、沉默、脸白”这类通用反应

### 红线 6.4：别平均铺叙，关键一秒别被总述抹平

正文别把最要命的那一下写成：
- `他看了一眼`
- `众人顿时明白`
- `局势一下变了`
- `双方很快交手`
- `然后事情就翻了`

先看清：
- 这些句子不是完全不能出现
- 但要是让它们扛关键翻面、关键受阻、关键立威的主活，一般就是把戏写成了流水账

一看就不对的情况：
- 关键翻面只用 `一下 / 顿时 / 很快 / 立刻 / 转眼` 一类时间词带过去
- 本应最有代入感的一拍，反而是段落里最抽象、最短、最像摘要的一拍
- 删掉某个总述句后，发现根本没有补上对应近景动作、器物变化、失态反应

靠谱改法：
1. 找出本段最重要的 `1-2` 秒
2. 把它拆成：
   - 谁先碰到什么
   - 什么先响、先偏、先断、先停
   - 谁先失手、先退、先把话收回去
3. 别再拿总述句顶过去

### 红线 6.7：解释句不能只看词面，得继续往下看根因

碰上解释腔 / 下结论句 / 作者替角色下结论的句子时，别只写“这句太解释”。

还要继续往下分到以下四类之一：
- `G1 信息抢跑`：正文抢着替读者说明设定、局势、价值
- `G2 现场东西不够`：动作、后果、失态、现实收益没写够，只能靠判断句补缝
- `G3 提纲语言漏出`：章节意图、走法调度、线索调度直接漏进正文
- `G4 人物失声`：本该由人物动作 / 对话来写的内容，被旁白替说了

先看清：
- `G1 / G4` 一般至少算句子层失败
- `G2` 一般先判场面没真正写进场里，别只删句
- `G3` 一般先判提纲语言漏出来了，别只换词

若还不能归到 `G1-G4` 之一，说明仍停留在现象层，别结束回修。

### 红线 6.72：别让 POV 松成“监控视角”

正文如果走有限视角，就别随手滑出当前人物当下能看到、能听到、能想到的边界。

常见别急着说行的情形：
- 主角背对某人，却先写清对方脸色和完整心思
- 一段里不断切换“这边看到什么、那边同时怎么想、围观者又整体怎么判断”
- 场景描写像俯拍总览，谁都看见了，唯独没有主角自己的注意力顺序

一看就不对的情况：
- 删掉人物名字后，整段像无主镜头扫场，不像某个人在现场里承压
- 读者更像站在天花板上看混乱，而不是站在主角肩后吃风、吃压、吃误判

靠谱改法：
- 锁回当前人物：
  - 先看到什么
  - 先听到什么
  - 先误会什么
  - 先决定什么
- 其他信息只能通过动作、声音、器物、结果倒推出来

### 红线 6.75：别让旁白发力句伪装成“有劲的开场句”或“有钩子的收束句”

下面这些地方先重点抽查：
- 章首前 `200` 字
- 第一爆点前后
- 主角与高位敌人第一次正面对顶时
- 章尾最后 `5-15` 行

先别急着说行的情形：
- 章首靠一句“先乱的不是……而是……”硬拉气势
- 章中靠一句“这不是在……，是在……”硬翻译冲突
- 章尾靠一句“真正要命的是 / 真正重要的是……”硬抬钩子
- 同一章里，旁白比人物更会说、更会下判、更会收句

修法要求：
- 开场要回到现场具体东西：谁挤、谁卡、谁先掉、谁先喊
- 中段要回到冲突具体东西：谁护物、谁把话收回去、谁失手、谁被迫换地方
- 章尾要回到结果具体东西：新动作、新物证、新名单、新说法、新翻面

### 红线 6.8：人物那点算盘别老靠旁白直译

正文别主要靠下面这些说法来证明人物“像活人”：
- `他最怕的是……`
- `他真正想要的是……`
- `他不是为了X，而是为了Y`
- `他不是想帮谁，他只是……`
- `他终于算明白了……`
- `他怕的不是A，是B`

这些句子偶尔冒一句还能忍，但别拿它们当人物成立的主要依靠。

一看就不对的情况：
- 删掉这句后，角色就只剩身份和剧情作用
- 该角色的欲望、损失点、私心、站哪边，没有通过动作 / 台词 / 停顿 / 失手 / 回避自行露出来
- 同一角色在同一章里反复靠这类句子补“他到底在想什么”
- 同一角色在同一章里碰上 `2` 句及以上同类直译句，一般至少按人物出场分量先别急着说行

靠谱改法：
1. 先把人物这场最想拿到什么，转成他正在护的具体东西
2. 再把人物最怕失去什么，转成身体先泄露出来的反应
3. 再把人物真正站哪边，转成一句带回避、转移、压价、甩锅、装硬的台词
4. 能让读者看懂时，不再补旁白翻译

更该补的是这些现场东西：
- 手里反复碰的物件
- 说到关键处突然把原话吞回去
- 明明想往前，却先退半步
- 嘴上压人，动作却先护住别的东西
- 沉默、岔话、跳答
- 抢先去挡 / 去删 / 去藏 / 去压的那一下
- 被点破后先护住的门卡 / 名册 / 护腕 / 算盘 / 钥匙 / 芯片

一般先往这边判：
- 多数情况先判 `G4 人物失声`
- 若该句同时在替整段补场面，则再加判 `G2 现场东西不够`

### 红线 6.5：别替读者先把轻重算完

正文别靠下面这些句型来代替“资源升级已经在现场站住了”：

- `更该先看的其实是……`
- `更硬的是……`
- `更实的是……`
- `这一项更……`
- `比……更……`
- `现在真正该抢的是……`
- `最该抢的已经不是……`
- `他终于明白，真正重要的……`

尤其是这些句子要是去扛了：
- 资源升级说明
- 爽点翻面说明
- 这一章到底吃到了什么好处
- 章尾钩子这一拍到底想吊什么

这几样要是让它扛主活，直接算没写对。

为什么这类话不行：
- 这类句子还是旁白先替读者把轻重算完了

### 红线 6.6：别把“说明口术语”直接塞进正文

正文别直接把纲层术语写成人话叙述，尤其是：

- `第一步……`
- `第二层……`
- `接下来那一下……`
- `这一层……`
- `哪一层……`
- `抽象收益总括句`
- `接头用的东西`
- `谁先开口就能定事的说法`

毛病就在这：
- 这些词在纲层只是压缩标签，进正文以后就会变成人物不会说、自然叙述也不该这么写的术语话
- 高频出现后，正文会越来越像办事记录、拆解报告、复盘纪要，不像小说

直接这么看：
- 若角色对白撞上这类问题，先判 `G4 人物失声`
- 若旁白叙述撞上这类问题，先判 `G2 现场东西不够`
- 若章尾靠这类术语话过桥，直接记为章尾先别急着说行

靠谱改法：
- 把抽象术语拆回 `具体人 / 具体物 / 具体地点 / 具体规矩 / 具体动作`
- 不写“接下来就该看资源这一线”
- 改写为“他脚下一偏，先去军械库”
- 不写“第一步现实变化已经拿到”
- 改写为“簿子先落了字，牌也没被人拿走”
1. 先写资源到底怎么换了归属
2. 再写谁因为这份资源挪地方、改动作、把话收回去
3. 再写谁因此活下来 / 被压下去 / 被挤出去
4. 别再补一句“所以这一下更值钱 / 更顶用 / 更实在”

可照着这个方向改：
- 不写：`这份收益比夜票更实`
- 改写成：`夜票只能让他挤上车。可刚才这一推一挤，活着站进前厢的人，已经真的换了。`

- 不写：`真正重要的是下一站谁先下车`
- 改写成：`站外那七个冷黄位置只亮了七道，多一个人挤过去，后面就得有人被压回门里。`

## 三、正文一写歪了就按这个顺序往回退

### 红线 7：别只删词，要回到句子到底在干什么

发现口号腔、解释腔、后台句时，回修顺序固定为：
1. 先看句子这一下是不是写歪了
2. 再看场面层是不是没写够
3. 再看细纲是不是根本没把这章撑住

别这样修：
- 只删单个词
- 只换近义词
- 只把句子缩短

### 红线 7.5：写完后还得跑正文场面检查

固定做法：
- 正文写完
- 运行 `scripts/scene_lint.py <正文文件>`
- 按实际碰上的问题整句或整段回修
- 复扫对上后才能收进正文

如果没跑脚本，一般就算这轮还没查完。

### 红线 8：句子层、章节层、整层问题别混着修

句子层：
- 解释句
- 作者下结论句
- 抽象效果句

章节层：
- 场面不足
- 爽点不落地
- 围观层/阻力层/代价层缺失

整层问题：
- 书名主打开始写偏
- 连续多章只查不兑现
- 副驱动反客为主

- 句子层才允许局改
- 章节层就得把场面和正文分量补回来
- 整层问题就得回细纲或连续章节组

## 四、动笔前后自己先问

写正文前，先问：
1. 这段是不是先有动作，再有感受？
2. 这段删掉效果词后还成立吗？
3. 这段有没有作者站出来替读者总结？
4. 这段是不是人物现场，而不是提纲后台？

写正文后，再问：
1. 谁赢了？
2. 谁亏了？
3. 谁被打疼、打丢脸、打失态？
4. 现实局面到底改掉了哪一块？
5. 删掉最后一句，章尾会不会更强？
6. 删掉最后两句，章尾现场还成立吗？
7. 最后一句是在新增现场，还是在解释前面的现场？

任一问答不上来，就先别往下走。
