
# 网文质量检查清单

这份就是写完以后拿来过稿和回炉的。

别一上来就盯句子，先看结构、推进、情绪、高潮和结尾是不是站住了。
如果已经跑了内部审计，默认还要同时看：

- `segment_scores`
- `paragraph_scores`
- `high_risk_segments`
- [heavy-audit-breakdown.md](heavy-audit-breakdown.md)

不能只看全文总分。
回炉时默认优先级是：`同桥承重件 / 后果链` > `高风险片段` > `高风险段落` > `短段节奏`。短段不是主刀位。
如果 `global_risk_shape.shape = single_global_block`，优先级再提前一层：

- 先修整篇主桥起手、顺序、场戏分工、人物偏手、烂关系漏出
- 禁止先做句面回修、短段整理、同义词替换
- 先看粗粒度大块，再看正文块，再看局部段落

如果 `global_risk_shape.shape = coarse_blocks`：

- 先修前 1-2 个粗粒度大块
- 每个大块内部仍按 `起手 -> 顺序 -> 承重件 -> 句面` 回炉

## 先判是不是“过闸稿”

下面这些信号，只要同时出现 2 条以上，就先怀疑这版正在往“安全施工稿 / 过闸稿”滑，不要继续顺着润：

- 命中下降了，但人物更会解释自己的心情和关系
- 句子更顺了，但场面里的失手、岔开、掉位、心虚反而少了
- 台词更完整了，但角色更像同一种会成熟沟通的人
- 逻辑更直了，但后果更像作者替人物总结
- 段落更匀了，但事情更不像被逼出来，只像成熟模板块

硬口径：

- `更像会过审计` 不等于 `更像能发表`
- 回修后如果 `安全感上升、活人感下降`，视为失败
- 这类失败先回桥段、场戏、人物偏手，不先继续润句

如果有 `bridge_rule_audit`，桥段层再细分成：

- `opening_pattern_missing`
- `recommended_sequence_missing / recommended_sequence_out_of_order`
- `must_keep_missing`
- `must_avoid_hit / fake_signal_hit`

顺序不能倒。不要先修短段、热点和句壳，再回头补桥段。

重审计相关的判断口径固定分两层：

- `直接计分层`：句壳、模板反应、统计平滑、重复热点
- `结构不过层`：桥段承重件、顺序件、单场戏功能堆叠、人物偏手、烂关系漏出

具体拆解见：

- [heavy-audit-breakdown.md](heavy-audit-breakdown.md)

## 一、通用检查

### 章节结构
- [ ] 开头有钩子（不是天气/风景/日常开场）
- [ ] 中段有推进（有事件发生）
- [ ] 局势有变化（读完这章，世界跟之前不一样了）
- [ ] 结尾落在变化上（不是总结）
- [ ] 这一章不是“把上章情绪换个说法再说一遍”

### 开篇检查（前 300-500 字）
- [ ] 有钩子，能抓住注意力
- [ ] 不从天气/风景/日常开始
- [ ] 主角快速出场
- [ ] 卖点或危机可见
- [ ] 读者能看见后面最想等的结果

### 章节推进
- [ ] 有核心事件
- [ ] 局势有变化
- [ ] 不是水字数（删掉这章会影响理解吗？不会 = 水了）
- [ ] 推进了主线/关系/设定中的至少一项

### 信息传递
- [ ] 没有大段设定说明文
- [ ] 信息跟着冲突走（通过事件传递设定）
- [ ] 设定量可控（一章不超 3 个新概念）

### 场景检查
- [ ] 场景有目标（人物要什么）
- [ ] 场景有阻碍（什么挡着）
- [ ] 场景有变化（结束后跟之前不同）
- [ ] 人物在做事情，不是在感觉事情
- [ ] 没有可删除的段落
- [ ] 主角不是纯挨打，至少有一个主动动作

### 章尾
- [ ] 结尾落在变化上
- [ ] 有危机/决定/发现/反转中的至少一个
- [ ] 不是总结式结尾
- [ ] 拉住读者翻下一页

### 语言
- [ ] 没有空洞的抒情段落
- [ ] 没有连续多段同一情绪
- [ ] 对话符合人物身份（不同人说话方式不同）
- [ ] 情绪通过动作落地（不是直接说"他很难过"）
- [ ] 没有明显“作者替人物下判词”的句子
- [ ] 没有明显“作者为了新鲜硬拧出来”的句子
- [ ] 没有明显“为了显得稳，把人物写得更会解释、更会总结”的句子

### 连载连续性
- [ ] 没有遗忘之前的承诺/伏笔
- [ ] 没有突然塞入大量新设定
- [ ] 伏笔有推进
- [ ] 故事引擎还在运转

### 水（filler）检测

下面这些信号一旦出现，就先怀疑这段是不是在水：
- 全章对话没有任何新信息
- 同一个情绪写了 3 段以上
- 场景描写超过 500 字但不推进剧情
- 角色回忆之前发生的事但没有任何新视角
- 连续 2 章以上没有冲突
- 主角连续几场都只是受气，没有任何位置变化

## 二、长篇专项

### 黄金三章检查
- [ ] 第一章前 500 字有钩子？
- [ ] 主角第一章就出场？
- [ ] 第一章有事件（不是纯铺垫）
- [ ] 第二章有升级（矛盾加深）
- [ ] 第三章有追读理由（给出继续看的动力）
- [ ] 前三章有至少 2 个爽点？
- [ ] 世界观没有大段说明文？
- [ ] 每章结尾有悬念？

### 节奏检查
- [ ] 最近 5 章是否有明确进展？
- [ ] 爽点间隔是否超过 5000 字？
- [ ] 有没有连续 2 章以上没有冲突？

### 人物检查
- [ ] 主角行为符合人设？
- [ ] 配角是否有存在感？
- [ ] 反派逼格是否匹配当前阶段？

### 五维评分标准

每个维度 0-100 分。分不是重点，重点是最后知道该往哪修。

### 维度 1：核心一致度

先看最根上的东西有没有乱：

- 关键冲突前后是不是一回事
- 人物动机是不是说变就变
- 关键行动是不是和人设打架

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 人物动机突然改变无铺垫 | critical | 补充动机转变的触发事件 |
| 核心冲突前后不一致 | high | 回溯修改冲突设定 |
| 关键行动与人物性格矛盾 | high | 调整行动或补充解释 |
| 次要矛盾遗忘 | medium | 回收或弱化 |

### 维度 2：表层重写度

再看表层句子是不是太像模板，是不是一眼就有 AI 腔或套话感。

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 照搬原句导致语气不自然 | medium | 只在确有 AI 腔时改写，正常原句可保留 |
| 大量使用 AI 标志词 | high | 替换为具体描述 |
| 同一句式重复出现 | medium | 变换表达方式 |
| 描写过于文学化 | medium | 改为口语化/动作化 |
| 所有角色说话像一个人 | high | 回原文补口气模板，区分称呼/句长/找补词/失控方式 |

### 维度 3：格式一致度

再看版面和格式是不是稳：

- 段长是否乱跳
- 节拍是否太机械
- 工作稿和投稿版格式有没有串

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 段落长度差异过大 | medium | 调整段落划分 |
| 连续大量等长短句密排 | high | 合并或拉开部分段落，补入正常长度叙述句和人物主观句，打破机械节拍 |
| 章节字数偏离目标 | **high** | 写作/大纲修复时先回到细纲补足计划内情节点，再展开；去AI味已有正文时不得新增剧情 |
| 格式混乱（对话/描写不统一）| low | 按目标平台统一；工作稿默认 `""`，知乎投稿版默认 `「」`，如平台明确要求 `【】` 则覆盖 |

### 维度 4：可读性

这一步就盯一个事：读起来是不是顺，是不是活，是不是像人写的。

| AI 腔特征 | 如何识别 | 如何修复 |
|-----------|----------|----------|
| 空泛总结 | 「他终于明白了」「一切尽在不言中」 | 删掉，用行动代替 |
| 套路修辞 | 「命运仿佛在和他开玩笑」 | 删掉或换成具体描述 |
| 情绪标签 | 「他感到一阵悲伤」 | 改为行为表现 |
| 大段心理描写 | 连续 3 句以上内心独白 | 压缩为 1 句或转为对话 |
| 模板节拍 | 连续多段都写成「动作一句 + 心理一句 + 台词一句」 | 打散节拍，合并同镜头信息，改成不同长度和不同句法的自然推进 |
| 生造口气 | 词新鲜但不像真人会说 | 回退常用口语层，先保句子功能再修词 |
| 代判句 | 作者替人物、替读者把反应说透 | 改成现场反应、动作、停顿、接不上话 |

### 维度 5：逻辑连贯

最后看逻辑有没有串：

- 设定有没有打架
- 时间线有没有错
- 因果链有没有断

| 问题 | 严重度 | 修复 |
|------|--------|------|
| 设定前后矛盾 | critical | 查找并统一设定 |
| 时间线错误 | high | 标记时间线并修正 |
| 角色信息不一致 | high | 建立角色档案对照 |
| 因果链断裂 | medium | 补充过渡 |

### 精修策略

分出主要问题后，再选修法，不要所有问题都拿“润色”硬修。

| 主要问题 | 策略 | 说明 |
|----------|------|------|
| 核心一致度低 | rewrite | 围绕核心冲突重写相关段落 |
| 字数超标 | compress | 先判断是否只是轻微超标；轻微超标时不要优先为压字数破坏桥段承重件、人味和场面秩序，只在结构稳定后再删减不推动剧情的内容 |
| AI 腔重 | de_ai | 替换禁用词、改写句式 |
| 小问题多 | polish | 打磨语言细节 |
| 全篇太平 | restructure | 回大纲补公开场、掉位变化、结果期待 |

字数处理硬口径：

- `8000-13000` 是目标区间，但不是高于一两百字就必须立刻动刀的死板触发器。
- 如果只是轻微超标，且当前正文的桥段承重件、后果链、公开场秩序、人物偏手还没完全站稳，**不要先压字数**。
- 轻微超标默认先继续做桥段/结构/人味修正，等核心问题稳定后，再删不承重的重复句、解释句、匀速段。
- 只有明显超标，或者已经确认存在不推动剧情的废段、重复说明、成套流程复述时，才把 `compress` 提到前面。

## 三、短篇专项

### 起盘卡验收

真正开始写正文前，先看起盘卡是不是已经把承重字段填实了。缺字段时，不要靠正文硬补。

必查字段：
- [ ] 平台明确
- [ ] 母结构明确
- [ ] 主冲突明确
- [ ] 副升级器明确，且不和主冲突撞车
- [ ] 关系场明确
- [ ] 开头钩子明确
- [ ] 开头钩子类型明确
- [ ] 异物明确
- [ ] 第二层冲突明确
- [ ] 高潮公开场明确
- [ ] 公开场后果明确
- [ ] 关键证据明确
- [ ] 角色口气模板至少有主角 / 对手 / 接住者
- [ ] 新状态明确
- [ ] 虐点对照细节明确
- [ ] 关系重组方式明确
- [ ] 情绪母线明确
- [ ] 情绪为什么爽明确
- [ ] 爽点类型明确
- [ ] 付费期待明确

如果是 `仿写 / 融合 / 去原作化` 稿件，额外必查：
- [ ] 上游参考稿已有 `样本分级与可学层`
- [ ] 已确认上游参考稿的 `样本等级`
- [ ] 如果上游是 `B类骨架样本`，本轮没有把它当句法 `DNA` 源使用
- [ ] 如果上游是 `C类负样本`，本轮没有把它并入正向融合 `profile`
- [ ] 上游参考稿已有 `导语拆解表`
- [ ] 上游参考稿已有 `顺序事件表`
- [ ] 上游参考稿已有 `物件表`
- [ ] 上游参考稿已有 `动作表`
- [ ] 上游参考稿已有 `对白功能表`
- [ ] 上游参考稿已有 `对话衔接表`
- [ ] 上游参考稿已有 `误判表`
- [ ] 上游参考稿已有 `钩子表`
- [ ] 上游参考稿已有 `微动作表`
- [ ] 上游参考稿已有 `安静压迫场表`
- [ ] 上游参考稿已有 `人物偏手表`
- [ ] 上游参考稿已有 `失控说话表`
- [ ] 上游参考稿已有 `作者DNA指纹`
- [ ] 上游参考稿已有 `仿写约束_禁写清单`
- [ ] 上游参考稿已有 `同桥段过检规则`
- [ ] 已重新生成本轮使用的 `book.profile.json / project.profile.json`
- [ ] 当前使用的融合包不是旧版本遗留包
- [ ] 如果 `run_revision_cycle.py` 提示当前 `project profile` 早于 `拆文库` 中的单书 profile，已先自动重生或手工重生
- [ ] 本轮审计实际使用的是最新重生后的 `project profile`

如果已经有 `profile`，桥段专项还要再过这组检查：
- [ ] 本轮主桥段有对应的 `opening_pattern`
- [ ] 本轮主桥段有对应的 `recommended_sequence`
- [ ] 本轮主桥段有对应的 `fake_signals`
- [ ] 正文起手没有跳过 `opening_pattern`
- [ ] 正文没有把桥段后果提前说穿
- [ ] 正文没有命中当前桥段的 `fake_signals`
- [ ] 回修时没有把现场对话压成转述或总结
- [ ] 上游参考稿已有 `烂关系漏出表`
- [ ] 上游参考稿已有 `book.profile.json`
- [ ] `book.profile.json` 已含 `style_assets`
- [ ] 上游参考稿已有 `写作资产/作者DNA指纹.md`
- [ ] 上游参考稿已有 `写作资产/仿写约束_禁写清单.md`
- [ ] 上游参考稿已有 `写作资产/同桥段过检规则.md`

判定口径：
- 缺 `主冲突 / 副升级器 / 第二层冲突 / 公开场后果` 里的任意 2 项，不开写
- 缺 `角色口气模板 / 新状态 / 虐点对照细节` 里的任意 2 项，说明表演层还没立住，正文容易空
- 如果缺 `样本分级与可学层`，默认不能直接使用上游样本提 DNA
- 如果上游是 `B类骨架样本`，却还在学它的现成句法和口气壳，默认开写方向错误，先退回修参考策略
- 如果上游是 `C类负样本`，却还把它当正向融合源，默认直接阻断
- 如果是 `仿写 / 融合 / 去原作化` 稿件，缺 `导语拆解表 / 顺序事件表 / 物件表 / 动作表 / 对白功能表 / 对话衔接表 / 误判表 / 钩子表 / 微动作表 / 安静压迫场表` 里的任意 2 项，不开写
- 如果是 `仿写 / 融合 / 去原作化` 稿件，缺 `人物偏手表 / 失控说话表 / 烂关系漏出表 / book.profile.json / style_assets` 里的任意 2 项，默认还没到可直接压味和防假的层
- 如果是 `仿写 / 融合 / 去原作化` 稿件，缺 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 里的任意 1 项，不开写
- 如果只有故事梗概、题材概括、起承转合，没有上面 10 张表，默认还停在概括层，先回拆文阶段补细
- 如果没有 `同桥段过检规则`，默认还不能直接拿去写“同桥段换皮稿”或做去AI味回修
- `情绪为什么爽` 讲不清，说明你知道情节怎么走，但还没抓住读者为什么会爽
- `爽点类型` 写不出来，说明结尾落点和高潮兑现大概率会飘
- `开头钩子类型` 写不出来，说明开头多半只是“看着像开头”，还不是功能开头
- `关系重组方式` 只能写成“她成长了”，说明结尾还没落到具体位置变化
- `付费期待` 说不清，说明读者后面最想看的那一下还没被钉住

和上游对齐的准入结论：

- `ready-for-write`：上游可直接仿写层完整，这 10 张表不是空壳，且 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 全齐，能直接支持起盘、场景写作和同桥段防假
- `need-more-deconstruction`：缺任意 2 张以上，或只有故事梗概、题材概括、起承转合，或缺 3 份硬门槛文件中的任意 1 份
- `blocked-on-assets`：骨架有了，但 `对话衔接 / 微动作 / 安静压迫场 / 同桥段过检规则` 等表演层与过检层资产不够，开写只会空

### 仿写 / 融合稿开写前硬闸

别只问“我是不是已经有大纲了”，先问“我是不是已经拆到能直接仿了”。

开写前硬闸：
- [ ] 参考稿不是只剩概括摘要
- [ ] 已读过 `样本分级与可学层`，知道这本参考稿到底是提DNA、提骨架，还是只做反面规则
- [ ] 知道 `导语拆解表` 里前 20 / 60 / 80 字为什么能把人拉住
- [ ] 能顺着 `顺序事件表` 说清原文怎么一步步往前推
- [ ] 知道关键 `物件` 为什么值钱，不会只机械换名词
- [ ] 知道关键 `动作` 为什么有用，不会只靠解释句推进
- [ ] 知道不同角色的 `对白功能` 和 `口气差异`
- [ ] 知道 `对话衔接` 怎么做压回、冷接、装傻接，不会把对白写成平推
- [ ] 知道全文最大的 `误判` 在哪翻
- [ ] 知道每一拍的 `钩子` 为什么能把人往下拖
- [ ] 知道 `微动作` 怎么替代解释句
- [ ] 知道 `安静压迫场` 怎么替代纯吵架高潮
- [ ] 已读过 `作者DNA指纹`，知道这一稿要学的不是情节名词，而是切句、口气、落点方式
- [ ] 已读过 `仿写约束_禁写清单`，知道哪些总结句、过渡句、解释句不能再写
- [ ] 已读过 `同桥段过检规则`，知道原桥承重件和最容易写假的位置

分级补充口径：

- `A类正样本`：可学句法、口气、动作落点、桥段承重件
- `B类骨架样本`：只学骨架、承重件、后果链、场面秩序，不学现成句法壳
- `C类负样本`：只学禁写点和易假桥提醒，不进入正向融合

判定：

- 这 14 条里缺 2 条以上，不是句子问题，是参考稿还没拆透
- 缺 `作者DNA指纹 / 仿写约束_禁写清单 / 同桥段过检规则` 里的任意 1 条，直接退回上游补拆文
- 这种情况不要精修，不要强写，先回上游补拆文
- 如果这 14 条基本都能答上，但对白、场面、公开场仍然明显发空，优先回上游补 `对话衔接表 / 微动作表 / 安静压迫场表`，不要误判成单纯文笔问题

### 局部回炉验收

如果已经跑了 `run_full_ai_audit.py`，回炉前默认再加这几条：

- [ ] 如果当前题材已经做过校准，本轮审计实际带了 `--internal-standard`
- [ ] 先看 `当前判定`，不是只看内部总分
- [ ] 先看 `内部最高块风险分`，不是先看 `内部整体风险分`
- [ ] 先看 `display_block_scores / 正文块风险排行`，这轮先修前 1 到 3 个正文块
- [ ] 先看 `high_risk_segments`，这轮只修前 3 到 5 个片段
- [ ] 再看 `paragraph_scores`，每个片段只挑最危险的 2 到 4 段先动刀
- [ ] 如果已经跑了 `auto_revise_ai_flavor.py`，检查 `task_validation.bridge_alignment_ok = true`
- [ ] 如果已经跑了 `auto_revise_ai_flavor.py`，检查 `task_validation.short_paragraph_priority_ok = true`
- [ ] 如果已经跑了 `auto_revise_ai_flavor.py`，确认 `display_block_focus` 前排就是本轮最该动的大块
- [ ] 如果只有局部片段高，不为了几段高分回写整篇
- [ ] 全文分下降了，但 `high_risk_segments` 里还有 `medium/high`，不算修干净
- [ ] 正文块优先级高于片段，片段优先级高于通用问题列表
- [ ] 段落修法只解决局部问题，不顺手全文润色

送检前的代理线口径：

- [ ] `内部最高块风险分 > 0.75` 时，不直接送检
- [ ] `内部最高块风险分 0.60 - 0.75` 时，默认继续回修
- [ ] `内部最高块风险分 < 0.60` 时，才进入外部终检
- [ ] 即使 `内部整体风险分` 下降，只要 `内部最高块风险分` 仍高，就按未过处理

### 虐爽节奏检查

短篇最容易翻车的地方之一，就是虐和爽的分布不对。

```
理想分布：虐1 → 虐2 → 虐3 → 爽1(小) → 虐4 → 爽2(大)
禁忌分布：虐1 → 虐2 → 虐3 → 虐4 → 虐5（无爽点，读者流失）
禁忌分布：爽1 → 爽2 → 爽3 → 爽4（无铺垫，爽感疲劳）
```

检查规则：
- 每 300 字至少 1 个小冲突
- 虐点间隔不超过全文的 30%
- 最大爽点必须出现在全文的 70-85% 位置
- 结尾必须有情绪落点
- 不能从头到尾都只是在“忍”，中段必须开始出现动作、布局或位置变化

### 对话密度检查

| 指标 | 合格标准 | 警戒线 |
|------|----------|--------|
| 对话占全文比 | 45-65% | <30%（太干）或>75%（太水） |
| 每章对话条数 | 12-18条/千字 | <8条（缺交互）或>25条（碎片化） |
| 伤人性对话占比 | 35-45%（虐文）/ 20-30%（爽文） | 虐文<20%（不够痛）/ 爽文>40%（太压抑） |

### 主角冷静度检查（打脸/复仇文专用）

| 检查项 | 标准 |
|--------|------|
| 主角是否有标志性冷静动作？ | 必须有（端水杯/整理西装/转笔等） |
| 主角是否有情绪失控场面？ | 复仇文最多1次，且必须在前30% |
| 反派是否比主角更歇斯底里？ | 必须是——反差是爽感来源 |
| 主角台词是否短于反派？ | 平均短30-50%——越短越有力 |
| 是否有"审判式对话"？ | 至少2处（主角提问→对方自爆） |
| 主角有没有把对方往更难看的位置推？ | 最好有，不然爽点容易只停在嘴上 |

## 反例对照：什么叫“越修越假”

### 反例 1：人物变得太会解释

**不合格**

- “我不是不爱你，只是我终于明白，我们之间从一开始就错了。”

为什么假：

- 这句把关系判断、主题总结、人物成熟认知一次说完了
- 很像为了过审计，把情绪和冲突都压成了标准解释

**更合理**

- “我看着她，半天没接话。”
- “那枚戒指被我推回桌上。”
- “我说，算了。”

为什么更好：

- 人物没有替作者把题讲完
- 关系冷掉是靠动作和断句落地，不是靠成熟总结句收口

### 反例 2：后果被写成说明

**不合格**

- “直到这一刻，他才真正意识到，自己失去她了。”

为什么假：

- 这是说明，不是后果
- 它把本该出现在动作、秩序、身体反应里的东西，提前总结掉了

**更合理**

- “她把门卡留在鞋柜上，没再回头。”
- “他追出去两步，又站住了。”

为什么更好：

- 读者自己能看见后果
- 文本还留着场面余压，不会一下变成主题汇报

### 反例 3：命中降了，但人物同脸了

**不合格**

- 每个人都说完整句
- 每个人都能准确说出自己委屈、愤怒、后悔在哪
- 每个人都在有条理地推动情节

为什么假：

- 这不是“更高级”，而是人物被统一成一种会表达的功能人
- 它很容易让正文变成安全成熟块

**更合理**

- 有的人先岔开
- 有的人先翻旧账
- 有的人根本不解释，只做动作
- 有的人越想装稳，越漏出真实偏向

为什么更好：

- 人物不同脸才是正文活性
- 同一场戏里，说话权限和动作权限本来就不该平均分配

### 证据链完整性检查（复仇/打脸文专用）

| 检查项 | 标准 |
|--------|------|
| 证据是否分章释放？ | 至少分3次揭露，不能一次全给 |
| 每个证据是否有铺垫？ | 前文必须埋过线索 |
| 反派是否每次都先得意再被打脸？ | 必须是——先扬后抑 |
| 最终证据是否最致命？ | 最后一个证据要改变全局认知 |
| 是否有"定时炸弹"证据？ | 至少1个主角提前布局的证据 |

### 毒点 / 代入感 / 震惊 / 开头 / 结尾 / 情绪 / 期待感速查

> 这里是速查版。真要细查毒点，再去看 `anti-ai-writing.md`。

| 检查项 | 标准 |
|--------|------|
| 爽文不爽 | 金手指效果必须展示清楚 |
| 压制无目的 | 每次压制必须服务于后续爆发 |
| 反派结局与主角无关 | 反派之死必须因主角行动导致 |
| 经济/战力崩坏 | 设定前后一致，以普通人为锚点 |
| 女主/女配降智 | 人设正常可写，老套路别写 |
| 读者期待长时间不满足 | 适时满足或引入新期待 |
| 主角行为不可理解 | 必须能理解、能共鸣 |
| 氛围被突兀梗破坏 | 氛围 > 突兀梗（乐子文除外） |
| 钩子不回收 | 钩子不回收 = 烂尾 |
| 震惊分层 | 点→网→深度，不能只写点震惊 |
| 震惊阶梯递进 | 阶梯式上升，非直线 |
| 震惊有广度 | 关系网都要有反应 |
| 前50字有冲突/异常 | 不能是背景铺垫 |
| 前100字知核心矛盾 | 必须知道 |
| 开头情绪强度 | ≥7（1-10） |
| 结尾是具体的 | 动作/对话/画面，禁止总结/反思 |
| 结尾有余韵 | 读者还想往下看 |
| 结尾情绪强度 | 虐≥8，爽≥7，治愈≥6 |
| 情绪对等释放 | 虐多少补多少，反派结局关联主角 |
| 期待管理 | 两长一短不断，核心卖点贯穿 |
| 付费期待 | 读者始终知道“后面最该看的那一下”还没来 |
| 情绪为什么爽 | 说得清前面替谁憋、中段靠什么攒、最后靠什么值回这口气 |
| 爽点类型匹配 | 高潮和结尾给出的爽，确实是起盘承诺的那种爽，不要挂羊头卖狗肉 |
