# 可直接仿写层资产合同

适用场景：

- `仿写`
- `融合`
- `去原作化后再写`
- `先拆后写`

原则：

- 这些表是接口，不是让 skill 主文件塞进大词库
- 表里记录的是功能、顺序、口气差异、表演层资产，不是堆漂亮话
- 缺关键表时，优先回上游补拆，不要拿概括结论硬写

## 最小字段

### 1. 导语拆解表

| 字段 | 说明 |
|------|------|
| 前20字 | 是否已经起事 |
| 前60字 | 是否完成关系错位 / 事故信息 / 后果预期中的至少 2 项 |
| 前80字 | 是否出现第二推进点 |
| 第一句功能 | 越界现场 / 结果先行 / 关系错位 / 反常反应 |
| 第二句加码 | 抬后果 / 抬危险 / 抬掉位 |
| 继续滑动力 | 读者为什么会继续往下滑 |

### 2. 顺序事件表

| 字段 | 说明 |
|------|------|
| 编号 | `01 / 02 / 03 ...` |
| 谁做了什么 | 必须具体到人 |
| 具体动作 | 不接受空泛概括 |
| 这一拍功能 | 起事 / 压位 / 留钩 / 翻刀 / 抬后果 |
| 可迁移写法 | 换壳后还能怎么用 |

### 3. 物件表

| 字段 | 说明 |
|------|------|
| 物件 | 关键承重件 |
| 首次出现 | 节位 / 场面 |
| 后续回收 | 回收点 |
| 伤害层 | 体面 / 旧伤 / 身份 / 关系位置 / 亲情 |
| 可替换功能件 | 换题材后的同功能件 |

### 4. 动作表

| 字段 | 说明 |
|------|------|
| 动作 | 抢 / 扔 / 拦 / 护 / 留证 / 装无辜等 |
| 谁做 | 动作发起人 |
| 对谁做 | 动作承受人 |
| 当下效果 | 当场怎么压位 |
| 后续效果 | 为后文埋了什么 |

### 5. 对白功能表

| 字段 | 说明 |
|------|------|
| 角色 | 主角 / 压迫者 / 接住者 / 旁观者等 |
| 典型说法类型 | 挑衅 / 找补 / 压人 / 装无辜 / 切割关系 |
| 这类话负责什么 | 拉仇恨 / 洗白自己 / 逼位 / 翻盘 |
| 口吻特征 | 称呼、长短句、先忍还是先刺 |
| 明显不像他说的话 | 用来拦错配口气 |

### 6. 对话衔接表

| 字段 | 说明 |
|------|------|
| 上句功能 | 上一句在逼什么 |
| 下句接法 | 反问 / 冷接 / 装傻 / 转移 / 压回 / 停顿后接 |
| 动作垫句 | 有没有用动作垫一下再说 |
| 称呼变化 | 是否改称呼制造关系变化 |
| 衔接效果 | 为什么这一接更刺人 / 更压人 / 更显心冷 |

### 7. 误判表

| 字段 | 说明 |
|------|------|
| 误判方 | 读者 / 角色 / 群体 |
| 先误判了什么 | 早期认知 |
| 从哪开始翻 | 触发点 |
| 翻完后前文怎么变 | 重读价值 |

### 8. 钩子表

| 字段 | 说明 |
|------|------|
| 位置 | 段末 / 节末 / 场末 |
| 钩子内容 | 留了什么问号或后果 |
| 钩子类型 | 越界现场 / 结果先行 / 信息差 / 掉位后果等 |
| 读者下一步在等什么 | 明确期待 |
| 回收位置 | 哪一拍兑现 |

### 9. 微动作表

| 字段 | 说明 |
|------|------|
| 动作本体 | 捏杯口 / 把话咽回去 / 手指停住等 |
| 对应情绪 | 心冷 / 发紧 / 憋屈 / 防御 |
| 角色状态 | 忍 / 试探 / 硬撑 / 装平静 |
| 替代的解释句 | 替掉了哪类“她很难过”式解释 |
| 可迁移题材 | 适合迁去什么题材 |

### 10. 安静压迫场表

| 字段 | 说明 |
|------|------|
| 场面压力来源 | 谁压谁，靠什么压 |
| 谁没说话 | 沉默者是谁 |
| 环境音 / 物件 | 关门声、筷子声、手机震动等 |
| 未说破结果 | 大家都知道但没说破的后果 |
| 适用公开场 | 饭桌 / 医院 / 婚礼 / 家族群 / 签字场等 |

## 推荐读取顺序

1. `导语拆解表`
2. `顺序事件表`
3. `物件表`
4. `动作表`
5. `对白功能表`
6. `对话衔接表`
7. `误判表`
8. `钩子表`
9. `微动作表`
10. `安静压迫场表`

规则：

- 先读导语和事件，再读对白和表演层
- 先读功能，再换壳
- 找到这些表时，先吃表，再读 `拆文报告.md`

---

## 缺库处理

- 缺 `导语拆解表`：不要直接写开头
- 缺 `顺序事件表`：不要直接重组骨架
- 缺 `对白功能表` 或 `对话衔接表`：不要硬写对白
- 缺 `微动作表`：不要只靠情绪词和解释句顶场
- 缺 `安静压迫场表`：高潮和公开场容易只会写吵
- 缺任意 2 张以上：默认还停在概括层，先回拆文阶段补

---

## 主文件中该怎么用

- `story-short-analyze`：把这 10 张表当作固定产出合同
- `story-short-write`：把这 10 张表当作固定读取合同
- 需要扩充例句、词库、反例时，优先放进独立 reference，不把主 skill 写成大词表
