# profile_source 模板

这份文件用于 `写作资产/profile_source.md`。

定位：

- 这是模型先提的原始规则层
- 不是最终 `book.profile.json`
- 作用是把脚本难抽出的隐性信息先固定下来，再交给脚本标准化

使用要求：

- 必须基于原文和完整拆书结果填写
- 不要写空泛题材介绍
- 每个字段尽量给“短语 + 简短说明”
- 能落到桥段、物件、动作、场面、句壳，就不要只写抽象结论
- 先做 `样本分级`，再决定哪些字段能进正向 DNA，哪些只能进禁写规则
- `开头高信息量信号` 里，脚本硬读的是独立多行的 `- 开头信号：`
- `禁句 / 禁写法` 里，验收脚本硬读的是独立多行的 `- 为什么假：`
- 不要把 3 个开头信号并到 1 行，也不要把 2 条“为什么假”写成 `为什么假1 / 为什么假2`
- `桥段承重件` 每个桥段必须补齐 6 个字段，不然脚本会把 `must_keep / why_order_matters / why_original_passes` 抽空
