# 全局成文形状人工审查

这份审查解决局部窗口通过、整篇仍像“AI按大纲展开”的问题。它不是要求故意写乱，而是检查全篇是否保留真实叙述的不均匀性。

## 写作前必须消费的拆书结论

如果项目有主体拆书，写作模型在生成设定、大纲和正文前必须读取并逐项消费：

- `拆文报告.md` 的 `### 全局成文形状审计`
- `写作手法.md` 的 `## 10. 全局成文形状审计`
- `写作资产/样本分级与可学层.md` 的 `## 4.4 全局成文形状审计`

这三处不是重复阅读项，而是三种用途：

- `拆文报告.md`：确认原作整本的结构风险和叙事事实。
- `写作手法.md`：确认哪些章法、对白和细节是可学层，哪些是反面成品感。
- `样本分级与可学层.md`：确认当前融合稿能继承什么、必须避开什么。

五项结论必须在规则执行台账中逐项留下 `source_contract_reviews`：

1. `全局结构形状`
2. `章尾收束模式`
3. `主角不规则性`
4. `专业细节功能性`
5. `全文对白模式`

台账中的 `applied` 只表示该结论确实转译进设定、大纲或正文；`not_selected` 必须说明
为什么当前题材不适用；只写“已读取”不算消费。写作阶段不要求机械复制原作问题，必须
先判断它是正向 DNA、反面规则、题材限制还是本稿不适用。

## 四项必查

### 1. 全局结构与章尾

- 章节数量、标题和每章弧线是否过度同构。
- 是否每章都按“起事 -> 解释 -> 决断 -> 精准短句”收束。
- 至少一处应以动作、旁人打断、生活杂音或未完成事务收束，而不是主题句、象征句或漂亮短句。
- 结构必须服务故事，不得为了制造“人味”随机插入无关事件。

### 2. 主角不规则性

- 主角是否每次都先做正确判断、说准确的话、立刻采取最优动作。
- 至少人工确认一处真实的迟疑、误判、动作失手、答非所问或不体面反应；若题材确实不允许，必须记录理由，不能默认放行。
- 不得把“专业正确”误判成“人格完整”。职业能力和情绪反应要分开审。

### 3. 专业细节功能性

- 每个专业术语、编号、版本号、器材名是否改变判断、权限、风险或后果。
- 连续出现的术语若只负责营造真实感，应删减或换成现场动作。
- 专业细节不能集中堆在开头、高潮或结尾，形成“展示专业度”的成品块。

### 4. 对白模式变化

- 不能只检查某个窗口的连续对白；要看全文是否反复出现“我问 -> 他答 -> 我确认 -> 他解释”的同一回路。
- 至少人工确认一处对白被动作、误听、第三方闲枝、生活事务或答非所问打断。
- 对白模式变化不是强行增加口语废话；每一处保留或回修都要写明场面功能。

## 执行分工

- 脚本：章节数、章尾短句比例、章尾重复、技术细节候选密度、连续对白和重复回路的预扫。
- 人工：是否真的像大纲展开、主角是否过度正确、术语是否承担功能、对白变化是否自然。
- 回执：四项必须逐项填写正文证据、判断、`keep/revise` 决策；任一缺项都不能通过写后人工复核。

## 失败处理

- 全局形状问题先回场景分工和章尾，不先替换同义词。
- 主角过度正确先补人物反应或重写动作选择，不机械添加“我不知道”。
- 术语堆叠先删无功能术语，不用新增漂亮感官描写补空。
- 对白重复先拆回避、动作和旁人干扰，不把每句改成碎句。
