# 短篇格式规范与小节结构

> 写作前先看这个。下面是当前仓库默认的正文交付格式；用户或目标平台有明确要求时，以用户或平台要求覆盖。
>
> **适用范围**：段落格式（紧密段落、单段通常 ≤60字）和对话格式适用于所有体裁。小节(beat)结构和字数标准仅适用于短篇。长篇小说字数以当前 skill 的字数硬约束为准。

---

## 章节标记

默认格式（按灵活度排序）：

| 格式 | 平台适用 | 示例 |
|------|----------|------|
| `###1.` | 黑岩、点众（主流） | `###1.` `###2.` `###3.` |
| `###第一章` | 部分平台 | `###第一章` `###第二章` |
| `1.`（纯数字） | 知乎 | `1.` `2.` `3.`（无 ### 前缀） |

**规则**：全文统一一种格式，不要混用。短篇默认推荐 `###1.` 或纯数字，够直接，也方便回修。

---

## 段落格式

### 核心规则：紧密段落，不机械等长

默认交付排版是**紧密排列**，不是“全篇所有段都切成一样短”。

- 每段 1-3 句，不超 60 字
- 段与段之间**不加空行**（紧密排列）
- 无缩进（平台渲染器自行处理，不需要 `　　` 或空格）
- 一段只做一个动作：叙事/对话/心理/动作，不混合
- 三维度织入后仍要按镜头断段：新动作、新物件、新信息、新对话另起一段，不拆成动作层/感知层/反应层
- 同一场景允许 1 句段、2 句段、偏长句段交替，不要把正文排成整页等长短句密排

### 段落节奏

连续的短段落（1-2 句）可以拉紧张感。中间偶尔给 2-3 句段当呼吸点，挂主观判断、补充信息或轻微拐弯。但**绝不**出现超过 60 字的大段落，也别把连续很多段都切成差不多长度。

## 对话格式

### 对话标记

按目标平台或用户要求选择；没指定就用默认格式：

| 优先级 | 格式 | 适用平台 |
|--------|------|----------|
| 首选 | `"说话内容"` | 黑岩、点众、番茄（主流） |
| 次选 | `「说话内容」` | 古言、知乎投稿版、部分平台 |
| 平台指定 | `【说话内容】` | 仅在用户或平台后台明确要求时使用 |

**默认用 `""`**。如果用户指定知乎盐选投稿版，默认改为 `「」`；只有用户或平台后台明确要求时，才改为 `【】`。

### 对话规则

1. 对话**独立成行**，不嵌在叙述段落中
2. 用动作代替高频对话标签：别满篇都是“他说”“她道”，改成动作描写 + 对话
3. 两人对话连续出现时，能省标签就省，靠内容区分说话人

**正确示例**：
```
她把杯子放下。
"你走吧。"
他没有动。
"我说，你走吧。"
```

**错误示例**：
```
她把杯子放下，说道："你走吧。"他没有动，她又说："我说，你走吧。"
```

## 小节（beat）结构

### 基本规则

- 用数字编号（`1` `2` `3`）分割小节，每个小节都是一个完整的叙事 beat
- 每小节 800-1500 字（爽文可压缩至 500-800 字/节，见 `genre-writing-formulas.md`）
- 8-15 个小节覆盖全文（按目标字数反推；8000÷1000≈8节，12000÷800≈15节）
- 每小节推进一个明确的情节点

### 小节内部结构

每个小节应该有：

1. **一个主事件** + **3-5 个子事件**（主事件推进核心情节，子事件把层次撑起来；初稿偏短时先回到小节或细纲补足计划内子事件、对话或冲突，再写正文；去AI味已有正文时不得新增剧情）
2. **一个情绪变化**（读者的感受如何变化）
3. **一条读者新获知的信息**
4. **3-5 轮对话交锋**（揭示关系/升级冲突，参考 `writing-craft.md` 对话权力博弈。特定场景如独自发现/翻阅日记可标零，按当前 skill 的细纲/设计任务要求执行）
5. **每个子事件三维度织入**：发生+感知+反应织入同一段正文（参考 `writing-craft.md` 场景写法）


### 小节之间的衔接

- 小节结尾留一个钩子（悬念、没解掉的情绪、或新问题）
- 下一节开头快速接续，不要重新铺垫
- 情绪跨节递进：每一节的情绪强度 ≥ 上一节。例外：峰值情绪（反转节）后允许维持 1 节不降，但不允许骤降

## 平台对话格式覆盖表

| 平台 | 章节标记 | 对话格式 | 特殊要求 |
|------|----------|----------|----------|
| 黑岩 | `###1.` | `""` | 无 |
| 点众 | `###1.` | `""` | 无 |
| 知乎盐言 | `1.` | `「」` | 导语需单独标注；若平台后台要求 `【】` 则覆盖 |
| 番茄 | `###第一章` | `""` | 首段需有吸引力 |
| 红果 | `###1.` | `""` | 无 |

**通用原则**：用户没指定平台时，默认使用黑岩或点众格式（`###1.` + `""`）。

## 5 条绝对禁止

下面这 5 条，写作全程都生效，不因题材或风格改变：

1. **禁止大段落**：超过 60 字的段落必须拆分
2. **禁止段间空行**：段落紧密排列，不留空行
3. **避免对话标签机械化**：高频或公式化的「他说」「她道」「他笑了笑说」用动作或上下文替代；普通“说”可以保留
4. **禁止缩进**：不使用 `　　`（全角空格）或半角空格缩进
5. **禁止正文段落 Markdown 渲染**：除统一的小节/章节标记（如 `###1.`）外，正文段落中不使用加粗 `**`、斜体 `*`、标题 `#`、分隔线 `---` 等 Markdown 语法

## 快速检查

写完每一段后，用这张表自己过一遍：

- [ ] 每段 ≤ 60 字？
- [ ] 是否避免连续大量等长短句密排？
- [ ] 三维度织入段是否按镜头/信息变化断开，而不是一段到底？
- [ ] 段间无空行？
- [ ] 对话独立成行且用正确引号？
- [ ] 对话标签是否避免高频/公式化，普通“说”是否按语境保留？
- [ ] 无缩进？
- [ ] 每个小节有事件+情绪+信息？
