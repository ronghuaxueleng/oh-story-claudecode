# `writing-workflow.md`：短篇写作工作流

这份是 `story-short-write` 的详细工作流。主 `SKILL.md` 只留总口径，这里把真正落地怎么做说细。

---

## Phase 2 设计任务（框架定住后再做）

### 开始前先把该读的先读掉

- 写结构物件 → 加载 `writing-craft.md` ✅/❌
- 写反派/揭露方式 → 加载 `villain-and-reveal.md` ✅/❌
- 写大纲 → 已有 `genre-writing-formulas.md` 的题材速查表即可 ✅/❌
- 反转信息差验证 → 需加载 `reversal-toolkit.md` 或 `writing-craft.md` ✅/❌
- 伏笔回查 → 需加载 `writing-craft.md` ✅/❌
- 某项如果确实不适用，比如这篇根本没有反派，标注原因后可以跳过，不要硬凑
- 两份读取门禁通过后 → 已初始化 `规则执行台账.json`，已导出并由当前写作模型逐族阅读 `cases`、归纳 `canonical_rule_text`，确认角色、修复目标、执行方式和适用性，可合并项已归一 ✅/❌
- 本阶段每执行完一条规则就立即标记并补证据，不允许阶段结束后批量补“已使用” ✅/❌

### 设计步骤

1. **设计结构物件**（1-2 个）：用 `writing-craft.md`「结构物件系统」模板
   - 填写：`- 第1现（第X节）：{场景}（{含义}）` 重复三次，含节号定位
2. **设计反派**（如有）：用 `villain-and-reveal.md` 反派模板
   - 填写：`{身份} + {动机} + {作恶方式} + {致命弱点} + {报应}`
3. **确定揭露方式**：从 `villain-and-reveal.md` 4 种真相揭露机制中选一种
4. **写 `小节大纲.md`**：字段必须写全，基准字段如下：
   `主事件 | 子事件×3-5 | 情绪 | 读者新获知什么 | 钩子 | 伏笔/物件 | 动静 | 对话密度 | 目标字数`
   - **默认推荐写成 Markdown 好读的格式**：每节一个 `## 节次. 标题`，下面用项目符号列出这些字段。
   - 如果只是做机器处理，或者用户明确要纯计划表，再压成“每节 1 行”。
   - 不管哪种排版，字段顺序和信息都不能缺。
   - `小节大纲.md` 是规划文档，不使用 `正文.md` 的 `###1.` 之类正文小节标记。
   - 子事件必须有明确功能（推剧情 / 铺反转 / 升情绪），别塞那种看着热闹、实际不承重的子事件
   - 子事件不够 3 个时，用扩展规则补（见 `writing-craft.md` 第 7 节「子事件不够时怎么扩」）
   - 子事件用 `->` 连接，每个子事件包含具体情节点描述+功能标签：
     `具体发生了什么{功能标签}` 如 `发现账单上多了一笔4800{发现}` / `把戒指冲进马桶{冲突}` / `他笑了{递进}`
     功能标签：`对话` / `冲突` / `伏笔` / `回忆` / `发现` / `递进`
     情节点描述要具体到"谁做了什么"，不能只写标签如`对话`或`冲突`
   - 动静标记：`动` / `静` / `动静`，相邻两节尽量交替；如果题材需要连着静，也得保证节内情绪还在往上走
   - 对话密度：`高`（3+轮）/ `中`（1-2轮）/ `零`（全节无对话），尽量别连着两节都标 `零`；真要连着静场，也得靠感官、动作和身体反应把情绪托住
5. **反转信息差验证**（大纲写完立刻验，别等正文写歪了再回头）：
   - 回答：读者在第几节知道真相？用「揭穿节号 ÷ 全文总节数」计算位置百分比
     - < 50% → 这更像延迟揭露，不太像真正的反转
     - 50-60% → 灰区，要再看误导和后劲够不够
     - > 60% → 位置通常更稳
   - 检查：至少有 1 条真能误导读者的假线索，不是那种一眼就穿的摆设
   - 检查：反转前至少铺了 3 条真线索，线索要藏着，不要明晃晃把答案摆出来
   - 验证：找个人，或者自己模拟一遍，只看大纲能不能太早猜到真相；太早猜到，就说明设计还不够
6. **伏笔回查清单**（大纲写完就查）：
   - 列出结尾所有关键元素（物件/信息/角色/关系变化）
   - 每个结尾元素在前文有 ≥1 次铺垫？没有 = 天降，必须补铺垫
   - 结构物件三现完整？（第 1 现在铺垫段，第 2 现在反转段，第 3 现在结尾段）
   - 铺垫节号与揭穿节号的间距 ≥ 3 节（间距太短铺垫无效）

### 起盘补强口径

- 写大纲前，先单独写一句 `付费期待`，别等正文写一半才发现读者根本不知道要等什么
- `主冲突` 负责一句话里的不公平，`副升级器` 负责给中段供氧、给题面抬层，两个不能撞车
- 最显眼的矛盾，和中段再加的那层事，不能写成一个东西。前者负责把人拉进来，后者负责把人留下来
- 开头那一节要先把“最想看的后果”亮出来，不要只摆处境，不给结果期待
- 对标原文时，不只借情节，必须顺手写出主要角色的 `对白口气模板`
- 开头钩子不只写内容，还要标 `钩子类型`，优先从 `越界现场钩 / 结果先行钩 / 反常反应钩 / 身份误差钩 / 群体围剿钩 / 不可逆后果钩 / 旧账回钩 / 怪物件钩` 里选
- 起盘卡里必须补出 `异物`，它负责把这一题从普通老桥段里拧出来
- 起盘卡里必须补出 `第二层冲突`，它不能和表层矛盾重复
- 起盘卡里必须补出 `关系重组方式`，要能回答“翻盘后谁离开旧位置，谁接手新位置”
- 起盘卡里必须补出 `公开场后果`，要能回答“高潮炸完以后，对方具体掉了什么位、丢了什么脸、失了什么人”
- 起盘卡里最好顺手补 `新状态`，也就是主角中段会进入什么新生活 / 新关系 / 新行动结构
- 想写虐点时，先补 `虐点对照细节`，不要只写“很惨”，要写同一人物对不同对象的态度差，或者一句很轻的轻慢动作
- 起盘卡里必须回答 `情绪为什么爽`：前半段读者先替她憋什么，中段靠什么把这口气越攒越大，最后靠什么把这口气值回来
- 起盘卡里必须标 `爽点类型`，至少写清这是 `打脸爽 / 追不回爽 / 位置反转爽 / 认知翻车爽 / 自由脱身爽 / 被坚定选择爽 / 价值爽点` 里的哪一种或哪两种
- 如果大纲看起来“什么都有”，但没有哪一下特别疼、特别狠、特别丢脸、特别掉位，说明还没起稳
- 题面不等于故事。题面只是入口，后面至少还要有起因、关系错位、再加的那层事、高潮场合和关键证据
- 开头不要只让主角难受，要让读者知道“她后面会怎么翻，或者对方后面会怎么倒霉”
- 回炉排序固定为：桥段承重件/后果链 > 场戏功能和人物关系漏出 > 段落局部压味 > 短段节奏微调。不要先拿“短段不短段”当主问题。
- 字数回炉也服从同一个顺序：如果只是轻微超出目标字数，不要先为压字数删桥段承重件、微动作、秩序件和人味段。先把桥段写对、场面写稳，再回头压重复解释、匀速段和废流程。
- 如果已经有 `book.profile.json / project.profile.json`，桥段层默认不只检查 `must_keep`，还要一起检查：
  - `opening_pattern`
  - `recommended_sequence`
  - `fake_signals`
- 桥段回炉默认顺序：
  - 先看原桥段怎么起手
  - 再看原桥段顺序有没有被写乱
  - 再看承重件缺没缺
  - 最后才看句子顺不顺
- 如果内部审计把全文判成 `single_global_block`：
  - 禁止先做句面回修、短段整理、同义词替换
  - 必须先重构主桥起手、桥段顺序、场戏分工、人物偏手、烂关系漏出
  - 必须先看粗粒度大块，再看正文块，再看局部段落
- 如果内部审计判成 `coarse_blocks`：
  - 先修前 1-2 个粗粒度大块
  - 每个大块里仍按 `起手 -> 顺序 -> 承重件 -> 句面` 的顺序回炉

### 原文里反复出现、要写进方法的几条

这几条不是二手总结，是回看原文摘要后反复撞出来的：

- 开头不是介绍，是先把关系、偏心和倒霉位置压清楚
- 不要围着事件本身打转，要拆事件带来的连续情绪反应
- 付费点前要持续压情绪，不是一件事写完就算
- 情绪不是靠大词堆出来的，而是靠一连串更难受的瞬间顶出来的
- 前期反击可以失败，失败本身也能继续加重她的难受和心死
- 重点不是把背景交代全，而是很快让读者知道她为什么难受、为什么还不走、为什么后面一定会走

### 题面和导语别起平了

从原文看，题面和导语能不能拉点击，靠的不是“信息全”，而是“先把最刺人的那一下亮出来”。

- 先写关系错位，再写事件
- 先写丢脸、背叛、让位、误认、迟到后悔这些感觉，再补背景
- 先让读者知道后面最想看的结果是什么，再补细节

常见起法：

- `意外发现`：以为自己是 A，结果根本不是
- `强对比`：她在低处忍，他在高处踩
- `三人同场`：旧人新人和她碰到一处
- `开场即离`：签字、退婚、搬走、断联当天出事

导语只做三件事：

1. 先把最刺的关系摆出来
2. 再把最想看的后果亮出来
3. 最后给一个还会更狠的口子

### 导语怎么写得更有感觉

导语不是只交代事实，而是：

- `事实`
- `情绪方向`

读者不只要知道发生了什么，还要立刻感觉到这篇文是：

- 疼
- 恨
- 憋
- 冷
- 舍不得

最稳的 5 种导语起法：

1. `关系 + 伤口`
   先交代我和他是什么关系，再交代这段关系留下了什么伤口
2. `结果 + 余痛`
   我已经离开/不要他了，但这件事留下的痛感还在
3. `静态表面 + 动态心口`
   表面没出事，心里已经凉了、乱了、死了
4. `一句话埋关系倾斜`
   他爱谁，我在什么位置
5. `一句话埋结果预期`
   现在还没彻底翻，但读者能感觉到后面一定会炸

导语写完后，至少过这 4 条：

1. 读者知道人物关系了吗
2. 读者知道这段关系哪里不对了吗
3. 读者能感觉到一种情绪了吗
4. 读者会不会想看后面结果

### 题面包装检查

题面不是写得文艺，是让人三秒内看懂：

1. 这是什么赛道
2. 冲突够不够狠
3. 卖点硬不硬

最稳的题面结构：

- `困境 + 反击`
- `关系 + 背刺`
- `场景 + 炸点`
- `旧题 + 新变量`

题面不要写成：

- 只有情绪，没有事件
- 只有题材，没有冲突
- 只有设定名，没有代价

快速检查：

- 删掉修饰词后，核心冲突还成立吗
- 编辑三秒内能看懂卖点吗

### 平台适配硬检查

平台别只定名字，定完以后还要立刻过这 5 条：

1. 这个平台要的是快钩，还是慢后劲
2. 题面要多直
3. 开头能不能再提速
4. 高潮更偏价值回收，还是更偏公开翻车
5. 结尾要不要更硬、更外显

最容易写错的几种情况：

- 把小红书那种讲拆法的口气直接拿来写正文
- 把盐选的慢后劲直接套去小程序，结果开头不够猛
- 把小程序那种土猛狠直接套进盐选，结果只剩响，不剩后劲
- 题面像一个平台，正文却像另一个平台

同一题换平台时，最该一起调的是：

- 题面力度
- 开头速度
- 中段事件密度
- 高潮表现方式
- 结尾落点口气

### 融合写作调用顺序

如果用户明确要“融合写作”，或者项目里已经有拆出来的题材素材和结构素材，不要直接拼桥段，固定按这个顺序来：

1. 先读参考稿的 `导语拆解表`
2. 再读 `顺序事件表`
3. 再读 `物件表`
4. 再读 `动作表`
5. 再读 `对白功能表`
6. 再读 `对话衔接表`
7. 再读 `误判表`
8. 再读 `钩子表`
9. 再读 `微动作表`
10. 再读 `安静压迫场表`
11. 再定 `表面题材`
12. 再定这篇最特别的地方
13. 再定故事怎么走
14. 再补中段要再加的那层事
15. 再选 `事件链`
16. 再选 `开头钩子`
17. 再补 `异物 / 第二层冲突`
18. 再选高潮放在哪个场合、靠什么证据翻过来、炸完后对方怎么掉位
19. 最后才补 `题面 / 导语 / 口气`

把它再压成能力模块，就是：

- `结构模块`
  负责：故事怎么走、中段怎么再加事、高潮怎么回收
- `题面模块`
  负责：题面、导语、平台口气、一句话卖点
- `开头模块`
  负责：前三句、第一屏、起事速度、后果预期
- `情绪模块`
  负责：主情绪母线、心冷点、切割点、结尾后劲
- `细节模块`
  负责：微动作、眉眼、说话动态、物件、小件刺激、题材局部件

后面一律按模块调用，不按任何“某人负责什么”的口径来。

补充硬规则：

- 没读到 `导语拆解表 / 顺序事件表 / 物件表 / 动作表 / 对白功能表 / 对话衔接表 / 误判表 / 钩子表 / 微动作表 / 安静压迫场表` 之前，不要直接做融合
- 只拿“故事梗概”去融，最后大概率会变成拼盘
- 先读功能，再换外壳；不要先抄名词，再硬改设定

### 融合 profile 重生口径

如果这一轮：

- 新增了拆书
- 补强了 `profile_source.md`
- 补强了 `同桥段过检规则.md`
- 重生了单书 `book.profile.json`

那写新稿前，默认先重生一次本轮题材的 `project.profile.json`，不要沿用旧融合包。

默认命令：

```bash
python3 "$CODEX_HOME/skills/story-short-write/scripts/generate_story_profile.py" \
  --merge-profile-dir '拆文库' \
  --name '{项目名}' \
  --output 'profiles/{项目名}.project.profile.json'
```

口径：

- 有整批题材拆文时，优先用 `--merge-profile-dir`
- 只在你明确只选少数几本书做融合时，才手工写多个 `--merge-profile`
- 重生完以后，正文和审计一律读取这次新出的 `project.profile.json`
- 不要一边补单书 profile，一边继续拿昨天的旧融合包写今天的新稿

### 融合写作落地卡

融合不是把两个热梗缝一起。

最稳的落地法是：

1. 先定 `大骨架`
2. 再定 `中段再加的那层事`
3. 再定 `高潮公开场`
4. 再定 `情绪母线`
5. 最后才补 `题面和导语`

可以直接这样填：

```text
表面题材：
大骨架：
主冲突：
副升级器：
中段再加的那层事：
关系场：
开头钩子：
开头钩子类型：
异物：
第二层冲突：
高潮公开场：
公开场后果：
关键证据：
情绪母线：
情绪为什么爽：
爽点类型：
关系重组方式：
角色口气模板：
新状态：
虐点对照细节：
结尾落点：
一句话题面：
一句话导语：
```

硬规则：

- 大骨架只定一个，别贪多
- 主冲突和副升级器不能抢同一个功能位
- 中段再加的那层事不能和大骨架重复
- 异物要能把这篇从同题材里拧出来，不能只是换个名词
- 第二层冲突要真把事情推到更坏，不是把表层冲突再说一遍
- 开头钩子类型要服务主卖点，不要为了花而花
- 高潮场一定要选对方最怕丢脸、最怕掉位的地方
- 公开场后果必须能看见，不能炸完就完
- 情绪为什么爽必须能写成一句完整人话，不能只写“很爽”“很解气”
- 爽点类型必须和结尾落点对得上，别前面写追不回爽，最后却只给嘴炮打脸
- 关系重组方式必须落到具体人，不要只写“女主成长了”
- 角色口气模板至少写主角、对手、接住者 3 个
- 新状态必须和前段受压状态不一样，不然中段容易塌
- 虐点对照细节优先写具体轻慢，不优先写抽象惨词
- 情绪母线最好只压一条主线
- 题面负责点击，不负责替你推进全文
- 如果想冲更高价位，至少再补一层：更狠的冲突 / 更强的升级器 / 更清楚的新变量
- 上游参考稿如果没有拆出 `导语拆解表 / 顺序事件表 / 物件表 / 动作表 / 对白功能表 / 对话衔接表 / 误判表 / 钩子表 / 微动作表 / 安静压迫场表`，先回拆文阶段补，不要硬起新稿
- 上游参考稿如果没有 `样本分级与可学层.md`，先回拆文阶段补，不要直接把整本当 DNA 源
- 上游参考稿如果是 `B类骨架样本`，只拿它的骨架、承重件、后果链和场面秩序，不拿它的现成句法壳
- 上游参考稿如果是 `C类负样本`，只并入反面规则，不并入正向融合 profile
- 上游资料如果新增了 `profile_source.md`、补强了 `同桥段过检规则.md`，或新增了拆书样本，开写前必须重生 `project.profile.json`
- 融合包不允许跨书把同序号桥段糊成同一条通用桥段规则

### 最短闭环检查链

如果你怀疑“资料都补了，但写出来还是平”，不要回头漫修，直接按这条短链查：

1. 起盘卡里有没有 `异物`
2. `第二层冲突` 有没有真的把事情推到更坏
3. 上游有没有 `顺序事件表`
4. 上游有没有 `物件表 + 动作表`
5. `角色口气模板 / 对白功能表` 有没有写到能直接拿来写对白
6. 高潮有没有 `公开场后果`
7. 结尾有没有完成 `关系重组`

这 7 条里缺 2 条以上，说明不是句子问题，是融合和骨架还没立住。

再补 2 条情绪检查：

6. `情绪为什么爽` 说不说得清
7. `爽点类型` 和结尾落点对不对得上

这 2 条答不上来，说明你可能知道故事在发生什么，但还不知道读者为什么会爽。

### bridge_rule_audit 口径

如果内部审计已经产出 `bridge_rule_audit`，回炉顺序固定按这 5 步走：

1. `opening_pattern_missing`
2. `recommended_sequence_missing / recommended_sequence_out_of_order`
3. `must_keep_missing`
4. `must_avoid_hit / fake_signal_hit`
5. 句壳、热点、短段节奏

不要倒着做。先修句壳，再回补桥段，通常只会把正文修成更顺的说明稿。

### 更值钱的爽点长什么样

不是所有爽点都值钱。

低档爽点常见问题：

- 只会骂
- 只会打一巴掌立刻赢
- 只有一层冲突

更值钱的爽点通常多一层：

- 关系更狠
- 后果更大
- 创新更清楚
- 回收更完整

最稳的高价位爽点：

1. 不是单点打脸，而是整局回收
   前面受的气、中间埋的线、后面一起爆
2. 不是普通情伤，而是叠加利益、名分、身份
3. 不是主角嘴更狠，而是主角位置变了
4. 不是常规追妻复合，而是追不回

所以真要冲更高价，不要只问“这句狠不狠”，要问：

- 这次翻盘有没有让前文所有受气一起值回来
- 这次反击除了情绪，还动了钱、名分、身份、位置里的哪一层

### 高潮卡点和二次翻刀怎么安排

高潮不值钱，通常不是因为不够吵，而是只做了一件事：

- 把真相说了
- 让主角赢了
- 让反派丢脸了

更稳的高潮，至少要同时做到这 3 条：

1. 把前文一直压着的东西放出来
2. 在最该公开的场面炸开
3. 炸完以后，让读者回头看前文时觉得意思变了

高潮前最稳的卡点法：

- 先给大场快到了
- 再给主角异常平静
- 再让读者知道她手里不是空的
- 再让对方把局做得更大

二次翻刀最常见的几种：

- 动机翻过来：原来她不是忍，是在等
- 关系翻过来：原来站队根本不是表面那样
- 利益翻过来：原来不只是感情，是有人一直在吃她的位
- 真相翻过来：原来前文那句、那件旧物、那次沉默都另有意思

高潮禁区：

- 只靠吵
- 硬牌临时从天上掉下来
- 高潮放在太私密的场，炸不开
- 翻完就完，没有后效应
- 知道了真相，但前文没有一起变得更狠

### 融合写作的抽卡分工

融合时不要整套照搬某一份参考，按功能抽：

- `故事走法清单`：决定这篇到底怎么走
- `事件链模板库`：决定中段怎么不发平
- `情绪节点模板库`：决定读者每一步收到什么感觉
- `高频变量池`：负责替换导火索、关系场、当众摊开的场面、关键证据、切割动作
- `题面/导语模板`：负责点击，不负责全篇推进
- `对白口气模板`：负责人物不是一个声道

抽卡时再补 3 条硬规则：

1. 一次只定 `1 个母结构 + 1 个副升级器 + 1 条主情绪母线`
2. 表演层素材和题材局部件只能给骨架服务，不能反过来抢戏
3. 如果一个新点子既不加重不公平，也不抬高后果，就先别塞

最稳的融合顺序不是“多”，而是“深”：

- 先把最显眼的不公平钉死
- 再把中段那层新麻烦钉死
- 再把高潮公开场和硬牌钉死
- 最后才给人物补口气、动作、题材件

如果写着写着发现：

- 题面很狠，但中段只是重复吵架
- 情节很多，但没有哪一下真疼
- 设定很花，但高潮不用它也能成立

那就不是融合写法，是拼盘。

融合时最容易写废的三种错法：

- 只换题材词，不换中段升级法
- 只会加设定，不会让设定参与对抗
- 高潮只靠骂赢，没有证据、没有场合、没有掉位

### 反直觉怎么用

反直觉不是故弄玄虚，是让读者第一反应先走错一步，后面再被真正值钱的结构拽回来。

最稳的 4 种用法：

1. `换主角反应`
   常规是哭、闹、争；反直觉是先顺着演、先签、先退，后面再收网
2. `换核心矛盾`
   表面是追妻，底下其实是身份、利益、选择权
3. `换受害者位置`
   读者以为最惨的是这一层，后面发现还有更深一层被吞掉
4. `换结局预期`
   表面像要复合，最后其实是弃爱重选、二次择偶、利益切割

反直觉必须同时满足两条：

- 前面有铺垫
- 后面能回收

不然就是硬拧。

### 从原文里拿写作技巧，不只是拿情节

至少要顺手拿这几种：

1. 开头前三句怎么分工
2. 主角怎么不靠解释，只靠动作让人看懂她变了
3. 对方怎么一步步越界，而不是一次坏完
4. 中段怎么插新事，避免一直重复受气
5. 高潮怎么先让对方得意，再让他掉下来
6. 结尾怎么不只是把事说完，而是多留一下余味

### 情绪怎么一层层往上加

原文里讲情绪，重点不是“这节她多难过”，而是“这节又多了一层什么难受”。

可以按这个顺序加：

1. 先受委屈
2. 再被误解
3. 再被区别对待
4. 再被公开羞辱
5. 再发现自己连解释机会都没有
6. 最后才走到心冷、断掉、离开

如果连续两三节只是同一种受气，读者会觉得平。

再记一条总口径：

- 情绪不是角色心情，是主角当前处境先砸给读者的感觉

所以写情绪时优先问：

1. 这一节读者会先替她感觉到什么
2. 这种感觉是因为谁做了什么，不是因为作者说她很难受
3. 这一节比上一节多出来的，不公平具体在哪

如果答不出第 3 条，说明这一节大概率没升级。

### 情绪模板怎么套

按原文和你现有模板库看，短篇最稳的不是“情绪越多越好”，而是先定一条主情绪母线，再让每一节往前走。

常用母线：

- `委屈 -> 心寒 -> 决绝 -> 释然`
  适合：追妻、婚恋轻贱、身份错位
- `得意/依赖 -> 错愕 -> 羞辱感 -> 觉醒`
  适合：重生火葬场、旧情被打翻
- `压抑 -> 反复拉扯 -> 高潮脱身 -> 新生`
  适合：囚禁、病娇、束缚关系、家庭压迫
- `心疼 -> 被珍视感 -> 安心 -> 满足`
  适合：男二上位、双向救赎、后段转甜
- `愤怒 -> 不甘 -> 成长 -> 掌控感`
  适合：大女主、重生改命、低位翻盘
- `边气边疼 -> 动摇 -> 坚定 -> 温暖收束`
  适合：复杂关系、真假千金、古言关系文

套法不是把这几个词写进正文，而是每一段先问：

1. 这一段要让读者替她感觉到什么
2. 这一段比上一段更难受在哪里
3. 这一段有没有把她往下一阶段推

例如：

- `委屈 -> 心寒` 之间，通常要补一刀“原本以为他至少会站我，结果他站了别人”
- `心寒 -> 决绝` 之间，通常要补一刀“她发现解释也没用，于是不再求明白”
- `决绝 -> 释然` 之间，通常要补一个“新位置已经站稳”的收束

如果只写成：

- 她委屈
- 她还是委屈
- 她更委屈

那不是情绪推进，是重复受气。

套情绪模板时，再过这一轮：

1. 这段是 `受气`、`看清`、`抽身`、`反击` 里的哪一步
2. 这段有没有一个能看见的承重动作
3. 这段结束后，人物关系有没有更偏一点
4. 这段是不是把读者往下一个情绪阶段送了一步

最容易写错的几种情绪段：

- 只有情绪词，没有事件承重
- 只有事件，没有主角反应
- 只有主角难受，读者却不知道她为什么还不走
- 还没压够，就急着让她翻盘

所以情绪母线不是挂在墙上的词，是每节都得落地成：

- 具体越界
- 具体反应
- 具体关系变化
- 具体后果预期

### 人物反应模板怎么套

人物不是标签，是反应模式。

不要只记：

- 清冷
- 嘴硬
- 病娇
- 美强惨

这些只是壳。真正让人物立住的，是他在关键时刻怎么反应。

最该先定的 4 种时刻：

1. `受伤时`
   是沉下去、硬撑、嘴更硬，还是反过来笑
2. `动心时`
   是靠近、回避、转移话题，还是先装没事
3. `被误解时`
   是解释、不解释、只解释一半，还是把委屈吞回去
4. `失去时`
   是立刻慌、后知后觉、面上稳动作乱，还是嘴上狠心里塌

写作时直接这样做：

- 给每个核心人物先定一套稳定反应模式
- 同一种情绪，不同人物必须反应不一样
- 不要总用解释句定义人物，要让人物自己做出定义

例如：

- 不写：她是个嘴硬的人
- 改写成：她明明疼得发抖，还是把眼泪咽回去

这样人物才不是功能块。

### 表演层怎么调用

前面抓到的那些微动作、眉眼、哭笑、心痛、心动、吃醋，不是拿来堆辞藻的，是拿来替代解释句的。

最稳的用法只有一句：

- `别直接说她怎么了，要先让读者看见她怎么了`

落地时按这 4 步走：

1. 先定这段最先砸中的情绪
2. 再选一个最贴身的身体反应
3. 再补一个能暴露关系位置的细节
4. 最后才决定要不要让人物把话说出来

常见替换法：

- 不直接写“她委屈”，先写手指蜷住、喉咙发紧、话到嘴边又咽回去
- 不直接写“他动心了”，先写眼神停住、语气放轻、动作慢半拍
- 不直接写“她吃醋了”，先写称呼变掉、回话变短、站位往后撤
- 不直接写“他起杀意了”，先写视线冷下来、指节收紧、动作反而更稳

硬规则：

- 一次只抓 1 到 2 个最有效的细节，别满身都在动
- 微动作要和当前关系挂钩，不能只是漂亮
- 眉眼细节优先用来放大态度变化，不要只写好看
- 哭笑描写优先写“忍”“压”“失控前一瞬”，不要只写“她哭了”“他笑了”
- 心痛、遗憾、崩溃这类重情绪，优先拆成身体反应 + 动作中断 + 说话失真

最容易写废的 3 种情况：

- 细节太满，像写真，不像推进剧情
- 只写美感，不写关系和处境
- 明明已经用动作写出来了，后面又补一句解释，把劲卸掉

### 对话和说话动态怎么调用

对话不是把信息说出来，是借说话方式把人分开。

写对话前，先定这 4 件事：

1. 这个人平时怎么叫人
2. 这个人平时句子长还是短
3. 这个人急了会更碎、更冲，还是突然更冷
4. 这个人最常用什么方式护脸面

说话动态别当词库硬塞，主要拿来做这几件事：

- 区分人物声道
- 让情绪升级更自然
- 让潜台词落地
- 让场面里的人际位置更清楚

最稳的写法：

- 强势的人，少解释，多截断，多改口称呼
- 心虚的人，爱补句、绕话、找理由、先替自己开脱
- 已经心凉的人，话会变短，甚至不接那句最该接的话
- 表面温和、内里发狠的人，语气越轻，动作越稳，越危险

对话检查时过这 5 条：

1. 去掉人名，还能听出来是谁在说话吗
2. 这句是在推事，还是只是在解释
3. 这句说完，关系有没有变化
4. 这句要是换成沉默，会不会更有力
5. 这几个人是不是都在用同一种体面话

### 题材局部件怎么用

古言、权谋、玄幻这些题材，代称、称谓、官职、店铺名、法器名、功法名这些东西，都是局部件，不是故事本身。

用法只有两条：

1. 先有冲突和关系，再上题材件
2. 题材件必须参与推进，不能只负责装饰

例如：

- 古言称谓不是越多越对，而是看尊卑、亲疏、翻脸时怎么变
- 官职不是摆百科，是用来决定谁能压谁、谁敢不敢当众顶回去
- 店铺名、宅院名不是摆设，要能挂住会面、做局、掉马、切割
- 法器、阵法、功法不是名词堆，要和输赢手段、代价、身份挂上

写题材局部件时，先问三句：

1. 这个东西不写，剧情会不会照样成立
2. 这个东西有没有把关系、身份或危险抬高
3. 这个东西是不是只在这里出现一次，后面再也不管

如果答案分别是“会”“没有”“是”，那它大概率只是装饰件。

### 转场、伏笔、悬念怎么落地

转场不是换地方，是把上一场剩下的劲带进下一场。

最稳的 4 种接法：

- 上一句留下没说完的话，下一场接结果
- 上一场压住没爆，下一场换场合再爆
- 用物件、称呼、动作延续前一场的情绪
- 先给结果异样，再回补发生了什么

伏笔和悬念别分开想，短篇里它们经常是一套动作：

- 先给一个当下解释不全的小异样
- 中段让它再出现一次，意思变重
- 高潮让它改写前文理解

写的时候别只记“埋了什么”，还要记：

- 第一次出现时，读者会怎么理解
- 第二次出现时，读者会不会开始不安
- 最后回收时，前两次是不是都值了

如果一个悬念只是拖着不说，但没有越拖越重，那不叫勾人，叫拖字数。

### 追妻后效应怎么写

追妻文最值钱的，不是男主跪得多惨，而是女主已经变了。

最稳的顺序：

1. 先真受伤
2. 再慢慢心冷
3. 再开始抽身
4. 再让男主后知后觉
5. 最后让读者明白，已经追不回了

最值钱的三种痛点：

- 她最需要他的时候，他站错了队
- 她已经凉了，他才看见自己失去什么
- 他终于肯爱了，但她已经不想要了

如果最后最大的爽点还是“他终于说爱她”，通常还不够。

### 前 60 字硬闸

前 60 字不是写漂亮，是最少完成下面 3 件事里的 2-3 件：

1. 立关系
2. 出冲突
3. 给结果预期

最好还能顺手给出：

4. 主角正在吃什么亏

最稳的 5 种起手：

- `关系 + 背刺`
- `结果先行`
- `场景起爆`
- `信息差起爆`
- `反直觉起爆`

前 60 字不要出现：

- 空背景
- 纯天气/环境
- 纯回忆
- 纯总结关系
- 没有后果的委屈
- 纯回忆
- 没代价的设定介绍
- 只有情绪，没有事

写完后只问四句：

1. 读者知道这两个人什么关系了吗
2. 读者知道现在出了什么事了吗
3. 读者会想看后面怎么收场吗
4. 这 60 字删掉一半后，信息还成立吗

### 情绪浓度怎么判断

按原文的口径，可以粗分三档：

- `一般情绪`：失望、意外、无奈、心累、遗憾
- `中等情绪`：痛苦、羞辱、无助、绝望、愤怒、心寒、死心
- `强烈触点`：践踏尊严、栽赃嫁祸、谋财害命、精神控制、众叛亲离、致残害命

判断时别只问“她难不难受”，要问：

1. 这件事只是让她不舒服，还是已经伤到尊严、关系、命运
2. 付费点前，情绪有没有从一般推到中等，最好再碰一次强烈触点
3. 后文是不是又退回普通拌嘴和普通委屈了

如果导语和开头用了很炸的强烈触点，后面正文却只剩普通争吵，读者会立刻觉得塌。

### 信息密度怎么查

很多“看着没问题但就是平”，不是句子不顺，是信息密度不够。

什么叫不够：

- 连续几段没有新变化
- 情绪在重复
- 关系没有偏移
- 冲突没有升级

每 300-500 字，至少要出现一种变化：

- 新人物入场
- 新证据出现
- 新误会生成
- 新危险逼近
- 主角立场变化
- 对方态度升级
- 关系重新定义
- 旧账翻出
- 场景切换到更危险的位置

最容易把文写平的几种情况：

- 连续解释
- 连续重复委屈
- 只吵不变
- 爆点来得太晚

逐段检查时，每段后面都问：

1. 这一段推进了什么
2. 删掉这一段，后文还成立吗
3. 这一段是新信息，还是旧信息换个说法

如果答案是“没推进 / 删掉也没事 / 只是换说法”，这段大概率该删、该并、该换。

### 题材怎么长期扩，不是一次性空想

如果手里已经有一批同题材资料，最稳的扩题顺序不是直接想新梗，而是：

1. 先拆结构
2. 再按题材归类
3. 再把同题材变体聚到一起
4. 再从这些变体里反推新题

比如追妻，不要只记标题，要记成循环：

1. 开头先给关系裂口
2. 中段加深误判
3. 女主转入抽离
4. 男主后知后觉
5. 结尾完成迟来惩罚

这样你不是在攒素材堆，而是在攒会自己长题的题库。

### 通用素材库怎么用

素材库不是为了堆满，是为了在起盘时少空想。

具体条目底库见 `short-story-material-bank.md`。这份负责：

- 给导火索具体清单
- 给关系场具体清单
- 给切割动作、第二层冲突、公开场、硬牌具体清单
- 给题面词、导语口气、结尾落点具体清单

这里继续只管“怎么抽、怎么排顺序、怎么防止乱拼”。

最常用的 7 类承重件：

1. `导火索`
   比如白月光回国、离婚当天翻脸、确诊后被逼懂事、奖项被临时换人
2. `关系场`
   比如原配 vs 白月光、长女 vs 偏心父母、元老员工 vs 空降新人
3. `切割动作`
   比如搬家、提离婚、退群、断供、公开划线、当场辞职
4. `中段再加的那层事`
   比如财产线、病情线、旧案线、身份误判、家族利益线
5. `高潮公开场`
   比如年夜饭、婚礼、满月酒、公司大会、病房外、直播间
6. `关键证据`
   比如录音、流水、房本、DNA、体检单、审计结果、聊天记录
7. `结尾落点`
   比如追不回、位置变化、价值抬高、平静新生活、迟到惩罚

调用顺序永远是：

- 先导火索
- 再关系场
- 再中段那层事
- 再切割动作
- 再高潮场和关键证据
- 最后才定结尾落点

别反过来先想“我要一个很炸的结尾”，那样中间大多会空。

可以直接用这张素材卡：

```text
导火索：
关系场：
切割动作：
第二层冲突：
公开场：
硬牌：
主情绪：
题面关键词：
导语口气：
结尾落点：
```

最稳的用法不是每栏都塞很多，而是：

- 一个导火索
- 一个关系场
- 一个切割动作
- 一条第二层冲突
- 一个公开场
- 一张硬牌
- 一条主情绪
- 一个结尾落点

按这个顺序抽，出来的题更稳。

### 表演层素材怎么用

素材型资料最有用的，不是替你定结构，而是补“人味”和“画面”。

最该拿来补的 6 类东西：

1. `微动作`
   比如手停一下、指节收紧、拽住袖口、把杯子放回去
2. `眉眼细节`
   比如眼神发直、眼尾压住、嘴角绷紧、笑意浮一下又没了
3. `说话动态`
   比如话说到一半停住、尾音压低、先笑后顶回去
4. `情绪躯体化`
   比如发堵、发冷、发麻、呼吸滞住、肩背发僵
5. `身份气质词`
   比如清冷、慵懒、温润、孤傲，不直接照抄，要拆成动作和反应
6. `题材局部件`
   比如古言代称、权谋话术、甜梗、接吻场景、吃醋反应

硬规则：

- 这些素材只负责补表现层，不负责替代结构
- 不要一段里堆三个以上漂亮描写件，够用就停
- 优先拿来区分人物，不要拿来把所有人都写成一个腔调
- 如果一处描写不能推动情绪、身份或关系变化，就删

最稳的用法：

- 先把结构和场景功能定住
- 再给每个核心人物补 1-2 个稳定微动作
- 再给关键场景补 1 个眉眼细节、1 个说话动态、1 个身体反应

这样补出来的“人味”比直接加一堆形容词稳得多。

### 素材型资料和结构型资料别混用

后面调用资料时，先分清楚它们负责哪一层：

- `结构型`
  负责：故事怎么走、事件链怎么推、中段怎么防平、高潮怎么回收
- `方法型`
  负责：开头钩子、信息密度、反直觉、情绪递进、自检口径
- `素材型`
  负责：微动作、眉眼、说话状态、题材局部件、剧情点库、替换词

错用的典型结果：

- 拿素材型资料硬拼结构，最后只剩词很多、事很空
- 拿结构型资料硬学句子，最后句子不稳、人也不像活人

所以调用前先问：

- 我现在缺的是骨架、检查口径，还是表演层

### 编辑视角回看

完稿后，别只从“我觉得还行”看一遍，再从编辑视角过一遍：

1. 题材是不是主流短篇赛道
2. 开头是不是直，前两节有没有明显冲突
3. 卖点能不能一句话说明白
4. 创新点是不是一眼能看见
5. 爽点是不是集中，高潮是不是值钱
6. 反击是不是比受气更让人记住

如果想冲更高价，不要只问“顺不顺”，要问现在最缺哪一层：

- 更新的题面
- 更狠的冲突
- 更强的升级器
- 更清楚的创新点

过稿前最后一问：

- 如果把这篇文放进编辑邮箱标题里，它最值钱的那一点到底是什么

编辑常见退回理由，别到最后才猜，直接反推：

- 觉得“套路化”
  先查有没有第二层冲突、有没有副升级器、有没有二次翻刀
- 觉得“张力不足”
  先查开头有没有后果预期、中段有没有持续升级、高潮前有没有卡点
- 觉得“文笔一般”
  先查是不是解释太多、事件太少、人物说话不分

### 写后自查顺序

正文写完后，不要第一时间埋头修句子。

先区分两层：

- **脚本预扫**：格式、段长、重复频率、固定词句、显式模式和风险块。
- **人工语义复核**：作者代判、叙述者/作者边界、多余解释、现场依据、对白是否过度高效。

脚本报“零命中”不能替代人工语义复核。尤其是“他以为”“像终于找到一种解释”“显然”“其实”等句子，必须结合上下文判断，不能只靠关键词定罪或放行。

固定先按这个顺序查：

1. 开头是不是抓人
2. 事件是不是够密
3. 情绪是不是在走，不是在重复
4. 人物是不是站住了
5. 节尾是不是挂住了后面
6. 结尾是不是兑现了，还留下了值钱的后劲

如果这里只查句子顺不顺，通常会越修越偏。

写后自查时，再补一个固定问法：

1. 这一篇最值钱的不公平到底是哪一下
2. 中段有没有真的把这一下越推越重
3. 高潮是不是用最该公开的场把它炸开
4. 炸完以后，有没有让前文一起更值钱

如果这 4 条里有 2 条答不上来，不要先修句子，先回大纲。

正文最终修改后，按 [../governance/post-write-human-review-gate.md](../governance/post-write-human-review-gate.md) 生成人工回执。局部或专项回炉必须传母稿，逐条检查所有新增/改写句，并复扫全文旧句。

进入人工回执前，先按 [../governance/rule-execution-ledger.md](../governance/rule-execution-ledger.md) 绑定最终设定、大纲、正文，逐项复核 skill 规则和拆书规则的执行证据，并通过 `rule_execution_gate`。

### 回炉顺序

发现问题后，也不要乱修。

先判断它属于哪一层：

- 骨架问题
- 开头问题
- 中段问题
- 高潮问题
- 情绪问题
- 句子问题

回炉时固定顺序是：

1. 先看 `high_risk_segments`，定这轮只修哪几个片段
2. 再看 `paragraph_scores`，定每个片段先动哪几个局部段落
3. 再修骨架
4. 再修开头
5. 再修中段升级
6. 再修高潮回收
7. 再修情绪走法
8. 最后才修句子和表演层

顺序反了，最容易出现“句子修了很多遍，根子没动”。

回炉时的判断口径再压成一句：

- `先看是不是没事，再看是不是不狠，再看是不是不像人，最后才看顺不顺`

分别对应：

- 没事：开头不立事、中段没升级、高潮没公开场
- 不狠：不公平不够刺、后果不够大、对方没掉位
- 不像人：人物口气混、动作空、解释句太多
- 不顺：句子拧、节奏散、段落太齐

局部回炉时再补一句：

- `先定片段，再定段落，再定改法；不要一看到问题列表就整篇乱修`

新增固定口径：

- 读取 `rulebook_audit / rulebook_impact_items`
- 命中 `开头反假`
  - 先拆 `先讲道理 / 卖点串烧 / 抽象定性 / 漂亮收刀`
- 命中 `信息漏出顺序`
  - 先拆 `事故关系前史结论一口气到齐 / 物证一出立刻定性 / 前史解释压过当前动作`
- 命中 `后果链`
  - 先拆 `手续流 / 安顿流 / 场场完整回击`
- 命中 `人物偏手`
  - 先拆 `男主过早解释 / 女主完整情绪宣言`
- 命中 `失控说话`
  - 先拆 `对白句句正答 / 追妻期成熟完整忏悔`
- 命中 `烂关系漏出`
  - 先拆 `关系只靠台词解释 / 物件像证据桌`
- 命中 `外部分块审计影响项映射`
  - 先拆 `作者替角色下结论`
- 命中 `作者站位过高`
  - 先拆 `角色当下不可能想这么明白 / 台词更像写给读者听 / 段尾总在点题`
- 这些项的优先级高于纯句面润色

补一条硬口径：

- `../governance/虚词模板词典.json` 不直接进自动批量替换
- 只可作为人工改写时的松句、钝化、补废气参考
- 原因见 `myconfig-rule-integration.md`

如果要把这一轮跑成标准化产物，默认直接用：

```bash
python3 "$CODEX_HOME/skills/story-short-write/scripts/run_revision_cycle.py" \
  当前短篇目录 \
  --internal-standard profiles/internal_audit_standard.json
```

它会把：

- `审计`
- `任务单`
- `任务单自检`
- `受限重写 gate 执行单`
- `gate 回执 JSON`

放进同一轮目录，避免手工散着跑。

不传 `--output-root` 时，默认使用 `当前书目录/数据/审计循环`。
同一篇正文连续回炉时，默认复用同一个 `output-root`。脚本会自动找上一轮 `cycle_summary.json` 对应的 `full_audit.json` 做本轮对比，不用每次手填上一轮路径。

注意：

- 这里的“循环”不是让审计脚本对同一份没改过的正文反复空跑
- 正确闭环是：`审计 -> 看桥段和大块 -> 改正文 -> 再审计`
- 桥段没对齐前，不要先为了总分去修短段、节奏或局部漂亮句

如果目录里有 `profiles/`，脚本还会自动找最新的规则包。也就是说，标准目录下默认可以直接在书目录级别启动，不必每次手写 `正文.md` 和 profile 路径。
如果当前书目录还没有规则包，但已经有 `01_主骨架与融合方案.md`，脚本会先从里面提取主骨架/辅桥来源，再去 `拆文库/{书名}/book.profile.json` 自动合成一本当前书专属 `profiles/{当前书名}.project.profile.json`，然后本轮直接使用。

如果目录里暂时还没有标准 `正文.md`，脚本也会兼容旧版版本稿命名，自动选最新的“含正文字样”的 `.md` 文件。这样旧项目不需要先重排文件名，也能先接上回炉闭环。

如果已经出了 `model_rewrite_task.json / .md`，默认再验两句：

- `桥段任务前排` 是否覆盖了 `segment_focus` 里真正命中的桥段
- `paragraph_focus` 前排里是否混进了只有 `短段对白密` 的段落
- `display_block_focus` 前排是否真的覆盖了 `内部最高块风险分` 对应的正文块

这两句对应任务单里的 `task_validation`。只要有一项不通过，先修任务单，不直接回修正文。

如果已经出了 `rewrite_gate_task.md / failure_gate_task.md`，还要再验：

- `rewrite_gate_receipt.json` 是否已经从 `pending` 改成真实执行结果
- `failure_gate_receipt.json` 是否已经写明 `passed / failed`

只生成执行单、不回填回执，视为这一轮 gate 没真正执行完。

如果这个题材还没有做过校准，先补这一步，再进入日常回炉：

```bash
python3 "$CODEX_HOME/skills/story-short-write/scripts/compare_with_external_block_audit.py" \
  block_audit_test/题材目录 \
  --audit-dir block_audit_test/题材目录/_score_align \
  --output profiles/external_block_audit_alignment.csv \
  --summary-output profiles/external_block_audit_alignment_summary.json \
  --internal-standard-output profiles/internal_audit_standard.json
```

后续就固定使用同一份：

- `profiles/internal_audit_standard.json`

来驱动：

- `run_full_ai_audit.py`
- `auto_revise_ai_flavor.py`
- `run_revision_cycle.py`

日常判断口径固定为：

- 先看 `当前判定`
- 再看 `内部最高块风险分`
- 再看 `正文块风险排行`
- 最后才看全文重审计分

这里要额外区分两层：

- `直接计分层`：会直接推高 `heavy_summary.score`
- `结构不过层`：不一定直接推高分数，但会让桥段、正文块、片段长期不过

不要把这两层混成一个“总分问题”。
详细拆解见：

- [../heavy-audit-breakdown.md](../heavy-audit-breakdown.md)

### 短篇回炉自检单

回炉前至少问这几句：

- 前 60 字里有没有关系、冲突、后果预期
- 第一屏里有没有真正的事，而不只是气氛
- 每 300-500 字有没有新变化
- 有没有连续两段都在解释
- 中段有没有明确升级器
- 这篇文有没有至少一个反直觉点
- 读者的第一预判有没有被打偏一次
- 情绪是在递进，还是在重复
- 高潮是不是只靠骂
- 高潮有没有回收前文铺垫
- 高潮后的代价够不够痛
- 这篇文最值钱的一点是什么
- 如果删掉设定壳子，核心结构还成不成立
- 如果编辑说“套路化”，最可能卡在哪一段
- 如果编辑说“张力不足”，最可能是哪个升级器没补上
- 自动脚本抓到了什么；哪些结论脚本根本无权判断
- 每个新增评价句有没有现场可观察依据
- 每个新增评价句是在插嘴，还是在替人物或读者总结
- 动作和对话已经说明后，是否又补了一句解释
- 是否只审了本轮新增句，漏掉母稿旧代判句

### 写后高风险信号

如果写完出现下面这些，不要先修句子，先回结构层：

- 开头平
- 中段塌
- 人物口气混成一个腔
- 事件不升级
- 情绪只有词，没有体感
- 高潮只是知道真相，没有炸场
- 结尾只是结束，没有落点
- 全文只是句子顺，没有最刺人的那一下

### 正文禁区

正式写正文时，下面这些一旦多起来，通常就会发平：

- 一整段都在解释背景
- 一整段都在回忆，但不改当前局面
- 只有人物感受，没有现实动作
- 对话只是把刚发生的事再说一遍
- 同类羞辱重复两三次，但没有更坏后果
- 主角一直挨打，没有一次主动动作

判断办法很简单：

- 删掉这一段，后面的局会不会照样成立

如果会，这段大概率就该压缩或重写。

### 情绪要落到眼神、身体、说话、动作

别只写“她难受”“他生气”“她绝望”。

每场都尽量补到这四层里至少两层：

- 眼神和脸上：盯、躲、发直、笑僵、嘴角绷住
- 身体反应：手停住、背发僵、呼吸乱、肩塌下去、站位后退
- 说话变化：更短、更硬、更轻，或者干脆接不上
- 动作选择：放下、推开、收回、绕开、当没听见、直接走

这样人物才不是一团说明书。

### 素材池怎么调，不要乱拼

素材池不是看见好的都塞进去，而是只调这些真正撑事的部分：

- 一个最显眼的矛盾
- 一层让事情变得更麻烦的新问题
- 一个核心关系场
- 一条主事件链
- 一个高潮发生的场合
- 一个最致命的证据
- 一个结尾落点

如果一篇里同时出现 2 个最显眼的矛盾、2 条事件链、3 个同等级的大证据，通常就是过满了。

### 素材库怎么接进来

正确顺序：

1. 先选起因
2. 再选人物关系
3. 再选中段再加的那层事
4. 再选切割动作
5. 再选高潮场合
6. 再选关键证据
7. 最后选结尾落点

抽素材时再补两条：

1. 前 5 步选的是骨架件，必须先定
2. 口气、微动作、题材替换词这些都属于后补件，不能抢在骨架前面

最稳的调用顺序就是：

- 先让故事站住
- 再让人物像活人
- 最后再让它更有味

示例一：

- 起因：离婚当天翻脸
- 关系：前妻 vs 前夫 + 前任
- 再加的事：财产转移
- 切割动作：搬家、切共同账户
- 高潮场合：家宴
- 关键证据：流水、房本、录音
- 结尾落点：她走了，他知道晚了

示例二：

- 起因：确诊后被逼懂事
- 关系：长女 vs 偏心父母
- 再加的事：让她签字让房
- 切割动作：断供、退群、公开划线
- 高潮场合：年夜饭
- 关键证据：转账记录、房产安排
- 结尾落点：她断干净，他们才知道失去的是什么

### 追妻文怎么把男主那条线写顺

别写成“前面虐女主，后面男主突然醒悟”。

更稳的写法是：

1. 先有旧情，不然恨和悔都轻
2. 再有误会或旧伤，把两人砸开
3. 再让他一次次选错边、信错人、说错话
4. 中间给他看清一点点的机会，但每次都错过去
5. 女主不是一下子死心，是被一层层磨冷
6. 真相到了以后，他也别立刻全懂，先继续认错重点
7. 最后让他知道问题不是“补多少”，而是她已经走了

这么写，追妻才不是流程表，更像真的迟到。

### 防套路起盘硬闸

起盘完成后，正式写前，必须单独回答这 6 个问题：

1. 这篇表面上是什么题材？
2. 这篇最特别的地方是什么？
3. 中段又加了什么更麻烦的事？
4. 主角哪一步是 `非标反应`？
5. 哪个高潮会让人回头再看前文？
6. 结尾除了处刑，还有没有 `位置变化 / 价值抬高 / 后面还让人难受一下`？

这 6 个里缺 2 个以上，不要落正文，先回大纲。

### 素材调用口径

项目里如果已经拆出了方法和素材，优先这样调：

- 想补结构，先看 `故事怎么走 / 事件链 / 中段再加什么事`
- 想补开头，先看 `开头钩子 / 导语起手 / 第一屏先给了什么`
- 想补情绪，先看 `情绪节点 / 处境给读者的感觉 / 爽后还留了什么感觉`
- 想补新鲜感，先看 `真正特别的地方 / 反套路手段 / 非标反应`
- 想补中段不平，先看 `让事情更麻烦的那层事 / 当众摊开的场面 / 关键证据 / 高潮后有没有再往上抬一下`

别反过来。

### 正式写前，再把必要参考补一遍（同时别超过 3 个）

- 必读：`format-and-structure.md`
- 必读：`anti-ai-writing.md`（写作时自检 AI 腔，**先查篇章级骨架味**）
- 必读：`craft/narrator-voice.md`（第一人称叙述者有没有"嘴"——实时评价、骤断短句、幽默/自嘲；去AI味核心，回炉时必查）
- 硬闸：三份必读文件必须逐项回填 `写作规则读取回执.json` 并通过 `validate_writing_rule_gate.py`；任一文件变化后必须重新读取，不能沿用旧回执。
- 硬闸：读取通过后必须初始化 `规则执行台账.json`。所有拆书文件逐文件判断，16 表和承重资产逐规则展开；额外挂载的参考规则用 `--skill-rule-file` 加入台账。
- 分类硬闸：脚本先按小节和资产文件形成规则族；当前写作模型再逐族分流流程、格式、设定、大纲、正文、审计、拆书候选和禁用规则。完全重复族自动合并，近义族由模型归入 canonical。台账全量执行不等于全量修改正文。
- 脚本项只做 SHA、格式、频率、禁词、固定模式和完整性；人物偏手、失控说话、注意力漂移、认知局限、作者代判和对白生活性必须人工执行；长窗节奏、对白效率、桥段相似度和 profile 覆盖必须混合执行。
- 按题材加载 1 个：`genre-writing-formulas.md` 中对应题材速查表
- 按需加载：`writing-craft.md` / `villain-and-reveal.md` / `emotional-methods.md`

### 工作目录结构

```
项目根/
├── 拆文库/          # story-short-analyze 默认产出（source of truth）
│   └── {参考小说}/
│       ├── 拆文报告.md
│       ├── 情节节点.md
│       └── 写作手法.md
└── {标题}/
    ├── 设定.md      # 核心框架 + 人设 + 反转铺垫 + 结构物件（含三现追踪表）+ 反派设计 + 对标摘要
    ├── 小节大纲.md  # 小节大纲：默认用 Markdown 可读格式；字段固定为主事件/子事件×3-5/情绪/读者新获知什么/钩子/伏笔或物件/动静/对话密度/目标字数
    ├── 正文.md      # 完整正文
    └── 对标/        # 当前短篇引用视图（可选，优先于项目根拆文库）
        └── {参考小说}/
            ├── 拆文报告.md
            ├── 情节节点.md
            └── 写作手法.md
```

### 基本原则

- 正文直接写进文件，不要只停在对话里
- 精修时先读文件再改，不要靠记忆硬修
- 结构物件的修改，直接体现在 `设定.md` 的追踪表里
- 设定.md 维护结构物件追踪表：
  `| 物件 | 第1现(节号/含义) | 第2现(节号/含义) | 第3现(节号/含义) |`
- `设定.md` 里最好单列一个“对白口气模板”区，至少给主角、压迫者、旁观者各记 3 条
- 只要感觉这篇“有点顺，但不够狠”，先回大纲查是不是最显眼的矛盾太常规、高潮场面不够大、对方摔得还不够狠

## Phase 4 精修打磨

### 精修检查清单

```
## 精修清单

### 开头
- [ ] 前 3 句话能抓住人？
- [ ] 没有无意义的背景介绍？
- [ ] 包含一个开篇钩子？

### 情绪
- [ ] 情绪曲线是否有明确走向？
- [ ] 反转前的情绪铺垫够不够？
- [ ] 反转后的冲击够不够？
- [ ] 读者此刻最想看的结果是否一直被吊着？

### 反转
- [ ] 反转是否出乎意料？
- [ ] 反转是否合情合理（回看有铺垫）？
- [ ] 反转时机是否合适？
- [ ] 反转后有没有让前文重新变得更狠，而不是只多知道一个信息？

### 节奏
- [ ] 有没有拖沓的部分？
- [ ] 每句话是否都有存在价值？
- [ ] 字数是否在目标范围内？
- [ ] 有没有连续几节都只是在重复受气，没有位置变化？

### 格式
- [ ] 段落不超 60 字？无大段落？
- [ ] 是否避免连续大量等长短句密排？
- [ ] 段间无空行？
- [ ] 对话独立成行、用正确引号？
- [ ] 无「他说/她道」？
- [ ] 如果要导出知乎盐选投稿版：是否已改成数字节位，且对话默认改为 `「」`？如平台后台明确要求 `【】`，是否已按要求覆盖？

### 钩子
- [ ] 每 2-3 小节有钩子？
- [ ] 章末有钩子（悬念或余韵）？

### AI 腔
- [ ] 无禁止比喻：「命运的齿轮」「如潮水般」「仿佛春风」「心猛地一沉」「眼眶泛红」
- [ ] 同一身体部位/情绪描写不超 5 次（统计：手指/心/膝盖/眼睛，超限必须替换）
- [ ] 「像」使用频率：全文不超 10 处（含「像是」「不像」）
- [ ] 没有连续多段都写成「动作一句 + 心理一句 + 台词一句」的模板节拍
- [ ] 没有“作者替人物说透了”的代判句
- [ ] “作者站位过高 0”是否仅被当成脚本结果，而不是人工通过结论？
- [ ] 是否已逐句引用正文完成人工语义复核？
- [ ] `规则执行台账.json` 中每条适用规则是否已完成，脚本/人工/混合证据是否齐全？
- [ ] 是否已确认每条规则的角色和修复目标，并确保只有失败的适用正文约束进入正文修改单？
- [ ] 重复规则是否已合并到有效 canonical，拆书候选未采用和禁用规则未命中是否分别留有理由/范围复核？
- [ ] 每个拆书文件是否已逐项判断，跳过项是否写了具体理由？
- [ ] 最终设定、大纲、正文 SHA 是否已绑定，正文修改后是否重新核验证据？
- [ ] 没有“词很新鲜但不像人会说”的生造口气句
- [ ] 完整 AI 腔黑名单见 `anti-ai-writing.md`

### 技法
- [ ] 身体细节：有无情绪词可替换为身体状态？
- [ ] 结构物件：三现是否完整？（查设定.md追踪表）
- [ ] 一动一静：每节是否有动有静？
- [ ] 开头密度：前 100 字事件 ≥ 3？
- [ ] 主要角色说话能不能不看人名也大概分出来？

### 结尾
- [ ] 结尾是否有余韵？
- [ ] 读者会不会想转发？
```

### 删减原则

1. 不推动剧情的对话 → 删
2. 不铺垫反转的描写 → 删
3. 不推高情绪的心理活动 → 删
4. 读者能猜到的内容 → 缩短
5. 重复表达的情绪 → 合并
6. 只是“像短篇套路里应该有”的桥段，但放进这篇里不增疼感、不增爽感、不增掉位 → 删

## 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 开头不抓人 | 在做背景铺垫 | 直接从冲突开始 |
| 中间拖沓 | 信息密度太低 | 删减或合并场景 |
| 反转没力度 | 铺垫不够或太明显 | 增加误导线索 |
| 结尾无力 | 反转后拖太长 | 反转后 500 字内收尾 |
| 全篇平淡 | 情绪曲线太平 | 加大情绪落差 |
| 感觉像流水账 | 只有经过，没有掉位和结果期待 | 补人多的场面、补后果、补位置变化 |
| 人物像一个人嘴里出来的 | 只借了情节，没借原文口气模板 | 回原文提炼称呼、句长、找补词、失控方式 |
