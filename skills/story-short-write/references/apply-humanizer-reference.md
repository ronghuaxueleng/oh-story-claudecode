# apply_humanizer 参考卡

这份不是让你直接运行 `scripts/apply_humanizer.py` 去改短篇正文。

这份只回答：

- 这个脚本里有哪些思路能借
- 哪些做法不能直接照搬到情绪短篇

---

## 一、先下结论

`apply_humanizer.py` 更适合做：

1. 系统腔降压
2. 说明句钝化
3. 规则怪谈 / 悬疑文本去公告味
4. 过硬句子的局部松开

不适合直接做：

1. 追妻火葬场正文自动改稿
2. 情绪短篇整篇自动人味化
3. 对白层自动优化
4. 桥段层自动重写

原因很简单：

- 它主要靠 `字面替换`
- 它处理的是 `句子壳`
- 它不会判断 `桥段承重件`
- 也不会判断 `人物偏手`
- 更不会判断 `同一桥为什么原文能过`

所以这类脚本只能当：

- `人工改写动作参考`

不能当：

- `正文自动改稿器`

---

## 二、里面真正可借的 5 类动作

### 1. 把过硬的规则句压回日常句

脚本里大量在做这件事：

- `不可` -> `不能`
- `必须` -> `都要`
- `当前` -> `现在 / 目前`
- `将被视为` -> `就是`
- `优先承担` -> `先承担`

可借的不是具体替换词，
而是这条判断：

`凡是像公告、规则、说明书的句子，都要先往“人会这么说吗”压一层。`

适合人工使用的场景：

- 规则句太硬
- 说明句太像面板
- 程序化表达太重

不适合的场景：

- 高情绪对峙句
- 关键桥段承重句
- 章尾刀句

---

### 2. 把精确情绪模板改钝

脚本里会做这种事：

- `背后一凉` -> `背后感到一阵凉`
- `心脏猛地一沉` -> `心里顿时感到很沉重`
- `瞳孔缩紧` -> `瞳孔一缩`

这能借的不是“统一替换”，
而是这个思路：

`某些太像模型默认情绪模板的句子，可以先钝一点，再决定要不要改成动作。`

正确人工顺序：

1. 先判断这句是不是模板情绪句
2. 如果是
3. 先决定删掉
4. 删不掉再钝化
5. 最稳的还是改成动作或现场阻滞

所以：

- `钝化` 只是过渡手段
- 不是最终改法

---

### 3. 去模板词，而不是去所有口语词

脚本会压这些：

- `眼中闪过`
- `嘴角勾起`
- `心中暗道`
- `身子一颤`
- `心头一紧`

这条可以直接借。

因为这些确实很容易长成模型默认壳。

但要注意：

`去模板词` 不等于 `全文都改平`

更稳的人工改法是：

- `眼中闪过` -> 改成视线、停顿、看向谁
- `嘴角勾起` -> 改成笑了一下 / 没笑出来 / 扯了下嘴角
- `心头一紧` -> 改成手停住 / 没接话 / 把东西翻过去又翻回来

---

### 4. 打断太平滑的纯叙述段

脚本里有个思路是：

- 如果一整段太顺、太长、太纯叙述
- 就拆开

这条能借，
但只能当末位节奏修整。

人工使用时要注意：

- 先看桥段和场戏问题
- 再看这段是不是太平
- 最后才考虑打断

因为：

`平滑段落` 只是表层问题

不能拿它去替代：

- 桥段重排
- 对白控场
- 烂关系漏出

---

### 5. 给说明句加一点“废气”

脚本里本质上在做：

- `发生了` -> `就发生了`
- `他看向` -> `他先朝`
- `围观的人反应过来` -> `围观的人这时候才反应过来`

这个思路可以借，
但只能少量人工使用。

它解决的是：

- 句子太直
- 句子太说明书
- 句子太干

它不能解决：

- 桥段假
- 人物假
- 情绪假

所以正确用法是：

- 已经确定这句功能对
- 只是太硬
- 才补一点废气

不是：

- 全文批量加 `就 / 才 / 先 / 还`

---

## 三、里面不能直接照搬的 5 类做法

### 1. 批量连词替换

脚本里会统计和替换连接词。

这个不适合直接拿来改短篇正文。

原因：

- 很容易统一腔调
- 很容易把文本改油
- 很容易改成另一种模板味

---

### 2. 批量 narrative particles 注入

也就是：

- 自动补 `就 / 才 / 先 / 还 / 已经`

这个最危险。

原因：

- 少量人工点缀有用
- 批量注入会非常假

它最多只能当：

- 人工参考库

不能当：

- 自动改写规则

---

### 3. 基于对齐样本学习字面替换

脚本支持：

- before / after 学习替换对

这条对实验文本有用，
但不适合直接迁移到短篇写作。

原因：

- 会把某一份稿子的局部语言壳硬继承下来
- 容易把项目特定脏数据学进去
- 对桥段和人物没有判断力

---

### 4. 把“钝”误当“人味”

脚本里很多改法都在做：

- 去精致
- 去利索
- 去压缩

这能降一部分 AI 感，
但不等于有人味。

人味真正更强的层还是：

1. 人物偏手
2. 烂关系漏出
3. 同桥承重件
4. 场戏功能排序

---

### 5. 自动改对白

这个脚本不该碰对白主层。

因为对白真假主要取决于：

- 人是不是会这样说
- 这时候是不是只会说半句
- 他是在控场还是在解释

这些都不是字面替换能解决的。

---

## 四、对 `story-short-write` 的实际使用口径

当前 skill 对 `apply_humanizer.py` 的吸收方式应该固定为：

1. 只吸收 `思路`
2. 不直接调用脚本改正文
3. 不把它接成自动批量替换链

可吸收的思路：

- 去规则腔
- 去说明书腔
- 去模板情绪词
- 把过硬句钝一层
- 给少数说明句补一点废气

不可直接自动化的：

- 虚词批量注入
- 连接词批量注入
- 对齐样本替换
- 全文自动松句

---

## 五、最短实操卡

当你人工改一段时，如果发现：

- 句子功能是对的
- 但太硬、太直、太说明

可以按这个顺序试：

1. 先删模板情绪词
2. 再把规则腔压成日常句
3. 再决定要不要补一个 `先 / 就 / 才 / 还`
4. 如果还能改成动作，优先改动作

永远不要反过来：

1. 先批量补虚词
2. 再希望它自然有人味

这条基本一定会翻车。
