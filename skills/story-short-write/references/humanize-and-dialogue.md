# 人话、对白与导语规则

适用时机：

- 写对白前
- 写开头导语前
- 初稿去生硬感
- 回炉时排查 AI 味

---

## 一、导语与题面

### 题面和导语怎么起

参考原文时，题面和导语不要只追求“信息完整”，要先追求“读者会不会顺手点开”。

- 题面优先写最扎人的那一下，不优先写背景
- 能用关系错位，就别只写事件名词
- 能把羞辱、背叛、误认、让位、迟到后悔这些感觉压进去，就别写成平铺简介
- 题面最好自带结果期待，让人一眼知道后面要看谁翻、谁倒霉、谁后悔

常用起法：

- `意外发现`：我以为怎样，结果发现根本不是那样
- `强对比`：我在低处受气，他在高处得意
- `三人修罗场`：我、旧人、新人同框，当场失衡
- `开场即离`：签字、离席、退婚、断联、搬走、过户当天出事

导语别写成剧情摘要，最好只干三件事：

1. 先交代最扎人的关系错位
2. 再亮出最想看的后果
3. 最后留一个“事情还会更大”的口

### 导语硬闸

前 20 / 60 / 80 字至少检查这三件事：

- 前 20 字：有没有起事
- 前 60 字：有没有完成 `关系错位 / 事故信息 / 后果预期` 里的至少 2 项
- 前 80 字：有没有第二推进点

开头禁区：

- 还在解释背景
- 只有气氛，没有事
- 只有委屈，没有后果
- 纯回忆、纯总结关系

## 二、对白与人话

### 对白硬约束

只参考事件和结构不够，人物对白必须参考拆文库原文或拆文结果提炼出的“口气模板”来写。至少区分主角、压迫者、求助者、旁观/清醒者的嘴型差异。

禁止：

- 所有角色共用一套书面表达
- 每个人都只会“完整解释自己”
- 把称呼习惯、口头禅、推责方式、压人方式、求情方式写平

### 人话硬约束

对白不能只追求“有劲”“新鲜”“像金句”，还必须像真人会这么说。写每场对话前，先过这道口：

- 这句话如果不看文艺效果，只看口语，真人会不会这么说？
- 这句是在表达人物，还是在表达“作者想造一个新说法”？
- 如果把其中最怪的词换成常见口语，人物功能是否完全不变？

若答案是“换成 `插嘴 / 掺和 / 接话 / 打岔 / 顶话 / 顶雷 / 找罪受` 之类常见口语也完全成立”，优先回退到常用口语层，不要硬拧。

### 改句顺序

任何句子在改写前，先判它当前承担的句子功能，再决定怎么修。

最少先判断属于哪一类：

- `陈述`
- `反问`
- `嘲讽`
- `顶嘴`
- `下判词`
- `求情`
- `敷衍`

如果原句本质是 `反问 / 嘲讽 / 顶嘴`，修的时候禁止先把它修成顺滑的平陈句。

优先级永远是：

1. 先保住句子功能
2. 再修不通的词
3. 最后才修顺口度

例：

- 错误修法：把 `他都带着怀孕的前任上门了，我还得跟他说话客气？` 修成 `他带着怀孕的前任上门，我说话难听。`
- 正确修法：先保留 `我还得……？` 这种反问骨架，再替换别扭词

## 三、前置拦截

### 1. 非人话表达 / 生造口气句

初稿阶段就拦，不要等 review 再抓。命中表现：

- 动词搭配生硬，像临时拼出来的
- 句子字面能懂，但不像这个人物身份、年龄、脾气会当场这么说
- 句子主要价值来自“陌生说法”，不是来自人物情绪、立场、压迫感
- 改成常见口语后，人物功能和情绪完全不变

命中后优先判断根因：

- `角色口条失真`
- `作者造句感过重`
- `口气模板未本地化`

默认修法：

- 回退到该角色平时真会说的口语层
- 缩短句子
- 去掉为了出挑而硬加的怪搭配
- 用动作 / 停顿 / 打断补锋利，不靠生造词顶
- 若原句本身带 `反问 / 嘲讽 / 顶嘴` 功能，先保住这个功能，再回退口语层

### 2. 抽象词悬空 / 对象省略过度

常见命中：

- `那时候我脑子里全是喜欢。`
- `她心里只剩委屈。`
- `他最在意的就是体面。`
- `我忽然觉得难堪。`

检查：

- 这个抽象词的对象是谁？
- 这个感受挂在谁身上？
- 不把对象写出来，读者会不会停一下才懂？

默认修法：

- 给抽象词补对象
- 给情绪补来源
- 给判断补落点
- 不要为了短和狠，把真正承重的信息一刀省掉

### 3. 人物反应代判句

常见命中：

- `她大概没想到我会这么平`
- `他显然没料到我会这么说`
- `他这才意识到自己输了`
- `我看着他，只觉得空`

默认修法：

- 先补表情、动作、沉默、打断、接不上话
- 再看是否还需要保留主观判断
- 优先“让读者自己看出来”，不要“替读者说出来”

### 4. 感官逻辑断裂句

常见命中：

- `门一关，客厅里一下静下来，静得我耳朵都嗡。`
- `我看着屏幕，心里一下静了。`
- `天一黑，后背都发麻。`

默认修法：

- 要么只保留环境状态
- 要么拆成两拍
- 要么把后半句改成前半句能自然推出的感受

禁止把“安静”“发懵”“耳鸣”“心口发闷”这类不同感受硬捏成一句求新鲜。

### 5. 听觉描写解释过满

如果只是“听见了”某种声音，却直接把它翻译成角色体内的细感受，默认按 `解释过满` 先拦。

原则：

- 听见什么就先写什么
- 身体细感受若不是现场可证，不要替人物多判半层

### 6. 效果判词句 / 起效代判句

常见命中：

- `我知道，我这句话进去了。`
- `这句一出去，起效了。`
- `我知道，他听进去了。`
- `这一下算是扎到了。`

默认修法：

- 删掉 `进去了 / 起效了 / 听进去了 / 扎到了` 这类效果结论
- 改成可见反应
- 若确实要保留判断，也必须先有现场承接，且对象说清

### 7. 事实指向错位句 / 场景指代错误

常见命中：

- 明明当前是在第三方现场
- 句子却把场景说成了 `你的婚礼 / 你的场子 / 你家席面`

默认修法：

- 保住句子功能
- 先把事实指向改对
- 再考虑口气强弱

### 8. 风险等级落轻句 / 动作结果写虚句

常见命中：

- 真实风险是 `动手 / 伤人 / 砸人`
- 句子却只落成 `碰 / 弄 / 搞一下`

默认修法：

- 先判句子功能：`提醒 / 制止 / 防伤害 / 现场止损`
- 再把真实风险后果写实

## 四、写后统一回看

写后优先回看这几类高危：

- `非人话表达 / 生造口气句`
- `人物反应代判句`
- `抽象词悬空 / 对象省略过度`
- `断段过散`

原则：

- 去 AI 味不是把字句抹平，而是恢复“作者偏手”和“角色偏手”
- 不要把正文修得过于规整
- 要允许少量“真人顺手话”，但不能破坏事实和节奏
